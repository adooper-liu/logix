# 预警系统不显示问题诊断指南

## 🐛 问题现象

统计面板、Tooltip 详情、甘特图标识都没有显示。

---

## 🔍 可能原因分析

### 原因 1：数据中没有 `portOperations` 或 `lastFreeDate`

**检查方法**：

打开浏览器 Console，输入：

```javascript
// 查看加载的货柜数据
console.log('Containers:', containers)

// 检查第一个货柜的 portOperations
console.log('First container portOperations:', containers[0]?.portOperations)

// 检查是否有 lastFreeDate
const hasLastFreeDate = containers.some(c => c.portOperations?.some(op => op.lastFreeDate))
console.log('Has lastFreeDate:', hasLastFreeDate)
```

**预期结果**：

- ✅ `portOperations` 应该是一个数组
- ✅ 至少有一个货柜的 `portOperations` 包含 `lastFreeDate` 字段

**如果数据为空**：

- 后端没有返回 `portOperations` 数据
- 需要检查后端的 `enrichContainersList` 方法是否正确加载了港口操作数据

### 原因 3：预警逻辑判断条件不匹配

**当前预警规则**：

```typescript
// 即将超期（Warning）
const daysUntilDeadline = dayjs(lastFreeDate).diff(dayjs(), 'day')
return daysUntilDeadline >= 0 && daysUntilDeadline <= 3

// 逾期未提（Danger）
return (
  dayjs().isAfter(dayjs(lastFreeDate)) &&
  container.logisticsStatus?.toLowerCase() !== 'picked_up' &&
  container.logisticsStatus?.toLowerCase() !== 'unloaded' &&
  container.logisticsStatus?.toLowerCase() !== 'returned_empty'
)
```

**调试代码**：

在 `useGanttLogic.ts` 中添加：

```typescript
// 在 getContainerAlerts 函数中添加调试
const getContainerAlerts = (container: Container): AlertRule[] => {
  const alerts = alertRules.filter(rule => rule.condition(container))

  // 调试日志

    console.log('[Alert] Container has alerts:', {
      containerNumber: container.containerNumber,
      alerts: alerts.map(a => ({ name: a.name, level: a.level })),
    })
  }

  return alerts
}
```

## 🐞 常见错误示例

### 错误 1：字段名不匹配

```typescript
// ❌ 错误：使用了错误的字段路径
const lastFreeDate = container.lastFreeDate

// ✅ 正确：从 portOperations 中提取
const lastFreeDate = container.portOperations?.find(
  op => op.portType === 'destination'
)?.lastFreeDate
```

### 错误 2：日期格式问题

```typescript
// ❌ 可能失败：如果 lastFreeDate 不是有效日期
dayjs(lastFreeDate).diff(dayjs(), 'day')

// ✅ 更安全：先验证
if (!lastFreeDate || !dayjs(lastFreeDate).isValid()) return false
```

### 错误 3：状态码大小写

```typescript
// ❌ 可能失败：如果 status 是 'AT_PORT' 而不是 'at_port'
container.logisticsStatus !== 'picked_up'

// ✅ 安全做法
container.logisticsStatus?.toLowerCase() !== 'picked_up'
```
