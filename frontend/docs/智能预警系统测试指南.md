### 方法 1：使用现有数据

如果数据库中已有以下类型的货柜，可直接测试：

2. **逾期未提的货柜**（lastFreeDate 已过去且状态不是 picked_up/unloaded/returned_empty）
3. **正常的货柜**（作为对照）

INSERT INTO port_operations (

#### TypeScript 脚本（推荐）

创建一个临时脚本来生成测试数据：

```typescript
// scripts/create-test-alerts.ts
import { containerService } from '@/services/container'

async function createTestAlerts() {
  // 获取所有容器
  const response = await containerService.getContainers({
    page: 1,
    pageSize: 100,
  })

  const containers = response.items

  console.log('找到容器数量:', containers.length)

  // 筛选出可能触发预警的容器
  const warningContainers = containers.filter(c => {
    const lastFreeDate = c.portOperations?.find(op => op.lastFreeDate)?.lastFreeDate
    if (!lastFreeDate) return false

    const daysUntilDeadline = dayjs(lastFreeDate).diff(dayjs(), 'day')
    return daysUntilDeadline >= 0 && daysUntilDeadline <= 3
  })

  const dangerContainers = containers.filter(c => {
    const lastFreeDate = c.portOperations?.find(op => op.lastFreeDate)?.lastFreeDate
    if (!lastFreeDate) return false

    const status = c.logisticsStatus?.toLowerCase()
    const isPickedUp =
      status === 'picked_up' || status === 'unloaded' || status === 'returned_empty'

    return dayjs().isAfter(dayjs(lastFreeDate)) && !isPickedUp
  })

  console.log('\n===== 预警统计 =====')
  console.log('即将超期（Warning）:', warningContainers.length)
  console.log('逾期未提（Danger）:', dangerContainers.length)

  console.log('\n===== 即将超期货柜 =====')
  warningContainers.forEach(c => {
    const lastFreeDate = c.portOperations?.find(op => op.lastFreeDate)?.lastFreeDate
    const days = dayjs(lastFreeDate).diff(dayjs(), 'day')
    console.log(`${c.containerNumber} - 剩余${days}天 - ${c.destinationPort}`)
  })

  console.log('\n===== 逾期租柜 =====')
  dangerContainers.forEach(c => {
    const lastFreeDate = c.portOperations?.find(op => op.lastFreeDate)?.lastFreeDate
    const daysOverdue = dayjs().diff(dayjs(lastFreeDate), 'day')
    console.log(
      `${c.containerNumber} - 已逾期${daysOverdue}天 - ${c.logisticsStatus} - ${c.destinationPort}`
    )
  })
}

createTestAlerts()
```

## 🧪 测试场景

### 场景 1：即将超期货柜

**测试数据特征**：

- `lastFreeDate` = 今天 + 2 天
- `logisticsStatus` = `at_port`

**预期结果**：

- ✅ 橙色边框（2px）
- ✅ 温和的脉冲动画
- ✅ Tooltip 显示"⚠️ 距离最晚提柜不足 3 天"
- ✅ 统计面板"即将超期"计数 +1

### 场景 2：逾期未提租柜

**测试数据特征**：

- `lastFreeDate` = 今天 - 5 天
- `logisticsStatus` = `at_port`（不是 picked_up）

**预期结果**：

- ✅ 红色边框（3px）
- ✅ 强烈的脉冲动画
- ✅ Tooltip 显示"⚠️ 已超过最晚提柜日期"
- ✅ 统计面板"逾期未提"计数 +1
- ✅ 日期显示红色加粗

### 场景 3：已提柜的超期货柜

**测试数据特征**：

- `lastFreeDate` = 今天 - 5 天
- `logisticsStatus` = `picked_up` 或 `unloaded` 或 `returned_empty`

**预期结果**：

- ❌ 不触发预警
- ❌ 没有彩色边框
- ❌ Tooltip 没有预警信息

### 场景 4：没有 lastFreeDate 的货柜

**测试数据特征**：

- `portOperations` 中没有 `lastFreeDate` 字段

**预期结果**：

- ❌ 不触发预警
- ❌ 没有彩色边框

### 场景 5：多个预警同时存在

**测试数据特征**：

- 同时满足多个预警条件（理论上可能）

**预期结果**：

- ✅ 显示所有预警信息
- ✅ 按最高级别设置边框颜色
- ✅ Tooltip 列出所有预警

## 🐛 常见问题排查

### 问题 1：看不到预警标识

**可能原因**：

1. 数据库中没有符合条件的货柜
2. 数据未正确加载
3. 代码未正确编译

**排查步骤**：

```bash
# 1. 检查控制台是否有错误
查看浏览器 Console

# 2. 检查数据是否加载
在 Console 输入：
window.__VUE_DEVTOOLS_GLOBAL_HOOK__

# 3. 手动检查数据
console.log('容器数据:', containers)
```

### 问题 2：统计面板数字为 0

**可能原因**：

- 确实没有预警货柜
- 筛选条件排除了预警货柜

**解决方法**：

1. 重置所有筛选条件
2. 检查数据范围是否正确
3. 确认 lastFreeDate 字段有值

### 问题 3：动画不流畅

**可能原因**：

- 浏览器性能问题
- CSS 动画兼容性问题

**解决方法**：

1. 尝试刷新页面
2. 检查浏览器是否是最新版本
3. 尝试其他浏览器

### 问题 4：Tooltip 不显示预警信息

**可能原因**：

- 预警判断逻辑有误
- 数据格式不正确

**排查方法**：

```javascript
// 在 Console 中检查
const container = tooltipContainer.value
console.log('预警检查结果:', getContainerAlerts(container))
console.log('是否有预警:', hasAlert(container))
console.log('预警级别:', getAlertLevel(container))
```
