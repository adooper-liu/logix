# 预警系统问题排查步骤

## 🎯 当前状态

我已经为预警系统添加了详细的调试日志。现在需要你按照以下步骤操作，我们就能快速定位问题所在。

---

## 📋 操作步骤

### 步骤 1：启动应用并打开甘特图页面

```bash
cd frontend
npm run dev
```

在浏览器中打开甘特图页面（通常是 `http://localhost:5173/simple-gantt` 或类似路径）

### 步骤 2：打开浏览器开发者工具

按 **F12** 打开开发者工具，切换到 **Console** 标签

### 步骤 3：刷新页面

刷新甘特图页面，等待数据加载完成

### 步骤 4：查看 Console 输出

你应该看到以下类型的日志：

#### A. 数据加载日志

```
[useGanttLogic] 找到 lastFreeDate: {
  containerNumber: "MSKU1234567",
  portType: "destination",
  lastFreeDate: 2026-03-20T00:00:00.000Z
}
```

这说明成功从货柜的港口操作记录中找到了 `lastFreeDate` 字段

#### B. 统计面板日志

```
[StatisticsPanel] 收到 containers: 156
[StatisticsPanel] 第一个货柜：{ ... }
[StatisticsPanel] 第一个货柜的 portOperations: [ ... ]
[StatisticsPanel] 有 lastFreeDate 的货柜数量：45
```

- `收到 containers: X` - 表示传入统计面板的货柜数量
- `有 lastFreeDate 的货柜数量：Y` - 表示有多少个货柜包含 `lastFreeDate` 字段

#### C. 预警触发日志

```
[Alert] 货柜有预警：{
  containerNumber: "MSKU1234567",
  logisticsStatus: "at_port",
  alerts: [
    { name: "即将超期", level: "warning", message: "距离最晚提柜不足 3 天" }
  ]
}
```

这说明某个货柜触发了预警规则

## 🔍 诊断结果判断

### ✅ 正常情况

如果你看到了上述所有类型的日志，特别是：

- ✅ 有 `[useGanttLogic] 找到 lastFreeDate` 的输出
- ✅ 有 `[Alert] 货柜有预警` 的输出
- ✅ 统计面板显示在页面上且有数字

**但是界面上看不到预警标识** → 这是样式问题

**解决方案**：检查 CSS 是否正确编译，可能需要重启开发服务器

---

### ❌ 异常情况 1：没有 `lastFreeDate` 日志

**现象**：Console 中没有 `[useGanttLogic] 找到 lastFreeDate` 的输出

**原因**：后端返回的数据中没有包含 `portOperations.lastFreeDate` 字段

**解决方法**：

1. 打开 Network 面板（F12 → Network）
2. 找到 `/containers` 请求
3. 查看响应数据中的 `portOperations`：

```json
{
  "items": [
    {
      "containerNumber": "MSKU1234567",
      "portOperations": []
    }
  ]
}
```

如果 `portOperations` 为空或缺少 `lastFreeDate`，需要检查后端代码

---

### ❌ 异常情况 2：有数据但无预警

**现象**：

- ✅ 看到了 `找到 lastFreeDate` 的日志
- ❌ 但没有 `货柜有预警` 的日志

**可能原因**：预警规则的条件判断不匹配

**检查方法**：

在 Console 中运行：

```javascript
const testContainer = containers[0]
const lastFreeDate = testContainer.portOperations?.find(
  op => op.portType === 'destination'
)?.lastFreeDate
console.log('lastFreeDate:', lastFreeDate)
console.log('当前日期:', new Date())
console.log('相差天数:', dayjs(lastFreeDate).diff(dayjs(), 'day'))
```

**结果分析**：

- 如果 `相差天数 > 3`，说明这个货柜不会触发预警（正常）
- 如果 `相差天数 < 0` 且状态不是已提柜，应该触发逾期预警

---

### ❌ 异常情况 3：统计面板不显示

**现象**：Console 中有日志，但页面上看不到统计面板

**检查方法**：

1. 在 Elements 面板中搜索 `.gantt-statistics-panel`
2. 如果元素存在但不显示，可能是：
   - CSS 样式问题（display: none）
   - 被其他元素遮挡
   - 父容器 overflow: hidden

## 🐞 常见问题速查

### Q1: 统计面板显示但数字都是 0

**原因**：数据中的字段名不匹配或计算逻辑有问题

**调试**：

```javascript
const c = containers[0]
console.log('物流状态:', c.logisticsStatus)
console.log('是否有 portOperations:', !!c.portOperations)
console.log(
  '是否有 lastFreeDate:',
  c.portOperations?.some(op => op.lastFreeDate)
)
```

### Q2: Tooltip 不显示预警信息

**原因**：Tooltip 模板中的条件判断失败

**检查**：

- `hasAlert(tooltipContainer)` 是否返回 true
- `getContainerAlerts(tooltipContainer)` 是否返回数组

### Q3: 甘特图上看不到带边框的圆点

**原因**：

1. 没有货柜触发预警
2. 或者边框样式未正确应用

**检查**：

- 右键点击任意圆点 → Inspect
- 查看元素是否有 `has-warning` class
- 查看内联样式中是否有 `border` 属性
