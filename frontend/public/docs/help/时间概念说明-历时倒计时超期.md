# 时间概念说明：历时、倒计时、超期

## 概述

LogiX 物流系统使用三种不同的时间概念来描述货物状态：**历时**、**倒计时**、**超期**。这些概念帮助您：

- **历时**：了解流程效率（已花多久）
- **倒计时**：规划未来工作（还有多久）
- **超期**：识别风险预警（迟了多久）

---

## 1. 历时 - 了解流程效率

### 定义

从一个时间点到另一个时间点已经过去的时间长度。

### 计算方式

**物流路径节点**：
```
历时 = 当前节点开始时间 - 上一节点结束时间
```

**关键日期**：
```
历时 = 后一节点日期 - 当前节点日期
```

### 应用场景

| 场景 | 说明 | 示例 |
|------|------|------|
| **物流路径** | 衡量两个物流节点之间的衔接效率 | "卸船后 4 小时开始提柜" |
| **关键日期** | 了解从一个关键日期到下一个关键日期的耗时 | "从 ATA 到卸船用了 2 天" |

### 显示样式

```
历时 1天
历时 2天 3小时
```

**颜色**：蓝色（中性分析）

**用途**：
- 分析流程效率
- 发现瓶颈环节
- 优化衔接时间

---

## 2. 倒计时 - 规划未来工作

### 定义

从现在到未来某个目标日期的剩余时间。

### 判断条件

**关键日期倒计时显示条件**：
- 后一节点不存在
- 当前节点在未来（`目标日期 - 当前日期 > 0`）

### 计算方式

```
倒计时数值 = 目标日期 - 当前日期
```

### 应用场景

| 场景 | 说明 | 示例 |
|------|------|------|
| **关键日期** | 提前安排提柜、还箱等操作 | "最晚提柜日还有 3 天" |
| **计划安排** | 合理规划人力资源 | "ETA 还有 5 天，可以提前准备" |

### 显示样式

```
今天到期
倒计时 1天
倒计时 5天
```

**颜色规则**：
- 🟠 **橙色**：今天到期或剩余 1-3 天（即将到期）
- 🟢 **绿色**：剩余 >3 天（安全）

**用途**：
- 提前预警
- 合理规划
- 避免延误

---

## 3. 超期 - 识别风险预警

### 定义

实际用时超过标准时长的部分（物流路径）或已过期天数（关键日期）。

### 判断条件

**关键日期超期显示条件**：
- 后一节点不存在
- 当前节点在过去（`当前日期 - 目标日期 > 0`）
- 已过天数超过标准天数（`已过天数 > 标准天数`）

### 计算方式

**关键日期（最晚提柜/最晚还箱）**：
```
超期数值 = 当前日期 - 目标日期（已过天数）
超期判断条件 = 当前日期 - 目标日期 > 0（目标日期在过去） AND 已过天数 > 标准天数
```

> **重要说明**：
> - 超期显示的是已过天数，不是减去标准后的数值
> - 红色超期：只有当已过天数超过标准天数时才显示红色
> - 标准耗时只是判断是否显示红色的阈值，不参与超期数值的计算

**物流路径节点**：
```
超期数值 = 当前时间 - 节点开始时间 - 标准耗时
超期判断 = 当前时间 - 节点开始时间 > 标准耗时
```

### 应用场景

| 场景 | 说明 | 示例 |
|------|------|------|
| **关键日期倒计时** | 提前规划提柜、还箱等操作 | "最晚提柜日还有 3 天" |
| **关键日期超期** | 识别最晚提柜/还箱日已过期 | "最晚提柜日已过 10 天，已超过 7 天标准" |
| **物流路径超期** | 发现货物在某个环节停留时间超过标准 | "卸船后 26 小时仍未提柜，超期 22 小时（标准 4 小时）" |

### 显示样式

**物流路径**：
```
超期 1天
超期 1天 5小时
```

**关键日期**：
```
超期 10天
超期 3天
```

**颜色**：红色 + 脉冲动画（风险预警）

**标准耗时配置**：
- 最晚提柜：7 天（168 小时）
- 最晚还箱：7 天（168 小时）
- 物流路径各节点：详见下文"标准耗时配置"

**用途**：
- 风险预警
- 人工干预
- 避免额外费用

## 4. 统一判断标准（关键日期 + 物流路径）

### 核心原则

**关键日期时间线**和**物流路径**采用**完全相同的判断标准**：基于**后一节点是否存在**决定显示历时、倒计时还是超期。

### 显示规则（统一标准）

| 条件 | 显示类型 | 计算方式 | 颜色 |
|------|---------|---------|------|
| **后一节点存在** | 历时 | `后一节点 - 当前节点` | 🔵 蓝色 |
| **后一节点不存在 + 当前节点在未来** | 倒计时 | `当前节点 - 今天` | 🟠 橙色 / 🟢 绿色 |
| **后一节点不存在 + 当前节点在过去 + 已超过标准** | 超期 | `今天 - 当前节点` | 🔴 红色 |
| **后一节点不存在 + 当前节点在过去 + 未超过标准** | 历时 | `今天 - 当前节点` | 🔵 蓝色 |

### 判断流程（统一标准）

```
当前节点
  ↓
是否有后一节点？
  ├─ 是 → 显示历时（蓝色）= 后一节点 - 当前节点
  └─ 否 → 判断当前节点相对于今天
         ├─ 在未来 → 显示倒计时（橙色/绿色）= 当前节点 - 今天
         └─ 在过去 → 显示超期/历时
                ├─ 已超过标准 → 显示超期（红色）= 今天 - 当前节点
                └─ 未超过标准 → 显示历时（蓝色）= 今天 - 当前节点
```

## 5. 关键时间线节点顺序

### 节点顺序

关键时间线按以下顺序排列：

```
出运 → ETA → 修正ETA → ATA → 卸船 → 最晚提柜 → 实际提柜 → 最晚还箱 → 实际还箱
```

### 示例
         ├─ 在未来 → 显示倒计时（橙色/绿色）
         └─ 在过去 → 显示超期（判断是否超过标准）
```

### 示例

**场景 1：完整时间线（关键日期）**
```
出运（2024-01-01） → ETA（2024-01-05） → 修正ETA（2024-01-06） → ATA（2024-01-08）
  ↓ [历时 4 天]          ↓ [历时 1 天]          ↓ [历时 2 天]          ↓ [倒计时 ?]
```

- 出运 → ETA：历时 4 天（后一节点存在）
- ETA → 修正ETA：历时 1 天（后一节点存在）
- 修正ETA → ATA：历时 2 天（后一节点存在）
- ATA：无后一节点，判断是否在今天之后

**场景 2：物流路径节点**
```
装船（2024-01-01 08:00）→ 航行中（2024-01-03 10:00）→ 抵港（2024-01-05 14:00）
  ↓ [历时 2天2小时]       ↓ [历时 2天4小时]       ↓ [倒计时 ?]
```

- 装船 → 航行中：历时 2天2小时（后一节点存在）
- 航行中 → 抵港：历时 2天4小时（后一节点存在）
- 抵港：无后一节点，判断是否在今天之后

**场景 3：修正ETA场景**
```
出运（2024-01-01） → ETA（2024-01-05） → 修正ETA（2024-01-06）
  ↓ [历时 4 天]          ↓ [历时 1 天]          ↓ [倒计时 2 天]
```

- 出运 → ETA：历时 4 天（后一节点存在）
- ETA → 修正ETA：历时 1 天（后一节点存在）
- 修正ETA：无后一节点，在今天之前 → 倒计时 2 天（绿色）

**场景 4：超期货柜**
```
ATA（2024-01-08） → 最晚提柜（2024-01-15）
  ↓ [历时 7 天]        ↓ [超期 10 天]
```

- ATA → 最晚提柜：历时 7 天（后一节点存在）
- 最晚提柜：无后一节点，在今天之前，已过 10 天 > 7 天标准 → 超期 10 天（红色）

**场景 4：部分数据缺失**
```
ETA（2024-01-10）
  ↓ [倒计时 5 天]
```

- ETA：无后一节点，在今天之前 → 倒计时 5 天（绿色）

## 6. 物流路径与关键日期对比

### 共同点（统一标准）

| 维度 | 关键日期 | 物流路径 |
|------|---------|---------|
| **判断标准** | 后一节点是否存在 | 后一节点是否存在 |
| **历时显示** | 有后一节点 → 历时（蓝色） | 有后一节点 → 历时（蓝色） |
| **倒计时显示** | 无后一节点 + 未来 → 倒计时 | 无后一节点 + 未来 → 倒计时 |
| **超期显示** | 无后一节点 + 过去 + 超标准 → 超期 | 无后一节点 + 过去 + 超标准 → 超期 |
| **超期颜色** | 红色（脉冲） | 红色（脉冲） |
| **倒计时颜色** | 橙色（≤3天）/ 绿色（>3天） | 绿色 |

### 差异点

| 维度 | 关键日期 | 物流路径 |
|------|---------|---------|
| **节点类型** | 固定日期点 | 动态物流节点 |
| **时间精度** | 天 | 小时 |
| **标准耗时** | 仅最晚提柜/最晚还箱 | 所有节点都有标准耗时 |
| **超期计算** | `今天 - 目标日期`（已过天数） | `今天 - 节点开始 - 标准`（超过标准时长） |

## 7. 核心区别对比

### 快速对比表

| 维度 | 历时 | 倒计时 | 超期 |
|------|------|--------|------|
| **方向** | 向后看 | 向前看 | 向前看（风险） |
| **性质** | 中性指标 | 正向指标 | 负面指标 |
| **用途** | 分析效率 | 规划工作 | 风险预警 |
| **颜色** | 蓝色 | 橙色/绿色 | 红色（脉冲） |
| **显示时机** | 有后一节点 | 无后一节点 + 未来 | 无后一节点 + 过去 + 超标准 |
| **计算基准** | 相对时间点 | 相对今天 | 相对今天 + 标准耗时 |

### 一句话总结

| 概念 | 核心含义 |
|------|---------|
| **历时** | 告诉你 **"已经多久"** |
| **倒计时** | 告诉你 **"还有多久"**（目标日期在未来） |
| **超期** | 告诉你 **"已过期多久"**（关键日期）或 **"已超过标准多久"**（物流路径）|

## 使用场景详解

### 场景 1：正常流程（无超期）

**物流路径**：
```
已装船 2024-01-01
  ↓ [历时 2小时]
航行中 2024-01-01 12:00
  ↓ [历时 1天20小时]
中转抵港 2024-01-03 08:00
```

**关键日期**：
```
出运     2024-01-01  → 2024-01-15 [倒计时 14天] 🟢
ATA     2024-01-08  → 2024-01-15 [倒计时 5天] 🟢
最晚提柜 2024-01-15 → 2024-01-15 [今天到期] 🟠
```

### 场景 2：超期预警

**物流路径**：
```
卸船      2024-01-08 08:00
  ↓ [历时 4小时]
当前节点  2024-01-08 12:00 [⚠️超期 1天2小时] 🔴
                       （标准耗时：4小时，超期 = 26小时 - 4小时 = 22小时）
```

**关键日期**：
```
最晚提柜 2024-01-15 → 2024-01-25 [⚠️已超期10天] 🔴
                          （显示：已过10天，判断：10天 > 7天标准）
```

### 场景 3：已完成流程

**物流路径**：
```
已装船  2024-01-01
  ↓ [历时 7天]
ATA     2024-01-08
  ↓ [历时 8天]
实际提柜 2024-01-16
  ↓ [历时 4天]
实际还箱 2024-01-20
```

**关键日期**：
```
出运     2024-01-01 → 2024-02-01 [历时 31天] 🔵
ATA      2024-01-08 → 2024-02-01 [历时 24天] 🔵
实际提柜 2024-01-16 → 2024-02-01 [历时 16天] 🔵
实际还箱 2024-01-20 → 2024-02-01 [历时 12天] 🔵
```

## 常见问题

### Q1：为什么"历时"是蓝色而不是红色？

**A**："历时"是**中性指标**，用于分析和了解流程效率。蓝色表示这是正常的历史数据，不需要引起过度关注。只有"超期"才用红色表示风险。

### Q2："倒计时"什么时候变橙色？

**A**：当剩余时间 ≤ 3 天时，倒计时会变成橙色，提醒您即将到期，需要尽快处理。

### Q3：关键时间线如何判断显示"历时"、"倒计时"还是"超期"？

**A**：关键时间线的显示逻辑基于**后一节点是否存在**：

```
如果有后一节点：
  → 显示"历时"（蓝色）= 后一节点 - 当前节点
  
如果没有后一节点：
  → 判断当前节点相对于今天
    ├─ 在未来 → 显示"倒计时"（橙色/绿色）= 当前节点 - 今天
    └─ 在过去 → 显示"超期"（红色/蓝色）= 今天 - 当前节点
       └─ 如果是关键节点且已过天数 > 标准天数 → 红色
```

**示例 1：完整时间线**
```
出运（2024-01-01）→ ETA（2024-01-05）→ ATA（2024-01-08）
  ↓ [历时 4 天]        ↓ [历时 3 天]        ↓ [倒计时 ?]
```
- 出运：有后一节点 → 历时 4 天
- ETA：有后一节点 → 历时 3 天
- ATA：无后一节点，在未来 → 倒计时

**示例 2：超期货柜**
```
ATA（2024-01-08）→ 最晚提柜（2024-01-15）
  ↓ [历时 7 天]        ↓ [超期 10 天]
```
- ATA：有后一节点 → 历时 7 天
- 最晚提柜：无后一节点，在过去，已过 10 天 > 7 天标准 → 超期 10 天（红色）

### Q4：什么情况下会显示"超期"？

**A**：满足以下条件时显示"超期"：

**物流路径节点**：
- 节点停留时间超过标准耗时
- 显示格式：`超期 X天Y小时`（超过标准时长的部分）

**关键日期（最晚提柜/最晚还箱）**：
- 无后一节点
- 目标日期已过去（`当前日期 - 目标日期 > 0`）
- 已过天数超过标准天数（`已过天数 > 标准天数`）
- 显示格式：`超期 X天`（已过天数，不是减去标准后的数值）

例如（关键日期）：
- 最晚提柜日：2024-01-15
- 标准耗时：7 天
- 当前日期：2024-01-25
- 已过天数：10 天
- 超期显示：`超期10天`（显示已过10天）
- 判断条件：10天 > 7天标准 → 显示红色超期

### Q5：物流路径中的"超期"和关键日期中的"超期"有什么区别？

**A**：

| 维度 | 物流路径超期 | 关键日期超期 |
|------|------------|------------|
| **适用范围** | 当前正在进行的节点 | 最晚提柜/最晚还箱日 |
| **计算方式** | (当前时间 - 节点开始 - 标准) | 当前日期 - 目标日期 |
| **判断条件** | 停留时间 > 标准耗时 | 已过天数 > 标准天数 |
| **显示位置** | 物流路径时间线 | 关键日期时间线 |
| **含义** | 超过标准时长的部分 | 已过期天数 |

### Q6：标准耗时可以修改吗？

**A**：当前标准耗时是硬编码的。未来版本将支持：
- 从后端数据库动态读取
- 按港口/航线/货代公司配置不同标准
- 用户自定义标准耗时

## 最佳实践

### 使用"历时"优化流程

1. **识别瓶颈**：找到"历时"较长的节点
2. **分析原因**：为什么这个环节耗时较长？
3. **制定改进**：优化流程，缩短衔接时间

### 使用"倒计时"提前规划

1. **关注橙色**：倒计时 ≤ 3 天时立即处理
2. **提前准备**：根据倒计时合理安排资源
3. **避免延误**：在倒计时到期前完成操作

### 处理"超期"风险

1. **立即关注**：红色超期需要优先处理
2. **评估影响**：超期可能产生额外费用或影响后续操作
3. **采取措施**：联系相关方，尽快解决问题
