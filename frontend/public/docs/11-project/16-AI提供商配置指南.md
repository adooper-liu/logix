# AI 提供商配置指南

## 概述

LogiX AI 聊天系统支持多种 AI 提供商，可以在运行时动态切换：

1. **SiliconFlow (硅基流动)** - 云端 API 服务
2. **Ollama Local** - 本地部署的 Ollama 服务
3. **Ollama Cloud** - Ollama 云端服务

---

## 环境变量配置

### 1. SiliconFlow (默认)

```bash
# .env 文件
AI_PROVIDER=siliconflow
SILICON_FLOW_API_KEY=your_api_key_here
SILICON_FLOW_MODEL=deepseek-ai/DeepSeek-V2-Chat
SILICON_FLOW_BASE_URL=https://api.siliconflow.cn/v1  # 可选
```

**获取 API Key**:

- 访问 https://siliconflow.cn
- 注册账号并获取 API Key
- 支持 DeepSeek、Qwen、Yi 等国产大模型

---

### 2. Ollama Local (本地部署)

```bash
# .env 文件
AI_PROVIDER=ollama-local
OLLAMA_MODEL=llama3.2
OLLAMA_LOCAL_BASE_URL=http://localhost:11434  # 可选，默认值
```

**安装步骤**:

1. **下载 Ollama**

   ```bash
   # macOS
   brew install ollama

   # Windows
   # 从 https://ollama.ai 下载安装包

   # Linux
   curl -fsSL https://ollama.ai/install.sh | sh
   ```

2. **启动 Ollama 服务**

   ```bash
   ollama serve
   ```

3. **拉取模型**

   ```bash
   # 推荐模型（轻量级）
   ollama pull llama3.2

   # 其他可用模型
   ollama pull mistral
   ollama pull qwen2.5
   ollama pull deepseek-r1
   ```

4. **验证安装**
   ```bash
   curl http://localhost:11434/api/tags
   ```

**优点**:

- ✅ 完全免费
- ✅ 数据隐私安全（本地运行）
- ✅ 无网络延迟
- ✅ 可离线使用

**缺点**:

- ⚠️ 需要本地 GPU/CPU 资源
- ⚠️ 模型质量可能不如云端
- ⚠️ 首次拉取模型较慢

---

### 3. Ollama Cloud (云端服务)

```bash
# .env 文件
AI_PROVIDER=ollama-cloud
OLLAMA_CLOUD_API_KEY=your_ollama_cloud_api_key
OLLAMA_CLOUD_MODEL=llama3.2
OLLAMA_CLOUD_BASE_URL=https://ollama.ai/api  # 可选，默认值
```

**注意**:

- Ollama Cloud 目前处于 Beta 阶段
- 请参考官方文档: https://docs.ollama.com/cloud
- 需要注册 Ollama Cloud 账号获取 API Key

## 前端使用

### 查看当前提供商

访问 http://localhost:5173/#/ai-chat，点击右上角的状态标签，可以看到：

- **当前使用的提供商**（绿色标签显示）
- **所有可用的提供商列表**
- **每个提供商的配置状态**（已配置/未配置）

### 切换提供商

1. 点击状态标签（如"硅基流动"）
2. 在下拉菜单中选择目标提供商
3. 系统会自动切换并重新连接
4. 切换成功后会显示提示消息

**注意**:

- 只能切换到"已配置"的提供商
- 未配置的提供商显示为灰色且不可点击
- 切换后会立即生效，无需刷新页面

### 3. 切换提供商

```http
POST /api/v1/ai/providers/switch
Content-Type: application/json

{
  "provider": "siliconflow"
}
```

**响应示例**:

```json
{
  "success": true,
  "message": "Successfully switched to siliconflow",
  "data": {
    "current": "siliconflow"
  }
}
```

**错误响应**:

```json
{
  "success": false,
  "message": "Provider ollama-cloud is not configured or disabled"
}
```

## 故障排查

### 问题 1: Ollama Local 连接失败

**症状**: 健康检查返回 `missing_config` 或 `error`

**解决方案**:

```bash
# 1. 检查 Ollama 是否运行
ollama list

# 2. 检查服务是否监听正确端口
curl http://localhost:11434/api/tags

# 3. 如果未运行，启动服务
ollama serve

# 4. 检查防火墙设置
# Windows: 允许 11434 端口
# macOS: 系统偏好设置 -> 安全性与隐私 -> 防火墙
```

---

### 问题 2: SiliconFlow API Key 无效

**症状**: 健康检查返回 `missing_api_key`

**解决方案**:

1. 检查 `.env` 文件中是否正确配置了 `SILICON_FLOW_API_KEY`
2. 登录 SiliconFlow 控制台验证 API Key 是否有效
3. 重启后端服务使配置生效

---

### 问题 3: 切换提供商后无法对话

**症状**: 切换成功，但发送消息时失败

**解决方案**:

1. 检查新提供商的健康状态
2. 查看后端日志确认错误信息
3. 尝试切换回原来的提供商
4. 如果是 Ollama，确认模型已正确拉取

```bash
# 检查已安装的模型
ollama list

# 如果模型不存在，重新拉取
ollama pull llama3.2
```

## 高级配置

### 自定义超时时间

```bash
# Ollama Local
OLLAMA_LOCAL_TIMEOUT=120000  # 120秒

# Ollama Cloud
OLLAMA_CLOUD_TIMEOUT=120000

# SiliconFlow
SILICON_FLOW_TIMEOUT=60000  # 60秒
```

### 调整模型参数

```bash
# 温度（创造性）
OLLAMA_TEMPERATURE=0.7
SILICON_FLOW_TEMPERATURE=0.7

# 最大 Token 数
OLLAMA_MAX_TOKENS=4096
SILICON_FLOW_MAX_TOKENS=4096
```

### 多模型切换

在代码中可以动态指定模型：

```typescript
// 前端调用时指定模型
await aiService.chat(message, context, {
  model: 'qwen2.5', // 覆盖默认模型
})
```

```typescript

  await chatWithProvider(backup)
}
```
