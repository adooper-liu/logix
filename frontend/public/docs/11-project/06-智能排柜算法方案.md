## 一、ContainerPlanning 代码解读摘要

### 1.1 核心数据结构

| 结构 | 说明 |
|------|------|
| **Container** | 柜号、ETA、提柜/送仓/卸柜/还箱日（相对天数）、港口、仓库、卸柜方式（Drop unload / Live unload）、最晚提柜/还箱日、候选仓库列表、fixed* 标志 |
| **SolutionResult** | 解（Container 列表）、总成本、仓库占用 |
| **TruckResult** | 车队、拖车费 |

### 1.2 费用模型（总成本 = 滞港 + 堆存 + 滞箱 + DDC + 堆场 + 拖车）

| 费用项 | 计算逻辑 | 匹配维度 |
|--------|----------|----------|
| **滞港费 Demurrage** | T_g = max(0, latestPick → pick 超免费期天数)；按阶梯费率（1–60 天、60+）计费 | 目的港、船公司、货代 |
| **堆存费 Storage** | 同滞港起算；T_g 超免费期按 storageFees 阶梯 | 同上 |
| **滞箱费 Detention** | T_c = pick → return 用箱天数；超免费期按 detFees 阶梯 | 同上 |
| **滞港&滞箱 DDC** | T_b = latestPick → return；若 DDC 非全 0 则用 DDC 替代 Dem+Det | 同上 |
| **堆场费 Yard** | yardDays = delivery − pick（Drop 模式下柜在车队堆场天数）；cost = 单柜价格×天数 + 运费 | 车队、港口 |
| **拖车费 Truck** | 仓库×港口 定价表，选最低价车队 | 仓库、港口、车队 |

### 1.3 资源约束与「最早可用日」算法

| 资源 | 数据结构 | 查找逻辑 |
|------|----------|----------|
| **仓库日产能** | mapWarehouseCapacity(warehouse, date) | `findEarliestAvailableDay`：从 startDay 起找第一个「当日占用 < 产能」且非周末的日期 |
| **车队送还能力** | mapFleetReturnCapacity / mapFleetReturnUsage（fleet_warehouse, date） | `findEarliestAvailableFleetReturnDay`：找第一个有可用车队送还能力的日期；`findEarliestAvailableDeliveryDay`：送仓日受车队送还能力约束（同一能力池） |
| **车队堆场** | mapFleetYardCapacity(fleet, port) | `checkYardUsability`：有堆场则 Drop 模式下 pick/delivery/unload 可错开；无则 delivery=unload=return |
| **港口-仓库** | mapPortWarehouse | 候选仓库来自港口对应关系 |
| **周末** | weekendDays 列表 | 跳过周六、周日 |

### 1.4 初始解生成（generateInitialSolution）

- **排序**：按 ETA 升序（先到先排）
- **卸柜方式分支**：
  - **Drop unload**：提柜→送仓→卸柜→还箱 可错开；有车队堆场时 pick < delivery < unload < return
  - **Live unload**：delivery = unload = return（同日）
- **固定状态分支**（Case A–D）：
  - 已固定卸柜：直接分配还箱日与车队送还能力
  - 已固定送仓：找最早可用卸柜日 → 还箱日
  - 已固定提柜：找最早卸柜日 → 检查堆场 → 送仓日（受车队送还约束）→ 还箱日
  - 全未固定：遍历 possibleWarehouses，找第一个「仓库×日期」有剩余能力的组合，再排 pick/delivery/unload/return
- **车队选择**：`getAvailableFleetReturnCapacityByPrice` 按价格排序，选最低价且有剩余能力的车队；`allocateFleetReturnCapacity` 扣减占用

### 1.5 局部搜索（localSearch）

- **邻域操作**：随机选柜 + 随机选操作
  - `change_warehouse`：换仓（校验新仓当日产能、车队送还能力）
  - `change_unload`：卸柜日 ±{−2,−1,1,2,3,4,5} 天
  - `change_pick`：提柜日在 [eta+1, delivery−1] 内随机
  - `change_delivery`：送仓日在 [pick, unload−1] 内随机，校验车队送还能力
- **接受准则**：仅接受 cost ≤ currentCost（贪心下降，无模拟退火）
- **固定约束**：fixed* 的字段不参与对应操作

## 二、行业最佳实践补充

| 实践 | 说明 |
|------|------|
| **先到先得 + 紧迫度** | 按 ETA 或 last_free_date 排序；即将超期柜优先排 |
| **费用阶梯** | 滞港/滞箱按 1–7、8–14、15–30、31–60、60+ 等阶梯计费，与船东标准一致 |
| **工作日/自然日** | 免费期与计费期支持 Natural/Working/Natural+Working 基准 |
| **三种方案决策** | 直接送仓 / 先提送堆场再送仓 / 滞港等仓；总费用最少选一（见 04、05 方案） |
| **Drop vs Live** | Drop 可落地、还箱可滞后；Live 当天卸当天还，占用实际卸柜能力 |
| **模拟退火/遗传** | 大规模时可用 SA 或 GA 接受劣解以跳出局部最优 |

## 三、算法方案（与 LogiX 对接）

### 3.1 总体流程

```
输入：货柜列表、日期范围、国家、配置表
  ↓
1. 过滤：排除 schedule_status ≠ initial
2. 排序：按 清关可放行日(ATA/ETA) 升序，同日内按 last_free_date 升序（紧迫优先）
  ↓
3. 初始解：规则引擎（见 3.2）
  ↓
4. 局部搜索：邻域操作 + 贪心接受（可选 SA 扩展）
  ↓
5. 输出：五节点计划时间、资源占用、总成本
```

### 3.2 规则引擎（初始解）

| 步骤 | 逻辑 |
|------|------|
| 1 | 对每柜：按**备货单的港口-仓库关系**确定 possibleWarehouses |
| 2 | **计划清关日** = 清关可放行日（ATA 或 修正 ETA） |
| 3 | **计划提柜日** = 计划清关日 + N 天，且 ≤ last_free_date |
| 4 | **计划卸柜日** = 从计划提柜日起，`findEarliestAvailableDay`（仓库日产能、跳过周末） |
| 5 | **计划送仓日**：Live loading = 计划卸柜日；Drop off = 计划卸柜日 或 前一日（视拖车档期） |
| 6 | **计划还箱日**：Live loading = 计划卸柜日；Drop off = 计划卸柜日 + 1～2 天，且 ≤ last_return_date |
| 7 | **拖车选择**：按 仓库×港口 定价表，选**最低价且有剩余日容量**的车队；扣减拖车档期 |
| 8 | **车队送还能力**（若 Drop + 还箱滞后）：按 仓库×港口×日期 查可用车队，选最低价，扣减占用 |
| 9 | **第三堆场**：若采用「先提送堆场再送仓」，yardDays = 堆场停留天数；costYard = 容量费用表费率×天数 + 运费 |

### 3.3 三种方案决策（仓库满负荷时）

| 方案 | 计划时间 | 费用项 |
|------|----------|--------|
| **直接送仓** | 顺延至有仓库能力日 | 滞港费（若有）+ 拖车 + 卸柜 |
| **先提送堆场** | pick→堆场→等 N 天→送仓→卸柜→还箱 | 拖车(港→堆场) + 堆场费×N + 拖车(堆场→仓) + 卸柜 + 多环节风险成本 |
| **滞港等仓** | 等 N 天→提柜→送仓→卸柜→还箱 | 滞港费×N + 拖车 + 卸柜 |

比较总费用，选最小方案；若滞港与堆场费相差不大，优先滞港（见 04 方案 2.3.1）。

### 3.4 局部搜索邻域

| 操作 | 条件 | 邻域 |
|------|------|------|
| change_warehouse | 非 fixedWarehouse | 在 possibleWarehouses 内换仓，校验新仓当日产能与车队送还 |
| change_unload | 非 fixedUnload | unload ± {−2,−1,1,2,3,4,5}，校验产能、顺序、last_return_date |
| change_pick | 非 fixedPick | pick 在 [清关日, delivery−1] 内随机，且 ≤ last_free_date |
| change_delivery | 非 fixedDelivery | delivery 在 [pick, unload−1] 内随机，校验拖车档期 |

### 3.5 费用计算（与 demurrage 文档一致）

- **滞港**：起算日 = ATA/ETA/卸船；截止日 = 提柜日；T_g = 超免费期天数；按阶梯计费
- **滞箱**：起算日 = 提柜日；截止日 = 还箱日；T_c = 超免费期天数；按阶梯计费
- **堆场**：yardDays × 单柜日费 + 运费（来自第三堆场容量与费用参数表）
- **拖车**：仓库×港口×车队 定价表
