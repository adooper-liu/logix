# 智能排柜流程优化方案

## 1. 现状分析

### 1.1 当前排柜流程架构

#### 核心流程图

```
数据准备 -> 待排产筛选 -> 排序 -> 逐柜排产 -> 保存结果 -> 成本优化
   |            |           |         |          |          |
   |            |           |         |          |          └─> SchedulingCostOptimizerService
   |            |           |         |          |
   |            |           |         |          └─> 更新流程表 + 扣减产能
   |            |           |         |
   |            |           |         └─> IntelligentSchedulingService.scheduleSingleContainer
   |            |           |              ├─> 选择仓库 (WarehouseSelectorService)
   |            |           |              ├─> 选择车队 (TruckingSelectorService)
   |            |           |              ├─> 计算日期 (SchedulingDateCalculator)
   |            |           |              └─> 决定卸柜方式 (Drop off / Live load)
   |            |           |
   |            |           └─> 按清关可放行日排序 (sortByClearanceDate)
   |            |
   |            └─> ContainerFilterService.getContainersToSchedule
   |               ├─> schedule_status = 'initial'
   |               ├─> 国家/港口/日期过滤
   |               └─> LFD/免费期预处理
   |
   └─> 基础数据准备
      ├─> dict_warehouses (仓库能力)
      ├─> dict_trucking_companies (车队能力)
      ├─> dict_trucking_port_mapping (港口 - 车队映射)
      ├─> dict_warehouse_trucking_mapping (仓库 - 车队映射)
      └─> ext_*_occupancy (产能日历)
```

#### 关键服务与职责

| 服务名 | 职责 | 关键方法 |
|--------|------|----------|
| **IntelligentSchedulingService** | 智能排柜主服务，协调整个排产流程 | `batchSchedule()`, `scheduleSingleContainer()` |
| **ContainerFilterService** | 筛选待排产货柜 | `getContainersToSchedule()` |
| **SchedulingSorter** | 货柜排序 | `sortByClearanceDate()` |
| **WarehouseSelectorService** | 选择最优仓库 | `selectWarehouse()` |
| **TruckingSelectorService** | 选择最优车队 | `selectTruckingCompany()` |
| **SchedulingDateCalculator** | 计算各节点日期 | `calculatePlannedPickupDate()`, `calculateReturnDate()` |
| **OccupancyCalculator** | 扣减产能 | `decrementWarehouseOccupancy()`, `decrementTruckingOccupancy()` |
| **SchedulingCostOptimizerService** | 成本优化建议 | `suggestOptimalUnloadDate()`, `batchOptimize()` |

### 1.2 核心业务逻辑

#### 五节点调度流程

```
到港 (ATA/ETA) 
  ↓
提柜 (plannedPickupDate) ← 基于 ETA + etaBufferDays
  ↓
送仓 (plannedDeliveryDate) ← 基于卸柜方式决定
  ↓
卸柜 (plannedUnloadDate) ← 基于仓库产能日历
  ↓
还箱 (plannedReturnDate) ← 基于卸柜方式 + 车队还箱产能
```

#### 卸柜方式决策树

```
truckingCompany.hasYard = true
  ├─> Drop off 模式（提 < 送 = 卸）
  │    └─> 允许货柜在堆场临时存放
  │         └─> 计算外部堆场堆存费
  │
truckingCompany.hasYard = false
  └─> Live load 模式（提 = 送 = 卸）
       └─> 必须当天完成提送卸
```

#### 产能约束检查

**仓库产能**:
- 表：`ext_warehouse_daily_occupancy`
- 字段：`capacity` (总容量), `planned_count` (已计划), `remaining` (剩余)
- 查询：按 `warehouse_code + date` 唯一索引

**车队提柜产能**:
- 表：`ext_trucking_slot_occupancy`
- 字段：`capacity`, `planned_trips`, `remaining`
- 维度：`trucking_company_id + date + port_code + warehouse_code`

**车队还箱产能** (Drop off 模式):
- 表：`ext_trucking_return_slot_occupancy`
- 字段：`capacity`, `planned_count`, `remaining`
- 维度：`trucking_company_id + slot_date`

### 1.3 现有优化机制

#### 成本优化服务 (SchedulingCostOptimizerService)

**工作流程**:
1. 获取候选仓库列表
2. 生成所有可行方案 (Direct/Drop off/Expedited)
3. 评估每个方案的总成本
4. 选择成本最低的方案

**成本构成**:
```typescript
totalCost = demurrageCost + detentionCost + storageCost + 
            yardStorageCost + transportationCost + handlingCost
```

**优化策略**:
- **免费期内**: 从免费日往前找，优先零成本方案
- **已超期**: 从今天往后找，尽量减少损失

### 1.4 存在的问题

#### 问题 1: 预测算法过于简单

**现状**:
- 时间预测 TAB 使用固定周期推导 (到港 +2 天提柜、提柜 +1 天卸柜、卸柜 +3 天还箱)
- 未考虑实际产能约束
- 与智能排柜引擎脱节

**影响**:
- 预测准确率低下
- 无法指导实际操作
- 用户信任度低

#### 问题 2: 成本优化与实际排产分离

**现状**:
- `SchedulingCostOptimizerService` 仅作为事后建议
- 排产时未实时计算成本
- 优化建议可能违反产能约束

**影响**:
- 优化方案无法落地
- 需要二次调整
- 增加操作复杂度

#### 问题 3: 产能日历利用率不足

**现状**:
- 虽然建立了产能日历表 (`ext_*_occupancy`)
- 但仅用于简单的容量检查 (`remaining > 0`)
- 未实现智能日历能力 (Smart Calendar Capacity)

**缺失功能**:
- 节假日自动识别
- 周末差异化配置
- 动态能力调整
- 可视化档期查看

#### 问题 4: 前端交互复杂

**现状**:
- 排产结果卡片信息过载
- 成本明细展示不直观
- 缺少批量操作引导

**用户体验问题**:
- 注意力分散 (统计、日志、结果混在一起)
- 决策支持不足 (看不出为什么要优化)
- 操作路径长 (预览 → 确认 → 保存)

## 2. 优化方案设计

### 2.1 总体目标

**构建"预测 - 排产 - 优化"一体化智能引擎**:
1. **预测准确**: 基于真实产能和历史数据
2. **排产智能**: 实时成本约束 + 产能约束
3. **优化自动**: 无需人工干预的自动优化
4. **体验流畅**: 所见即所得的交互式排产

### 2.2 架构优化

#### 方案 A: 基于智能排柜引擎的预测 (推荐)

**核心思路**:
- 时间预测直接调用排产引擎
- 预测结果 = 模拟排产结果
- 预测与排产使用同一套逻辑

**实现步骤**:

#### 方案 B: 基于产能日历的启发式预测

**核心思路**:
- 使用智能日历能力 (Smart Calendar Capacity)
- 基于历史数据学习
- 逐层推导时间节点

**实现步骤**:

```typescript
// 1. 增强 smartCalendarCapacity 工具
export class SmartCalendarCapacity {
  /**
   * 查找下一个可用日期
   */
  async findNextAvailableDate(
    resourceType: 'warehouse' | 'trucking' | 'trucking-return',
    resourceId: string,
    startDate: Date,
    minOffset: number = 0
  ): Promise<Date> {
    let currentDate = dateTimeUtils.addDays(startDate, minOffset);
    
    while (true) {
      // 1. 检查是否为工作日
      if (await this.isWeekend(currentDate)) {
        currentDate = dateTimeUtils.addDays(currentDate, 1);
        continue;
      }
      
      // 2. 检查产能
      const occupancy = await this.ensureResourceOccupancy(
        resourceType,
        resourceId,
        currentDate
      );
      
      if (occupancy.remaining > 0) {
        return currentDate;
      }
      
      // 3. 产能不足，尝试下一天
      currentDate = dateTimeUtils.addDays(currentDate, 1);
    }
  }
  
  /**
   * 通用资源档期确保方法
   */
  private async ensureResourceOccupancy(
    resourceType: string,
    resourceId: string,
    date: Date
  ) {
    switch (resourceType) {
      case 'warehouse':
        return smartCalendarCapacity.ensureWarehouseOccupancy(resourceId, date);
      case 'trucking':
        return smartCalendarCapacity.ensureTruckingOccupancy(resourceId, date);
      case 'trucking-return':
        return smartCalendarCapacity.ensureTruckingReturnOccupancy(resourceId, date);
      default:
        throw new Error(`Unknown resource type: ${resourceType}`);
    }
  }
}

// 2. 重构 TimeService
export class TimeService {
  private smartCalendar = new SmartCalendarCapacity();
  
  async predictPickupTime(container: Container, trucking: any): Promise<Date | null> {
    if (trucking?.pickupDate) {
      return new Date(trucking.pickupDate);
    }
    
    if (container.portOperations) {
      const destPort = container.portOperations.find(op => op.portType === 'destination');
      const eta = destPort?.eta || container.seaFreight?.eta;
      
      if (!eta) return null;
      
      const startDate = new Date(eta);
      
      // 获取目的港
      const portCode = destPort?.portCode;
      
      // 获取默认车队 (基于国家和港口)
      const defaultTrucking = await this.getDefaultTruckingCompany(
        container.countryCode,
        portCode
      );
      
      if (!defaultTrucking) {
        // 降级：简单预测
        const predicted = new Date(startDate);
        predicted.setDate(predicted.getDate() + 2);
        return predicted;
      }
      
      // 考虑真实产能
      const pickupDate = await this.smartCalendar.findNextAvailableDate(
        'trucking',
        defaultTrucking.companyCode,
        startDate,
        2 // minOffset: 到港后至少 2 天
      );
      
      return pickupDate;
    }
    
    return null;
  }
}
```

**优势**:
- 考虑真实产能约束
- 支持灵活配置
- 可扩展性强

**实施难度**: 高 (需完善产能日历)

**预计工作量**: 2-3 周

#### 优化点 2: 批量优化前置

**现状**: 批量排产后逐个优化

**改进**: 排产前预先分组优化

```typescript
async batchSchedule(request: ScheduleRequest) {
  // 1. 筛选待排产货柜
  const containers = await this.getContainersToSchedule(request);
  
  // 2. 按免费期分组
  const withinFreePeriod = [];
  const overdue = [];
  
  for (const container of containers) {
    const lastFreeDate = await this.getLastFreeDate(container);
    const daysRemaining = dateTimeUtils.daysBetween(new Date(), lastFreeDate);
    
    if (daysRemaining >= 0) {
      withinFreePeriod.push({ container, lastFreeDate, daysRemaining });
    } else {
      overdue.push({ container, lastFreeDate, daysRemaining });
    }
  }
  
  // 3. 优先级排序
  overdue.sort((a, b) => a.daysRemaining - b.daysRemaining); // 超期越久越优先
  withinFreePeriod.sort((a, b) => b.daysRemaining - a.daysRemaining); // 免费期越短越优先
  
  // 4. 批量成本优化
  const optimizedGroups = await this.costOptimizerService.batchOptimize(
    [...overdue, ...withinFreePeriod].map(x => x.container.containerNumber),
    new Date()
  );
  
  // 5. 按优化后的顺序排产
  for (const group of optimizedGroups.results) {
    const result = await this.scheduleSingleContainer(group.container, request);
    results.push(result);
  }
}
```

**收益**:
- 全局最优而非局部最优
- 减少产能冲突
- 提升整体成本优化效果

### 2.4 产能日历增强

#### 功能 1: 智能日历能力 (Smart Calendar Capacity)

**目标**: 自动计算每日产能，支持节假日/周末配置

**实现**:

```typescript
// backend/src/utils/smartCalendarCapacity.ts (增强版)
export class SmartCalendarCapacity {
  /**
   * 确保仓库档期记录存在并设置正确的能力
   */
  async ensureWarehouseOccupancy(warehouseCode: string, date: Date) {
    // 1. 查找现有记录
    let occupancy = await this.warehouseOccupancyRepo.findOne({
      where: { warehouseCode, date }
    });
    
    if (occupancy && occupancy.capacity > 0) {
      return occupancy; // 已有有效记录
    }
    
    // 2. 计算当日产能
    const baseCapacity = await this.getBaseCapacity(warehouseCode);
    const multiplier = await this.getDayMultiplier(date);
    const finalCapacity = Math.floor(baseCapacity * multiplier);
    
    // 3. 更新或创建记录
    if (occupancy) {
      occupancy.capacity = finalCapacity;
      occupancy.remaining = Math.max(0, finalCapacity - occupancy.plannedCount);
      await this.warehouseOccupancyRepo.save(occupancy);
    } else {
      await this.warehouseOccupancyRepo.save({
        warehouseCode,
        date,
        plannedCount: 0,
        capacity: finalCapacity,
        remaining: finalCapacity
      });
    }
    
    return occupancy;
  }
  
  /**
   * 获取基础产能 (从仓库配置)
   */
  private async getBaseCapacity(warehouseCode: string): Promise<number> {
    const warehouse = await this.warehouseRepo.findOne({
      where: { warehouseCode },
      select: ['dailyUnloadCapacity']
    });
    return warehouse?.dailyUnloadCapacity || 20; // 默认 20
  }
  
  /**
   * 获取日期倍率 (周末/节假日)
   */
  private async getDayMultiplier(date: Date): Promise<number> {
    // 1. 检查是否为周末
    if (await this.isWeekend(date)) {
      const weekendConfig = await this.getConfig('weekend_capacity_multiplier');
      return parseFloat(weekendConfig?.configValue || '0'); // 周末默认 0
    }
    
    // 2. 检查是否为节假日
    if (await this.isHoliday(date)) {
      const holidayConfig = await this.getConfig('holiday_capacity_multiplier');
      return parseFloat(holidayConfig?.configValue || '0'); // 节假日默认 0
    }
    
    // 3. 工作日
    return 1.0;
  }
  
  /**
   * 检查是否为周末
   */
  private async isWeekend(date: Date): Promise<boolean> {
    const day = date.getDay();
    return day === 0 || day === 6; // 周日或周六
  }
  
  /**
   * 检查是否为节假日
   */
  private async isHoliday(date: Date): Promise<boolean> {
    // 从 dict_holidays 表查询
    const holiday = await this.holidayRepo.findOne({
      where: {
        date: date.toISOString().split('T')[0],
        isActive: true
      }
    });
    return !!holiday;
  }
}
```

**前端展示**:
- 日历视图显示每日产能
- 颜色区分工作日/周末/节假日
- 点击查看详情 (基础产能、倍率、最终产能、已占用、剩余)
