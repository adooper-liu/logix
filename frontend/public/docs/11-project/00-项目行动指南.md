# LogiX 项目行动指南

## 一、项目概览

### 1.1 技术架构

| 层级 | 技术栈 | 端口 |
|------|--------|------|
| 前端 | Vue 3.4 + TypeScript + Element Plus + ECharts + Pinia | 5173 |
| 后端 | Node.js 18+ + Express + TypeORM | 3001 |
| 数据库 | PostgreSQL 15 + TimescaleDB 2.15 | 5432 |
| 缓存 | Redis 7 | 6379 |

### 1.2 核心业务域

- **货柜全生命周期**：备货单 → 海运 → 港口操作（多港经停）→ 拖卡 → 仓库卸柜 → 还空箱
- **五节点调度**：清关、拖卡、卸柜、还箱、查验
- **费用与风险**：滞港费、滞箱费、最晚提柜/还箱、异常预警
- **外部数据**：飞驼 API 等船公司/码头数据接入

## 三、未来需求与规划

### 3.1 五节点调度与可视化（方案已定）

**参考**：[04-五节点调度与可视化方案.md](./04-五节点调度与可视化方案.md)

| 节点 | 计划字段 | 实际字段 | 倒计时依据 |
|------|----------|----------|------------|
| 清关 | planned_customs_date | actual_customs_date | ETA/修正 ETA |
| 拖卡 | planned_pickup_date, planned_delivery_date | pickup_date, delivery_date | last_free_date |
| 卸柜 | planned_unload_date | unload_date | planned_unload_date / last_return_date |
| 还箱 | planned_return_date | return_time | last_return_date |

### 4.3 依赖关系

```
P0 滞港费闭环
  └─ last_free_date 计算依赖标准匹配
  └─ 滞港费计算依赖 last_free_date + 阶梯费率

P1 五节点可视化
  └─ 依赖现有 process_* 表计划字段
  └─ 仓库能力、第三堆场为 P2 前置

P2 智能排柜
  └─ 依赖 P0（last_free_date、滞港费标准）
  └─ 依赖 P1（仓库能力、第三堆场、schedule_status）

P3 甘特图资源
  └─ 依赖 P2 资源占用表
  └─ 依赖 P2 计划时间产出
```
