# 甘特图资源动态展示方案

> 文档版本：1.0
> 创建日期：2026-03-15
> 状态：**已实施**

---

## 一、概述

### 1.1 背景

当前甘特图（SimpleGanttChartRefactored.vue）已实现五节点（清关/提柜/送仓/卸柜/还箱）时间线展示，但**缺少资源动态占用显示**。用户无法直观看到：

- 某仓库某天已安排多少柜子、剩余产能
- 某车队某天可安排多少车次
- 满负荷预警

### 1.2 目标

在现有五节点甘特图基础上，在**卸柜节点格子内嵌入资源占用统计**，让用户一眼看到「某仓库某天已安排多少柜子、还剩多少容量」。

### 2.4 资源占用现状

**❌ 当前没有资源占用统计**，格子内只显示货柜圆点，不显示资源占用数。

## 三、问题分析

### 3.1 当前痛点

| 问题 | 说明 |
|------|------|
| 不知道仓库满不满 | 某天某个仓库安排了多少柜子，剩余产能看不清 |
| 不知道车队有没有档期 | 某天某车队可安排多少车次，看不到 |
| 无法发现冲突 | 两个柜子同天同仓库卸柜，无法提前预警 |
| 无法做费用平衡 | 滞港费 vs 堆场费决策没有数据支撑 |

### 3.2 核心需求

> **每个格子内显示**：该资源当日已占用 / 总容量

## 四、方案设计

### 4.1 界面设计

在现有五节点泳道基础上，在**卸柜节点格子内嵌入资源占用统计**：

```
【卸柜】节点行
│
├─ 2026-03-15 列
│   ├─ [●] Container A
│   ├─ [●] Container B
│   ├─ ─────────────────
│   └─ 📊 WH01仓: 8/10 (余2)   ← 资源占用统计
│
├─ 2026-03-16 列
│   ├─ [●] Container C
│   ├─ [●] Container D
│   ├─ [●] Container E
│   ├─ ─────────────────
│   └─ 📊 WH01仓: 10/10 (满)   ← 红色预警
│
└─ ...
```

### 4.2 工具栏开关

添加「资源占用」开关，控制是否显示资源统计：

```vue
<el-switch
  v-model="showResourceStats"
  active-text="资源占用"
  @change="loadWarehouseOccupancy"
/>
```

### 4.3 格子状态样式

| 状态 | 条件 | 样式 |
|------|------|------|
| 正常 | `占用 < 容量 * 80%` | 绿色文字 `#67c23a` |
| 预警 | `容量 * 80% <= 占用 < 容量` | 橙色文字 `#e6a23c` |
| 满负荷 | `占用 >= 容量` | 红色文字 `#f56c6c` |

### 4.4 显示格式

```
WH01仓: 8/10 (余2)
│     │  │  │   │
│     │  │  │   └── 剩余数量
│     │  │  └────── 容量
│     │  └───────── 已占用
│     └──────────── 仓库代码
```

## 九、核心原则

### 卸柜日为核心锚点

```
         ←────────── 向前（依赖卸柜日） ──────────→          ←── 向后（依赖卸柜日）
         
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  清关   │ →  │  提柜   │ →  │  送仓   │ →  │  卸柜   │ →  │  还箱   │
│ customs │    │ pickup  │    │delivery │    │ unload  │    │ return │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     ↑              ↑              ↑              ★              ↑
     │              │              │         【核心锚点】         │
     │              │              │              │              │
  清关日         提柜日         送仓日         卸柜日          还箱日
```

**核心逻辑**：
1. 先确定卸柜日（找第一个有产能的日期）
2. 根据卸柜日确定卸柜方式（Live load / Drop off）
3. 向前计算送仓日、提柜日
4. 向后计算还箱日

甘特图中「按仓库」泳道应按 `plannedUnloadDate` 落格，资源占用统计同理。
