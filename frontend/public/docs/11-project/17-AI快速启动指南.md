## 方案二：切换到 Ollama Local（推荐开发环境）

### 步骤 1: 安装 Ollama

**macOS**:

```bash
brew install ollama
```

**Windows**:

1. 访问 https://ollama.ai
2. 下载 Windows 安装包
3. 运行安装程序

**Linux**:

```bash
curl -fsSL https://ollama.ai/install.sh | sh
```

### 步骤 2: 拉取模型

```bash
# 推荐模型（轻量级，适合本地运行）
ollama pull llama3.2

# 其他可选模型
ollama pull mistral      # Mistral 7B
ollama pull qwen2.5      # Qwen 2.5
ollama pull deepseek-r1  # DeepSeek R1
```

### 步骤 3: 启动 Ollama 服务

```bash
# 终端 1: 启动 Ollama 服务
ollama serve

# 终端 2: 验证服务
curl http://localhost:11434/api/tags
```

预期输出:

```json
{
  "models": [
    {
      "name": "llama3.2:latest",
      "modified_at": "2026-04-08T10:00:00.000000000Z",
      "size": 2019000000,
      "digest": "abc123..."
    }
  ]
}
```

### 步骤 4: 更新 .env 配置

编辑 `backend/.env` 文件：

```bash
# 切换提供商
AI_PROVIDER=ollama-local

# 模型配置（保持默认即可）
OLLAMA_MODEL=llama3.2
OLLAMA_LOCAL_BASE_URL=http://localhost:11434
```

### 步骤 5: 重启后端服务

```bash
# 停止当前服务
# Ctrl+C

# 重新启动
npm run dev
```

### 步骤 6: 验证配置

访问 http://localhost:5173/#/ai-chat，点击右上角状态标签，应该显示"Ollama本地"。

## 方案三：使用 Ollama Cloud

### 步骤 1: 注册 Ollama Cloud

1. 访问 https://ollama.ai/cloud
2. 注册账号
3. 获取 API Key

### 步骤 2: 更新 .env 配置

编辑 `backend/.env` 文件：

```bash
# 切换提供商
AI_PROVIDER=ollama-cloud

# 填入 API Key
OLLAMA_CLOUD_API_KEY=your_api_key_here
OLLAMA_CLOUD_MODEL=llama3.2
```

### 步骤 3: 重启后端服务

```bash
npm run dev
```

## 在前端切换提供商

无需修改配置文件，可以直接在 UI 中切换：

1. 访问 http://localhost:5173/#/ai-chat
2. 点击右上角的状态标签（如"硅基流动"）
3. 在下拉菜单中选择目标提供商

**注意**:

- 只能切换到"已配置"的提供商
- Ollama Local 需要先安装并启动 Ollama 服务
- Ollama Cloud 需要配置 API Key

## 故障排查

### 问题 1: Ollama Local 连接失败

**症状**: 健康检查返回 `missing_config` 或 `error`

**解决方案**:

```bash
# 1. 检查 Ollama 是否运行
ollama list

# 2. 检查服务端口
curl http://localhost:11434/api/tags

# 3. 如果未运行，启动服务
ollama serve

# 4. 检查防火墙
# Windows: 允许 11434 端口
# macOS: 系统偏好设置 -> 安全性与隐私 -> 防火墙
```

### 问题 2: 模型不存在

**症状**: 调用 API 时返回 "model not found"

**解决方案**:

```bash
# 查看已安装的模型
ollama list

# 如果模型不存在，重新拉取
ollama pull llama3.2
```

### 问题 3: 响应速度慢

**原因**: Ollama Local 首次加载模型较慢

**解决方案**:

- 等待模型加载完成（首次可能需要 10-30 秒）
- 后续请求会快很多
- 考虑使用更小的模型（如 llama3.2:1b）

## 环境变量说明

### 通用配置

```bash
# 选择 AI 提供商
AI_PROVIDER=siliconflow  # siliconflow | ollama-local | ollama-cloud
```

### SiliconFlow 配置

```bash
SILICON_FLOW_API_KEY=sk-xxxxx          # API Key（必填）
SILICON_FLOW_MODEL=deepseek-ai/...     # 模型名称
SILICON_FLOW_BASE_URL=https://...      # API 地址（可选）
SILICON_FLOW_TEMPERATURE=0.7           # 温度参数（0-1）
SILICON_FLOW_MAX_TOKENS=4096           # 最大 Token 数
SILICON_FLOW_TIMEOUT=60000             # 超时时间（毫秒）
```

### Ollama Local 配置

```bash
OLLAMA_MODEL=llama3.2                  # 模型名称
OLLAMA_LOCAL_BASE_URL=http://localhost:11434  # 服务地址
OLLAMA_TEMPERATURE=0.7                 # 温度参数
OLLAMA_MAX_TOKENS=4096                 # 最大 Token 数
OLLAMA_TIMEOUT=60000                   # 超时时间
```

### Ollama Cloud 配置

```bash
OLLAMA_CLOUD_API_KEY=xxxxx             # API Key（必填）
OLLAMA_CLOUD_MODEL=llama3.2            # 模型名称
OLLAMA_CLOUD_BASE_URL=https://ollama.ai/api  # API 地址
OLLAMA_CLOUD_TEMPERATURE=0.7           # 温度参数
OLLAMA_CLOUD_MAX_TOKENS=4096           # 最大 Token 数
OLLAMA_CLOUD_TIMEOUT=60000             # 超时时间
```
