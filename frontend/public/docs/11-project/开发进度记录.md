#### 3. 字段混淆修复
**问题**：计划提柜日字段映射错误，使用了错误的数据库字段
**解决方案**：
- 明确字段定义：`process_trucking_transport.planned_pickup_date` 为计划提柜日
- 修复读取逻辑：从`truckings[0].plannedPickupDate`读取计划提柜日
- 区分字段：`last_pickup_date`（拖卡表的最晚提柜日）≠ `planned_pickup_date`（计划提柜日）

**涉及文件**：
- `backend/src/services/demurrage.service.ts` (第462-466行)

#### 4. 前端错误提示优化
**问题**：滞港费计算失败时提示不友好，只显示"计算失败"
**解决方案**：
- 创建分类错误提示配置（`ERROR_MESSAGES`）
- 根据`reason`字段显示不同的错误提示
- 每个错误包含：图标、标题、消息、建议
- 系统错误根据HTTP状态码和网络错误类型显示具体提示

**错误提示分类**：
| reason | 图标 | 标题 | 说明 |
|--------|------|------|------|
| `no_arrival_at_dest` | 🚢 | 货柜未实际到港 | 未到港时不计算 |
| `missing_arrival_dates` | 📅 | 缺少到港日期 | 缺少ATA/ETA/卸船日 |
| `missing_dates` | 📋 | 计划提柜日未维护 | 预测模式下缺少计划提柜日 |
| `no_matching_standards` | 🔍 | 未匹配到滞港费标准 | 未找到匹配的标准 |

**系统错误分类**：
| 错误类型 | 标题 | 建议 |
|---------|------|------|
| 网络连接失败 | 网络连接失败 | 请检查网络连接后重试 |
| 服务器错误(5xx) | 服务器内部错误 | 服务器暂时不可用，请稍后重试或联系系统管理员 |
| 404 | 请求的资源不存在 | 请检查货柜编号是否正确 |
| 401 | 未授权 | 请重新登录后重试 |
| 超时 | 请求超时 | 请求响应超时，请稍后重试 |

**涉及文件**：
- `frontend/src/components/demurrage/DemurrageDetailSection.vue` (第22-106行, 第145-182行, 第302-344行)

### 📝 核心技术要点

#### 1. 计划提柜日字段说明
- **字段位置**：`process_trucking_transport.planned_pickup_date`
- **数据来源**：
  - 智能调度计划：系统自动计算生成（未来功能）
  - 手工录入：用户手动维护
- **用途**：
  - 预测模式的前置条件检查
  - 预测模式下计算最晚还箱日

#### 2. 最晚还箱日计算模式区分
```typescript
if (calculationMode === 'actual') {
  pickupBasisForDetention = params.calculationDates.pickupDateActual ?? null;
} else {
  pickupBasisForDetention = params.calculationDates.plannedPickupDate ?? null;
}
```

#### 3. 滞港费计算预测模式规则
- **实际到港**：正常计算滞港费
- **未实际到港 + 有计划提柜日**：基于ETA和计划提柜日预测计算
- **未实际到港 + 无计划提柜日**：不计算，返回提示
