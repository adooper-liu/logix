# 飞驼数据对接说明与验证

> 说明飞驼数据与主服务货柜业务数据的对接架构、数据流向、验证方式，以及如何向外界证明数据无缝对接。

---

## 一、数据对接架构

### 1.1 两条接入路径

| 路径 | 入口 | 中间层 | 落库表 | data_source |
|------|------|--------|--------|-------------|
| **API 同步** | FeiTuoAdapter | ExternalDataController → applyFeiTuoToCoreFields | ext_container_status_events + process_* | `FeituoAPI` |
| **Excel 导入** | FeituoImportService | mergeTable1ToCore / mergeTable2ToCore | ext_feituo_import_* → ext_container_status_events + biz_* + process_* | `Feituo` |

### 1.2 中间层（适配器）

- **FeiTuoAdapter**（`backend/src/adapters/FeiTuoAdapter.ts`）
  - 调用飞驼官方 API：`POST /application/v1/query`
  - 将飞驼节点格式转换为标准状态事件
  - 返回时 `dataSource: 'FeituoAPI'`（与 Excel 导入的 `Feituo` 区分）

- **ExternalDataController**
  - 接收同步请求，调用 FeiTuoAdapter
  - 写入 `ext_container_status_events`（`data_source = 'FeituoAPI'`）
  - 通过 `FeiTuoStatusMapping` 更新核心字段：`process_port_operations.ata_dest_port`、`gate_out_time`、`process_sea_freight.shipment_date`、`process_empty_return.return_time` 等
  - 触发 `calculateLogisticsStatus` 重算 `biz_containers.logistics_status`

### 1.3 数据落库与追溯

| 表 | 飞驼相关字段 | 说明 |
|----|--------------|------|
| ext_container_status_events | status_code, status_name, occurred_at, location, **data_source** | 原始状态事件，`data_source` 标识来源 |
| process_port_operations | ata_dest_port, gate_out_time, last_free_date, **data_source** | 港口操作，可追溯是否由飞驼更新 |
| process_sea_freight | shipment_date | 装船日期（LOBD 映射） |
| process_empty_return | return_time | 还空箱时间（RCVE 映射） |
| biz_containers | logistics_status, current_status_desc_cn | 由状态机重算 |

## 二、API 与 Excel 的数据源区分

**当前实现**：API 同步标记为 `FeituoAPI`，Excel 导入保持 `Feituo`，便于追溯与审计。

| 来源 | data_source | 前端展示 |
|------|-------------|----------|
| 飞驼 API 同步 | `FeituoAPI` | 显示为「API」 |
| 飞驼 Excel 导入 | `Feituo` | 显示为「Excel」 |

---

## 三、如何证明数据无缝对接且无错误

### 3.1 技术证明

1. **数据流可追溯**
   - `ext_container_status_events.data_source` 记录来源
   - `process_port_operations.data_source` 记录港口操作来源
   - 核心字段（ata_dest_port、shipment_date 等）由 `FeiTuoStatusMapping` 统一映射

2. **状态机一致性**
   - 飞驼事件写入后触发 `calculateLogisticsStatus` 重算
   - 货柜列表与详情的 `logistics_status` 与状态事件一致

3. **去重与覆盖**
   - 同步时按 `container_number + status_code + occurred_at` 去重
   - 核心字段按「实际优先于预计」更新

### 3.2 业务验证

1. **抽样对比**：选取若干货柜，对比飞驼官网/Excel 与 LogiX 展示的节点、时间是否一致
2. **统计校验**：`GET /api/v1/external/stats` 查看各 data_source 事件数、最近更新货柜
3. **前端验证页**：见下文「四、验证界面」

## 四、验证界面

### 4.1 现有界面

| 页面 | 路径 | 功能 | 数据来源展示 |
|------|------|------|--------------|
| 飞驼数据导入 | /import/feituo | API 同步、Excel 导入 | 同步结果有成功/失败 |
| **飞驼数据验证** | **/import/feituo-verify** | **外部数据统计与货柜列表** | **总事件数、按 data_source 分布、已有外部数据的货柜列表（含 API/Excel 标签）** |
| 货柜详情-状态事件 | /shipments/:id（Tab: 状态事件） | 展示状态时间轴 | **不包含** ext_container_status_events，仅来自 process_*、trucking、warehouse、empty_return |
| 货柜详情-物流路径 | /shipments/:id（Tab: 物流路径） | 展示物流路径 | 来自 ext_container_status_events，**节点上展示 data_source 标签（API/Excel）** |

### 4.2 飞驼数据验证页功能

- 调用 `GET /api/v1/external/stats` 展示：总事件数、数据源种类、按 data_source 分布、最近更新货柜
- 调用 `GET /api/v1/external/containers` 展示：已有外部数据的货柜列表，支持按 data_source 筛选（FeituoAPI/Feituo）
- 点击货柜号跳转至货柜详情「物流路径」Tab

### 4.3 待完善

- **货柜详情「状态事件」Tab**：当前 `getContainerStatusEvents` 只聚合 process_*、trucking、warehouse、empty_return，**未包含** `ext_container_status_events` 的原始飞驼事件。

## 五、API 接口速查

| 接口 | 说明 |
|------|------|
| `POST /api/v1/external/sync/:containerNumber` | 同步单个货柜（飞驼 API） |
| `POST /api/v1/external/sync/batch` | 批量同步 |
| `GET /api/v1/external/events/:containerNumber` | 获取货柜的 ext_container_status_events（含 data_source） |
| `GET /api/v1/external/stats` | 数据源统计、最近更新货柜 |
| `GET /api/v1/external/containers?dataSource=&page=&pageSize=` | 已有外部数据的货柜列表（验证页用） |
| `POST /api/v1/import/feituo-excel` | 飞驼 Excel 导入 |

## 六、总结

| 问题 | 答案 |
|------|------|
| 飞驼与主服务是否打通？ | **是**。API 与 Excel 两条路径均写入 ext_container_status_events 并更新核心业务表。 |
| 如何证明无缝对接？ | 通过 data_source 追溯、状态机重算、抽样对比、统计接口、验证页。 |
| 是否有验证界面？ | **是**。`/import/feituo-verify` 飞驼数据验证页，展示统计、按 data_source 分布、货柜列表。 |
| 哪些货柜有飞驼数据？ | 验证页或 `GET /api/v1/external/containers` 按 data_source 筛选。 |
| 是否应标记为「API」？ | **已实现**：API 同步用 `FeituoAPI`（前端展示「API」），Excel 用 `Feituo`（展示「Excel」）。 |
