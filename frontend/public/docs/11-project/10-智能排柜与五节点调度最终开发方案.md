## 一、综合架构总览

### 1.1 核心架构澄清

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  输入：货柜 + ETA + last_free_date + 备货单（港口/目标仓） + 映射 + 约束      │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  智能排柜引擎（中间环节）                                                      │
│  - 映射关系：港口→车队、仓库→车队（dict_trucking_port_mapping、               │
│              dict_warehouse_trucking_mapping）                                │
│  - 车队能力：日容量、港口-仓库路线                                            │
│  - 仓库能力：日产能、卸柜方式（Drop off / Live loading）                      │
│  - ETA / last_free_date / last_return_date                                   │
│  - 三种方案决策：直接送仓 / 先提送堆场 / 滞港等仓 → 总费用最少                 │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  输出：分派的车队、仓库、五节点计划时间（planned_*）                           │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  流程表（存储结果）                                                           │
│  process_trucking_transport（trucking_company_id、planned_pickup_date 等）  │
│  process_warehouse_operations（warehouse_id、planned_unload_date 等）        │
│  process_empty_return（planned_return_date）                                 │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  甘特图、列表、统计、执行跟踪                                                 │
└─────────────────────────────────────────────────────────────────────────────┘
```

**关键约定**：流程表中的车队、仓库是**智能排柜引擎分派后的结果**，引擎是中间分派环节，流程表仅存储结果。

## 二、五节点定义与约束（来自 04）

### 2.1 五节点

| 节点 | 英文 | 计划字段 | 实际字段 | 主表 |
|------|------|----------|----------|------|
| 清关 | Customs | planned_customs_date | actual_customs_date | process_port_operations |
| 拖卡 | Trucking | planned_pickup_date, planned_delivery_date | pickup_date, delivery_date | process_trucking_transport |
| 卸柜 | Unload | planned_unload_date | unload_date | process_warehouse_operations |
| 还箱 | Empty Return | planned_return_date | return_time | process_empty_return |
| 查验 | Inspection | inspection_planned_date（扩展） | 查验记录（扩展） | ext_inspection_records |

### 2.2 节点顺序与依赖

```
到港(ATA) → 清关(放行) → 拖卡(提柜→送仓) → 卸柜(仓库卸货) → 还箱(空箱还回)
                ↑
            查验(可能插入清关前后)
```

### 2.3 三种方案决策（总费用最少）

| 方案 | 说明 | 费用项 |
|------|------|--------|
| **直接送仓** | 按仓库能力排期，直接提柜送仓卸柜还箱 | 拖车 + 卸柜；若无能力则顺延产生滞港费 |
| **先提送堆场** | 先提柜至第三堆场暂存，等仓有能力再送仓 | 港→堆场拖车 + 堆场费×N + 堆场→仓拖车 + 卸柜 + 风险成本 |
| **滞港等仓** | 在港口等 N 天，待仓有能力再提柜直接送仓 | 滞港费×N + 拖车 + 卸柜 |

比较总费用，选最小方案；滞港与堆场费相差不大时优先滞港。

### 2.4 资源约束

| 资源 | 约束对象 | 数据/配置 |
|------|----------|-----------|
| 港口免费期 last_free_date | 计划提柜日 | 计算得出 |
| 免费用箱期 last_return_date | 计划还箱日 | 计算得出 |
| 仓库日产能 | 计划卸柜日 | ext_warehouse_unload_capacity, ext_warehouse_daily_occupancy |
| 卸柜方式 Drop/Live | 送仓/卸柜/还箱节奏 | unload_mode_plan |
| 拖车日容量 | 提柜/送仓日 | ext_trucking_slot_occupancy + 港口-仓库路线 |
| 第三堆场 | 方案二决策 | dict_yards, ext_yard_capacity_fee |

## 三、智能排柜引擎（来自 05、06）

### 3.1 阶段一：规则引擎（先到先得 + 约束校验）

**流程**：
1. 过滤：排除 schedule_status ≠ initial
2. 排序：按清关可放行日(ATA/ETA)升序，同日内按 last_free_date 升序
3. 对每柜：
   - 按**备货单港口-仓库关系**确定候选仓库（港口→车队、仓库→车队映射推导）
   - 计划清关日 = 清关可放行日
   - 计划提柜日 = 计划清关日 + N 天，且 ≤ last_free_date
   - 计划卸柜日 = findEarliestAvailableDay（仓库日产能）
   - 计划送仓日 = Live 时=卸柜日；Drop 时=卸柜日或前一日
   - 计划还箱日 = Live 时=卸柜日；Drop 时=卸柜日+1～2天，且 ≤ last_return_date
   - 车队选择：按 仓库×港口 定价表，选最低价且有剩余能力的车队
   - 写回 planned_* 字段；扣减仓库日产能、拖车档期

### 3.2 阶段二：三种方案决策

仓库满负荷时，对三种方案分别估算总费用，选最小方案并生成计划。

### 3.3 阶段三：智能优化（可选）

启发式、整数规划、遗传/模拟退火，多目标优化。

### 3.4 触发方式

| 方式 | 说明 |
|------|------|
| 手动一键排产 | 甘特/列表页点击「一键排产」，选择日期范围、国家 |
| 导入后自动排产 | Excel 导入后对 schedule_status=initial 的柜自动排产 |
| 定时任务 | 定时对未排产柜执行排产 |

### 3.5 覆盖规则

| schedule_status | 智能排产 | 手工拖拽 |
|-----------------|----------|----------|
| initial | ✅ 可覆盖 | ✅ 可调整；调整后→adjusted |
| issued | ❌ 不可覆盖 | 可配置 |
| adjusted | ❌ 不可覆盖 | ✅ 可继续调整 |

## 四、甘特图与资源（来自 07）

### 4.1 泳道类型

| 泳道 | 货柜落格规则 | 数据来源 |
|------|--------------|----------|
| 按到港 | ETA/修正 ETA/ATA | process_port_operations |
| 按提柜计划 | planned_pickup_date | process_trucking_transport |
| 按最晚提柜 | last_free_date | process_port_operations |
| 按最晚还箱 | last_return_date | process_empty_return |
| 按计划清关 | planned_customs_date | process_port_operations |
| 按计划卸柜 | planned_unload_date | process_warehouse_operations |
| 资源泳道 | 车队/仓库/堆场 × 日期 | 按占用日期落格 |

### 4.2 三池评估

- **价值**：流程可视化、调度直观
- **劣势**：空间割裂、时间轴弱化、实现复杂
- **建议**：优先做强泳道甘特，再视需求引入三池或单资源池

### 4.3 手工拖拽

- 校验：last_free_date、last_return_date、仓库日产能、拖车档期、顺序约束
- 更新：planned_*、资源占用、schedule_status=adjusted
- API：PATCH /containers/:id/schedule

### 4.4 实体与虚拟任务

| 类型 | 说明 | 可操作性 |
|------|------|----------|
| 实体任务 | 有持久化数据，对应 process_* | 可拖拽、可编辑 |
| 虚拟占位 | 由主任务派生，无独立持久化 | 不可单独拖拽；随主任务联动 |

## 七、分阶段实施路径（最终）

> **实施顺序**：先**手工排柜**，再**智能排柜**。手工排柜建立数据流与校验，智能排柜在此基础上自动化。

### 阶段 P1：五节点可视化与基础能力（约 2–3 周）

| 序号 | 任务 | 产出 |
|------|------|------|
| P1-1 | 甘特图扩展泳道 | 按计划清关、按计划卸柜 |
| P1-2 | 五节点统一时间线 | 货柜详情内计划 vs 实际 |
| P1-3 | schedule_status 字段 | initial / issued / adjusted |
| P1-4 | 仓库能力与卸柜方式 | ext_warehouse_unload_capacity、unload_mode 逻辑 |
| P1-5 | 第三堆场参数表 | dict_yards / ext_yard_capacity_fee |

### 阶段 P2：手工排柜（约 2–3 周）— 先实现

| 序号 | 任务 | 产出 |
|------|------|------|
| P2-1 | 计划编辑表单/详情 | 货柜详情或弹窗内编辑 车队、仓库、计划时间（planned_*） |
| P2-2 | 甘特图手工拖拽 | 拖拽货柜至车队/仓库泳道 × 日期格子，落点校验 |
| P2-3 | PATCH schedule API | 更新单柜计划，校验 last_free_date、last_return_date、顺序、映射关系 |
| P2-4 | 约束校验 | 前端 + 后端校验：港口→车队、仓库→车队 映射；顺序约束 |
| P2-5 | 覆盖规则 | schedule_status=adjusted 时记录手工调整 |

### 阶段 P3：智能排柜引擎（约 2–3 周）— 后实现

| 序号 | 任务 | 产出 |
|------|------|------|
| P3-1 | 资源占用表 | ext_warehouse_daily_occupancy、ext_trucking_slot_occupancy、ext_yard_daily_occupancy |
| P3-2 | 规则引擎（阶段一） | 先到先得 + 约束校验，生成五节点计划时间 |
| P3-3 | 三种方案决策（阶段二） | 直接送仓 / 先提送堆场 / 滞港等仓，总费用最少 |
| P3-4 | 智能排柜 API | POST /containers/batch-schedule、schedule-preview |
| P3-5 | 一键排产入口 | 甘特/列表页「一键排产」，仅覆盖 schedule_status=initial |

### 阶段 P4：资源泳道与扩展（按需）

| 序号 | 任务 | 产出 |
|------|------|------|
| P4-1 | 资源泳道 | 车队、仓库、堆场泳道 + 占用数展示 |
| P4-2 | 智能优化（阶段三） | 启发式/整数规划/遗传算法（可选） |
| P4-3 | 查验全过程 | 查验记录、状态、费用 |
| P4-4 | 其他船公司适配器 | MSC、Maersk、COSCO、CustomApi |

## 八、依赖关系

```
P0 滞港费闭环（已完成）
  └─ last_free_date、last_return_date 计算

P1 五节点可视化
  └─ 依赖现有 process_* 表计划字段
  └─ 仓库能力、第三堆场为 P2 前置

P2 手工排柜（先实现）
  └─ 依赖 P1（甘特图、五节点时间线、schedule_status）
  └─ 依赖 dict_trucking_port_mapping、dict_warehouse_trucking_mapping（校验用）
  └─ 产出：计划编辑、拖拽、PATCH API、约束校验

P3 智能排柜（后实现）
  └─ 依赖 P0（last_free_date、滞港费标准）
  └─ 依赖 P2（手工排柜已建立数据流与校验，智能排产复用）
  └─ 依赖 P1（仓库能力、第三堆场、schedule_status）
  └─ 产出：规则引擎、batch-schedule、一键排产

P4 资源泳道
  └─ 依赖 P3 资源占用表
```
