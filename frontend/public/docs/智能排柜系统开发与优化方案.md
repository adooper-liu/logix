### 3.1 后端性能优化

#### 方案 1: Redis 缓存策略

**问题**: 重复查询 mapping 表和档期数据

**解决方案**:
```typescript
// 1. 添加缓存层
import { Redis } from 'ioredis';

const redis = new Redis({
  host: 'localhost',
  port: 6379,
  keyPrefix: 'logix:scheduling:'
});

// 2. 缓存 mapping 数据（5 分钟）
async function getCachedMapping(key: string) {
  const cached = await redis.get(key);
  if (cached) return JSON.parse(cached);
  
  // 查询数据库
  const data = await queryDatabase(key);
  await redis.setex(key, 300, JSON.stringify(data)); // 5 分钟过期
  return data;
}

// 3. 使用示例
const truckingMappings = await getCachedMapping(
  `mapping:port:${portCode}:country:${countryCode}`
);
```

**预期收益**:
- Mapping 查询减少 90%
- 批量排产速度提升 40%

**实现步骤**:
1. 安装 Redis: `npm install ioredis @types/ioredis`
2. 添加缓存工具类
3. 在以下方法中集成缓存:
   - `getCandidateWarehouses()`
   - `selectTruckingCompany()`
   - `findEarliestAvailableWarehouse()`

```typescript
// 1. 预加载所有 mapping 数据
const uniquePortCountryPairs = getUniquePairs(containers);

// 2. 内存分组
const mappingsByPair = groupBy(allMappings, pair => `${pair.port}:${pair.country}`);

// 3. 使用缓存数据
for (const container of containers) {

  const warehouses = mappingsByPair[key];
  // ...

```

**预期收益**:

### 3.2 代码质量改进

#### 改进 1: 函数重构（降低复杂度）

**问题函数**: `scheduleSingleContainer()` (180 行，复杂度过高)

**重构方案**:
```typescript
// 拆分为小函数
private async scheduleSingleContainer(container: Container): Promise<ScheduleResult> {
  // 1. 验证基础数据
  const validation = await this.validateContainerData(container);
  if (!validation.valid) return validation.error;

  // 2. 计算计划日期
  const dates = await this.calculatePlannedDates(container);
  
  // 3. 选择资源
  const resources = await this.selectResources(container, dates);
  if (!resources.valid) return resources.error;

  // 4. 确定卸柜方式
  const unloadMode = this.determineUnloadMode(resources.truckingCompany);

  // 5. 调整日期（如需）
  const adjustedDates = this.adjustDatesIfNeeded(dates, unloadMode, resources.warehouse);

  // 6. 写入数据库
  await this.persistSchedule(container, adjustedDates, resources, unloadMode);

  // 7. 扣减档期
  await this.decrementOccupancies(resources, adjustedDates, unloadMode);

  return this.buildSuccessResult(container, adjustedDates, resources, unloadMode);
}
```

**优点**:
- 单一职责原则
- 易于测试和维护
- 降低认知负担

#### 改进 2: 错误处理增强

**当前问题**: 错误信息不统一，缺少分类

**改进方案**:
```typescript
// 1. 定义错误类型
export enum SchedulingErrorCode {
  NO_PORT_OPERATION = 'NO_PORT_OPERATION',
  NO_ETA_ATA = 'NO_ETA_ATA',
  NO_WAREHOUSE_MAPPING = 'NO_WAREHOUSE_MAPPING',
  WAREHOUSE_CAPACITY_EXCEEDED = 'WAREHOUSE_CAPACITY_EXCEEDED',
  NO_TRUCKING_MAPPING = 'NO_TRUCKING_MAPPING',
  TRUCKING_CAPACITY_EXCEEDED = 'TRUCKING_CAPACITY_EXCEEDED'
}

// 2. 统一错误结构
export class SchedulingError extends Error {
  constructor(
    public code: SchedulingErrorCode,
    message: string,
    public context?: Record<string, any>
  ) {
    super(message);
  }
}

// 3. 使用示例
if (warehouses.length === 0) {
  throw new SchedulingError(
    SchedulingErrorCode.NO_WAREHOUSE_MAPPING,
    '无映射关系中的仓库',
    { countryCode, portCode }
  );
}
```

**优点**:
- 错误分类清晰
- 便于前端国际化

```typescript
describe('IntelligentSchedulingService', () => {
  describe('Live load Mode', () => {
    it('should schedule with Live load when has_yard=false', async () => {
      // Arrange
      const container = createTestContainer({ eta: '2026-03-20' });
      const trucking = createTestTrucking({ hasYard: false });

      const result = await service.scheduleSingleContainer(container);

      expect(result.success).toBe(true);
      expect(result.plannedData.unloadModePlan).toBe('Live load');
      expect(result.plannedData.plannedPickupDate)

    });
  });

  describe('Drop off Mode', () => {
    it('should schedule with Drop off when has_yard=true', async () => {
      // Arrange
      const container = createTestContainer({ eta: '2026-03-20' });
      const trucking = createTestTrucking({ hasYard: true });
      
      // Act
      const result = await service.scheduleSingleContainer(container);
      
      // Assert
      expect(result.success).toBe(true);
      expect(result.plannedData.unloadModePlan).toBe('Drop off');
```
