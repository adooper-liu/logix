# 业务流程

LogiX 核心业务流程说明。

## 7层流转

```
第1层: 备货单 (biz_replenishment_orders)
   ↓ 创建货柜
第2层: 货柜 (biz_containers)
   ↓ 安排出运
第3层: 海运 (process_sea_freight)
   ↓ 港口操作
第4层: 港口 (process_port_operations)
   ↓ 拖卡运输
第5层: 拖卡 (process_trucking_transport)
   ↓ 仓库操作
第6层: 仓库 (process_warehouse_operations)
   ↓ 还空箱
第7层: 还空箱 (process_empty_return)
```

## 货柜生命周期

### 1. 创建备货单

```
用户创建备货单
  ↓
系统生成 order_number
  ↓
添加货柜信息
  ↓
状态: not_shipped
```

### 2. 出运

```
备货单确认出运
  ↓
创建海运记录
  ↓
更新货柜状态: shipped
  ↓
同步飞驼数据
```

### 3. 在途

```
获取物流节点更新
  ↓
更新港口操作记录
  ↓
状态流转: in_transit → at_port
  ↓
计算 ETA
```

### 4. 到港

```
实际到港 (ATA)
  ↓
更新港口操作
  ↓
状态: at_port
  ↓
计算滞港费
```

### 5. 提柜

```
安排拖卡运输
  ↓
更新拖卡记录
  ↓
更新计划提柜日
  ↓
状态: picked_up
```

### 6. 卸柜

```
仓库卸柜
  ↓
更新仓库操作
  ↓
状态: unloaded
  ↓
计算最晚还箱日
```

### 7. 还箱

```
安排还箱
  ↓
更新还空箱记录
  ↓
状态: returned_empty
```

## 数据导入流程

### Excel 导入

```
1. 用户上传 Excel 文件
2. 解析 Excel（使用 xlsx 库）
3. 验证必填字段
4. 映射到实体字段
5. 写入数据库
6. 返回导入结果
```

### 飞驼数据同步

```
1. 定时任务触发（或手动）
2. 调用飞驼 API 获取数据
3. 解析响应数据
4. 更新港口操作记录
5. 触发状态机重新计算
```

## 统计查询流程

### 按到港统计

```
1. 接收日期范围参数
2. 构建 SQL 子查询
3. 按条件分组统计
4. 返回统计结果
```

### 分类逻辑

```
已到目的港
├── 今日到港
├── 之前未提柜
└── 之前已提柜

已到中转港
├── ETA 逾期
├── ETA 3日内
├── ETA 7日内
└── ETA 7日后

预计到港（无中转）
├── ETA 逾期
├── ETA 3日内
├── ETA 7日内
└── ETA 7日后
```

## 滞港费计算流程

### 计算条件

```
1. 匹配标准
   - 进口国
   - 目的港
   - 船公司
   - 货代
   - 有效期内

2. 计算免费天数
   - 按到港: 从 ATA 开始
   - 按卸船: 从卸船日开始

3. 计算费用
   - 超期天数 × 日费率
   - 多行费用项合计
```

### 计算模式

```
actual 模式: 使用实际日期计算
forecast 模式: 使用计划日期预测
```

## 智能排产流程

### 锚点逻辑

```
以卸柜日为核心锚点
  ↓
根据仓库可用性确定锚点日期
  ↓
向前推导:
  - 计划还箱日
  - 计划送仓日
  - 计划提柜日
  - 计划清关日
  ↓
向后推导:
  - 计划卸柜日
```

### 约束条件

```
- 仓库可用性
- 日产能限制
- 运输天数
- 清关时间
- 节假日
```
