# 滞港费合计与预警可视化方案

> 结合行业最佳实践，为全部货柜滞港费用合计及预警提供可视化设计。  
**数据口径**：与 Shipments 页一致，按**出运日期**（`actual_ship_date` / `shipment_date`）筛选货柜子集。

---

## 一、行业最佳实践概览

| 实践 | 说明 |
|------|------|
| **KPI 先行** | 总费用、柜数、单柜均值等核心指标置于顶部，一眼可见 |
| **多维度拆解** | 按目的港、船公司、货代、时间等维度拆解，便于定位高成本来源 |
| **预警分级** | 红/黄/绿分级，超阈值自动突出；支持可配置阈值 |
| **钻取联动** | 卡片/图表点击可下钻到货柜列表，与现有 Shipments 过滤联动 |
| **趋势对比** | 月度/周度趋势、同比环比，支持成本预测 |
| **Top N 清单** | 高费用货柜清单，便于优先处理 |

---

## 二、数据层设计

### 2.1 新增 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `GET /api/v1/demurrage/summary` | 汇总统计 | 按日期范围返回：总费用、柜数、单柜均值、分项汇总 |
| `GET /api/v1/demurrage/summary/by-dimension` | 维度拆解 | 按目的港/船公司/货代等维度聚合费用 |
| `GET /api/v1/demurrage/top-containers` | Top 货柜 | 费用最高的 N 个货柜，支持分页 |
| `GET /api/v1/demurrage/alerts` | 预警列表 | 超阈值货柜、即将超期货柜等 |

**请求参数**（与 `statistics-detailed` 一致）：
- `startDate`、`endDate`：出运日期范围
- `dimension`（by-dimension）：`destination_port` | `shipping_company` | `origin_forwarder` | `charge_type`

### 2.2 汇总计算逻辑

- **货柜子集**：与 `getStatisticsDetailed` 同口径，按出运日期筛选
- **单柜费用**：调用 `DemurrageService.calculateForContainer(containerNumber)` 得到 `totalAmount`
- **批量策略**：可先获取子集柜号列表，再并行/分批调用计算，最后聚合；或引入缓存/预计算表以提升性能

### 2.3 预警规则（可配置）

| 规则类型 | 说明 | 默认阈值 |
|----------|------|----------|
| **总费用超限** | 日期范围内滞港费合计 > 阈值 | 如 $10,000 |
| **单柜费用超限** | 单柜滞港费 > 阈值 | 如 $500 |
| **高费用柜数** | 费用 > 单柜阈值的柜数 > N | 如 10 个 |
| **即将超期** | 复用 `lastPickupDistribution` 的 expired/urgent | 与最晚提柜卡片联动 |
| **无标准匹配** | 有到港/卸船但无滞港费标准匹配的柜数 | 提示配置缺失 |

## 四、核心组件设计

### 4.1 滞港费合计卡片（DemurrageSummaryCard）

| 指标 | 说明 | 展示形式 |
|------|------|----------|
| 总费用 | 日期范围内所有货柜滞港费合计 | 大号数字 + 货币 |
| 产生费用柜数 | 费用 > 0 的货柜数量 | 数字 + 「柜」 |
| 单柜均值 | 总费用 / 产生费用柜数 | 数字 + 货币 |
| 无费用柜数 | 费用 = 0 或无法计算的柜数 | 小字说明（可选） |

**样式**：与 `CountdownCard` 风格一致，使用 `useColors()` 变量；可增加环比/同比小字（若有历史数据）。

### 4.2 预警卡片（DemurrageAlertCard）

| 状态 | 条件 | 视觉 |
|------|------|------|
| 正常 | 无超阈值项 | 绿色边框，`CircleCheck` 图标 |
| 预警 | 有 1 项超阈值 | 黄色边框，`Warning` 图标 |
| 严重 | 总费用或高费用柜数超限 | 红色边框，`Warning` 图标 |

**内容**：
- 摘要：如「2 项需关注」
- 明细：总费用超限 / 单柜超限柜数 / 即将超期柜数（可点击下钻）
- 操作：「查看详情」→ 跳转或展开预警列表

### 4.3 维度拆解图表

- **按目的港**：柱状图，X 轴目的港，Y 轴费用；支持点击筛选该目的港货柜
- **按船公司**：同上
- **按费用类型**：饼图，Demurrage / Detention / Demurrage & Detention 占比
- **按月份**：折线图，展示月度趋势（若数据量足够）

**图表库**：与项目现有一致（如 ECharts / Chart.js），使用 `useColors()` 保证配色统一。

### 4.4 高费用货柜表格

- 列：柜号、目的港、船公司、费用(USD)、计费天数、最晚提柜日、状态标签、操作
- 排序：默认按费用降序
- 操作：点击「详情」进入货柜详情页滞港费 Tab
- 与 Shipments 表格风格一致

## 五、预警规则与交互

### 5.1 预警级别

| 级别 | 名称 | 颜色 | 触发条件 |
|------|------|------|----------|
| 0 | 正常 | success | 无超阈值 |
| 1 | 关注 | warning | 单柜超限或即将超期 |
| 2 | 严重 | danger | 总费用超限或高费用柜数超限 |

### 5.2 阈值配置（后续扩展）

- 初期可硬编码默认阈值
- 后续可增加「系统设置」或「滞港费设置」页，支持配置：
  - 总费用阈值（USD）
  - 单柜费用阈值（USD）
  - 高费用柜数阈值（个）

### 5.3 与现有卡片联动

- **按最晚提柜**：`expired` / `urgent` 与滞港费预警重叠，可复用同一批货柜
- 点击「滞港费预警」中的「即将超期」可联动 `activeFilter`，筛选 `expired` 或 `urgent` 货柜

## 七、性能考虑与预计算（已实现）

- **预计算存储**：定时任务（默认每 6 小时，可配置 `DEMURRAGE_WRITEBACK_SCHEDULER_INTERVAL=1440` 实现每日一次）将各柜滞港费写入 `ext_demurrage_records`
- **临时/永久**：未还箱柜为临时数据（`is_final=false`），每次覆盖；已还箱柜为永久数据（`is_final=true`），计算一次后不再更新
- **Summary 接口**：优先从 `ext_demurrage_records` 读取，无记录时回退实时计算
- **手动触发**：`POST /api/v1/demurrage/batch-compute-records` 可立即执行预计算

---

## 八、小结

- **合计**：总费用、柜数、单柜均值，与出运日期范围一致
- **预警**：总费用/单柜/柜数阈值 + 即将超期联动，红黄绿分级
- **可视化**：KPI 卡片 + 维度图表 + Top 货柜表，风格与 Shipments 统一
- **数据层**：新增 summary、top-containers、alerts 接口；考虑预计算或异步以保障性能
