# 监控系统 - 真实数据模式说明

## 📊 概述

LogiX监控系统已升级为真实数据监控模式，不再使用模拟数据。系统从后端API获取实时的系统性能指标、优化数据和告警信息。

## 🔧 技术实现

### 后端实现

#### 1. 监控控制器
- **文件**: `backend/src/controllers/monitoring.controller.ts`
- **功能**: 提供监控数据API接口

#### API端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/monitoring` | GET | 获取综合监控数据 |
| `/monitoring/refresh` | GET | 刷新监控数据 |
| `/monitoring/performance` | GET | 获取性能指标 |
| `/monitoring/optimization` | GET | 获取优化数据 |
| `/monitoring/alerts` | GET | 获取告警信息 |
| `/monitoring/health` | GET | 获取服务健康度 |
| `/monitoring/trend` | GET | 获取性能趋势 |

#### 数据来源

1. **性能指标**
   - CPU使用率: 基于Node.js `process` 模块计算
   - 内存使用率: `process.memoryUsage()` 实时统计
   - 响应时间: 基于请求时间戳计算
   - 吞吐量: 估算每秒请求数

2. **优化数据**
   - 缓存命中率: 从Redis获取（暂用估算）
   - 搜索节省: 防抖节省的请求数统计
   - 慢组件: 前端性能监控上报
   - 加载时间: 页面平均加载时间

3. **服务健康度**
   - API服务: 后端服务状态
   - 数据库: PostgreSQL连接状态
   - Redis缓存: Redis连接状态
   - 飞驼适配器: 外部服务连接状态
   - 物流适配器: 物流服务状态

4. **告警生成**
   - CPU > 80%: 严重告警
   - CPU > 60%: 警告
   - 内存 > 90%: 严重告警
   - 内存 > 75%: 警告
   - 响应时间 > 1000ms: 严重告警
   - 响应时间 > 500ms: 警告
   - 服务健康度 < 80%: 严重告警
   - 服务健康度 < 95%: 警告

## 📈 监控指标说明

### 性能指标

| 指标 | 说明 | 正常范围 | 告警阈值 |
|------|------|----------|----------|
| CPU使用率 | 后端服务器CPU使用率 | < 60% | > 80% |
| 内存使用率 | 后端Node.js进程内存使用率 | < 75% | > 90% |
| 响应时间 | API平均响应时间 | < 300ms | > 500ms |
| 每秒请求数 | 系统吞吐能力 | 取决于业务 | - |

### 优化指标

| 指标 | 说明 | 优秀值 |
|------|------|--------|
| 缓存命中率 | API请求从缓存获取的比例 | > 50% |
| 搜索节省 | 搜索防抖节省的请求数 | 越高越好 |
| 慢组件数量 | 渲染时间 > 50ms的组件数 | 0 |
| 平均加载时间 | 页面平均加载时间 | < 2秒 |

### 服务健康度

| 服务 | 说明 | 健康标准 |
|------|------|----------|
| API服务 | 后端API服务可用性 | 100% |
| 数据库 | PostgreSQL连接状态 | 100% |
| Redis缓存 | Redis连接状态 | > 95% |
| 飞驼适配器 | 外部数据服务连接 | > 95% |
| 物流适配器 | 物流服务状态 | 100% |

## 🚨 告警级别

### 🔴 严重（Error）
需要立即处理的问题：
- CPU使用率 > 80%
- 内存使用率 > 90%
- 响应时间 > 1000ms
- 服务健康度 < 80%

### 🟡 警告（Warning）
需要关注的问题：
- CPU使用率 > 60%
- 内存使用率 > 75%
- 响应时间 > 500ms
- 服务健康度 < 95%

### 🔵 信息（Info）
系统正常运行的提示信息

## 🛠 故障排查

### 问题1: 监控数据不更新

**检查步骤**:
1. 查看浏览器控制台是否有错误
2. 检查后端服务是否正常运行
3. 检查API端点是否可访问
4. 查看网络请求状态码

**解决方法**:
- 重启后端服务: `npm run dev`
- 检查数据库连接
- 清除浏览器缓存

### 问题2: 图表不显示

**检查步骤**:
1. 查看是否有JavaScript错误
2. 检查ECharts是否正确加载
3. 检查DOM元素是否存在

**解决方法**:
- 刷新页面
- 检查网络连接
- 查看浏览器兼容性

### 问题3: 数据异常

**检查步骤**:
1. 查看后端日志
2. 检查数据库连接状态
3. 检查Redis连接状态

**解决方法**:
- 重启数据库服务
- 重启Redis服务
- 检查系统资源使用情况
