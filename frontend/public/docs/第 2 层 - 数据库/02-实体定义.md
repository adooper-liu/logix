# 实体定义

TypeORM 实体与数据库表的映射。

## 权威来源

实体定义的权威来源：`backend/src/entities/*.ts`

## 命名规则

```
数据库: snake_case  →  实体: camelCase
例: container_number  →  containerNumber
```

## Container 实体

文件：`backend/src/entities/Container.ts`
表名：`biz_containers`

```typescript
@Entity('biz_containers')
export class Container {
  @PrimaryColumn({ type: 'varchar', length: 100, name: 'container_number' })
  containerNumber!: string;

  @Column({ type: 'varchar', length: 100, name: 'container_type_code' })
  containerTypeCode!: string;

  @Column({ type: 'text', nullable: true, name: 'cargo_description' })
  cargoDescription!: string;

  @Column({ type: 'decimal', precision: 10, scale: 2, nullable: true, name: 'gross_weight' })
  grossWeight!: number;

  @Column({ type: 'decimal', precision: 10, scale: 2, nullable: true, name: 'net_weight' })
  netWeight!: number;

  @Column({ type: 'decimal', precision: 8, scale: 2, nullable: true, name: 'cbm' })
  cbm!: number;

  @Column({ type: 'int', nullable: true, name: 'packages' })
  packages!: number;

  @Column({ type: 'varchar', length: 100, nullable: true, name: 'seal_number' })
  sealNumber!: string;

  @Column({ type: 'boolean', default: false, name: 'inspection_required' })
  inspectionRequired!: boolean;

  @Column({ type: 'boolean', default: false, name: 'is_unboxing' })
  isUnboxing!: boolean;

  @Column({ type: 'varchar', length: 20, default: 'not_shipped', name: 'logistics_status' })
  logisticsStatus!: string;

  @Column({ type: 'jsonb', nullable: true, name: 'gantt_derived' })
  ganttDerived?: GanttDerivedPayload | null;

  @Column({ type: 'varchar', length: 20, default: 'initial', name: 'schedule_status' })
  scheduleStatus?: 'initial' | 'issued' | 'dispatched' | 'adjusted';

  @Column({ type: 'text', nullable: true, name: 'remarks' })
  remarks!: string;

  @Column({ type: 'varchar', length: 50, nullable: true, name: 'bill_of_lading_number' })
  billOfLadingNumber?: string;

  @CreateDateColumn({ name: 'created_at' })
  createdAt!: Date;

  @UpdateDateColumn({ name: 'updated_at' })
  updatedAt!: Date;
}
```

## SchedulingHistory 实体

```typescript
@Entity('hist_scheduling_records')
export class SchedulingHistory {
  @PrimaryGeneratedColumn({ type: 'integer' })
  id: number;

  @Column({ type: 'varchar', length: 50, name: 'container_number' })
  containerNumber: string;

  @Column({ type: 'int', name: 'scheduling_version', default: 1 })
  schedulingVersion: number;

  @Column({ type: 'varchar', length: 20, name: 'scheduling_mode' })
  schedulingMode: 'MANUAL' | 'AUTO' | 'BATCH';

  @Column({ type: 'varchar', length: 20 })
  strategy: 'Direct' | 'Drop off' | 'Expedited';

  @Column({ type: 'date', nullable: true, name: 'planned_pickup_date' })
  plannedPickupDate?: Date;

  @Column({ type: 'date', nullable: true, name: 'planned_unload_date' })
  plannedUnloadDate?: Date;

  @Column({ type: 'decimal', precision: 12, scale: 2, nullable: true, name: 'total_cost' })
  totalCost?: number;

  @Column({ type: 'varchar', length: 20, default: 'CONFIRMED', name: 'scheduling_status' })
  schedulingStatus: 'CONFIRMED' | 'CANCELLED' | 'SUPERSEDED';

  @CreateDateColumn({ name: 'created_at' })
  createdAt: Date;

  @UpdateDateColumn({ name: 'updated_at' })
  updatedAt: Date;
}
```

## 类型对应

| 数据库类型 | TypeORM 类型 |
|------------|--------------|
| VARCHAR(n) | string (length: n) |
| DECIMAL(p,s) | decimal (precision: p, scale: s) |
| DATE | date |
| TIMESTAMP | timestamp |
| TIMESTAMPTZ | timestamptz |
| INT | int |
| BOOLEAN | boolean |
| TEXT | text |
| JSONB | jsonb |

## 字段映射原则

1. 实体属性使用 camelCase
2. @Column 使用 name 参数指定数据库字段名
3. 主键使用 @PrimaryColumn 或 @PrimaryGeneratedColumn
4. 时间戳使用 @CreateDateColumn / @UpdateDateColumn
5. 关联关系使用 @ManyToOne / @OneToMany / @JoinColumn
