# 表结构

LogiX 数据库表结构速查。

## 权威来源

数据库表结构的权威来源：

- `backend/src/entities/*.ts` - TypeORM 实体定义（第一优先级）
- 数据库实际表结构 - 通过 `\d table_name` 查看

## 核心业务表

### biz_containers (货柜表)

主键：container_number

| 字段 | 类型 | 说明 |
|------|------|------|
| container_number | VARCHAR(100) | 箱号（主键） |
| container_type_code | VARCHAR(100) | 箱型代码 |
| cargo_description | TEXT | 货物描述 |
| gross_weight | DECIMAL(10,2) | 毛重 |
| net_weight | DECIMAL(10,2) | 净重 |
| cbm | DECIMAL(8,2) | 体积 |
| packages | INT | 件数 |
| seal_number | VARCHAR(100) | 封条号 |
| inspection_required | BOOLEAN | 是否查验 |
| is_unboxing | BOOLEAN | 是否拆箱 |
| logistics_status | VARCHAR(20) | 物流状态 |
| schedule_status | VARCHAR(20) | 排产状态 |
| gantt_derived | JSONB | 甘特派生数据 |
| remarks | TEXT | 备注 |
| bill_of_lading_number | VARCHAR(50) | 提单号 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

## 索引

常用索引：

```sql
CREATE INDEX idx_containers_status ON biz_containers(logistics_status);
CREATE INDEX idx_containers_schedule_status ON biz_containers(schedule_status);
CREATE INDEX idx_port_ops_container ON process_port_operations(container_number);
CREATE INDEX idx_port_ops_eta ON process_port_operations(eta);
CREATE INDEX idx_trucking_container ON process_trucking_transport(container_number);
CREATE INDEX idx_warehouse_ops_container ON process_warehouse_operations(container_number);
```

## 字段命名规范

| 规范 | 示例 |
|------|------|
| 数据库字段 | snake_case (container_number) |
| 实体属性 | camelCase (containerNumber) |
| 主键 | container_number / id |
| 外键 | xxx_code / xxx_id |
| 时间字段 | xxx_date / xxx_time / created_at |
