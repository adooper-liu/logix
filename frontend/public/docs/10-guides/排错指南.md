# LogiX 排错指南

> LogiX 项目常见问题的排查与解决方法

## 快速诊断命令

### 环境诊断

```bash
# 诊断国家筛选问题
npm run diagnose:country-filter

# 验证统计筛选一致性
npm run verify:stats-filter
```

### 服务健康检查

```bash
# 后端健康检查
curl http://localhost:3001/health

# 数据库连接测试
docker exec -it logix-timescaledb-dev psql -U postgres -d logix_db -c "SELECT 1"

# Redis 连接测试
docker exec -it logix-redis-dev redis-cli ping
```

## 常见问题分类

### 1. 环境问题

#### 端口被占用

```powershell
# Windows 查看端口占用
netstat -ano | findstr "3001"

# 查找进程
tasklist | findstr <PID>

# 终止进程
taskkill /PID <PID> /F
```

#### Docker 数据库启动失败

```powershell
# 检查 Docker 状态
docker info

# 查看容器日志
docker logs logix-timescaledb-dev

# 重建容器
docker-compose -f docker-compose.timescaledb.yml down
docker-compose -f docker-compose.timescaledb.yml up -d
```

#### 依赖安装失败

```bash
# 清理 node_modules 重新安装
cd backend
Remove-Item -Recurse -Force node_modules
Remove-Item package-lock.json
npm install
```

### 2. 数据库问题

#### 数据库连接失败

检查 `.env` 配置：
```env
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=postgres
DB_DATABASE=logix_db
```

测试连接：
```bash
docker exec -it logix-timescaledb-dev psql -U postgres -d logix_db
```

#### 表结构不一致

以 `backend/03_create_tables.sql` 为准：
```bash
# 查看实际表结构
docker exec -it logix-timescaledb-dev psql -U postgres -d logix_db -c "\d biz_containers"
```

#### 数据导入问题

**问题表现**：导入的数据查询不到

**排查步骤**：
1. 检查导入日志，确认导入成功
2. 查询数据库确认数据存在
3. 检查日期筛选条件是否正确
4. 检查国家筛选是否生效

**重要**：禁止使用 UPDATE 修补导入错误，正确流程：
```sql
-- 删除错误数据
DELETE FROM biz_containers WHERE container_number = '错误数据';

-- 修复导入代码后重新导入
```

### 3. API 问题

#### 请求返回 404

```bash
# 检查路由是否注册
curl http://localhost:3001/api/v1/containers

# 检查后端日志
docker logs logix-backend
```

#### 请求返回 500

检查后端日志：
```bash
docker logs logix-backend --tail 100
```

常见原因：
- 数据库查询错误
- 实体字段与数据库不一致
- 必填字段缺失

#### 跨域问题

检查 `backend/.env`：
```env
CORS_ORIGIN=http://localhost:5173
```

### 4. 前端问题

#### 前端无法连接后端

1. 检查后端服务是否启动
```bash
curl http://localhost:3001/health
```

2. 检查前端 API 配置
```typescript
// frontend/src/api/index.ts
baseURL: '/api/v1'
```

3. 检查浏览器控制台 CORS 错误

#### 页面空白/白屏

1. 打开浏览器开发者工具（F12）
2. 查看 Console 错误
3. 查看 Network 错误

常见原因：
- 依赖未安装完整
- TypeScript 编译错误
- 组件导入路径错误

#### 筛选返回空结果

排查步骤：
1. 检查 API 返回数据
2. 检查筛选条件映射（前端 → 后端）
3. 检查日期筛选范围
4. 检查国家筛选是否生效

```bash
# 直接调用 API 测试
curl "http://localhost:3001/api/v1/containers/statistics/arrival?filterCondition=today"
```

### 5. 统计问题

#### 统计数字与查询结果不一致

**问题表现**：统计接口返回的数量与列表查询返回的记录数不匹配

**排查步骤**：
1. 确认统计方法和查询方法使用相同 SQL 条件
2. 检查日期筛选是否一致
3. 检查筛选条件映射是否正确

```bash
# 验证统计一致性
npm run verify:stats-filter
```

#### 日期筛选问题

**问题表现**：筛选返回 0 条数据，实际有数据

**原因**：JS Date 对象带时区，与数据库日期比较时产生偏移

**解决**：使用纯字符串日期（YYYY-MM-DD），避免使用 new Date()

```typescript
// 错误
const startDate = new Date('2026-01-01');

// 正确
const startDate = '2026-01-01';
```

### 6. 滞港费计算问题

#### 滞港费计算结果为 0

检查项：
1. 货柜是否实际到港（有 ATA 或 at_port 状态）
2. 滞港费标准是否匹配（进口国、目的港、船公司、货代）
3. 是否在免费天数内
4. 预测模式下是否有计划提柜日

#### 滞港费标准未匹配

```sql
-- 查询已配置的滞港费标准
SELECT * FROM dict_demurrage_rules
WHERE effective_date <= CURRENT_DATE
AND (expiry_date IS NULL OR expiry_date >= CURRENT_DATE);
```

### 7. 物流状态问题

#### 状态流转不正确

**排查步骤**：
1. 检查货柜的当前状态
2. 检查相关时间字段（ATA、提柜日、卸柜日等）
3. 检查状态机逻辑

```sql
-- 查询货柜状态
SELECT container_number, logistics_status, ata_dest_port, 
       gate_out_time, gate_in_time
FROM biz_containers c
LEFT JOIN process_port_operations po ON c.container_number = po.container_number
WHERE c.container_number = 'TCNU1234567';
```

## 日志分析

### 后端日志

```bash
# 实时查看后端日志
docker logs -f logix-backend

# 查看错误日志
docker logs logix-backend 2>&1 | grep -i error
```

- Console：JavaScript 错误
- Network：API 请求响应

### 数据库日志

```bash
# 开启查询日志（开发环境）
docker exec -it logix-timescaledb-dev psql -U postgres -d logix_db -c \
  "ALTER DATABASE logix_db SET log_statement = 'all';"
```
