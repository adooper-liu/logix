# LogiX 开发环境准备指南

> LogiX 项目开发入门第一步：环境准备与启动

## 前置要求

| 工具 | 版本要求 | 用途 |
|------|----------|------|
| Node.js | >= 18.0.0 | 后端/前端运行环境 |
| npm | >= 9.0.0 | 包管理 |
| Docker Desktop | 最新版 | 数据库/缓存/监控服务 |
| Git | 任意 recent 版本 | 版本控制 |

## 环境变量配置

### 1. 复制环境配置模板

```bash
# 进入后端目录
cd backend

# 复制环境配置模板
copy .env.example .env
```

### 2. 关键配置项说明

```env
# 应用配置
NODE_ENV=development
PORT=3001
API_PREFIX=/api/v1

# CORS 配置（前端开发服务器地址）
CORS_ORIGIN=http://localhost:5173
SOCKET_CORS_ORIGIN=http://localhost:5173

# 数据库配置
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=postgres
DB_DATABASE=logix_db

# Redis 配置
REDIS_HOST=localhost
REDIS_PORT=6379

# 飞驼API配置（可选，留空仅Excel导入可用）
FEITUO_CLIENT_ID=
FEITUO_CLIENT_SECRET=
```

## 启动开发环境

### 方式一：PowerShell 脚本（推荐 Windows）

```powershell
# 启动完整开发环境（数据库 + 后端 + 前端 + 微服务）
.\start-logix-dev.ps1
```

### 方式二：Make 命令（推荐 Linux/Mac）

```bash
# 启动数据库服务
make tsdb-up

# 分别启动后端和前端
cd backend && npm run dev
cd frontend && npm run dev
```

### 方式三：Docker Compose 手动启动

```bash
# 启动数据库和缓存
docker-compose -f docker-compose.timescaledb.yml up -d

# 启动后端（独立终端）
cd backend && npm run dev

# 启动前端（独立终端）
cd frontend && npm run dev
```

## 启动后的服务地址

| 服务 | 地址 | 说明 |
|------|------|------|
| 前端 | http://localhost:5173 | Vue 3 开发服务器 |
| 后端 | http://localhost:3001 | Express API 服务 |
| 数据库 | localhost:5432 | TimescaleDB |
| Redis | localhost:6379 | 缓存服务 |
| Adminer | http://localhost:8080 | 数据库管理（可选） |
| pgAdmin | http://localhost:5050 | PostgreSQL 客户端（可选） |
| Prometheus | http://localhost:9090 | 监控（可选） |
| Grafana | http://localhost:3000 | 监控面板（可选，账号 admin/admin） |

## 依赖安装

```bash
# 根目录（lint 配置）
npm install

# 后端
cd backend && npm install

# 前端
cd frontend && npm install
```

## 验证环境

### 1. 验证后端服务

```bash
# 后端健康检查
curl http://localhost:3001/health
```

预期响应：
```json
{
  "status": "ok",
  "timestamp": "2026-03-27T10:00:00.000Z"
}
```

### 2. 验证前端服务

访问 http://localhost:5173 ，页面正常显示即表示环境正常。

### 3. 验证数据库连接

```bash
# 通过 Docker 进入数据库
docker exec -it logix-timescaledb-dev psql -U postgres -d logix_db

# 或者使用 Make 命令
make tsdb-db
```

## 常见问题

### Q1: 端口被占用

```powershell
# Windows 查看占用端口的进程
netstat -ano | findstr "3001"

# 终止进程
taskkill /PID <PID> /F
```

### Q2: Docker 数据库启动失败

```powershell
# 检查 Docker Desktop 是否运行
docker info

# 重启 Docker
docker-compose -f docker-compose.timescaledb.yml down
docker-compose -f docker-compose.timescaledb.yml up -d
```

### Q3: 依赖安装失败

```powershell
# 清理 node_modules 重新安装
cd backend
Remove-Item -Recurse -Force node_modules
Remove-Item package-lock.json
npm install
```

### Q4: 前端无法连接后端

检查后端 `.env` 配置：
```env
CORS_ORIGIN=http://localhost:5173
```

确保与前端实际访问地址一致。
