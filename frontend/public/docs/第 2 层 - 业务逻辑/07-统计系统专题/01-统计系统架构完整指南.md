## 📋 概述

统计系统是 LogiX 物流系统的数据分析核心，提供多维度货柜统计、状态分布、月度货量等功能。

### 核心价值

1. **多维度统计**: 5 大统计维度（到港、ETA、计划提柜、最晚提柜、最晚还箱）
2. **实时预警**: 识别超期、即将超期等风险货柜
3. **状态分布**: 完整的货柜生命周期分布
4. **高性能**: SQL 子查询模板优化，批量查询

## 一、统计维度详解

### 1.1 按到港统计

**用途**: 统计货柜到达目的港/中转港的情况

```
已到目的港
├── 今日到港 (arrivalToday)
├── 之前未提柜 (arrivedBeforeTodayNotPickedUp)
└── 之前已提柜 (arrivedBeforeTodayPickedUp)

已到中转港 (transit)
├── ETA 逾期 (transitOverdue)
├── ETA 3 日内 (transitWithin3Days)
├── ETA 7 日内 (transitWithin7Days)
└── ETA 7 日后 (transitOver7Days)

预计到港 (无中转)
├── ETA 逾期 (expectedArrivalOverdue)
├── ETA 3 日内 (expectedArrivalWithin3Days)
├── ETA 7 日内 (expectedArrivalWithin7Days)
└── ETA 7 日后 (expectedArrivalOver7Days)
```

**关键字段**:

- `ata_dest_port`: 实际到港日
- `eta_dest_port`: 预计到港日
- `port_type='destination'`: 只统计目的港，过滤中转港
- `port_sequence`: 取最新一条港口操作

---

### 1.2 按 ETA 统计

**用途**: 统计在途货柜的预计到达情况

```
在途 (shipped/in_transit)
├── 逾期未到 (overdue)
├── 3 日内到港 (within3Days)
├── 7 日内到港 (within7Days)
└── 7 日后到港 (over7Days)
```

**关键字段**:

- `eta_dest_port`: 预计到港日
- `shipment_date`: 海运出运日

---

### 1.3 按计划提柜统计

**用途**: 统计有计划提柜日的货柜分布

```
有计划提柜日
├── 逾期 (overduePlanned)
├── 今日 (todayPlanned)
├── 3 日内 (plannedWithin3Days)
└── 7 日内 (plannedWithin7Days)

无计划提柜日 (pendingArrangement)
```

**关键字段**:

- `planned_pickup_date`: 计划提柜日
- `process_trucking_transport.pickup_date`: 实际提柜日

---

### 1.4 按最晚提柜统计 ⭐

**用途**: 统计免费期截止情况，识别超期风险

```
有最后免费日
├── 已超时 (expired) - last_free_date < today
├── 即将超时 (urgent) - today <= last_free_date <= 3 days
├── 预警 (warning) - 3 days < last_free_date <= 7 days
└── 正常 (normal) - last_free_date > 7 days

无最后免费日 (noLastFreeDate)
```

**关键字段**:

- `process_port_operations.last_free_date`: 最晚提柜日（可自动计算写回）
- **数据来源**:
  - Excel 导入或人工维护
  - 滞港费计算自动写回（当 DB 值为空时）

---

### 1.5 按最晚还箱统计 ⭐

**用途**: 统计免费用箱期截止情况

```
有最后还箱日
├── 已超时 (returnExpired)
├── 即将超时 (returnUrgent)
├── 预警 (returnWarning)
└── 正常 (returnNormal)

无最后还箱日 (noLastReturnDate)
```

**关键字段**:

- `process_empty_return.last_return_date`: 最晚还箱日（可自动计算写回）
- **数据来源**:
  - Excel 导入或人工维护
  - 滞港费计算自动写回（当 DB 值为空时）

## 三、数据流与业务流程

### 3.1 数据流

```
用户访问 Shipments 页面
        ↓
前端调用 ContainerService.getStatistics()
        ↓
后端 ContainerStatisticsService.getStatistics()
        ↓
根据维度委托给对应子服务：
├─ 按到港 → ArrivalStatisticsService
├─ 按 ETA → EtaStatisticsService
├─ 按计划提柜 → PlannedPickupStatisticsService
├─ 按最晚提柜 → LastPickupStatisticsService
└─ 按最晚还箱 → LastReturnStatisticsService
        ↓
子服务使用 SQL 子查询模板统计
        ↓
返回统计结果
        ↓
前端展示统计卡片
```

---

### 3.2 业务流程

#### 场景 1: 查看今日到港货柜

```
1. 用户点击"今日到港"统计卡片
2. 前端筛选条件设置为 days='arrivalToday'
3. 后端 ArrivalStatisticsService.arrivalToday()
4. 使用 SQL 子查询模板统计
5. 返回货柜列表
```

#### 场景 2: 查看即将超期免费期货柜

```
1. 用户点击"即将超时"统计卡片（按最晚提柜）
2. 前端筛选条件设置为 days='urgent'
3. 后端 LastPickupStatisticsService.urgent()
4. 统计 today <= last_free_date <= 3 days 的货柜
5. 返回货柜列表
```

## 四、SQL 子查询模板

### 4.1 今日到港

```sql
SELECT COUNT(DISTINCT c.container_number) as count
FROM biz_containers c
INNER JOIN process_port_operations po
  ON c.container_number = po.container_number
WHERE po.port_type = 'destination'
  AND po.ata_dest_port = $1
  AND po.port_sequence = (
    SELECT MAX(port_sequence)
    FROM process_port_operations
    WHERE container_number = c.container_number
      AND port_type = 'destination'
  )
```

**关键点**:

- ✅ 使用 `port_sequence` 确保取最新一条港口操作
- ✅ 使用 `port_type='destination'` 过滤中转港
- ✅ 使用 `DISTINCT` 避免重复计数

---

### 4.2 最晚提柜日统计

```sql
-- 即将超时 (urgent)
SELECT COUNT(DISTINCT c.container_number) as count
FROM biz_containers c
INNER JOIN process_port_operations po
  ON c.container_number = po.container_number
WHERE po.port_type = 'destination'
  AND po.last_free_date >= CURRENT_DATE
  AND po.last_free_date <= CURRENT_DATE + INTERVAL '3 days'
  AND po.port_sequence = (
    SELECT MAX(port_sequence)
    FROM process_port_operations
    WHERE container_number = c.container_number
      AND port_type = 'destination'
  )
```

## 五、写回机制 ⭐

### 5.1 last_free_date 自动计算

**触发时机**:

1. 用户打开滞港费详情时（`calculateForContainer` 返回后，异步写回）
2. 定时任务：每 6 小时批量计算并写回

**写回条件**:

- ✅ `process_port_operations.last_free_date IS NULL`
- ✅ demurrage 计算出 `lastPickupDateComputed`
- ✅ 无实际提柜日（`pickup_date IS NULL`）

**写回策略**:

- ✅ 不覆盖人工维护（仅当 DB 值为空时写回）
- ✅ 幂等（多次计算，结果一致时可重复写回）
- ✅ 可配置（通过开关控制是否启用）

---

### 5.2 last_return_date 自动计算

**触发时机**: 同 last_free_date

**写回条件**:

- ✅ `process_empty_return.last_return_date IS NULL`
- ✅ demurrage 计算出 `lastReturnDateComputed`
- ✅ 无实际还箱日（`return_time IS NULL`）

## 六、性能优化

### 6.1 批量查询

```typescript
// ❌ 坏的做法：N+1 查询
async function getStatistics() {
  const arrivalToday = await this.arrivalToday()
  const arrivedBeforeTodayNotPickedUp = await this.arrivedBeforeTodayNotPickedUp()
  // ... 每个维度单独查询
}

// ✅ 好的做法：批量查询
async function getStatistics() {
  const sql = `
    SELECT
      COUNT(DISTINCT FILTER (WHERE condition1)) as arrivalToday,
      COUNT(DISTINCT FILTER (WHERE condition2)) as arrivedBeforeTodayNotPickedUp,
      -- ... 其他维度
    FROM ...
  `
  return await this.containerRepository.query(sql)
}
```
