# 统计

LogiX 统计功能设计与实现。

## 一、统计维度

### 1.1 按到港统计

```
已到目的港
├── 今日到港 (arrivalToday)
├── 之前未提柜 (arrivedBeforeTodayNotPickedUp)
└── 之前已提柜 (arrivedBeforeTodayPickedUp)

已到中转港 (transit)
├── ETA 逾期 (transitOverdue)
├── ETA 3日内 (transitWithin3Days)
├── ETA 7日内 (transitWithin7Days)
└── ETA 7日后 (transitOver7Days)

预计到港 (无中转)
├── ETA 逾期 (expectedArrivalOverdue)
├── ETA 3日内 (expectedArrivalWithin3Days)
├── ETA 7日内 (expectedArrivalWithin7Days)
└── ETA 7日后 (expectedArrivalOver7Days)
```

### 1.2 按 ETA 统计

```
在途 (shipped/in_transit)
├── 逾期未到 (overdue)
├── 3日内到港 (within3Days)
├── 7日内到港 (within7Days)
└── 7日后到港 (over7Days)
```

### 1.3 按计划提柜统计

```
有计划提柜日
├── 逾期 (overduePlanned)
├── 今日 (todayPlanned)
├── 3日内 (plannedWithin3Days)
└── 7日内 (plannedWithin7Days)

无计划提柜日 (pendingArrangement)
```

### 1.4 按最晚提柜统计

```
有最后免费日
├── 已超时 (expired) - last_free_date < today
├── 即将超时 (urgent) - today <= last_free_date <= 3 days
├── 预警 (warning) - 3 days < last_free_date <= 7 days
└── 正常 (normal) - last_free_date > 7 days

无最后免费日 (noLastFreeDate)
```

### 1.5 按最晚还箱统计

```
有最后还箱日
├── 已超时 (returnExpired)
├── 即将超时 (returnUrgent)
├── 预警 (returnWarning)
└── 正常 (returnNormal)

无最后还箱日 (noLastReturnDate)
```

## 二、SQL 实现

### 2.1 子查询模板

统计服务使用 SQL 子查询模板实现：

```sql
-- 按到港：今日到港
SELECT c.container_number
FROM biz_containers c
INNER JOIN process_port_operations po
  ON c.container_number = po.container_number
WHERE po.port_type = 'destination'
  AND po.ata_dest_port = :today
  AND po.port_sequence = (
    SELECT MAX(port_sequence)
    FROM process_port_operations
    WHERE container_number = c.container_number
      AND port_type = 'destination'
  )
```

### 2.2 统计方法

```typescript
async function getArrivedToday(today: Date): Promise<number> {
  const sql = `
    SELECT COUNT(DISTINCT c.container_number) as count
    FROM biz_containers c
    INNER JOIN process_port_operations po
      ON c.container_number = po.container_number
    WHERE po.port_type = 'destination'
      AND po.ata_dest_port = $1
  `;
  const result = await this.containerRepository.query(sql, [today]);
  return parseInt(result[0]?.count || '0');
}
```

## 五、实施坑点

### 5.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 统计数量与列表不一致 | SQL 条件不统一 | 使用统一模板 |
| 日期边界问题 | JS Date 转 timestamptz | SQL 层用 `::date` |
| 中转港误统计 | 未过滤 port_type | 只统计 destination |
| 性能问题 | N+1 查询 | 批量查询 |

### 5.2 排查步骤

1. 检查 SQL 子查询模板是否一致
2. 检查日期过滤条件
3. 检查 port_type 过滤
4. 检查国家筛选条件
