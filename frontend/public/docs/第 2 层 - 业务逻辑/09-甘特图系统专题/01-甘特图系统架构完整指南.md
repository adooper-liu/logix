## 📋 概述

甘特图系统是 LogiX 物流系统的核心可视化组件，用于展示货柜物流全生命周期的时间线，支持拖拽调整、预警提示、多维度过滤等功能。

### 核心价值

1. **可视化追踪**: 直观展示货柜从清关到还箱的完整流程
2. **智能预警**: 自动识别超期、即将超期等风险货柜
3. **交互操作**: 支持拖拽改期、右键菜单、详情查看
4. **多维度分组**: 按港口、节点、供应商三级分组展示
5. **性能优化**: 虚拟滚动、懒加载、计算属性缓存

```typescript
function getNodeAndSupplier(container: Container): Array<{
  node: string
  supplier: string
}> {
  const result: Array<{ node: string; supplier: string }> = []

  // 1. 清关节点
  if (container.customsDeclarationTime) {
    result.push({
      node: '清关',
      supplier: container.customsBroker || '未知清关行',
    })
  }

  // 2. 提柜节点
  if (container.actualPickupDate || container.plannedPickupDate) {
    result.push({
      node: '提柜',
      supplier: container.truckingCompany || '未知车队',
    })
  }

  // 3. 卸柜节点
  if (container.actualUnloadDate || container.plannedUnloadDate) {
    result.push({
      node: '卸柜',
      supplier: container.warehouseCode || '未知仓库',
    })
  }

  // 4. 还箱节点
  if (container.actualReturnDate || container.plannedReturnDate) {
    result.push({
      node: '还箱',
      supplier: container.truckingCompany || '未知车队',
    })
  }

  // 5. 查验节点（如有）
  if (container.inspectionRequired) {
    result.push({
      node: '查验',
      supplier: '查验机构',
    })
  }

  return result
}
```

### 2.3 圆点显示规则

**核心原则**: 基于 `ganttDerived` 字段判断主/虚/销毁状态

```typescript
interface GanttDisplayItem {
  type: 'main' | 'virtual' | 'destroyed'
  status: 'planned' | 'actual' | 'partial' | 'empty'
  plannedDate?: Date
  actualDate?: Date
  displayDate?: Date // 优先显示的实际日期
}

function getDisplayItemsFromGanttDerived(
  container: Container,
  gd: GanttDerived,
  nodeStatus: ContainerNodeStatus
): GanttDisplayItem[] {
  const displayItems: GanttDisplayItem[] = []

  // 遍历五节点
  for (const nodeName of ['清关', '查验', '提柜', '卸柜', '还箱']) {
    const node = nodeStatus.nodes[nodeName]

    // 判断节点状态
    let item: GanttDisplayItem

    if (gd.nodes?.[nodeName]?.status === 'actual') {
      // ✅ 已完成：显示实心圆点（绿色）
      item = {
        type: 'main',
        status: 'actual',
        plannedDate: gd.nodes?.[nodeName]?.plannedDate,
        actualDate: gd.nodes?.[nodeName]?.actualDate,
        displayDate: gd.nodes?.[nodeName]?.actualDate,
      }
    } else if (gd.nodes?.[nodeName]?.status === 'planned') {
      // ⏰ 未完成：显示空心圆点（橙色）
      item = {
        type: 'main',
        status: 'planned',
        plannedDate: gd.nodes?.[nodeName]?.plannedDate,
        displayDate: gd.nodes?.[nodeName]?.plannedDate,
      }
    } else if (node.hasActualDate && !gd.nodes?.[nodeName]) {
      // 💀 已销毁：实际有日期但 ganttDerived 没有
      item = {
        type: 'destroyed',
        status: 'actual',
        displayDate: node.actualDate,
      }
    } else {
      // 👻 虚节点：上游未完成，下游不可能完成
      item = {
        type: 'virtual',
        status: 'empty',
      }
    }

    displayItems.push(item)
  }

  return displayItems
}
```

## 四、核心算法

### 4.0 节点日期优先级（v1.1 新增）

**核心函数**: `getNodePlannedDate(container, nodeName)`

用于确定货柜在指定节点（五节点：清关/提柜/卸柜/还箱/查验）的计划日期，实现圆点正确落格。

#### 4.0.1 各节点日期来源优先级

| 节点     | 优先级 | 字段来源                              | 说明                       |
| -------- | ------ | ------------------------------------- | -------------------------- |
| **清关** | 1      | `actualCustomsDate`                   | 实际清关日期               |
|          | 2      | `plannedCustomsDate`                  | 计划清关日期               |
|          | 3      | `ataDestPort`                         | 实际到港日期               |
|          | 4      | `etaCorrection`                       | 修正ETA                    |
|          | 5      | `etaDestPort`                         | 预计到港日期（原始）       |
|          | 6      | `container.ataDestPort`               | 货柜级ATA                  |
|          | 7      | `container.etaCorrection`             | 货柜级修正ETA              |
|          | 8      | `container.etaDestPort`               | 货柜级ETA                  |
| **提柜** | 1      | `deliveryDate`                        | 实际送仓日期               |
|          | 2      | `plannedDeliveryDate`                 | 计划送仓日期               |
|          | 3      | `pickupDate`                          | 实际提柜日期               |
|          | 4      | `plannedPickupDate`                   | 计划提柜日期               |
|          | 5      | `warehouseOp.actualUnloadDate + 1天`  | 基于卸柜日推算             |
|          | 6      | `warehouseOp.plannedUnloadDate + 1天` | 基于计划卸柜日推算         |
| **卸柜** | 1      | `actualUnloadDate`                    | 实际卸柜日期               |
|          | 2      | `plannedUnloadDate`                   | 计划卸柜日期               |
|          | 3      | `unloadDate`                          | 卸柜日期（兼容字段）       |
| **还箱** | 1      | `returnTime`                          | 实际还箱时间               |
|          | 2      | `lastReturnDate`                      | 最晚还箱日期               |
|          | 3      | `plannedReturnDate`                   | 计划还箱日期               |
|          | 4      | `lastFreeDate + 7天`                  | 基于最晚提柜日推算         |
| **查验** | 1      | `actualCustomsDate`                   | 实际清关日期（与清关同步） |
|          | 2      | `plannedCustomsDate`                  | 计划清关日期               |

#### 4.0.2 实现代码

```typescript
/** 获取货柜在指定节点的计划日期（用于圆点落格） */
const getNodePlannedDate = (container: any, nodeName: string): Date | null => {
  const destPortOp = container.portOperations?.find((op: any) => op.portType === 'destination')
  const trucking = container.truckingTransports?.[0]
  const warehouseOp = container.warehouseOperations?.[0]
  const emptyReturn = container.emptyReturns?.[0]

  switch (nodeName) {
    case '清关':
      // 实际清关 > 计划清关 > ATA > ETA修正 > ETA原始
      if (destPortOp?.actualCustomsDate) return new Date(destPortOp.actualCustomsDate)
      if (destPortOp?.plannedCustomsDate) return new Date(destPortOp.plannedCustomsDate)
      if (destPortOp?.ataDestPort) return new Date(destPortOp.ataDestPort)
      if (destPortOp?.etaCorrection) return new Date(destPortOp.etaCorrection)
      if (destPortOp?.etaDestPort) return new Date(destPortOp.etaDestPort)
      if (container.ataDestPort) return new Date(container.ataDestPort)
      if (container.etaCorrection) return new Date(container.etaCorrection)
      if (container.etaDestPort) return new Date(container.etaDestPort)
      return null
    case '提柜':
      // 实际送仓 > 计划送仓 > 实际提柜 > 计划提柜
      if (trucking?.deliveryDate) return new Date(trucking.deliveryDate)
      if (trucking?.plannedDeliveryDate) return new Date(trucking.plannedDeliveryDate)
      if (trucking?.pickupDate) return new Date(trucking.pickupDate)
      if (trucking?.plannedPickupDate) return new Date(trucking.plannedPickupDate)
      // 备选：基于卸柜日（到港后1天）
      if (warehouseOp?.actualUnloadDate) {
        const date = new Date(warehouseOp.actualUnloadDate)
        date.setDate(date.getDate() + 1)
        return date
      }
      if (warehouseOp?.plannedUnloadDate) {
        const date = new Date(warehouseOp.plannedUnloadDate)
        date.setDate(date.getDate() + 1)
        return date
      }
      return null
    case '卸柜':
      // 实际卸柜 > 计划卸柜
      if (warehouseOp?.actualUnloadDate) return new Date(warehouseOp.actualUnloadDate)
      if (warehouseOp?.plannedUnloadDate) return new Date(warehouseOp.plannedUnloadDate)
      if (warehouseOp?.unloadDate) return new Date(warehouseOp.unloadDate)
      return null
    case '还箱':
      // 实际还箱 > 最晚还箱 > 计划还箱
      if (emptyReturn?.returnTime) return new Date(emptyReturn.returnTime)
      if (emptyReturn?.lastReturnDate) return new Date(emptyReturn.lastReturnDate)
      if (emptyReturn?.plannedReturnDate) return new Date(emptyReturn.plannedReturnDate)
      // 备选：最晚提柜日 + 7天
      if (destPortOp?.lastFreeDate) {
        const date = new Date(destPortOp.lastFreeDate)
        date.setDate(date.getDate() + 7)
        return date
      }
      return null
    case '查验':
      // 与清关使用相同日期来源
      if (destPortOp?.actualCustomsDate) return new Date(destPortOp.actualCustomsDate)
      if (destPortOp?.plannedCustomsDate) return new Date(destPortOp.plannedCustomsDate)
      return null
    default:
      return null
  }
}
```

#### 4.0.3 关键设计原则

1. **实际优先于计划**: 优先使用已完成节点的实际日期
2. **多级回退机制**: 每级找不到时自动回退到下一级
3. **日期推算**: 基于相邻节点日期自动推算（如最晚提柜日+7天=最晚还箱日）
4. **与Tooltip一致**: `getNodePlannedDate` 与 Tooltip 中的日期辅助方法保持一致

### 4.1 动态日期范围计算

```typescript
/**
 * 根据货柜数据动态计算最佳显示范围
 */
function calculateDynamicDateRange(containers: Container[]): [Date, Date] {
  if (containers.length === 0) {
    // 无数据时显示今天起 30 天
    const today = new Date()
    return [today, dayjs(today).add(30, 'day').toDate()]
  }

  // 收集所有日期（计划 + 实际）
  const allDates: Date[] = []

  containers.forEach(container => {
    const dates = [
      container.customsDeclarationTime,
      container.actualPickupDate,
      container.plannedPickupDate,
      container.actualUnloadDate,
      container.plannedUnloadDate,
      container.actualReturnDate,
      container.plannedReturnDate,
    ]
      .filter(Boolean)
      .map(d => new Date(d!))

    allDates.push(...dates)
  })

  if (allDates.length === 0) {
    // 无任何日期时显示今天起 30 天
    const today = new Date()
    return [today, dayjs(today).add(30, 'day').toDate()]
  }

  // 找到最小和最大日期
  const minDate = new Date(Math.min(...allDates.map(d => d.getTime())))
  const maxDate = new Date(Math.max(...allDates.map(d => d.getTime())))

  // 前后各扩展 7 天
  const startDate = dayjs(minDate).subtract(7, 'day').toDate()
  const endDate = dayjs(maxDate).add(7, 'day').toDate()

  return [startDate, endDate]
}
```

```typescript
interface AlertRule {
  name: string

  check: (container: Container) => boolean
  message: (container: Container) => string

const alertRules: AlertRule[] = [
  {
    name: 'last_free_date_overdue',

    check: container => {

      const today = new Date()
      return lfd && lfd < today

  },
  {
    name: 'last_free_date_urgent',

      if (!lfd) return false

      const diffDays = Math.floor((lfd.getTime() - today.getTime()) / (1000 * 60 * 60 * 24))
      return diffDays >= 0 && diffDays <= 2
    },
    message: container => {

      if (diffDays === 1) return '免费期明天到期'
      return `免费期剩余 ${diffDays} 天`
    },
  },
  {
    name: 'missing_planned_date',
    level: 'warning',
    check: container => {
      // 检查是否有节点缺少计划日期
      return (
        !container.plannedPickupDate || !container.plannedUnloadDate || !container.plannedReturnDate
      )
    },
    message: () => '缺少计划日期',
  },
]

/**
 * 获取货柜的所有预警

function getContainerAlerts(container: Container): Alert[] {
  const alerts: Alert[] = []

  for (const rule of alertRules) {
    if (rule.check(container)) {
      alerts.push({
        rule: rule.name,
        level: rule.level,
        message: rule.message(container),

    }
  }
```

### 4.3 拖拽改期算法

#### 4.3.1 节点字段映射（NODE_TO_FIELD_MAP）

```typescript
/** 节点类型到更新字段的映射 */
const NODE_TO_FIELD_MAP: Record<string, { field: string; label: string }> = {
  清关: { field: 'plannedCustomsDate', label: '计划清关日' },
  查验: { field: 'plannedCustomsDate', label: '计划查验日' },
  提柜: { field: 'plannedPickupDate', label: '计划提柜日' },
  卸柜: { field: 'plannedUnloadDate', label: '计划卸柜日' },
  还箱: { field: 'plannedReturnDate', label: '计划还箱日' },
  未分类: { field: 'plannedPickupDate', label: '计划提柜日' },
}
```

#### 4.3.2 拖拽流程

```
用户拖拽货柜圆点
       ↓
handleDragStart(container, event) → 设置 draggingContainer
       ↓
document.dragover → handleDragOver(event)
       ↓
updateDragOverState(event) → 计算落点日期格
       ↓
用户松手 → drop 事件
       ↓
handleDrop(date) → pendingDropConfirm 存储待确认数据
       ↓
dragend 事件 → handleDragEnd()
       ↓
requestAnimationFrame × 2 → 弹窗确认
       ↓
确认后调用 API 更新日期
```

#### 4.3.3 完整实现代码

```typescript
// 拖拽状态
const draggingContainer = ref<Container | null>(null)
const dragOverDate = ref<Date | null>(null)
const pendingDropConfirm = ref<{
  container: Container
  newDate: string
  updateField: string
  fieldLabel: string
  confirmMsg: string
} | null>(null)

// handleDragStart
const handleDragStart = (container: Container, event: DragEvent) => {
  draggingContainer.value = container
  if (event.dataTransfer) {
    event.dataTransfer.effectAllowed = 'move'
    event.dataTransfer.setData('text/plain', container.containerNumber)
  }
}

// RAF 节流
let rafId = 0
let lastEvent: DragEvent | null = null
const updateDragOverState = (event: DragEvent) => {
  const elementUnderCursor = document.elementFromPoint(event.clientX, event.clientY) as HTMLElement
  const dateCell =
    elementUnderCursor?.closest('.date-cell') ??
    (event.target as HTMLElement)?.closest('.date-cell')
  if (dateCell) {
    const dateIndexAttr = dateCell.getAttribute('data-date-index')
    let dateIndex = -1
    if (dateIndexAttr !== null) {
      dateIndex = parseInt(dateIndexAttr, 10)
      if (isNaN(dateIndex)) dateIndex = -1
    } else {
      const parent = dateCell.parentElement
      const siblings = parent ? Array.from(parent.children) : []
      dateIndex = siblings.indexOf(dateCell)
    }
    if (dateIndex >= 0 && dateIndex < dateRange.value.length) {
      const newDate = dateRange.value[dateIndex]
      const rect = dateCell.getBoundingClientRect()
      dragOverDate.value = newDate
      // 更新落点高亮位置...
    }
  }
}

// handleDragOver
const handleDragOver = (event: DragEvent) => {
  event.preventDefault()
  lastEvent = event
  if (rafId === 0) {
    rafId = requestAnimationFrame(() => {
      if (lastEvent) updateDragOverState(lastEvent)
      rafId = 0
    })
  }
}

// handleDrop
const handleDrop = (date: Date) => {
  if (!draggingContainer.value || !dragOverDate.value) return
  const container = draggingContainer.value
  const newDate = dayjs(dragOverDate.value).format('YYYY-MM-DD')

  // 根据 filterCondition 确定要更新的字段
  const condition = filterCondition.value
  let updateField = 'plannedPickupDate'
  let fieldLabel = '计划提柜日'

  if (condition.startsWith('return')) {
    updateField = 'plannedReturnDate'
    fieldLabel = '计划还箱日'
  } else if (condition.startsWith('unload')) {
    updateField = 'plannedUnloadDate'
    fieldLabel = '计划卸柜日'
  } else if (condition.startsWith('customs') || condition.startsWith('inspection')) {
    updateField = 'plannedCustomsDate'
    fieldLabel = '计划清关日'
  }

  const confirmMsg = `确定要将货柜 ${container.containerNumber} 的${fieldLabel}调整为 ${formatDateShort(dragOverDate.value)} 吗？`
  pendingDropConfirm.value = { container, newDate, updateField, fieldLabel, confirmMsg }
}

// handleDragEnd（关键：双重 RAF 避免取消需点二次）
const handleDragEnd = () => {
  if (rafId) {
    cancelAnimationFrame(rafId)
    rafId = 0
  }
  const pending = pendingDropConfirm.value
  pendingDropConfirm.value = null
  draggingContainer.value = null
  dragOverDate.value = null

  if (!pending) return

  const { container, newDate, updateField, fieldLabel, confirmMsg } = pending
  // 在 dragend 之后、下一帧再弹窗，确保浏览器已释放指针
  requestAnimationFrame(() => {
    requestAnimationFrame(() => {
      ElMessageBox.confirm(confirmMsg, '确认调整日期', {
        confirmButtonText: '确定',
        cancelButtonText: '取消',
        type: 'warning',
      })
        .then(async () => {
          const updateData: Record<string, string> = { [updateField]: newDate }
          const result = await containerService.updateSchedule(
            container.containerNumber,
            updateData
          )
          if (result.success) {
            ElMessage.success('日期调整成功')
            await loadData()
          } else {
            ElMessage.error(result.message || '更新失败')
          }
        })
        .catch((err: unknown) => {
          if (err !== 'cancel') {
            ElMessage.error((err as Error)?.message || '操作失败')
          }
        })
    })
  })
}
```

#### 4.3.4 关键技术点

1. **elementFromPoint**: 精确定位鼠标下的元素，而非依赖 event.target
2. **data-date-index**: 每个 date-cell 设置索引属性，支持精确定位
3. **document 级监听**: 在 document 上监听 dragover/drop，支持拖到任意区域
4. **RAF 节流**: 每帧最多更新一次高亮状态，兼顾流畅与性能
5. **双重 RAF**: 确保弹窗在 dragend 之后显示，避免首次点击被浏览器消费

### 5.1 滚动性能优化（v1.1 新增）

**问题**: 快速滚动时频繁触发 tooltip 显示导致卡顿

**解决方案**: 滚动节流 + 快速滚动时隐藏 tooltip

```typescript
// 滚动节流
let scrollRAFId = 0
let lastScrollTop = 0
const isScrollingFast = ref(false)

const handleScroll = (event: Event) => {
  const target = event.target as HTMLElement
  const scrollTop = target.scrollTop
  const scrollLeft = target.scrollLeft

  // 判断是否快速滚动
  const delta = Math.abs(scrollTop - lastScrollTop)
  if (delta > 30) {
    isScrollingFast.value = true
    if (scrollRAFId) cancelAnimationFrame(scrollRAFId)
    scrollRAFId = requestAnimationFrame(() => {
      setTimeout(() => {
        isScrollingFast.value = false
      }, 150)
    })
  }
  lastScrollTop = scrollTop

  // 同步表头滚动
  const headerEl = document.querySelector('.gantt-header-row') as HTMLElement
  if (headerEl && headerEl.scrollLeft !== scrollLeft) {
    headerEl.scrollLeft = scrollLeft
  }
}
```

**Tooltip 智能边界检测（v1.1 新增）**:

问题：增强版 Tooltip 包含 5 个分区，内容较多，底部容易超出屏幕。

解决方案：智能边界检测，自动调整位置。

```typescript
// Tooltip 尺寸预估（用于边界检测）
const TOOLTIP_WIDTH = 320
const TOOLTIP_HEIGHT = 420

const showTooltip = (container: Container, event: MouseEvent) => {
  tooltipContainer.value = container

  // 基础偏移量
  const offsetX = 15
  const offsetY = 15

  // 获取视口尺寸
  const viewportWidth = window.innerWidth
  const viewportHeight = window.innerHeight

  // 计算 tooltip 位置
  let x = event.clientX + offsetX
  let y = event.clientY + offsetY

  // 右侧边界检测：如果 tooltip 会超出右边界，则向左偏移
  if (x + TOOLTIP_WIDTH > viewportWidth - 10) {
    x = event.clientX - TOOLTIP_WIDTH - offsetX
  }

  // 底部边界检测：如果 tooltip 会超出底边界，则向上偏移
  if (y + TOOLTIP_HEIGHT > viewportHeight - 10) {
    y = event.clientY - TOOLTIP_HEIGHT - offsetY
  }

  // 左边界检测：确保不超出左边界
  if (x < 10) {
    x = 10
  }

  // 上边界检测：确保不超出上边界
  if (y < 10) {
    y = 10
  }

  tooltipPosition.value = { x, y }
  tooltipVisible.value = true
}
```

**边界检测优先级**:

1. **右侧优先**：优先显示在鼠标右侧（正常情况）
2. **底部翻转**：如果右侧不够空间，尝试显示在鼠标左侧
3. **底部翻转**：如果底部超出，则显示在鼠标上方
4. **极端边界**：最后确保不超出视口最小边界
