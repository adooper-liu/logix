```
## 三、技术实现

### 3.1 核心代码

```

/\*\*

- 🎯 应用最优方案
  \*/
  const applyOptimalSolution = async () => {
  if (!optimizationResult.value) {
  ElMessage.warning('没有可应用的优化方案')
  return
  }

try {
const {
containerNumber,
suggestedPickupDate,
suggestedStrategy,
currentPickupDate,
} = optimizationResult.value

    // 1. 确认对话框
    await ElMessageBox.confirm(
      `确定要应用最优方案吗？\n\n` +
        `当前提柜日: ${currentPickupDate}\n` +
        `建议提柜日: ${suggestedPickupDate}\n` +
        `建议策略: ${suggestedStrategy}\n\n` +
        `系统将自动更新日期并重新计算还箱日。`,
      '应用最优方案',
      {
        confirmButtonText: '确定应用',
        cancelButtonText: '取消',
        type: 'warning',
      }
    )

    // 2. 构建更新数据
    const updateData: Record<string, string> = {
      plannedPickupDate: suggestedPickupDate,
    }

    // 根据策略决定是否需要更新卸柜日
    // Direct/Expedited: 卸柜日 = 提柜日
    // Drop off: 卸柜日由后端根据仓库能力确定（智能搜索）
    // 注意：如果未提供 plannedUnloadDate，后端默认使用 pickupDate + 2天 作为估算值
    if (suggestedStrategy === 'Direct' || suggestedStrategy === 'Expedited') {
      updateData.plannedUnloadDate = suggestedPickupDate
    }
    // Drop off 模式：不设置 plannedUnloadDate，让后端根据仓库能力计算

    // 3. 调用后端 API 更新
    const result = await containerService.updateSchedule(containerNumber, updateData)

    if (result.success) {
      // 4. 显示成功消息
      ElMessage.success({
        message: `✅ 最优方案已应用！提柜日已更新为 ${suggestedPickupDate}`,
        duration: 3000,
      })

      // 5. 刷新甘特图数据
      await loadData()

      // 6. 关闭优化面板
      closeOptimizationPanel()
    } else {
      // 7. 显示错误消息
      ElMessage.error({
        message: result.message || '应用失败，请重试',
        duration: 3000,
      })
    }

} catch (error: any) {
if (error !== 'cancel') {
// 8. 处理异常
ElMessage.error({
message: `应用失败：${error.message || '未知错误'}`,
duration: 5000,
})
}
}
}

```

### 3.2 关键设计决策

#### 决策 1: 同时更新提柜日和卸柜日

**原因**:
- ✅ 确保日期一致性（提柜和卸柜同一天）
- ✅ 符合 Drop off/Direct/Expedited 策略的实际业务逻辑
- ✅ 后端会根据这两个日期自动计算还箱日

**代码**:
``typescript
const updateData = {
  plannedPickupDate: suggestedPickupDate,
  plannedUnloadDate: suggestedPickupDate,  // ← 同时更新
}
```

#### 决策 2: 不直接更新还箱日

**原因**:

- ✅ 还箱日由后端自动计算（`mergeReturnDateIntoUpdateData`）
- ✅ 避免前端重复实现计算逻辑
- ✅ 确保前后端规则一致

**后端逻辑** (已在 `executeUpdateWithData` 中):
``typescript
if (updateData.plannedUnloadDate) {
mergeReturnDateIntoUpdateData(container, updateData, um)
}

```

#### 决策 3: 更新成功后立即刷新数据

**原因**:
- ✅ 确保甘特图显示最新数据
- ✅ 触发防抖优化检查（可能产生新的优化建议）
- ✅ 用户体验流畅

**代码**:
``typescript
if (result.success) {
  await loadData()  // ← 刷新数据
  closeOptimizationPanel()
}
```

## 九、性能影响

### 9.1 API 调用次数

**应用前**:

- 拖拽触发优化: 1 次 API（带防抖）
- 查看备选方案: 0 次 API（使用缓存）

**应用后**:

- 更新日期: 1 次 API (`updateSchedule`)
- 刷新数据: 1 次 API (`loadData`)
- 可能触发新优化: 1 次 API（带防抖+缓存）

**总计**: 2-3 次 API 调用

### 9.2 响应时间

| 操作       | 耗时   | 说明         |

| 确认对话框 | <100ms | 本地渲染     |

| 数据刷新   | ~1s    | 查询数据库   |
