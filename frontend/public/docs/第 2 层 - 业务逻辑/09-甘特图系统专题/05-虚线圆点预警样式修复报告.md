# 甘特图虚线圆点预警样式修复报告

## 修复时间
2026-04-04

## 问题描述

**现象**：虚线圆点（dashed-task）始终显示为"透明背景 + 灰色虚线边框"，无法体现预警状态。

**根本原因**：
1. CSS 中使用了 `background: transparent !important;` 覆盖了内联样式的背景色
2. CSS 中使用了固定边框颜色 `border: 2px dashed #c0c4cc;` 覆盖了内联样式的边框色
3. 一级/二级泳道的模板缺少 `borderColor` 绑定

## 修复内容

### 1. CSS 样式修改

**文件**：`frontend/src/components/common/SimpleGanttChartRefactored.vue`

**修改前**（Line 3820-3823）：
```css
.container-dot.dashed-task {
  background: transparent !important; /* 空心圆点，保持白色填充 */
  border: 2px dashed #c0c4cc;
}
```

**修改后**（Line 3820-3833）：
```css
/* 虚线任务 - 虚线圆点（计划中） */
.container-dot.dashed-task {
  background: transparent; /* 空心圆点，透明背景 */
  border: 2px dashed #c0c4cc; /* 灰色虚线边框 */
}

/* 虚线任务 + 预警状态 */
.container-dot.dashed-task.has-warning {
  background: rgba(230, 162, 60, 0.3); /* 半透明橙色背景 */
  border: 2px dashed #f56c6c; /* 红色虚线边框 */
}
```

**改进点**：
- ✅ 移除了 `!important`，允许内联样式覆盖（如果需要）
- ✅ 添加了 `.has-warning` 修饰符，支持预警状态的视觉区分
- ✅ 预警虚线圆点：半透明橙色背景 + 红色虚线边框 + 脉冲动画

---

### 2. 模板样式绑定统一

**修改位置**：所有渲染 `container-dot` 的模板（共 5 处）

**修改前**：
```vue
:style="{ backgroundColor: getStatusColor(container.logisticsStatus) }"
```

**修改后**：
```vue
:style="{
  backgroundColor: getStatusColor(container.logisticsStatus),
  borderColor: getContainerBorderColor(container),
}"
```

**影响范围**：
1. Line 156-159：一级港口泳道 - 清关节点
2. Line 200-203：未分类港口泳道 - 清关节点
3. Line 320-323：三级供应商泳道 - 五节点（已存在）
4. Line 700-703：弹窗 - 一级港口泳道
5. Line 810-813：弹窗 - 五节点泳道

## 技术细节

### 1. CSS 优先级

**问题**：`!important` 会覆盖内联样式

**解决方案**：
- 移除 `!important`，使用更具体的选择器（`.dashed-task.has-warning`）
- 保留默认样式作为回退

### 2. 预警状态判断

**函数**：`hasAlert(container)`

**返回条件**：
- `danger`：已逾期 → 红色边框 + 脉冲动画
- `warning`：3天内到期 → 橙色边框 + 脉冲动画
- `normal`：正常 → 灰色边框

### 3. 颜色映射

**背景色**（`getStatusColor`）：
- `picked_up` / `unloaded` / `returned_empty` → `#67c23a`（绿色）
- `shipped` / `in_transit` → `#409eff`（蓝色）
- `at_port` → `#e6a23c`（橙色）
- `not_shipped` → `#909399`（灰色）

**边框色**（`getContainerBorderColor`）：
- `danger` → `#f56c6c`（红色）
- `warning` → `#e6a23c`（橙色）
- `normal` → `#c0c4cc`（灰色）
