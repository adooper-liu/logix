# 甘特图圆点颜色定义规范

## 一、圆点类型与样式

### 1. 实心圆点（main-task）- 当前活跃节点

**CSS 样式**（Line 3815-3817）：
```css
.container-dot.main-task {
  border: 2px solid; /* 实线边框，颜色由内联样式 borderColor 决定 */
}
```

**内联样式绑定**：
- **背景色**：`getStatusColor(container.logisticsStatus)` - 根据物流状态动态设置
- **边框色**：`getContainerBorderColor(container)` - 根据预警等级动态设置

**显示规则**：
- 当 `getNodeDisplayType(container, node) === 'main'` 时显示
- 表示该节点是当前活跃节点（前置已完成 + 有计划日期）

---

### 2. 虚线圆点（dashed-task）- 计划中但未激活

**CSS 样式**（Line 3820-3828）：
```css
.container-dot.dashed-task {
  background: transparent !important; /* 空心圆点，透明背景 */
  border: 2px dashed #c0c4cc; /* 灰色虚线边框 */
}

.container-dot.dashed-task:hover {
  border-color: #909399; /* 悬停时边框变深灰 */
  transform: scale(1.5);
}
```

**内联样式绑定**：
- **背景色**：`getStatusColor(container.logisticsStatus)` - **但被 CSS 的 `!important` 覆盖，实际为透明**
- **边框色**：`getContainerBorderColor(container)` - **但被 CSS 的固定值 `#c0c4cc` 覆盖**

**显示规则**：
- 当 `getNodeDisplayType(container, node) === 'dashed'` 时显示
- 表示该节点有计划日期，但前置节点未完成（待激活状态）

**问题**：
- ⚠️ 内联样式的 `backgroundColor` 和 `borderColor` 都被 CSS 覆盖了
- ⚠️ 虚线圆点始终显示为"空心 + 灰色虚线"，无法体现预警状态

---

### 3. 已完成圆点（completed-task）- 带✓标记

**CSS 样式**（Line 3831-3844）：
```css
.container-dot.completed-task {
  position: relative;
}

.container-dot.completed-task::after {
  content: '✓';
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  font-size: 8px;
  color: white;
  font-weight: bold;
}
```

**注意**：
- ✅ 已完成节点应该被销毁（`getNodeDisplayType` 返回 `null`），不应该渲染
- ✅ 这个类名目前可能没有实际使用场景

## 二、颜色映射表

### 1. 物流状态颜色（getStatusColor）

**定义位置**：`useGanttLogic.ts` Line 78-86

| 物流状态 | 英文标识 | 颜色值 | 说明 |
|---------|---------|--------|------|
| 未出运 | not_shipped | `#909399` | 灰色 |
| 已出运 | shipped | `#409eff` | 蓝色 |
| 运输中 | in_transit | `#409eff` | 蓝色 |
| 到港 | at_port | `#e6a23c` | 橙色 |
| 已提柜 | picked_up | `#67c23a` | 绿色 |
| 已卸柜 | unloaded | `#67c23a` | 绿色 |
| 已还箱 | returned_empty | `#67c23a` | 绿色 |
| 未知状态 | - | `#909399` | 灰色（默认） |

**函数实现**（Line 719-722）：
```typescript
const getStatusColor = (status?: string): string => {
  if (!status) return '#909399' // 默认灰色
  return statusColors[status.toLowerCase()] || '#909399'
}
```

---

### 2. 预警边框颜色（getContainerBorderColor）

**定义位置**：`useGanttLogic.ts` Line 220-230

| 预警等级 | 条件 | 颜色值 | 说明 |
|---------|------|--------|------|
| danger | 已逾期 | `#f56c6c` | 红色 |
| warning | 3天内到期 | `#e6a23c` | 橙色 |
| normal | 正常状态 | `#c0c4cc` | 灰色 |

**函数实现**：
```typescript
const getContainerBorderColor = (container: Container): string => {
  const alertLevel = getAlertLevel(container)
  switch (alertLevel) {
    case 'danger':
      return '#f56c6c' // 红色（已逾期）
    case 'warning':
      return '#e6a23c' // 橙色（3天内到期）
    default:
      return '#c0c4cc' // 灰色（正常状态）
  }
}
```

---

### 3. 预警视觉效果（has-warning）

**CSS 样式**（Line 3896-3899）：
```css
.container-dot.has-warning {
  box-shadow: 0 0 8px rgba(245, 108, 108, 0.6); /* 红色阴影 */
  animation: pulse-warning 2s ease-in-out infinite; /* 脉冲动画 */
}
```

**脉冲动画**（Line 3902-3910）：
```css
@keyframes pulse-warning {
  0%, 100% {
    box-shadow: 0 0 8px rgba(245, 108, 108, 0.6);
  }
  50% {
    box-shadow: 0 0 16px rgba(245, 108, 108, 0.9); /* 更强的阴影 */
  }
}
```

**触发条件**：
- 当 `hasAlert(container)` 返回 `true` 时添加 `has-warning` 类
- 表示货柜有预警（逾期或即将到期）

## 三、模板中的样式绑定

### 1. 一级港口泳道 & 二级五节点泳道（Line 156, 200, 694, 801）

```vue
:style="{ backgroundColor: getStatusColor(container.logisticsStatus) }"
```

**特点**：
- 只设置背景色，不设置边框色
- 边框色由 CSS 的 `.main-task { border: 2px solid; }` 继承内联样式的 `borderColor`（如果有的话）
- 实际上这些模板**没有绑定 `borderColor`**，所以边框颜色可能是透明的或由浏览器默认决定

---

### 2. 三级供应商泳道（Line 320-323）

```vue
:style="{
  backgroundColor: getStatusColor(container.logisticsStatus),
  borderColor: getContainerBorderColor(container),
}"
```

**特点**：
- 同时设置背景色和边框色
- 这是唯一明确绑定 `borderColor` 的模板

**问题**：
- ⚠️ 对于虚线圆点（dashed-task），这两个值都会被 CSS 覆盖
- ⚠️ 导致虚线圆点无法体现预警状态（始终是灰色虚线）

---

## 四、当前存在的问题

### 问题1：虚线圆点的颜色被 CSS 覆盖

**现象**：
- 虚线圆点应该有预警颜色（橙色背景 + 红色边框），但实际显示为透明背景 + 灰色虚线

**原因**：
```css
.container-dot.dashed-task {
  background: transparent !important; /* ← 覆盖了内联的 backgroundColor */
  border: 2px dashed #c0c4cc; /* ← 覆盖了内联的 borderColor */
}
```

**影响**：
- 用户无法通过颜色区分虚线圆点的预警状态
- 所有虚线圆点看起来都一样（灰色）

---

### 问题2：一级/二级泳道的边框颜色未绑定

**现象**：
- 一级港口泳道和二级五节点泳道的圆点只有背景色，没有明确的边框色

**原因**：
```vue
<!-- Line 156, 200, 694, 801 -->
:style="{ backgroundColor: getStatusColor(container.logisticsStatus) }"
<!-- 缺少 borderColor -->
```

**影响**：
- 实心圆点的边框颜色可能不一致
- 无法体现预警状态（红色边框）

---

## 五、建议的修复方案

### 方案A：保持当前设计（空心虚线）

**适用场景**：
- 虚线圆点表示"计划中但未激活"，不需要体现预警状态
- 预警信息通过其他方式展示（如 Tooltip、统计面板）

**优点**：
- 视觉清晰，实心和虚线有明显区别
- 符合"空心=计划中"的直觉

**缺点**：
- 无法通过颜色快速识别哪些虚线圆点有预警

---

### 方案B：虚线圆点也显示预警颜色

**修改 CSS**：
```css
.container-dot.dashed-task {
  /* 移除 background: transparent !important; */
  background: transparent; /* 允许内联样式覆盖 */
  border: 2px dashed; /* 移除固定颜色，允许内联样式覆盖 */
}
```

**效果**：
- 虚线圆点会有背景色（绿色/蓝色等）和边框色（灰色/橙色/红色）
- 可以体现预警状态

**缺点**：
- 视觉上可能不够清晰（有色背景的虚线圆点）

---

### 方案C：虚线圆点使用特殊预警样式

**新增 CSS 类**：
```css
/* 虚线 + 预警 */
.container-dot.dashed-task.has-warning {
  background: rgba(230, 162, 60, 0.3); /* 半透明橙色背景 */
  border: 2px dashed #f56c6c; /* 红色虚线边框 */
}
```

**效果**：
- 普通虚线圆点：透明背景 + 灰色虚线
- 预警虚线圆点：半透明橙色背景 + 红色虚线 + 脉冲动画

**优点**：
- 既能区分虚实，又能体现预警
- 视觉层次清晰

---

## 六、统一修复建议

### 1. 统一所有模板的样式绑定

**修改前**（一级/二级泳道）：
```vue
:style="{ backgroundColor: getStatusColor(container.logisticsStatus) }"
```

**修改后**：
```vue
:style="{
  backgroundColor: getStatusColor(container.logisticsStatus),
  borderColor: getContainerBorderColor(container),
}"
```

---

### 2. 调整虚线圆点的 CSS

**推荐方案C**：
```css
/* 虚线任务 - 默认样式 */
.container-dot.dashed-task {
  background: transparent;
  border: 2px dashed #c0c4cc;
}

/* 虚线任务 + 预警 */
.container-dot.dashed-task.has-warning {
  background: rgba(230, 162, 60, 0.3); /* 半透明橙色 */
  border: 2px dashed #f56c6c; /* 红色虚线 */
}
```

---

### 3. 验证反向链式依赖

确保以下节点的显示逻辑正确：
- ✅ 已提柜 → 清关和提柜销毁
- ✅ 已卸柜 → 清关、提柜、卸柜销毁
- ✅ 已还箱 → 清关、提柜、卸柜、还箱销毁

## 七、总结

| 圆点类型 | 背景色 | 边框色 | 边框样式 | 预警效果 |
|---------|--------|--------|---------|---------|
| **实心（main）** | 物流状态颜色 | 预警等级颜色 | 实线 2px | 红色阴影 + 脉冲 |
| **虚线（dashed）** | 透明（被覆盖） | 灰色 #c0c4cc（被覆盖） | 虚线 2px | ❌ 无（被覆盖） |
| **完成（completed）** | 物流状态颜色 | 预警等级颜色 | 实线 2px | ✓ 标记 |

**核心问题**：
- 虚线圆点的颜色被 CSS 的 `!important` 和内联固定值覆盖
- 一级/二级泳道缺少 `borderColor` 绑定

**建议**：
- 采用方案C，为虚线圆点添加预警样式
- 统一所有模板的样式绑定
