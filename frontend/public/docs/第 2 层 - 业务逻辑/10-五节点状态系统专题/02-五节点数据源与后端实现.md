## 数据库表结构

### 核心表关系

```
biz_containers (货柜主表)
  ├─ logistics_status (物流状态)
  ├─ inspection_required (是否需要查验)
  │
  ├─ process_port_operations (港口操作)
  │   └─ customs_status (清关状态, port_type='destination')
  │
  ├─ process_trucking_transport (拖卡运输)
  │   ├─ planned_pickup_date (计划提柜日)
  │   └─ pickup_date (实际提柜日)
  │
  └─ process_empty_return (还箱记录)
      └─ return_time (实际还箱时间)
```

### 关键字段说明

| 表名 | 字段名 | 类型 | 说明 | 五节点用途 |
|------|--------|------|------|-----------|
| biz_containers | logistics_status | varchar(20) | 七层物流状态 | 推断卸柜/还箱状态 |
| biz_containers | inspection_required | boolean | 是否需要查验 | 查验节点 |
| process_port_operations | customs_status | varchar(20) | 清关状态 | 清关节点 |
| process_port_operations | port_type | varchar(20) | 港口类型 | 筛选目的港 |
| process_trucking_transport | planned_pickup_date | date | 计划提柜日 | 提柜节点 |
| process_trucking_transport | pickup_date | date | 实际提柜日 | 提柜节点 |
| process_empty_return | return_time | timestamp | 实际还箱时间 | 还箱节点 |

## SQL 查询逻辑

### 列表查询(Shipments 表格)

文件位置: `backend/src/services/container.service.ts`

```sql
SELECT 
  c.container_number,
  c.logistics_status,
  c.inspection_required,
  
  -- 清关状态(目的港)
  po_dest.customs_status,
  
  -- 提柜日期
  tt.planned_pickup_date,
  tt.pickup_date,
  
  -- 还箱时间
  er.return_time
  
FROM biz_containers c

-- 目的港港口操作(清关状态)
LEFT JOIN process_port_operations po_dest 
  ON c.container_number = po_dest.container_number 
  AND po_dest.port_type = 'destination'

-- 拖卡运输(提柜日期) - 取最新一条
LEFT JOIN LATERAL (
  SELECT planned_pickup_date, pickup_date
  FROM process_trucking_transport
  WHERE container_number = c.container_number
  ORDER BY created_at DESC
  LIMIT 1
) tt ON true

-- 还箱记录 - 取最新一条
LEFT JOIN LATERAL (
  SELECT return_time
  FROM process_empty_return
  WHERE container_number = c.container_number
  ORDER BY created_at DESC
  LIMIT 1
) er ON true

WHERE ...  -- 其他筛选条件
ORDER BY c.created_at DESC
LIMIT 50 OFFSET 0
```

**性能优化**:
- 使用 `LEFT JOIN LATERAL` 避免笛卡尔积
- 只查询需要的字段,不使用 `SELECT *`
- 通过 `LIMIT 1` 确保每个货柜只返回一条拖卡/还箱记录

### 单个货柜详情查询

文件位置: `backend/src/services/fiveNodeService.ts`

```typescript
async getFiveNodeInfo(containerNumber: string) {
  // 1. 获取货柜基本信息
  const container = await this.containerRepository.findOne({
    where: { containerNumber },
    relations: ['seaFreight']
  });

  // 2. 获取目的港港口操作
  const destPort = await this.portOperationRepository.findOne({
    where: { containerNumber, portType: 'destination' },
    order: { portSequence: 'DESC' }
  });

  // 3. 获取拖卡运输
  const trucking = await this.truckingRepository.findOne({
    where: { containerNumber },
    order: { createdAt: 'DESC' }
  });

  // 4. 获取还箱记录
  const emptyReturn = await this.emptyReturnRepository.findOne({
    where: { containerNumber },
    order: { createdAt: 'DESC' }
  });

  // 5. 组装五节点数据
  return {
    containerNumber: container.containerNumber,
    fiveNodes: {
      customs: {
        status: destPort?.actualCustomsDate ? 'cleared' : 'pending',
        plannedDate: destPort?.plannedCustomsDate,
        actualDate: destPort?.actualCustomsDate,
        latestStatus: destPort?.customsStatus
      },
      trucking: {
        status: trucking?.pickupDate ? 'pickedUp' : 'notPickedUp',
        plannedDate: trucking?.plannedPickupDate,
        actualDate: trucking?.pickupDate
      },
      unloading: {
        status: warehouseOp?.unloadDate ? 'unloaded' : 'notUnloaded',
        plannedDate: warehouseOp?.plannedUnloadDate,
        actualDate: warehouseOp?.unloadDate
      },
      emptyReturn: {
        status: emptyReturn?.returnTime ? 'returned' : 'notReturned',
        plannedDate: emptyReturn?.plannedReturnDate,
        actualDate: emptyReturn?.returnTime
      },
      inspection: {
        status: inspection?.inspectionDate ? 'inspected' : 'notInspected',
        plannedDate: inspection?.inspectionPlannedDate,
        actualDate: inspection?.inspectionDate
      }
    }
  };
}
```

## TypeORM 实体映射

### Container 实体

文件位置: `backend/src/entities/Container.ts`

```typescript
@Entity('biz_containers')
export class Container {
  @PrimaryColumn({ type: 'varchar', length: 50, name: 'container_number' })
  containerNumber!: string;

  @Column({
    type: 'varchar',
    length: 20,
    default: 'not_shipped',
    name: 'logistics_status'
  })
  logisticsStatus!: string;

  @Column({ type: 'boolean', default: false, name: 'inspection_required' })
  inspectionRequired!: boolean;

  // 关联关系
  @OneToMany(() => PortOperation, (portOp) => portOp.container)
  portOperations!: PortOperation[];

  @OneToMany(() => TruckingTransport, (tt) => tt.container)
  truckingTransports!: TruckingTransport[];

  @OneToMany(() => EmptyReturn, (er) => er.container)
  emptyReturns!: EmptyReturn[];
}
```

## 数据同步机制

### 五节点数据的更新时机

| 触发事件 | 更新的节点 | 更新方式 |
|---------|-----------|---------|
| 飞驼事件同步 | 清关、卸柜、还箱 | 自动更新流程表 → 重新计算 |
| Excel 导入 | 所有节点 | 导入时写入流程表 |
| 智能排柜 | 提柜(计划日期) | 更新 planned_pickup_date |
| 手工编辑 | 任意节点 | 通过 ScheduleEditDialog |
| 状态机重算 | 卸柜、还箱 | 调用 calculateLogisticsStatus |

### 状态一致性保证

**问题**: 流程表更新了,但 `logistics_status` 没有同步

**解决方案**:
```typescript
// 在更新流程表后,触发状态重算
async function updateTruckingPickupDate(containerNumber: string, pickupDate: Date) {
  // 1. 更新拖卡表
  await this.truckingRepository.update(
    { containerNumber },
    { pickupDate }
  );
  
  // 2. 重新计算物流状态
  await this.containerStatusService.recalculateStatus(containerNumber);
  
  // 3. 清除缓存(如果有)
  this.cacheService.invalidate(`container:${containerNumber}`);
}
```

## 性能优化

### 1. 索引优化

```sql
-- 货柜号索引(所有流程表都需要)
CREATE INDEX idx_port_ops_container ON process_port_operations(container_number);
CREATE INDEX idx_trucking_container ON process_trucking_transport(container_number);
CREATE INDEX idx_empty_return_container ON process_empty_return(container_number);

-- 复合索引(加速筛选)
CREATE INDEX idx_port_ops_type_container ON process_port_operations(port_type, container_number);
CREATE INDEX idx_containers_status ON biz_containers(logistics_status);
```

### 2. 查询优化

**避免 N+1 查询**:
```typescript
// ❌ 错误: 循环查询
for (const container of containers) {
  const trucking = await this.truckingRepository.findOne({
    where: { containerNumber: container.containerNumber }
  });
}

// ✅ 正确: 一次性查询
const containerNumbers = containers.map(c => c.containerNumber);
const truckings = await this.truckingRepository.find({
  where: { containerNumber: In(containerNumbers) }
});
```

**使用 DataLoader**:
```typescript
class TruckingDataLoader {
  private cache = new Map<string, TruckingTransport>();
  
  async load(containerNumber: string): Promise<TruckingTransport | null> {
    if (!this.cache.has(containerNumber)) {
      const trucking = await this.truckingRepository.findOne({
        where: { containerNumber },
        order: { createdAt: 'DESC' }
      });
      this.cache.set(containerNumber, trucking);
    }
    return this.cache.get(containerNumber) || null;
  }
}
```

### 3. 缓存策略

**Redis 缓存**:
```typescript
// 缓存键: five_node:{containerNumber}
// 过期时间: 5 分钟

async function getFiveNodeWithCache(containerNumber: string) {
  const cacheKey = `five_node:${containerNumber}`;
  
  // 尝试从缓存读取
  const cached = await redis.get(cacheKey);
  if (cached) {
    return JSON.parse(cached);
  }
  
  // 查询数据库
  const data = await fiveNodeService.getFiveNodeInfo(containerNumber);
  
  // 写入缓存
  await redis.setex(cacheKey, 300, JSON.stringify(data));
  
  return data;
}
```

**缓存失效时机**:
- 流程表更新时
- 状态机重算时
- 手工编辑时

## 数据完整性检查

### 常见问题

#### 问题 1: 清关状态为 null

**原因**:
- 目的港港口操作记录不存在
- `port_type` 不是 'destination'
- 数据导入时遗漏

**排查 SQL**:
```sql
SELECT 
  c.container_number,
  po.id as port_op_id,
  po.port_type,
  po.customs_status
FROM biz_containers c
LEFT JOIN process_port_operations po 
  ON c.container_number = po.container_number 
  AND po.port_type = 'destination'
WHERE c.container_number = 'TCLU1234567';
```

**修复方案**:
- 检查数据导入脚本
- 手动补充港口操作记录
- 确保 `port_type` 字段正确

---

#### 问题 2: 提柜日期缺失

**原因**:
- 拖卡运输记录不存在
- 有计划日期但没有实际日期
- 智能排柜未执行

**排查 SQL**:
```sql
SELECT 
  c.container_number,
  tt.id as trucking_id,
  tt.planned_pickup_date,
  tt.pickup_date
FROM biz_containers c
LEFT JOIN process_trucking_transport tt 
  ON c.container_number = tt.container_number
WHERE c.container_number = 'TCLU1234567';
```

**修复方案**:
- 执行智能排柜生成计划日期
- 手工录入实际提柜日期
- 检查车队分配逻辑

---

#### 问题 3: 还箱时间为 null

**原因**:
- 还箱记录不存在
- 还未到还箱环节
- 数据同步延迟

**排查 SQL**:
```sql
SELECT 
  c.container_number,
  er.id as empty_return_id,
  er.return_time,
  er.planned_return_date
FROM biz_containers c
LEFT JOIN process_empty_return er 
  ON c.container_number = er.container_number
WHERE c.container_number = 'TCLU1234567';
```

**修复方案**:
- 等待还箱操作完成
- 检查飞驼事件同步
- 手动录入还箱时间

## 监控与告警

### 关键指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| 清关状态 null 比例 | > 5% | 数据导入问题 |
| 提柜日期缺失比例 | > 10% | 智能排柜未执行 |
| 还箱时间缺失比例 | > 20% | 正常(很多货柜还未还箱) |
| 五节点查询耗时 | > 500ms | 需要优化查询 |

### 监控 SQL

```sql
-- 清关状态分布
SELECT 
  customs_status,
  COUNT(*) as count,
  ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER(), 2) as percentage
FROM process_port_operations po
JOIN biz_containers c ON po.container_number = c.container_number
WHERE po.port_type = 'destination'
GROUP BY customs_status;

-- 提柜日期缺失统计
SELECT 
  COUNT(*) FILTER (WHERE tt.pickup_date IS NULL) as missing_pickup,
  COUNT(*) as total,
  ROUND(COUNT(*) FILTER (WHERE tt.pickup_date IS NULL) * 100.0 / COUNT(*), 2) as missing_rate
FROM biz_containers c
LEFT JOIN process_trucking_transport tt ON c.container_number = tt.container_number;
```
