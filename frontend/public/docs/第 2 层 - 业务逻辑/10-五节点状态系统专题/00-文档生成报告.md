## 关键技术点记录

### 1. 提柜状态修正

**问题**: 原代码有计划提柜日就显示"已完成"

**修正**:
```typescript
// 修正前
const pickup = plannedPickupDate || pickupDate ? 'ok' : 'bad'

// 修正后
const pickup = pickupDate ? 'ok' : (plannedPickupDate ? 'warn' : 'bad')
```

**影响**: 
- 更新了 `containerDisplay.ts` 第 195 行
- 更新了 `getFiveNodeRows()` 第 247 行
- 新增了测试用例 `should distinguish pickup status correctly`

### 2. 五节点与七层状态机的关系

**区别**:
- 五节点: 列表快速概览,粗粒度(5个节点)
- 七层状态机: 详细物流路径,细粒度(7层+多个子节点)

**关联**:
- 卸柜/还箱节点基于 `logisticsStatus` 推断
- 七层状态机更新会触发五节点重新计算

### 3. 数据来源权威性

| 五节点 | 数据表 | 字段 | 权威级别 |

| 卸柜 | biz_containers | logistics_status | P1 (推断) |
