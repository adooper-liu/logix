# 状态机

LogiX 物流状态机设计与实现。

## 一、物流状态

### 1.1 状态定义

| 状态 | 代码 | 说明 |
|------|------|------|
| 未出运 | `not_shipped` | 尚未装船 |
| 已出运 | `shipped` | 已装船 |
| 在途 | `in_transit` | 航行中 |
| 已到港 | `at_port` | 到达港口 |
| 已提柜 | `picked_up` | 已提柜到仓库/目的地 |
| 已卸柜 | `unloaded` | 已卸货 |
| 已还箱 | `returned_empty` | 已归还空箱 |

### 1.2 状态流转

```
not_shipped → shipped → in_transit → at_port → picked_up → unloaded → returned_empty
```

## 二、状态计算逻辑

### 2.1 核心规则

```typescript
function calculateStatus(container, portOperations, seaFreight, trucking, warehouse, emptyReturn) {
  // 1. 检查是否已还箱
  if (emptyReturn?.returnTime) {
    return 'returned_empty'
  }

  // 2. 检查是否已卸柜
  if (warehouse?.unloadDate) {
    return 'unloaded'
  }

  // 3. 检查是否已提柜
  if (trucking?.pickupDate) {
    return 'picked_up'
  }

  // 4. 检查是否已到港（目的港）
  const destPort = portOperations?.find(p => p.portType === 'destination')
  if (destPort?.ata || container?.logisticsStatus === 'at_port') {
    return 'at_port'
  }

  // 5. 检查是否在途
  if (seaFreight?.shipmentDate) {
    return 'in_transit'
  }

  return 'not_shipped'
}
```

### 2.2 计算模式

| 模式 | 使用场景 | 计算依据 |
|------|----------|----------|
| **actual** | 已到目的港 | 实际日期 ATA、提柜日、还箱日 |
| **forecast** | 未到目的港 | ETA、计划提柜日、计划还箱日 |

## 三、港口操作状态

### 3.1 港口类型

| 类型 | 代码 | 说明 |
|------|------|------|
| 起运港 | `origin` | 货物出发港口 |
| 中转港 | `transit` | 途径港口 |
| 目的港 | `destination` | 最终目的港口 |

### 3.2 多港经停

一个货柜可以有多个港口操作记录：

```sql
SELECT * FROM process_port_operations
WHERE container_number = 'xxxxx'
ORDER BY port_sequence;
```

### 3.3 统计规则

只统计目的港（port_type = 'destination'）记录，排除中转港。

## 四、时间节点

### 4.1 核心时间字段

| 字段 | 说明 |
|------|------|
| `shipment_date` | 开船日 |
| `eta_dest_port` | 预计到港日 |
| `ata_dest_port` | 实际到港日 |
| `revised_eta_dest_port` | 修正ETA |
| `last_free_date` | 最后免费日 |
| `last_return_date` | 最晚还箱日 |

### 4.2 时间线

```
shipment_date (开船日)
    ↓
eta_dest_port (预计到港)
    ↓
ata_dest_port (实际到港)
    ↓
last_free_date (最后免费日)
    ↓
last_return_date (最晚还箱日)
```

## 五、飞驼数据同步

### 5.1 状态映射

```
飞驼状态码 → DetailedStatus → SimplifiedStatus → LogiX 物流状态
```

### 5.2 核心字段映射

飞驼状态码更新对应的核心时间字段：

| 飞驼状态码 | 核心字段 |
|-----------|----------|
| ARRIVE/ATA/ARRI/BDAR | `ata` |
| ETA | `eta` |
| GATE_IN/GTIN/GITM | `gate_in_time` |
| GATE_OUT/GTOT/STCS | `gate_out_time` |
| DISCHARGED/DISC/DSCH | `dest_port_unload_date` |
| PCAB/AVLE/AVAIL | `available_time` |
| TRANSIT_ARRIVE/TSBA | `transit_arrival_date` |

### 5.3 更新规则

```typescript
async function syncFromFeituo(containerNumber: string, statusEvent: StatusEvent) {
  // 1. 更新港口操作记录
  await updatePortOperation(containerNumber, statusEvent)

  // 2. 触发状态机重新计算
  await recalculateLogisticsStatus(containerNumber)
}
```

## 六、实施坑点

### 6.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 状态计算错误 | 核心时间字段未更新 | 飞驼同步后触发状态机重算 |
| 中转港误统计 | 未过滤 port_type | 只统计 destination 类型 |
| 状态滞后 | 异步更新延迟 | 使用调度器批量更新 |

### 6.2 排查步骤

1. 检查货柜的核心时间字段（ata、pickup_date 等）
2. 检查港口操作记录的 port_type
3. 检查状态机计算逻辑
4. 检查飞驼同步日志
