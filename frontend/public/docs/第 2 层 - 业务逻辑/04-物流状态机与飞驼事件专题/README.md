## 📚 核心知识体系

### 一、物流状态机（7 层简化状态）

#### 状态定义与流转

```
not_shipped → shipped → in_transit → at_port → picked_up → unloaded → returned_empty
未出运        已出运      在途         已到港     已提柜     已卸柜     已还箱
```

#### 状态计算优先级

| 优先级 | 判断条件       | 触发字段                                          | 状态             |
| ------ | -------------- | ------------------------------------------------- | ---------------- |
| **1**  | 还空箱日期     | `emptyReturn.returnTime`                          | `returned_empty` |
| **2**  | 仓库卸柜确认   | `warehouseOperation.wmsConfirmDate`               | `unloaded`       |
| **3**  | 拖车提柜日期   | `truckingTransport.pickupDate`                    | `picked_up`      |
| **4**  | 目的港实际到港 | `process_port_operations.ata`                     | `at_port`        |
| **4a** | 目的港可提货   | `process_port_operations.availableTime`           | `at_port`        |
| **5**  | 中转港 ATA/ETA | `transitPort.ata/eta` + `seaFreight.shipmentDate` | `in_transit`     |
| **6**  | 海运出运       | `seaFreight.shipmentDate`                         | `shipped`        |
| **7**  | 默认           | 无                                                | `not_shipped`    |

**代码位置**:

- 后端：`backend/src/utils/logisticsStatusMachine.ts`
- 前端：`frontend/src/utils/logisticsStatusMachine.ts`

### 二、飞驼事件与映射

#### 飞驼状态码分类

| 分类           | 状态码示例                          | 映射核心字段            | 说明         |
| -------------- | ----------------------------------- | ----------------------- | ------------ |
| **目的港相关** | ARRIVE, ATA, ARRI, BDAR, FETA, POCA | `ata`                   | 实际到港时间 |
| **预计到港**   | ETA, FETA                           | `eta`                   | 预计到港时间 |
| **闸口操作**   | GATE_IN, GITM, GTIN                 | `gate_in_time`          | 进场时间     |
| **闸口操作**   | GATE_OUT, GTOT, STCS                | `gate_out_time`         | 出场时间     |
| **码头操作**   | DISCHARGED, DISC, DSCH              | `dest_port_unload_date` | 卸船时间     |
| **可提货状态** | PCAB, AVLE, AVAIL                   | `availableTime`         | 可提货时间   |
| **中转港**     | TRANSIT_ARRIVE, TSBA                | `transit_arrival_date`  | 中转港到港   |

**完整映射表**: `backend/src/constants/FeiTuoStatusMapping.ts`

---

#### is_estimated 原则（核心原则）

| 类型         | is_estimated  | 是否更新核心字段 | 说明                   |
| ------------ | ------------- | ---------------- | ---------------------- |
| **实际事件** | `false` / `N` | ✅ **是**        | 已经实际发生，不可改变 |
| **预计事件** | `true` / `Y`  | ❌ **否**        | 预测未来，可能变化     |

**核心原则**:

1. ✅ 只有实际事件才能更新核心时间字段
2. ❌ 预计事件不能覆盖实际数据
3. ⚠️ 预计事件即使标记为"最终状态"也不能更新

**代码示例**:

```typescript
// ✅ 正确逻辑
if (!status.isEstimated && status.occurredAt) {
  await this.updateCoreFieldsFromStatus(...);
}

// ❌ 错误逻辑（已修复）
if ((!status.isEstimated || isFinalStatus) && status.occurredAt) {
  await this.updateCoreFieldsFromStatus(...);
}
```

### 三、港口操作记录（process_port_operations）

#### 港口类型

| 类型       | port_type     | 说明                 |
| ---------- | ------------- | -------------------- |
| **起运港** | `origin`      | 货物出发港口         |
| **中转港** | `transit`     | 途径港口（多港经停） |
| **目的港** | `destination` | 最终目的港口         |

#### 核心时间字段

| 字段                    | 说明                                       | 数据来源             |
| ----------------------- | ------------------------------------------ | -------------------- |
| `ata`                   | 实际到港时间 (Actual Time of Arrival)      | 飞驼 ATA/ARRIVE 等   |
| `eta`                   | 预计到港时间 (Estimated Time of Arrival)   | 飞驼 ETA             |
| `atd`                   | 实际离港时间 (Actual Time of Departure)    | 飞驼 ATD             |
| `etd`                   | 预计离港时间 (Estimated Time of Departure) | 飞驼 ETD             |
| `gate_in_time`          | 进场时间                                   | 飞驼 GATE_IN         |
| `gate_out_time`         | 出场时间                                   | 飞驼 GATE_OUT        |
| `availableTime`         | 可提货时间                                 | 飞驼 PCAB/AVLE/AVAIL |
| `dest_port_unload_date` | 卸船时间                                   | 飞驼 DISCHARGED      |

### 四、数据同步流程

#### 飞驼 API 同步流程

```
1. 调用飞驼 API (POST /application/v1/query)
   ↓
2. 获取 status[] 数组
   ↓
3. 解析每个状态节点
   - statusCode (事件代码)
   - eventTime (发生时间)
   - isEsti (是否预计 Y/N)
   ↓
4. 写入 ext_feituo_status_events (原始数据)
   ↓
5. 判断 isEstimated
   ↓ isEstimated = false (实际事件)
6. 更新 process_port_operations 核心字段
   ↓
7. 触发状态机重新计算 logistics_status
```

**关键代码**:

- API 适配器：`backend/src/adapters/FeiTuoAdapter.ts`
- 外部数据服务：`backend/src/services/externalDataService.ts`
- 飞驼导入服务：`backend/src/services/feituoImport.service.ts`

### 常见错误码定位

| **状态一直是 not_shipped**  | 缺少 shipment_date    | 01-物流状态机 → 二、2 节 |
