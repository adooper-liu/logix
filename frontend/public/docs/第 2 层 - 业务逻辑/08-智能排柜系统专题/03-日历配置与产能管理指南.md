## 📋 概述

日历配置和产能管理是智能排柜系统的基础，决定了仓库和车队的工作能力、休息日安排以及档期扣减机制。

### 核心价值

1. **精细化管理**: 按天配置仓库和车队的产能
2. **灵活调度**: 支持周末、节假日等休息日配置
3. **实时扣减**: 排产后自动扣减档期，避免超卖
4. **可视化监控**: 随时查看剩余产能

## 一、仓库能力配置

### 1.1 仓库档期表结构

```sql
CREATE TABLE ext_warehouse_daily_occupancy (
    id                  SERIAL PRIMARY KEY,
    warehouse_code      VARCHAR(50) NOT NULL,          -- 仓库代码
    date                DATE NOT NULL,                 -- 日期
    total_capacity      INTEGER NOT NULL DEFAULT 20,   -- 总容量
    used_capacity       INTEGER NOT NULL DEFAULT 0,    -- 已使用容量
    remaining_capacity  INTEGER NOT NULL,              -- 剩余容量
    is_rest_day         BOOLEAN DEFAULT FALSE,         -- 是否休息日
    notes               TEXT,                          -- 备注
    created_at          TIMESTAMPTZ DEFAULT NOW(),
    updated_at          TIMESTAMPTZ DEFAULT NOW(),
    
    UNIQUE (warehouse_code, date)                      -- 唯一约束
);

-- 索引
CREATE INDEX idx_warehouse_occupancy_wh_date 
  ON ext_warehouse_daily_occupancy(warehouse_code, date);
CREATE INDEX idx_warehouse_occupancy_remaining 
  ON ext_warehouse_daily_occupancy(remaining_capacity);

-- 触发器：自动计算剩余容量
CREATE OR REPLACE FUNCTION update_remaining_capacity()
RETURNS TRIGGER AS $$
BEGIN
  NEW.remaining_capacity := NEW.total_capacity - NEW.used_capacity;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_update_remaining_capacity
BEFORE INSERT OR UPDATE ON ext_warehouse_daily_occupancy
FOR EACH ROW EXECUTE FUNCTION update_remaining_capacity();
```

### 1.4 查询可用日期

**算法实现**:

```typescript
async findEarliestAvailableDay(
  warehouseCode: string,
  startDate: Date,
  maxDaysToSearch: number = 30
): Promise<Date | null> {
  let currentDate = new Date(startDate);
  
  for (let i = 0; i < maxDaysToSearch; i++) {
    // 1. 检查是否为周末
    if (this.isWeekend(currentDate)) {
      currentDate = dateTimeUtils.addDays(currentDate, 1);
      continue;
    }
    
    // 2. 查询档期表
    const occupancy = await this.occupancyRepo.findOne({
      where: {
        warehouseCode,
        date: currentDate
      }
    });
    
    if (!occupancy) {
      // 无记录，跳过
      currentDate = dateTimeUtils.addDays(currentDate, 1);
      continue;
    }
    
    // 3. 检查是否为休息日
    if (occupancy.isRestDay) {
      currentDate = dateTimeUtils.addDays(currentDate, 1);
      continue;
    }
    
    // 4. 检查剩余产能
    if (occupancy.remainingCapacity > 0) {
      return currentDate; // 找到可用日期
    }
    
    // 5. 继续下一天
    currentDate = dateTimeUtils.addDays(currentDate, 1);
  }
  
  return null; // 搜索范围内无可用日期
}

private isWeekend(date: Date): boolean {
  const day = date.getDay();
  return day === 0 || day === 6; // 周日或周六
}
```

### 1.5 档期扣减

**扣减逻辑**:

```typescript
async decrementWarehouseOccupancy(
  warehouseCode: string,
  date: Date,
  amount: number = 1
): Promise<void> {
  const occupancy = await this.occupancyRepo.findOne({
    where: {
      warehouseCode,
      date
    }
  });
  
  if (!occupancy) {
    throw new Error(`No occupancy record found for ${warehouseCode} on ${date}`);
  }
  
  if (occupancy.remainingCapacity < amount) {
    throw new Error(
      `Insufficient capacity: ${warehouseCode} on ${date}, ` +
      `requested: ${amount}, available: ${occupancy.remainingCapacity}`
    );
  }
  
  // 扣减档期
  await this.occupancyRepo.update(occupancy.id, {
    usedCapacity: occupancy.usedCapacity + amount
    // remainingCapacity 会自动由触发器重新计算
  });
  
  logger.info(
    `[Occupancy] Decremented warehouse ${warehouseCode} on ${date.toISOString().split('T')[0]}: ` +
    `${amount} unit(s), remaining: ${occupancy.remainingCapacity - amount}`
  );
}
```

## 三、档期扣减机制

### 3.1 完整扣减流程

```typescript
async decrementAllOccupancies(params: {
  containerNumber: string;
  warehouse: Warehouse;
  truckingCompany: TruckingCompany;
  plannedPickupDate: Date;
  plannedUnloadDate: Date;
  plannedReturnDate: Date;
  portCode: string;
  unloadMode: string;
}): Promise<void> {
  const {
    warehouse,
    truckingCompany,
    plannedPickupDate,
    plannedUnloadDate,
    plannedReturnDate,
    portCode,
    unloadMode
  } = params;
  
  try {
    // ========== 1. 扣减仓库卸柜档期 ==========
    await this.decrementWarehouseOccupancy(
      warehouse.warehouseCode,
      plannedUnloadDate,
      1
    );
    
    logger.info(
      `[IntelligentScheduling] Decremented warehouse occupancy: ` +
      `${warehouse.warehouseCode} on ${plannedUnloadDate.toISOString().split('T')[0]}`
    );
    
    // ========== 2. 扣减车队送柜档期 ==========
    await this.decrementTruckingOccupancy(
      truckingCompany.companyCode,
      plannedPickupDate,  // 使用提柜日作为送柜档期
      portCode,
      warehouse.warehouseCode
    );
    
    logger.info(
      `[IntelligentScheduling] Decremented trucking occupancy: ` +
      `${truckingCompany.companyCode} on ${plannedPickupDate.toISOString().split('T')[0]}`
    );
    
    // ========== 3. 如果是 Drop off 模式，扣减还箱档期 ==========
    if (unloadMode === 'Drop off') {
      await this.decrementTruckingReturnOccupancy(
        truckingCompany.companyCode,
        plannedReturnDate,
        portCode,
        warehouse.warehouseCode
      );
      
      logger.info(
        `[IntelligentScheduling] Decremented trucking return occupancy (Drop off): ` +
        `${truckingCompany.companyCode} on ${plannedReturnDate.toISOString().split('T')[0]}`
      );
    }
    
  } catch (error) {
    logger.error(
      `[IntelligentScheduling] Failed to decrement occupancies for ${params.containerNumber}:`,
      error
    );
    throw error;
  }
}
```

    usedCapacity: occupancy.usedCapacity + amount
  });
  
  logger.debug(
    `[Occupancy] Decremented trucking ${truckingCompanyId} ` +
    `on ${date.toISOString().split('T')[0]}: ${amount} unit(s)`
  );
}

### 4.2 产能告警规则

```typescript
interface CapacityAlert {
  warehouseCode: string;
  date: Date;
  alertLevel: 'warning' | 'critical' | 'full';
  utilizationPercent: number;
  remainingCapacity: number;
}

async checkCapacityAlerts(
  warehouseCode: string,
  startDate: Date,
  endDate: Date
): Promise<CapacityAlert[]> {
  const alerts: CapacityAlert[] = [];
  
  const occupancies = await this.occupancyRepo.find({
    where: {
      warehouseCode,
      date: Between(startDate, endDate)
    },
    order: { date: 'ASC' }
  });
  
  for (const occupancy of occupancies) {
    if (occupancy.isRestDay) {
      continue; // 跳过休息日
    }
    
    const utilizationPercent = (occupancy.usedCapacity / occupancy.totalCapacity) * 100;
    
    if (utilizationPercent >= 100) {
      alerts.push({
        warehouseCode,
        date: occupancy.date,
        alertLevel: 'full',
        utilizationPercent,
        remainingCapacity: occupancy.remainingCapacity
      });
    } else if (utilizationPercent >= 90) {
      alerts.push({
        warehouseCode,
        date: occupancy.date,
        alertLevel: 'critical',
        utilizationPercent,
        remainingCapacity: occupancy.remainingCapacity
      });
    } else if (utilizationPercent >= 75) {
      alerts.push({
        warehouseCode,
        date: occupancy.date,
        alertLevel: 'warning',
        utilizationPercent,
        remainingCapacity: occupancy.remainingCapacity
      });
    }
  }
  
  return alerts;
}
```

## 五、常见问题排查

### 5.1 问题 1: 档期扣减失败

**现象**:
```
Error: Insufficient capacity: WH001 on 2026-03-25, requested: 1, available: 0
```

**排查步骤**:

```sql
-- Step 1: 检查当天档期
SELECT 
  warehouse_code,
  date,
  total_capacity,
  used_capacity,
  remaining_capacity,
  is_rest_day
FROM ext_warehouse_daily_occupancy
WHERE warehouse_code = 'WH001'
  AND date = '2026-03-25';

-- Step 2: 检查最近 7 天的使用情况
SELECT 
  date,
  total_capacity,
  used_capacity,
  remaining_capacity,
  ROUND(used_capacity * 100.0 / total_capacity, 2) as utilization
FROM ext_warehouse_daily_occupancy
WHERE warehouse_code = 'WH001'
  AND date BETWEEN '2026-03-18' AND '2026-03-25'
ORDER BY date;
```

**常见原因**:
- ❌ 档期已满（remaining_capacity=0）
- ❌ 配置的总容量太小
- ❌ 触发器未正确计算剩余容量

**解决方案**:
- ✅ 增加总容量：`UPDATE ext_warehouse_daily_occupancy SET total_capacity = 30 WHERE ...`
- ✅ 检查触发器是否正常工作
- ✅ 调整排产日期到有空闲的日期

---

### 5.2 问题 2: 休息日仍然排产

**现象**:
```
排产成功但日期是周末（is_rest_day=true）
```

**排查步骤**:

```sql
-- 检查周末的档期配置
SELECT 
  warehouse_code,
  date,
  EXTRACT(DOW FROM date) as day_of_week,
  is_rest_day,
  total_capacity
FROM ext_warehouse_daily_occupancy
WHERE warehouse_code = 'WH001'
  AND date >= CURRENT_DATE
  AND EXTRACT(DOW FROM date) IN (0, 6)  -- 周六、周日
ORDER BY date;
```

**常见原因**:
- ❌ is_rest_day 配置错误（false）
- ❌ 排产逻辑未检查 is_rest_day
- ❌ 手动修改了档期但未同步更新 is_rest_day

**解决方案**:
- ✅ 批量更新周末为休息日：
  ```sql
  UPDATE ext_warehouse_daily_occupancy
  SET is_rest_day = TRUE, total_capacity = 0
  WHERE EXTRACT(DOW FROM date) IN (0, 6)
    AND warehouse_code = 'WH001';
  ```
- ✅ 检查排产逻辑中的日期验证代码

### 5.3 问题 3: 档期数据不一致

**现象**:
```
used_capacity + remaining_capacity ≠ total_capacity
```

**排查步骤**:

```sql
-- 查找数据不一致的记录
SELECT 
  id,
  warehouse_code,
  date,
  total_capacity,
  used_capacity,
  remaining_capacity,
  (used_capacity + remaining_capacity) as calculated_total,
  (total_capacity - used_capacity - remaining_capacity) as difference
FROM ext_warehouse_daily_occupancy
WHERE total_capacity != used_capacity + remaining_capacity;
```

**修复方案**:

```sql
-- 修复数据：以 total_capacity 和 used_capacity 为准
UPDATE ext_warehouse_daily_occupancy
SET remaining_capacity = total_capacity - used_capacity
WHERE total_capacity != used_capacity + remaining_capacity;

-- 验证修复结果
SELECT COUNT(*) as inconsistent_count
FROM ext_warehouse_daily_occupancy
WHERE total_capacity != used_capacity + remaining_capacity;
-- 应该返回 0
```
