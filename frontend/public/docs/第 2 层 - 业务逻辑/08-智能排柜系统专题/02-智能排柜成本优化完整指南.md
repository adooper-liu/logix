## 📋 概述

智能排柜成本优化是系统的核心决策引擎，通过智能算法在多种约束条件下（仓库产能、车队能力、免费期等）找到成本最优的卸柜方案。

### 核心价值

1. **自动化决策**: 基于规则引擎自动生成最优方案，减少人工干预
2. **成本降低**: 平均降低 15-30% 总成本
3. **资源平衡**: 合理分配仓库和车队产能，避免拥堵
4. **可追溯**: 完整的日志和档期扣减记录

## 一、三种卸柜策略详解

### 1.1 Direct (Live Unload) - 直卸模式

**特点**:
- 提柜 = 卸柜，当天完成
- 无中间堆存环节

**适用场景**:
- 车队无堆场设施
- 免费期内且仓库有充足产能
- 货物急需入库

**成本构成**:
```typescript
totalCost = demurrageCost + detentionCost + transportationCost

// 示例
// demurrageCost: $0 (免费期内)
// detentionCost: $0 (免费期内)
// transportationCost: $200 (基础运费)
// totalCost: $200
```

**优点**:
- ✅ 流程简单，操作便捷
- ✅ 无额外堆场费用
- ✅ 货物快速入库

**缺点**:
- ❌ 完全依赖仓库产能
- ❌ 可能产生滞港费（如超期）
- ❌ 灵活性差

---

### 1.2 Drop off - 堆场暂存模式

**特点**:
- 提柜后先暂存外部堆场
- 再安排送仓卸柜
- 两段运输

**适用场景**:
- 车队有堆场设施
- 仓库产能不足
- 需要缓冲时间

**成本构成**:
```typescript
totalCost = demurrageCost + detentionCost + 
            yardStorageCost + yardOperationFee + 
            transportationCost * 2.0

// 示例
// demurrageCost: $0 (免费期内)
// detentionCost: $0 (免费期内)
// yardStorageCost: $100/天 × 2 天 = $200
// yardOperationFee: $50 (一次性操作费)
// transportationCost: $200 × 2 = $400 (往返)
// totalCost: $650
```

**优点**:
- ✅ 缓解仓库压力
- ✅ 灵活安排卸柜时间
- ✅ 可能降低总成本（避免滞港费）

**缺点**:
- ❌ 增加堆场费用
- ❌ 操作流程复杂
- ❌ 两次运输成本高

---

### 1.3 Expedited - 加急模式

**特点**:
- 免费期内加急处理
- 优先安排卸柜
- 支付加急操作费

**适用场景**:
- 即将超期（剩余 1-2 天）
- 高价值货物
- 客户紧急需求

**成本构成**:
```typescript
totalCost = handlingCost + transportationCost

// 示例
// handlingCost: $50 (加急操作费)
// transportationCost: $200
// totalCost: $250
```

**优点**:
- ✅ 避免滞港费
- ✅ 快速处理
- ✅ 客户满意度高

**缺点**:
- ❌ 加急费用较高
- ❌ 需要额外协调资源

## 二、成本优化算法

### 2.1 决策树

```
货柜到达目的港
        ↓
获取免费期截止日 (last_free_date)
        ↓
判断是否超期？
├─ 已超期 → 尽早安排（减少损失）
│         ↓
│     计算每天产生的费用
│         ↓
│     选择最早可用日期（成本最低）
│
└─ 未超期 → 是否使用 Drop off？
            ↓
        比较：堆场费 vs 滞港费
            ↓
        如果 堆场费 < 滞港费 × 0.7
            ↓
        选择 Drop off（节省 30% 以上）
            ↓
        否则选择 Direct
```

### 2.2 搜索窗口算法

**核心逻辑**:
```typescript
async suggestOptimalUnloadDate(
  containerNumber: string,
  warehouse: Warehouse,
  truckingCompany: TruckingCompany,
  basePickupDate: Date,
  lastFreeDate?: Date
): Promise<OptimizationSuggestion> {
  
  // 1. 获取准确的免费期截止日
  const demurrageResult = await this.demurrageService.calculateForContainer(containerNumber);
  const effectiveLastFreeDate = demurrageResult.result?.calculationDates?.lastPickupDateComputed || lastFreeDate;
  
  // 2. 计算当前方案成本
  const currentOption: UnloadOption = {
    containerNumber,
    warehouse,
    plannedPickupDate: basePickupDate,
    strategy: truckingCompany.hasYard ? 'Drop off' : 'Direct',
    truckingCompany,
    isWithinFreePeriod: basePickupDate <= effectiveLastFreeDate
  };
  const originalCost = (await this.evaluateTotalCost(currentOption)).totalCost;
  
  // 3. 向前/向后探索±7 天
  const candidates = [];
  for (let offset = -7; offset <= 7; offset++) {
    const candidateDate = dateTimeUtils.addDays(basePickupDate, offset);
    
    // 跳过过去和周末
    if (candidateDate < new Date() || this.isWeekend(candidateDate)) {
      continue;
    }
    
    // 检查仓库档期
    if (!await this.isWarehouseAvailable(warehouse, candidateDate)) {
      continue;
    }
    
    // 评估不同策略
    const strategies = [
      'Direct',
      ...(truckingCompany.hasYard ? ['Drop off'] : []),
      ...(candidateDate <= effectiveLastFreeDate ? ['Expedited'] : [])
    ];
    
    for (const strategy of strategies) {
      const option: UnloadOption = {
        containerNumber,
        warehouse,
        plannedPickupDate: candidateDate,
        strategy,
        truckingCompany
      };
      
      const breakdown = await this.evaluateTotalCost(option);
      candidates.push({
        pickupDate: candidateDate,
        strategy,
        totalCost: breakdown.totalCost
      });
    }
  }
  
  // 4. 找到成本最低的方案
  const optimalCandidate = candidates.reduce(
    (min, curr) => curr.totalCost < min.totalCost ? curr : min
  );
  
  const optimizedCost = optimalCandidate.totalCost;
  const savings = originalCost - optimizedCost;
  const savingsPercent = originalCost > 0 ? (savings / originalCost) * 100 : 0;
  
  // 5. 返回前 3 个最优方案
  const sortedCandidates = candidates
    .sort((a, b) => a.totalCost - b.totalCost)
    .slice(0, 3);
  
  return {
    suggestedPickupDate: optimalCandidate.pickupDate,
    suggestedStrategy: optimalCandidate.strategy,
    originalCost,
    optimizedCost,
    savings,
    savingsPercent,
    alternatives: sortedCandidates
  };
}
```

### 2.3 成本计算公式

#### Direct 模式

```typescript
async evaluateDirectCost(option: UnloadOption): Promise<CostBreakdown> {
  const { plannedPickupDate, warehouse, truckingCompany } = option;
  const plannedUnloadDate = plannedPickupDate; // 提=卸
  
  // 计算还箱日
  const plannedReturnDate = this.calculateReturnDate(
    plannedUnloadDate,
    'Live load',
    truckingCompany
  );
  
  // 使用 DemurrageService 计算 D&D 费用
  const totalCostResult = await this.demurrageService.calculateTotalCost(
    option.containerNumber,
    {
      mode: 'forecast',
      plannedDates: {
        plannedPickupDate,
        plannedUnloadDate,
        plannedReturnDate
      },
      includeTransport: true,
      warehouse,
      truckingCompany,
      unloadMode: 'Live load'
    }
  );
  
  return {
    demurrageCost: totalCostResult.demurrageCost,
    detentionCost: totalCostResult.detentionCost,
    storageCost: totalCostResult.storageCost,
    transportationCost: totalCostResult.transportationCost,
    yardStorageCost: 0,
    handlingCost: 0,
    totalCost: totalCostResult.totalCost
  };
}
```

#### Drop off 模式

```typescript
async evaluateDropOffCost(option: UnloadOption): Promise<CostBreakdown> {
  const { plannedPickupDate, warehouse, truckingCompany } = option;
  
  // Drop off: 提柜后 2 天卸柜
  const plannedUnloadDate = dateTimeUtils.addDays(plannedPickupDate, 2);
  
  // 计算还箱日
  const plannedReturnDate = this.calculateReturnDate(
    plannedUnloadDate,
    'Drop off',
    truckingCompany
  );
  
  // 使用 DemurrageService 计算 D&D 费用
  const totalCostResult = await this.demurrageService.calculateTotalCost(
    option.containerNumber,
    {
      mode: 'forecast',
      plannedDates: {
        plannedPickupDate,
        plannedUnloadDate,
        plannedReturnDate
      },
      includeTransport: true,
      warehouse,
      truckingCompany,
      unloadMode: 'Drop off'
    }
  );
  
  // 计算堆场费用
  const yardStorageDays = dateTimeUtils.daysBetween(
    plannedPickupDate,
    plannedUnloadDate
  );
  
  const tpMapping = await this.truckingPortMappingRepo.findOne({
    where: {
      truckingCompanyId: truckingCompany.companyCode,
      portCode: option.portCode,
      isActive: true
    }
  });
  
  const yardDailyRate = Number(tpMapping?.standardRate || 100);
  const yardOperationFee = Number(tpMapping?.yardOperationFee || 50);
  const yardStorageCost = (yardStorageDays * yardDailyRate) + yardOperationFee;
  
  return {
    demurrageCost: totalCostResult.demurrageCost,
    detentionCost: totalCostResult.detentionCost,
    storageCost: totalCostResult.storageCost,
    transportationCost: totalCostResult.transportationCost,
    yardStorageCost,
    handlingCost: 0,
    totalCost: totalCostResult.totalCost + yardStorageCost
  };
}
```

#### Expedited 模式

```typescript
async evaluateExpeditedCost(option: UnloadOption): Promise<CostBreakdown> {
  const { plannedPickupDate, warehouse, truckingCompany } = option;
  const plannedUnloadDate = plannedPickupDate; // 提=卸
  
  // 计算还箱日
  const plannedReturnDate = this.calculateReturnDate(
    plannedUnloadDate,
    'Live load',
    truckingCompany
  );
  
  // 使用 DemurrageService 计算 D&D 费用（免费期内为 0）
  const totalCostResult = await this.demurrageService.calculateTotalCost(
    option.containerNumber,
    {
      mode: 'forecast',
      plannedDates: {
        plannedPickupDate,
        plannedUnloadDate,
        plannedReturnDate
      },
      includeTransport: true,
      warehouse,
      truckingCompany,
      unloadMode: 'Live load'
    }
  );
  
  // 加急操作费
  const handlingConfig = await this.schedulingConfigRepo.findOne({
    where: { configKey: 'expedited_handling_fee' }
  });
  const handlingCost = Number(handlingConfig?.configValue || 50);
  
  return {
    demurrageCost: 0, // 免费期内
    detentionCost: 0, // 免费期内
    storageCost: 0,   // 免费期内
    transportationCost: totalCostResult.transportationCost,
    yardStorageCost: 0,
    handlingCost,
    totalCost: totalCostResult.transportationCost + handlingCost
  };
}
```

## 五、常见问题排查

### 5.1 问题 1: 优化建议不准确

**现象**:
```
优化建议的成本与实际计算不符
```

**排查步骤**:

```sql
-- Step 1: 检查滞港费标准配置
SELECT * FROM ext_demurrage_standard
WHERE port_code = 'USLAX'
  AND is_active = true
ORDER BY free_days, tier_days;

-- Step 2: 检查货柜的免费期
SELECT 
  po.last_free_date,
  po.available_time,
  c.container_number
FROM process_port_operations po
JOIN biz_containers c ON c.container_number = po.container_number
WHERE c.container_number = 'HMMU6232153'
  AND po.port_type = 'destination';

-- Step 3: 检查排柜配置
SELECT * FROM dict_scheduling_config
WHERE config_key IN (
  'expedited_handling_fee',
  'external_storage_daily_rate',
  'yard_operation_fee'
);
```

**常见原因**:
- ❌ 滞港费标准配置错误
- ❌ 免费期数据不准确
- ❌ 堆场费率配置缺失

**解决方案**:
- ✅ 校正 ext_demurrage_standard 数据
- ✅ 更新 process_port_operations.last_free_date
- ✅ 补充 dict_scheduling_config 配置

### 5.2 问题 2: 找不到可行方案

**现象**:
```
优化结果为空：No feasible solution found
```

**排查步骤**:

```sql
-- Step 1: 检查仓库档期
SELECT 
  warehouse_code,
  date,
  total_capacity,
  used_capacity,
  remaining_capacity
FROM ext_warehouse_daily_occupancy
WHERE warehouse_code = 'WH001'
  AND date BETWEEN '2026-03-20' AND '2026-04-20'
ORDER BY date;

-- Step 2: 检查车队映射
SELECT 
  wtm.warehouse_code,
  wtm.trucking_company_id,
  tpm.port_code,
  tpm.has_yard
FROM dict_warehouse_trucking_mapping wtm
LEFT JOIN dict_trucking_port_mapping tpm 
  ON wtm.trucking_company_id = tpm.trucking_company_id
WHERE wtm.warehouse_code = 'WH001'
  AND wtm.country = 'US';
```

**常见原因**:
- ❌ 仓库档期已满（30 天内无产能）
- ❌ 车队 - 港口映射缺失
- ❌ 搜索窗口太小（默认±7 天）

**解决方案**:
- ✅ 增加仓库产能或调整休息日
- ✅ 补充车队映射关系
- ✅ 扩大搜索窗口至±14 天

### 5.3 问题 3: Drop off 总是被选中

**现象**:
```
即使成本更高，也总是推荐 Drop off 模式
```

```sql

  trucking_company_id,
  port_code,
  standard_rate,
  yard_operation_fee

SELECT * FROM dict_scheduling_config
WHERE config_key = 'dropoff_threshold_percent';
```

**常见原因**:
- ❌ 车队 has_yard=true 但实际无堆场
- ❌ 堆场费率配置过低
- ❌ Drop off 阈值设置不合理

**解决方案**:
- ✅ 校正车队 has_yard 字段
- ✅ 调整 standard_rate 和 yard_operation_fee
- ✅ 修改阈值为 0.5（更严格）
