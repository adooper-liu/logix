## 一、目标与范围

### 1.1 业务目标

1. **费用计算**：对每个货柜可计算或预估  
   - **滞港费**（Demurrage）：在港/码头超出免费期产生的费用  
   - **滞箱费**（Detention）：用箱超出船东免费用箱期产生的费用  
   - **港口堆存费**（Port Storage）：港口堆场堆存超出免费期产生的费用  

2. **排期规划**：在以下约束下，为每个货柜规划：  
   - **提柜日期**（何时从港口提柜）：直送仓库时 = 送仓日；先送外部堆场时 = 送外部堆场日。  
   - **送仓日期**（**仅指何时送达仓库**）：直送时为当日到仓的日期；经外部堆场时为从堆场送到仓库的日期。  
   - **卸柜日期**（何时在仓库完成卸柜）  
   - **还空箱日期**（何时还空箱）  

3. **约束条件**：  
   - 仓库**卸柜能力**（每日可卸柜数/门数）  
   - 港口**免费期**（last_free_date 等，避免滞港）  
   - 船东**免费用箱天数**（避免滞箱费）  

4. **缓冲与优化**：  
   - 可借助**外部堆场**暂存，缓解仓库压力  
   - 在「滞港费 + 滞箱费 + 港口堆存费」与「外部堆场费用」之间做**成本最优**决策  
   - 输出**拖车路线**：直送仓库 vs 先到外部堆场 → 待仓库有空再送仓  

### 1.2 输出物（ per 柜）

| 输出项 | 说明 |
|--------|------|
| 提柜日 | 从港口提柜的日期。直送仓库时 = 送仓日；先送外部堆场时 = 送外部堆场日。 |
| 送仓日期 | **仅指送达（目标）仓库的日期**：直送 = 到仓日；经堆场 = 从外部堆场送到仓库的日期。 |
| 卸柜日期 | 在仓库完成卸柜的日期（若经外部堆场，则为送仓后的卸柜日） |
| 还箱日期 | 空箱还回船东/堆场的日期 |
| 拖车路线 | 直送仓库 / 先到外部堆场再送仓（含在堆场停留天数或送仓顺序） |

## 二、费用口径与数据来源

### 2.1 滞港费（Demurrage）

- **含义**：货柜在港口停留超过免费期产生的费用；从到港/卸船到提柜（或计算日）的停留中，超出免费期的部分按费率计费。详见 [01-DEMURRAGE_LOGIC_FROM_CONTAINER_SYSTEM.md](./01-DEMURRAGE_LOGIC_FROM_CONTAINER_SYSTEM.md)。  

- **起算日（免费期开始）**：由「计算方式」决定（与 container-system 一致）：  
  - **按到港**：优先 **ATA**（`ata_dest_port`），无则 **ETA**（`eta_dest_port`）；若有**修正 ETA**（`eta_correction`）且无 ATA 时可用修正 ETA。  
  - **按卸船**：**卸船日**（`dest_port_unload_date` / 目的港卸船时间）。  

- **截止日（计费截止）**：**实际提柜日**（`process_trucking_transport.pickup_date`）；无提柜日时为当前日期或计划提柜日（用于预估）。提柜后不再产生滞港费。  

- **免费期与计费天数**：  
  - **免费期结束日** = 起算日 + 免费天数（按**免费天数基准**折算，见下）；也可直接使用 `last_free_date`（当 last_free_date 按同规则计算时，等价）。  
  - **计费天数** = max(0, 截止日与免费期结束日之间的天数)，按所选基准（自然日/工作日）计算。  

- **免费天数基准类型**（与 container-system 一致，见 01-DEMURRAGE_LOGIC_FROM_CONTAINER_SYSTEM.md §三）：  

| 类型 | 免费期 | 计费期 | 说明 |
|------|--------|--------|------|
| Natural Days | 自然日 | 自然日 | 每天都计入 |
| Working Days | 工作日 | 工作日 | 周末/节假日不计 |
| **Natural+Working** | 工作日 | 自然日 | 免费期按工作日，超期后按自然日，**最常用** |
| Working+Natural | 自然日 | 工作日 | 免费期自然日，计费期工作日 |
| Working Days-Excluding Holidays | 工作日（不含法定假） | 工作日 | 明确排除法定节假日 |

- **阶梯费率**：标准中可配置多档（如 1–5 天 0 元、6–10 天 50 元/天、11–30 天 120 元/天、31+ 天 200 元/天）；计费天数按阶梯从低到高填满，每档「天数 × 日费率」累加为总滞港费。  

- **与现有数据**：  
  - `process_port_operations`：`last_free_date`、`ata_dest_port`、`eta_dest_port`、`eta_correction`、`dest_port_unload_date`。  
  - `process_trucking_transport.pickup_date`：实际提柜日（计费截止）。  
- **需配置**：滞港费标准（港口/船东 → 免费天数、免费天数基准、计算方式按到港/按卸船、日费率或阶梯表）。  

### 2.2 滞箱费（Detention）

- **含义**：货柜提走后超过免费期未还空箱产生的费用（见 [01-DEMURRAGE_LOGIC_FROM_CONTAINER_SYSTEM.md](./01-DEMURRAGE_LOGIC_FROM_CONTAINER_SYSTEM.md) 术语表）。用箱时间超出船东免费用箱期（如 7/10/14 天），按超期天数计费。  

- **计算口径（与 last_free_date 一致，需配套更新机制）**：  
  - **默认**：按**最晚提柜日 last_free_date**（目的港）作为用箱起算参考；免费用箱截止日 = last_free_date + **滞箱费标准中的免费天数**。  
  - **若有实际提柜日**（`process_trucking_transport.pickup_date`）：则用**实际提柜日** + 滞箱费标准中的免费天数，得到「免费用箱截止日」；还箱日晚于该日则产生滞箱费。  

- **与现有数据**：  
  - 起算/参考：**last_free_date**（无实际提柜时）或**实际提柜日**（有则用）。  
  - 还箱日：`process_empty_return.last_return_date` 或 `return_time`。  
  - 滞箱费标准：需配置船东/航线/柜型 → 免费天数、免费天数基准（可选）、费率或阶梯。  

- **计算思路**：  
  - 免费用箱截止日 =（实际提柜日 ?? last_free_date）+ 免费天数（若支持工作日基准，需按同一套免费天数基准类型折算）。  
  - 滞箱费超期天数 = max(0, 还箱日 − 免费用箱截止日)，按日费率或阶梯计。  

### 2.3 港口堆存费（Port Storage）

- **含义**：在港口堆场堆存超免费期后的堆存费；在 container-system 中常作为独立费用项「Storage Charge」与滞港费分别配置（见 01-DEMURRAGE_LOGIC_FROM_CONTAINER_SYSTEM.md §四 多费用项）。  
- **与现有数据**：  
  - 同滞港：可用 `last_free_date`、卸船/可用时间 + 配置免费天数；计算方式可为「按到港」或「按卸船」。  
- **计算思路**：  
  - 与滞港类似：免费期结束日由起算日（到港/卸船）+ 免费天数（及免费天数基准）得到；按「提柜日 vs 免费截止日」算超期天数，再按堆存费率或阶梯计。  
  - 若业务上滞港与堆存为同一张单，可合并为一个费用项，仅展示名称不同。  

### 2.4 外部堆场费用

- **含义**：柜子先拉到外部堆场暂存，按天或按柜计费，直到送仓。  
- **与现有数据**：  
  - 当前无「外部堆场」实体，需新增：堆场主数据、每日/每柜费率、容量（可选）。  
- **计算思路**：  
  - 停留天数 × 日费率（或按柜一口价），与「滞港+滞箱+港口堆存」加总对比，用于优化决策。

## 三、约束与能力建模

### 3.1 港口免费期与 last_free_date 计算

- **已有**：`process_port_operations.last_free_date`（目的港）。  
- **使用**：规划提柜日时，尽量 ≤ last_free_date，否则计入滞港/堆存成本。  

#### last_free_date 计算口径（建议，与 01-DEMURRAGE_LOGIC_FROM_CONTAINER_SYSTEM 对齐）

**基准日（免费期起算日）**：由标准中的「计算方式」决定。  

- **按到港**（与 2.1 一致）：按优先级取一个  
  1. **实际到港日**（`ata_dest_port`）或**实际卸船日**（`dest_port_unload_date`）→ 用实际到港日或实际卸船日（业务可约定优先其一，如优先卸船日）；  
  2. 否则**修正 ETA**（`eta_correction`）→ 用修正 ETA 的日期；  
  3. 否则 **ETA**（`eta_dest_port`）。  
- **按卸船**：仅用**实际卸船日**（`dest_port_unload_date`）；若无则不可算 last_free_date 或退回按到港逻辑。  

**免费期结束日 last_free_date**：由「基准日 + 免费天数」按**免费天数基准**折算得到（见 2.1 表）：  

- **Natural Days**：last_free_date = 基准日 + (免费天数 − 1) 的自然日。  
- **Working Days**：从基准日起数第 N 个**工作日**对应的日期。  
- **Natural+Working**：免费期按工作日数满 N 天，得到结束日（最常用）。  

即：先根据计算方式取基准日（ETA/修正 ETA/ATA/卸船），再根据滞港费标准中的免费天数与免费天数基准得到 last_free_date。  

#### 当前是否有更新与计算机制

- **现状**：当前代码**没有**对 `last_free_date` 的自动计算与写回。  
  - 数据来源为 **Excel 导入**或人工维护，直接写入 `process_port_operations.last_free_date`。  
  - 飞驼适配器会更新 `ata_dest_port`、`eta_dest_port`、`dest_port_unload_date` 等，但**不会**根据这些字段重算并更新 `last_free_date`。  
- **建议**：增加 **last_free_date 计算与更新机制**：  
  - **触发时机**：目的港港口操作写入或更新后（如 Excel 导入后、飞驼同步写库后、单柜/批量状态更新后），对**目的港**记录按上述口径算一次基准日，再根据配置的免费天数得到 `last_free_date`，写回同一条 `process_port_operations`。  
  - **配置**：滞港费标准（港口/船东 → 免费天数），若暂无则用全局默认免费天数。  
  - **覆盖规则**：若业务希望「仅在没有手工填写 last_free_date 时自动计算」，可做成：仅当当前 `last_free_date` 为空时才写回；或提供「强制重算」接口。  

#### 扩展：标准匹配（与 container-system 一致）

- 若引入滞港费标准表，可复用 **四字段 + 有效期** 匹配（见 01-DEMURRAGE_LOGIC_FROM_CONTAINER_SYSTEM.md §四）：**进口国**通过标准表 **foreign_company_code / foreign_company_name**（境外/分公司）识别；目的港、船公司、货代对应 **destination_port_code**、**shipping_company_code**、**origin_forwarder_code**；`effective_date <= 当前` 且（`expiry_date` 为空或 >= 当前）。匹配到多条时可为多费用项（滞港费、堆存费、滞箱费等）分别取免费天数与费率。  
- 配置表建议字段：上述四维度（foreign_company、destination_port、shipping_company、origin_forwarder）、免费天数、**免费天数基准**、**计算方式**（按到港/按卸船）、日费率或阶梯 JSON。  

### 3.2 仓库卸柜能力

- **含义**：每个仓库（或仓库组）每日可卸柜数量或卸柜门数上限。  
- **现状**：有 `dict_warehouses`、`process_warehouse_operations`（含 planned_warehouse、planned_unload_date 等），无「每日容量」字段。  
- **建议**：  
  - 新增「仓库/仓库组 × 日期」的**每日卸柜能力**（如每日最大卸柜数），或按周/月容量 + 简单均分。  
  - 规划卸柜日时：同一仓库同一天的「计划卸柜数」≤ 当日能力。  

### 3.3 外部堆场容量（可选）

- **含义**：堆场每日可接收/暂存柜数上限，避免无限制缓冲。  
- **建议**：堆场主数据中增加「日容量」或「总容量」，排程时堆场每日占用量 ≤ 容量。  

### 3.4 船东免费用箱天数与滞箱费免费截止日

- **含义**：从提柜（或 last_free_date）到还箱的免费用箱天数，超则滞箱费；见 2.2 口径。  
- **建议**：配置表（船东/航线/柜型 → 免费天数、费率），用于滞箱费计算与「还箱日」约束。  

#### 滞箱费免费截止日：是否也要更新与计算机制

- **建议：要，与 last_free_date 同套思路。**  
  - **口径**：免费用箱截止日 = **实际提柜日**（有则用 `process_trucking_transport.pickup_date` 的日期）+ **滞箱费标准免费天数**；若无实际提柜日，则用目的港 **last_free_date** + 滞箱费标准免费天数。  
  - **触发时机**：与 last_free_date 类似，在以下任一发生后对受影响柜号重算并写回：  
    - 目的港 **last_free_date** 更新（含自动计算写回后）；  
    - **实际提柜日**写入或更新（拖车表 `pickup_date`）；  
    - 还箱记录写入/更新（若业务需要按还箱日反推展示「是否已超免费用箱期」）。  
  - **存储**：可新增字段，如 `process_empty_return.last_free_return_date`（免费用箱截止日 / 最晚免滞箱费还箱日），便于列表/统计/提醒与 last_free_date 一致；若暂不落库，也可仅做实时计算用于费用估算。  
  - **配置**：与 last_free_date 共用或单独「滞箱费标准」（船东/航线/柜型 → 免费天数），若与滞港费免费天数不同则分开配置。

## 四、优化问题抽象

### 4.1 决策变量（per 柜）

- **提柜日** \( d_{pickup} \)：从港口提柜的日期。直送仓库时 = 送仓日；先送外部堆场时 = 送外部堆场日。  
- **送仓日** \( d_{delivery} \)：**仅指送达仓库的日期**。直送 = 到仓日；经堆场 = 从堆场送到仓库的日期。  
- **卸柜日** \( d_{unload} \)：在目标仓库的卸柜日。  
- **还箱日** \( d_{return} \)。  
- **路线** \( r \in \{ \text{直送仓库}, \text{先外部堆场再送仓} \} \)  
  - 直送：提柜日 = 送仓日。  
  - 经堆场：可增加「送外部堆场日」（= 提柜日）、「送仓日」（从堆场送仓），便于算堆场费用与容量。  

### 4.2 约束（简要）

1. **时间顺序**：  
   - 直送：提柜日 = 送仓日 ≤ 卸柜日 ≤ 还箱日。  
   - 经堆场：提柜日 = 送外部堆场日 &lt; 送仓日 ≤ 卸柜日 ≤ 还箱日。  
2. **港口免费期**：  
   - 提柜日 &gt; last_free_date 时，产生滞港/堆存费用（见上）。  
3. **仓库能力**：  
   - 按仓库+日期汇总计划卸柜数，不超过当日卸柜能力。  
4. **船东免费用箱**：  
   - 还箱日 − 起算日 尽量 ≤ 免费天数，否则计入滞箱费。  
5. **外部堆场容量**（若启用）：  
   - 堆场每日占用量 ≤ 容量。  

### 4.3 目标函数（最小化总成本）

- **方案 A（直送仓库）**：  
  - 滞港费 + 滞箱费 + 港口堆存费（由提柜日、还箱日等推导）+ 拖车费（港口→仓库）。  
- **方案 B（经外部堆场）**：  
  - 滞港费（可能因更早提柜而减少）+ 滞箱费 + 港口堆存费 + 拖车费（港口→堆场 + 堆场→仓库）+ 外部堆场费。  
- **优化**：对每个柜在 A/B 及不同日期组合下估算总成本，在满足约束下选总成本最小的方案，并输出对应提柜日、送仓日、卸柜日、还箱日与路线。  

可先做**规则+启发式**（如：优先在 last_free_date 前提柜、按仓库能力排队定卸柜日、比较 A/B 总价），再考虑整数规划/排程算法做批量优化。

## 五、与现有系统衔接

### 5.1 已有表与字段（可直接用）

| 表 | 字段 | 用途 |
|----|------|------|
| process_port_operations | last_free_date, ata_dest_port, dest_port_unload_date, available_time | 免费期、滞港/滞箱起算 |
| process_trucking_transport | planned_pickup_date, pickup_date, planned_delivery_date, delivery_date | 提柜/送仓计划与实际 |
| process_warehouse_operations | planned_unload_date, unload_date, warehouse_id, planned_warehouse | 卸柜计划与实际、仓库 |
| process_empty_return | last_return_date, return_time | 还箱计划与实际 |

### 5.2 建议新增或扩展

1. **费用与规则配置**（新表或配置模块，与 01-DEMURRAGE_LOGIC_FROM_CONTAINER_SYSTEM 对齐）  
   - **滞港费/堆存费标准**：港口/船东（或四字段：进口国、目的港、船公司、货代）、**免费天数**、**免费天数基准**（Natural Days / Working Days / Natural+Working）、**计算方式**（按到港/按卸船）、日费率或**阶梯费率**（如 minDays、maxDays、rate）、有效期（effective_date、expiry_date）。  
   - **滞箱费标准**：船东/航线/柜型 → 免费用箱天数、免费天数基准（可选）、费率或阶梯。  
   - **外部堆场**：日费率或按柜费、可选日容量。  
   - 多费用项时可按**收费标志**（is_chargeable）区分是否计费；分别计算后合计。  

2. **仓库能力**  
   - 在 `dict_warehouses` 或新表「仓库日历」：日期 + 每日最大卸柜数（或按门数折算）。  

3. **外部堆场主数据**  
   - 堆场编码、名称、日容量、费率、与仓库的距离/拖车成本（若需）。  

4. **排程结果存储**（可选）  
   - 可为每个柜存「推荐提柜日、送仓日、卸柜日、还箱日、路线、预估费用」等，便于与人工调整后的实际值对比。  
   - 可放在 process_trucking_transport / process_warehouse_operations / process_empty_return 的「计划」字段，或单独一张「排程建议」表（container_number + 各建议日期 + 路线 + 费用摘要）。

## 六、实施阶段建议

### 阶段 1：费用计算与展示（不改变排程）

- 明确滞港、滞箱、港口堆存的口径与配置（免费天数、费率）。  
- 用现有 last_free_date、ata、卸船、还箱日等，实现「当前/计划下的费用估算」逻辑。  
- 在货柜详情或列表展示：预估滞港费、滞箱费、堆存费（及合计）。  
- **不引入**外部堆场与优化排程。  
- **可选**：实现 **last_free_date 计算与更新**（见 3.1）：在导入/飞驼同步后根据 ETA / 修正 ETA / ATA / 卸船日 + 免费天数写回目的港 last_free_date，便于后续统计与费用一致。  
- **可选**：实现 **滞箱费免费截止日**的计算与更新（见 3.4）：有实际提柜日则用实际提柜日 + 滞箱费免费天数，否则用 last_free_date + 滞箱费免费天数；在 last_free_date 或实际提柜日更新时触发，可写回 `process_empty_return` 新字段（如 last_free_return_date）或仅实时计算用于费用展示。  

### 阶段 2：约束与单柜排期建议

- 配置仓库卸柜能力、船东免费用箱天数。  
- 在「给定当前计划」下，校验是否超免费期、是否超仓库能力；若超，给出「建议提柜日/卸柜日/还箱日」的简单规则（如：尽量 last_free_date 前提柜、按能力排队卸柜）。  
- 仍可先不建外部堆场，仅直送仓库。  

### 阶段 3：外部堆场与成本对比

- 维护外部堆场主数据与费率。  
- 对每个柜计算两种成本：直送 vs 经堆场（含堆场停留天数估算），展示对比与推荐路线。  
- 可选：支持「先到堆场再送仓」的计划日期（堆场到达日、送仓日、卸柜日、还箱日）。  

### 阶段 4：批量优化与自动排程

- 在满足仓库能力、堆场容量、免费期等约束下，对多柜做批量优化（启发式或整数规划），输出每柜的提柜日、送仓日、卸柜日、还箱日与路线。  
- 结果写入「排程建议」或写入现有计划字段，供人工确认与执行。

## 七、小结

- **费用**：滞港、滞箱、港口堆存、外部堆场，依赖港口免费期、卸船/还箱日及配置的费率和免费天数；滞港费起算按「按到港」或「按卸船」，计费截止为实际提柜日，免费期按**免费天数基准**（自然日/工作日/Natural+Working）折算，详见 [01-DEMURRAGE_LOGIC_FROM_CONTAINER_SYSTEM.md](./01-DEMURRAGE_LOGIC_FROM_CONTAINER_SYSTEM.md)。  
- **排期**：提柜日、送仓日、卸柜日、还箱日，受港口免费期、仓库卸柜能力、船东免费用箱天数、可选的外部堆场容量约束。  
- **优化**：在「滞港+滞箱+堆存」与「外部堆场+二次拖车」之间做成本最优，并输出拖车路线（直送 vs 经堆场）。  
- **落地**：先做费用计算与展示，再加约束与单柜建议，再引入外部堆场与成本对比，最后做批量优化与自动排程；数据上复用 process_port_operations、process_trucking_transport、process_warehouse_operations、process_empty_return，并补充配置表（含免费天数基准、计算方式、阶梯）与仓库/堆场能力。
