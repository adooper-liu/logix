## 📋 概述

货柜（Container）是 LogiX 物流系统的核心实体，贯穿从备货、海运、港口操作、拖车运输、仓库卸货到还箱的全生命周期。

### 核心价值

1. **单一真相源**: `biz_containers` 表是货柜数据的权威源
2. **全链路追踪**: 通过 container_number 关联所有流程表
3. **状态快照**: `logistics_status` 和 `gantt_derived` 落库保存计算结果
4. **多维度汇总**: 支持按订单、国家、状态等多维度统计

### 1.2 biz_containers 表结构详解

```sql
CREATE TABLE biz_containers (
    -- ========== 主键与基本信息 ==========
    container_number      VARCHAR(100) PRIMARY KEY,     -- 集装箱号（主键）
    order_number          VARCHAR(100) NOT NULL,        -- 备货单号（外键）
    container_type_code   VARCHAR(50) NOT NULL,         -- 柜型编码（20GP/40HQ 等）

    -- ========== 货物信息 ==========
    cargo_description     VARCHAR(500),                 -- 货物描述
    gross_weight          NUMERIC(10,2),                -- 毛重（KG）
    net_weight            NUMERIC(10,2),                -- 净重（KG）
    cbm                   NUMERIC(10,3),                -- 体积（CBM）
    packages              INTEGER,                      -- 件数
    seal_number           VARCHAR(100),                 -- 封条号

    -- ========== 业务标记 ==========
    inspection_required   BOOLEAN DEFAULT FALSE,        -- 是否需要查验
    is_unboxing           BOOLEAN DEFAULT FALSE,        -- 是否开箱
    requires_pallet       BOOLEAN DEFAULT FALSE,        -- 是否需要打托
    requires_assembly     BOOLEAN DEFAULT FALSE,        -- 是否需要组装
    is_rolled             BOOLEAN DEFAULT FALSE,        -- 是否甩柜

    -- ========== 物流状态（状态机计算结果） ==========
    logistics_status      VARCHAR(20),                  -- 简化状态（7 层）
    gantt_derived         JSONB,                        -- 甘特派生快照
    schedule_status       VARCHAR(20),                  -- 排产状态

    -- ========== 物理属性 ==========
    over_length           NUMERIC(8,2),                 -- 超长（CM）
    over_height           NUMERIC(8,2),                 -- 超高（CM）
    danger_class          VARCHAR(10),                  -- 危险品等级

    -- ========== 人员信息 ==========
    operator              VARCHAR(100),                 -- 操作员
    container_holder      VARCHAR(200),                 -- 持柜人

    -- ========== 元数据 ==========
    created_at            TIMESTAMPTZ DEFAULT NOW(),
    updated_at            TIMESTAMPTZ DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_containers_order ON biz_containers(order_number);
CREATE INDEX idx_containers_status ON biz_containers(logistics_status);
CREATE INDEX idx_containers_type ON biz_containers(container_type_code);
```

---

#### 核心字段说明

**container_number** (VARCHAR(100)):

- **用途**: 集装箱号，全球唯一
- **格式**: 4 个字母 + 7 个数字（如 `HMMU6232153`）
- **特点**:
  - 主键，全局唯一标识
  - 所有流程表的外键
  - 一旦创建不可修改

**logistics_status** (VARCHAR(20)):

- **用途**: 物流状态机计算的当前状态
- **枚举值**:
  - `not_shipped` - 未出运
  - `shipped` - 已出运
  - `in_transit` - 在途
  - `at_port` - 已到港
  - `picked_up` - 已提柜
  - `unloaded` - 已卸柜
  - `returned_empty` - 已还箱
- **更新机制**: 由 `ContainerStatusService.updateStatus()` 自动计算并更新

**gantt_derived** (JSONB):

- **用途**: 甘特图五阶段快照
- **结构**: 详见"甘特派生快照"章节
- **更新机制**: 与 `logistics_status` 同步计算并落库

#### process_port_operations（港口操作表）

```sql
CREATE TABLE process_port_operations (
    id                  SERIAL PRIMARY KEY,
    container_number    VARCHAR(100) NOT NULL,          -- FK
    port_code           VARCHAR(50),                    -- 港口代码
    port_name           VARCHAR(200),                   -- 港口名称
    port_type           VARCHAR(20),                    -- origin/transit/destination
    port_sequence       INTEGER,                        -- 港口顺序

    -- 时间字段
    eta                 DATE,                           -- 预计到港
    ata                 DATE,                           -- 实际到港
    etd                 DATE,                           -- 预计离港
    atd                 DATE,                           -- 实际离港

    -- 特殊字段
    available_time      DATE,                           -- 可提货时间（PCAB 触发）
    last_free_date      DATE,                           -- 最晚免费日
    customs_status      VARCHAR(20),                    -- 清关状态

    -- 元数据
    data_source         VARCHAR(50),                    -- 数据来源
    created_at          TIMESTAMPTZ DEFAULT NOW(),
    updated_at          TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_port_ops_container ON process_port_operations(container_number);
CREATE INDEX idx_port_ops_type ON process_port_operations(port_type);
```

**核心字段**:

- `port_type`: 起运港/中转港/目的港，状态机只统计目的港
- `ata`: 实际到港时间，触发 `at_port` 状态（优先级 4）
- `availableTime`: 可提货时间，也可触发 `at_port` 状态（优先级 4a）
- `last_free_date`: 最晚提柜免费日，用于滞港费计算和倒计时

- `pickup_date`: 提柜日，触发 `picked_up` 状态（优先级 3）
- `planned_pickup_date`: 智能排柜引擎生成的计划日期
- `gate_out_time`: 码头出场时间，自动同步到 `pickup_date`

### 2.2 GanttDerived 快照

```typescript
export interface GanttDerived {
  phase: 1 | 2 | 3 | 4 | 5 // 五阶段
  phaseLabel: string // 清关/提柜/卸柜/还箱/完成
  primaryNode: GanttNodeKey | null // 主节点 key
  nodes: GanttDerivedNode[] // 节点数组
  ruleVersion: string // 规则版本
  derivedAt: string // 计算时间
}

export type GanttNodeKey = 'customs' | 'pickup' | 'unload' | 'return'

export interface GanttDerivedNode {
  key: GanttNodeKey // 节点 key
  taskRole: 'main' | 'dashed' | 'none' // 任务角色
  completed: boolean // 是否已完成
  plannedDate?: string | null // 计划日期（ISO 字符串）
  actualDate?: string | null // 实际日期（ISO 字符串）
}
```

**五阶段详解**:

| 阶段  | phase | phaseLabel | primaryNode | 说明               |
| ----- | ----- | ---------- | ----------- | ------------------ |
| **1** | 1     | 清关       | customs     | 准备清关资料、申报 |
| **2** | 2     | 提柜       | pickup      | 安排车队提柜       |
| **3** | 3     | 卸柜       | unload      | 货物送达仓库卸柜   |
| **4** | 4     | 还箱       | return      | 空箱归还堆场       |
| **5** | 5     | 完成       | -           | 全部流程结束       |

```typescript
// backend/src/services/container.service.ts

async getContainerWithDetails(containerNumber: string): Promise<ContainerWithDetails> {
  // 1. 查询货柜主表
  const container = await this.containerRepository.findOne({
    where: { containerNumber }
  });

  // 2. 并行查询所有关联表
  const [seaFreight, portOperations, truckingTransports,
         warehouseOperations, emptyReturns] = await Promise.all([
    this.seaFreightRepository.findOne({ where: { containerNumber } }),
    this.portOperationRepository.find({
      where: { containerNumber },
      order: { portSequence: 'ASC' }
    }),
    this.truckingTransportRepository.find({ where: { containerNumber } }),
    this.warehouseOperationRepository.find({ where: { containerNumber } }),
    this.emptyReturnRepository.find({ where: { containerNumber } })
  ]);

  // 3. 合并返回
  return {
    container,
    seaFreight,
    portOperations,
    truckingTransports,
    warehouseOperations,
    emptyReturns
  };
}
```

#### 复杂查询（JOIN 多表）

```sql
-- 查询所有已到港的货柜及其详细信息
SELECT
  c.container_number,
  c.logistics_status,
  sf.bill_of_lading_number,
  sf.vessel_name,
  po.port_name,
  po.ata,
  tt.pickup_date
FROM biz_containers c
LEFT JOIN process_sea_freight sf
  ON c.container_number = sf.container_number
LEFT JOIN process_port_operations po
  ON c.container_number = po.container_number
  AND po.port_type = 'destination'
LEFT JOIN process_trucking_transport tt
  ON c.container_number = tt.container_number
WHERE c.logistics_status = 'at_port'
ORDER BY po.ata DESC;
```

### 4.1 国家概念统一

**业务背景**:

- `country` 字段实质指该国分公司（子公司类型客户）
- 统一存 `dict_countries.code`（国家代码）
- `sell_to_country` 特例存子公司名称

**实现方式**:

```typescript
// backend/src/middleware/scopedCountry.middleware.ts

export function scopedCountryMiddleware(req: Request, res: Response, next: NextFunction) {
  // 从请求头读取国家代码
  const countryCode = req.headers['x-country-code'] as string

  if (countryCode) {
    // 写入请求上下文，供后续查询使用
    ;(req as any).scopedCountryCode = countryCode
  }

  next()
}
```

### 4.3 前端请求拦截器

```typescript
// frontend/src/services/container.ts

class ContainerService {
  constructor() {
    this.api = axios.create({
      baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:3001/api/v1',
      timeout: 120000,
    })

    // 请求拦截器
    this.api.interceptors.request.use(config => {
      // 1. Token 认证
      const token = localStorage.getItem('token')
      if (token) {
        config.headers.Authorization = `Bearer ${token}`
      }

      // 2. 全局国家筛选
      const appStore = useAppStore()
      if (appStore.scopedCountryCode) {
        config.headers['X-Country-Code'] = appStore.scopedCountryCode
      }

      // 3. 避免浏览器缓存
      if (config.method?.toLowerCase() === 'get') {
        config.headers['Cache-Control'] = 'no-cache'
        config.headers['Pragma'] = 'no-cache'
      }

      return config
    })
  }
}
```

### 5.1 问题 1: 货柜列表为空

**现象**: Shipments 页面加载后显示空列表

**排查步骤**:

```sql
-- Step 1: 检查是否有货柜数据
SELECT COUNT(*) FROM biz_containers;

-- Step 2: 检查国家筛选是否生效
SELECT
  container_number,
  logistics_status,
  created_at
FROM biz_containers
LIMIT 10;

-- Step 3: 检查后端日志
# 查看是否收到 X-Country-Code header
tail -f backend.log | grep "X-Country-Code"
```

**常见原因**:

- ❌ `appStore.scopedCountryCode` 为某个特定国家，但数据库中无该国家数据
- ❌ 后端中间件未正确读取 header
- ❌ 数据库确实无数据

**解决方案**:

- ✅ 清除 localStorage 中的 `logix_scoped_country_code`
- ✅ 检查前端 app Store 初始化逻辑
- ✅ 验证后端 middleware 配置

### 5.3 问题 3: 路由参数乱码

**现象**: 访问 `/containers/HMMU6232153` 时显示乱码

**排查步骤**:

```typescript
// Step 1: 检查路由配置
// router/index.ts
{
  path: '/containers/:containerNumber',
  component: ContainerDetail,
  // ✅ 确保没有错误的 decode 处理
}

// Step 2: 检查组件内解码
const route = useRoute();
const containerNumber = computed(() => {
  const p = route.params.containerNumber as string;
  return p ? decodeURIComponent(p) : ''; // ✅ 正确解码
});
```

**常见原因**:

- ❌ URL 包含特殊字符未编码
- ❌ 重复 decodeURIComponent

**解决方案**:

- ✅ 使用 `encodeURIComponent()` 编码后再传参
- ✅ 接收方使用 `decodeURIComponent()` 解码
