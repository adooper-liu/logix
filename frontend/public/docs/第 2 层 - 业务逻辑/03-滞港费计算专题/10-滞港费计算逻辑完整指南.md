## 一、核心概念与费用类型

### 1.1 四种费用类型

LogiX 系统处理以下四种港口相关费用：

| 类型 | 代码标识 | 起算日 | 截止日 | 说明 | 依赖条件 |
|------|----------|--------|--------|------|----------|
| **Demurrage** | 滞港费 | ATA/ETA/卸柜日 | 实际提柜日/当天 | 港区占用费 | 需要到港信息 |
| **Detention** | 滞箱费 | 实际/计划提柜日 | 实际/计划还箱日/当天 | 箱体占用费 | **必须有提柜日** |
| **Storage** | 堆存费 | ATA/卸柜日 | 实际/计划提柜日/当天 | 港区堆存费 | **预测模式需计划提柜日** |
| **D&D Combined** | 合并费 | ATA/ETA/卸柜日 | 实际/计划还箱日/当天 | 滞港 + 滞箱合并 | 需要到港或 ETA |

### 1.2 费用类型识别规则

系统通过 `chargeTypeCode` 和 `chargeName` 自动识别费用类型：

```typescript
// 识别逻辑（backend/src/services/demurrage.service.ts）

/** 判断是否为「Demurrage & Detention」合并费用项：一个费用项覆盖到港→还箱整段 */
function isCombinedDemurrageDetention(std: {
  chargeTypeCode?: string | null;
  chargeName?: string | null;
}): boolean {
  const code = (std.chargeTypeCode ?? '').toUpperCase();
  const name = (std.chargeName ?? '').toLowerCase();
  const hasDem = code.includes('DEMURRAGE') || name.includes('demurrage') || name.includes('滞港');
  const hasDet = code.includes('DETENTION') || name.includes('detention') || name.includes('滞箱');
  return hasDem && hasDet;
}

/** 判断是否为纯滞箱费（Detention）标准：起算日=提柜，截止日=还箱；排除合并类型 */
function isDetentionCharge(std: {
  chargeTypeCode?: string | null;
  chargeName?: string | null;
}): boolean {
  if (isCombinedDemurrageDetention(std)) return false;
  const code = (std.chargeTypeCode ?? '').toUpperCase();
  const name = (std.chargeName ?? '').toLowerCase();
  return code.includes('DETENTION') || name.includes('detention') || name.includes('滞箱');
}

/**
 * 堆存费（Storage）：港区堆存，区间为「到港侧起算日 → 提柜侧截止日」（非提柜→还箱）。
 * - forecast 且无 ATA/实际卸船：起算=修正 ETA / ETA；截止=max(计划提柜日, 当天)。
 * - actual，或 forecast 但已有 ATA（或按标准已有实际卸船日）：起算=「按到港」用 ATA，「按卸船」用卸船日；截止=实际提柜或当天。
 */
function isStorageCharge(std: {
  chargeTypeCode?: string | null;
  chargeName?: string | null;
}): boolean {
  if (isCombinedDemurrageDetention(std)) return false;
  if (isDetentionCharge(std)) return false;
  const code = (std.chargeTypeCode ?? '').toUpperCase();
  const name = (std.chargeName ?? '').toLowerCase();
  return code.includes('STORAGE') || name.includes('storage') || name.includes('堆存');
}

/** 判断是否为纯滞港费（Demurrage）标准：到港侧起算，提柜截止；排除合并、滞箱、堆存类型 */
function isDemurrageCharge(std: {
  chargeTypeCode?: string | null;
  chargeName?: string | null;
}): boolean {
  if (isCombinedDemurrageDetention(std)) return false;
  if (isDetentionCharge(std)) return false;
  if (isStorageCharge(std)) return false;
  const code = (std.chargeTypeCode ?? '').toUpperCase();
  const name = (std.chargeName ?? '').toLowerCase();
  return code.includes('DEMURRAGE') || name.includes('demurrage') || name.includes('滞港');
}
```

### 1.3 关键区别

**滞港费 vs 堆存费**：
- **滞港费 (Demurrage)**: 针对货物占用港区资源的费用，从到港开始计算
- **堆存费 (Storage)**: 针对集装箱在堆场存放的费用，可能与滞港费重叠

**滞箱费 vs D&D 合并**：
- **滞箱费 (Detention)**: 仅计算提柜后的箱体占用费
- **D&D 合并**: 从到港到还箱的整段期间，通常比分开计算更优惠

## 二、计算模式详解

### 2.1 两种计算模式

系统根据货柜状态自动选择计算模式：

| 模式 | 使用场景 | 起算日来源 | 截止日来源 | 金额特性 |
|------|----------|------------|------------|----------|
| **actual** | 已到目的港并产生实际作业 | ATA、实际卸柜日 | 实际提柜日、实际还箱日 | 固定值（封顶） |
| **forecast** | 未到目的港或计划阶段 | ETA、修正 ETA、计划提柜日 | 计划提柜日、计划还箱日 | 滚动更新（随 today 变化） |

### 2.2 模式自动判断逻辑

```typescript
// 判断逻辑（demurrage.service.ts）

function isArrivedAtDestinationPortForDemurrage(
  logisticsStatus: string,
  currentPortType?: 'origin' | 'destination' | 'transit'
): boolean {
  // actual 模式（已到达）
  if (['PICKED_UP', 'UNLOADED', 'RETURNED_EMPTY'].includes(logisticsStatus)) {
    return true;
  }
  
  if (logisticsStatus === 'AT_PORT' && currentPortType === 'destination') {
    return true;
  }
  
  // forecast 模式（未到达）
  if (['NOT_SHIPPED', 'SHIPPED', 'IN_TRANSIT'].includes(logisticsStatus)) {
    return false;
  }
  
  if (logisticsStatus === 'AT_PORT' && currentPortType !== 'destination') {
    return false;
  }
  
  // 默认保守处理
  return false;
}
```

### 2.3 时序封顶逻辑

**actual 模式**：
- 一旦切换到 actual 模式，截止日固定为实际提柜日/实际还箱日
- 金额不再随日期变化，形成"封顶"
- 即使后续发现更早的实际日期，也只向下调整

**forecast 模式**：
- 每日滚动更新
- 截止日 = `max(Today, 计划日期)`
- 金额会随 `today` 变化而增长

## 三、费用计算逻辑

### 3.1 滞港费 (Demurrage)

#### 起算日规则

| 计算模式 | calculation_basis | 起算日优先级 |
|----------|------------------|--------------|
| **actual** | 按到港 | ATA → ETA → 卸柜日 |
| **actual** | 按卸船 | 卸柜日 |
| **forecast** | 按到港 | ATA → 修正 ETA → ETA |
| **forecast** | 按卸船 | 卸柜日 → 修正 ETA → ETA |

#### 截止日规则

```typescript
// forecast 模式
if (calculationMode === 'forecast') {
  // 截止日 = max(Today, 计划提柜日)
  endDate = maxDate(today, plannedPickupDate ?? today);
  endDateSource = plannedPickupDate ? '计划提柜日' : '当前日期';
}

// actual 模式
if (calculationMode === 'actual') {
  // 截止日 = 实际提柜日（有则用），无则 Today
  endDate = pickupDateActual ?? today;
  endDateSource = pickupDateActual ? '实际提柜日' : '当前日期';
  // ✅ 关键：有实际提柜后金额封顶
}
```

#### 计算公式

```typescript
// 免费天数计算
const n = Math.max(0, freeDays - 1);
const lastFreeDate = freeDaysBasis === 'working_days'
  ? addWorkingDays(startDate, n)  // 工作日模式
  : addDays(startDate, n);         // 自然日模式

// 计费天数计算
const chargeDays = calculationMode === 'forecast'
  ? daysBetween(lastFreeDate, endDate)  // forecast: 包含当天
  : daysBetween(lastFreeDate, endDate); // actual: 包含当天

// 费用计算（支持阶梯费率）
const { totalAmount, tierBreakdown } = calculateSingleDemurrage(
  startDate,      // 起算日
  endDate,        // 截止日
  freeDays,       // 免费天数
  ratePerDay,     // 基础日费率
  tiers,          // 阶梯费率配置
  currency,       // 币种
  freeDaysBasis   // 免费天数基准
);
```

#### 示例

**场景 1: forecast 模式，有计划提柜日**
```
起算日：2024-04-01 (ETA)
免费天数：5 天（自然日）
最晚提柜日：2024-04-05
计划提柜日：2024-04-08
Today: 2024-04-06

截止日 = max(2024-04-06, 2024-04-08) = 2024-04-08
计费天数 = 2024-04-08 - 2024-04-05 = 3 天
费用 = 3 × 日费率
```

**场景 2: actual 模式，有实际提柜日**
```
起算日：2024-04-01 (ATA)
免费天数：5 天
最晚提柜日：2024-04-05
实际提柜日：2024-04-10
Today: 2024-04-15

截止日 = 2024-04-10 (实际提柜日，金额封顶)
计费天数 = 2024-04-10 - 2024-04-05 = 5 天
费用 = 5 × 日费率（固定值，不再增长）
```

### 3.2 滞箱费 (Detention)

#### 前置条件

**必须满足以下条件之一才能计算**：
- **forecast 模式**: 必须有 `planned_pickup_date`（计划提柜日）
- **actual 模式**: 必须有 `pickup_date_actual`（实际提柜日）

**缺少提柜日的处理**：
```typescript
if (calculationMode === 'forecast' && !plannedPickupDate) {
  skippedItems.push({
    reasonCode: 'missing_planned_pickup_date',
    reason: '缺少计划提柜日（planned_pickup_date），该费用项暂不计算'
  });
  continue; // 跳过该项
}

if (calculationMode === 'actual' && !pickupDateActual) {
  skippedItems.push({
    reasonCode: 'missing_pickup_date_actual',
    reason: '缺少实际提柜日（pickup_date），该费用项暂不计算'
  });
  continue; // 跳过该项
}
```

#### 起算日与截止日

| 计算模式 | 起算日 | 截止日 |
|----------|--------|--------|
| **forecast** | 计划提柜日 (`planned_pickup_date`) | `max(Today, 计划还箱日)` |
| **actual** | 实际提柜日 (`pickup_date_actual`) | `实际还箱日 (return_time) 或 Today` |

#### 计算公式

```typescript
// 起算日 = 实际/计划提柜日
startDate = calculationMode === 'actual' 
  ? pickupDateActual 
  : plannedPickupDate;

// 截止日 = 实际/计划还箱日 或 Today
endDate = calculationMode === 'actual'
  ? returnTime ?? today
  : maxDate(today, plannedReturnDate ?? today);

// 免费期计算（从起算日开始）
const n = Math.max(0, freeDays - 1);
const lastFreeDate = freeDaysBasis === 'working_days'
  ? addWorkingDays(startDate, n)
  : addDays(startDate, n);

// 计费天数
const chargeDays = daysBetween(lastFreeDate, endDate);

// 费用
const amount = chargeDays × ratePerDay;
```

#### 示例

**场景 1: forecast 模式，有计划日期**
```
计划提柜日：2024-04-10
计划还箱日：2024-04-15
免费天数：7 天
Today: 2024-04-12

起算日 = 2024-04-10
最晚还箱日 = 2024-04-16 (起算日 + 7 天)
截止日 = max(2024-04-12, 2024-04-15) = 2024-04-15
计费天数 = 2024-04-15 - 2024-04-16 = 0 天（在免费期内）
费用 = 0
```

**场景 2: actual 模式，有实际日期**
```
实际提柜日：2024-04-10
实际还箱日：2024-04-20
免费天数：7 天
Today: 2024-04-25

起算日 = 2024-04-10
最晚还箱日 = 2024-04-16
截止日 = 2024-04-20 (实际还箱日，金额封顶)
计费天数 = 2024-04-20 - 2024-04-16 = 4 天
费用 = 4 × 日费率
```

### 3.3 堆存费 (Storage)

#### 特殊规则

**堆存费的计算逻辑最为复杂**，因为它有两种子模式：

```typescript
// 判断是否使用实际区间
let storageUsesActualInterval = false;
if (isStorage) {
  const basisStorage = (std.calculationBasis ?? '').toLowerCase();
  const useDischargeStorage = basisStorage.includes('卸船');
  
  // 检查是否有实际起算日（ATA 或卸船日）
  const hasStorageActualStart = useDischargeStorage
    ? !!params.calculationDates.dischargeDate  // 按卸船：只需卸船日
    : !!(params.calculationDates.ataDestPort ?? params.calculationDates.dischargeDate);
  
  // 全局 actual，或 forecast 但已有 ATA/实际卸船 → 按实际区间计费
  storageUsesActualInterval =
    calculationMode === 'actual' || 
    (calculationMode === 'forecast' && hasStorageActualStart);
}
```

#### 两种子模式对比

| 子模式 | 触发条件 | 起算日 | 截止日 | 依赖条件 |
|--------|----------|--------|--------|----------|
| **actual 区间** | global actual，或 forecast 但已有 ATA/卸船日 | ATA/卸船日 | 实际/计划提柜日 | 需要 ATA 或卸船日 |
| **forecast 区间** | forecast 且无 ATA/卸船日 | 修正 ETA/ETA | `max(Today, 计划提柜日)` | **必须有计划提柜日** |

#### 前置条件检查

```typescript
// 堆存费：按实际区间时需 ATA 或卸船日；纯预测区间需计划提柜 + 修正 ETA/ETA
if (isStorage) {
  const basisStorage = (std.calculationBasis ?? '').toLowerCase();
  const useDischargeStorage = basisStorage.includes('卸船');
  const hasStorageActualStart = useDischargeStorage
    ? !!params.calculationDates.dischargeDate
    : !!(params.calculationDates.ataDestPort ?? params.calculationDates.dischargeDate);
  
  if (storageUsesActualInterval) {
    // 实际区间模式
    if (!hasStorageActualStart) {
      skippedItems.push({
        reasonCode: 'missing_arrival_storage_actual',
        reason: '堆存 (实际口径) 需目的港 ATA 或卸船日（按标准「按卸船」时仅需卸船日）'
      });
      continue;
    }
  } else {
    // 预测区间模式
    if (!plannedPickupDate) {
      skippedItems.push({
        reasonCode: 'missing_planned_pickup_date',
        reason: '堆存 (预测) 需计划提柜日（planned_pickup_date）'
      });
      continue;
    }
    
    // 检查是否有起算日（修正 ETA/ETA）
    if (!(params.calculationDates.revisedEtaDestPort ?? params.calculationDates.etaDestPort)) {
      skippedItems.push({
        reasonCode: 'missing_eta_storage_forecast',
        reason: '堆存 (预测) 需 ETA 或修正 ETA'
      });
      continue;
    }
  }
}
```

#### 计算公式

```typescript
// 实际区间模式（已有 ATA/卸船日）
if (storageUsesActualInterval) {
  // 起算日 = ATA 或卸船日
  startDate = useDischargeStorage
    ? params.calculationDates.dischargeDate
    : params.calculationDates.ataDestPort ?? params.calculationDates.dischargeDate;
  
  // 截止日 = 实际/计划提柜日 或 Today
  endDate = calculationMode === 'actual'
    ? pickupDateActual ?? today
    : maxDate(today, plannedPickupDate ?? today);
}

// 预测区间模式（无 ATA/卸船日）
else {
  // 起算日 = 修正 ETA 或 ETA
  startDate = params.calculationDates.revisedEtaDestPort ?? params.calculationDates.etaDestPort;
  
  // 截止日 = max(Today, 计划提柜日)
  endDate = maxDate(today, plannedPickupDate ?? today);
}

// 免费期和费用计算与其他费用类型一致
const n = Math.max(0, freeDays - 1);
const lastFreeDate = freeDaysBasis === 'working_days'
  ? addWorkingDays(startDate, n)
  : addDays(startDate, n);

const chargeDays = daysBetween(lastFreeDate, endDate);
const amount = chargeDays × ratePerDay;
```

#### 示例

**场景 1: forecast 模式，无 ATA，有计划提柜日**
```
修正 ETA: 2024-04-01
计划提柜日：2024-04-08
免费天数：5 天
Today: 2024-04-06

起算日 = 2024-04-01 (修正 ETA)
最晚提柜日 = 2024-04-05
截止日 = max(2024-04-06, 2024-04-08) = 2024-04-08
计费天数 = 2024-04-08 - 2024-04-05 = 3 天
费用 = 3 × 日费率
```

**场景 2: forecast 模式，已有 ATA，有计划提柜日**
```
ATA: 2024-04-01 (已到港)
计划提柜日：2024-04-08
免费天数：5 天
Today: 2024-04-06

// ✅ 切换到实际区间模式
起算日 = 2024-04-01 (ATA)
最晚提柜日 = 2024-04-05
截止日 = max(2024-04-06, 2024-04-08) = 2024-04-08
计费天数 = 3 天
费用 = 3 × 日费率
```

**场景 3: 没有计划提柜日**
```
修正 ETA: 2024-04-01
计划提柜日：null ❌
Today: 2024-04-06

结果：跳过堆存费计算
原因：'堆存 (预测) 需计划提柜日（planned_pickup_date）'
```

### 3.4 D&D 合并费用

#### 起算日规则

| 计算模式 | calculation_basis | 起算日优先级 |
|----------|------------------|--------------|
| **actual** | 按到港 | ATA → 卸柜日 |
| **actual** | 按卸船 | 卸柜日 |
| **forecast** | 任意 | 修正 ETA → ETA |

#### 截止日规则

与**滞箱费**一致：
- **forecast**: `max(Today, 计划还箱日)`
- **actual**: `实际还箱日 或 Today`

#### 前置条件

```typescript
if (calculationMode === 'forecast') {
  // forecast 模式需 ETA/修正 ETA
  if (!(params.calculationDates.revisedEtaDestPort ?? params.calculationDates.etaDestPort)) {
    skippedItems.push({
      reasonCode: 'missing_eta_combined_forecast',
      reason: 'D&D 合并 (预测) 需 ETA 或修正 ETA'
    });
    continue;
  }
} else {
  // actual 模式需 ATA/卸船日
  const basisCombined = (std.calculationBasis ?? '').toLowerCase();
  const useDischargeCombined = basisCombined.includes('卸船');
  const hasCombinedActualStart = useDischargeCombined
    ? !!params.calculationDates.dischargeDate
    : !!(params.calculationDates.ataDestPort ?? params.calculationDates.dischargeDate);
  
  if (!hasCombinedActualStart) {
    skippedItems.push({
      reasonCode: 'missing_arrival_for_combined_actual',
      reason: 'D&D 合并 (实际口径) 需目的港 ATA 或卸船日'
    });
    continue;
  }
}
```

## 四、标准匹配规则

### 4.1 四字段匹配

滞港费标准通过以下四个字段与货柜匹配：

```
1. 进口国/海外公司 (foreign_company_code)
   来源：备货单客户.overseas_company_code
   
2. 目的港 (destination_port_code)
   来源：process_port_operations.port_code 或 process_sea_freight.port_of_discharge
   
3. 船公司 (shipping_company_code)
   来源：process_sea_freight.shipping_company_id
   
4. 货代 (origin_forwarder_code)
   来源：process_sea_freight.freight_forwarder_id
```

### 4.2 匹配优先级

```sql
-- 匹配顺序（从高到低）

-- 1. 完全匹配（四字段 + 有效期）
SELECT * FROM ext_demurrage_standard
WHERE foreign_company_code = ?
  AND destination_port_code = ?
  AND shipping_company_code = ?
  AND origin_forwarder_code = ?
  AND is_chargeable = 'N'
  AND effective_date <= CURRENT_DATE
  AND (expiry_date IS NULL OR expiry_date >= CURRENT_DATE)
ORDER BY sequence_number ASC;

-- 2. 宽松匹配（逐步减少字段）
-- 2.1 去掉货代
WHERE foreign_company_code = ?
  AND destination_port_code = ?
  AND shipping_company_code = ?
  ...

-- 2.2 去掉船公司
WHERE foreign_company_code = ?
  AND destination_port_code = ?
  ...

-- 2.3 默认标准
WHERE destination_port_code = ?
  AND is_chargeable = 'N'
  ...
```

### 4.3 有效期验证

```typescript
// 有效期规则
const isValidEffectiveDate = (standard: ExtDemurrageStandard): boolean => {
  const today = toDateOnly(new Date());
  const effectiveDate = standard.effectiveDate 
    ? toDateOnly(standard.effectiveDate) 
    : null;
  const expiryDate = standard.expiryDate 
    ? toDateOnly(standard.expiryDate) 
    : null;
  
  // 生效日期 <= 当前日期
  if (!effectiveDate || effectiveDate > today) {
    return false;
  }
  
  // 结束日期 IS NULL OR 结束日期 >= 当前日期
  if (expiryDate && expiryDate < today) {
    return false;
  }
  
  return true;
};
```

### 4.4 is_chargeable 标志

```
Y = 不收费，跳过（用于标记不适用的标准）
N = 收费，参与计算（默认值）
```

### 6.5 完整计算示例

**场景：加拿大温哥华港，40HC 柜，智能排柜方案**

```
基本信息:
- 货柜号：MSKU1234567
- 目的国：Canada
- 目的港：CA-VAN
- 船公司：CMA-CGM
- 货代：Flexport

日期信息:
- 修正 ETA: 2024-04-01
- ATA: null (未到港)
- 计划提柜日：2024-04-08
- 计划送仓日：2024-04-10
- 计划卸柜日：2024-04-10
- 计划还箱日：2024-04-15
- Today: 2024-04-06

匹配标准:
1. Demurrage: 5 天免费，USD 100/天
2. Detention: 7 天免费，USD 80/天
3. Storage: 3 天免费，USD 120/天（按到港）
4. D&D Combined: 10 天免费，USD 150/天（按到港）

计算过程:

1. Demurrage (forecast):
   起算日 = 2024-04-01 (修正 ETA)
   最晚提柜日 = 2024-04-05 (起算日 + 5 天)
   截止日 = max(2024-04-06, 2024-04-08) = 2024-04-08
   计费天数 = 2024-04-08 - 2024-04-05 = 3 天
   费用 = 3 × USD 100 = USD 300

2. Detention (forecast):
   起算日 = 2024-04-08 (计划提柜日)
   最晚还箱日 = 2024-04-14 (起算日 + 7 天)
   截止日 = max(2024-04-06, 2024-04-15) = 2024-04-15
   计费天数 = 2024-04-15 - 2024-04-14 = 1 天
   费用 = 1 × USD 80 = USD 80

3. Storage (forecast 区间，因为无 ATA):
   起算日 = 2024-04-01 (修正 ETA)
   最晚提柜日 = 2024-04-03 (起算日 + 3 天)
   截止日 = max(2024-04-06, 2024-04-08) = 2024-04-08
   计费天数 = 2024-04-08 - 2024-04-03 = 5 天
   费用 = 5 × USD 120 = USD 600

4. D&D Combined (forecast):
   起算日 = 2024-04-01 (修正 ETA)
   最晚还箱日 = 2024-04-10 (起算日 + 10 天)
   截止日 = max(2024-04-06, 2024-04-15) = 2024-04-15
   计费天数 = 2024-04-15 - 2024-04-10 = 5 天
   费用 = 5 × USD 150 = USD 750

总费用汇总:
- Demurrage: USD 300
- Detention: USD 80
- Storage: USD 600
- D&D Combined: USD 750
- Transportation: USD 500 (车队 + 仓库)
- Yard Storage: USD 200 (Drop off 模式堆场费)
- Total: USD 2,430
```

## 九、常见问题排查

### 9.1 费用为 0

| 可能原因 | 排查方法 | 解决方案 |
|----------|----------|----------|
| 费用标准未配置 | 查询 `ext_demurrage_standard` 表 | 补充标准配置 |

### 9.2 匹配失败

**排查步骤**:


### 9.3 日期问题

**问题 1: 日期边界偏移**
- **原因**: JS `new Date('2024-04-01')` 转为 timestamptz 产生±1 天偏移
- **解决**: SQL 层用 `::date` 强制转换，代码层用 `toDateOnly()`

**问题 2: 工作日计算错误**
- **原因**: 未正确处理跨年、跨月的工作日
- **解决**: 使用统一的 `addWorkingDays()` 函数

**问题 3: forecast 模式无计划提柜日**
- **原因**: 智能排柜未生成或数据未同步
- **解决**: 执行智能排柜或手工维护计划提柜日
