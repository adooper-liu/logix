### 一、费用类型识别

| 类型 | 起算日 | 截止日 | 必需条件 |
|------|--------|--------|----------|
| **Demurrage (滞港费)** | ATA/ETA/卸船日 | 提柜日/Today | 到港信息 |
| **Detention (滞箱费)** | 提柜日 | 还箱日/Today | **提柜日** |
| **Storage (堆存费)** | ATA/卸船日/ETA | 提柜日/Today | **提柜日** (预测模式) |
| **D&D Combined (合并)** | ATA/ETA/卸船日 | 还箱日/Today | 到港信息或 ETA |

### 二、计算模式

```
┌─────────────────────────────────────┐
│   forecast 模式 (未到达)            │
│   - 起算日：修正 ETA > ETA          │
│   - 截止日：max(Today, 计划日期)    │
│   - 金额滚动更新                    │
└─────────────────────────────────────┘
              ↓
       判断是否到达
              ↓
┌─────────────────────────────────────┐
│   actual 模式 (已到达)              │
│   - 起算日：ATA/卸船日              │
│   - 截止日：实际日期 (金额封顶)     │
│   - 金额固定不变                    │
└─────────────────────────────────────┘
```

### 三、标准匹配规则

**四字段匹配**:
1. 进口国/海外公司 (`foreign_company_code`)
2. 目的港 (`destination_port_code`)
3. 船公司 (`shipping_company_code`)
4. 货代 (`origin_forwarder_code`)

**匹配优先级**:
```
完全匹配 (四字段) 
  ↓
宽松匹配 (去掉货代)
  ↓
更宽松匹配 (去掉船公司)
  ↓
默认标准
```

### 四、关键公式

```typescript
// 免费期计算
n = Math.max(0, freeDays - 1);
lastFreeDate = startDate + n 天;

// 计费天数
chargeDays = endDate - lastFreeDate;

// 费用计算
amount = chargeDays × ratePerDay;
```

### 一、Storage Charge 深度解析

#### 问题：没有 planned_pickup_date 会不会计算 Storage Charge？

**答案：分三种情况**

| 情况 | 有 ATA/卸船日 | 有计划提柜日 | 结果 |
|------|--------------|-------------|------|
| **情况 1** | ❌ 无 | ❌ 无 | ❌ **不计算**<br>reason: missing_planned_pickup_date |
| **情况 2** | ❌ 无 | ✅ 有 | ✅ **计算**<br>forecast 区间模式 |
| **情况 3** | ✅ 有 | ❌ 无 | ✅ **计算**<br>actual 区间模式 (用 Today) |
| **情况 4** | ✅ 有 | ✅ 有 | ✅ **计算**<br>actual 区间模式 |

#### 示例代码

```typescript
// Storage Charge 判断逻辑
let storageUsesActualInterval = false;

if (isStorage) {
  const hasStorageActualStart = useDischargeStorage
    ? !!params.calculationDates.dischargeDate
    : !!(params.calculationDates.ataDestPort ?? params.calculationDates.dischargeDate);
  
  // 全局 actual，或 forecast 但已有 ATA/实际卸船 → 按实际区间计费
  storageUsesActualInterval =
    calculationMode === 'actual' || 
    (calculationMode === 'forecast' && hasStorageActualStart);
}

// 情况 1: forecast 且无 ATA → 需要 planned_pickup_date
if (!storageUsesActualInterval && !plannedPickupDate) {
  skippedItems.push({
    reason: '堆存 (预测) 需计划提柜日（planned_pickup_date）'
  });
  continue; // 跳过
}

// 情况 3: forecast 但有 ATA → 不需要 planned_pickup_date
if (storageUsesActualInterval && hasStorageActualStart) {
  // 使用 actual 区间，用 Today 替代截止日
  endDate = today;
  // 正常计算...
}
```

### 二、四种费用类型完整对比矩阵

| 费用类型 | forecast 无计划 | forecast 有计划 | actual 无实际 | actual 有实际 |
|----------|---------------|---------------|-------------|-------------|
| **Demurrage** | ✅ (ETA+Today) | ✅ (计划提柜) | ✅ (ATA+Today) | ✅ (实际封顶) |
| **Detention** | ❌ **跳过** | ✅ (必需) | ❌ **跳过** | ✅ (实际封顶) |
| **Storage** | ⚠️ (有 ATA 时可) | ✅ (必需) | ✅ (有 ATA 可) | ✅ (实际封顶) |
| **D&D** | ⚠️ (需 ETA) | ✅ (计划还箱) | ⚠️ (需 ATA) | ✅ (实际封顶) |

### 三、智能排柜成本计算实战

#### 成本计算流程

```
智能排柜引擎
    ↓
生成 N 个候选方案
(不同的日期组合)
    ↓
for each 方案:
    ├── 调用 DemurrageService.calculateTotalCost()
    │     ├── Demurrage: USD 300
    │     ├── Detention: USD 80
    │     ├── Storage: USD 600
    │     ├── D&D: USD 750
    │     └── Transportation: USD 500
    │
    └── 总成本 = USD 2,230
          ↓
选择最低成本方案
    ↓
输出最优日期 + 成本明细
```

#### 示例：快速提柜 vs 延迟提柜

```
方案 A (快速提柜):
- 计划提柜日：2024-04-05
- 总成本：USD 740  ✅ 推荐

方案 B (延迟提柜):
- 计划提柜日：2024-04-08
- 总成本：USD 2,530

成本差异：USD 1,790
节省比例：70.8%
```

### Storage Charge 判断逻辑

```typescript
// demurrage.service.ts (第 1547-1608 行)

/** 堆存费：全局 actual，或 forecast 但已有 ATA/实际卸船 → 按实际区间计费 */
let storageUsesActualInterval = false;
if (isStorage) {
  const basisStorage = (std.calculationBasis ?? '').toLowerCase();
  const useDischargeStorage = basisStorage.includes('卸船');
  const hasStorageActualStart = useDischargeStorage
    ? !!params.calculationDates.dischargeDate
    : !!(params.calculationDates.ataDestPort ?? params.calculationDates.dischargeDate);
  
  storageUsesActualInterval =
    calculationMode === 'actual' || 
    (calculationMode === 'forecast' && hasStorageActualStart);
}

// 滞箱费：actual 需实际提柜；forecast 需计划提柜
if (isDetention) {
  if (calculationMode === 'actual' && !pickupDateActual) {
    skippedItems.push({ /* ... */ });
    continue;
  }
  if (calculationMode === 'forecast' && !plannedPickupDate) {
    skippedItems.push({ /* ... */ });
    continue;
  }
}

// 堆存费：按实际区间时需 ATA 或卸船日；纯预测区间需计划提柜 + 修正 ETA/ETA
if (isStorage) {
  const basisStorage = (std.calculationBasis ?? '').toLowerCase();
  const useDischargeStorage = basisStorage.includes('卸船');
  const hasStorageActualStart = useDischargeStorage
    ? !!params.calculationDates.dischargeDate
    : !!(params.calculationDates.ataDestPort ?? params.calculationDates.dischargeDate);
  
  if (storageUsesActualInterval) {
    if (!hasStorageActualStart) {
      skippedItems.push({
        reasonCode: 'missing_arrival_storage_actual',
        reason: '堆存 (实际口径) 需目的港 ATA 或卸船日（按标准「按卸船」时仅需卸船日）'
      });
      continue;
    }
  } else {
    if (!plannedPickupDate) {
      skippedItems.push({
        reasonCode: 'missing_planned_pickup_date',
        reason: '堆存 (预测) 需计划提柜日（planned_pickup_date）'
      });
      continue;
    }
    
    if (!(params.calculationDates.revisedEtaDestPort ?? params.calculationDates.etaDestPort)) {
      skippedItems.push({
        reasonCode: 'missing_eta_storage_forecast',
        reason: '堆存 (预测) 需 ETA 或修正 ETA'
      });
      continue;
    }
  }
}
```

### 费用类型识别函数

```typescript
// demurrage.service.ts (第 435-462 行)

/** 判断是否为堆存费（Storage）标准 */
function isStorageCharge(std: {
  chargeTypeCode?: string | null;
  chargeName?: string | null;
}): boolean {
  if (isCombinedDemurrageDetention(std)) return false;
  if (isDetentionCharge(std)) return false;
  const code = (std.chargeTypeCode ?? '').toUpperCase();
  const name = (std.chargeName ?? '').toLowerCase();
  return code.includes('STORAGE') || name.includes('storage') || name.includes('堆存');
}

/** 判断是否为纯滞港费（Demurrage）标准 */
function isDemurrageCharge(std: {
  chargeTypeCode?: string | null;
  chargeName?: string | null;
}): boolean {
  if (isCombinedDemurrageDetention(std)) return false;
  if (isDetentionCharge(std)) return false;
  if (isStorageCharge(std)) return false;
  const code = (std.chargeTypeCode ?? '').toUpperCase();
  const name = (std.chargeName ?? '').toLowerCase();
  return code.includes('DEMURRAGE') || name.includes('demurrage') || name.includes('滞港');
}

/** 判断是否为滞箱费（Detention）标准 */
function isDetentionCharge(std: {
  chargeTypeCode?: string | null;
  chargeName?: string | null;
}): boolean {
  if (isCombinedDemurrageDetention(std)) return false;
  const code = (std.chargeTypeCode ?? '').toUpperCase();
  const name = (std.chargeName ?? '').toLowerCase();
  return code.includes('DETENTION') || name.includes('detention') || name.includes('滞箱');
}

/** 判断是否为 D&D 合并费用标准 */
function isCombinedDemurrageDetention(std: {
  chargeTypeCode?: string | null;
  chargeName?: string | null;
}): boolean {
  const code = (std.chargeTypeCode ?? '').toUpperCase();
  const name = (std.chargeName ?? '').toLowerCase();
  return code.includes('D&D') || 
         code.includes('DEMURRAGE_AND_DETENTION') ||
         (name.includes('demurrage') && name.includes('detention'));
}
```
