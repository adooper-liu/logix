# 滞港费

LogiX 滞港费计算逻辑与实施经验。

## 一、核心概念

### 1.1 四种费用类型

| 类型 | 代码标识 | 起算日 | 截止日 | 说明 |
|------|----------|--------|--------|------|
| **Demurrage** | 滞港费 | ATA/ETA/卸柜日 | 提柜日 | 港区占用费 |
| **Detention** | 滞箱费 | 提柜日 | 还箱日 | 箱体占用费 |
| **Storage** | 堆存费 | ATA/卸柜日 | 提柜日 | 港区堆存费 |
| **D&D Combined** | 合并费 | ATA/ETA/卸柜日 | 还箱日 | 滞港+滞箱合并 |

### 1.2 计算模式

| 模式 | 使用场景 | 起算日来源 | 截止日来源 |
|------|----------|------------|------------|
| **actual** | 已到目的港 | ATA、实际卸柜日 | 实际提柜日、实际还箱日 |
| **forecast** | 未到目的港 | ETA、修正ETA、计划提柜日 | 计划提柜日、计划还箱日 |

**模式自动判断逻辑**：
- 已提柜/已卸柜/已还箱 → actual
- AT_PORT 且目的港 → actual
- AT_PORT 中转港/在途/未出运 → forecast

## 二、标准匹配

### 2.1 匹配四字段

```
1. 进口国 (foreign_company_code)
2. 目的港 (destination_port_code)
3. 船公司 (shipping_company_code)
4. 货代 (origin_forwarder_code)
```

**匹配规则**：支持代码或名称匹配（模糊匹配）

### 2.2 有效期验证

```
生效日期 <= 当前日期
AND (结束日期 IS NULL OR 结束日期 >= 当前日期)
```

### 2.3 is_chargeable 标志

- `N` = 收费，参与计算
- `Y` = 不收费，跳过

### 2.4 优先级

1. 四字段 + 有效期都匹配 → 使用
2. 有效期无匹配，但四字段匹配 → 取最新生效日期（effective_date 降序）

## 三、日期计算

### 3.1 起算日优先级

```
ATA > 修正ETA > ETA > 卸柜日
```

**按到港**：ATA → ETA → 卸柜日
**按卸船**：只用卸柜日（calculation_basis 包含"卸船"）

### 3.2 免费天数计算

```typescript
// 核心公式
n = Math.max(0, freeDays - 1);
lastFreeDate = startDate + n 天;

// 工作日模式：跳过周六、周日
// 自然日模式：连续计算
```

**示例**：
- 免费5天，起算日3月20日(周五)
- 自然日：lastFreeDate = 3月24日(周二)
- 工作日：lastFreeDate = 3月26日(周四)，跳过周末

### 3.3 计费计算

```
chargeDays = endDate - lastFreeDate - 1
amount = chargeDays × 日费率
```

**阶梯费率**：支持多级阶梯，计算方式见 3.4 节

### 3.4 完整日期结构

```typescript
calculationDates: {
  ataDestPort: Date | null;        // 实际到港日
  etaDestPort: Date | null;        // 预计到港日
  revisedEtaDestPort: Date | null; // 修正ETA
  dischargeDate: Date | null;      // 卸柜日

  lastPickupDate: Date | null;     // 最晚提柜日(last_free_date)
  plannedPickupDate: Date | null;  // 计划提柜日
  lastReturnDate: Date | null;    // 最晚还箱日
  plannedReturnDate: Date | null; // 计划还箱日

  pickupDateActual: Date | null;   // 实际提柜日
  returnTime: Date | null;         // 实际还箱时间

  shipmentDate: Date | null;       // 出运日
  today: Date;                     // 当前日期
}
```

## 四、实施坑点

### 4.1 常见错误原因

| reason | 说明 | 解决方案 |
|--------|------|----------|
| `no_arrival_at_dest` | 未到达目的港 | 等待到港或维护计划提柜日 |
| `missing_arrival_dates` | 缺少到港/卸船日起算日 | 维护ATA/ETA/卸柜日 |
| `no_matching_standards` | 无匹配标准 | 检查四字段是否正确 |
| `missing_dates` | 缺少关键日期 | forecast模式必须维护计划提柜日 |
| `missing_pickup_date_actual` | actual模式缺实际提柜日 | 等待提柜或更新实际提柜日 |
| `missing_planned_pickup_date` | forecast模式缺计划提柜日 | 维护计划提柜日 |
| `missing_eta_combined_forecast` | D&D合并费用缺ETA | 维护ETA |
| `missing_arrival_for_combined_actual` | D&D合并费用缺ATA | 等待到港 |

### 4.2 匹配失败排查

**步骤1：检查标准表数据**
```sql
-- 查看有效标准数量
SELECT COUNT(*) FROM ext_demurrage_standard
WHERE is_chargeable = 'N'
AND effective_date <= CURRENT_DATE
AND (expiry_date IS NULL OR expiry_date >= CURRENT_DATE);
```

**步骤2：检查四字段匹配**
```sql
-- 查看货柜的四个匹配字段值
SELECT c.container_number,
       c.destination_country,
       po.port_code as dest_port,
       sf.shipping_company_id,
       sf.freight_forwarder_id
FROM biz_containers c
LEFT JOIN process_port_operations po ON c.container_number = po.container_number
LEFT JOIN process_sea_freight sf ON c.container_number = sf.container_number
WHERE c.container_number = 'xxx';
```

**步骤3：检查标准表字段**
```sql
-- 对比货柜字段与标准表
SELECT destination_port_code, shipping_company_code,
       origin_forwarder_code, foreign_company_code
FROM ext_demurrage_standard
WHERE is_chargeable = 'N';
```

### 4.3 日期问题

**问题1：日期边界**
- JS `new Date('2026-01-01')` 转为 timestamptz，边界易产生±1天偏移
- 解决：SQL 层用 `::date` 强制转换

**问题2：工作日计算**
- 需正确识别周六、周日
- 需处理跨年、跨月的工作日计算

**问题3：预测模式前置条件**
- 无ATA且无计划提柜日时，forecast模式返回 `reason='missing_dates'`
- 前端需正确处理并提示用户

### 4.4 编码匹配问题

**问题：标准表编码与货柜数据编码不一致**
- 船公司常用 SCAC 代码（如 CMA）
- 货代常用中文名
- 解决：使用 `dict_universal_mapping` 通用映射表

## 八、数据表

### 8.1 ext_demurrage_standard（滞港费标准表）

```sql
CREATE TABLE ext_demurrage_standard (
  id SERIAL PRIMARY KEY,
  foreign_company_code VARCHAR(20),      -- 进口国/海外公司
  foreign_company_name VARCHAR(100),
  destination_port_code VARCHAR(20),     -- 目的港
  destination_port_name VARCHAR(100),
  shipping_company_code VARCHAR(20),      -- 船公司
  shipping_company_name VARCHAR(100),
  origin_forwarder_code VARCHAR(20),      -- 货代
  origin_forwarder_name VARCHAR(100),
  charge_type_code VARCHAR(20),           -- 费用类型代码
  charge_name VARCHAR(100),               -- 费用名称
  is_chargeable CHAR(1) DEFAULT 'N',     -- Y=不收费, N=收费
  sequence_number INTEGER,                -- 优先级
  port_condition VARCHAR(20),            -- 港口条件(如"差")
  free_days_basis VARCHAR(20),            -- 免费天数基准(自然日/工作日)
  free_days INTEGER,                     -- 免费天数
  calculation_basis VARCHAR(20),          -- 计算方式(按到港/按卸船)
  effective_date DATE,                   -- 生效日期
  expiry_date DATE,                      -- 结束日期
  daily_rate DECIMAL(10,2),              -- 日费率
  rate_tiers JSONB                       -- 阶梯费率JSON
);
```

### 8.2 ext_demurrage_record（滞港费记录表）

```sql
CREATE TABLE ext_demurrage_record (
  id SERIAL PRIMARY KEY,
  container_number VARCHAR(20) NOT NULL,
  calculation_date DATE NOT NULL,
  charge_type_code VARCHAR(20),
  charge_name VARCHAR(100),
  start_date DATE,
  end_date DATE,
  free_days INTEGER,
  charge_days INTEGER,
  daily_rate DECIMAL(10,2),
  amount DECIMAL(12,2),
  currency VARCHAR(3) DEFAULT 'USD',
  calculation_mode VARCHAR(10),           -- actual/forecast
  created_at TIMESTAMP DEFAULT NOW()
);
```
