**修复**:
```typescript
// 优先级：plannedData.warehouseId > warehouseCode > 其他备用字段
const warehouseCode =
  row.plannedData?.warehouseId || // ✅ 首选：排产数据中的 warehouseId
  row.plannedData?.warehouseCode ||
  row.warehouseCode ||
  row.warehouseId ||
  row.destinationWarehouse ||
  row.warehouseName?.split(" ")[0]
```

### 1.5 成本优化候选日期参数覆盖无效

**问题**: 成本优化功能中，不同候选日期的费用计算结果都一样（如 chargeDays 都是 38 天）。

**日志证据**:
```log
{"level":"info","message":"[Demurrage] Temporarily overriding planned dates for ECMU5397691:",
 "newPickup":"2026-03-31T10:17:05.895Z"}  // ✅ 覆盖成功

{"amount":2200,"chargeName":"Storage Charge",
 "rangeEnd":"2026-03-27T00:00:00.000Z",  // ❌ 但 rangeEnd 还是今天！
 "chargeDays":38  // ❌ 所有日期的 chargeDays 都是 38！
 ...}
```

**根因分析**:
```
calculateTotalCostWithPlannedDates
    │
    ├── 覆盖 params.calculationDates.plannedPickupDate = '2026-03-31'
    │
    └── 调用 calculateForContainer()
            │
            └── 内部重新调用 getContainerMatchParams() ← 从数据库重新获取！
            │
            └── paramsOverride 被覆盖无效 ❌
```

**修复方案**:
```typescript
// demurrage.service.ts 修改
async calculateForContainer(
  containerNumber: string,
  options?: {
    freeDateWriteMode?: 'none' | 'actual' | 'forecast';
    paramsOverride?: ContainerMatchParams;  // ✅ 新增：允许外部传入参数覆盖
  }
)

// calculateTotalCostWithPlannedDates 中传递覆盖后的 params
const params = {
  ...matchParams,
  calculationDates: {
    ...matchParams.calculationDates,
    plannedPickupDate: plannedPickupDate!,  // 覆盖为候选日期
  }
};

const result = await this.calculateForContainer(containerNumber, {
  paramsOverride: params  // ✅ 传入覆盖后的参数
});
```

**修复效果**:
- 候选日期 2026-03-27：rangeEnd = 2026-03-27，chargeDays = 38
- 候选日期 2026-03-30：rangeEnd = 2026-03-30，chargeDays = 35 ✅ 不同了！
- 候选日期 2026-03-31：rangeEnd = 2026-03-31，chargeDays = 34 ✅ 不同了！
