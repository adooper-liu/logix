# 甘特图

LogiX 甘特图可视化设计与实现。

## 一、核心概念

### 1.1 时间维度

甘特图支持四种时间维度筛选：

| 维度 | 说明 | filterCondition |
|------|------|-----------------|
| 按到港 | 按实际/预计到港日 | `arrival*` |
| 按计划提柜 | 按计划提柜日 | `pickup*` |
| 按最晚提柜 | 按最晚提柜日 | `lastPickup*` |
| 按最晚还箱 | 按最晚还箱日 | `return*` |

### 1.2 日期取值优先级

```
1. container.ataDestPort（实际到港日）
2. container.etaCorrection（修正ETA）
3. container.etaDestPort（预计到港日）
4. container.portOperations[].ataDestPort/etaDestPort
5. container.seaFreight.eta（海运表ETA - 最后fallback）
```

### 1.3 五阶段

| 阶段 | 标签 | 主节点 | 说明 |
|------|------|--------|------|
| 1 | 清关 | customs | 海关申报 |
| 2 | 提柜 | pickup | 提柜到仓库 |
| 3 | 卸柜 | unload | 卸货入库 |
| 4 | 还箱 | return | 归还空箱 |
| 5 | 完成 | - | 流程结束 |

### 1.4 节点状态

```typescript
type GanttNodeKey = 'customs' | 'pickup' | 'unload' | 'return'

interface GanttDerivedNode {
  key: GanttNodeKey
  taskRole: 'main' | 'dashed' | 'none'  // 主任务/虚线/无
  completed: boolean
  plannedDate?: string | null
  actualDate?: string | null
}
```

## 三、分组结构

### 3.1 三级分组

```
目的港 (Port)
  └── 节点 (Node): customs/pickup/unload/return
        └── 供应商 (Supplier): 清关/拖车/仓库
```

### 3.2 分组逻辑

```typescript
// 按目的港分组
groupedByPort: Record<string, PortGroup>

interface PortGroup {
  portName: string
  portCode: string
  nodes: Record<GanttNodeKey, NodeGroup>
}

interface NodeGroup {
  nodeKey: GanttNodeKey
  suppliers: Record<string, SupplierGroup>
}

interface SupplierGroup {
  containers: Container[]
}
```

### 3.3 折叠/展开

- 支持按目的港折叠/展开
- 支持全局展开/折叠
- 状态持久化到 localStorage

## 四、视图模式

### 4.1 独立表格模式

货柜行直接显示在主表格中，支持：

- 目的港汇总行
- 节点汇总行
- 供应商行
- 货柜行（四级嵌套）

### 4.2 弹窗详情模式

点击货柜弹出详情面板：

- 货柜基本信息
- 甘特节点详情
- 时间线展示

### 4.3 日期范围

| 类型 | 说明 |
|------|------|
| 动态 | 根据数据自动计算范围 |
| 7天 | 固定显示最近7天 |
| 15天 | 固定显示最近15天 |
| 30天 | 固定显示最近30天 |

## 六、交互功能

### 6.1 搜索与筛选

- 货柜号搜索

## 七、实施坑点

### 7.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 甘特图无数据 | filterCondition 不匹配 | 检查 useGanttLogic 与后端路由 |
| 日期显示错位 | dateRange 计算错误 | 检查 dayjs 日期计算 |
| 分组不正确 | ganttDerived 为空 | 调用 rebuildGanttDerivedSnapshot |
| 颜色不显示 | statusColors 未注入 | 检查组件 props |

### 7.2 排查步骤

1. 检查 ganttFilterStore 状态
2. 检查 containerService 返回数据
3. 检查 ganttDerived 字段
4. 检查日期范围计算逻辑

## 九、数据流

```
后端 gantt_derived
       ↓
useGanttLogic.loadData()
       ↓
containers.value = response.items
       ↓
groupedByPort = groupByPort(containers)
       ↓
SimpleGanttChartRefactored.render()
       ↓
三级分组渲染（目的港→节点→供应商→货柜）
```

**关键路径**：

1. **数据加载**：`loadData()` → `containerService.getContainers()`
2. **分组计算**：`groupByPort()` → `groupByNode()` → `groupBySupplier()`
3. **日期范围**：`calculateDisplayRange()` → `generateDateRange()`
4. **交互响应**：拖拽 → 弹窗确认 → `containerService.updateSchedule()`
