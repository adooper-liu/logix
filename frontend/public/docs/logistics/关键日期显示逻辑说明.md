# 关键日期显示逻辑说明

## 核心概念

关键日期时间线显示三种不同的时间状态：**历时**、**超期**、**倒计时**。

---

## 1. 倒计时 - 未来日期

**定义**：
距离目标日期还有多少天

**适用范围**：
所有未来的日期

**计算公式**：
```
倒计时 = 目标日期 - 当前日期
```

**标记示例**：
```
今天到期
倒计时1天
倒计时5天
```

**颜色规则**：
- **橙色**：今天到期或剩余1-3天
- **绿色**：剩余>3天

**视觉样式**：
```
橙色：#E6A23C (warning-color)
绿色：#67C23A (success-color)
```

---

## 2. 历时 - 过去日期

**定义**：
从目标日期到现在已经过去多少天

**适用范围**：
- 所有过去的普通日期
- 关键节点未超期的情况

**计算公式**：
```
历时 = 当前日期 - 目标日期
```

**标记示例**：
```
今天到期
历时1天
历时5天
```

**颜色规则**：
- **蓝色**：所有显示"历时"的日期

**视觉样式**：
```
蓝色：#409EFF (info-color)
字体粗细：500
```

---

## 3. 超期 - 关键节点预警

**定义**：
关键节点已超过标准耗时的时长

**适用范围**：
**仅关键节点**（最晚提柜日、最晚还箱日）且已超过标准时间

**计算公式**：
```
超期 = 当前日期 - 目标日期 - 标准耗时
```

**标记示例**：
```
超期1天
超期5天
```

**颜色规则**：
- **红色**：已超期的关键节点

**视觉样式**：
```
红色：#F56C6C (danger-color)
字体粗细：600
动画：脉冲动画 (pulse 2s infinite)
```

## 代码实现

### 状态文字生成

```typescript
const getDateStatusText = (date: Date, label?: string): string => {
  const now = new Date()

  // 1. 未来日期：显示倒计时
  if (date > now) {
    const diffDays = Math.floor((date.getTime() - now.getTime()) / (1000 * 60 * 60 * 24))
    if (diffDays === 0) return '今天到期'
    if (diffDays === 1) return '倒计时1天'
    return `倒计时${diffDays}天`
  }

  // 2. 过去日期
  const diffDays = Math.floor((now.getTime() - date.getTime()) / (1000 * 60 * 60 * 24))
  if (diffDays === 0) return '今天到期'

  // 2.1 关键节点（最晚提柜/最晚还箱）：判断是否超期
  if (label && (label === '最晚提柜' || label === '最晚还箱')) {
    const standardHours = STANDARD_DURATIONS[label] ?? 0
    if (standardHours > 0) {
      const standardDays = standardHours / 24
      // 如果已超过标准时间，显示"超期"
      if (diffDays > standardDays) {
        const overdueDays = diffDays - standardDays
        if (overdueDays === 1) return '超期1天'
        return `超期${overdueDays}天`
      }
    }
  }

  // 2.2 普通节点或未超期：显示"历时"
  if (diffDays === 1) return '历时1天'
  return `历时${diffDays}天`
}
```

### 颜色判断

```typescript
const getDateAlertColor = (date: Date, label?: string): 'red' | 'orange' | 'green' | 'blue' => {
  const now = new Date()

  // 未来日期：倒计时
  if (date > now) {
    const diffDays = Math.ceil((date.getTime() - now.getTime()) / (1000 * 60 * 60 * 24))
    if (diffDays <= 3) return 'orange'
    return 'green'
  }

  // 过去日期
  const diffDays = Math.floor((now.getTime() - date.getTime()) / (1000 * 60 * 60 * 24))

  // 关键节点（最晚提柜/最晚还箱）：判断是否超期
  if (label && (label === '最晚提柜' || label === '最晚还箱')) {
    const standardHours = STANDARD_DURATIONS[label] ?? 0
    if (standardHours > 0) {
      const standardDays = standardHours / 24
      // 已超期：红色
      if (diffDays > standardDays) return 'red'
    }
  }

  // 未超期或普通节点：蓝色（历时）
  return 'blue'
}
```

## 关键节点特殊处理

### 最晚提柜日 / 最晚还箱日

**计算逻辑**：
1. 检查是否为关键节点（label === '最晚提柜' 或 label === '最晚还箱'）
2. 如果已超过目标日期：
   - 计算实际已过天数：`diffDays = 当前日期 - 目标日期`
   - 获取标准耗时：`standardDays = 标准耗时 / 24`
   - 如果 `diffDays > standardDays`：
     - 显示"超期"：`超期${diffDays - standardDays}天`
     - 使用红色脉冲样式
   - 否则：
     - 显示"历时"：`历时${diffDays}天`
     - 使用蓝色样式

**示例**：
- 目标日期：2024-01-15
- 标准耗时：7天
- 当前日期：2024-01-20

计算：
- 实际已过：5天
- 标准：7天
- 未超期（5 < 7）
- 显示：**历时5天**（蓝色）

---

当前日期：2024-01-25

计算：
- 实际已过：10天
- 标准：7天
- 超期：3天（10 - 7）
- 显示：**⚠️超期3天**（红色脉冲）

## 与物流路径标签的对比

| 维度 | 物流路径标签 | 关键日期标签 |
|------|------------|------------|
| **历时** | 节点间衔接时长 | 从目标日期到现在 |
| **超期** | 超过标准耗时 | 仅关键节点超期 |
| **倒计时** | 无 | 未来日期剩余时间 |
| **适用范围** | 所有物流节点 | 关键日期时间线 |
| **颜色方案** | 蓝/红 | 红/橙/绿/蓝 |
