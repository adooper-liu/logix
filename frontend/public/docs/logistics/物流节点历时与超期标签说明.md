# 物流节点历时与超期标签说明

## 核心概念

### 1. 历时 - 衡量历史衔接效率

**定义**：
从上一节点结束到当前节点开始的时间差

**计算公式**：
```
历时 = 当前节点开始时间 - 上一节点结束时间
```

**用途**：
- 衡量流程衔接效率
- 向后看历史（中性指标）
- 用于数据分析和优化

**标记示例**：
```
历时 2天 3小时
```

**应用场景**：
- 货物卸船后多久开始清关
- 清关完成后多久开始提柜
- 提柜后多久到达仓库

**适用范围**：
- 所有已完成或正在进行的节点
- 不分节点状态，全部显示

---

### 2. 超期 - 风险预警指标

**定义**：
当前节点已停留时间与该节点标准耗时的差值

**计算公式**：
```
超期 = 当前时间 - 当前节点开始时间 - 标准耗时
```

**用途**：
- 风险预警
- 向前看未来（负面指标）
- 用于人工干预

**标记示例**：
```
超期 1天 5小时
```

**应用场景**：
- 货物在港口停留时间超过标准
- 清关处理时间超过预期
- 还空箱延迟预警

**适用范围**：
- **仅当前正在进行的节点**
- 已完成的节点不显示超期（已无风险）

---

## 关键区别对比

| 维度 | 历时 | 超期 |
|------|------|------|
| **方向** | 向后看（历史） | 向前看（风险） |
| **性质** | 中性指标 | 负面指标 |
| **用途** | 分析效率 | 干预预警 |
| **适用节点** | 所有节点 | 仅当前节点 |
| **时间基准** | 相对上一节点 | 相对标准耗时 |
| **显示频率** | 始终显示 | 超期时才显示 |

## 当前节点判断逻辑

```typescript
const isCurrentNode = (node: StatusNode, index: number): boolean => {
  // 节点状态为进行中 或 是最后一个节点
  return node.nodeStatus === 'IN_PROGRESS' ||
         index === path.value.nodes.length - 1
}
```

**判断规则**：
- 节点状态为 `IN_PROGRESS`（进行中）
- 或节点是整个路径的最后一个节点

## 代码实现

### 历时计算
```typescript
const getElapsedDuration = (index: number): string => {
  if (!path.value?.nodes?.length || index === 0) return ''

  const prev = path.value.nodes[index - 1]
  const curr = path.value.nodes[index]
  const prevTime = new Date(prev.timestamp).getTime()
  const currTime = new Date(curr.timestamp).getTime()
  const diffHours = (currTime - prevTime) / (1000 * 60 * 60)

  if (diffHours <= 0) return ''

  const days = Math.floor(diffHours / 24)
  let hours = Math.round(diffHours % 24)
  if (hours === 24) { hours = 0 }

  return `${days}天${hours}小时`
}
```

### 超期计算
```typescript
const getOverdueDuration = (node: StatusNode, index: number): string | null => {
  // 仅计算当前正在进行的节点
  if (!isCurrentNode(node, index)) return null

  const standardHours = STANDARD_DURATIONS[node.status] ?? 0
  if (standardHours === 0) return null // 无标准耗时不计算超期

  const nodeStartTime = new Date(node.timestamp).getTime()
  const now = Date.now()
  const elapsedHours = (now - nodeStartTime) / (1000 * 60 * 60)

  // 计算超期时长
  const overdueHours = elapsedHours - standardHours
  if (overdueHours <= 0) return null // 未超期

  const days = Math.floor(overdueHours / 24)
  let hours = Math.round(overdueHours % 24)
  if (hours === 24) { hours = 0 }

  return `${days}天${hours}小时`
}
```

### 获取节点时长信息
```typescript
const getNodeDurationInfo = (node: StatusNode, index: number): {
  elapsed?: string
  overdue?: string
  isOverdue: boolean
} => {
  const elapsed = getElapsedDuration(index)
  const overdue = getOverdueDuration(node, index)
  return {
    elapsed,
    overdue,
    isOverdue: !!overdue
  }
}
```

## 使用示例

### 场景1：正常流程（无超期）
```
已装船 | 2024-01-01 10:00 | 上海港 (CNSHA)
  ↓
航行中 | 2024-01-01 12:00 | — | 历时 2小时
  ↓
中转抵港 | 2024-01-03 08:00 | 新加坡港 (SGSIN) | 历时 1天20小时
```

### 场景2：异常流程（有超期）
```
已到港 | 2024-01-05 10:00 | 洛杉矶港 (USLAX) | 历时 1天2小时
  ↓
当前节点 | 2024-01-05 14:00 | — | 历时 4小时 | 超期 1天2小时
```

## 一句话总结

**历时告诉你"过去花了多久"，超期告诉你"现在卡了多久"**

## 未来优化方向

1. **标准耗时动态配置**
   - 从后端数据库读取标准耗时
   - 支持按港口/航线/货代公司区分标准
   - 支持用户自定义标准耗时

2. **超期预警等级**
   - 轻度超期（黄色预警）
   - 中度超期（橙色预警）
   - 严重超期（红色预警）

3. **历史数据分析**
   - 历时统计报表
   - 效率趋势分析
   - 瓶颈环节识别

4. **智能推荐**
   - 基于历史数据推荐标准耗时
   - 异常模式识别
   - 预测性维护
