# 按到港统计分类完整文档（修正版）

## 概述

本文档详细说明了按到港维度的货柜统计分类规则，确保所有有目的港记录的货柜都能准确分类。

**目标集合**：所有有目的港记录（`port_type='destination'`）的货柜，取每个货柜的最后一个目的港记录（最大 `port_sequence`）。

**分组结构（修正版）**：
1. **已到目的港**：今日到港 + 今日之前到港未提柜 + 今日之前到港已提柜
2. **已到中转港**：有中转港记录，目的港无ATA
3. **预计到港**：逾期未到港 + 3天内 + 7天内 + 7天以上 + 其他

## 关键规则说明

### 1. 目的港选择规则

- 只统计目的港（`port_type='destination'`），排除中转港（`port_type='transit'`）
- 对于有多港经停的货柜，只统计最后一个目的港（最大 `port_sequence`）

### 2. 物流状态分类

**已到港状态**：`at_port`, `picked_up`, `unloaded`, `returned_empty`
- 这些状态表示货柜已经到达目的港或已完成后续流程
- 用于"无ATA"分类（分类4）

**在途状态**：`shipped`, `in_transit`
- 这些状态表示货柜尚未到港
- 用于逾期未到港（分类5）和预计在途（分类6）

### 3. 到港判断

**有ATA时**：使用 ATA 判断是否到港
- ATA = 今日：今日到港（分类1）
- ATA < 今日：已到港（分类2或3）
- ATA > 今日：不存在的异常情况

**无ATA时**：根据物流状态判断
- 状态为 `at_port/picked_up/unloaded/returned_empty`：已到港但无ATA记录（分类4）
- 状态为 `shipped/in_transit`：未到港（根据ETA判断）

### 4. 提柜状态判断

**未提柜**：`logistics_status NOT IN ('picked_up', 'unloaded', 'returned_empty')`
- 包含：`not_shipped`, `shipped`, `in_transit`, `at_port`

**已提柜**：`logistics_status IN ('picked_up', 'unloaded', 'returned_empty')`
- 已完成提柜、卸货或还空箱

## 验证方法

确保所有货柜都被准确分类，执行以下验证：

```sql
-- 检查是否有未分类的货柜
SELECT DISTINCT c.container_number, c.logistics_status, po.ata_dest_port, po.eta_dest_port
FROM biz_containers c
INNER JOIN process_port_operations po ON c.container_number = po.container_number
WHERE po.port_type = 'destination'
AND po.port_sequence = (
  SELECT MAX(po2.port_sequence)
  FROM process_port_operations po2
  WHERE po2.container_number = po.container_number
  AND po2.port_type = 'destination'
)
AND NOT (
  -- 分类1：今日到港
  (po.ata_dest_port IS NOT NULL AND DATE(po.ata_dest_port) = CURRENT_DATE)
  OR
  -- 分类2：今日之前到港未提柜
  (po.ata_dest_port IS NOT NULL AND DATE(po.ata_dest_port) < CURRENT_DATE AND c.logistics_status NOT IN ('picked_up', 'unloaded', 'returned_empty'))
  OR
  -- 分类3：今日之前到港已提柜
  (po.ata_dest_port IS NOT NULL AND DATE(po.ata_dest_port) < CURRENT_DATE AND c.logistics_status IN ('picked_up', 'unloaded', 'returned_empty'))
  OR
  -- 分类4：今日之前到港无ATA
  (po.ata_dest_port IS NULL AND c.logistics_status IN ('at_port', 'picked_up', 'unloaded', 'returned_empty'))
  OR
  -- 分类5：逾期未到港
  (po.eta_dest_port IS NOT NULL AND DATE(po.eta_dest_port) < CURRENT_DATE AND po.ata_dest_port IS NULL AND c.logistics_status IN ('shipped', 'in_transit'))
  OR
  -- 分类6：预计在途
  (po.eta_dest_port IS NOT NULL AND DATE(po.eta_dest_port) >= CURRENT_DATE AND po.ata_dest_port IS NULL AND c.logistics_status IN ('shipped', 'in_transit'))
  OR
  -- 分类7：无ETA无ATA
  (po.eta_dest_port IS NULL AND po.ata_dest_port IS NULL AND c.logistics_status NOT IN ('at_port', 'picked_up', 'unloaded', 'returned_empty'))
);
```

如果查询返回结果为空，说明所有货柜都已正确分类。
