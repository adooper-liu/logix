# Shipments 表格功能梳理与增强方案

本文档针对 `http://localhost:5173/#/shipments` 页的集装箱表格做**功能梳理**与**增强方案**，便于后续按优先级落地。

---

## 一、现状功能梳理

### 1.1 数据与筛选

| 能力 | 现状 | 说明 |
|------|------|------|
| **数据来源** | `GET /api/v1/containers`（分页） + `getContainersByFilterCondition`（按卡片条件） | 与 `SHIPMENTS_TABLE_BINDING.md` 一致 |
| **日期范围** | 顶部 DateRangePicker，与统计卡片、表格共用 | 变更日期会触发重新拉列表 + 统计 |
| **关键词搜索** | 单输入框，支持「集装箱号、备货单号、提单号」 | 回车/点击搜索，后端 `search` 参数 |
| **卡片条件筛选** | 点击 5 组统计卡片内 Badge 后，表格仅显示该条件货柜 | 使用 `getContainersByFilterCondition(filterCondition, startDate, endDate)`，有筛选时前端分页 |
| **重置** | 重置按钮清空关键词 + 卡片条件，并重新 loadContainers | - |
| **刷新** | 「刷新列表」「刷新统计」分别重拉表格、统计 | - |

### 1.2 表格展示

| 能力 | 现状 | 说明 |
|------|------|------|
| **列** | 约 22 列：集装箱号、出运日期、备货单号、提单号、柜型、物流状态、查验、开箱、目的港、当前位置、预计到港、修正ETA、实际到港、清关状态、计划提柜日、最晚提柜日、最晚还箱日、实际还箱日、货物描述、最后更新、操作 | 列与后端字段绑定见 `SHIPMENTS_TABLE_BINDING.md` |
| **固定列** | 集装箱号左固定，操作右固定 | `fixed="left"` / `fixed="right"` |
| **列宽** | 多为固定 width，货物描述 min-width | 无列拖拽调整 |
| **溢出** | 货物描述 `show-overflow-tooltip` | 其余列无统一溢出策略 |
| **排序** | 无 | 未使用 `sortable` |
| **列显隐** | 无 | 不可自定义显示列 |

### 1.3 分页与交互

| 能力 | 现状 | 说明 |
|------|------|------|
| **分页** | 后端分页（无筛选时）或前端分页（有卡片筛选时） | pageSize: 10/20/50/100，layout 含 total/sizes/prev/pager/next/jumper |
| **行操作** | 每行「查看」「编辑」 | 查看跳详情页，编辑仅 Message |
| **多选/批量** | 无 | 无勾选列、无批量操作 |
| **行高亮/展开** | 无 | - |

### 1.4 其他

| 能力 | 现状 | 说明 |
|------|------|------|
| **导出** | 无 | 不支持 Excel/CSV 导出 |
| **甘特图** | 有入口按钮 | 跳转 `/shipments/gantt-chart` |
| **筛选态展示** | 有筛选时顶部显示 el-tag（类型 + 条件文案），可关闭 | `getFilterLabel` 映射 days → 中文 |

## 二、增强方案（建议优先级）

### 2.1 高优先级（体验与数据准确）

| 序号 | 增强项 | 说明 | 实现要点 |
|------|--------|------|----------|
| 1 | **列排序** | 关键列支持前端或后端排序（出运日期、预计到港、实际到港、计划提柜日、最晚提柜日等） | 先做前端排序（当前页或当前数据集）；若数据量大再考虑后端 `sortBy`/`order` 参数 |
| 2 | **空态与加载态统一** | 无数据时明确「无数据」提示；加载时表格区域统一 loading，避免布局跳动 | 使用 el-table 的 `empty` 插槽；loading 已存在，可核对遮罩范围 |
| 3 | **筛选条件下分页一致性** | 有卡片筛选时，后端若支持分页则改为后端分页，避免一次拉全量再前端切页 | 需后端 `getContainersByFilterCondition` 支持 `page`/`pageSize`；若暂不支持，保留前端分页并在 UI 上注明「当前为前端分页」 |

### 2.2 中优先级（效率与可配置）

| 序号 | 增强项 | 说明 | 实现要点 |
|------|--------|------|----------|
| 4 | **列显隐/列序** | 用户可勾选显示列、拖拽调整列顺序 | 列配置存 localStorage；el-table 用 `v-for` 动态列 + `columnKey`；或引入 table 列配置组件 |
| 5 | **导出当前列表** | 支持将当前表格数据导出为 Excel 或 CSV（含当前筛选、分页或当前页） | 前端用 xlsx 或 csv 库，数据源用 `filteredContainers` 或当前页；可加「导出当前页/导出全部（当前条件）」选项 |
| 6 | **表格密度/尺寸** | 支持紧凑/默认/宽松或小/默认/大字号 | el-table 的 `size` 或自定义 class；可放在工具栏或设置里 |
| 7 | **快捷筛选条** | 在表格上方增加常用筛选（如物流状态、是否逾期、目的港等） | 与现有卡片筛选可并存；可做成 tag 或下拉多选，对应后端参数或前端过滤 |

### 2.3 低优先级（扩展能力）

| 序号 | 增强项 | 说明 | 实现要点 |
|------|--------|------|----------|
| 8 | **多选 + 批量操作** | 勾选多行，支持批量导出、批量打标签或后续业务操作 | el-table-column type="selection"；工具栏「批量导出」「批量…」按选中行操作 |
| 9 | **列宽拖拽** | 表头可拖拽调整列宽 | el-table 默认不支持，需第三方或自封装带 resize 的列 |
| 10 | **行详情展开** | 点击展开行显示港口操作、拖车、还箱等子信息 | el-table 的 `type="expand"` + 插槽，数据可从详情接口或列表 enrich 字段取 |
| 11 | **从表格跳转甘特图并带选中柜号** | 支持勾选若干柜号后「在甘特图中查看」 | 甘特图路由支持 query 如 `?containers=xxx,yyy`，甘特图页只高亮或只显示这些柜 |

---

## 三、实现顺序建议（已按序实现）

1. **Phase 1（快速收益）**：列排序（1）、空态/加载态（2）、导出当前页/当前条件（5）。✅
2. **Phase 2（配置与体验）**：列显隐（4）、表格密度（6）、快捷筛选条（7）。✅ 列序未做拖拽，仅显隐。
3. **Phase 3（后端配合）**：筛选条件下后端分页（3）。⏸ 需后端支持 `page`/`pageSize` 后接入。
4. **Phase 4（扩展）**：多选与批量导出（8）、行展开（10）、甘特图带选中柜号（11）。✅ 列宽拖拽（9）未做（需第三方或自封装）。

---

## 四、与现有文档的关系

- **SHIPMENTS_TABLE_BINDING.md**：表列与后端字段对应、数据流，改列或改接口时需同步更新。
- **SHIPMENTS_CARD_STATISTICS_DESIGN_CONFIRM.md**：统计卡片口径与筛选条件，表格筛选与之一致。
- 本方案仅针对**表格**本身的梳理与增强，不改变统计卡片逻辑与接口约定。

## 五、实现状态与后续

### 5.1 已实现（见 Phase 1–4 标注）

- 列排序、空态/加载态、导出（当前页/全部条件）、列显隐、表格密度、快捷筛选条、多选与批量导出、行详情展开、甘特图带选中柜号（Shipments 页已传 `query.containers`）。

### 5.2 待配合或后续

- **甘特图页使用 query.containers**：当前仅 Shipments 跳转时携带 `containers` 与日期；甘特图页（`GanttChart.vue`）尚未根据 `route.query.containers` 过滤或只展示这些柜号，可后续在甘特图侧读取并过滤。
- **筛选条件下后端分页**：需后端 `getContainersByFilterCondition` 支持 `page`/`pageSize` 及 total，前端再接入。
- **列宽拖拽**：需可拖拽列宽的 table 组件或自定义表头。

### 5.3 可补充

- 具体交互稿（列显隐弹窗、导出弹窗等）
- 后端接口变更清单（如排序参数、筛选分页参数）
- 可访问性（键盘、焦点、屏幕阅读器）与性能（大列表虚拟滚动）要求
