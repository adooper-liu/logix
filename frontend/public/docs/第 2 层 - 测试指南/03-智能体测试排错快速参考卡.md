# 智能体测试与排错快速参考卡

## 一、测试流程速查

### 开发前检查（30 秒）

```
1. 运行现有测试 -> npm run test
2. 查看测试覆盖率 -> 核心逻辑是否≥80%
3. 确认测试文件位置 -> 同级或 tests/目录
```

### 测试编写决策树

```
需要修改代码？
  ├─ Bug 修复 -> 先写复现问题的失败测试
  ├─ 新功能 -> 先写 TDD 测试用例
  ├─ 重构 -> 确保现有测试通过
  └─ 性能优化 -> 先写性能基准测试
```

### 测试执行策略

| 阶段 | 执行范围 | 命令 |
|------|---------|------|
| 开发中 | 相关单元测试 | `npm run test -- [文件名]` |
| 提交前 | 全部单元测试 + 关键集成 | `npm run test` |
| CI/CD | 完整测试套件 | `npm run test && npm run test:e2e` |

## 二、排错流程速查

### 5 分钟快速诊断

```bash
# 1. 看日志（2 分钟）
tail -f backend/logs/error.log
grep "[错误关键词]" backend.log

# 2. 查数据库（2 分钟）
psql -c "SELECT * FROM [表] WHERE [条件];"

# 3. 测 API（1 分钟）
curl http://localhost:3001/api/v1/[接口]
```

### 问题分类与应对

| 症状 | 可能原因 | 排查工具 |
|------|---------|---------|
| 页面无数据 | API 失败/前端过滤/网络问题 | DevTools Network |
| 数据错误 | 数据库/业务逻辑/映射错误 | SQL 查询 + 调试脚本 |
| 功能失效 | 权限/配置/依赖服务 | 后端日志 + Postman |
| 性能问题 | SQL 慢查询/循环嵌套 | EXPLAIN + Profiling |

### 根因分析模板

## 四、数据库查询速查

### 基础查询

```sql
-- 检查数据量
SELECT COUNT(*) FROM biz_containers;

-- 按状态分组
SELECT logistics_status, COUNT(*) 
FROM biz_containers 
GROUP BY logistics_status;

-- 按日期范围查询
SELECT * FROM biz_containers 
WHERE actual_ship_date >= '2024-01-01' 
  AND actual_ship_date <= '2024-12-31';
```

### 关联查询

```sql
-- 货柜 + 港口操作
SELECT c.container_number, po.ata, po.port_type
FROM biz_containers c
JOIN process_port_operations po ON c.container_number = po.container_number
WHERE c.container_number = 'XXXXX';

-- 货柜 + 海运 + 备货单
SELECT c.*, sf.shipment_date, ro.order_number
FROM biz_containers c
LEFT JOIN process_sea_freight sf ON c.bill_of_lading_number = sf.bill_of_lading_number
LEFT JOIN biz_replenishment_orders ro ON c.container_number = ro.container_number
WHERE c.container_number = 'XXXXX';
```

### 数据验证

```sql
-- 检查不一致状态
SELECT container_number, logistics_status, ata_dest_port
FROM biz_containers
WHERE logistics_status = 'at_port' 
  AND ata_dest_port IS NULL;

-- 检查缺失数据
SELECT c.container_number
FROM biz_containers c
LEFT JOIN process_sea_freight sf ON c.bill_of_lading_number = sf.bill_of_lading_number
WHERE sf.bill_of_lading_number IS NULL;
```

## 五、常见错误快速修复

### 错误 1: 国家筛选无数据

**症状**: 下拉框为空或筛选无效

**快速检查**:
```bash
# 1. 查数据库
psql -c "SELECT COUNT(*) FROM dict_countries;"

# 2. 测 API
curl http://localhost:3001/api/v1/countries

# 3. 看前端
# DevTools -> Network -> /countries
```

**修复步骤**:
1. 数据库无数据 -> 运行字典表初始化 SQL
2. API 报错 -> 检查后端路由和控制器
3. 前端问题 -> 检查服务和组件

### 错误 2: 统计显示为 0

**症状**: 统计卡片全为 0

**快速检查**:
```bash
# 1. 测试 API
node scripts/test-statistics-fix.js

# 2. 检查日期范围
# 确认顶部日期选择器有值

# 3. 查数据库
psql -c "SELECT COUNT(*) FROM biz_containers WHERE actual_ship_date >= CURRENT_DATE - INTERVAL '30 days';"
```

### 错误 3: 导入失败

**症状**: Excel 导入报错

**快速检查**:
```bash
# 1. 查看错误日志
grep "import" backend/logs/error.log

# 2. 检查映射
psql -c "SELECT * FROM dict_trucking_port_mapping;"

# 3. 验证数据
node scripts/check-imported-data.js
```
