# LogiX 智能体测试与排错规范

## 📚 文档概述

这套规范为 LogiX 项目的智能体（AI 助手）和开发人员提供完整的测试与排错标准作业程序（SOP）。

**核心理念**:
- **测试优先**: 任何代码修改前必须先理解现有测试
- **证据驱动**: 基于日志、数据库查询等客观证据排查问题
- **流程标准化**: 统一的诊断流程和记录模板
- **知识沉淀**: 每个问题都是学习和改进的机会

### 03-快速参考卡（打印置顶）

**核心价值**:
- **5 分钟快速诊断**: 日志、数据库、API 三步定位
- **问题分类决策树**: 根据症状快速判断方向
- **常用脚本索引**: 所有调试脚本位置和使用方法
- **SQL 查询速查**: 基础查询、关联查询、数据验证
- **常见错误修复**: 国家筛选、统计 API、导入失败等

**使用建议**: A4 双面打印，贴在工作台

### 04-检查清单（执行时对照）

**检查阶段**:
```
1. 开发前测试检查
   ├─ 现有测试覆盖检查
   └─ 测试策略确定

2. 单元测试编写检查
   ├─ 测试结构
   ├─ 测试用例设计
   ├─ Mock 检查
   └─ 断言检查

3. 问题诊断检查
   ├─ 信息收集（5W1H）
   ├─ 日志检查
   ├─ 数据库检查
   └─ API 检查

4. 根因分析检查
   ├─ 五问法分析
   ├─ 二分法定位
   └─ 常见原因排查

5. 解决方案验证检查
   ├─ 代码修复验证
   ├─ 测试验证
   ├─ 功能验证
   └─ 数据验证

6. 自动化检查执行
   └─ 提交前检查（必须 100% 通过）

7. 问题升级检查
8. 知识沉淀检查
9. 测试数据管理检查
10. 持续改进检查
```

**使用建议**: 每项完成后打勾✓，重要任务拍照存档

## 🚀 典型使用场景

```

```

## 📋 每日开发流程

### 早上开始工作

```bash
# 1. 运行全部测试，确认基线正常
cd backend && npm run test
cd frontend && npm run test:unit

# 2. 查看测试结果
# - 如有失败，优先修复
# - 如全部通过，开始今日工作
```

### 开发过程中

对于每次代码修改:
```
1. 思考：需要什么类型的测试？
2. 检查：现有测试是否覆盖？
3. 编写：补充必要的单元测试
4. 验证：运行测试确认通过
```

### 提交代码前

```bash
# 必须执行的检查（检查清单 6.1）
npm run lint          # ESLint
npm run type-check    # TypeScript
npm run format        # Prettier
npm run test          # 全部单元测试
npm run skill:check   # SKILL 合规

# 确认所有检查 100% 通过才能提交
```

## 🛠️ 核心工具集

### 测试工具

| 工具 | 用途 | 运行命令 |
|------|------|----------|
| Jest | 后端单元测试 | `cd backend && npm run test` |
| Vitest | 前端单元测试 | `cd frontend && npm run test:unit` |
| Playwright | E2E 测试 | `cd frontend && npm run test:e2e` |

### 调试脚本

| 脚本 | 用途 | 位置 |
|------|------|------|
| diagnose-country-filter.js | 国家筛选诊断 | `scripts/` |
| test-statistics-fix.js | 统计 API 测试 | `scripts/` |
| debug-lastpickup.ts | 最晚提柜检查 | `backend/` |
| analyze-return-date-logs.ps1 | 还箱日日志分析 | `scripts/` |

### 检查工具

| 工具 | 用途 | 命令 |
|------|------|------|
| dev-paradigm-check.ts | 开发范式检查 | `npx ts-node scripts/dev-paradigm-check.ts` |
| check-skill-compliance.js | SKILL 合规检查 | `cd backend && npm run skill:check` |
