## 1. 五个计划日期概述

智能排柜模块（Intelligent Scheduling）以 **卸柜日 (plannedUnloadDate)** 为核心锚点，向前向后推导其他4个日期。

### 五个计划日期定义

| 序号 | 日期字段 | 中文名 | 说明 |
|------|----------|--------|------|
| 1 | `plannedCustomsDate` | 计划清关日 | 锚点向前推导：ETA - 1天 |
| 2 | `plannedPickupDate` | 计划提柜日 | 锚点向前推导：清关日 + 1天 ≤ lastFreeDate |
| 3 | `plannedDeliveryDate` | 计划送仓日 | 锚点向前推导：提柜日 + 运输天数 |
| 4 | `plannedUnloadDate` | 计划卸柜日 | **核心锚点**：基于仓库可用性计算 |
| 5 | `plannedReturnDate` | 计划还箱日 | 锚点向后推导：卸柜日 + 在仓天数 |

### 日期推导方向图

```
         ←─────────── 向前推导 ───────────→        ←── 向后推导 ──→

┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   计划清关日    计划提柜日    计划送仓日    计划卸柜日    计划还箱日        │
│  ───────────    ──────────    ──────────    ──────────    ─────────        │
│                                                                             │
│   planned      planned        planned        planned        planned         │
│  CustomsDate  PickupDate    DeliveryDate    UnloadDate    ReturnDate        │
│                                                                             │
│       ↑            ↑             ↑             ↑              ↑            │
│       │            │             │             │              │            │
│    ETA-1天     customs+1天    提+运输天      锚点          卸+在仓天       │
│                 (≤lastFree)                                           │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 2. 三种排产模式概述

智能排柜模块支持 **三种排产模式**，它们在日期计算逻辑上有细微差异：

### 模式对比表

| 模式 | 触发方式 | dryRun参数 | 数据保存 | 用途 |
|------|----------|------------|----------|------|
| **① 预览排产** | 前端点击"预览排产" | `true` | ❌ 不保存 | 预览方案，审查后批量保存 |
| **② 单柜优化** | 前端点击"优化"按钮 | `false` | ❌ 可选保存 | 针对单个货柜的成本优化 |
| **③ 批量排产** | 前端点击"执行排产" | `false` | ✅ 保存到DB | 正式排产，批量处理 |

## 3. 锚点确定逻辑

### 锚点选择规则

```
卸柜日 (plannedUnloadDate) 是核心锚点，通过仓库可用性确定：

Step 1: 从计划提柜日(plannedPickupDate)开始
Step 2: 检查仓库当日产能 (dailyUnloadCapacity)
Step 3: 如果产能已满，向后查找下一个可用日
Step 4: 返回最早可用日作为卸柜日
```

### 核心代码逻辑

```typescript
async findEarliestAvailableWarehouse(warehouses: Warehouse[], pickupDate: Date) {
  for (const warehouse of warehouses) {
    // 从提柜日开始，依次检查每一天
    let checkDate = new Date(pickupDate);
    for (let i = 0; i < 30; i++) { // 最多查30天
      const isAvailable = await checkWarehouseCapacity(warehouse, checkDate);
      if (isAvailable) {
        return { warehouse, plannedUnloadDate: checkDate };
      }
      checkDate.setDate(checkDate.getDate() + 1); // 向后找
    }
  }
  return null; // 30天内无产能
}
```

## 4. 各日期计算公式

### 4.1 计划清关日 (plannedCustomsDate)

**计算公式**:
```
plannedCustomsDate = ATA/ETA - 1天 - ETA缓冲天数
```

**参数说明**:

| 参数 | 来源 | 说明 |
|------|------|------|
| ATA | port_operations.ata_dest_port | 实际到港日期 |
| ETA | port_operations.eta_dest_port | 预计到港日期 |
| ETA缓冲天数 | 请求参数 etaBufferDays | 前端传入的顺延天数 |

### 4.2 计划提柜日 (plannedPickupDate)

**计算公式**:
```
plannedPickupDate = plannedCustomsDate + 1天，且 ≤ lastFreeDate
```

**约束条件**:
- 必须 ≥ 今天（如果计算结果 ≤ 今天，调整为明天）
- 必须 ≤ lastFreeDate（最晚提柜日）
- 跳过周末（如果配置 skip_weekends = true）

### 4.3 计划送仓日 (plannedDeliveryDate)

**计算公式**:
```
卸柜模式 = 'Live load' 时:  plannedDeliveryDate = plannedPickupDate
卸柜模式 = 'Drop off' 时:  plannedDeliveryDate = plannedUnloadDate
```

### 4.4 计划卸柜日 (plannedUnloadDate)

**说明**: 这是核心锚点，基于仓库可用性动态计算。

**计算逻辑**:
```
输入: 候选仓库列表, 计划提柜日
输出: 最早可用仓库 + 对应卸柜日

步骤:
1. 遍历候选仓库
2. 从计划提柜日开始，检查每日产能
3. 找到第一个产能未满的日期
4. 返回仓库和卸柜日
```

### 4.5 计划还箱日 (plannedReturnDate)

**计算公式**:
```
卸柜模式 = 'Drop off' 时:  plannedReturnDate = 卸柜日 + 1天 (受还箱能力约束)
卸柜模式 = 'Live load' 时:  plannedReturnDate = 卸柜日 (同日)
```

**fallback规则**:
```typescript
// 1. 优先从 EmptyReturn 表获取 lastReturnDate
if (emptyReturn?.lastReturnDate) {
  lastReturnDate = emptyReturn.lastReturnDate;
}
// 2. fallback: lastFreeDate + 7天
else if (destPo.lastFreeDate) {
  lastReturnDate = destPo.lastFreeDate + 7;
}
```

## 5. 场景模拟

### 场景 1: Live load 模式（现场装卸）

**数据条件**:
- ATA: 2026-02-01
- lastFreeDate: 2026-02-10
- 仓库产能: 充足
- 车队堆场: 无 (hasYard = false)

**计算过程**:

| 步骤 | 计算 | 结果 |
|------|------|------|
| 1 | plannedCustomsDate = ATA - 1 | 2026-02-01 - 1 = 2026-01-31 |
| 2 | plannedPickupDate = customs + 1 | 2026-02-01 (≤ lastFreeDate) |
| 3 | unloadMode = Live load (无堆场) | - |
| 4 | plannedUnloadDate = 仓库可用性 | 2026-02-01 (当天有产能) |
| 5 | plannedDeliveryDate = pickup (Live load) | 2026-02-01 |
| 6 | plannedReturnDate = unload (Live load) | 2026-02-01 |

**结果**:
```
01-31  →  02-01  →  02-01  →  02-01  →  02-01
清关       提柜      送仓       卸柜       还箱

全部同一天完成
```

### 场景 2: Drop off 模式（有堆场）

**数据条件**:
- ATA: 2026-02-01
- lastFreeDate: 2026-02-15
- 仓库产能: 提柜日当天满，次日有产能
- 车队堆场: 有 (hasYard = true)

**计算过程**:

| 步骤 | 计算 | 结果 |
|------|------|------|
| 1 | plannedCustomsDate = ATA - 1 | 2026-01-31 |
| 2 | plannedPickupDate = customs + 1 | 2026-02-01 (≤ lastFreeDate) |
| 3 | unloadMode = Drop off (有堆场) | - |
| 4 | 仓库产能检查 | 02-01 满 → 02-02 有产能 |
| 5 | plannedUnloadDate | 2026-02-02 |
| 6 | plannedDeliveryDate = unload (Drop off) | 2026-02-02 |
| 7 | plannedReturnDate = unload + 1 | 2026-02-03 |

## 6. 日期链完整性约束

### 约束规则

```
1. 清关日 < 提柜日
2. 提柜日 ≤ 送仓日 ≤ 卸柜日
3. 卸柜日 ≤ 还箱日
4. 提柜日 ≤ lastFreeDate (最晚提柜日)
5. 还箱日 ≤ lastReturnDate (最晚还箱日)
```

### 约束验证代码

```typescript
function validateDateChain(dates: {
  plannedCustomsDate: Date;
  plannedPickupDate: Date;
  plannedDeliveryDate: Date;
  plannedUnloadDate: Date;
  plannedReturnDate: Date;
}): boolean {
  return (
    dates.plannedCustomsDate < dates.plannedPickupDate &&
    dates.plannedPickupDate <= dates.plannedDeliveryDate &&
    dates.plannedDeliveryDate <= dates.plannedUnloadDate &&
    dates.plannedUnloadDate <= dates.plannedReturnDate
  );
}
```
