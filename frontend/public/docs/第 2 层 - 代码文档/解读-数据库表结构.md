## 六、表关系架构 (7层流转)

```
┌─────────────────────────────────────────────────────────────────┐
│  Layer 1: 备货单 (biz_replenishment_orders)                     │
│  order_number (PK)                                              │
└────────────────────────────┬────────────────────────────────────┘
                             │ 1:N
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  Layer 2: 货柜 (biz_containers)                                 │
│  container_number (PK), order_number (FK)                       │
│  logistics_status: not_shipped → shipped → in_transit → ...     │
└────────────────────────────┬────────────────────────────────────┘
                             │ 1:N
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  Layer 3: 海运 (process_sea_freight)                            │
│  id, container_number (FK), bill_of_lading_number               │
└────────────────────────────┬────────────────────────────────────┘
                             │ 1:N
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  Layer 4: 港口操作 (process_port_operations)                    │
│  id, container_number (FK), port_type (origin/transit/dest)    │
│  port_sequence, eta_dest_port, ata_dest_port                     │
└────────────────────────────┬────────────────────────────────────┘
                             │ 1:N
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  Layer 5: 拖卡运输 (process_trucking_transport)                │
│  id, container_number (FK), planned_pickup_date, gate_in_time   │
└────────────────────────────┬────────────────────────────────────┘
                             │ 1:N
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  Layer 6: 仓库操作 (process_warehouse_operations)               │
│  id, container_number (FK), warehouse_code, unload_date         │
└────────────────────────────┬────────────────────────────────────┘
                             │ 1:N
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  Layer 7: 还空箱 (process_empty_returns)                         │
│  id, container_number (FK), return_date                         │
└─────────────────────────────────────────────────────────────────┘
```

## 七、物流状态流转

```
not_shipped (未出运)
    │ 排产成功
    ▼
shipped (已出运)
    │ 到达起运港
    ▼
in_transit (在途)
    │ 到达目的港
    ▼
at_port (已到港)
    │ 提柜
    ▼
picked_up (已提柜)
    │ 卸柜
    ▼
unloaded (已卸柜)
    │ 还箱
    ▼
returned_empty (已还箱)
```

## 八、关键索引

```sql
-- 货柜状态索引
CREATE INDEX idx_containers_status ON biz_containers(logistics_status);
CREATE INDEX idx_containers_order ON biz_containers(order_number);
CREATE INDEX idx_containers_port ON biz_containers(destination_port);

-- 港口操作索引
CREATE INDEX idx_port_operations_container ON process_port_operations(container_number);
CREATE INDEX idx_port_operations_type ON process_port_operations(port_type);
CREATE INDEX idx_port_operations_eta ON process_port_operations(eta_dest_port);

-- 拖卡运输索引
CREATE INDEX idx_trucking_container ON process_trucking_transport(container_number);
CREATE INDEX idx_trucking_pickup ON process_trucking_transport(planned_pickup_date);

-- 仓库操作索引
CREATE INDEX idx_warehouse_container ON process_warehouse_operations(container_number);
CREATE INDEX idx_warehouse_date ON process_warehouse_operations(unload_date);

-- 滞港费标准索引
CREATE INDEX idx_demurrage_standard_lookup ON ext_demurrage_standards(
  overseas_company_code, destination_port_code, shipping_company_code, freight_forwarder_code
);
```

## 九、常用查询

### 9.1 按到港统计
```sql
SELECT c.*, po.ata_dest_port
FROM biz_containers c
JOIN process_port_operations po ON c.container_number = po.container_number
WHERE po.port_type = 'destination'
  AND po.port_sequence = (
    SELECT MAX(port_sequence)
    FROM process_port_operations
    WHERE container_number = c.container_number
      AND port_type = 'destination'
  )
  AND DATE(po.ata_dest_port) = CURRENT_DATE;
```

### 9.2 按计划提柜统计
```sql
SELECT c.*, tt.planned_pickup_date
FROM biz_containers c
JOIN process_trucking_transport tt ON c.container_number = tt.container_number
WHERE tt.planned_pickup_date < CURRENT_DATE
  AND c.logistics_status NOT IN ('picked_up', 'unloaded', 'returned_empty');
```
