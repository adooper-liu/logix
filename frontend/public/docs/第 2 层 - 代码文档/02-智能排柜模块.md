# 智能排柜模块

## 1. 核心计算逻辑

### 锚点日期计算

智能排柜以 **卸柜日(plannedUnloadDate)** 为锚点，向前向后推导其他日期：

```
5个关键日期关系:
plannedCustomsDate   ← 计划清关日
        ↓
plannedPickupDate    ← 计划提柜日 (锚点前)
        ↓
plannedUnloadDate    ← 计划卸柜日 (核心锚点)
        ↓
plannedDeliveryDate  ← 计划送仓日
        ↓
plannedReturnDate    ← 计划还箱日
```

### 计算规则

| 日期                | 计算规则                                   |
| ------------------- | ------------------------------------------ |
| plannedCustomsDate  | plannedPickupDate - 1天                    |
| plannedPickupDate   | plannedCustomsDate + 1天 (若已过期则≥今天) |
| plannedUnloadDate   | 基于仓库可用性，从提柜日开始向后查找       |
| plannedDeliveryDate | plannedPickupDate + 运输天数               |
| plannedReturnDate   | plannedUnloadDate + 在仓天数               |

## 2. 场景模拟

### 场景 1: 仓库有足够产能

**参数**:

- 仓库: UK-S005 (日产能: 5柜)
- 提柜日候选: 2026-03-28
- 运输天数: 1天
- 在仓天数: 3天

**计算过程**:

```
Step 1: 确定卸柜日
  2026-03-28 + 1天 = 2026-03-29 (运输日)
  检查 2026-03-29 仓库产能: 3/5 (可用)
  选定: 2026-03-29

Step 2: 向前推导
  还箱日: 2026-03-29 + 3天 = 2026-04-01
  送仓日: 2026-03-28 + 1天 = 2026-03-29
  提柜日: 2026-03-28
  清关日: 2026-03-28 - 1天 = 2026-03-27

Step 3: 结果
  plannedCustomsDate:    2026-03-27
  plannedPickupDate:     2026-03-28
  plannedUnloadDate:     2026-03-29
  plannedDeliveryDate:   2026-03-29
  plannedReturnDate:     2026-04-01
```

**结果**: 所有日期有效

### 场景 2: 仓库产能不足，需要顺延

**参数**:

- 仓库: UK-S005 (日产能: 5柜)
- 已有排产: 2026-03-29 已排 5柜 (满)
- 提柜日候选: 2026-03-28

**计算过程**:

```
Step 1: 尝试 2026-03-29 卸柜
  仓库产能: 5/5 (已满)
  无法安排 → 顺延

Step 2: 尝试 2026-03-30 卸柜
  仓库产能: 2/5 (可用)
  选定: 2026-03-30

Step 3: 结果
  plannedCustomsDate:    2026-03-27
  plannedPickupDate:     2026-03-28
  plannedUnloadDate:     2026-03-30 (顺延1天)
  plannedDeliveryDate:   2026-03-29
  plannedReturnDate:     2026-04-02
```

**结果**: 自动顺延到有产能的日期

### 场景 3: 多日产能不足

**参数**:

- 仓库: UK-S005 (日产能: 3柜)
- 连续3天已满: 03-29, 03-30, 03-31
- 提柜日候选: 2026-03-28

**计算过程**:

```
Step 1: 搜索可用日期
  03-29: 满 → 跳过
  03-30: 满 → 跳过
  03-31: 满 → 跳过
  04-01: 1/3 (可用)
  选定: 2026-04-01

Step 2: 结果
  plannedCustomsDate:    2026-03-27
  plannedPickupDate:     2026-03-28
  plannedUnloadDate:     2026-04-01 (顺延3天)
  plannedDeliveryDate:   2026-03-29
  plannedReturnDate:     2026-04-04
```

**结果**: 自动顺延到04-01

## 3. 排产策略对比

| 策略     | 说明                                 | 适用场景     |
| -------- | ------------------------------------ | ------------ |
| 最早可用 | 从候选日期向后找第一个有产能的日期   | 追求最快完成 |
| 成本最优 | 结合滞港费计算，选择总成本最低的日期 | 成本敏感     |
| 负载均衡 | 优先选择产能利用率低的日期           | 资源平衡     |

## 4. 三种排产模式

| 模式     | 触发方式           | dryRun | 数据保存 | 用途                     |
| -------- | ------------------ | ------ | -------- | ------------------------ |
| 预览排产 | 前端点击"预览排产" | true   | 不保存   | 预览方案，审查后批量保存 |
| 单柜优化 | 前端点击"优化"按钮 | false  | 可选保存 | 针对单个货柜的成本优化   |
| 批量排产 | 前端点击"执行排产" | false  | 保存到DB | 正式排产，批量处理       |
