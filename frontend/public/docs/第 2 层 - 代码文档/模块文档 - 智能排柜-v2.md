## 一、模块概述

智能排柜模块负责根据 ETA、目的港、仓库产能、车队能力等因素，自动分派仓库和车队，并计算最优排产方案。

### 核心功能

- 批量排产（自动分派资源）
- 排产预览（不保存）
- 成本优化（多方案对比）
- 资源配置管理
- 产能日历查看

## 三、API 接口

### 3.1 API 路由定义

**路由文件**: `backend/src/routes/scheduling.routes.ts`

```typescript
// 智能排柜路由
router.post('/batch-schedule', controller.batchSchedule) // 批量排产

router.get('/resources/overview', controller.getResourcesOverview) // 资源概览
router.get('/resources/mapped', controller.getMappedResources) // 映射资源

router.post('/batch-optimize', controller.batchOptimizeContainers) // 批量优化
```

### 3.3 重要变更说明

#### ❌ 已废弃的 API

```markdown
// 旧版本（已废弃）
POST /api/v1/scheduling/batch ← 不再使用
POST /api/v1/scheduling/preview ← 不存在，已删除
POST /api/v1/scheduling/cost-optimize ← 不再使用

// 新版本（当前使用）
POST /api/v1/scheduling/batch-schedule ← ✅ 正确
POST /api/v1/scheduling/:id/schedule-preview ← ✅ 正确（带 ID）
POST /api/v1/scheduling/optimize-cost ← ✅ 正确
GET /api/v1/scheduling/cost-comparison/:containerNumber ← ✅ 正确（参数为 containerNumber）
```

**变更原因**:

- 批量排产使用更准确的命名 `batch-schedule`
- 预览功能需要指定具体的排产 ID
- 成本优化使用更清晰的命名 `optimize-cost`
- 成本对比的参数应明确为 `containerNumber`

### 4.2 Service 层

**文件**: `backend/src/services/intelligentScheduling.service.ts`

**核心服务**:

- `IntelligentSchedulingService` - 智能排柜核心服务
- `SchedulingCostOptimizerService` - 成本优化服务
- `WarehouseSelectorService` - 仓库选择服务
- `TruckingSelectorService` - 车队选择服务
- `SchedulingDateCalculator` - 排产日期计算

**依赖关系**:

