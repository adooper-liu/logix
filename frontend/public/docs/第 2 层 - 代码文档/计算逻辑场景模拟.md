## 二、滞港费计算模块 (Demurrage)

### 2.1 核心计算逻辑

#### 计算公式

```
费用 = chargeDays × ratePerDay
```

其中：
- `chargeDays` = endDate - lastFreeDate - 1 (收费天数)
- `lastFreeDate` = startDate + freeDays - 1 (最后免费日)

#### 关键参数

| 参数 | 说明 | 来源 |
|------|------|------|
| startDate | 起始日期 | ATA (实际到港) 或 ETA (预计到港) |
| endDate | 截止日期 | min(today, plannedPickupDate) |
| freeDays | 免费天数 | 滞港费标准表 |
| freeDaysBasis | 免费日基准 | '自然日' 或 '工作日' |
| ratePerDay | 日费率 | 滞港费标准表 |

---

### 2.2 场景模拟

#### 场景 1: 免费期内 (未超期)

**参数**:
- ATA (实际到港): 2026-02-01
- 计划提柜日: 2026-03-10
- 免费天数: 7 天
- 日费率: $100/天
- 计算日期: 2026-03-05

**计算过程**:
```
lastFreeDate = 2026-02-01 + 7 - 1 = 2026-02-07
endDate = min(2026-03-05, 2026-03-10) = 2026-03-05

因为 endDate (03-05) <= lastFreeDate (02-07)
所以 chargeDays = 0
费用 = 0 × $100 = $0
```

**结果**: ✅ 免费期内，无需缴费

---

#### 场景 2: 已超期 3 天

**参数**:
- ATA: 2026-02-01
- 计划提柜日: 2026-02-05 (已过)
- 免费天数: 7 天
- 日费率: $100/天
- 计算日期: 2026-02-10

**计算过程**:
```
lastFreeDate = 2026-02-01 + 7 - 1 = 2026-02-07
endDate = min(2026-02-10, 2026-02-05) = 2026-02-05 (取小值)

chargeStart = lastFreeDate + 1 = 2026-02-08
chargeDays = 2026-02-05 - 2026-02-08 + 1 = -2 (负数，取0)

实际: chargeDays = 2月8日到2月5日 = -2天 → 按0处理
费用 = 0 × $100 = $0
```

**问题**: 此场景下 endDate 取了计划日，但应该取 today!

**修正计算**:
```
endDate = min(2026-02-10, 2026-02-05) = 2026-02-05
chargeStart = 2026-02-08
chargeDays = 0 (因为 02-05 <= 02-07)

实际应该是: chargeDays = 2026-02-10 - 2026-02-08 + 1 = 3 天
```

**结果**: ❌ 计费逻辑需要修正 (已修复: paramsOverride)

---

#### 场景 3: 超期 5 天，有分段费率

**参数**:
- ATA: 2026-02-01
- 计划提柜日: 2026-02-20 (已过期)
- 免费天数: 7 天
- 日费率: $100/天
- 分段费率:
  - 1-7天: $100/天
  - 8-14天: $150/天
  - 15天以上: $200/天
- 计算日期: 2026-02-12

**计算过程**:
```
lastFreeDate = 2026-02-07
endDate = min(2026-02-12, 2026-02-20) = 2026-02-12
chargeStart = 2026-02-08
chargeDays = 2026-02-12 - 2026-02-08 + 1 = 5 天

分段计算:
├── 第1段: 1-5天 (第1-5天), $100/天 → 5 × $100 = $500
└── 总费用: $500
```

**结果**: ✅ 费用 = $500

### 2.3 免费日计算基准 (freeDaysBasis)

#### 免费日类型说明

| freeDaysBasis | 免费期计算 | 计费期计算 | 说明 |
|----------------|------------|------------|------|
| `自然日` | 7天 | 7天 | 包含所有日历日 |
| `工作日` | 7天 | 7天 | 只计算周一到周五 |
| `工作+自然日` | 7工作日 | 7天 | 免费期排除周末 |
| `自然+工作日` | 7天 | 7工作日 | 计费期排除周末 |

#### 工作日计算规则

```typescript
// 工作日判断：周日(0)和周六(6)不计入
function isWeekend(d: Date): boolean {
  const dow = d.getUTCDay();
  return dow === 0 || dow === 6;
}

// 添加N个工作日
function addWorkingDays(start: Date, n: number): Date {
  if (n <= 0) return new Date(start.getTime());
  const result = new Date(start.getTime());
  let count = 0;
  while (count < n) {
    if (!isWeekend(result)) count++;
    if (count < n) result.setUTCDate(result.getUTCDate() + 1);
  }
  return result;
}
```

#### 场景 4: 免费日基准对比 (自然日 vs 工作日)

**参数**:
- ATA (实际到港): 2026-02-01 (周六)
- 免费天数: 7天
- 计算日期: 2026-02-15

**场景 4A: freeDaysBasis = "自然日"**

```
Step 1: 计算 lastFreeDate
lastFreeDate = ATA + 7 - 1 = 2026-02-01 + 6天 = 2026-02-07

Step 2: 判断是否超期
endDate = 2026-02-15
因为 2026-02-15 > 2026-02-07 (lastFreeDate)
所以已超期

Step 3: 计算收费天数
chargeStart = lastFreeDate + 1 = 2026-02-08
chargeDays = 2026-02-15 - 2026-02-08 + 1 = 8天

结果: chargeDays = 8天, 超期 1 天
```

**场景 4B: freeDaysBasis = "工作日"**

```
Step 1: 计算 lastFreeDate (跳过周末)
起始: 2026-02-01 (周六)
  02-02 (周日) - 跳过
  02-03 (周一) - 第1个工作日
  02-04 (周二) - 第2个
  02-05 (周三) - 第3个
  02-06 (周四) - 第4个
  02-07 (周五) - 第5个
  02-08 (周六) - 跳过
  02-09 (周日) - 跳过
  02-10 (周一) - 第6个工作日
  02-11 (周二) - 第7个工作日 ← lastFreeDate

lastFreeDate = 2026-02-11 (周二)

Step 2: 判断是否超期
因为 2026-02-15 > 2026-02-11
所以已超期

Step 3: 计算收费天数 (按工作日)
chargeStart = 2026-02-12
  02-12 (周三) - 第1个工作日
  02-13 (周四) - 第2个
  02-14 (周五) - 第3个
  02-15 (周六) - 跳过
  → chargeDays = 3天

结果: chargeDays = 3天 (比自然日少5天周末)
```

**对比表**:

| 基准类型 | lastFreeDate | chargeDays | 费用($50/天) |
|----------|--------------|------------|--------------|
| 自然日 | 02-07 | 8天 | $400 |
| 工作日 | 02-11 | 3天 | $150 |

## 三、智能排柜模块 (Scheduling)

### 3.1 核心计算逻辑

#### 锚点日期计算

智能排柜以 **卸柜日(plannedUnloadDate)** 为锚点，向前向后推导其他日期：

```
5个关键日期关系:
┌──────────────────────────────────────────────────────────────┐
│  plannedCustomsDate   ← 计划清关日                          │
│        ↓                                                  │
│  plannedPickupDate    ← 计划提柜日 (锚点前)                 │
│        ↓                                                  │
│  plannedUnloadDate   ← 计划卸柜日 (核心锚点)               │
│        ↓                                                  │
│  plannedDeliveryDate ← 计划送仓日                          │
│        ↓                                                  │
│  plannedReturnDate    ← 计划还箱日                          │
└──────────────────────────────────────────────────────────────┘
```

#### 计算规则

| 日期 | 计算规则 |
|------|----------|
| plannedCustomsDate | plannedPickupDate - 1天 |
| plannedPickupDate | plannedCustomsDate + 1天 (若已过期则≥今天) |
| plannedUnloadDate | 基于仓库可用性，从提柜日开始向后查找 |
| plannedDeliveryDate | plannedPickupDate + 运输天数 |
| plannedReturnDate | plannedUnloadDate + 在仓天数 |

---

## 四、成本优化模块 (Cost Optimization)

### 4.1 核心计算逻辑

#### 成本构成

```
TotalCost = DemurrageCost + DetentionCost + StorageCost
```

其中:
- **Demurrage (滞港费)**: 超过免费期后，货物滞留在港口的费用
- **Detention (滞箱费)**: 超过免费期后，集装箱滞留在手中的费用
- **Storage (堆存费)**: 港口堆存费用

#### 优化目标

在满足约束条件下，找到**总成本最低**的候选日期组合。

---

修正: rangeEnd = min(2026-03-31, 2026-03-27) = 2026-03-27

## 六、总结

### 6.1 各模块计算对比

| 模块 | 核心参数 | 计算目标 | 输出 |
|------|----------|----------|------|
| 滞港费 | ATA, freeDays, rate | 收费天数 × 日费率 | 费用金额 |
| 智能排柜 | 仓库产能, 运输天数 | 日期推导 | 5个计划日期 |
| 成本优化 | 候选日期列表 | 总成本最低 | 最优日期 |
| 统计 | 筛选条件 | 分类计数 | 统计数量 |

### 6.2 计算逻辑要点

1. **日期边界处理**: `min()` vs `max()` 决定费用方向
2. **计算模式区分**: actual 用实际日期，forecast 用计划日期
3. **分段费率**: 超过免费期后，不同天数段费率不同
4. **仓库产能**: 排产时需要检查并更新可用容量

### 6.3 免费日计算要点

| 要点 | 说明 |
|------|------|
| **lastFreeDate 公式** | `起算日 + freeDays - 1` |
| **起算日类型** | ATA (按到港) 或 dischargeDate (按卸船) |
| **自然日** | 所有日历日都计入 |
| **工作日** | 周六、周日不计入 |
| **计算方式** | "按到港" 用ATA，"按卸船" 用卸船日 |
| **最晚还箱日** | 从计划/实际提柜日起算 |

## 七、各类费用计算详解

### 7.1 费用类型识别逻辑

LogiX 系统通过费用名称自动识别费用类型：

```typescript
// 费用类型识别函数
function getChargeType(std: { chargeTypeCode?: string; chargeName?: string }): string {
  const code = (std.chargeTypeCode ?? '').toUpperCase();
  const name = (std.chargeName ?? '').toLowerCase();
  
  // 1. 合并 D&D (同时包含 Demurrage 和 Detention)
  if (code.includes('DEMURRAGE') && code.includes('DETENTION')) return 'D&D';
  if (name.includes('demurrage') && name.includes('detention')) return 'D&D';
  
  // 2. 滞箱费 (Detention)
  if (code.includes('DETENTION') || name.includes('detention') || name.includes('滞箱')) 
    return 'Detention';
  
  // 3. 港口堆存费 (Storage)
  if (code.includes('STORAGE') || name.includes('storage') || name.includes('堆存')) 
    return 'Storage';
  
  // 4. 滞港费 (Demurrage) - 默认
  if (code.includes('DEMURRAGE') || name.includes('demurrage') || name.includes('滞港')) 
    return 'Demurrage';
  
  return 'Unknown';
}
```

### 7.2 滞港费 (Demurrage) 计算

#### 定义

- **区间**: 到港/卸船 → 实际提柜日（无则当天）
- **起算日**: ATA 或卸船日 (dischargeDate)
- **截止日**: 实际提柜日 或 今天 (取小值)
- **免费日**: lastFreeDate = 起算日 + freeDays - 1

#### 计算公式

```
chargeDays = max(0, endDate - lastFreeDate - 1)
费用 = chargeDays × ratePerDay
```

#### 场景模拟

**参数**:
- ATA: 2026-02-01
- 卸船日: 2026-02-03
- 计划提柜日: 2026-02-15
- 实际提柜日: 2026-02-20
- 免费天数: 7天
- 日费率: $100/天
- 计算日期: 2026-02-25

**计算过程**:

| 步骤 | 计算 | 结果 |
|------|------|------|
| 1. 起算日 | 取ATA | 2026-02-01 |
| 2. lastFreeDate | 02-01 + 7 - 1 | 2026-02-07 |
| 3. 截止日 | min(实际提柜日, today) | 2026-02-20 |
| 4. chargeStart | lastFreeDate + 1 | 2026-02-08 |
| 5. chargeDays | 02-20 - 02-08 | 13天 |
| 6. 费用 | 13 × $100 | $1,300 |

**结果**: $1,300

### 7.3 滞箱费 (Detention) 计算

#### 定义

- **区间**: 实际提柜日 → 实际还箱日（无则当天）
- **起算日**: 实际提柜日 或 计划提柜日 (forecast模式)
- **截止日**: 实际还箱日 或 今天 (取大值)
- **免费日**: lastReturnDate = 提柜日起算 + freeDays - 1

#### 核心逻辑

```typescript

} else {

if (actual模式) {
  detentionEndDate = 实际还箱日 ?? 今天
} else {
  detentionEndDate = max(今天, 计划还箱日)
}
```

### 7.4 D&D (Demurrage & Detention) 合并计算

#### 定义

- **区间**: 预测模式用 ETA → 计划还箱日；实际模式用 ATA/卸船日 → 实际还箱日
- **起算日**: 同滞港费 (按到港/按卸船)
- **截止日**: 同滞箱费
- **特点**: 一个费用项覆盖到港→还箱整段时间

#### 计算逻辑

```typescript
if (forecast模式) {
  // 起算: ETA 或 修正ETA
  rangeStart = revisedEtaDestPort ?? etaDestPort
  // 截止: 同滞箱费 forecast
  rangeEnd = max(today, plannedReturnDate)
} else {
  // actual模式
  // 起算: 按标准 "按到港"用ATA，"按卸船"用卸船日
  rangeStart = calculationBasis.includes('卸船') 
    ? dischargeDate 
    : (ataDestPort ?? dischargeDate)
  // 截止: 实际还箱日或今天
  rangeEnd = returnTime ?? today
}
```

  hasYard &&

### 8.2 状态优先级计算

状态机按 **优先级顺序** 计算，取最高优先级的状态作为当前状态。

#### 优先级规则

| 优先级 | 条件 | 状态 | 港口类型 |
|--------|------|------|----------|
| **1** | 还空箱日期 (returnTime) | `returned_empty` | - |
| **2** | WMS已确认卸柜 | `unloaded` | - |
| **3** | 拖车提柜日期 (pickupDate) | `picked_up` | - |
| **4** | 目的港 ATA | `at_port` | destination |
| **4a** | 目的港可提货 (availableTime) | `at_port` | destination |
| **5** | 中转港到港 (ata/gate_in_time) | `at_port` | transit |
| **6** | 海运出运日期 (shipmentDate) | `in_transit` | - |
| **7** | 默认 | `not_shipped` | - |

#### 核心代码逻辑

```typescript
function calculateLogisticsStatus(container, portOperations, seaFreight, truckingTransport, warehouseOperation, emptyReturn): LogisticsStatusResult {
  // 优先级1: 还空箱
  if (emptyReturn?.returnTime) {
    return { status: 'returned_empty', reason: '已还空箱' };
  }
  
  // 优先级2: WMS已确认
  if (isWmsConfirmed(warehouseOperation)) {
    return { status: 'unloaded', reason: '仓库已卸柜（WMS确认）' };
  }
  
  // 优先级3: 拖车提柜
  if (truckingTransport?.pickupDate) {
    return { status: 'picked_up', reason: '已提柜' };
  }
  
  // 优先级4: 目的港ATA
  const destWithArrival = destPorts.find(po => po.ata);
  if (destWithArrival) {
    return { status: 'at_port', currentPortType: 'destination', reason: '目的港已到港（ATA）' };
  }
  
  // 优先级4a: 目的港可提货
  const destWithAvailable = destPorts.find(po => po.availableTime);
  if (destWithAvailable) {
    return { status: 'at_port', currentPortType: 'destination', reason: '目的港可提货' };
  }
  
  // 优先级5: 中转港到港
  const transitWithArrival = transitPorts.find(po => po.ata || po.gateInTime || po.transitArrivalDate);
  if (transitWithArrival) {
    return { status: 'at_port', currentPortType: 'transit', reason: '中转港已到港' };
  }
  
  // 优先级6: 在途
  if (seaFreight?.shipmentDate) {
    return { status: 'in_transit', reason: '已出运（在途）' };
  }
  
  // 优先级7: 未出运
  return { status: 'not_shipped', reason: '未出运' };
}
```

### 9.2 五个计划日期概述

### 9.3 锚点确定逻辑

#### 锚点选择规则

```
卸柜日 (plannedUnloadDate) 是核心锚点，通过仓库可用性确定：

Step 1: 从计划提柜日(plannedPickupDate)开始
Step 2: 检查仓库当日产能 (dailyUnloadCapacity)
Step 3: 如果产能已满，向后查找下一个可用日
Step 4: 返回最早可用日作为卸柜日
```

#### 核心代码逻辑

```typescript
async findEarliestAvailableWarehouse(warehouses: Warehouse[], pickupDate: Date) {
  for (const warehouse of warehouses) {
    // 从提柜日开始，依次检查每一天
    let checkDate = new Date(pickupDate);
    for (let i = 0; i < 30; i++) { // 最多查30天
      const isAvailable = await checkWarehouseCapacity(warehouse, checkDate);
      if (isAvailable) {
        return { warehouse, plannedUnloadDate: checkDate };
      }
      checkDate.setDate(checkDate.getDate() + 1); // 向后找
    }
  }
  return null; // 30天内无产能
}
```

### 9.4 各日期计算公式

#### 9.4.1 计划清关日 (plannedCustomsDate)

**计算公式**:
```
plannedCustomsDate = ATA/ETA - 1天 - ETA缓冲天数
```

**参数说明**:

| 参数 | 来源 | 说明 |
|------|------|------|
| ATA | port_operations.ata_dest_port | 实际到港日期 |
| ETA | port_operations.eta_dest_port | 预计到港日期 |
| ETA缓冲天数 | 请求参数 etaBufferDays | 前端传入的顺延天数 |

**计算逻辑**:
```typescript
// 1. 优先使用 ATA，其次使用 ETA
const clearanceDate = ata ?? eta;

// 2. 减去1天（清关需在到港前完成）
plannedCustomsDate = clearanceDate - 1;

// 3. 加上ETA缓冲天数（可选）
if (etaBufferDays > 0) {
  plannedCustomsDate += etaBufferDays;
}
```

---

#### 9.4.2 计划提柜日 (plannedPickupDate)

**计算公式**:
```
plannedPickupDate = plannedCustomsDate + 1天，且 ≤ lastFreeDate
```

**约束条件**:
- 必须 ≥ 今天（如果计算结果 ≤ 今天，调整为明天）
- 必须 ≤ lastFreeDate（最晚提柜日）
- 跳过周末（如果配置 skip_weekends = true）

**计算逻辑**:
```typescript
function calculatePlannedPickupDate(customsDate: Date, lastFreeDate?: Date): Date {
  // 1. 清关后次日提柜
  let pickupDate = customsDate + 1;

  // 2. 受 lastFreeDate 约束
  if (lastFreeDate && pickupDate > lastFreeDate) {
    pickupDate = lastFreeDate; // 不能超过免费日
  }

  // 3. 必须 ≥ 今天
  if (pickupDate <= today) {
    pickupDate = today + 1;
    // 同步调整清关日
    customsDate = pickupDate - 1;
  }

  // 4. 跳过周末（可选）
  if (skipWeekends) {
    pickupDate = skipWeekends(pickupDate);
  }

  return pickupDate;
}
```

---

#### 9.4.3 计划送仓日 (plannedDeliveryDate)

**计算公式**:
```
卸柜模式 = 'Live load' 时:  plannedDeliveryDate = plannedPickupDate
卸柜模式 = 'Drop off' 时:  plannedDeliveryDate = plannedUnloadDate
```

**模式说明**:

| 模式 | 描述 | 提→送关系 | 送→卸关系 |
|------|------|-----------|-----------|
| **Live load** | 现场装卸 | 提 = 送 = 卸 (同日) | - |
| **Drop off** | 堆场中转 | 提 < 送 = 卸 | - |

**计算逻辑**:
```typescript
function calculatePlannedDeliveryDate(
  pickupDate: Date,
  unloadMode: string,
  unloadDate: Date
): Date {
  if (unloadMode === 'Live load') {
    // 现场装卸：提=送=卸（同日）
    return new Date(pickupDate);
  }
  // Drop off：送=卸（送仓日即卸柜日）
  return new Date(unloadDate);
}
```

---

#### 9.4.4 计划卸柜日 (plannedUnloadDate)

**说明**: 这是核心锚点，基于仓库可用性动态计算。

**计算逻辑**:
```
输入: 候选仓库列表, 计划提柜日
输出: 最早可用仓库 + 对应卸柜日

步骤:
1. 遍历候选仓库
2. 从计划提柜日开始，检查每日产能
3. 找到第一个产能未满的日期
4. 返回仓库和卸柜日
```

**产能检查**:
```typescript
async checkWarehouseCapacity(warehouse: Warehouse, date: Date): Promise<boolean> {
  const occupancy = await getOccupancy(warehouse.code, date);
  if (!occupancy) return true; // 无记录，默认有产能
  return occupancy.plannedCount < occupancy.capacity;
}
```

---

#### 9.4.5 计划还箱日 (plannedReturnDate)

**计算公式**:
```
卸柜模式 = 'Drop off' 时:  plannedReturnDate = 卸柜日 + 1天 (受还箱能力约束)
卸柜模式 = 'Live load' 时:  plannedReturnDate = 卸柜日 (同日)
```

**还箱能力约束**:
- 检查车队当日还箱槽位 (trucking return slots)
- 如果产能已满，向后查找下一个可用日
- 最晚不超过 lastReturnDate（最晚还箱日红线）

**fallback规则**:
```typescript
// 1. 优先从 EmptyReturn 表获取 lastReturnDate
if (emptyReturn?.lastReturnDate) {
  lastReturnDate = emptyReturn.lastReturnDate;
}
// 2. fallback: lastFreeDate + 7天
else if (destPo.lastFreeDate) {
  lastReturnDate = destPo.lastFreeDate + 7;
}
```

**计算逻辑**:
```typescript
async function calculatePlannedReturnDate(
  unloadDate: Date,
  unloadMode: string,
  truckingCompanyId: string,
  lastReturnDate?: Date,
  plannedPickupDate?: Date
): Promise<{ returnDate: Date; adjustedUnloadDate?: Date }> {
  if (unloadMode === 'Live load') {
    // Live load: 还 = 卸（同日）
    const availableDate = await findEarliestReturnSlot(
      truckingCompanyId,
      unloadDate,
      lastReturnDate
    );
    // 如果需调整卸柜日
    if (availableDate !== unloadDate) {
      return { returnDate: availableDate, adjustedUnloadDate: availableDate };
    }
    return { returnDate: unloadDate };
  }

  // Drop off: 还 = 卸 + 1天
  let returnDate = unloadDate + 1;
  
  // 受还箱能力约束
  const availableDate = await findEarliestReturnSlot(
    truckingCompanyId,
    returnDate,
    lastReturnDate
  );
  
  // 不能超过 lastReturnDate 红线
  if (lastReturnDate && availableDate > lastReturnDate) {
    returnDate = lastReturnDate;
  } else {
    returnDate = availableDate;
  }

  return { returnDate };
}
```

| 4 | 调整为 tomorrow | 2026-02-21 |
| 5 | 同步调整清关日 | 2026-02-20 (02-21 - 1) |

**结果**:
```
┌────────────────────────────────────────────────────────────┐
│       过期 ETA                                    today   │
│         ↓                                           ↓     │
│  01-31  →  [02-01] ──→ ──→  02-20  →  02-21  →  02-22    │
│ 清关       提柜(过期)              清关(调整)  提柜(新)   │
│                                    ↑                       │
│                             自动调整为 >= today           │
└────────────────────────────────────────────────────────────┘
```

### 9.6 五个计划日期对比汇总表

| 日期字段 | 中文名 | 计算基准 | 约束条件 | 模式影响 |
|----------|--------|----------|----------|----------|
| plannedCustomsDate | 计划清关日 | ATA/ETA - 1天 | ETA缓冲天数 | 无 |
| plannedPickupDate | 计划提柜日 | customs + 1天 | ≤ lastFreeDate, ≥ today | 无 |
| plannedDeliveryDate | 计划送仓日 | 提柜日/卸柜日 | 模式决定 | Live load=提, Drop off=卸 |
| plannedUnloadDate | 计划卸柜日 | 仓库产能锚点 | 日产能限制 | 无 |
| plannedReturnDate | 计划还箱日 | 卸柜日 + 1天 | ≤ lastReturnDate, 还箱能力 | Live load=卸, Drop off=卸+1 |

### 9.7 日期链完整性约束

#### 约束规则

```
1. 清关日 < 提柜日
2. 提柜日 ≤ 送仓日 ≤ 卸柜日
3. 卸柜日 ≤ 还箱日
4. 提柜日 ≤ lastFreeDate (最晚提柜日)
5. 还箱日 ≤ lastReturnDate (最晚还箱日)
```

#### 约束验证代码

```typescript
function validateDateChain(dates: {
  plannedCustomsDate: Date;
  plannedPickupDate: Date;
  plannedDeliveryDate: Date;
  plannedUnloadDate: Date;
  plannedReturnDate: Date;
}): boolean {
  return (
    dates.plannedCustomsDate < dates.plannedPickupDate &&
    dates.plannedPickupDate <= dates.plannedDeliveryDate &&
    dates.plannedDeliveryDate <= dates.plannedUnloadDate &&
    dates.plannedUnloadDate <= dates.plannedReturnDate
  );
}
```

## 十、历时、超期、倒计时计算逻辑

### 10.1 三个概念概述

物流时间线中需要区分三个核心概念：

| 概念 | 英文 | 含义 | 方向 | 颜色 | 用途 |
|------|------|------|------|------|------|
| **倒计时** | Countdown | 距离未来节点还有多久 | 向前看 | 橙色(≤3天) / 绿色(>3天) | 计划预警 |
| **历时** | Elapsed | 历史节点持续了多久 | 向后看 | 蓝色 | 流程分析 |
| **超期** | Overdue | 已超过标准耗时 | 风险看 | 红色脉冲 | 干预提醒 |

### 10.2 核心判断规则

### 10.3 各概念计算公式

#### 10.3.1 倒计时 (Countdown)

**定义**：未来日期距离当前时间还有多久

**计算公式**：
```
倒计时 = 目标日期 - 当前时间
```

**代码实现**：
```typescript
// DurationDisplay.vue:153-163
const getCountdownText = (): string => {
  const now = new Date()
  const diffDays = Math.ceil((date.getTime() - now.getTime()) / (1000 * 60 * 60 * 24))

  if (diffDays === 0) return '今天到期'
  if (diffDays === 1) return '倒计时1天'
  return `倒计时${diffDays}天`
}
```

**颜色规则**：
```typescript
// KeyDatesTimeline.vue:243-248
if (date > now) {
  // 未来日期：倒计时
  const diffDays = Math.ceil((date.getTime() - now.getTime()) / (1000 * 60 * 60 * 24))
  if (diffDays <= 3) return 'orange'   // ≤3天：橙色预警
  return 'green'                        // >3天：绿色安全
}
```

---

#### 10.3.2 历时 (Elapsed)

**定义**：历史节点从开始到结束持续了多久

**计算公式**：
```
有后一节点：历时 = 后一节点日期 - 当前节点日期
无后一节点：历时 = 当前节点日期 - 上一节点日期
```

**代码实现**：
```typescript
// DurationDisplay.vue:166-184
const getElapsedText = (): string => {
  // 优先使用后一节点计算
  if (nextDate) {
    const diffDays = Math.floor((nextDate.getTime() - date.getTime()) / (1000 * 60 * 60 * 24))
    if (diffDays <= 0) return ''
    if (diffDays === 1) return '历时1天'
    return `历时${diffDays}天`
  }

  // fallback：使用上一节点
  if (prevDate) {
    const diffDays = Math.floor((date.getTime() - prevDate.getTime()) / (1000 * 60 * 60 * 24))
    if (diffDays <= 0) return ''
    if (diffDays === 1) return '历时1天'
    return `历时${diffDays}天`
  }
  return ''
}
```

**显示规则**：
- 有后一节点（业务闭环）：显示「历时X天」（蓝色）
- 无后一节点 + 过去日期 + 非关键节点：显示「历时X天」（蓝色）

---

#### 10.3.3 超期 (Overdue)

**定义**：关键节点已超过标准耗时，需要人工干预

**计算公式**：
```
超期 = 当前时间 - 目标节点日期 - 标准耗时
```

**代码实现**：
```typescript
// DurationDisplay.vue:202-208
const overdueDays = computed(() => {
  const time = overdueTime.value  // 当前时间 - 目标节点时间
  if (!time) return 0
  const diffDays = Math.floor(time / (1000 * 60 * 60 * 24))
  return diffDays
})

// 超期文本格式化
function formatOverdueCore(days: number): string {
  if (days === 0) return ''
  const base = days === 1 ? '超期1天' : `超期${days}天`
  if (props.nextMilestoneLabel) {
    return `已${base}→${props.nextMilestoneLabel}`  // 如：已超期2天→还空箱
  }
  return base
}
```

**关键节点超期判断**：
```typescript
// KeyDatesTimeline.vue:253-261
if (label && (label === '最晚提柜' || label === '最晚还箱')) {
  const standardHours = STANDARD_DURATIONS[label] ?? 0
  if (standardHours > 0) {
    const standardDays = standardHours / 24
    // 已超期：红色
    if (diffDays > standardDays) return 'red'
  }
}
```

### 10.5 三概念对比汇总表

| 维度 | 倒计时 | 历时 | 超期 |
|------|--------|------|------|
| **时间方向** | 未来 → 当前 | 过去 → 过去 | 当前 ← 过去 |

| **动画** | 无 | 无 | 脉冲动画 |
| **用途** | 计划预警 | 流程分析 | 干预提醒 |
| **文案格式** | 倒计时X天 | 历时X天 | 超期X天→下一节点 |
