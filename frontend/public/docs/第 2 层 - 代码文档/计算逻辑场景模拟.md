## 二、滞港费计算模块

### 2.1 核心计算逻辑

**计算公式**:
```
费用 = chargeDays × ratePerDay
```

**关键参数**:
| 参数 | 说明 | 来源 |
|------|------|------|
| startDate | 起始日期 | ATA 或 ETA |
| endDate | 截止日期 | min(today, plannedPickupDate) |
| freeDays | 免费天数 | 滞港费标准表 |
| freeDaysBasis | 免费日基准 | 自然日/工作日 |
| ratePerDay | 日费率 | 滞港费标准表 |

### 2.2 场景模拟

#### 场景 1: 免费期内

**参数**:
- ATA: 2026-02-01
- 计划提柜日: 2026-03-10
- 免费天数: 7天
- 日费率: $100/天
- 计算日期: 2026-03-05

**计算过程**:
```
lastFreeDate = 2026-02-01 + 7 - 1 = 2026-02-07
endDate = min(2026-03-05, 2026-03-10) = 2026-03-05

因为 endDate (03-05) > lastFreeDate (02-07)
所以 chargeDays = 03-05 - 02-07 = 27天
费用 = 27 × $100 = $2,700
```

**结果**: 超期27天，费用$2,700

#### 场景 2: 已超期

**参数**:
- ATA: 2026-02-01
- 实际提柜日: 2026-02-20
- 免费天数: 7天
- 日费率: $100/天

**计算过程**:
```
lastFreeDate = 2026-02-01 + 7 - 1 = 2026-02-07
endDate = 2026-02-20 (实际提柜日)
chargeDays = 02-20 - 02-07 = 13天
费用 = 13 × $100 = $1,300
```

**结果**: 超期13天，费用$1,300

#### 场景 3: 分段费率

**参数**:
- ATA: 2026-02-01
- 实际提柜日: 2026-02-20
- 免费天数: 7天
- 分段费率:
  - 1-7天: $100/天
  - 8-14天: $150/天

**计算过程**:
```
lastFreeDate = 2026-02-07
chargeDays = 13天 (02-08 到 02-20)

分段计算:
├── 第1段: 02-08 到 02-14 (7天), $100/天 → 7 × $100 = $700
├── 第2段: 02-15 到 02-20 (6天), $150/天 → 6 × $150 = $900
└── 总费用: $700 + $900 = $1,600
```

**结果**: 费用 = $1,600

### 2.3 免费日计算基准

| freeDaysBasis | 说明 |
|---------------|------|
| 自然日 | 包含所有日历日 |
| 工作日 | 只计算周一到周五 |
| 工作+自然日 | 免费期排除周末 |
| 自然+工作日 | 计费期排除周末 |

#### 场景 4: 自然日 vs 工作日对比

**参数**:
- ATA: 2026-02-01 (周六)
- 免费天数: 7天
- 计算日期: 2026-02-15

| 基准类型 | lastFreeDate | chargeDays | 费用($50/天) |
|----------|--------------|------------|--------------|
| 自然日 | 02-07 | 8天 | $400 |
| 工作日 | 02-11 | 3天 | $150 |

### 2.4 计算模式对比

| 场景 | 计算模式 | 起始日 | 截止日 | 费用 |
|------|----------|--------|--------|------|
| 实际到港，未提柜 | actual | ATA | min(today, plannedPickup) | 有 |
| 实际到港，已提柜 | actual | ATA | min(实际提柜日, plannedPickup) | 有 |
| 预测模式，未到港 | forecast | ETA | min(计划提柜日, today) | 可能为0 |
| 预测模式，有ATA | forecast | ATA | min(计划提柜日, today) | 有 |

## 三、智能排柜模块

### 3.1 核心计算逻辑

**5个关键日期关系**:
```
plannedCustomsDate   ← 计划清关日
        ↓
plannedPickupDate    ← 计划提柜日
        ↓
plannedUnloadDate    ← 计划卸柜日 (核心锚点)
        ↓
plannedDeliveryDate  ← 计划送仓日
        ↓
plannedReturnDate    ← 计划还箱日
```

### 3.2 场景模拟

#### 场景 1: 仓库有足够产能

**参数**:
- 仓库: UK-S005 (日产能: 5柜)
- 提柜日候选: 2026-03-28
- 运输天数: 1天
- 在仓天数: 3天

**结果**:
| 日期 | 值 |
|------|-----|
| plannedCustomsDate | 2026-03-27 |
| plannedPickupDate | 2026-03-28 |
| plannedUnloadDate | 2026-03-29 |
| plannedDeliveryDate | 2026-03-29 |
| plannedReturnDate | 2026-04-01 |

#### 场景 2: 仓库产能不足，需要顺延

**参数**:
- 仓库: UK-S005 (日产能: 5柜)
- 已有排产: 2026-03-29 已排 5柜 (满)
- 提柜日候选: 2026-03-28

**结果**:
| 日期 | 值 |
|------|-----|
| plannedCustomsDate | 2026-03-27 |
| plannedPickupDate | 2026-03-28 |
| plannedUnloadDate | 2026-03-30 (顺延1天) |
| plannedDeliveryDate | 2026-03-29 |
| plannedReturnDate | 2026-04-02 |

### 3.3 排产模式对比

| 模式 | dryRun | 数据保存 | 用途 |
|------|--------|----------|------|
| 预览排产 | true | 不保存 | 预览方案 |
| 单柜优化 | false | 可选保存 | 成本优化 |
| 批量排产 | false | 保存到DB | 正式排产 |

```
TotalCost = DemurrageCost + DetentionCost + StorageCost + TransportationCost
```

### 4.2 场景模拟

## 五、各类费用计算详解

### 5.1 费用类型识别

| 费用类型 | 识别关键词 |
|----------|------------|
| D&D | demurrage + detention |
| 滞箱费 | detention / 滞箱 |
| 港口存储 | storage / 堆存 |
| 滞港费 | demurrage / 滞港 |

### 5.2 费用类型对比

| 费用类型 | 区间起点 | 区间终点 | 免费日基准 |
|----------|----------|----------|------------|
| 滞港费 | ATA/卸船 | 实际提柜日 | ATA/卸船+freeDays |
| 滞箱费 | 实际/计划提柜日 | 实际/计划还箱日 | 提柜日+freeDays |
| D&D | ETA/ATA | 计划/实际还箱日 | 到港日+freeDays |
| 港口存储 | ETA/ATA/卸船 | 实际提柜日 | 到港日+freeDays |
| 车队堆场 | 提柜日 | 送仓日 | 无免费日 |

## 七、五个计划日期计算

### 7.1 日期定义

| 序号 | 日期字段 | 中文名 | 计算基准 |
|------|----------|--------|----------|
| 1 | plannedCustomsDate | 计划清关日 | ATA/ETA - 1天 |
| 2 | plannedPickupDate | 计划提柜日 | customs + 1天 |
| 3 | plannedDeliveryDate | 计划送仓日 | 提柜日/卸柜日 |
| 4 | plannedUnloadDate | 计划卸柜日 | 仓库产能锚点 |
| 5 | plannedReturnDate | 计划还箱日 | 卸柜日 + 1天 |

### 7.2 日期链约束

```
1. 清关日 < 提柜日
2. 提柜日 ≤ 送仓日 ≤ 卸柜日
3. 卸柜日 ≤ 还箱日
4. 提柜日 ≤ lastFreeDate
5. 还箱日 ≤ lastReturnDate
```

## 八、历时、超期、倒计时计算

### 8.1 概念对比

| 概念 | 方向 | 颜色 | 用途 |
|------|------|------|------|
| 倒计时 | 未来 → 当前 | 橙/绿 | 计划预警 |
| 历时 | 过去 → 过去 | 蓝 | 流程分析 |
| 超期 | 当前 ← 过去 | 红 | 干预提醒 |

### 8.2 计算公式

**倒计时**:
```
倒计时 = 目标日期 - 当前时间
颜色: ≤3天=橙色, >3天=绿色
```

**历时**:
```
有后一节点: 历时 = 后一节点日期 - 当前节点日期
无后一节点: 历时 = 当前节点日期 - 上一节点日期
颜色: 蓝色
```

**超期**:
```
超期 = 当前时间 - 目标节点日期 - 标准耗时
标准: 最晚提柜/最晚还箱 = 7天
颜色: 红色 + 脉冲动画
```

## 九、免费日计算公式汇总

```
最晚提柜日:
lastFreeDate = 起算日 + freeDays - 1
起算日 = ATA (按到港) 或 dischargeDate (按卸船)

最晚还箱日:
lastReturnDate = 提柜日起算 + freeDays - 1
提柜日起算 = 计划提柜日 (forecast) 或 实际提柜日 (actual)

收费天数:
chargeDays = endDate - lastFreeDate - 1

费用:
totalAmount = chargeDays × ratePerDay
```
