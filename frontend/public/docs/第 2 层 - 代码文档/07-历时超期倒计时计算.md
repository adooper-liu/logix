## 1. 三个概念概述

物流时间线中需要区分三个核心概念：

| 概念 | 英文 | 含义 | 方向 | 颜色 | 用途 |
|------|------|------|------|------|------|
| **倒计时** | Countdown | 距离未来节点还有多久 | 向前看 | 橙色(≤3天) / 绿色(>3天) | 计划预警 |
| **历时** | Elapsed | 历史节点持续了多久 | 向后看 | 蓝色 | 流程分析 |
| **超期** | Overdue | 已超过标准耗时 | 风险看 | 红色脉冲 | 干预提醒 |

## 3. 各概念计算公式

### 3.1 倒计时 (Countdown)

**定义**: 未来日期距离当前时间还有多久

**计算公式**:
```
倒计时 = 目标日期 - 当前时间
```

**代码实现**:
```typescript
const getCountdownText = (): string => {
  const now = new Date()
  const diffDays = Math.ceil((date.getTime() - now.getTime()) / (1000 * 60 * 60 * 24))

  if (diffDays === 0) return '今天到期'
  if (diffDays === 1) return '倒计时1天'
  return `倒计时${diffDays}天`
}
```

**颜色规则**:
```typescript
if (date > now) {
  // 未来日期：倒计时
  const diffDays = Math.ceil((date.getTime() - now.getTime()) / (1000 * 60 * 60 * 24))
  if (diffDays <= 3) return 'orange'   // ≤3天：橙色预警
  return 'green'                        // >3天：绿色安全
}
```

---

### 3.2 历时 (Elapsed)

**定义**: 历史节点从开始到结束持续了多久

**计算公式**:
```
有后一节点：历时 = 后一节点日期 - 当前节点日期
无后一节点：历时 = 当前节点日期 - 上一节点日期
```

**代码实现**:
```typescript
const getElapsedText = (): string => {
  // 优先使用后一节点计算
  if (nextDate) {
    const diffDays = Math.floor((nextDate.getTime() - date.getTime()) / (1000 * 60 * 60 * 24))
    if (diffDays <= 0) return ''
    if (diffDays === 1) return '历时1天'
    return `历时${diffDays}天`
  }

  // fallback：使用上一节点
  if (prevDate) {
    const diffDays = Math.floor((date.getTime() - prevDate.getTime()) / (1000 * 60 * 60 * 24))
    if (diffDays <= 0) return ''
    if (diffDays === 1) return '历时1天'
    return `历时${diffDays}天`
  }
  return ''
}
```

---

### 3.3 超期 (Overdue)

**定义**: 关键节点已超过标准耗时，需要人工干预

**计算公式**:
```
超期 = 当前时间 - 目标节点日期 - 标准耗时
```

**代码实现**:
```typescript
const overdueDays = computed(() => {
  const time = overdueTime.value  // 当前时间 - 目标节点时间
  if (!time) return 0
  const diffDays = Math.floor(time / (1000 * 60 * 60 * 24))
  return diffDays
})

// 超期文本格式化
function formatOverdueCore(days: number): string {
  if (days === 0) return ''
  const base = days === 1 ? '超期1天' : `超期${days}天`
  if (props.nextMilestoneLabel) {
    return `已${base}→${props.nextMilestoneLabel}`  // 如：已超期2天→还空箱
  }
  return base
}
```

## 4. 场景模拟

### 场景 1: 正常出运流程（显示历时）

**数据条件**:
- 预计到港 (ETA): 2026-03-01
- 实际到港 (ATA): 2026-03-02
- 实际卸船: 2026-03-02

**判断逻辑**:
- ETA → ATA：有后一节点 → 显示「历时1天」(蓝色)
- ATA → 卸船：有后一节点 → 显示「历时0天」(蓝色)
- 卸船 → 最晚提柜：无后一节点 → 过去日期 → 非关键 → 显示「历时X天」(蓝色)

---

### 场景 2: 等待最晚提柜（显示倒计时）

**数据条件**:
- 当前日期: 2026-03-05
- 最晚提柜日: 2026-03-08
- 实际提柜日: 未发生

**判断逻辑**:
- 最晚提柜 > 当前时间 → 未来日期
- diffDays = 3 ≤ 3 → 橙色预警

---

### 场景 3: 最晚提柜已超期（显示超期）

**数据条件**:
- 当前日期: 2026-03-15
- 最晚提柜日: 2026-03-08
- 标准耗时: 7天

**判断逻辑**:
- 最晚提柜 < 当前时间 → 过去日期
- diffDays = 7 > 标准7天 → 超期
- 颜色 = 红色 + 脉冲动画

---

### 场景 4: 最晚还箱未还箱（显示倒计时）

**数据条件**:
- 当前日期: 2026-03-10
- 最晚还箱日: 2026-03-12
- 实际还箱: 未发生

**判断逻辑**:
- 最晚还箱 > 当前时间 → 未来日期
- diffDays = 2 ≤ 3 → 橙色预警

---

### 场景 5: 最晚还箱已超期（显示超期）

**数据条件**:
- 当前日期: 2026-03-20
- 最晚还箱日: 2026-03-12
- 标准耗时: 7天

**判断逻辑**:
- 最晚还箱 < 当前时间 → 过去日期
- diffDays = 8 > 标准7天 → 超期
- 实际还箱 > 最晚还箱 → 有后一节点 → 显示「历时3天」(蓝色)

## 5. 三概念对比汇总表

| 维度 | 倒计时 | 历时 | 超期 |
|------|--------|------|------|
| **时间方向** | 未来 → 当前 | 过去 → 过去 | 当前 ← 过去 |
| **计算基准** | 目标日期 - 当前时间 | 后一节点 - 当前节点 | 当前时间 - 目标日期 - 标准 |
| **显示条件** | 未来日期 | 有后一节点 | 关键节点 + 超过标准 |
| **颜色** | 橙色(≤3天) / 绿色(>3天) | 蓝色 | 红色 |
| **动画** | 无 | 无 | 脉冲动画 |
| **用途** | 计划预警 | 流程分析 | 干预提醒 |
| **文案格式** | 倒计时X天 | 历时X天 | 超期X天→下一节点 |

## 6. 颜色编码系统

| 颜色 | 状态 | 场景 |
|------|------|------|
| 🔴 红色 | 超期风险 | 最晚提柜/最晚还箱超过7天 |
| 🟠 橙色 | 即将到期 | 倒计时 ≤ 3天 |
| 🟢 绿色 | 安全/未来 | 倒计时 > 3天 |
| 🔵 蓝色 | 中性/历时 | 历史节点或有后一节点 |
