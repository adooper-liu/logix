# 滞港费计算逻辑

**项目**: LogiX 物流管理系统  
**相关文档**: [返回目录](./README.md)

---

## 1. 核心计算逻辑

### 计算公式

```
费用 = chargeDays × ratePerDay
```

其中：
- `chargeDays` = endDate - lastFreeDate - 1 (收费天数)
- `lastFreeDate` = startDate + freeDays - 1 (最后免费日)

### 关键参数

| 参数 | 说明 | 来源 |
|------|------|------|
| startDate | 起始日期 | ATA (实际到港) 或 ETA (预计到港) |
| endDate | 截止日期 | min(today, plannedPickupDate) |
| freeDays | 免费天数 | 滞港费标准表 |
| freeDaysBasis | 免费日基准 | '自然日' 或 '工作日' |
| ratePerDay | 日费率 | 滞港费标准表 |

## 2. 场景模拟

### 场景 1: 免费期内 (未超期)

**参数**:
- ATA (实际到港): 2026-02-01
- 计划提柜日: 2026-03-10
- 免费天数: 7 天
- 日费率: $100/天
- 计算日期: 2026-03-05

**计算过程**:
```
lastFreeDate = 2026-02-01 + 7 - 1 = 2026-02-07
endDate = min(2026-03-05, 2026-03-10) = 2026-03-05

因为 endDate (03-05) <= lastFreeDate (02-07)
所以 chargeDays = 0
费用 = 0 × $100 = $0
```

**结果**: ✅ 免费期内，无需缴费

---

### 场景 2: 已超期 3 天

**参数**:
- ATA: 2026-02-01
- 计划提柜日: 2026-02-05 (已过)
- 免费天数: 7 天
- 日费率: $100/天
- 计算日期: 2026-02-10

**计算过程**:
```
lastFreeDate = 2026-02-01 + 7 - 1 = 2026-02-07
endDate = min(2026-02-10, 2026-02-05) = 2026-02-05 (取小值)

chargeStart = lastFreeDate + 1 = 2026-02-08
chargeDays = 2026-02-05 - 2026-02-08 + 1 = -2 (负数，取0)

实际: chargeDays = 2月8日到2月5日 = -2天 → 按0处理
费用 = 0 × $100 = $0
```

**修正计算**:
```
endDate = min(2026-02-10, 2026-02-05) = 2026-02-05
chargeStart = 2026-02-08
chargeDays = 0 (因为 02-05 <= 02-07)

实际应该是: chargeDays = 2026-02-10 - 2026-02-08 + 1 = 3 天
```

**结果**: 需修正计费逻辑 (已修复: paramsOverride)

---

### 场景 3: 超期 5 天，有分段费率

**参数**:
- ATA: 2026-02-01
- 计划提柜日: 2026-02-20 (已过期)
- 免费天数: 7 天
- 日费率: $100/天
- 分段费率:
  - 1-7天: $100/天
  - 8-14天: $150/天
  - 15天以上: $200/天
- 计算日期: 2026-02-12

**计算过程**:
```
lastFreeDate = 2026-02-07
endDate = min(2026-02-12, 2026-02-20) = 2026-02-12
chargeStart = 2026-02-08
chargeDays = 2026-02-12 - 2026-02-08 + 1 = 5 天

分段计算:
├── 第1段: 1-5天 (第1-5天), $100/天 → 5 × $100 = $500
└── 总费用: $500
```

**结果**: ✅ 费用 = $500

## 3. 免费日计算基准 (freeDaysBasis)

### 免费日类型说明

| freeDaysBasis | 免费期计算 | 计费期计算 | 说明 |
|---------------|------------|------------|------|
| `自然日` | 7天 | 7天 | 包含所有日历日 |
| `工作日` | 7天 | 7天 | 只计算周一到周五 |
| `工作+自然日` | 7工作日 | 7天 | 免费期排除周末 |
| `自然+工作日` | 7天 | 7工作日 | 计费期排除周末 |

### 工作日计算规则

```typescript
// 工作日判断：周日(0)和周六(6)不计入
function isWeekend(d: Date): boolean {
  const dow = d.getUTCDay();
  return dow === 0 || dow === 6;
}

// 添加N个工作日
function addWorkingDays(start: Date, n: number): Date {
  if (n <= 0) return new Date(start.getTime());
  const result = new Date(start.getTime());
  let count = 0;
  while (count < n) {
    if (!isWeekend(result)) count++;
    if (count < n) result.setUTCDate(result.getUTCDate() + 1);
  }
  return result;
}
```

### 场景 4: 免费日基准对比 (自然日 vs 工作日)

**参数**:
- ATA (实际到港): 2026-02-01 (周六)
- 免费天数: 7天
- 计算日期: 2026-02-15

#### 场景 4A: freeDaysBasis = "自然日"

```
Step 1: 计算 lastFreeDate
lastFreeDate = ATA + 7 - 1 = 2026-02-01 + 6天 = 2026-02-07

Step 2: 判断是否超期
endDate = 2026-02-15
因为 2026-02-15 > 2026-02-07 (lastFreeDate)
所以已超期

Step 3: 计算收费天数
chargeStart = lastFreeDate + 1 = 2026-02-08
chargeDays = 2026-02-15 - 2026-02-08 + 1 = 8天

结果: chargeDays = 8天, 超期 1 天
```

#### 场景 4B: freeDaysBasis = "工作日"

```
Step 1: 计算 lastFreeDate (跳过周末)
起始: 2026-02-01 (周六)
  02-02 (周日) - 跳过
  02-03 (周一) - 第1个工作日
  02-04 (周二) - 第2个
  02-05 (周三) - 第3个
  02-06 (周四) - 第4个
  02-07 (周五) - 第5个
  02-08 (周六) - 跳过
  02-09 (周日) - 跳过
  02-10 (周一) - 第6个工作日
  02-11 (周二) - 第7个工作日 ← lastFreeDate

lastFreeDate = 2026-02-11 (周二)

Step 2: 判断是否超期
因为 2026-02-15 > 2026-02-11
所以已超期

Step 3: 计算收费天数 (按工作日)
chargeStart = 2026-02-12
  02-12 (周三) - 第1个工作日
  02-13 (周四) - 第2个
  02-14 (周五) - 第3个
  02-15 (周六) - 跳过
  → chargeDays = 3天

结果: chargeDays = 3天 (比自然日少5天周末)
```

**对比表**:

| 基准类型 | lastFreeDate | chargeDays | 费用($50/天) |
|----------|--------------|------------|--------------|
| 自然日 | 02-07 | 8天 | $400 |
| 工作日 | 02-11 | 3天 | $150 |

## 4. 计算方式对比 (按到港 vs 按卸船)

### 场景 5A: calculationBasis = "按到港"

```
起算日 = ATA = 2026-02-10
lastFreeDate = 2026-02-10 + 7 - 1 = 2026-02-16

若今日 = 2026-02-20:
chargeDays = 2026-02-20 - 2026-02-16 = 4天
费用 = 4 × $100 = $400
```

### 场景 5B: calculationBasis = "按卸船"

```
起算日 = 卸船日 = 2026-02-12
lastFreeDate = 2026-02-12 + 7 - 1 = 2026-02-18

若今日 = 2026-02-20:
chargeDays = 2026-02-20 - 2026-02-18 = 2天
费用 = 2 × $100 = $200
```

**对比表**:

| 计算方式 | 起算日 | lastFreeDate | chargeDays | 费用 |
|----------|--------|--------------|------------|------|
| 按到港 | 02-10 | 02-16 | 4天 | $400 |
| 按卸船 | 02-12 | 02-18 | 2天 | $200 |

## 5. 最晚提柜日 vs 最晚还箱日

### 最晚提柜日 (lastFreeDate)

- 起算日: ATA 或卸船日
- 公式: `lastFreeDate = 起算日 + freeDays - 1`
- 存储位置: `process_port_operations.last_free_date`

### 最晚还箱日 (lastReturnDate)

- 起算日: 计划提柜日 或 实际提柜日
- 公式: `lastReturnDate = 提柜日起算 + freeDays - 1`
- 存储位置: `process_port_operations.last_return_date`

#### 场景 6A: 预测模式 (forecast)

```
参数:
- ATA: 2026-02-01
- freeDays (提柜): 7天
- freeDays (还箱): 7天
- 计划提柜日: 2026-02-20
- 计划还箱日: 2026-02-27

计算:
lastFreeDate = 2026-02-01 + 7 - 1 = 2026-02-07
lastReturnDate = 2026-02-20 + 7 - 1 = 2026-02-26
```

#### 场景 6B: 实际模式 (actual)

```
参数:
- ATA: 2026-02-01
- 实际提柜日: 2026-02-10
- 实际还箱日: 2026-02-18

计算:
lastFreeDate = 2026-02-01 + 7 - 1 = 2026-02-07 (不变)
lastReturnDate = 2026-02-10 + 7 - 1 = 2026-02-16
```

**对比表**:

| 模式 | lastFreeDate | lastReturnDate | 依据 |
|------|--------------|----------------|------|
| forecast | 02-07 | 02-26 | 计划提柜日 |
| actual | 02-07 | 02-16 | 实际提柜日 |

## 6. 计算模式对比

| 场景 | 计算模式 | 起始日 | 截止日 | 费用 |
|------|----------|--------|--------|------|
| 实际到港，未提柜 | actual | ATA | min(today, plannedPickup) | 有 |
