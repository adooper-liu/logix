## 1. 状态机概述

LogiX 物流状态机采用 **7层简化状态** 模型，用于桑基图展示和费用计算判断。

### 简化状态枚举

| 状态值           | 中文   | 描述           |
| ---------------- | ------ | -------------- |
| `not_shipped`    | 未出运 | 尚未出运       |
| `shipped`        | 已出运 | 已装船/已出运  |
| `in_transit`     | 在途   | 航行中         |
| `at_port`        | 已到港 | 中转港或目的港 |
| `picked_up`      | 已提柜 | 已从码头提柜   |
| `unloaded`       | 已卸柜 | 已从货柜卸货   |
| `returned_empty` | 已还箱 | 已归还空箱     |

## 2. 状态优先级计算

状态机按 **优先级顺序** 计算，取最高优先级的状态作为当前状态。

### 优先级规则

| 优先级 | 条件                                                                            | 状态             | 港口类型    |
| ------ | ------------------------------------------------------------------------------- | ---------------- | ----------- |
| **1**  | 还空箱日期 (returnTime)                                                         | `returned_empty` | -           |
| **2**  | WMS已确认卸柜 (wmsStatus=WMS已完成 或 ebsStatus=已入库 或 wmsConfirmDate不为空) | `unloaded`       | -           |
| **3**  | 拖车提柜日期 (pickupDate)                                                       | `picked_up`      | -           |
| **4**  | 目的港 ATA                                                                      | `at_port`        | destination |
| **4a** | 目的港可提货 (availableTime)                                                    | `at_port`        | destination |
| **5**  | 中转港到港 (ata/gate_in_time/transit_arrival_date)                              | `at_port`        | transit     |
| **6**  | 海运出运日期 (shipmentDate)                                                     | `in_transit`     | -           |
| **7**  | 默认                                                                            | `not_shipped`    | -           |

### 核心代码逻辑

```typescript
/**
 * 基于货柜相关数据自动计算物流状态
 * Calculate logistics status automatically based on container data
 */
export interface LogisticsStatusResult {
  status: SimplifiedStatus
  currentPortType: 'origin' | 'transit' | 'destination' | null
  latestPortOperation: PortOperation | null
  /** 触发状态变更的字段及值 */
  triggerFields?: Record<string, unknown>
  /** 状态计算的原因说明 */
  reason?: string
}

export const calculateLogisticsStatus = (
  container: Container,
  portOperations: PortOperation[],
  seaFreight?: SeaFreight,
  truckingTransport?: TruckingTransport,
  warehouseOperation?: WarehouseOperation,
  emptyReturn?: EmptyReturn
): LogisticsStatusResult => {
  // 分类港口操作记录
  const transitPorts = portOperations.filter(po => po.portType === 'transit')
  const destPorts = portOperations.filter(po => po.portType === 'destination')

  let status = SimplifiedStatus.NOT_SHIPPED
  let currentPortType: 'origin' | 'transit' | 'destination' | null = null
  let latestPortOperation: PortOperation | null = null
  let triggerFields: Record<string, unknown> = {}
  let reason = ''

  // 优先级1: 还空箱日期（最高优先级）
  if (emptyReturn?.returnTime) {
    status = SimplifiedStatus.RETURNED_EMPTY
    triggerFields = { returnTime: emptyReturn.returnTime }
    reason = '已还空箱'
    return { status, currentPortType, latestPortOperation, triggerFields, reason }
  }

  // 优先级2: 仓库卸柜（WMS已确认）
  if (isWmsConfirmed(warehouseOperation)) {
    status = SimplifiedStatus.UNLOADED
    triggerFields = {
      wmsStatus: warehouseOperation?.wmsStatus,
      ebsStatus: warehouseOperation?.ebsStatus,
      wmsConfirmDate: warehouseOperation?.wmsConfirmDate,
    }
    reason = '仓库已卸柜（WMS确认）'
    return { status, currentPortType, latestPortOperation, triggerFields, reason }
  }

  // 优先级3: 拖车提柜日期
  if (truckingTransport?.pickupDate) {
    status = SimplifiedStatus.PICKED_UP
    triggerFields = { pickupDate: truckingTransport.pickupDate }
    reason = '已提柜'
    return { status, currentPortType, latestPortOperation, triggerFields, reason }
  }

  // 优先级4: 目的港有实际到港时间
  const destWithArrival = destPorts.find(po => po.ata)
  if (destWithArrival) {
    status = SimplifiedStatus.AT_PORT
    currentPortType = 'destination'
    latestPortOperation = destWithArrival
    triggerFields = { ata: destWithArrival.ata }
    reason = '目的港已到港（ATA）'
    return { status, currentPortType, latestPortOperation, triggerFields, reason }
  }

  // 优先级4a: 目的港有可提货时间（飞驼状态码 PCAB/AVLE/AVAIL 触发）
  const destWithAvailable = destPorts.find(po => po.availableTime)
  if (destWithAvailable) {
    status = SimplifiedStatus.AT_PORT
    currentPortType = 'destination'
    latestPortOperation = destWithAvailable
    triggerFields = { availableTime: destWithAvailable.availableTime }
    reason = '目的港可提货（PCAB/AVLE/AVAIL）'
    return { status, currentPortType, latestPortOperation, triggerFields, reason }
  }

  // 优先级5: 中转港有到港/进闸（ata、gate_in_time 或 transit_arrival_date）
  const transitWithArrival = transitPorts.find(
    po => po.ata || po.gateInTime || po.transitArrivalDate
  )
  if (transitWithArrival) {
    status = SimplifiedStatus.AT_PORT
    currentPortType = 'transit'
    latestPortOperation = transitWithArrival
    triggerFields = {
      ata: transitWithArrival.ata,
      gateInTime: transitWithArrival.gateInTime,
      transitArrivalDate: transitWithArrival.transitArrivalDate,
    }
    reason = '中转港已到港'
    return { status, currentPortType, latestPortOperation, triggerFields, reason }
  }

  // 优先级6: 有海运记录（已实际出运）→ 在途
  if (seaFreight?.shipmentDate) {
    status = SimplifiedStatus.IN_TRANSIT
    // 在途时仍需展示「预计到港/修正ETA」，取目的港港口操作（按 port_sequence 取最后一条目的港）
    const destForEta = destPorts.length
      ? [...destPorts].sort((a, b) => (b.portSequence ?? 0) - (a.portSequence ?? 0))[0]
      : null
    latestPortOperation = destForEta ?? null
    triggerFields = { shipmentDate: seaFreight.shipmentDate }
    reason = '已出运（在途）'
    return { status, currentPortType, latestPortOperation, triggerFields, reason }
  }

  // 优先级7: 默认状态（未出运）
  status = SimplifiedStatus.NOT_SHIPPED
  triggerFields = {}
  reason = '未出运'
  return { status, currentPortType, latestPortOperation, triggerFields, reason }
}

/**
 * 判断是否WMS已确认（已卸柜）
 * Check if WMS is confirmed (container unloaded)
 */
export const isWmsConfirmed = (warehouseOperation?: WarehouseOperation): boolean => {
  if (!warehouseOperation) return false

  // 满足任一条件即可
  return (
    warehouseOperation.wmsStatus === 'WMS已完成' ||
    warehouseOperation.ebsStatus === '已入库' ||
    warehouseOperation.wmsConfirmDate !== null
  )
}
```

## 5. 状态与费用计算关联

### 基础费用计算表

| 状态           | 滞港费    | 滞箱费    | 堆存费    | D&D 费用  | 说明           |
| -------------- | --------- | --------- | --------- | --------- | -------------- |
| not_shipped    | ❌ 不计算 | ❌ 不计算 | ❌ 不计算 | ❌ 不计算 | 尚未出运       |
| shipped        | ❌ 不计算 | ❌ 不计算 | ❌ 不计算 | ❌ 不计算 | 已出运但未到港 |
| in_transit     | ❌ 不计算 | ❌ 不计算 | ❌ 不计算 | ❌ 不计算 | 航行中         |
| at_port        | ✅ 计算   | ❌ 不计算 | ✅ 计算   | ❌ 不计算 | 已到港未提柜   |
| picked_up      | ✅ 计算   | ✅ 计算   | ❌ 不计算 | ✅ 计算   | 已提柜未卸柜   |
| unloaded       | ✅ 计算   | ✅ 计算   | ❌ 不计算 | ✅ 计算   | 已卸柜未还箱   |
| returned_empty | ❌ 不计算 | ✅ 计算   | ❌ 不计算 | ✅ 计算   | 已还箱         |

**说明**:

- **D&D 费用**: 指合并计算的滞港费 + 滞箱费（D&D Combined），一个费用项覆盖从到港到还箱的完整期间
- **计算逻辑**: 当存在合并 D&D 费用项时，不再单独计算滞港费和滞箱费，而是统一按 D&D 标准计算

### 5.1 物流状态对 D&D 费用计算的影响

D&D 费用（Demurrage & Detention）的计算模式直接依赖于物流状态机的判定结果。

#### 状态机判定逻辑

**核心函数**: `isArrivedAtDestinationPortForDemurrage()`  
**代码位置**: `backend/src/services/demurrage.service.ts:474-487`

```typescript
function isArrivedAtDestinationPortForDemurrage(ls: LogisticsStatusResult): boolean {
  const { status, currentPortType } = ls

  // 已提柜/已卸柜/已还箱 → 视为已过到港节点
  if (
    status === SimplifiedStatus.PICKED_UP ||
    status === SimplifiedStatus.UNLOADED ||
    status === SimplifiedStatus.RETURNED_EMPTY
  ) {
    return true
  }

  // 已到目的港且为 destination 类型
  if (status === SimplifiedStatus.AT_PORT && currentPortType === 'destination') {
    return true
  }

  return false
}
```

#### Actual vs Forecast 计算模式

| 物流状态                | 计算模式     | 说明                              |
| ----------------------- | ------------ | --------------------------------- |
| `at_port` (destination) | **actual**   | 已到目的港，按实际发生日期计算    |
| `picked_up`             | **actual**   | 已提柜，按实际提柜日计算          |
| `unloaded`              | **actual**   | 已卸柜，按实际卸柜日计算          |
| `returned_empty`        | **actual**   | 已还箱，费用计算终止              |
| `shipped`               | **forecast** | 已出运未到港，按计划/预测日期计算 |
| `in_transit`            | **forecast** | 在途中，按计划/预测日期计算       |
| `not_shipped`           | **forecast** | 未出运，按计划/预测日期计算       |

### 5.2 不同费用类型的起止日规则

#### 1. 滞港费（Demurrage）

| 计算模式     | 起算日         | 截止日                 | 说明                 |
| ------------ | -------------- | ---------------------- | -------------------- |
| **forecast** | 修正 ETA / ETA | max(今天，计划提柜日)  | 未到港，每日滚动更新 |
| **actual**   | ATA / 卸船日   | 实际提柜日（无则今天） | 已到港，提柜后封顶   |

#### 2. 滞箱费（Detention）

| 计算模式     | 起算日     | 截止日                 | 说明               |
| ------------ | ---------- | ---------------------- | ------------------ |
| **forecast** | 计划提柜日 | max(今天，计划还箱日)  | 未实际提柜，按计划 |
| **actual**   | 实际提柜日 | 实际还箱日（无则今天） | 已实际提柜，按实际 |

#### 3. 堆存费（Storage）

| 计算模式     | 起算日         | 截止日                 | 说明                 |
| ------------ | -------------- | ---------------------- | -------------------- |
| **forecast** | 修正 ETA / ETA | max(今天，计划提柜日)  | 未到港               |
| **actual**   | ATA / 卸船日   | 实际提柜日（无则今天） | 已有 ATA，按实际区间 |

#### 4. 合并费用（D&D Combined）

| 计算模式     | 起算日         | 截止日                 | 说明               |
| ------------ | -------------- | ---------------------- | ------------------ |
| **forecast** | 修正 ETA / ETA | max(今天，计划还箱日)  | 一个费用项覆盖整段 |
| **actual**   | ATA / 卸船日   | 实际还箱日（无则今天） | 不强制依赖实际提柜 |

### 5.3 状态快照在费用计算中的应用

**代码位置**: `backend/src/services/demurrage.service.ts:1430-1434`

```typescript
// 第一步：状态机判定是否到达目的港（或提柜/卸柜/还箱），再决定 actual vs forecast
const logisticsSnapshot = await this.getLogisticsStatusSnapshot(containerNumber)
const arrivedAtDestinationPort = logisticsSnapshot
  ? isArrivedAtDestinationPortForDemurrage(logisticsSnapshot)
  : false
const calculationMode: 'actual' | 'forecast' = arrivedAtDestinationPort ? 'actual' : 'forecast'
```

#### 关键要点

1. **状态优先原则**: 先通过状态机判断货柜当前位置，再决定使用哪种计算模式
2. **Actual 模式**: 已到目的港或之后环节 → 使用实际发生日期，费用金额固定
3. **Forecast 模式**: 未到目的港 → 使用计划/预测日期，每日滚动更新预计金额

## 6. 状态流转规则

### 合法状态流转

| 当前状态       | 可流转到的状态                 |
| -------------- | ------------------------------ |
| not_shipped    | shipped                        |
| shipped        | in_transit, at_port, picked_up |
| in_transit     | at_port, picked_up             |
| at_port        | picked_up, unloaded            |
| picked_up      | unloaded, returned_empty       |
| unloaded       | returned_empty                 |
| returned_empty | 无（最终状态）                 |

### 状态流转验证

```typescript
/**
 * 验证状态流转是否合法
 */
export const isValidSimplifiedTransition = (
  fromStatus: SimplifiedStatus,
  toStatus: SimplifiedStatus
): boolean => {
  const validTargets = SimplifiedStatusTransitions[fromStatus] || []
  return validTargets.includes(toStatus)
}
```

## 9. 常见问题与解决方案

### 状态计算异常

| 问题                           | 原因                                     | 解决方案                     |
| ------------------------------ | ---------------------------------------- | ---------------------------- |
| 状态显示为"未出运"但有海运记录 | shipmentDate为空                         | 检查海运记录是否正确导入     |
| 已到港但状态仍为"在途"         | ATA或availableTime为空                   | 检查港口操作记录中的到港时间 |
| WMS已确认但状态未更新          | wmsStatus/ebsStatus/wmsConfirmDate未设置 | 检查仓库操作记录             |
| 中转港到港但状态未更新         | 中转港标识错误                           | 确保portType设置为"transit"  |

### 外部API状态映射

| 问题             | 原因                   | 解决方案                     |
| ---------------- | ---------------------- | ---------------------------- |
| 飞驼状态码未映射 | 新状态码未添加到映射表 | 更新ExternalApiToDetailedMap |
| 状态映射错误     | 映射关系不正确         | 检查映射规则是否符合业务逻辑 |

## 10. 最佳实践

1. **状态数据完整性**
   - 确保所有相关数据字段（如returnTime、pickupDate、ata等）及时更新
   - 定期检查数据一致性，避免状态计算错误

2. **状态流转管理**
   - 使用isValidSimplifiedTransition()验证状态流转
   - 避免直接修改状态，应通过数据更新触发状态重新计算

3. **性能优化**
   - 状态计算结果可缓存，避免重复计算
   - 仅在相关数据变更时重新计算状态

4. **监控与预警**
   - 监控异常状态（如HOLD、DETENTION等）
   - 及时处理状态异常，避免影响业务流程
