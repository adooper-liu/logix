# LogiX 开发流程指南

> 掌握 LogiX 项目功能开发的标准流程

## 核心原则

1. **数据库优先**：所有变更以数据库表结构为基准
2. **禁止反向**：代码对齐数据库，不反向改库补数据
3. **数据完整**：导入错误需删除数据 → 修复映射 → 重新导入

## 开发流程概览

```
┌─────────────────────────────────────────────────────────────┐
│  1. 数据库变更    →  2. 实体定义    →  3. API 开发        │
│       ↓                   ↓                 ↓              │
│  03_create_tables.sql   Entity.ts      Controller/Service │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 前端开发      →  5. 联调测试    →  6. 代码审查        │
│       ↓                   ↓                 ↓              │
│  Vue 组件/API     功能验证         lint/type-check        │
└─────────────────────────────────────────────────────────────┘
```

## 步骤详解

### 1. 数据库变更

#### 新增表

在 `backend/03_create_tables.sql` 中添加表定义：

```sql
-- 示例：创建新表
CREATE TABLE biz_example (
    id SERIAL PRIMARY KEY,
    code VARCHAR(50) NOT NULL UNIQUE,
    name VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_example_code ON biz_example(code);
```

#### 新增字段

```sql
ALTER TABLE biz_containers
ADD COLUMN IF NOT EXISTS new_field VARCHAR(50);

COMMENT ON COLUMN biz_containers.new_field IS '新字段说明';
```

### 2. 实体定义

在 `backend/src/entities/` 中创建或更新实体：

```typescript
// backend/src/entities/Example.ts
import { Entity, PrimaryGeneratedColumn, Column, CreateDateColumn, UpdateDateColumn } from 'typeorm';

@Entity({ name: 'biz_example' })
export class Example {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ name: 'code', type: 'varchar', length: 50, unique: true })
  code: string;

  @Column({ name: 'name', type: 'varchar', length: 100, nullable: true })
  name: string;

  @CreateDateColumn({ name: 'created_at' })
  createdAt: Date;

  @UpdateDateColumn({ name: 'updated_at' })
  updatedAt: Date;
}
```

### 3. API 开发

#### 创建 Service

```typescript
// backend/src/services/example.service.ts
import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { Example } from '../entities/Example';

@Injectable()
export class ExampleService {
  constructor(
    @InjectRepository(Example)
    private exampleRepository: Repository<Example>,
  ) {}

  async findAll(): Promise<Example[]> {
    return this.exampleRepository.find();
  }

  async findOne(id: number): Promise<Example> {
    return this.exampleRepository.findOne({ where: { id } });
  }

  async create(data: Partial<Example>): Promise<Example> {
    const example = this.exampleRepository.create(data);
    return this.exampleRepository.save(example);
  }

  async update(id: number, data: Partial<Example>): Promise<Example> {
    await this.exampleRepository.update(id, data);
    return this.findOne(id);
  }

  async delete(id: number): Promise<void> {
    await this.exampleRepository.delete(id);
  }
}
```

#### 创建 Controller

```typescript
// backend/src/controllers/example.controller.ts
import { Request, Response } from 'express';
import { ExampleService } from '../services/example.service';

export class ExampleController {
  constructor(private readonly exampleService: ExampleService) {}

  async findAll(req: Request, res: Response) {
    const data = await this.exampleService.findAll();
    res.json({ success: true, data });
  }

  async findOne(req: Request, res: Response) {
    const { id } = req.params;
    const data = await this.exampleService.findOne(Number(id));
    res.json({ success: true, data });
  }

  async create(req: Request, res: Response) {
    const data = await this.exampleService.create(req.body);
    res.json({ success: true, data });
  }

  async update(req: Request, res: Response) {
    const { id } = req.params;
    const data = await this.exampleService.update(Number(id), req.body);
    res.json({ success: true, data });
  }

  async delete(req: Request, res: Response) {
    const { id } = req.params;
    await this.exampleService.delete(Number(id));
    res.json({ success: true });
  }
}
```

#### 注册路由

在 `backend/src/routes/index.ts` 中添加路由：

```typescript
import { ExampleController } from '../controllers/example.controller';
import { ExampleService } from '../services/example.service';

const exampleService = new ExampleService();
const exampleController = new ExampleController(exampleService);

// 添加路由
router.get('/examples', exampleController.findAll.bind(exampleController));
router.get('/examples/:id', exampleController.findOne.bind(exampleController));
router.post('/examples', exampleController.create.bind(exampleController));
router.patch('/examples/:id', exampleController.update.bind(exampleController));
router.delete('/examples/:id', exampleController.delete.bind(exampleController));
```

### 4. 前端开发

#### API 调用封装

```typescript
// frontend/src/api/example.ts
import axios from './index';

export const exampleApi = {
  list: () => axios.get('/examples'),
  get: (id: number) => axios.get(`/examples/${id}`),
  create: (data: any) => axios.post('/examples', data),
  update: (id: number, data: any) => axios.patch(`/examples/${id}`, data),
  delete: (id: number) => axios.delete(`/examples/${id}`),
};
```

#### 组件开发

遵循 Vue 3 Composition API 规范：

```vue
<!-- frontend/src/views/example/ExampleList.vue -->
<script setup lang="ts">
import { ref, onMounted } from 'vue';
import { exampleApi } from '@/api/example';

const examples = ref<any[]>([]);
const loading = ref(false);

const fetchList = async () => {
  loading.value = true;
  try {
    const res = await exampleApi.list();
    examples.value = res.data.data;
  } finally {
    loading.value = false;
  }
};

onMounted(fetchList);
</script>

<template>
  <div class="example-list">
    <el-table :data="examples" v-loading="loading">
      <el-table-column prop="code" label="编码" />
      <el-table-column prop="name" label="名称" />
    </el-table>
  </div>
</template>
```

## 功能修改流程

### 修改已有功能

1. **分析影响**：确定修改影响的范围
2. **数据库变更**：创建 migration 脚本
3. **同步实体**：更新 Entity 字段
4. **修改 API**：更新 Controller/Service 逻辑
5. **前端适配**：更新组件和调用
6. **验证功能**：确保原有功能不受影响

### Bug 修复流程

1. **复现问题**：确认 bug 表现
2. **定位根因**：分析数据流和代码逻辑
3. **修复代码**：从数据源到展示的完整链路
4. **验证修复**：测试确认问题解决

## 数据库迁移规范

### 脚本位置

- **正式迁移**：`migrations/` 目录
- **诊断修复**：`scripts/` 目录

### 命名规范

```
动作_目标_说明.sql

示例：
add_schedule_status_to_containers.sql
create_resource_occupancy_tables.sql
backfill_last_free_date.sql
```

### 执行方式

```bash
# 通过 Docker 执行
docker exec -i logix-timescaledb-dev psql -U postgres -d logix_db < migrations/your_script.sql

# 或使用 Make
make db-migrate
```
