# Shipments 页与甘特图统计数据一致性对比

本文档说明 **http://localhost:5173/#/shipments** 统计卡片与 **甘特图**（/shipments/gantt-chart）各泳道行上的数量是否一致、在什么条件下一致、以及如何逐项对比。

---

## 一、数据来源对比

| 项目 | Shipments 页（统计卡片） | 甘特图（泳道行数量） |
|------|--------------------------|------------------------|
| **接口** | `GET /api/v1/containers/statistics-detailed?startDate=&endDate=` | 无单独统计接口 |
| **计算位置** | 后端：按出运日期范围内**全库**货柜做聚合（各 Statistics 子服务 count） | 前端：对**当前已加载的货柜列表**做 `getGroupContainersSubset(containers, laneName, group.label).length` |
| **货柜集合** | 出运日期在 [startDate, endDate] 的**全部**货柜（无条数上限） | 来自 `getContainers` 或 `getContainersByFilterCondition` 的 **items**（当前实现 pageSize=5000，即最多 5000 条） |

结论：  
- **口径**：分组与子项定义已对齐（见《甘特图四泳道与 Shipments 统计卡片子集逻辑对比》），同一货柜在相同条件下会归入同一分组。  
- **集合**：Shipments 卡片 = 全库该日期范围内的数量；甘特图 = 当前请求返回的那一批货柜（≤5000）的数量。  
因此：**只有在「甘特图加载的货柜集合 = 该出运日期范围内全量货柜」且总条数 ≤5000 时，两者数量才会一致。**

---

## 二、何时数量会一致

满足以下条件时，同一分组/子项的数字应对齐：

1. **同一出运日期范围**  
   - Shipments 与甘特图使用相同的 startDate、endDate（例如都选「过去 90 天～今天」，或从 Shipments 跳转甘特图时带上相同 query）。

2. **甘特图未做「选中柜号」过滤**  
   - 即未通过 `route.query.containers` 只保留部分柜号。

3. **该日期范围内货柜总数 ≤ 5000**  
   - 甘特图当前用 `getContainers({ page: 1, pageSize: 5000, startDate, endDate })`，只取前 5000 条；若全量超过 5000，甘特图只是子集，数量会小于或等于 Shipments 卡片。

4. **无 filterCondition 或两边用同一 filterCondition**  
   - 若甘特图带 `filterCondition`（从 Shipments 点卡片后跳转），则甘特图列表与 Shipments 当前筛选列表一致，此时应在「同一筛选条件下」对比该条件对应分组。

若不满足 2～4（例如全量 >5000、或只看了部分柜号），则 Shipments 卡片与甘特图**不必相等**，且甘特图表示的是「当前加载集合」的统计，不是全库。

## 三、逐项对应关系（可对比项）

以下为「统计卡片上的键」与「甘特图泳道行标签」的对应关系，用于人工或自动化对比。  
**按到港**、**按提柜计划**、**按最晚提柜**、**按最晚还箱** 四类在两边都有对应项；**按状态** 仅 Shipments 有，甘特图无该泳道。

### 3.1 按到港

| Shipments 卡片（arrivalDistribution 或子项） | 甘特图「按到港」泳道行标签 | 说明 |
|---------------------------------------------|----------------------------|------|
| `today` | 今日到港 | 一致 |
| `arrivedBeforeTodayNotPickedUp` | 今日之前到港未提柜 | 一致 |
| `arrivedBeforeTodayPickedUp` | 今日之前到港已提柜 | 一致 |
| `overdue`（预计到港子项） | 已逾期未到港 | 一致 |
| `within3Days` | 3日内预计到港 | 一致 |
| `within7Days` | 7日内预计到港 | 一致 |
| `over7Days` | 7日后预计到港 | 一致 |
| `other` | 其他记录 | 一致 |
| `arrivedAtDestination` | （主分组） | 卡片主分组 = today + arrivedBeforeTodayNotPickedUp + arrivedBeforeTodayPickedUp，甘特图为三行分别计数 |
| `arrivedAtTransit` / `expectedArrival` | （甘特图无单独「已到中转港」「预计到港」主分组行） | 甘特图仅子项行，无主分组汇总行 |

### 3.2 按提柜计划

| Shipments 卡片（pickupDistribution） | 甘特图「按提柜计划」泳道行标签 |
|-------------------------------------|--------------------------------|
| `overdue` | 逾期未提柜 |
| `todayPlanned` | 今日计划提柜 |
| `pending` | 待安排提柜 |
| `within3Days` | 3天内预计提柜 |
| `within7Days` | 7天内预计提柜 |

### 3.3 按最晚提柜

| Shipments 卡片（lastPickupDistribution） | 甘特图「按最晚提柜」泳道行标签 |
|-----------------------------------------|--------------------------------|
| `expired` | 已逾期 |
| `noLastFreeDate` | 最晚提柜日为空 |
| `urgent` | 紧急 |
| `warning` | 警告 |
| `normal` | 正常 |

### 3.4 按最晚还箱

| Shipments 卡片（returnDistribution） | 甘特图「按最晚还箱」泳道行标签 |
|-------------------------------------|--------------------------------|
| `expired` | 已逾期 |
| `noLastReturnDate` | 最后还箱日为空 |
| `urgent` | 紧急 |
| `warning` | 警告 |
| `normal` | 正常 |

## 四、如何自测「是否一致」

1. **准备**  
   - 选同一出运日期范围（如「过去 90 天～今天」）。  
   - 在 Shipments 页**不点任何卡片**（无筛选），记下四类卡片下各子项数量（或只记按到港/按提柜计划/按最晚提柜/按最晚还箱中一类）。

2. **甘特图**  
   - 用相同 startDate/endDate 打开甘特图（或从 Shipments 点「甘特图」带相同 query）。  
   - 确认未带 `containers` 参数。  
   - 依次切换四个泳道，看每行右侧的 `(n)`。

3. **对比**  
   - 用上表「Shipments 键 ↔ 甘特图行标签」逐项对比。  
   - 若该日期范围内货柜总数 ≤5000，且无其他筛选，数字应一致；若总数 >5000，甘特图可能偏小（子集效应）。

4. **带筛选时**  
   - 在 Shipments 点某卡片子项（如「3日内预计到港」），再点「甘特图」跳转（会带 `filterCondition`）。  
   - 甘特图加载的是该条件对应的货柜列表，此时应对比：  
     - Shipments 上该子项显示的数量 = 后端该条件全量；  
     - 甘特图「按到港」下「3日内预计到港」行数量 = 当前加载列表里该子集的数量。  
   - 若该条件返回条数 ≤5000 且甘特图未再裁切，两者应一致；若后端返回超过 5000 且甘特图只拿一批，则可能不一致。

## 五、小结

| 问题 | 结论 |
|------|------|
| 统计口径是否一致？ | 一致。甘特图分组逻辑已按《甘特图四泳道与 Shipments 统计卡片子集逻辑对比》以后端为准对齐。 |
| 数量是否一定相等？ | 不一定。Shipments 卡片 = 全库该日期范围（及当前筛选）的聚合；甘特图 = 当前加载的货柜列表（最多 5000）的聚合。 |
| 何时相等？ | 同一日期范围、同一筛选（或无筛选）、且该条件下货柜总数 ≤5000、甘特图未用 `containers` 裁切时，可预期相等。 |
| 如何逐项对比？ | 使用上表「Shipments 键 ↔ 甘特图行标签」对照四类（到港/提柜计划/最晚提柜/最晚还箱）即可。 |

如需「严格一致」，可考虑：甘特图在总条数 >5000 时拉全量（或分页汇总后汇总 count），或后端提供与 statistics-detailed 同口径的、按条件返回的 count 接口供甘特图展示；当前实现下，在≤5000 条且同一条件下两者应一致。
