# 数据来源流程与变更日志方案

## 一、四种数据来源的流程与数据影响

### 1. 业务数据导入（Excel 导入）

**入口**：`POST /api/v1/import/excel`、`POST /api/v1/import/excel/batch`  
**前端**：`ExcelImport.vue`、`DemurrageStandardsImport.vue`、`FeituoDataImport.vue` 等

**流程**：
```
用户选择 Excel → 解析并映射字段 → 拆分为各表数据 → 调用导入 API → 后端逐表写入
```

**数据影响**：
| 表 | 操作 | 说明 |
|---|---|---|
| biz_replenishment_orders | INSERT | 备货单（actual_ship_date 等） |
| biz_containers | INSERT | 货柜主表（bill_of_lading_number、logistics_status 等） |
| process_sea_freight | INSERT/UPDATE | 海运信息（shipment_date 等） |
| process_port_operations | INSERT/UPDATE | 港口操作（ata_dest_port、eta_dest_port、gate_in_time 等） |
| process_trucking_transport | INSERT/UPDATE | 拖卡运输（pickup_date 等） |
| process_warehouse_operations | INSERT/UPDATE | 仓库操作 |
| process_empty_return | INSERT/UPDATE | 还空箱（return_time） |

**特点**：
- 按出运日期范围筛选时，导入数据会参与统计
- 导入后若货柜已存在，会触发 `logistics_status` 重算
- 飞驼专用字段（route_code、status_code、data_source 等）**不**由 Excel 写入，仅由 API 写入

---

### 2. 飞驼数据（API 同步）

**入口**：`POST /api/v1/external/sync/:containerNumber`、`POST /api/v1/external/sync/batch`  
**适配器**：`FeiTuoAdapter` → `ExternalDataService`

**流程**：
```
调用飞驼 API 获取 trackingEvents → 转换为状态事件 → 写入 ext_container_status_events
→ 按 FeiTuoStatusMapping 更新 process_port_operations 核心字段
→ 触发 logistics_status 重算
```

**数据影响**：
| 表 | 操作 | 说明 |
|---|---|---|
| ext_container_status_events | INSERT | 状态事件（status_code、occurred_at、data_source='Feituo'） |
| process_port_operations | UPDATE | 核心字段：ata_dest_port、eta_dest_port、gate_in_time、gate_out_time、transit_arrival_date、dest_port_unload_date、available_time 等 |
| biz_containers | UPDATE | logistics_status（由状态机重算） |

**飞驼 Excel 导入**（ext_feituo_import_table1/table2）：
- 与 API 同步不同，是离线 Excel 导入到原始表
- 需通过 `feituoImportService` 解析后写入 process_port_operations，逻辑与 API 类似

---

### 3. 手工维护（CRUD）

**入口**：`PUT /api/v1/containers/:id`、货柜详情页编辑、港口操作/拖卡/还箱等表单

**流程**：
```
用户在前端编辑 → 调用对应 API（containers、port-operations 等）→ 后端 UPDATE 实体
```

**数据影响**：
| 操作 | 表 | 说明 |
|---|---|---|
| 货柜编辑 | biz_containers | bill_of_lading_number、logistics_status（若手动指定）等 |
| 港口操作编辑 | process_port_operations | ata_dest_port、eta_dest_port、last_free_date 等 |
| 拖卡编辑 | process_trucking_transport | pickup_date、planned_pickup_date 等 |
| 还空箱编辑 | process_empty_return | return_time |
| 其他流程表 | process_* | 各字段 |

**特点**：
- 手工维护会覆盖 Excel/API 写入的数据
- 若修改了影响状态机的字段，应触发 `logistics_status` 重算（部分接口已实现）

---

### 4. 状态更新（重算物流状态）

**入口**：`POST /api/v1/containers/:containerNumber/update-status`、`POST /api/v1/containers/update-statuses/batch`  
**服务**：`ContainerStatusService`、`calculateLogisticsStatus`（状态机）

**流程**：
```
读取货柜及关联流程表数据 → 按状态机优先级计算 logistics_status → 若变化则 UPDATE biz_containers
```

**数据影响**：
| 表 | 操作 | 说明 |
|---|---|---|
| biz_containers | UPDATE | 仅 logistics_status 字段 |

**触发场景**：
- 飞驼同步后自动调用
- Excel 导入后可选调用
- 定时任务（containerStatus.scheduler）批量重算
- 用户手动点击「更新状态」

---

## 二、数据流向与优先级（冲突时）

```
数据源优先级（写入时覆盖关系）：
- 手工维护 > 飞驼 API > Excel 导入（对同一字段而言）
- 状态更新：仅重算 logistics_status，不覆盖其他字段

字段分工：
- Excel：备货单、海运、港口、拖卡、仓库、还箱等业务字段（不含飞驼专用字段）
- 飞驼 API：ata_dest_port、gate_in_time、transit_arrival_date 等核心时间 + ext_container_status_events
- 手工：可修改任意字段
- 状态更新：仅 logistics_status
```

## 三、数据变更日志方案

### 3.1 现状

- **无**统一的数据变更审计表
- 仅有 `ext_container_status_events` 记录飞驼状态事件（非通用变更日志）
- 应用日志（logger）写入文件，不便于按业务维度查询

### 3.2 建议：新增 `sys_data_change_log` 表

**表结构**：

```sql
CREATE TABLE IF NOT EXISTS sys_data_change_log (
    id BIGSERIAL PRIMARY KEY,
    source_type VARCHAR(32) NOT NULL,   -- 'excel_import' | 'feituo_api' | 'manual' | 'status_update'
    entity_type VARCHAR(64) NOT NULL,  -- 'biz_containers' | 'process_port_operations' | ...
    entity_id VARCHAR(128),             -- 主键或业务键，如 container_number
    action VARCHAR(16) NOT NULL,       -- 'INSERT' | 'UPDATE' | 'DELETE'
    changed_fields JSONB,               -- 变更字段及新旧值，如 {"ata_dest_port": {"old": null, "new": "2025-03-10"}}
    batch_id VARCHAR(64),               -- 批次号（导入/批量操作时）
    operator_id VARCHAR(64),            -- 操作人（可选）
    operator_ip VARCHAR(64),             -- 请求 IP（可选）
    remark TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_change_log_source ON sys_data_change_log(source_type);
CREATE INDEX idx_change_log_entity ON sys_data_change_log(entity_type, entity_id);
CREATE INDEX idx_change_log_created ON sys_data_change_log(created_at DESC);
```

### 3.3 记录点

| 数据来源 | 记录点 | 建议记录内容 |
|---|---|---|
| Excel 导入 | ImportController.importExcelData | source_type='excel_import', entity_type='biz_containers' 等，batch_id=导入批次 |
| 飞驼同步 | ExternalDataService.updatePortOperationCoreFields | source_type='feituo_api', entity_type='process_port_operations', changed_fields |
| 手工维护 | ContainerController.updateContainer、各流程表 PUT | source_type='manual', changed_fields |
| 状态更新 | ContainerStatusService.updateStatus | source_type='status_update', entity_type='biz_containers', changed_fields={logistics_status} |

### 3.4 查询 API 示例

```
GET /api/v1/audit/changes?entityType=biz_containers&entityId=ABCD1234567
GET /api/v1/audit/changes?sourceType=feituo_api&startDate=2025-03-01&endDate=2025-03-31
GET /api/v1/audit/changes?containerNumber=ABCD1234567
```

### 3.5 实现步骤（已实现）

1. ✅ **迁移脚本**：`backend/migrations/add_sys_data_change_log.sql` 创建表
2. ✅ **Entity**：`backend/src/entities/SysDataChangeLog.ts`
3. ✅ **Service**：`AuditLogService` 提供 `logChange`、`queryChanges`
4. ✅ **埋点**：ImportController、ExternalDataService、ContainerController、ContainerStatusService
5. ✅ **查询 API**：`GET /api/v1/audit/changes`、`GET /api/v1/audit/changes/container/:containerNumber`
6. 待实现：前端货柜详情页「变更日志」Tab

**首次使用需执行迁移**：
```bash
psql -h localhost -U postgres -d logix_db -f backend/migrations/add_sys_data_change_log.sql
```
