# 清关与货柜查验 — 项目与数据库支持情况

## 一、清关（Customs / 清关）

### 1.1 数据库支持

| 表 | 字段 | 说明 |
|----|------|------|
| **process_port_operations** | `customs_status` | 清关状态（如 pending/processing/cleared） |
| | `planned_customs_date` | 计划清关日期 |
| | `actual_customs_date` | 实际清关日期 |
| | `customs_broker_code` | 目的港清关公司代码 → 关联 dict_customs_brokers |
| | `document_status` | 清关单据状态 |
| | `customs_remarks` | 异常原因（清关信息） |
| **process_sea_freight** | `customs_clearance_date` | 海运清关放行日期 |
| **dict_customs_brokers** | 清关公司字典 | broker_code, broker_name 等 |
| **biz_customers** | `customs_code` | 客户海关编码 |

- 主表 `03_create_tables.sql` 已包含上述字段；`04_fix_constraints.sql` 中有 `process_port_operations.customs_broker_code` → `dict_customs_brokers(broker_code)` 外键及 `customs_status` 索引。

### 1.2 后端

- **实体**：`PortOperation` 含 customsStatus、plannedCustomsDate、actualCustomsDate、customsBrokerCode、documentStatus、customsRemarks；`SeaFreight` 含 customsClearanceDate；`CustomsBroker` 对应 dict_customs_brokers。
- **业务**：`container.service` 列表 enrich 时带 `customsStatus`（来自最新港口操作）；`import.controller` 有清关公司校验与自动建码（validateCustomsBroker）；Excel 导入支持计划/实际清关日期、清关公司、清关状态、清关备注等。
- **状态机**：`logisticsStatusMachine` 中 `CUSTOMS_HOLD`（清关扣货）映射为 `at_port`（在港）。

### 1.3 前端

- **Shipments 列表**：列「清关状态」，展示 customsStatus（待清关/清关中/已清关等）。
- **货柜详情**：港口操作里展示清关状态、清关公司（PortOperations.vue）。
- **Excel 导入**：映射「计划清关日期」「实际清关日期」「目的港清关公司」「清关状态」「清关单据状态」「异常原因(清关信息表）」到 process_port_operations。
- **字典映射**：DictMapping 支持 CUSTOMS_BROKER → dict_customs_brokers。

**结论**：清关在库表、实体、导入、列表与详情展示上均有支持；若需更细的清关流程（如状态流转、报表），可在现有 customs_status / document_status 上扩展。

## 二、货柜查验（Inspection / 查验）

### 2.1 数据库支持

| 表 | 字段 | 说明 |
|----|------|------|
| **biz_containers** | `inspection_required` | 是否查验（BOOLEAN） |
| **biz_replenishment_orders** | `inspection_required` | 是否查验（BOOLEAN） |

- 无单独的「查验记录表」或「查验结果/日期」表；仅有「是否需查验」标记。

### 2.2 后端

- **实体**：Container、ReplenishmentOrder 均有 `inspectionRequired`。
- **业务**：列表/详情返回 `inspectionRequired`；Excel 导入支持「是否查验」映射到容器与备货单。
- **文档**：LOGISTICS_FLOW_COMPLETE 等文档中提及「查验标记 (inspection_required)」「查验 (hold) ← inspection_required=true」。

### 2.3 前端

- **Shipments 列表**：列「查验」，展示「是/否」（根据 inspectionRequired）。
- **Excel 导入**：映射「是否查验」到 biz_containers 与 biz_replenishment_orders。
- **多语言**：inspectionRequired 有中/英/日/德/法/意等文案。

**结论**：查验在库表与前后端仅体现为「是否需查验」布尔字段，**没有**：查验日期、查验结果、查验类型、查验状态流转等。若业务需要「查验进度/结果」，需在库表与实体中新增（例如 process_port_operations 或单独 ext_inspection_records）并扩展接口与前端。

## 三、汇总

| 维度 | 清关 | 货柜查验 |
|------|------|----------|
| **数据库** | 已支持：customs_status、计划/实际清关日期、清关公司、单据状态、备注；清关公司字典 | 已支持：仅 inspection_required（是否查验）；无查验记录/结果表 |
| **后端** | 实体、导入、列表/详情、状态机（CUSTOMS_HOLD）均已对接 | 实体、导入、列表/详情仅用 inspection_required |
| **前端** | 列表、详情、导入、字典映射均有 | 列表、导入、多语言有；无查验进度/结果页 |
| **可扩展** | 可做清关状态流转、清关报表 | 若需「查验日期/结果/状态」需新增表与接口 |

清关与查验的**现有**内容与数据库支持如上；后续若要做「国家全局过滤」或「帐号国家权限」，建议按 `GLOBAL_COUNTRY_FILTER_DESIGN.md` 的全局过滤方案实施，避免在每处传递 countryCode。

## 四、数据库与实现速查

### 4.1 清关（Customs）

| 层级 | 内容 |
|------|------|
| **库表** | `process_port_operations`: customs_status, planned_customs_date, actual_customs_date, customs_broker_code, document_status, customs_remarks；`process_sea_freight`: customs_clearance_date；`dict_customs_brokers` 清关公司字典；`04_fix_constraints.sql` 中 customs_broker_code → dict_customs_brokers(broker_code) 外键 |
| **实体** | PortOperation、SeaFreight、CustomsBroker（backend/src/entities） |
| **后端** | container.service 列表 enrich 带 customsStatus；import.controller 中 validateCustomsBroker、Excel 清关字段写入 port_operations；状态机 CUSTOMS_HOLD → at_port |
| **前端** | Shipments 列「清关状态」、PortOperations.vue 清关状态/清关公司；Excel 导入映射计划/实际清关日期、清关公司、清关状态等；DictMapping CUSTOMS_BROKER |
| **如何扩展** | 清关状态流转：在 customs_status / document_status 上定义枚举与流转规则；清关报表：基于 process_port_operations 按 customs_status、actual_customs_date 聚合 |

### 4.2 查验（Inspection）

| 层级 | 内容 |
|------|------|
| **库表** | `biz_containers.inspection_required`、`biz_replenishment_orders.inspection_required`（BOOLEAN）；**无**查验记录/结果/日期表 |
| **实体** | Container.inspectionRequired、ReplenishmentOrder.inspectionRequired |
| **后端** | 列表/详情返回 inspectionRequired；Excel 导入「是否查验」写容器与备货单 |
| **前端** | Shipments 列「查验」展示是/否；Excel 导入「是否查验」；多语言 inspectionRequired |
| **如何扩展** | 若需「查验日期、结果、状态」：在 `process_port_operations` 增字段（如 inspection_date, inspection_result, inspection_status）或新建 `ext_inspection_records`（container_number, inspection_date, result, status 等），再扩展实体、API 与前端展示 |
