# 七步开发流程

**版本**: v1.0  
**创建时间**: 2026-03-31  
**最后更新**: 2026-03-31  
**作者**: 刘志高  

---

## 概述

标准化的七步开发流程，确保每次开发都符合 SKILL 原则。

```
1. 理解需求
   ↓
2. 查阅 SKILL
   ↓
3. 编写测试（TDD）
   ↓
4. 编写代码
   ↓
5. 运行检查
   ↓
6. 创建/更新 SKILL
   ↓
7. 提交代码
```

---

## Step 1: 理解需求

### 目标

明确要解决的问题和业务场景。

### 活动

1. **阅读需求文档**
   - 业务背景
   - 功能要求
   - 验收标准

2. **识别关键信息**
   - 涉及哪些表？
   - 涉及哪些业务流程？
   - 有什么约束条件？

3. **澄清疑问**
   - 不明确的地方立即问
   - 不要假设
   - 确认所有细节

### 输出

- 需求理解文档（可选）
- 关键问题清单

---

## Step 2: 查阅 SKILL

### 目标

了解相关规范和已有知识。

### 活动

1. **阅读 SKILL 原则**
   - [SKILL 原则详解](01-SKILL 原则.md)
   - 理解五大核心原则

2. **查阅相关 SKILL 文档**
   - [文档索引](../../DOCS_INDEX.md)
   - 查找相关的业务知识

3. **查看代码示例**
   - 现有实现参考
   - 最佳实践案例

### 输出

- 相关 SKILL 文档列表
- 参考代码片段

---

## Step 3: 编写测试（TDD）

### 目标

先写测试，确保代码正确性。

### 活动

1. **编写失败的测试**
   ```typescript
   describe('新功能', () => {
     it('应该完成某项任务', () => {
       // 测试代码（此时会失败）
     });
   });
   ```

2. **验证测试失败**
   ```bash
   npm test  # 应该看到红色
   ```

3. **完善测试用例**
   - 正常场景
   - 边界情况
   - 错误处理

### 输出

- 测试文件
- 测试覆盖率报告

---

## Step 4: 编写代码

### 目标

实现功能，通过测试。

### 活动

1. **编写实现代码**
   ```typescript
   // 遵循 SKILL 原则
   // 单一职责、知识沉淀、索引清晰...
   ```

2. **运行测试**
   ```bash
   npm test  # 应该看到绿色
   ```

3. **重构优化**
   - 保持测试通过
   - 改进代码结构
   - 提取公共逻辑

### 输出

- 实现代码
- 通过的测试

---

## Step 5: 运行检查

### 目标

确保符合 SKILL 原则。

### 活动

1. **运行 SKILL Check**
   ```bash
   cd backend
   npm run skill:check
   ```

2. **检查输出**
   ```
   ✅ 通过：6
   ❌ 失败：0
   OK 恭喜！所有文件都符合 SKILL 原则！
   ```

3. **修复问题**
   - 如果有 FAIL，立即修复
   - 重新运行直到全绿

### 输出

- SKILL Check 报告
- 合规的代码

---

## Step 6: 创建/更新 SKILL

### 目标

沉淀业务知识。

### 活动

1. **判断是否需要 SKILL**
   - 新的业务逻辑？
   - 复杂的计算规则？
   - 容易踩坑的地方？

2. **创建 SKILL 文档**
   ```markdown
   # SKILL-{编号}-{主题}
   
   ## 业务场景
   ## 核心规则
   ## 代码实现
   ## 常见错误
   ```

3. **更新索引**
   - 更新 SKILL_INDEX.md
   - 添加相关链接

### 输出

- SKILL 文档（新增或更新）
- 更新的索引文件

---

## Step 7: 提交代码

### 目标

小步提交，完整记录。

### 活动

1. **准备提交**
   ```bash
   git add .
   ```

2. **编写提交信息**
   ```bash
   git commit -m "feat: 添加 XX 功能
   
   - 实现 XX 逻辑
   - 添加单元测试
   - 更新 SKILL 文档"
   ```

3. **推送代码**
   ```bash
   git push
   ```

### 输出

- Git 提交记录
- 推送远程仓库

---

## 检查清单

### 每一步完成后检查

- [ ] Step 1: 需求清晰，无疑问
- [ ] Step 2: 已查阅相关 SKILL
- [ ] Step 3: 测试已写且失败过
- [ ] Step 4: 代码已通过测试
- [ ] Step 5: SKILL Check 全绿
- [ ] Step 6: SKILL 文档已创建/更新
- [ ] Step 7: 代码已提交

---

## 最佳实践

### 1. 严格执行每一步

不要跳过任何步骤：

```
错误：直接写代码 -> 补测试 -> 忘记 SKILL
正确：按七步流程一步步来
```

### 2. 小步快跑

每步都要快速完成并验证：

```
错误：堆积几天的工作再检查
正确：每小时都运行一次检查
```

### 3. 持续改进

发现问题立即修正：

```
看到 SKILL Check FAIL -> 立即修复
发现文档缺失 -> 立即补充
```
