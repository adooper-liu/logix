# 测试指南

LogiX 项目测试规范与实践。

## 测试类型

| 类型 | 工具 | 位置 |
|------|------|------|
| 单元测试 | Jest | backend/src/test/ |
| 单元测试 | Vitest | frontend/src/ |
| E2E 测试 | Playwright | frontend/tests/e2e/ |

## 后端测试

### 运行测试

```bash
cd backend
npm run test
```

### Jest 配置

```javascript
// jest.config.js
module.exports = {
  preset: 'ts-jest',
  testEnvironment: 'node',
  roots: ['<rootDir>/src'],
  testMatch: ['**/__tests__/**/*.ts'],
  moduleFileExtensions: ['ts', 'js'],
};
```

### 测试示例

```typescript
// services/container.service.spec.ts
import { ContainerService } from './container.service';

describe('ContainerService', () => {
  let service: ContainerService;

  beforeEach(async () => {
    service = new ContainerService(containerRepository);
  });

  describe('getContainers', () => {
    it('should return container list', async () => {
      const result = await service.getContainers({ page: 1, limit: 20 });
      expect(result).toHaveProperty('data');
      expect(result).toHaveProperty('total');
    });

    it('should apply country filter', async () => {
      const result = await service.getContainers({
        page: 1,
        limit: 20,
        countryCode: 'US'
      });
      expect(result.data).toBeDefined();
    });
  });
});
```

## 前端测试

### 运行测试

```bash
# 单元测试
cd frontend
npm run test:unit

# E2E 测试
npm run test:e2e
```

### Vitest 配置

```typescript
// vite.config.ts
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    environment: 'jsdom',
    globals: true,
  },
});
```

### 组件测试

```typescript
// components/ContainerCard.spec.ts
import { mount } from '@vue/test-utils';
import { describe, it, expect } from 'vitest';
import ContainerCard from './ContainerCard.vue';

describe('ContainerCard', () => {
  it('should render container info', () => {
    const wrapper = mount(ContainerCard, {
      props: {
        container: {
          containerNumber: 'XXXXX',
          logisticsStatus: 'at_port'
        }
      }
    });

    expect(wrapper.text()).toContain('XXXXX');
    expect(wrapper.text()).toContain('at_port');
  });

  it('should emit delete event', async () => {
    const wrapper = mount(ContainerCard, {
      props: {
        container: { containerNumber: 'XXXXX' }
      }
    });

    await wrapper.find('.delete-btn').trigger('click');
    expect(wrapper.emitted('delete')).toBeTruthy();
  });
});
```

### Composable 测试

```typescript
// composables/useContainerData.spec.ts
import { describe, it, expect, vi } from 'vitest';
import { useContainerData } from './useContainerData';

vi.mock('@/services/container', () => ({
  default: {
    getList: vi.fn().mockResolvedValue({
      list: [{ containerNumber: 'A' }],
      total: 1
    })
  }
}));

describe('useContainerData', () => {
  it('should fetch containers', async () => {
    const { containers, fetchContainers } = useContainerData();
    await fetchContainers({ page: 1, limit: 20 });

    expect(containers.value).toHaveLength(1);
    expect(containers.value[0].containerNumber).toBe('A');
  });
});
```

## E2E 测试

### Playwright 配置

```typescript
// playwright.config.ts
import { defineConfig } from '@playwright/test';

export default defineConfig({
  testDir: './tests/e2e',
  use: {
    baseURL: 'http://localhost:5173',
  },
});
```

### 测试示例

```typescript
// e2e/shipments.spec.ts
import { test, expect } from '@playwright/test';

test('should display container list', async ({ page }) => {
  await page.goto('/shipments');
  await expect(page.locator('.container-table')).toBeVisible();
});

test('should filter containers by status', async ({ page }) => {
  await page.goto('/shipments');
  await page.selectOption('.status-filter', 'at_port');
  await expect(page.locator('.container-row')).toBeVisible();
});
```

## 最佳实践

### 测试命名

```
describe: [模块名]
it: should [预期行为]
```

### 测试覆盖

```
核心业务逻辑: 必须覆盖
API 接口: 必须覆盖
复杂组件: 必须覆盖
工具函数: 必须覆盖
```

### Mock 原则

```
外部依赖: 必须 Mock
数据库: 使用内存数据库或 Mock
API 调用: 使用 Mock Service
时间: Mock Date
```
