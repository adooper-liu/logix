## 📝 文档生成规则（AI 记忆）

### 📄 文档分类规则

1. **长期有效文档**（本目录下各分类）
   - 项目总纲和导航文档
   - 开发规范、代码规范、命名规范
   - 架构设计文档
   - API 文档、数据库文档
   - 使用指南、快速参考
   - **特征**：内容通用、长期有效、持续更新

2. **临时性文档**（`09-misc/`）
   - 特定问题的修复记录、验证报告、迁移记录
   - **特征**：带日期、针对特定场景、可能过期

### 🎯 文档创建规则

1. **创建文档前先分类**
   - 通用的、长期有效的 → 放入对应分类目录（01-standards、02-architecture 等）
   - 临时的、特定场景的 → `09-misc/`

2. **命名规范**
   - 分类内使用编号：`01-xxx.md`、`02-xxx.md`
   - 避免特殊字符（中文、emoji 仅在标题中使用）

3. **文档引用**
   - 正式文档间可以互相引用
   - 使用相对路径：`./11-project/00-项目行动指南.md`

4. **定期维护**
   - 定期清理 09-misc 中过期内容
   - 有价值的内容及时整合到正式文档

### ⚡ 快速判断清单

- 对 6 个月后的新开发者有价值？ → 对应分类目录
- 与特定 bug、验证、迁移相关？ → `09-misc/`
- 需在帮助文档系统中展示？ → 本目录下
