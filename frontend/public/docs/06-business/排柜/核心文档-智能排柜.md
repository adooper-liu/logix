## 一、系统定位

### 1.1 核心定义

智能排柜系统是 LogiX 物流管理平台的**核心业务引擎**，负责自动化安排集装箱的完整物流流程：

```
到港 (ATA/ETA) → 清关 → 提柜 → 送仓 → 卸柜 → 还箱
```

### 1.2 核心价值

| 价值 | 说明 |
|------|------|
| **自动化排产** | 根据到港日期、免费期、仓库产能自动计算最优排产计划 |
| **成本优化** | 对比多种卸柜方案 (Live load / Drop off / Expedited)，选择成本最低方案 |
| **资源管理** | 管理仓库日产能、车队档期，避免资源冲突 |
| **智能决策** | 基于规则引擎和成本模型，提供智能推荐 |

## 二、核心算法

### 2.1 锚点逻辑：以卸柜日为核心

智能排柜以 **plannedUnloadDate（计划卸柜日）** 为核心锚点，向前向后推导其他 5 个日期：

```
                          ┌─────────────────────┐
                          │  锚点: plannedUnloadDate  │
                          │  (基于仓库可用性计算)     │
                          └──────────┬──────────┘
                                     │
      ┌──────────────────────────────┼──────────────────────────────┐
      │                              │                              │
      ▼                              │                              ▼
┌─────────────────┐                 │                  ┌─────────────────┐
│  plannedCustoms │                 │                  │ plannedReturn  │
│  Date (向前)    │                 │                  │ Date (向后)     │
│  = 提柜日 - 1天│                 │                  │ = 卸柜日+在仓天数│
└─────────────────┘                 │                  └─────────────────┘
                                     │
      ┌──────────────────────────────┼──────────────────────────────┐
      │                              │                              │
      ▼                              ▼                              ▼
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│  plannedPickup  │      │ plannedDelivery │      │   lastFreeDate  │
│  Date (向前)    │      │ Date (向前)     │      │   (约束)        │
│  = 清关日 + 1天 │      │ = 提柜日+运时   │      │   ≤ 提柜日      │
│  ≤ lastFreeDate │      │                 │      │                 │
└─────────────────┘      └─────────────────┘      └─────────────────┘
```

### 2.2 五个日期推导规则

| 日期字段 | 计算方式 | 约束条件 |
|----------|----------|----------|
| **plannedCustomsDate** | 向前：提柜日 - 1 天 | 无 |
| **plannedPickupDate** | 向前：清关日 + 1 天 | ≤ lastFreeDate，≥ 今天，跳过周末 |
| **plannedDeliveryDate** | 向前：提柜日 + 运输天数 | 无 |
| **plannedUnloadDate** | **锚点**：基于仓库可用性计算 | ≥ 提柜日 |
| **plannedReturnDate** | 向后：卸柜日 + 在仓天数 | ≤ lastReturnDate（优先）或 lastFreeDate + 7 天 |

### 2.3 卸柜方式规则

| 方式 | hasYard | 提柜日 | 送仓日 | 卸柜日 | 还箱日 | 适用场景 |
|------|---------|--------|--------|--------|--------|----------|
| **Live load** | false | = | = | = | = | 无堆场，当日还箱 |
| **Drop off** | true | < | = | = | +1 | 有堆场，堆存后还箱 |
| **Expedited** | - | = | = | = | = | 免费期内，加急处理 |

## 三、核心服务

### 3.1 排产服务

**文件**: `backend/src/services/intelligentScheduling.service.ts` (1284+ 行)

#### 主流程：batchSchedule

```typescript
async batchSchedule(request: ScheduleRequest): Promise<BatchScheduleResponse> {
  // 1. 获取待排产货柜（scheduleStatus IN ['initial', 'issued']）
  // 2. 按清关日期排序（ATA > ETA，同日内按 lastFreeDate 升序）
  // 3. 滞港费预计算（批量写回 lastFreeDate，并发 CONCURRENCY=5）
  // 4. 分步排产（支持分页）
  // 5. 返回结果（更新数据库，扣减产能）
}
```

#### 单柜排产：scheduleSingleContainer（10 步流程）

```
步骤 1: 获取目的港操作记录
  └─ 提取 ATA/ETA、lastFreeDate

步骤 2: 计算计划清关日
  └─ plannedCustomsDate = ETA（无则用 ATA）

步骤 3: 计算计划提柜日
  └─ plannedPickupDate = 清关日 + 1 天
  └─ 约束：≤ lastFreeDate，≥ 今天，跳过周末

步骤 4: 确定候选仓库
  └─ 映射链：港口 → 车队 → 仓库
  └─ 严格匹配 dict_trucking_port_mapping + dict_warehouse_trucking_mapping

步骤 5: 选择最早可用仓库
  └─ 从提柜日起查询仓库日产能
  └─ 检查 ext_warehouse_daily_occupancy

步骤 6: 选择车队
  └─ 严格匹配映射关系

步骤 7: 决定卸柜方式
  └─ hasYard = true → Drop off
  └─ hasYard = false → Live load

步骤 8: 计算计划还箱日
  └─ Live load: 还 = 卸
  └─ Drop off: 还 = 卸 + 1

步骤 9: 选择清关公司
  └─ 根据国家匹配

步骤 10: 更新数据库
  └─ 更新 5 张流程表
  └─ 扣减仓库/车队/还箱产能
  └─ 同步状态
```

## 九、故障排查

### 9.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 排产失败 - "无映射关系中的仓库" | 映射链不完整 | 补充 dict_trucking_port_mapping / dict_warehouse_trucking_mapping |
| 产能不足 | 当日产能已满 | 增加 ETA 顺延天数或临时增加产能 |
| 日期计算异常 | 周末配置未生效 | 检查 dict_scheduling_config.skip_weekends |
| 成本为0 | 费率配置缺失 | 检查 dict_scheduling_config 费率配置 |

### 9.2 诊断 SQL

```sql
-- 检查车队-港口映射
SELECT * FROM dict_trucking_port_mapping
WHERE port_code = 'GBFXT' AND country = 'GB';

-- 检查仓库-车队映射
SELECT * FROM dict_warehouse_trucking_mapping
WHERE trucking_company_id = 'TC001';

-- 查看仓库占用情况
SELECT * FROM ext_warehouse_daily_occupancy
WHERE warehouse_code = 'WH001'
  AND date BETWEEN '2026-03-24' AND '2026-03-30'
ORDER BY date;

-- 检查排产配置
SELECT * FROM dict_scheduling_config
WHERE config_key IN ('skip_weekends', 'weekend_days', '%rate%');
```
