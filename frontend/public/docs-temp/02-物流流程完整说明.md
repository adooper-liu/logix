```
备货单 (replenishment_orders)
    ↓ (1:N)
货柜 (containers)
    ↓ (1:1)
海运 (sea_freights)
    ↓ (1:N)
```

## 状态机与流转规则

### 主服务简化状态 (7层)

| 状态代码 | 中文名称 | 层级 | 触发条件 |
|---------|---------|------|---------|
| `not_shipped` | 未出运 | 第1层 | 备货单创建，无出运信息 |
| `in_transit` | 在途 | 第2层 | 有中转港记录但未到目的港 |
| `at_port` | 已到港 | 第3层 | 目的港/中转港有实际到港时间 |
| `picked_up` | 已提柜 | 第4层 | 拖车运输有提柜日期 |
| `unloaded` | 已卸柜 | 第5层 | 仓库有卸柜日期 |
| `returned_empty` | 已还箱 | 第6层 | 有还箱日期 |

### 状态自动计算规则

```typescript
calculateLogisticsStatus(container, portOperations): string {
  // 优先级1: 还空箱 (最高)
  if (container.emptyReturnDate) return 'returned_empty';

  // 优先级2: 仓库卸柜
  if (container.warehouseUnloadDate) return 'unloaded';

  // 优先级3: 提柜
  if (container.truckingPickupDate) return 'picked_up';

  // 优先级4: 目的港实际到港
  if (portOperations.some(po => po.portType === 'destination' && po.ataDestPort)) {
    return 'at_port';
  }

  // 优先级5: 中转港到达
  if (portOperations.some(po => po.portType === 'transit' && po.transitArrivalDate)) {
    return 'at_port';
  }

  // 优先级6: 有目的港记录
  if (portOperations.some(po => po.portType === 'destination')) {
    return 'in_transit';
  }

  // 优先级7: 有中转港记录
  if (portOperations.some(po => po.portType === 'transit')) {
    return 'shipped'; // 已出运
  }

  // 优先级8: 有出运日期
  if (container.shipDate) return 'shipped';

  // 默认: 未出运
  return 'not_shipped';
}
```

### 支持跳转的状态流转

```
not_shipped → shipped → in_transit → at_port → picked_up → unloaded → returned_empty
     ↑           ↓            ↓           ↓            ↓           ↓
     └───────────┴────────────┴───────────┴────────────┴───────────┘
              支持跳转 (例如: shipped → at_port, in_transit → picked_up)
```

## 外部数据集成

### 飞驼 API 集成流程

> 飞驼 API 详见 [飞驼可视接口](https://doc.freightower.com/)，提供集装箱综合跟踪（28 节点）、异常预警推送、智能 ETA 等。LogiX 通过 FeiTuoAdapter 对接订阅与查询。

```
1. 请求物流状态
   GET /api/external/container/{containerNumber}/status-events

2. 调用飞驼API
   POST https://api.feituo.com/v1/track
   Body: { containerNumber, language: 'zh' }

3. 解析响应数据
   {
     "status": "ARRIVED",              // 标准状态
     "events": [                       // 状态事件
       { "time": "2026-02-20", "location": "洛杉矶", "description": "到达目的港" },
       { "time": "2026-02-15", "location": "上海", "description": "到达中转港" }
     ]
   }

4. 转换为标准格式
   events → ContainerStatusEvent[]

5. 保存到数据库
   container_status_events 表

6. 更新货柜状态
   container.logistics_status = calculateLogisticsStatus()
```

### 标准状态映射

| 飞驼状态 | 标准状态 | 简化状态 |
|---------|---------|---------|
| NOT_SHIPPED | NOT_SHIPPED | not_shipped |
| SAILING | SAILING | in_transit |
| TRANSIT_ARRIVED | TRANSIT_ARRIVED | at_port |
| ARRIVED | ARRIVED | at_port |
| DISCHARGED | DISCHARGED | at_port |
| GATE_OUT | GATE_OUT | picked_up |
| STRIPPED | STRIPPED | unloaded |
| RETURNED_EMPTY | RETURNED_EMPTY | returned_empty |

## 核心功能模块

### 1. 货柜管理

**功能点**:
- 创建/编辑/删除货柜
- 关联备货单
- 录入海运信息
- 查看货柜详情

**关键API**:
```
GET    /api/containers              # 获取货柜列表
GET    /api/containers/:number     # 获取货柜详情
POST   /api/containers             # 创建货柜
PUT    /api/containers/:number     # 更新货柜
DELETE /api/containers/:number     # 删除货柜
```

### 2. 港口操作

**功能点**:
- 多港经停支持
- 起运港/中转港/目的港区分
- 清关状态跟踪
- 查验标记

**关键API**:
```
GET    /api/containers/:number/port-operations    # 获取港口操作
POST   /api/containers/:number/port-operations    # 添加港口操作
PUT    /api/port-operations/:id                   # 更新港口操作
```

### 3. Excel导入/导出

**导入流程**:
```
1. 选择Excel文件
2. 读取并验证数据
3. 字段映射
4. 数据转换 (标准化)
5. 批量插入数据库
6. 显示导入结果
```

**导出流程**:
```
1. 选择筛选条件
2. 查询数据库
3. 格式化数据
4. 生成Excel文件
5. 自动下载
```

### 4. 倒计时卡片

**显示内容**:
```
┌─────────────────────────┐
│  📦 预计到港倒计时        │
├─────────────────────────┤
│  货柜号: MRKU4896861    │
│  目的港: 洛杉矶          │
│  ETA: 2026-03-06        │
│                         │
│  ⏱️ 剩余天数: 12 天     │
└─────────────────────────┘
```

**计算逻辑**:
```typescript
calculateDaysRemaining(etaDate): number {
  const now = new Date();
  const eta = new Date(etaDate);
  const diffTime = eta.getTime() - now.getTime();
  const diffDays = Math.ceil(diffTime / (1000 * 60 * 60 * 24));
  return diffDays;
}
```

### 5. 滞港费计算

**匹配条件** (必须同时满足):
1. 进口国
2. 目的港
3. 船公司
4. 货代公司
5. 有效期内 (生效日期 <= 当前日期)

**计算方式**:
- 按到港: 从 eta_dest_port 或 ata_dest_port 开始
- 按卸船: 从卸船时间开始

**计算公式**:
```typescript
逾期天数 = 当前日期 - (到港日期 + 免费天数)
滞港费 = 逾期天数 × 日费率
```
