# 飞驼集成效果验证

**版本**: v1.0  
**最后更新**: 2026-03-18  
**用途**: 验证飞驼 API 集成、Excel 导入、状态映射、数据流与状态机联动

## 一、验证方式总览

| 方式 | 工具/入口 | 验证内容 |
|------|-----------|----------|
| **1. API 接口** | POST/GET /api/external/* | 飞驼同步、事件查询、统计 |
| **2. 前端页面** | 导入 → 飞驼验证 | 数据源统计、货柜列表、事件时间线 |

## 二、验证步骤

### 2.1 前置条件

- [ ] 后端服务已启动（`npm run dev`）
- [ ] 数据库已初始化（含 `ext_container_status_events`、`process_port_operations` 等）
- [ ] 飞驼 API 配置（`.env` 中 `FEITUO_API_KEY`、`FEITUO_API_URL`，可选，Excel 导入不依赖）

### 2.2 方式一：API 接口验证

#### 2.2.1 同步单个货柜（需飞驼 API）

```bash
# 替换为真实货柜号
curl -X POST http://localhost:3001/api/v1/external/sync/EXAMPLE1234567 \
  -H "Content-Type: application/json" \
  -d '{"dataSource":"Feituo"}'
```

**预期**：
- 成功：`{"success":true,"message":"成功同步 N 个状态事件","data":{...}}`
- 失败（无 Token）：`{"success":false,"message":"飞驼 API 未配置或 Token 无效"}`（Excel 导入不受影响）

#### 2.2.2 查询状态事件

```bash
curl "http://localhost:3001/api/v1/external/events/EXAMPLE1234567?limit=20"
```

**预期**：返回 `status_code`、`occurred_at`、`data_source` 等字段

#### 2.2.3 获取统计

```bash
curl "http://localhost:3001/api/v1/external/stats"
```

**预期**：`totalEvents`、`dataSourceStats`、`estimatedStats` 等

#### 2.2.4 获取有外部数据的货柜列表

```bash
curl "http://localhost:3001/api/v1/external/containers?dataSource=FeituoAPI&page=1&pageSize=20"
```

**预期**：货柜号、数据源、事件数、首末事件时间

---

### 2.3 方式二：前端页面验证

1. 进入 **导入**：`/import`
2. 使用 **飞驼 Excel** 导入表一/表二
3. 进入 **飞驼验证**：`/import/feituo-verify`（或对应路由）
4. 检查：
   - 统计卡片：总事件数、数据源分布、预计/实际分布
   - 货柜列表：有 FeituoAPI 或 Feituo 的货柜
   - 点击货柜 → 跳转 Shipments 详情 → 物流路径 Tab：事件时间线、状态节点

**通过标准**：
- 导入后能在统计中看到事件
- 物流路径 Tab 能正确展示 BDAR/DSCH/STCS 等节点及时间

---

### 2.4 方式三：P0 修复脚本验证

```bash
cd backend
npx tsx scripts/test-p0-feituo-fix.ts
```

**验证内容**：
- 火车状态码 IRAR、IRDS、PLFD 映射
- PCAB 正确映射到 `available_time` 而非卸船日期
- 港口操作记录、状态事件写入正确

**预期**：控制台输出 `✅` 标记，无 `❌`

---

### 2.5 方式四：SQL 数据流验证

```bash
psql -U your_user -d your_db -f backend/scripts/verify-feituo-import-to-logistics-path.sql
```

**验证内容**：
- `ext_feituo_import_*` 表有导入记录
- `ext_container_status_events` 有状态事件
- `process_port_operations` 有目的港 ATA、卸船、可提货、提柜
- `process_trucking_transport`、`process_empty_return` 有对应日期

**通过标准**：单柜示例（如 MSKU0627486）在 `ext_container_status_events` 和 `process_port_operations` 有数据

---

### 2.6 方式五：状态机联动验证

1. 导入或同步飞驼数据后，查看货柜 `logistics_status`
2. 预期：`ata_dest_port` 有值 → `at_port`；`gate_out_time` 有值 → `picked_up`；`return_time` 有值 → `returned_empty`
3. 检查 Shipments 列表「按状态」卡片：点击「已到目的港」「已提柜」等，对应货柜应正确筛选

**验证命令**（可选）：

```sql
-- 抽查：有 ATA 的货柜，状态应为 at_port 或更后
SELECT c.container_number, c.logistics_status, po.ata_dest_port
FROM biz_containers c
LEFT JOIN process_port_operations po ON po.container_number = c.container_number AND po.port_type = 'destination'
WHERE po.ata_dest_port IS NOT NULL
LIMIT 5;
```

## 三、核心校验点

| 校验项 | 说明 | 通过标准 |
|--------|------|----------|
| **1. 状态码映射** | 20 个核心状态码 → 核心字段 | FeiTuoStatusMapping 含 AVLE/AVAIL/PCAB、DSCH、BDAR 等 |
| **2. 核心字段写入** | 飞驼事件 → process_port_operations | ata_dest_port、dest_port_unload_date、available_time、gate_out_time 等有值 |
| **3. 状态机重算** | 同步后触发 calculateLogisticsStatus | logistics_status 与核心字段一致 |
| **4. places 优先** | 有 places 时优先使用 | externalDataService 优先 processPlaces |

## 四、常见问题排查

| 现象 | 可能原因 | 排查步骤 |
|------|----------|----------|
| 同步返回「Token 无效」 | 未配置 FEITUO_API_KEY | 检查 .env，Excel 导入不受影响 |
| 事件数为 0 | 货柜号不存在或飞驼无数据 | 换真实货柜号，或先用 Excel 导入 |
| ata_dest_port 为空 | 无 BDAR/POCA 事件 | 查 ext_container_status_events 的 status_code |
| logistics_status 异常 | 状态机未重算 | 确认 sync 后调用了 recalculateLogisticsStatus |
| PCAB 误作卸船 | 旧逻辑 | 已修复，PCAB 仅映射 available_time |
