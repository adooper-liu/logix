### 2.2 不一致项：AVLE vs AVAIL/PCAB

| 来源 | 可提货状态码 | 核心字段 |
|------|--------------|----------|
| 状态码映射验证报告 | **AVLE** | available_time |
| 飞驼API集成完成报告 | **AVLE**（集装箱事件） | - |
| FeituoImportSpec.equipment | **AVLE** | - |
| feituoPlaces.processor.destination | **AVLE** | - |
| **FeiTuoStatusMapping.ts** | **AVAIL**, **PCAB** | available_time |

**结论**：文档与 FeituoImportSpec、feituoPlaces.processor 均使用 **AVLE**，但 FeiTuoStatusMapping 仅映射 AVAIL、PCAB，**未映射 AVLE**。

**影响**：当飞驼 API 返回 `statusCode: 'AVLE'` 时，`getCoreFieldName('AVLE')` 返回 `null`，该事件不会更新 `available_time`，可提货时间可能缺失。

**修复建议**：在 `FeiTuoStatusMapping.ts` 中补充 AVLE 映射：

```typescript
// FEITUO_STATUS_TO_CORE_FIELD_MAP
'AVLE': 'available_time',           // 可提货 Available (equipmentEvents)
'AVAIL': 'available_time',
'PCAB': 'available_time',

// FEITUO_STATUS_TO_PORT_TYPE_MAP
'AVLE': 'destination',

// FEITUO_STATUS_TYPE_MAP
'AVLE': 'AVAILABLE',
```

### 3.2 DEFAULT_STATUS_CODES 与文档

feituoPlaces.processor 中 `destination: ['BDAR', 'POCA', 'DSCH', 'AVLE', 'GTIN', 'RCVE']` 与文档一致，但 AVLE 在 FeiTuoStatusMapping 中缺失，若 places 后续按状态码更新核心字段会受影响。当前 places 主要通过 place.ata/place.eta 等直接更新，不依赖 statusCode，故 places 主流程正常。
