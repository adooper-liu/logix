# LogiX Shipments 统计卡片说明

> 文档版本：2025-03  
> 关联：`public/docs-temp/状态机.md`、Shipments 页面、`useContainerCountdown`

## 一、概述

Shipments 页面顶部有五张统计卡片，用于按不同维度展示货柜分布，支持点击筛选列表。所有统计均以**顶部日期范围**（出运日期口径）为筛选条件。

| 卡片 | 目标集 | 主分组数 | 用途 |
|------|--------|----------|------|
| 按状态 | 全部货柜 | 7 | 物流状态机分布 |
| 按到港 | 已出运至已还箱 | 4 | 到港时间分布 |
| 按提柜计划 | 已到目的港且未提柜 | 5 | 计划提柜进度 |
| 按最晚提柜 | 已到目的港且未提柜 | 5 | 免费期倒计时 |
| 按最晚还箱 | 已提柜或已卸柜且未还箱 | 5 | 还箱倒计时 |

---

## 二、统一约定

### 2.1 日期口径

- **字段**：`actual_ship_date`（备货单）或 `shipment_date`（海运）
- **子查询**：`createDateRangeSubQuery`（`backend/src/services/statistics/common/DateRangeSubquery.ts`）
- **范围**：`[startDate, endDate]`，endDate 使用当天 23:59:59.999

### 2.2 Fallback 逻辑

当传了日期且 **status 或 arrival 任一合计为 0** 时，**同时**回退为「不按日期」统计，避免两卡片使用不同日期范围导致总数不一致。

- **触发**：`statusTotal === 0` 或 `arrivalTotal === 0`
- **行为**：status 与 arrival 均改为全量统计
- **标识**：`dateFilterFallback: true`，前端可提示「所选日期范围内无出运记录，已显示全部货柜」

### 2.3 数据流

```
GET /api/v1/containers/statistics-detailed?startDate=&endDate=
    ↓
ContainerStatisticsService.getStatisticsDetailed()
    ↓
statusDistribution, arrivalDistribution, pickupDistribution, lastPickupDistribution, returnDistribution
    ↓
useContainerCountdown(statisticsData) → countdownByStatus, countdownByArrival, ...
    ↓
CountdownCard 展示 + 点击 → handleCountdownFilter → loadContainersByFilter(filterCondition)
```

## 三、五张卡片详解

### 3.1 按状态（物流状态分布）

| 项目 | 说明 |
|------|------|
| **目标集** | 全部货柜（出运日在日期范围内） |
| **数据源** | `StatusDistributionService.getDistribution()` |
| **主统计** | `container.logistics_status` 分组计数 |
| **补充** | `arrived_at_transit`、`arrived_at_destination` 用 SQL 复刻状态机 |

**分组结构：**

```
未出运 (not_shipped)
已出运 (shipped)
在途
├── 未到港 (in_transit)
└── 已到中转港 (arrived_at_transit)
已到目的港 (arrived_at_destination)
已提柜 (picked_up)
已卸柜 (unloaded)
已还箱 (returned_empty)
```

**互斥与穷尽**：✅ 互斥 ✅ 穷尽（每个柜只有一个 `logistics_status`）

---

### 3.2 按到港（到港时间分布）

| 项目 | 说明 |
|------|------|
| **目标集** | 已出运至已还箱（`logistics_status IN (shipped, in_transit, at_port, picked_up, unloaded, returned_empty)`） |
| **数据源** | `ArrivalStatisticsService` + `EtaStatisticsService` |

**分组结构：**

```
已到目的港
├── 今日到港
├── 之前未提柜
└── 之前已提柜
已到中转港
├── 已逾期
├── 3日内
├── 7日内
├── 7日后
└── 无ETA
预计到港（主数 = 子项之和）
├── 已逾期
├── 3天内
├── 7天内
├── 7天后
└── 其他
到港数据缺失
└── 目的港无ATA无ETA但状态显示已到港
```

**口径说明：**

- **已到目的港**：目的港有 ATA，按今日/之前未提柜/之前已提柜细分
- **已到中转港**：有中转港到港记录、目的港无 ATA，按目的港 ETA 细分
- **预计到港**：目的港无 ATA，按目的港 ETA 与今天比较（shipped/in_transit）
- **到港数据缺失**：目的港无 ATA 无 ETA，但状态为 at_port/picked_up/unloaded/returned_empty

**互斥与穷尽**：✅ 互斥 ✅ 穷尽

---

### 3.3 按提柜计划（计划提柜分布）

| 项目 | 说明 |
|------|------|
| **目标集** | 已到目的港且未提柜（与按状态「已到目的港」对齐） |
| **数据源** | `PlannedPickupStatisticsService.getDistribution()` |
| **基准字段** | `plannedPickupDate`（计划提柜日） |

**分组结构：**

```
逾期未提柜 (plannedPickupDate < 今天)
今日计划提柜
3天内计划提柜
7天内计划提柜
待安排提柜 (无 plannedPickupDate)
```

**互斥与穷尽**：✅ 互斥 ✅ 穷尽

---

### 3.4 按最晚提柜（最晚提柜倒计时）

| 项目 | 说明 |
|------|------|
| **目标集** | 与按提柜计划相同（已到目的港且未提柜） |
| **数据源** | `LastPickupStatisticsService.getDistribution()` |
| **基准字段** | `last_free_date`（目的港港口操作表，免费期截止日） |

**分组结构：**

```
已超时 (last_free_date < 今天)
即将超时(1-3天)
预警(4-7天)
时间充裕(7天以上)
最晚提柜日为空
```

**互斥与穷尽**：✅ 互斥 ✅ 穷尽

---

### 3.5 按最晚还箱（最晚还箱倒计时）

| 项目 | 说明 |
|------|------|
| **目标集** | 已提柜或已卸柜且未还箱（`logistics_status IN ('picked_up', 'unloaded')`） |
| **数据源** | `LastReturnStatisticsService.getDistribution()` |
| **基准字段** | `last_return_date`（来自 process_empty_return 或滞箱费计算） |

**分组结构：**

```
已逾期未还箱
紧急：倒计时3天内
警告：倒计时7天内
正常
最后还箱日为空
```

**互斥与穷尽**：✅ 互斥 ✅ 穷尽

## 四、单卡内部：互斥与穷尽分析表

### 4.1 主分组（顶级筛选项）

| 卡片 | 目标集 | 主分组 | 是否互斥 | 是否穷尽 | 说明 |
|------|--------|--------|----------|----------|------|
| **按状态** | 全部货柜 | 未出运、已出运、未到港、已到中转港、已到目的港、已提柜、已卸柜、已还箱 | ✅ 互斥 | ✅ 穷尽 | 每个柜只有一个 `logistics_status`，7 类互斥且覆盖全部 |
| **按到港** | 已出运至已还箱 | 已到目的港、已到中转港、预计到港、到港数据缺失 | ✅ 互斥 | ✅ 穷尽 | 按目的港 ATA / 中转港 / 在途 / 数据缺失分，四类互斥且覆盖目标集 |
| **按提柜计划** | 已到目的港且未提柜 | 逾期未提柜、今日计划提柜、3 天内、7 天内、待安排提柜 | ✅ 互斥 | ✅ 穷尽 | 每个柜只有一个计划提柜日（含空），5 类互斥且覆盖目标集 |
| **按最晚提柜** | 已到目的港且未提柜 | 已超时、1–3 天、4–7 天、7 天以上、最晚提柜日为空 | ✅ 互斥 | ✅ 穷尽 | 每个柜只有一个 `last_free_date`（含空），5 类互斥且覆盖目标集 |
| **按最晚还箱** | 已提柜或已卸柜且未还箱 | 已逾期未还箱、3 天内、7 天内、正常、最后还箱日为空 | ✅ 互斥 | ✅ 穷尽 | 每个柜只有一个 `last_return_date`（含空），5 类互斥且覆盖目标集 |

#### 按到港

| 主分组 | 子项 | 是否互斥 | 是否穷尽 | 说明 |

## 五、跨卡片口径差异

| 货柜状态 | 按状态 | 按到港 | 按提柜计划 | 按最晚提柜 | 按最晚还箱 |
|----------|--------|--------|------------|------------|------------|
| 未出运 | ✅ | ❌ | ❌ | ❌ | ❌ |
| 已出运 | ✅ | ✅ 预计到港 | ❌ | ❌ | ❌ |
| 在途（未到港） | ✅ | ✅ 预计到港 | ❌ | ❌ | ❌ |
| 已到中转港 | ✅ | ✅ 已到中转港 | ❌ | ❌ | ❌ |
| 已到目的港未提柜 | ✅ | ✅ 已到目的港 | ✅ | ✅ | ❌ |
| 已提柜 | ✅ | ✅ 已到目的港（之前已提柜） | ❌ | ❌ | ✅ |
| 已卸柜 | ✅ | ✅ 已到目的港（之前已提柜） | ❌ | ❌ | ✅ |
| 已还箱 | ✅ | ✅ 已到目的港（之前已提柜） | ❌ | ❌ | ❌ |

同一货柜出现在多张卡是设计如此（不同维度），非重复统计。

## 七、filterCondition 与列表筛选

点击卡片标签时，`filterCondition` 传给 `getContainersByCondition`，用于获取对应货柜列表。常用映射见 `FilterConditions.ts`，例如：

- `arrivedAtDestination`、`arrivedAtTransit`、`expectedArrival` → arrival
- `overdue`、`within3Days`、`over7Days`、`otherRecords` → eta
- `expired`、`urgent`、`warning`、`normal`、`noLastFreeDate` → lastPickup
- `returnExpired`、`returnUrgent`、`returnNormal`、`noLastReturnDate` → lastReturn
