# 字典数据提取工具使用指南

## 功能概述

字典数据提取工具用于从飞驼（FeiTuo）Excel数据文件中提取以下四类字典信息：

1. **船公司字典**：12+ 主要船公司（CMA, YML, WHL, MSK, MSC, HPL, HMM 等）
2. **港口字典**：20+ 主要港口（CNSHA, FRLEH, CNNKG, USSAV, USLGB 等）
3. **状态码全集**：RCVE, STSP, GITM, LOBD, DLPT, BDAR, POCA, DSCH, STCS 等
4. **运输模式**：大船、驳船、卡车、铁路全覆盖

## 主要用途

### 1. 初始化字典表
- 快速初始化 `dict_shipping_companies` 船公司字典表
- 快速初始化 `dict_ports` 港口字典表
- 快速初始化 `dict_status_codes` 状态码字典表
- 快速初始化 `dict_transport_modes` 运输模式字典表

### 2. 验证状态码映射完整性
- 验证飞驼状态码是否在系统映射配置中存在
- 识别未映射的状态码，防止数据丢失
- 确保多式联运场景下的状态转换正确

### 3. 测试多式联运场景
- 验证驳船+大船+铁路+卡车的完整运输链
- 提取各种运输模式下的状态码
- 确保跨运输方式的数据一致性

## 使用步骤

### 步骤 1：访问工具页面

```
访问地址: http://localhost:5173/#/import/dictionary-extractor
```

或者在系统菜单中：
- **数据导入** → **字典数据提取**

### 步骤 2：上传Excel文件

1. 准备飞驼Excel数据文件（.xlsx 或 .xls 格式）
2. 拖拽文件到上传区域，或点击选择文件
3. 确保文件包含以下列（至少包含一列）：
   - 船公司相关列："船公司", "船公司名称", "shipping company"
   - 港口相关列："目的港", "目的港名称", "port of discharge"
   - 状态码相关列："状态码", "status code", "事件代码"
   - 运输模式相关列："运输方式", "transport mode", "运输模式"

### 步骤 3：提取字典数据

点击 **"提取字典数据"** 按钮，系统将：

1. 解析Excel文件
2. 自动识别相关列
3. 提取唯一值并去重
4. 标准化数据格式（如船公司代码大写）
5. 显示提取结果汇总

**提取结果展示**：
- 船公司：显示提取到的船公司列表和数量
- 港口：显示提取到的港口列表和数量
- 状态码：显示提取到的状态码列表和数量
- 运输模式：显示提取到的运输模式列表和数量

### 步骤 4：验证映射完整性

点击 **"验证状态码映射"** 按钮，系统将：

1. 对比提取的状态码与系统已知映射
2. 对比提取的船公司与系统已知映射
3. 对比提取的港口与系统已知映射
4. 生成验证报告，显示：
   - ✓ 已映射的项目（绿色）
   - ✗ 未映射的项目（红色/黄色）

**验证结果说明**：
- **状态码**：严格匹配，未映射的状态码会导致数据处理失败
- **船公司**：宽松匹配，未映射的船公司会显示警告
- **港口**：宽松匹配，未映射的港口会显示信息提示

### 步骤 5：导出数据

根据需求选择导出格式：

#### 导出为CSV
- 适合数据分析、备份和共享
- 包含所有提取的字典数据
- 格式：数据类型,值,数量

#### 导出为SQL
- 适合直接初始化数据库字典表
- 自动生成INSERT语句
- 包含：船公司、港口、状态码、运输模式

#### 导出验证报告（如进行了验证）
- Markdown格式
- 包含完整的映射验证结果
- 提供未映射项目的处理建议

## 导出文件示例

### SQL导出示例

```sql
-- 船公司字典
INSERT INTO dict_shipping_companies (company_code, company_name, status) VALUES
('CMA', 'CMA', 'active'),
('YML', 'YML', 'active'),
('MSK', 'MSK', 'active');

-- 港口字典
INSERT INTO dict_ports (port_code, port_name, port_type, status) VALUES
('CNSHA', 'CNSHA', 'seaport', 'active'),
('FRLEH', 'FRLEH', 'seaport', 'active');

-- 状态码字典
INSERT INTO dict_status_codes (status_code, status_name, category, description) VALUES
('RCVE', 'RCVE', 'logistics', 'Auto imported status code'),
('STSP', 'STSP', 'logistics', 'Auto imported status code');

-- 运输模式字典
INSERT INTO dict_transport_modes (mode_code, mode_name, category) VALUES
('大船', '大船', 'transport'),
('驳船', '驳船', 'transport');
```

## 常见问题

### Q1: 上传文件后提示"未找到相关列"

**原因**：Excel文件缺少必要的列名

**解决方案**：
1. 确保Excel包含以下任一列名：
   - 船公司："船公司", "船公司名称", "shipping company"
   - 港口："目的港", "目的港名称", "port of discharge"
   - 状态码："状态码", "status code", "事件代码"
   - 运输模式："运输方式", "transport mode", "运输模式"

### Q2: 提取到的船公司显示为全称（如"CMA CGM"）

**原因**：系统自动提取船公司代码（如"CMA"）

**说明**：这是正常行为，系统会自动识别并提取标准代码

### Q3: 验证结果显示大量未映射的状态码

**原因**：飞驼数据包含系统尚未配置的状态码

**解决方案**：
1. 导出验证报告
2. 查看未映射的状态码列表
3. 更新 `backend/src/constants/FeiTuoStatusMapping.ts` 文件
4. 重新部署后端服务

### Q4: 港口代码显示为中文名称（如"上海港"）

**原因**：Excel中使用中文港口名称而非标准代码

**解决方案**：
1. 在 `dict_ports` 表中添加中文名称到标准代码的映射
2. 使用字典映射功能进行转换
3. 或手动将Excel中的中文名称替换为标准代码（CNSHA, etc.）

## 最佳实践

### 1. 首次初始化字典表

```bash
# 步骤
1. 上传完整的飞驼历史数据Excel
2. 提取所有字典数据
3. 验证映射完整性
4. 导出SQL脚本
5. 在数据库中执行SQL初始化字典表
```

### 2. 定期验证新数据

```bash
# 每月执行一次
1. 上传上月的飞驼数据Excel
2. 提取字典数据
3. 验证映射完整性
4. 如发现新状态码，及时更新映射配置
```

### 3. 多式联运场景测试

```bash
# 每季度执行一次
1. 准备包含各种运输模式的测试数据
2. 上传测试Excel
3. 验证所有状态码映射
4. 确保驳船/大船/铁路/卡车全覆盖
```
