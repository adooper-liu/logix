## 🎯 核心功能

### 功能定位

基于后端服务代码自动生成符合规范的计算逻辑场景模拟文档，格式参考：
`frontend/public/docs/第 2 层 - 代码文档/计算逻辑场景模拟.md`

## 🛠️ 维护策略

### 何时需要更新文档？

**必须更新**：

- 源代码计算公式变更
- 新增计算场景或分支
- 数据库表结构调整
- 业务规则变更（如免费天数从 7 天改为 10 天）

**无需更新**：

- 代码重构但逻辑不变
- 注释或日志修改
- 性能优化但不改变结果

## 📝 常见问题

### Q1: 为什么有些场景计算结果是 0？

A: 这可能是正常情况。例如：

- 在免费期内提柜，chargeDays = 0
- 预测模式下 ETA 未到，费用 = 0

关键是如实反映代码逻辑。

### Q2: 如何确定场景参数的合理性？

A: 参考以下来源：

1. 生产环境真实数据
2. 业务规则文档
3. 测试用例数据
4. 与业务分析师确认

### Q3: 代码中有多个计算方法怎么办？

A: 分章节说明：

1. 为每个方法单独创建章节
2. 说明各自的适用场景
3. 提供调用关系图
