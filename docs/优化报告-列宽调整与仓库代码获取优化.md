## 🔧 仓库代码获取优化

### 问题现象

点击"优化"按钮时，弹出提示：
```
缺少仓库信息，无法进行优化
```

### 根本原因

原代码只从两个字段获取仓库代码：
```typescript
// ❌ 问题：获取源太少，可能遗漏
const warehouseCode = row.plannedData?.warehouseCode || row.warehouseCode
```

### 优化方案

#### 1. 扩展获取源

**优化后：**
```typescript
// ✅ 优化：从多个可能的字段获取仓库代码
const warehouseCode = 
  row.plannedData?.warehouseCode ||    // 优先从 plannedData 获取
  row.warehouseCode ||                 // 从根对象获取
  row.destinationWarehouse ||          // 备用字段 1
  row.warehouseId                      // 备用字段 2

// ✅ 优化：从多个可能的字段获取车队 ID
const truckingCompanyId = 
  row.plannedData?.truckingCompanyId || 
  row.truckingCompanyId || 
  row.truckingCompany
```

#### 2. 增强错误提示

**优化前：**
```typescript
if (!warehouseCode) {
  ElMessage.warning('缺少仓库信息，无法进行优化')
  return
}
```

**优化后：**
```typescript
if (!warehouseCode) {
  console.error('[handleOptimizeContainer] 缺少仓库代码，所有可能的字段都为空')
  ElMessage.warning('缺少仓库信息，无法进行优化。请确认排产结果中是否包含仓库信息。')
  return
}
```

**改进点：**
- ✅ 添加控制台错误日志
- ✅ 提示更具体，指导用户检查排产结果
- ✅ 输出所有可能的字段名，方便调试

#### 3. 增强调试信息

**优化前：**
```typescript
console.log('[handleOptimizeContainer] 检查参数:', {
  containerNumber,
  warehouseCode,
  truckingCompanyId,
  basePickupDate,
  plannedData: row.plannedData,
})
```

**优化后：**
```typescript
console.log('[handleOptimizeContainer] 检查参数:', {
  containerNumber,
  warehouseCode,
  truckingCompanyId,
  basePickupDate,
  plannedData: row.plannedData,
  rowKeys: Object.keys(row),  // ✅ 新增：输出所有字段名
})
```

**作用：**
- ✅ 查看对象的所有字段
- ✅ 快速定位缺失的字段
- ✅ 方便排查数据结构问题

## 🎨 技术实现细节

### 列宽优化策略

**为什么减少 30%？**
- 原宽度：280px + 300px = 580px
- 新宽度：200px + 210px = 410px
- 节省：170px（约 29.3%）

**内容适配性：**
- ✅ 200px 足够显示 4 个日期（每行约 50px）
- ✅ 210px 足够显示树形费用明细
- ✅ 不会造成内容挤压或换行

### 字段获取策略

**优先级顺序：**
```typescript
// 仓库代码获取优先级
1. row.plannedData?.warehouseCode    // 最优：排产数据中的仓库代码
2. row.warehouseCode                 // 次优：根对象的仓库代码
3. row.destinationWarehouse          // 备选：目的仓库字段
4. row.warehouseId                   // 备选：仓库 ID 字段

// 车队 ID 获取优先级
1. row.plannedData?.truckingCompanyId    // 最优：排产数据中的车队 ID
2. row.truckingCompanyId                 // 次优：根对象的车队 ID
3. row.truckingCompany                   // 备选：车队字段
```
