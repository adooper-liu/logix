## 📋 **脚本体系总览**

LogiX项目的脚本文件按功能和位置分为以下几大类：

```
scripts/                          # 根目录脚本（~120 个）
├── 数据库查询与验证              # SQL 脚本（50+ 个）
├── 数据导入与清理                # 数据管理（20+ 个）
├── 功能测试与诊断                # 测试调试（15+ 个）
├── 代码质量检查                  # 质量保障（4 个）
└── 文档与报告                    # 说明文档（30+ 个）

backend/scripts/                  # 后端专用脚本（~50 个）
├── 数据库初始化                   # 建库建表（5 个）
├── 数据迁移                       # 迁移脚本（5 个）
├── 集成测试                       # 测试脚本（20+ 个）
├── 字典与配置                     # 配置脚本（5 个）
└── 工具脚本                       # 实用工具（10+ 个）

根目录                           # 运维脚本（~20 个）
├── Docker 管理                     # 容器编排（7 个）
├── TimescaleDB 管理               # 时序数据库（7 个）
├── 生产环境管理                   # 运维脚本（6 个）
└── 开发启动脚本                   # 开发工具（4 个）
```

## 🔧 **最佳实践**

### 1. 执行 SQL 脚本的最佳实践

```bash
# ✅ 推荐：在事务中执行（可回滚）
psql -U logix -d logix -c "BEGIN;" -f your-script.sql -c "COMMIT;"

# ✅ 推荐：先备份再执行
pg_dump -U logix logix > backup_before_change.sql
psql -U logix -d logix -f your-script.sql

# ❌ 避免：直接执行无保护
psql -f your-script.sql  # 危险！
```

### 2. 测试脚本使用规范

```bash
# ✅ 推荐：使用 npx ts-node（无需全局安装）
npx ts-node backend/scripts/test-xxx.ts

# ✅ 推荐：指定测试文件
npx ts-node --files backend/scripts/test-xxx.ts

# ❌ 避免：编译后执行
tsc && node dist/test-xxx.js  # 麻烦且易错
```

### 3. 跨平台脚本兼容

```bash
# ✅ 推荐：提供多平台版本
code-quality-check.bat   # Windows
code-quality-check.sh    # Linux/Mac

# ✅ 推荐：使用 Node.js/TypeScript（跨平台）
test-api.js              # 任何平台

# ❌ 避免：仅支持单一平台
only-windows.bat         # Mac/Linux 无法使用
```

### 高危操作警告

以下脚本会**删除或修改数据**，使用前务必备份：

```
🔴 高危脚本列表:
- cleanup-*.sql          # 清理数据
- fix-*.sql             # 修复数据
- verify-data.ps1       # 可能修改数据
- tsdb-clean.bat        # 清理时序数据库

**使用流程**:
1. 备份当前数据
2. 在测试环境验证
3. 获得授权批准
4. 在生产环境执行
5. 验证执行结果

### 数据保密要求

```
⚠️ 包含敏感信息的脚本:
- check-*.sql           # 可能包含业务数据
- query-*.sql           # 查询结果可能敏感

处理要求:
