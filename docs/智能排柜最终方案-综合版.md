# LogiX 智能排柜最终方案（综合版）

> **综合来源**：计划提柜日计算分析、数据库参数修改方案、Java 算法解读、智能排柜与五节点调度、优化提案  
> **版本**：v1.0 | 2026-03-17  
> **状态**：最终方案，作为后续开发唯一基准

---

## 一、方案总览

### 1.1 核心目标

将 LogiX 智能排柜从「满足约束」升级为「多约束条件下的成本优化」，与 Java/Python 外部脚本能力对齐。

### 1.2 综合结论

| 维度 | 当前 LogiX | 目标状态 |
|------|------------|----------|
| 提柜日计算 | 清关日 + 1，简单截断 | 多约束枚举 + 剪枝 + 成本评分 |
| 周末 | 未处理 | 跳过周六日 |
| 车队送还能力 | 仅送柜档期 | 送柜 + 还箱双档期 |
| 仓库产能 | 有检查 | 预检查 + 容量不足时提前失败 |
| 车队堆场 | 未区分 | has_yard 决定 Live/Drop |
| 成本优化 | 无 | 滞港费、堆场费、拖车费综合评分 |
| 三种方案 | 未实现 | 直接送仓 / 先提送堆场 / 滞港等仓 |

## 三、算法与计算逻辑（最终方案）

### 3.1 计算范式转变

**从「单向计算」改为「多约束枚举 + 剪枝 + 评分」**：

```
旧：清关日 + 1 → 提柜日 → 找仓库 → 卸柜日 → 还箱日
新：收集约束 → 枚举(提柜日 × 仓库) → 校验可行性 → 评分 → 取最优
```

### 3.2 硬约束（必须满足）

| 编号 | 约束 | 来源 |
|------|------|------|
| C1 | 提 ≥ 清关日 + 1 | 业务流程 |
| C2 | 提 ≤ last_free_date | 滞港费约束 |
| C3 | 提 ≥ ETA + 1 | 物理限制 |
| C4 | 提 ≥ 今天 | 时间限制 |
| C5 | 提 ≠ 周末 | 工作日限制 |
| C6 | 存在仓库卸柜容量 | 卸柜能力 |
| C7 | 存在卸柜日 ≥ 提 | 流程顺序 |
| C8 | 存在送柜日（提≤送≤卸） | 流程顺序 |
| C9 | 存在车队送还能力 | 还箱限制 |
| C10 | 还 ≤ 最晚还箱日 | 合同约束 |

### 3.3 软约束（优化目标）

| 编号 | 目标 | 说明 |
|------|------|------|
| S1 | 最小化滞港费 | 提 ≤ last_free_date |
| S2 | 最小化堆场费 | 有堆场时减少 yard_days |
| S3 | 最小化拖车费 | 选择价格最低车队 |
| S4 | 优先自营仓 | CA-S003 > CA-P003 |

### 3.4 卸柜方式决策

| 车队 has_yard | 仓库当日产能 | 卸柜方式 |
|---------------|--------------|----------|
| true | 有 | 可选 Live 或 Drop |
| true | 无 | Drop（提 < 送 = 卸） |
| false | 有 | Live（提 = 送 = 卸） |
| false | 无 | 顺延至有产能日 |

### 3.5 周末跳过

- 所有计划日期（提/送/卸/还）均需跳过周六日
- 配置项：`dict_scheduling_config.skip_weekends`，默认 true

## 五、核心接口设计

### 5.1 约束收集

```typescript
interface PickupConstraints {
  earliestPossiblePickup: Date;   // max(清关日+1, ETA+1, 今天)
  latestPossiblePickup: Date;    // min(last_free_date, 最晚还箱日-周期)
  candidateWarehouses: Warehouse[];
  skipWeekends: boolean;
  fleetReturnCapacity: Map<string, Map<string, number>>;  // fleetCode -> date -> remaining
}
```

### 5.2 计划结果

```typescript
interface OptimalPlan {
  pickup: Date;
  delivery: Date;
  unload: Date;
  return: Date;
  warehouse: string;
  truckingCompany: string;
  unloadMode: 'Live load' | 'Drop off';
  score: number;
}
```

### 5.3 API 保持不变

| 接口 | 用途 |
|------|------|
| POST /containers/batch-schedule | 批量排产 |
| POST /containers/:id/schedule-preview | 单柜模拟 |
| PATCH /containers/:id/schedule | 手工拖拽更新 |

## 六、测试用例（必须覆盖）

| 场景 | 输入 | 期望输出 |
|------|------|----------|
| 正常 | 清关日 3/21，last_free 3/28 | 提柜 3/22 |

| 周末冲突 | 清关日 3/20（周五），+1=周六 | 提柜 3/23（周一） |
| 产能不足 | 仓库 A 未来 3 天满 | 选仓库 B 或第 4 天 |
| 无可行解 | 所有仓库未来 30 天满 | 明确错误「仓库产能不足」 |
| 还箱档期满 | 还箱日车队无能力 | 顺延还箱日或换车队 |
