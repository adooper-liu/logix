# 排产预览批量操作栏位置调整

## 📋 优化概述

将**批量操作栏**从 TAB 区域内移动到**卡片标题栏下方**，采用**标签 + 按钮组**的形式，并内联到标题栏，体现"选择/取消 → 确认保存/放弃"的操作流程。

## 🎯 优化目标

### 问题

- 批量操作栏隐藏在 TAB 内部，用户不易发现
- 预览模式下，选择状态提示不够醒目
- 操作路径过长，需要先看 TAB 才能看到操作按钮
- 操作流程不清晰，选择→确认的逻辑未体现
- 冗余提示浪费空间
- 选择操作区独立成行，与确认按钮分离

### 目标

- **提升可见性**：批量操作栏移到标题栏下方，第一眼就能看到
- **强化提示**：选择状态以标签形式实时显示
- **缩短路径**：减少操作步骤，提升效率
- **清晰流程**：体现"选择 → 确认"的操作逻辑
- **简洁界面**：移除冗余提示，节省空间
- **内联布局**：选择按钮与确认按钮在同一行

## ✨ 调整内容

### 1. 批量操作栏新位置

```
排产结果卡片结构：
┌─────────────────────────────────────────────────────────┐
│  标题 + 标签 + [已选] [全选] [取消] │ [确认保存] [放弃]  │ ← 卡片头部
├─────────────────────────────────────────────────────────┤
│  📊 统计徽章（总柜数、成功、失败、成功率）                │
├─────────────────────────────────────────────────────────┤
│  TAB 过滤区（全部、成功、失败）                          │
│  ┌─────────────────────────────────────────────────┐    │
│  │  表格数据                                        │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

### 2. 视觉层次

**优化前：**

```
标题栏
  ├─ 标题 + 标签 + 确认/放弃按钮
选择操作区（独立容器）
  ├─ [已选 N 个] [全选成功项] [取消全选]
统计徽章
TAB 过滤区
  └─ 表格
```

**优化后：**

```
标题栏
  ├─ 标题 + 标签
  ├─ 选择操作区（内联）
  │   ├─ [已选 N 个] [全选成功项] [取消全选]
  └─ 确认操作区
      ├─ [确认保存 (N)] [放弃]
统计徽章
TAB 过滤区
  └─ 表格
```

## 🔧 技术实现

### 代码调整

### 1. 移动批量操作栏位置并内联到标题栏

```vue
<template #header>
  <div class="card-header-optimized">
    <!-- 标题 -->
  </div>

  <div class="header-right">
    <template v-if="isPreviewMode">
      <!-- ✅ 选择操作区（内联到标题栏） -->
      <div v-if="displayResults.length > 0" class="selection-actions-inline">
        <el-tag v-if="selectedPreviewContainers.length > 0" type="warning" size="small" effect="plain"> 已选 {{ selectedPreviewContainers.length }} 个 </el-tag>
        <el-button type="primary" size="small" plain @click="selectAllOnPage"> 全选成功项 </el-button>
        <el-button type="default" size="small" plain @click="clearSelection"> 取消全选 </el-button>
      </div>

      <!-- ✅ 确认操作区 -->
      <el-button type="success" size="small" @click="handleConfirmSave">
        <el-icon><Check /></el-icon> 确认保存 ({{ selectedPreviewContainers.length }})
      </el-button>
      <el-button type="info" size="small" plain @click="handleDiscardPreview">
        <el-icon><Close /></el-icon> 放弃
      </el-button>
    </template>
  </div>
</template>
```

## 🎨 操作流程设计

### 1. 视觉分组

```
标题栏（同一行）
┌──────────────────────────────────────────┐
│ [已选] [全选] [取消]  │  [确认] [放弃]   │
│    ↑ 选择操作区 ↑      ↑ 确认操作区 ↑    │
│         分隔线 →  │ ← 视觉分组           │
└──────────────────────────────────────────┘
         ↓
    选择要保存的
         ↓
    立即点击 [确认保存]
```

### 2. 色彩设计

**标题栏（同一行）：**

- 选择操作区：
  - 已选标签：`type="warning"`（橙色，醒目）
  - 全选按钮：`type="primary"` + `plain`（蓝色边框）
  - 取消按钮：`type="default"` + `plain`（灰色边框）
  - 分隔线：`#dcdfe6`（灰色，视觉分组）
- 确认操作区：
  - 确认保存：`type="success"`（绿色，强调）
  - 放弃：`type="info"` + `plain`（灰色，次要）

## 🎨 设计要点

### 1. 视觉层次清晰

- **第一层**：标题 + 核心操作（保存/放弃）
- **第二层**：选择操作区（已选标签 + 全选/取消按钮）← 新位置
- **第三层**：统计徽章（关键指标）
- **第四层**：TAB 过滤 + 表格

### 2. 操作流程明确

**选择 → 确认 两步走：**

```
第一步：选择（独立区域）
├─ 查看已选数量（橙色标签）
├─ 全选成功项（蓝色边框按钮）
└─ 取消全选（灰色边框按钮）

第二步：确认（右上角按钮）
├─ 确认保存（绿色按钮，强调）
└─ 放弃（灰色按钮，次要）
```

### 3. 色彩搭配

- **容器背景**：`#f5f7fa`（浅灰色，简洁中性）
- **容器边框**：`#e4e7ed`（灰色边框）
- **已选标签**：橙色（`type="warning"`，醒目）
- **全选按钮**：蓝色边框（`plain` 风格）
- **确认保存**：绿色（`type="success"`，强调操作）

### 4. 间距设计

- **容器边距**：`12px 0`（上下间距）
- **元素间距**：`8px`（标签和按钮之间）
- **内边距**：`12px 16px`（容器内边距）
