## 一、需求分析

### 1.1 业务目标

**成本优化功能**的核心目标：
1. ✅ 基于**原计划日期**（如 3-28 提柜）计算当前成本
2. ✅ 探索前后 7 天（共 15 天）的不同提柜日期方案
3. ✅ 对每个候选日期计算**预测成本**（包含滞港费、滞箱费等）
4. ✅ 比较所有方案，找到**成本最低**的方案
5. ✅ 展示**原方案 vs 优化方案**的费用对比和明细

### 1.2 关键数据流

```
原计划日期（3-28） → 计算原方案成本 → 探索前后 7 天 → 计算各方案成本 → 比较 → 返回最优
     ↓                      ↓                    ↓                ↓           ↓
  basePickupDate    originalCost      candidates[]     evaluated[]    optimal
```

### 1.3 预期行为

**输入**:
- 原计划提柜日：2026-03-28
- 免费期截止：2026-02-24（已过期）
- 当前日期：2026-03-27

**预期输出**:
```
📊 费用对比
┌──────────┬──────────────┬──────────────┐
│          │ 原方案 (3-28) │ 优化后 (3-30) │
├──────────┼──────────────┼──────────────┤
│ 滞港费   │ $2,200       │ $2,400       │ ← 应该递增
│ 运输费   │ $700         │ $700         │
│ 合计     │ $2,900       │ $3,100       │ ← 应该不同
└────────────────────────┴──────────────┘
```

```typescript
async selectBestOption(options: UnloadOption[]): Promise<{
  option: UnloadOption;
  costBreakdown: CostBreakdown;
}> {
  // 1. 评估所有方案的成本
  const evaluatedOptions = await Promise.all(
    options.map(async (option) => ({
      option,
      costBreakdown: await this.evaluateTotalCost(option) // ✅ 调用成本评估
    }))
  );
  
  // 2. 选择成本最低的方案
  const best = evaluatedOptions.sort(
    (a, b) => (a.option.totalCost || 0) - (b.option.totalCost || 0)
  )[0];
  
  return best;
}
```

**关键点**:
- ✅ 为每个 `option` 调用 `evaluateTotalCost`
- ✅ `evaluateTotalCost` 调用 `demurrageService.calculateTotalCost`
- ✅ `calculateTotalCost` 接收 `plannedDates` 参数
