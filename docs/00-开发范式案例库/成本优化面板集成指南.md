## 🎯 功能说明

### 用户操作流程

1. **选择货柜**
   - 在预览排产结果表格中勾选货柜
   - 或使用"全选"按钮选择所有成功的货柜

2. **点击批量优化**
   - 点击成本优化面板中的"🚀 批量优化"按钮
   - 系统自动计算每个货柜的最优方案

3. **查看优化结果**
   - 表格展示每个货柜的原始成本、优化后成本、节省金额
   - 显示建议的提柜日期
   - 标记是否需要优化

4. **应用优化**
   - **一键应用**: 点击"✅ 一键应用所有优化"批量应用
   - **单个应用**: 点击每个货柜右侧的"应用"按钮

5. **性能监控**
   - 底部显示性能指标：总柜数、处理耗时、平均耗时/柜

## ⚠️ 注意事项

### 1. 后端依赖

需要后端提供以下 API:

```typescript
POST /api/intelligent-scheduling/batch-optimize
Request: {
  containerNumbers: string[],
  options?: { forceRefresh?: boolean }
}
Response: {
  success: true,
  results: Array<{
    containerNumber: string,
    originalCost: number,
    optimizedCost: number,
    savings: number,
    suggestedPickupDate?: string,
    shouldOptimize: boolean
  }>,
  performance?: {
    totalContainers: number,
    totalTimeMs: number,
    avgTimePerContainer: number
  }
}
```

### 2. 数据源

组件使用 `selectedPreviewContainers` 作为数据源，这是用户在预览表格中选中的货柜。

**确保**: 
- 预览表格启用了 `selection-change` 事件
- `selectedPreviewContainers` 正确更新

### 3. 刷新逻辑

当前 `handleOptimizationApplied` 是空实现，如需刷新数据:

```typescript
const handleOptimizationApplied = async (containerNumber: string | 'all') => {

    await loadOverview() // 重新加载概览数据

    // 刷新单个货柜数据
    await refreshContainer(containerNumber)
  }
```
