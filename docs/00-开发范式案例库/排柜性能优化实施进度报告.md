## 🔧 关键技术点

### 1. SKILL 原则遵循

- **Specific**: 明确优化目标 (100 柜 < 15s)
- **Knowledge-driven**: 复用现有 Service 和工具
- **Incremental**: 分阶段实施 (后台方法 → UI 集成)
- **Leverage**:
  - 复用 `SchedulingCostOptimizerService.suggestOptimalUnloadDate()`
  - 复用现有的 Repository 和 Entity
  - 避免重新造轮子
- **Learning**: 记录性能日志，持续优化

### 2. 性能优化策略

- **缓存机制**: 仓库和车队信息跨批次共享
- **并发处理**: 批次内并发，批次间串行
- **分批处理**: 避免一次性加载过多数据
- **DB 查询优化**: 减少重复查询

### 3. 错误处理

- 单个货柜优化失败不影响其他货柜
- 详细的错误日志便于问题定位
- 返回 null 而非抛出异常 (上层统一处理)
