# 成本优化完整流程拆解与验证

## 📊 完整流程图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     成本优化完整流程（单柜）                              │
└─────────────────────────────────────────────────────────────────────────┘

Step 1: 初始化
  ├─ 输入参数：
  │   • containerNumber: 柜号
  │   • warehouse: 仓库信息
  │   • truckingCompany: 车队信息
  │   • basePickupDate: 基础提柜日（原计划）
  │   └─ lastFreeDate?: 免费期截止日（可选）
  │
  └─ 输出：无

Step 2: 获取货柜信息
  ├─ 查询数据库：
  │   • SELECT * FROM biz_containers WHERE container_number = ?
  │   └─ LEFT JOIN process_port_operations
  │
  └─ 输出：Container 对象（包含 portOperations）

Step 3: 计算免费期截止日
  ├─ 调用 DemurrageService.calculateForContainer()
  ├─ 优先级：
  │   1. DB: process_port_operations.last_free_date（最高优先级）
  │   2. 传入的 lastFreeDate 参数
  │   └─ 3. basePickupDate + 7 days（兜底）
  │
  └─ 输出：effectiveLastFreeDate (Date)

Step 4: 分类判断
  ├─ 调用 categorizeSingleContainer()
  ├─ 计算剩余免费天数：
  │   remainingDays = daysBetween(today, effectiveLastFreeDate)
  │
  └─ 输出：ContainerCategory
      • category: 'within_free_period' | 'overdue'
      • remainingDays: number
      └─ 其他元数据

Step 5: 计算原方案成本
  ├─ 构建当前方案：
  │   currentOption = {
  │     plannedPickupDate: basePickupDate,
  │     strategy: truckingCompany.hasYard ? 'Drop off' : 'Direct',
  │     isWithinFreePeriod: basePickupDate <= effectiveLastFreeDate
  │   }
  │
  ├─ 调用 evaluateTotalCost(currentOption)
  │   │
  │   ├─ 根据策略推导日期：
  │   │   Direct: 提=送=卸=basePickupDate, 还=basePickupDate
  │   │   Drop off: 提=basePickupDate, 卸=basePickupDate+2, 还=卸 +3
  │   │   └─ Expedited: 同 Direct
  │   │
  │   ├─ 调用 DemurrageService.calculateTotalCost()
  │   │   ├─ 滞港费 = f(提柜日，免费期，费率)
  │   │   ├─ 滞箱费 = f(提柜日，卸柜日，免费期，费率)
  │   │   ├─ 港口存储费 = f(提柜日，费率)
  │   │   ├─ 运输费 = f(距离，费率)
  │   │   └─ 外部堆场费 = f(堆存天数，费率)（仅 Drop off）
  │   │
  │   └─ 总成本 = Σ 各项费用
  │
  └─ 输出：originalCost (number), currentBreakdown (CostBreakdown)

Step 6: 选择优化策略
  ├─ 根据分类选择：
  │   • within_free_period → backward 策略
  │   │   searchStartOffset: 0 (从免费期截止日开始)
  │   │   searchEndOffset: -7 (往前 7 天)
  │   │   prioritizeZeroCost: true
  │   │   └─ allowSkipIfNoCapacity: true
  │   │
  │   └─ overdue → forward 策略
  │       searchStartOffset: 0 (从今天开始)
  │       searchEndOffset: 7 (往后 7 天)
  │       prioritizeZeroCost: false
  │       └─ allowSkipIfNoCapacity: false
  │
  └─ 输出：OptimizationStrategyConfig

Step 7: 生成智能搜索范围
  ├─ 调用 generateSearchRange()
  ├─ 约束条件（免费期内）：
  │   • date >= today（不能是过去）
  │   • date >= basePickupDate（不能早于原计划）
  │   • date != today（不能是当天，除非原计划就是今天）
  │   └─ date <= effectiveLastFreeDate（在免费期内）
  │
  ├─ 约束条件（已超期）：
  │   • date >= today（不能是过去）
  │   └─ 允许 date < basePickupDate（尽早处理）
  │
  └─ 输出：Date[] (搜索日期列表)

Step 8: 遍历候选日期
  ├─ FOR EACH candidateDate IN searchDates:
  │   │
  │   ├─ 跳过周末（如果配置了）
  │   │
  │   ├─ 检查仓库档期：
  │   │   isWarehouseAvailable(warehouse, candidateDate)
  │   │   └─ 查询 ExtWarehouseDailyOccupancy
  │   │
  │   ├─ 确定可用策略：
  │   │   strategies = ['Direct']
  │   │   if truckingCompany.hasYard → add 'Drop off'
  │   │   if candidateDate <= effectiveLastFreeDate → add 'Expedited'
  │   │
  │   ├─ FOR EACH strategy IN strategies:
  │   │   │
  │   │   ├─ 构建方案：
  │   │   │   option = {
  │   │   │     plannedPickupDate: candidateDate,
  │   │   │     strategy: strategy,
  │   │   │     isWithinFreePeriod: candidateDate <= effectiveLastFreeDate
  │   │   │   }
  │   │   │
  │   │   ├─ 调用 evaluateTotalCost(option)
  │   │   │   │
  │   │   │   ├─ 推导日期：
  │   │   │   │   Direct: 提=送=卸=candidateDate, 还=candidateDate
  │   │   │   │   Drop off: 提=candidateDate, 卸=candidateDate+2, 还=卸 +3
  │   │   │   └─ Expedited: 同 Direct
  │   │   │
  │   │   │   ├─ 计算费用明细：
  │   │   │   │   demurrageCost = f(提柜日，免费期)
  │   │   │   │   detentionCost = f(提柜日，卸柜日，免费期)
  │   │   │   │   storageCost = f(提柜日，费率)
  │   │   │   │   transportationCost = f(距离，费率)
  │   │   │   │   yardStorageCost = f(堆存天数，费率) (仅 Drop off)
  │   │   │   └─ handlingCost = f(配置) (仅 Expedited)
  │   │   │
  │   │   │   └─ totalCost = Σ 各项费用
  │   │   │
  │   │   ├─ 如果 prioritizeZeroCost=true 且 totalCost > 0 → 跳过
  │   │   │
  │   │   └─ 添加到 candidates 数组
  │   │
  │   └─ 如果 prioritizeZeroCost=true 且找到零成本 → 提前退出
  │
  └─ 输出：candidates[]
      [{
        pickupDate: Date,
        strategy: string,
        totalCost: number,
        breakdown: CostBreakdown
      }]

Step 9: 选择最优方案
  ├─ 如果 candidates.length === 0:
  │   └─ 返回原方案
  │
  ├─ 找到最低成本：
  │   optimalCandidate = min(candidates, by totalCost)
  │
  ├─ 计算节省：
  │   savings = originalCost - optimizedCost
  │   savingsPercent = savings / originalCost * 100
  │
  └─ 输出：optimalCandidate, savings, savingsPercent

Step 10: 返回 Top 3 方案
  ├─ 排序：candidates.sort(by totalCost)
  ├─ 截取：slice(0, 3)
  ├─ 映射：添加 savings 字段
  │
  └─ 输出：{
      suggestedPickupDate: Date,
      suggestedStrategy: string,
      originalCost: number,
      optimizedCost: number,
      savings: number,
      savingsPercent: number,
      alternatives: [
        {
          pickupDate: Date,
          strategy: string,
          totalCost: number,
          savings: number,
          breakdown: CostBreakdown
        }
      ]
    }
```

### Step 7: 生成搜索范围

**输入**:
```typescript
basePickupDate: 2026-03-28
effectiveLastFreeDate: 2026-03-27
strategy: {
  searchDirection: 'backward',
  searchStartOffset: 0,
  searchEndOffset: -7
}
category: 'within_free_period'
today: 2026-03-27
```

**处理过程**:
```typescript
// 从 lastFreeDate 开始往前找
for offset from 0 to -7:
  date = addDays(effectiveLastFreeDate, offset)
  
  // 约束检查
  if dateStr < todayStr: continue      // 过滤过去
  if dateStr < basePickupDateStr: continue  // 过滤早于原计划
  if dateStr === todayStr && dateStr !== basePickupDateStr: continue  // 过滤当天
  
  dates.push(date)
```

**生成的日期**（假设）:
```
offset = 0:  2026-03-27 → ❌ 过滤（< basePickupDate 2026-03-28）
offset = -1: 2026-03-26 → ❌ 过滤（< basePickupDate）
offset = -2: 2026-03-25 → ❌ 过滤（< basePickupDate）
...
```

**问题发现**:
❌ **所有日期都被过滤了！** 因为 `effectiveLastFreeDate (2026-03-27) < basePickupDate (2026-03-28)`

**解决方案**:
```typescript
// 如果 lastFreeDate < basePickupDate，说明原计划已经超期
// 应该使用 forward 策略而不是 backward
if (effectiveLastFreeDate < basePickupDate) {
  strategy = OPTIMIZATION_STRATEGIES.OVERDUE; // forward
}
```

**修正后的日期**（使用 forward 策略）:
```
offset = 0:  2026-03-27 (today) → ✅ 保留
offset = 1:  2026-03-28 → ✅ 保留
offset = 2:  2026-03-29 → ✅ 保留
offset = 3:  2026-03-30 → ✅ 保留
offset = 4:  2026-03-31 → ✅ 保留
offset = 5:  2026-04-01 → ✅ 保留
offset = 6:  2026-04-02 → ✅ 保留
offset = 7:  2026-04-03 → ✅ 保留
```

**输出**:
```typescript
[
  2026-03-27, 2026-03-28, 2026-03-29, 2026-03-30,
  2026-03-31, 2026-04-01, 2026-04-02, 2026-04-03
]
```

**验证点**:
- ✅ 免费期内：backward 搜索
- ✅ 已超期：forward 搜索
- ✅ 约束条件正确应用
