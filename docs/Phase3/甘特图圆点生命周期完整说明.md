# 甘特图圆点生命周期完整说明

## 📋 概述

甘特图中的货柜圆点分为两种视觉类型：

- **实心主任务（Main Task）**：实线边框，填充状态颜色
- **虚线任务（Dashed Task）**：虚线边框，透明背景

这两种状态随着货柜在五个节点（清关、提柜、卸柜、还箱、查验）的流转而动态变化。

## 🔄 生命周期流转规则

### 核心判断函数

#### 1. `getDisplayItems(container)` - 获取显示项列表

**功能**：计算货柜在所有节点的显示状态

**逻辑**：

```typescript
const getDisplayItems = (container: any): GanttDisplayItem[] => {
  // 1. 优先使用后端 ganttDerived.nodes（权威数据源）
  if (gd?.nodes?.length) {
    return getDisplayItemsFromGanttDerived(container, gd, nodeStatus)
  }

  // 2. 本地推导（fallback）
  const displayItems: GanttDisplayItem[] = []
  let foundActive = false

  nodeOrder.forEach(nodeName => {
    const node = nodeStatus.nodes[nodeName]

    if (node.supplier && node.supplier !== '未指定') {
      if (node.status === 'active') {
        // 当前活动节点 → 主任务
        displayItems.push({ type: 'main', ... })
        foundActive = true
      } else if (!foundActive && node.status === 'pending') {
        // 尚未开始的节点 → 虚线任务
        displayItems.push({ type: 'dashed', ... })
      }
      // completed 状态的节点不显示
    }
  })

  return displayItems
}
```

**关键点**：

- 每个货柜在同一时刻，每个节点最多只有一个显示项
- 已经找到 `active` 节点后，后续的 `pending` 节点不再显示（避免剧透）
- `completed` 状态的节点完全不显示（已销毁）

#### 2. `getNodeDisplayType(container, nodeName)` - 判断节点显示类型

**功能**：判断指定节点应该显示为什么类型

**逻辑**：

```typescript
const getNodeDisplayType = (container: any, nodeName: string): "main" | "dashed" | null => {
  const nodeStatus = calculateNodeStatus(container);
  const node = nodeStatus.nodes[nodeName];

  // 特殊情况：查验节点
  if (nodeName === "查验") {
    if (node.status === "completed") return null;
    if (node.status === "active") return "main";
    if (node.status === "pending") return "dashed";
  }

  // 优先使用后端 ganttDerived
  const gd = container.ganttDerived;
  if (gd?.nodes?.length) {
    const gn = gd.nodes.find((x) => x.key === key);
    if (!gn || gn.completed) return null; // 已完成 → 不显示（销毁）
    if (gn.taskRole === "main") return "main"; // 主任务
    if (gn.taskRole === "dashed") return "dashed"; // 虚线任务
  }

  // 本地状态判断
  if (node.status === "completed") return null; // 已完成 → 不显示
  if (node.status === "active") return "main"; // 进行中 → 主任务
  if (node.status === "pending") return "dashed"; // 计划中 → 虚线任务

  return null;
};
```

#### 3. `isMainTask(container, date)` - 判断是否为主任务

**功能**：判断货柜在指定日期是否显示为主任务圆点

**逻辑**：

```typescript
const isMainTask = (container: any, date: Date): boolean => {
  const displayItems = getDisplayItems(container);
  const targetDate = new Date(date);

  for (const item of displayItems) {
    if (item.type === "main" && item.plannedDate) {
      const itemDate = new Date(item.plannedDate);
      if (dayjs(itemDate).isSame(targetDate, "day")) {
        return true; // 日期匹配且为主任务
      }
    }
  }
  return false;
};
```

#### 4. `isDashedTask(container, date)` - 判断是否为虚线任务

**功能**：判断货柜在指定日期是否显示为虚线任务圆点

**逻辑**：

```typescript
const isDashedTask = (container: any, date: Date): boolean => {
  const displayItems = getDisplayItems(container);
  const targetDate = new Date(date);

  for (const item of displayItems) {
    if (item.type === "dashed" && item.plannedDate) {
      const itemDate = new Date(item.plannedDate);
      if (dayjs(itemDate).isSame(targetDate, "day")) {
        return true; // 日期匹配且为虚线任务
      }
    }
  }
  return false;
};
```

## 🎯 关键设计原则

### 1. 生成（Creation）

**何时生成圆点**：

- 当节点有供应商分配（`supplier !== '未指定'`）
- 且节点状态为 `active` 或 `pending`
- 且有计划日期（`plannedDate`）

**生成逻辑**：

```typescript
if (node.supplier && node.supplier !== '未指定') {
  if (node.status === 'active') {
    displayItems.push({ type: 'main', plannedDate: node.plannedDate, ... })
  } else if (!foundActive && node.status === 'pending') {
    displayItems.push({ type: 'dashed', plannedDate: node.plannedDate, ... })
  }
}
```

### 2. 移动（Movement）

**如何移动圆点**：

- 用户拖拽圆点到新的日期格子
- 调用 `handleDrop(date)` 更新计划日期
- 重新计算 `getDisplayItems()`，圆点在新日期重新渲染

**拖拽处理**：

```typescript
const handleDrop = (date: Date) => {
  const container = draggingContainer.value;
  const newDate = dayjs(dragOverDate.value).format("YYYY-MM-DD");

  // 更新计划日期字段
  const updateField = "plannedPickupDate"; // 或其他字段
  await containerService.updateSchedule(container.containerNumber, {
    [updateField]: newDate,
  });

  // 重新加载数据，圆点会在新位置重新生成
  await loadData();
};
```

### 3. 销毁（Destruction）

**何时销毁圆点**：

- 当节点状态变为 `completed`（已完成）
- 或者节点没有供应商分配（`supplier === '未指定'`）

**销毁逻辑**：

```typescript
// getNodeDisplayType 返回 null，不渲染圆点
if (node.status === "completed") return null;

// getDisplayItems 不添加到显示列表
// completed 状态的节点不会出现在 displayItems 中
```

**重要**：

- **销毁是永久性的**，除非回滚状态
- 已还箱的货柜，所有节点都已完成，所以所有圆点都被销毁
- 这是为什么你看不到已还箱货柜圆点的原因

## 🐛 常见问题诊断

### Q1: "已经还箱了，为什么看不到圆点？"

**答案**：这是正确的！

**原因**：

- 还箱完成后，`还箱` 节点状态变为 `completed`
- `getDisplayItems()` 不返回已完成节点的显示项
- 所有圆点都被销毁，从甘特图中移除

**验证方法**：

```javascript
// 检查容器状态
console.log(container.logisticsStatus); // 应该是 'returned_empty'

// 检查节点状态
const nodeStatus = calculateNodeStatus(container);
console.log(nodeStatus.nodes.还箱.status); // 应该是 'completed'

// 检查显示项
const items = getDisplayItems(container);
console.log(items); // 应该是空数组 []
```

### Q2: "为什么有些节点显示虚线，有些显示实线？"

**答案**：取决于节点状态

**判断逻辑**：

- `active`（进行中）→ 实心主任务
- `pending`（计划中）→ 虚线任务
- `completed`（已完成）→ 不显示（销毁）

### Q3: "圆点应该在哪个日期显示？"

**答案**：由 `plannedDate` 决定

**匹配逻辑**：

```typescript
// isMainTask / isDashedTask 都会检查日期匹配
const itemDate = new Date(item.plannedDate);
const targetDate = new Date(date);

if (dayjs(itemDate).isSame(targetDate, "day")) {
  return true; // 在这个日期格子显示圆点
}
```

## 📝 总结

### 圆点生命周期三阶段

| 阶段     | 触发条件                                          | 视觉表现                               | 技术实现                         |
| -------- | ------------------------------------------------- | -------------------------------------- | -------------------------------- |
| **生成** | 节点有供应商 + 有计划日期 + 状态为 active/pending | 实心圆点（主任务）或虚线圆点（计划中） | `getDisplayItems()` 添加显示项   |
| **移动** | 用户拖拽圆点到新日期                              | 圆点从旧日期消失，在新日期出现         | `handleDrop()` 更新日期字段      |
| **销毁** | 节点状态变为 completed                            | 圆点完全消失                           | `getDisplayItems()` 不返回显示项 |

### 核心设计哲学

1. **单一真相源**：优先使用后端 `ganttDerived.nodes`
2. **渐进式披露**：只显示当前和未来的任务，不剧透已完成任务
3. **视觉清晰**：实线=进行中，虚线=计划中，消失=已完成

### 对于"已还箱货柜看不到圆点"的结论

✅ **这是正确的行为！**

- 已还箱 = 所有节点都已完成 = 所有圆点都已销毁
- 如果需要在历史视图中查看已完成的货柜，应该使用筛选器或单独的历史查询功能
- 甘特图的设计目的是展示**正在进行和计划中**的任务流
