## 🐛 常见问题排查

### 问题 1: 日历不显示

**症状**: 页面空白或报错

**解决方案**:
```bash
# 1. 检查依赖是否安装
npm list @fullcalendar/vue3

# 2. 重新安装依赖
rm -rf node_modules package-lock.json
npm install

# 3. 重启开发服务器
npm run dev
```

### 问题 2: TypeScript 类型错误

**症状**: IDE 报红色下划线

**解决方案**: 创建类型声明文件

```typescript
// src/types/fullcalendar.d.ts
declare module '@fullcalendar/vue3' {
  const Calendar: any
  export default Calendar
}

declare module '@fullcalendar/daygrid' {
  const plugin: any
  export default plugin
}

declare module '@fullcalendar/interaction' {
  const plugin: any
  export default plugin
}
```

### 问题 3: 样式错乱

**症状**: 日历显示不正常

**解决方案**: 确保样式作用域正确

```scss
// 在组件样式中使用 :deep()
.calendar-capacity-view {
  :deep(.fc) {
    all: initial;
    font-family: inherit;
  }
}
```

### 问题 4: API 请求失败

**症状**: 控制台报 404 错误

**解决方案**: 
1. 检查后端服务是否启动
2. 检查 API 路径是否正确
3. 使用 Mock 数据先测试前端功能

## 📊 性能优化建议

### 1. 按需加载

对于大日期范围，建议分页加载：

```typescript
const loadCapacityData = async () => {
  // 只加载当前月份
  const start = dayjs().startOf('month').format('YYYY-MM-DD')
  const end = dayjs().endOf('month').format('YYYY-MM-DD')
  
  const response = await api.get(`/api/capacity/range?start=${start}&end=${end}`)
  // ...
}
```

### 2. 防抖处理

避免频繁刷新：

```typescript
import { debounce } from 'lodash-es'

const debouncedRefresh = debounce(() => {
  loadCapacityData()
}, 300)
```

### 3. 缓存数据

使用 Pinia 缓存：

```typescript
// stores/capacity.ts
import { defineStore } from 'pinia'

export const useCapacityStore = defineStore('capacity', {
  state: () => ({
    capacityCache: new Map<string, any>()
  }),
  
  actions: {
    setCapacity(date: string, data: any) {
      this.capacityCache.set(date, data)
    },
    
    getCapacity(date: string) {
      return this.capacityCache.get(date)
    }
  }
})
```
