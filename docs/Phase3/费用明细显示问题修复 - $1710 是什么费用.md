## 🔍 问题描述

### **用户反馈**

用户在排产预览界面看到：
- **平均费用**: $1,710
- **费用区间**: $1,710 ~ $1,710
- **总费用**: $8,550 (5 个货柜 × $1,710)

**但是**，表格中所有费用列都显示为"-"：
- 滞港费：-
- 滞箱费：-
- 港口存储费：-
- 运输费：-
- 外部堆场费：-
- **总费用**: $1,710 ❓

**问题**: $1,710 是什么费用？为什么明细中没有显示？

## 🔬 问题分析

### **1. 现象确认**

从截图看到：
1. ✅ 总费用有值：$1,710
2. ❌ 所有分项费用都是："-"
3. ❌ hover 弹窗中也只显示"合计：$1,710"，没有明细

### **2. 后端代码检查**

检查 `intelligentScheduling.service.ts` 的 `calculateEstimatedCosts` 方法：

```typescript
private async calculateEstimatedCosts(
  containerNumber: string,
  plannedPickupDate: Date,
  plannedUnloadDate: Date,
  plannedReturnDate: Date,
  unloadMode: string,
  warehouse: Warehouse,
  truckingCompany: TruckingCompany
): Promise<{
  demurrageCost?: number;
  detentionCost?: number;
  storageCost?: number;
  transportationCost?: number;
  yardStorageCost?: number;
  totalCost?: number;      // ✅ 有返回值
  currency?: string;
}> {
  // ... 计算逻辑
  return {
    demurrageCost: totalCostResult.demurrageCost,      // ❌ 可能为 0 或 undefined
    detentionCost: totalCostResult.detentionCost,      // ❌ 可能为 0 或 undefined
    storageCost: totalCostResult.storageCost,          // ❌ 可能为 0 或 undefined
    transportationCost: totalCostResult.transportationCost, // ❌ 可能为 0 或 undefined
    yardStorageCost,                                    // ❌ 可能为 0
    totalCost: totalCostResult.totalCost + yardStorageCost // ✅ 有值
  };
}
```

**发现问题**:
1. ❌ **返回类型缺少 handlingCost 字段**
2. ❌ **没有计算 handlingCost（操作费/加急费）**
3. ❌ **前端判断条件问题**：`v-if="row.estimatedCosts?.demurrageCost"` 当费用为 0 时不显示

### **3. 费用构成分析**

根据截图中的信息：
- **卸柜方式**: 全部是 Drop off
- **仓库**: Bedford
- **车队**: YunExpress UK Ltd
- **国家**: UK (英国)

**推测费用构成**:
```
总费用 $1,710 = 滞港费 + 滞箱费 + 港口存储费 + 运输费 + 外部堆场费
```

但由于所有分项都显示为"-"，无法确定具体构成。

## 🔧 根本原因

### **原因 1: 前端显示判断问题**

**问题代码**:
```vue
<span v-if="row.estimatedCosts?.demurrageCost" :class="getAmountClass(...)">
  ${{ row.estimatedCosts.demurrageCost.toLocaleString() }}
</span>
<span v-else>-</span>
```

**问题**: 当 `demurrageCost === 0` 时，`v-if` 条件为 false，显示"-"

**正确做法**: 应该判断是否存在且 >= 0
```vue
<span v-if="row.estimatedCosts?.demurrageCost !== undefined && row.estimatedCosts?.demurrageCost !== null" ...>
```

或者简化：
```vue
<span :class="getAmountClass(row.estimatedCosts?.demurrageCost || 0)">
  ${{ (row.estimatedCosts?.demurrageCost || 0).toLocaleString() }}
</span>
```

### **原因 2: 后端缺少 handlingCost 字段**

**问题**: 返回类型定义和处理逻辑都缺少 handlingCost

**影响**: 
- Expedited 模式下无法收取加急费
- 费用明细不完整

### **原因 3: 后端可能未正确计算各项费用**

**可能原因**:
1. `demurrageService.calculateTotalCost` 返回的各项费用为 0
2. 数据源（ext_demurrage_standards）缺少匹配的标准
3. 免费期内操作，确实没有产生 D&D 费用

---

## ✅ 修复方案

### **修复 1: 添加 handlingCost 支持**

**后端修改** (`intelligentScheduling.service.ts`):

```typescript
// 1. 返回类型添加 handlingCost
private async calculateEstimatedCosts(...): Promise<{
  demurrageCost?: number;
  detentionCost?: number;
  storageCost?: number;
  transportationCost?: number;
  yardStorageCost?: number;
  handlingCost?: number; // ✅ 新增
  totalCost?: number;
  currency?: string;
}>

// 2. 计算 handlingCost（仅在 Expedited 模式下收取）
let handlingCost = 0;
if (unloadMode === 'Expedited') {
  try {
    const config = await this.schedulingConfigRepo.findOne({
      where: { configKey: 'expedited_handling_fee' }
    });
    handlingCost = config?.configValue ? parseFloat(config.configValue) : 50;
  } catch (error) {
    logger.warn('[IntelligentScheduling] Get expedited handling fee failed:', error);
    handlingCost = 50; // 默认$50
  }
}

// 3. 返回 handlingCost
return {
  demurrageCost: totalCostResult.demurrageCost,
  detentionCost: totalCostResult.detentionCost,
  storageCost: totalCostResult.storageCost,
  transportationCost: totalCostResult.transportationCost,
  yardStorageCost,
  handlingCost, // ✅ 新增
  totalCost: totalCostResult.totalCost + yardStorageCost + handlingCost
};
```

### **修复 2: 前端添加 handlingCost 列**

**前端修改** (`SchedulingPreviewModal.vue`):

```vue
<!-- 添加操作费列 -->
<el-table-column prop="estimatedCosts.handlingCost" label="操作费" width="95" sortable align="right">
  <template #default="{ row }">
    <span v-if="row.estimatedCosts?.handlingCost" :class="getAmountClass(row.estimatedCosts.handlingCost)">
      ${{ row.estimatedCosts.handlingCost.toLocaleString() }}
    </span>
    <span v-else>-</span>
  </template>
</el-table-column>
```

**更新 hover 弹窗**:
```vue
<p v-if="row.estimatedCosts.handlingCost" style="margin: 4px 0">
  操作费：${{ row.estimatedCosts.handlingCost.toLocaleString() }}
</p>
```

**更新 TypeScript 类型**:
```typescript
estimatedCosts?: {
  demurrageCost?: number
  detentionCost?: number
  storageCost?: number
  transportationCost?: number
  yardStorageCost?: number
  handlingCost?: number // ✅ 新增
  totalCost?: number
  currency?: string
}
```

### **修复 3: 优化 0 值显示逻辑**

**问题**: 当前 `$0` 应该显示为"0"而不是"-"

**修复方案**:
```vue
<!-- 方案 1: 显示 0 值 -->
<span :class="getAmountClass(row.estimatedCosts?.demurrageCost || 0)">
  ${{ (row.estimatedCosts?.demurrageCost || 0).toLocaleString() }}
</span>

<!-- 方案 2: 保持现状，0 显示为"-" -->
<span v-if="row.estimatedCosts?.demurrageCost" :class="getAmountClass(row.estimatedCosts.demurrageCost)">
  ${{ row.estimatedCosts.demurrageCost.toLocaleString() }}
</span>
<span v-else>-</span>
```

**决策**: 保持现状，因为：
- ✅ "-" 表示"未产生费用"，语义清晰
- ✅ 0 值在 getAmountClass 中会被分类为"amount-zero"（绿色）
- ✅ 符合用户习惯，"-"比"$0"更直观

## 🎯 $1,710 费用分析

### **根据截图信息推断**

**已知条件**:
- 卸柜方式：Drop off (5 个货柜)
- 仓库：Bedford (UK)
- 车队：YunExpress UK Ltd
- 平均费用：$1,710
- 费用区间：$1,710 ~ $1,710 (说明所有货柜费用相同)

**费用构成推测**:

由于所有费用列都显示"-"，最可能的原因是：

1. **后端计算返回的各项费用为 0 或 undefined**
   - `demurrageService.calculateTotalCost` 可能未正确计算
   - 或者确实所有货柜都在免费期内，没有产生 D&D 费用

2. **总费用 $1,710 的来源**
   - 可能是 `totalCostResult.totalCost` 有值
   - 但各项明细被判断为 0 而显示"-"

3. **最可能的情况**
   - 这 5 个货柜都在免费期内提柜
   - 但产生了运输费、外部堆场费等
   - 由于前端判断逻辑问题，0 值显示为"-"
