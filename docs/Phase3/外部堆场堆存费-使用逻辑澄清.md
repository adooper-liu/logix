#### **场景分析**

| 场景 | 卸柜方式 | 车队有堆场 | 提=卸？ | 实际使用堆场 | 是否收费 |
|------|---------|-----------|--------|-------------|---------|

**关键判断标准**: 
- `hasYard = true` 只表示**有能力提供**堆场服务
- 是否收费取决于**货柜是否实际在堆场存放**
- 即使选择 Drop off 模式，也可能**不使用堆场**（直接送仓）

## 🔍 核心判断逻辑

### **如何判断“实际使用堆场”？**

根据业务逻辑，应该满足以下**三个条件**才收费：

1. **必要条件**: Drop off 模式
2. **必要条件**: 车队有堆场 (`hasYard = true`)
3. **关键条件**: **提柜日 < 送仓日** (货柜先在堆场存放) ⭐ **关键修复**

**判断公式**:
```typescript
const actuallyUsedYard = 
  unloadMode === 'Drop off' &&           // 条件 1: Drop off 模式
  truckingCompany.hasYard &&             // 条件 2: 车队有堆场
  plannedPickupDate < plannedDeliveryDate; // 条件 3: 提 < 送（实际使用）⭐
```

**关键发现**:
- ❌ ~~`plannedPickupDate < plannedUnloadDate`~~ (旧逻辑，不准确)
- ✅ **`plannedPickupDate < plannedDeliveryDate`** (正确逻辑)
- 原因：送仓日才是货柜离开堆场的日期

## 🔧 修复方案

### **修复前代码**

**问题**: 缺少"提 < 卸"的判断

### **修复后代码**

```typescript
// ✅ intelligentScheduling.service.ts (新代码)
let yardStorageCost = 0;
if (unloadMode === 'Drop off' && truckingCompany.hasYard) {
  try {
    // 判断是否实际使用了堆场：提柜日 < 卸柜日
    // 如果 提=卸，说明货柜直接从港口送到仓库，未使用堆场
    const pickupDayStr = plannedPickupDate.toISOString().split('T')[0];
    const unloadDayStr = plannedUnloadDate.toISOString().split('T')[0];
    
    if (pickupDayStr !== unloadDayStr) {
      // ✅ 提 < 卸，说明货柜在堆场存放了
      // 预计堆场存放天数
      const yardStorageDays = dateTimeUtils.daysBetween(plannedUnloadDate, plannedReturnDate);
      
      // 获取货柜的目的港信息
      const container = await this.containerRepo.findOne({
        where: { containerNumber },
        relations: ['portOperations']
      });
      
      if (container) {
        const destPo = container.portOperations?.find((po: any) => po.portType === 'destination');
        const portCode = destPo?.portCode || 'USLAX';
        const countryCode = warehouse.country || 'US';
        
        // 从 TruckingPortMapping 获取堆场费率
        const truckingPortMapping = await this.truckingPortMappingRepo.findOne({
          where: {
            country: countryCode,
            portCode,
            truckingCompanyId: truckingCompany.companyCode,
            isActive: true
          }
        });
        
        if (truckingPortMapping) {
          // 计算外部堆场堆存费 = 每日费率 × 天数 + 操作费
          yardStorageCost = 
            (truckingPortMapping.standardRate || 0) * yardStorageDays + 
            (truckingPortMapping.yardOperationFee || 0);
        }
      }
    } // ← 关键：只有提≠卸时才计费
  } catch (error) {
    logger.error(`[IntelligentScheduling] Yard storage cost calculation failed:`, error);
    yardStorageCost = 0; // 失败不影响整体
  }
}
```

**改进**:
- ✅ 添加了 `pickupDayStr !== unloadDayStr` 判断
- ✅ 只有**提 < 卸**时才计算费用
- ✅ 符合"**使用了才收费**"的业务逻辑

## 🎯 业务逻辑总结

### **三层判断逻辑**

```
┌─────────────────────────────────┐
│ 第 1 层：是否 Drop off 模式？     │
│ - 否 → ❌ 不收费                │
│ - 是 → 进入下一层               │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│ 第 2 层：车队是否有堆场？         │
│ - 否 → ❌ 不收费                │
│ - 是 → 进入下一层               │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│ 第 3 层：提柜日 < 卸柜日？        │
│ - 否 → ❌ 不收费（未使用）      │
│ - 是 → ✅ 收费（实际使用）      │
└─────────────────────────────────┘
```

---

### **核心原则**

1. **有能力 ≠ 已使用** ✅
   - `hasYard = true` 只表示有能力
   - 不代表一定会使用

2. **使用了才收费** ✅
   - 只有**实际占用堆场资源**才收费
   - 直接送仓（提=卸）不收费

3. **精确计算** ✅
   - 通过日期比较判断使用情况
   - 避免误收费
