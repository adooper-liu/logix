# 前端货币显示与总费用修复

## 🎯 问题描述

### **问题 1：总费用显示错误**
- **现象**：总费用显示为 `$1710700.0`，实际应为 `$1,710.70`
- **原因**：数字格式化时缺少小数位控制
- **影响**：所有费用显示都不规范

### **问题 2：货币符号硬编码**
- **现象**：所有金额都显示 `$` 符号
- **原因**：代码中硬编码了 `$`，未根据国家/地区自动选择
- **影响**：英国仓库显示 `$` 而非 `£`，欧盟显示 `$` 而非 `€`

---

## ✅ 修复方案

### **1. 添加 formatCurrency 函数**

在 `SchedulingPreviewModal.vue` 中添加统一的货币格式化函数：

```typescript
// ✅ SKILL 规范：货币格式化函数
const formatCurrency = (amount: number, countryCode?: string): string => {
  const country = countryCode || 'US'
  
  // 根据国家获取货币符号
  const currencyMap: Record<string, { symbol: string; code: string }> = {
    US: { symbol: '$', code: 'USD' },
    GB: { symbol: '£', code: 'GBP' },
    EU: { symbol: '€', code: 'EUR' },
    DE: { symbol: '€', code: 'EUR' },
    FR: { symbol: '€', code: 'EUR' },
    IT: { symbol: '€', code: 'EUR' },
    ES: { symbol: '€', code: 'EUR' },
    CA: { symbol: 'C$', code: 'CAD' },
    AU: { symbol: 'A$', code: 'AUD' },
    JP: { symbol: '¥', code: 'JPY' },
    CN: { symbol: '¥', code: 'CNY' }
  }
  
  const currency = currencyMap[country] || { symbol: '$', code: 'USD' }
  
  // 格式化数字（带千分位，2 位小数）
  const formattedAmount = amount.toLocaleString('en-US', {
    minimumFractionDigits: 2,
    maximumFractionDigits: 2
  })
  
  return `${currency.symbol}${formattedAmount}`
}
```

## 🔧 技术细节

### **1. toLocaleString 参数**

```typescript
amount.toLocaleString('en-US', {
  minimumFractionDigits: 2, // 最少 2 位小数
  maximumFractionDigits: 2  // 最多 2 位小数
})
```

**效果**：
- `1710.7` → `1,710.70`
- `1710` → `1,710.00`
- `1710.123` → `1,710.12`

---

### **2. 降级处理**

```typescript
const country = countryCode || 'US' // 未传参时默认 US
const currency = currencyMap[country] || { symbol: '$', code: 'USD' } // 未知国家时默认 USD
```

**效果**：
- 未传 `countryCode` → 使用 `USD`
- 传未知国家代码 → 使用 `USD`
- 传已知国家代码 → 使用对应货币
