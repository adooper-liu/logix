## 🎯 **费用项详细说明**

### **1. 滞港费 (Demurrage Cost)**

**定义**: 货柜在港口免费期内未提柜产生的费用

**数据源**: 
- 表：`ext_demurrage_standards`
- 字段：`rate_per_day`, `tiers`, `free_days`
- 类型：`charge_type_code = 'DEMURRAGE'`

**计算公式**:
```typescript
demurrageCost = Σ(每日费率 × 超期天数) // 阶梯费率
```

**计算逻辑**:
- 起算日：ATA Dest Port（实际到港）或 ETA Dest Port（预测到港）
- 截止日：实际提柜日或计划提柜日
- 免费天数：根据标准确定
- 计费天数：超出免费天数的天数

**收取方**: 港口/码头

---

### **2. 滞箱费 (Detention Cost)**

**定义**: 货柜提柜后未在规定时间内还空箱产生的费用

**数据源**: 
- 表：`ext_demurrage_standards`
- 字段：`rate_per_day`, `tiers`, `free_days`
- 类型：`charge_type_code = 'DETENTION'`

**计算公式**:
```typescript
detentionCost = Σ(每日费率 × 超期天数) // 阶梯费率
```

**计算逻辑**:
- 起算日：实际提柜日或计划提柜日
- 截止日：实际还箱日或计划还箱日
- 免费天数：根据标准确定
- 计费天数：超出免费天数的天数

**收取方**: 港口/码头

---

### **3. 港口存储费 (Storage Cost)**

**定义**: 货柜在港口堆场存放产生的费用

**数据源**: 
- 表：`ext_demurrage_standards`
- 字段：`rate_per_day`, `tiers`, `free_days`
- 类型：`charge_type_code = 'STORAGE'`

**计算公式**:
```typescript
storageCost = Σ(每日费率 × 超期天数) // 阶梯费率
```

**计算逻辑**:
- 起算日：卸船日或 ATA Dest Port
- 截止日：实际提柜日或计划提柜日
- 免费天数：根据标准确定
- 计费天数：超出免费天数的天数

**收取方**: 港口/码头

---

### **4. D&D 合并费 (Combined Demurrage & Detention)**

**定义**: 部分标准将滞港费和滞箱费合并计算的统一费用

**数据源**: 
- 表：`ext_demurrage_standards`
- 字段：`rate_per_day`, `tiers`, `free_days`
- 类型：`charge_type_code = 'COMBINED_DEMURRAGE_DETENTION'`

**计算公式**:
```typescript
ddCombinedCost = Σ(每日费率 × 总超期天数) // 阶梯费率
```

**计算逻辑**:
- 起算日：ATA Dest Port
- 截止日：实际还箱日或计划还箱日
- 免费天数：根据标准确定
- 计费天数：超出免费天数的天数

**收取方**: 港口/码头

**注意**: 如果存在 D&D 合并费，则不再单独计算滞港费和滞箱费

---

### **5. 外部堆场堆存费 (Yard Storage Cost)** ⭐ **关键修复**

**定义**: Drop off 模式下，货柜在车队堆场存放产生的费用

**数据源**: 
- 表：`dict_trucking_port_mapping`
- 字段：`standard_rate` (每日费率), `yard_operation_fee` (一次性操作费)
- 条件：`yard_capacity > 0` (车队有堆场)

**计算公式**:
```typescript
yardStorageCost = standard_rate × yard_storage_days + yard_operation_fee
```

**判断逻辑** ⭐ **关键修复**:
```typescript
const shouldChargeYardStorage = 
  unloadMode === 'Drop off' &&           // 条件 1: Drop off 模式
  truckingCompany.hasYard &&             // 条件 2: 车队有堆场
  plannedPickupDate < plannedDeliveryDate; // 条件 3: 提柜日 < 送仓日 ⭐

if (shouldChargeYardStorage) {
  const yardStorageDays = dateTimeUtils.daysBetween(
    plannedPickupDate, 
    plannedDeliveryDate // Drop off: 送仓日 = 卸柜日
  );
  yardStorageCost = standardRate * yardStorageDays + yardOperationFee;
}
```

**场景对比**:

| 场景 | 提 | 送 | 卸 | 是否收费 | 原因 |
|------|---|---|---|---------|------|
| Live load | D1 | D1 | D1 | ❌ 不收费 | 直接送仓，不经堆场 |
| 假 Drop off | D1 | D1 | D1 | ❌ 不收费 | 直接送仓，未使用堆场 |
| 真 Drop off | D1 | D3 | D3 | ✅ 收费 | 在堆场放了 3 天 |

**收取方**: 拖车车队

**类别**: 运输环节附加费

---

### **6. 运输费 (Transportation Cost)**

**定义**: 从港口到仓库的拖车运输费用

**数据源**: 
- 表：`dict_trucking_port_mapping`
- 字段：`transport_fee`

**计算公式**:
```typescript
transportationCost = transport_fee // 固定费用
```

**收取方**: 拖车车队

**类别**: 运输费

---

### **7. 操作费/加急费 (Handling Cost)**

**定义**: Expedited 模式下的加急操作费用

**数据源**: 
- 表：`dict_scheduling_config`
- 配置键：`expedited_handling_fee`

**计算公式**:
```typescript
handlingCost = expedited_handling_fee // 固定费用，默认 $50
```

**收取方**: 仓库/车队

**类别**: 操作费

**条件**: 仅在 Expedited 模式下收取

## 🧮 **总成本计算公式**

```typescript
totalCost = 
  demurrageCost +        // 1. 滞港费
  detentionCost +        // 2. 滞箱费
  storageCost +          // 3. 港口存储费
  ddCombinedCost +       // 4. D&D 合并费（如有）
  yardStorageCost +      // 5. 外部堆场堆存费（条件性）
  transportationCost +   // 6. 运输费
  handlingCost           // 7. 操作费/加急费（条件性）
```

**注意**:
- D&D 合并费与滞港费、滞箱费互斥（不同时计算）
- 外部堆场堆存费仅在真 Drop off 模式下收取
- 操作费/加急费仅在 Expedited 模式下收取

### **schedulingCostOptimizer.service.ts**

```typescript
async evaluateTotalCost(option: UnloadOption): Promise<CostBreakdown> {
  const breakdown: CostBreakdown = {
    demurrageCost: 0,
    detentionCost: 0,
    storageCost: 0,
    yardStorageCost: 0,
    transportationCost: 0,
    handlingCost: 0,
    totalCost: 0
  };

  // 1. 使用统一的 calculateTotalCost 方法计算 D&D 费用和运输费
  const totalCostResult = await this.demurrageService.calculateTotalCost(option.containerNumber, {
    mode: 'forecast',
    plannedDates: {
      plannedPickupDate,
      plannedUnloadDate: option.unloadDate,
      plannedReturnDate
    },
    includeTransport: true,
    warehouse: option.warehouse,
    truckingCompany: option.truckingCompany,
    unloadMode: option.strategy === 'Drop off' ? 'Drop off' : 'Live load'
  });

  breakdown.demurrageCost = totalCostResult.demurrageCost;
  breakdown.detentionCost = totalCostResult.detentionCost;
  breakdown.storageCost = totalCostResult.storageCost; // 港口存储费
  breakdown.transportationCost = totalCostResult.transportationCost;
  
  // 2. 计算外部堆场堆存费（仅在 Drop off 模式、车队有堆场且实际使用时）⭐
  breakdown.yardStorageCost = 0;
  if (option.strategy === 'Drop off' && option.truckingCompany) {
    try {
      const hasYard = option.truckingCompany.hasYard || false;
      
      if (hasYard) {
        // ✅ 关键修复：判断是否实际使用了堆场：提柜日 < 送仓日
        const plannedDeliveryDate = option.unloadDate; // Drop off: 送 = 卸
        
        const pickupDayStr = option.unloadDate.toISOString().split('T')[0];
        const deliveryDayStr = plannedDeliveryDate.toISOString().split('T')[0];
        
        if (pickupDayStr !== deliveryDayStr) {
          // ✅ 提 < 送，说明货柜在堆场存放了
          const yardStorageDays = dateTimeUtils.daysBetween(option.unloadDate, plannedDeliveryDate);
          
          // ... 计算费用
        }
      }
    } catch (error) {
      log.warn(`[CostOptimizer] Yard storage cost calculation failed:`, error);
      breakdown.yardStorageCost = 0;
    }
  }

  // 3. 计算加急费
  breakdown.handlingCost = await this.getConfigNumber('expedited_handling_fee', 50);

  // 4. 总计
  breakdown.totalCost =
    breakdown.demurrageCost +
    breakdown.detentionCost +
    breakdown.storageCost +
    breakdown.yardStorageCost +
    breakdown.transportationCost +
```

## 🎯 **关键发现总结**

### **发现 1: 有两种完全不同的堆存费** ⭐

- **港口存储费** (`storageCost`): D&D 费用，来自 `ext_demurrage_standards`
- **外部堆场堆存费** (`yardStorageCost`): 运输附加费，来自 `dict_trucking_port_mapping`

两者数据源、收取方、计算方式完全不同！

---

### **发现 2: 送仓日是判断堆场使用的关键** ⭐

- ❌ ~~旧逻辑~~: 比较 `提柜日 vs 卸柜日`
- ✅ **新逻辑**: 比较 `提柜日 vs 送仓日`

**判断标准**:
- `提 ≠ 送` → ✅ 真 Drop off，收费
- `提 = 送` → ❌ 假 Drop off，不收费
