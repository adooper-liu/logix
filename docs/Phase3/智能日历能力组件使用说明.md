## 🔌 API 服务

### calendarCapacity.ts

**位置**: `frontend/src/services/calendarCapacity.ts`

**提供的接口**:

```typescript
// 获取日期范围内的能力数据
getCapacityRange(startDate, endDate)

// 设置手动能力
setManualCapacity(date, capacity, reason)

// 批量设置手动能力
batchSetManualCapacity({ startDate, endDate, capacity, applyDays, reason })

// 恢复日历规则
resetToCalendarRule(date)

// 获取手动设置列表
getManualSettingsList(page, pageSize)

// 更新手动设置
updateManualSetting(date, capacity, reason)

// 删除手动设置
deleteManualSetting(date)
```

**后端 API 需求**:

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/capacity/range` | GET | 获取日期范围内的能力数据 |
| `/api/capacity/manual` | POST | 设置单个日期的手动能力 |
| `/api/capacity/manual/batch` | POST | 批量设置手动能力 |
| `/api/capacity/manual/list` | GET | 获取手动设置列表 |
| `/api/capacity/manual/:date` | DELETE | 删除指定日期的手动设置 |
| `/api/capacity/manual/:date` | PUT | 更新指定日期的手动设置 |

## 🎯 使用场景

### 场景 1: 查看每日能力

1. 打开排产页面
2. 在右侧看到智能日历
3. 不同颜色显示每日能力状态
4. 鼠标悬停查看详细数字

### 场景 2: 设置春节假期

1. 点击"手动设置"按钮
2. 选择日期范围（如 2026-01-28 至 2026-02-04）
3. 设置能力值为 0
4. 原因填写："春节假期"
5. 点击"应用批量设置"
6. 系统自动设置范围内所有日期

### 场景 3: 只设置周末关闭

1. 点击"手动设置"按钮
2. 选择日期范围（如未来 3 个月）
3. 设置能力值为 0
4. 应用星期：只勾选"周六"、"周日"
5. 原因填写："常规周末休息"
6. 点击应用

### 场景 4: 临时修改某天能力

1. 在日历上点击目标日期
2. 弹出详情对话框
3. 修改"设置能力"输入框
4. 填写原因："临时增加产能"
5. 点击"保存设置"

### 场景 5: 恢复日历规则

1. 在日历上点击已手动设置的日期（蓝色）
2. 在详情对话框中点击"恢复日历规则"
3. 确认操作
4. 日期恢复为自动计算（根据周末/节假日规则）

## ⚠️ 注意事项

### 1. 依赖冲突

如果项目中已有 FullCalendar 旧版本，可能需要升级：

```bash
# 检查版本
npm list @fullcalendar/*

# 如果有冲突，先卸载旧版本
npm uninstall @fullcalendar/*

# 重新安装新版本
npm install @fullcalendar/vue3@latest \
  @fullcalendar/daygrid@latest \
  @fullcalendar/interaction@latest \
  @fullcalendar/core@latest
```

### 2. TypeScript 类型错误

如果遇到类型错误，添加类型声明：

```typescript
// src/types/fullcalendar.d.ts
declare module '@fullcalendar/vue3' {
  const Calendar: any
  export default Calendar
}

declare module '@fullcalendar/daygrid' {
  const plugin: any
  export default plugin
}

declare module '@fullcalendar/interaction' {
  const plugin: any
  export default plugin
}
```

### 3. 样式冲突

如果 FullCalendar 样式与其他组件冲突，使用 scoped 样式：

```scss
.calendar-capacity-view {
  :deep(.fc) {
    // 覆盖 FullCalendar 样式
  }
}
```

### 4. 性能优化

对于大量日期数据，建议：

- ✅ 使用虚拟滚动（已内置）
- ✅ 限制初始加载范围（默认 2 个月）
- ✅ 按需加载更多数据
- ✅ 防抖搜索

## 💡 常见问题

### Q1: 日历不显示？

**A**: 检查以下几点：
1. 是否安装了 FullCalendar 依赖
2. 是否正确导入组件
3. 浏览器控制台是否有错误信息
4. 网络请求是否成功（F12 查看 Network）

### Q2: 手动设置不生效？

**A**: 检查后端 API 是否实现：
1. `/api/capacity/manual` POST 端点
2. 数据库表是否存在
3. 权限配置是否正确

### Q3: 周末颜色不对？

**A**: 检查日期格式：
1. 确保日期字符串是 `YYYY-MM-DD` 格式
2. 检查时区处理是否正确
3. 验证 `isWeekend` 判断逻辑

### Q4: 批量设置很慢？

**A**: 优化建议：
1. 减少日期范围（如不超过 3 个月）
2. 后端添加批量操作索引
3. 使用异步任务处理
