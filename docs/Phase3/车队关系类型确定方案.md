**创建日期**: 2026-03-26

**实施状态**: ✅ **已完成**

## 📊 **问题背景**

### **原始问题**

在 Phase 2 的关系评分中，需要确定车队的"关系类型"来给予不同的加分。但原数据库表 `dict_trucking_companies` **没有**以下字段：

- ❌ `property_type` (类似仓库的自营/平台/第三方)
- ❌ `tier` (车队等级：核心/普通/临时)
- ❌ `partnership_level` (合作伙伴级别：战略/优先/一般)

### **原有实现的问题**

```typescript
// ❌ 问题：用"运力大小"代替"关系类型"不够准确
if ((trucking?.dailyCapacity || 0) >= 50) {
  score += 15; // 大运力车队加分
}
```

**缺陷**:
1. ⚠️ 小运力但长期合作的车队无法获得加分
2. ⚠️ 无法区分战略合作伙伴和普通车队
3. ⚠️ 业务含义不清晰

---

## 💡 **解决方案：添加 partnership_level 字段**

### **方案对比**

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **A: 添加字段** | 明确定义、易管理、可扩展 | 需要数据库迁移 | ⭐⭐⭐⭐⭐ |
| B: 动态计算 | 无需改库、自动计算 | 间接指标、不准确 | ⭐⭐⭐ |
| C: 独立配置表 | 灵活、可追溯 | 过度复杂、维护成本高 | ⭐ |

**决策**: 采用**方案 A**（添加 `partnership_level` 字段）

```typescript
/** 合作关系级别：STRATEGIC=战略合作，CORE=核心，NORMAL=普通，TEMPORARY=临时 */
@Column({ 
  type: 'varchar', 
  length: 20, 
  default: 'NORMAL', 
  name: 'partnership_level' 
})
partnershipLevel?: 'STRATEGIC' | 'CORE' | 'NORMAL' | 'TEMPORARY';
```

**类型定义**:
- `STRATEGIC`: 战略合作伙伴（最高级别）
- `CORE`: 核心合作伙伴（重点维护）
- `NORMAL`: 普通合作伙伴（日常合作）
- `TEMPORARY`: 临时合作伙伴（试用/偶尔）

```typescript
  'STRATEGIC': 30, // 战略合作伙伴 +30 分
  'CORE': 20,      // 核心合作伙伴 +20 分
```

## 📈 **完整的评分模型**

### **关系评分构成**

```
relationshipScore = 基础分 (50) 
                  + 合作频次加分 (0-20) 
                  + 关系级别加分 (0-30) 
                  + 运力规模加分 (0-15) 
                  + 服务质量加分 (5)

总分范围：50-120 分（实际会截断到 100 分）
```

### **详细评分维度**

| 维度 | 分值范围 | 计算方法 | 数据来源 |
|------|---------|---------|---------|
| **基础分** | 50 分 | 固定 | 代码设定 |
| **合作频次** | 0-20 分 | `min(合作数×2, 20)` | 过去 30 天合作货柜数 |
| **关系级别** | 0-30 分 | STRATEGIC=30, CORE=20, NORMAL=10, TEMPORARY=0 | `partnership_level` 字段 |
| **运力规模** | 0-15 分 | `dailyCapacity >= 50 ? 15 : 0` | `daily_capacity` 字段 |
| **服务质量** | 5 分 | 固定 | 代码设定 |

### **示例计算**

#### **示例 1: 战略合作伙伴**
```
车队 A: STRATEGIC 级别
- 基础分：50
- 过去 30 天合作：10 次 → 10×2 = 20 分
- 关系级别：STRATEGIC → +30 分
- 日产能：60 → +15 分
- 服务质量：+5 分
- 总分：50 + 20 + 30 + 15 + 5 = 120 → 截断到 100 分 ✅
```

#### **示例 2: 普通合作伙伴**
```
车队 B: NORMAL 级别
- 基础分：50
- 过去 30 天合作：2 次 → 2×2 = 4 分
- 关系级别：NORMAL → +10 分
- 日产能：30 → 0 分
- 服务质量：+5 分
- 总分：50 + 4 + 10 + 0 + 5 = 69 分
```

#### **示例 3: 临时合作伙伴**
```
车队 C: TEMPORARY 级别
- 基础分：50
- 过去 30 天合作：0 次 → 0 分
- 关系级别：TEMPORARY → +0 分
- 日产能：20 → 0 分
- 服务质量：+5 分
- 总分：50 + 0 + 0 + 0 + 5 = 55 分
```

## 🎯 **业务应用场景**

### **场景 1: 成本相近时优先合作伙伴**

```
车队 A: STRATEGIC, 成本$200, 关系分 100 → 综合 85 分
车队 B: NORMAL, 成本$195, 关系分 69 → 综合 78 分

结果：选择车队 A（战略合作伙伴）✅
业务价值：维护长期合作关系
```

### **场景 2: 淡季扶持核心车队**

```
总体货柜量少时：
- 给 CORE 级别以上车队分配保底量
- 即使成本高 5-10% 也维持合作
- 通过关系评分体现政策倾斜
```

### **场景 3: 旺季优先保障产能**

```
繁忙期模式（调整权重）：
- 成本权重：40% → 30% ↓
- 能力权重：30% → 50% ↑
- 关系权重：30% → 20% ↓

结果：优先选择有能力的车队，确保可执行性
```

## 📊 **管理建议**

### **车队等级评定标准（参考）**

| 级别 | 评定标准 | 占比建议 |
|------|---------|---------|
| **STRATEGIC** | 合作 3 年以上<br/>年运输量>500 柜<br/>准点率>95%<br/>零投诉 | 5-10% |
| **CORE** | 合作 1-3 年<br/>年运输量>200 柜<br/>准点率>90%<br/>投诉率<2% | 20-30% |
| **NORMAL** | 合作 6 个月 -1 年<br/>年运输量>50 柜<br/>准点率>85% | 40-60% |
| **TEMPORARY** | 合作<6 个月<br/>或运输量少<br/>或试用期 | 10-20% |

### **等级调整流程**

1. **定期评估**（每季度/每年）
2. **绩效分析**（运输量、准点率、投诉率）
3. **业务评审**（采购团队意见）
4. **系统更新**（修改 `partnership_level`）
5. **结果通知**（告知车队）

### **数据维护**

```sql
-- 查看各等级车队数量
SELECT 
  partnership_level,
  COUNT(*) as count,
  AVG(daily_capacity) as avg_capacity
FROM dict_trucking_companies
WHERE status = 'ACTIVE'
GROUP BY partnership_level;

-- 调整某个车队的等级
UPDATE dict_trucking_companies 
SET partnership_level = 'CORE' 
WHERE company_code = 'TRUCK_001';

-- 降级处理（连续 3 个月绩效不达标）
UPDATE dict_trucking_companies 
SET partnership_level = 'NORMAL' 
WHERE company_code = 'TRUCK_002' 
  AND partnership_level = 'CORE';
```
