## 核心功能

### 1. 成本优化评估

#### 功能描述

提供卸柜方案的成本评估和对比功能，帮助用户选择最优方案。

#### 支持的方案类型

- **Direct**: 直送模式
- **Drop off**:  Drop off 模式
- **Expedited**: 加急模式

#### 成本构成

- 运输成本
- 仓储成本
- 人工成本
- 加急处理费
- 外部存储费

### 2. 日历能力配置

#### 功能模块

1. **仓库能力配置**
   - 按国别分组
   - 按资源类型分组
   - 容量限制设置

2. **车队能力配置**
   - 车队 - 仓库映射
   - 每日容量限制
   - 特殊日期配置

3. **可视化界面**
   - 日历视图
   - 颜色标识
   - 拖拽操作

### 3. 智能推荐

#### 推荐算法

基于以下因素计算最优方案：

1. **成本最低** (权重 40%)
2. **时间最优** (权重 30%)
3. **资源利用率** (权重 20%)
4. **客户偏好** (权重 10%)

#### 推荐结果

```typescript
interface Recommendation {
  optionType: string;           // 方案类型
  warehouse: string;            // 推荐仓库
  truckingCompany: string;      // 推荐车队
  estimatedCost: number;        // 预估成本
  savings: number;              // 节省金额
  confidence: number;           // 置信度 (0-1)
}
```

1. **数据库版本**: PostgreSQL 14+
2. **Node.js 版本**: 16.x+
3. **TypeORM 版本**: 0.3.x

### 部署步骤

#### 1. 数据库迁移

```bash
# 执行配置项 SQL
psql -U logix_user -d logix_db -f migrations/add_cost_optimization_config.sql

# 验证配置项
SELECT config_key, config_value 
FROM dict_scheduling_config 
WHERE config_key LIKE 'transport_%' 
   OR config_key IN ('external_storage_daily_rate', 'expedited_handling_fee');
```


```bash

cd frontend

npm run build

# 部署到 Nginx
cp -r dist/* /var/www/logix/frontend/
```

### 常见问题

#### Q1: 为什么显示"暂无可用方案"？

**原因**: 
- 当前没有符合条件的仓库
- 仓库容量已满
- 日期不可用

**解决方案**:
- 调整卸柜日期
- 更换卸柜策略
- 检查仓库映射配置

#### Q2: 成本计算为 0 怎么办？

**检查点**:
1. 配置表是否有费率配置
2. 仓库 - 车队映射是否正确
3. 距离数据是否完整

**SQL 查询**:
```sql
-- 检查费率配置
SELECT * FROM dict_scheduling_config 
WHERE config_key LIKE '%rate%';

-- 检查映射关系
SELECT * FROM warehouse_trucking_mapping 
WHERE warehouse_id = '你的仓库 ID';
```

#### Q3: 如何自定义推荐算法权重？

**修改配置文件**:

```typescript
// backend/src/config/scheduling.config.ts
export const RECOMMENDATION_WEIGHTS = {
  cost: 0.4,           // 成本权重
  time: 0.3,           // 时间权重
  utilization: 0.2,    // 资源利用率
  preference: 0.1      // 客户偏好
};
```
