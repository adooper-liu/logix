# 日历能力组件改进报告 - 支持仓库/车队分别管理

**改进日期**: 2026-03-17  
**改进类型**: 功能增强  
**状态**: ✅ **已完成**

---

## 🎯 改进目标

### 原始问题

原来的 CalendarCapacityView 组件只有一个统一的能力日历，无法区分：
- ❌ 仓库的卸柜能力（Unload Capacity）
- ❌ 车队的提柜能力（Pickup Capacity）

这导致：
- 用户无法查看特定仓库的卸柜档期
- 用户无法查看特定车队的提柜档期
- 能力设置过于笼统，不符合实际业务需求

### 改进后效果

✅ **仓库能力管理**:
- 按仓库查看每日卸柜能力
- 支持选择具体仓库（如 WH001, WH002）
- 显示该仓库的卸柜档期占用情况

✅ **车队能力管理**:
- 按车队查看每日提柜能力
- 支持选择具体车队（如 TC001, TC002）
- 显示该车队的提柜档期占用情况

## 🔧 技术实现

### 新增状态变量

```typescript
// 资源类型切换
const resourceType = ref<'warehouse' | 'trucking'>('warehouse')

// 仓库和车队列表
const warehouses = ref<any[]>([])
const truckingCompanies = ref<any[]>([])

// 选中的资源
const selectedWarehouseCode = ref<string>('')
const selectedTruckingCode = ref<string>('')

// 默认能力
const defaultWarehouseCapacity = ref(10)  // 仓库卸柜能力
const defaultTruckingCapacity = ref(8)    // 车队提柜能力
```

### 新增计算属性

```typescript
// 选中的仓库名称
const selectedWarehouseName = computed(() => {
  const wh = warehouses.value.find(w => w.warehouseCode === selectedWarehouseCode.value)
  return wh ? wh.warehouseName : ''
})

// 选中的车队名称
const selectedTruckingName = computed(() => {
  const tc = truckingCompanies.value.find(t => t.truckingCompanyId === selectedTruckingCode.value)
  return tc ? tc.truckingCompanyName : ''
})
```

### 新增方法

#### 1. onResourceTypeChange

```typescript
const onResourceTypeChange = () => {
  // 切换时重置选中的资源
  if (resourceType.value === 'warehouse') {
    selectedTruckingCode.value = ''
  } else {
    selectedWarehouseCode.value = ''
  }
  loadCapacityData()
}
```

#### 2. loadResources

```typescript
const loadResources = async () => {
  try {
    // 加载仓库列表
    const warehouseResponse = await api.get('/api/scheduling/resources/warehouses')
    if (warehouseResponse.data.success) {
      warehouses.value = warehouseResponse.data.data
      if (warehouses.value.length > 0) {
        selectedWarehouseCode.value = warehouses.value[0].warehouseCode
      }
    }
    
    // 加载车队列表
    const truckingResponse = await api.get('/api/scheduling/resources/truckings')
    if (truckingResponse.data.success) {
      truckingCompanies.value = truckingResponse.data.data
      if (truckingCompanies.value.length > 0) {
        selectedTruckingCode.value = truckingCompanies.value[0].truckingCompanyId
      }
    }
  } catch (error: any) {
    console.error('加载资源列表失败:', error)
  }
}
```

#### 3. loadCapacityData (增强版)

```typescript
const loadCapacityData = async () => {
  try {
    const startDate = dayjs().format('YYYY-MM-DD')
    const endDate = dayjs().add(2, 'month').format('YYYY-MM-DD')
    
    // 根据资源类型和选中的资源构建查询参数
    let params: any = { start: startDate, end: endDate }
    
    if (resourceType.value === 'warehouse') {
      params.resourceType = 'warehouse'
      if (selectedWarehouseCode.value) {
        params.warehouseCode = selectedWarehouseCode.value
      }
    } else {
      params.resourceType = 'trucking'
      if (selectedTruckingCode.value) {
        params.truckingCompanyId = selectedTruckingCode.value
      }
    }
    
    // 构建查询字符串
    const queryString = Object.entries(params)
      .map(([key, value]) => `${key}=${encodeURIComponent(String(value))}`)
      .join('&')
    
    const response = await api.get(`/api/capacity/range?${queryString}`)
    
    // ... 处理响应
  } catch (error: any) {
    console.error('加载能力数据失败:', error)
    ElMessage.error('加载能力数据失败：' + error.message)
  }
}
```

## 💡 使用场景

### 场景 1: 仓库经理查看卸柜档期

**用户**: 仓库经理

**需求**: 查看未来一个月的卸柜能力占用情况

**操作**:
1. 选择"🏭 仓库"模式
2. 选择自己管理的仓库
3. 查看每日的 remaining/capacity
4. 发现某天 capacity=10, remaining=0 → 已满
5. 考虑是否需要增加临时卸货人手

### 场景 2: 车队调度查看提柜档期

**用户**: 车队调度员

**需求**: 查看车队的提柜能力分配

**操作**:
1. 选择"🚛 车队"模式
2. 选择自己的车队
3. 查看每天的提柜安排
4. 发现周末 capacity=0 → 休息日
5. 合理安排司机排班

### 场景 3: 运营总监对比资源

**用户**: 运营总监

**需求**: 对比仓库和车队的匹配度

**操作**:
1. 切换到 WH001，看到卸柜能力=15
2. 切换到 TC001，看到提柜能力=8
3. 发现仓库能力 > 车队能力
4. 决策：增加车队或减少仓库投入
5. 优化整体供应链效率

### 场景 4: 设置特殊时期能力

**用户**: 运营经理

**需求**: 设置春节期间能力调整

**操作**:
1. 选择"🏭 仓库"模式
2. 点击"手动设置"
3. 选择日期范围（春节 7 天）
4. 设置 capacity=0
5. 原因填写："春节放假"
6. 批量应用
7. 切换到"🚛 车队"模式重复上述操作
