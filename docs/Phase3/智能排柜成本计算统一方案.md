# 智能排柜成本计算逻辑一致性分析与统一方案

**文档版本**: v1.0  
**创建日期**: 2026-03-25  
**问题严重性**: 🚨 P0 - 严重影响数据一致性和用户信任  

---

## 📊 问题概述

当前系统中存在**三套独立的成本计算逻辑**，分别用于不同场景：

1. **货柜详情页** - 完整计算（`DemurrageService.calculateForContainer`）
2. **预览排产** - 简化预测（`IntelligentScheduling.calculateEstimatedCosts`）
3. **自动排柜优化** - 独立服务（`SchedulingCostOptimizer.evaluateTotalCost`）

**核心问题**：
- ❌ 三套逻辑各自为政，零复用
- ❌ 计算结果不一致（同一货柜在不同页面显示不同费用）
- ❌ 部分功能缺失（如自动排柜的滞箱费未实现）
- ❌ 维护成本高（修改一处需同步三处）

### 三、数据流不一致示例

**场景**：用户查看货柜 `ECMU1234567`，然后在排产页面预览

#### **货柜详情页**：
```
输入：containerNumber = "ECMU1234567"
调用：demurrageService.calculateForContainer()

处理流程：
1. ✅ 状态机判定：arrivedAtDestinationPort = true (已有 ATA)
2. ✅ 计算模式：actual
3. ✅ 匹配标准：找到 3 条（DEMURRAGE $200, DETENTION $150, STORAGE $100）
4. ✅ 累加计算：
   - 滞港费：$200 (实际区间：ATA→提柜日)
   - 滞箱费：$150 (实际区间：提柜日→还箱日)
   - 堆存费：$100 (标准匹配，阶梯费率)
   - D&D 合并：$0 (不适用)
   
输出：totalCost = $450
```

#### **预览排产页**：
```
输入：containerNumber = "ECMU1234567", proposedUnloadDate = "2026-03-28"
调用：intelligentSchedulingService.calculateEstimatedCosts()

处理流程：
1. ⚠️ 简化判断：hasAtaOrDischarge = true → actual 模式
2. ⚠️ 调用 predictDemurrageForUnloadDate()：
   - 只取第一个 DEMURRAGE 标准
   - 计算：$200 (单一标准，非累加)
3. ⚠️ 调用 predictDetentionForReturnDate()：
   - 需要实际提柜日（但这是预测！）
   - 返回：$0 (无法计算)
4. ❌ 自己实现堆存费：
   - 从 TruckingPortMapping 读取：$50/天 × 3 天 = $150
   
输出：totalCost = $350 (滞港$200 + 堆存$150)

差异：$450 - $350 = $100 (22%) ⚠️
```

**用户困惑**："为什么两个地方显示的费用不一样？哪个是对的？"

### 问题 2：预测 vs 实际的混淆

**核心矛盾**：
- **货柜详情**：计算**已发生**的费用（actual 模式）
- **预览排产**：预测**将发生**的费用（forecast 模式）
- **自动排柜**：比较多个**未来方案**的成本（forecast 模式）

**当前处理**：
- ✅ `DemurrageService` 已支持 forecast 模式（通过状态机智能切换）
- ⚠️ `predictDemurrageForUnloadDate()` 有自己的简化判断
- ❌ `calculateEstimatedCosts()` 混合使用 actual 和 forecast

**正确做法**：
```typescript
// 统一的预测接口
interface CostPredictionOptions {
  mode: 'actual' | 'forecast';
  plannedDates?: {
    plannedPickupDate: Date;
    plannedUnloadDate: Date;
    plannedReturnDate: Date;
  };
}

// 所有地方调用同一个方法
const cost = await unifiedCostService.calculateTotalCost(containerNumber, {
  mode: 'forecast',
  plannedDates: { plannedPickupDate, plannedUnloadDate, plannedReturnDate }
});
```

### 方案 A：增强现有 DemurrageService（推荐）

**核心思路**：将 `DemurrageService` 打造为**唯一的费用计算权威源**

#### Step 1: 扩展现有方法

```typescript:backend/src/services/demurrage.service.ts
export class DemurrageService {
  /**
   * 统一的总费用计算入口（新增）
   */
  async calculateTotalCost(
    containerNumber: string,
    options?: {
      /** 计算模式：actual=实际已发生，forecast=预测将发生 */
      mode?: 'actual' | 'forecast';
      /** 预测模式必需：计划的日期 */
      plannedDates?: {
        plannedPickupDate: Date;
        plannedUnloadDate: Date;
        plannedReturnDate: Date;
      };
      /** 是否包含运输费（默认 false） */
      includeTransport?: boolean;
      /** 仓库和车队信息（运输费必需） */
      warehouse?: Warehouse;
      truckingCompany?: TruckingCompany;
    }
  ): Promise<UnifiedCostBreakdown> {
    // 1. 调用现有的 calculateForContainer()
    const demurrageResult = await this.calculateForContainer(containerNumber, {
      freeDateWriteMode: 'none'
    });
    
    // 2. 提取各项费用
    const costs: UnifiedCostBreakdown = {
      demurrageCost: 0,
      detentionCost: 0,
      storageCost: 0,
      ddCombinedCost: 0,
      transportationCost: 0,
      totalCost: 0,
      currency: 'USD',
      calculationMode: demurrageResult.result?.calculationMode || 'forecast'
    };
    
    if (demurrageResult.result) {
      // 分类汇总
      demurrageResult.result.items.forEach(item => {
        if (isDemurrageCharge(item)) costs.demurrageCost += item.amount;
        if (isDetentionCharge(item)) costs.detentionCost += item.amount;
        if (isStorageCharge(item)) costs.storageCost += item.amount;
        if (isCombinedDemurrageDetention(item)) costs.ddCombinedCost += item.amount;
      });
      
      costs.currency = demurrageResult.result.currency;
    }
    
    // 3. 计算运输费（可选）
    if (options?.includeTransport && options.warehouse && options.truckingCompany) {
      costs.transportationCost = await this.calculateTransportationCost(
        containerNumber,
        options.warehouse,
        options.truckingCompany
      );
    }
    
    // 4. 总计
    costs.totalCost = 
      costs.demurrageCost +
      costs.detentionCost +
      costs.storageCost +
      costs.ddCombinedCost +
      costs.transportationCost;
    
    return costs;
  }

    // ...
  }
}
```

```typescript:backend/src/services/schedulingCostOptimizer.service.ts
async evaluateTotalCost(
  containerNumber: string,
  option: UnloadOption
): Promise<CostBreakdown> {
  // ✅ 直接调用统一的 calculateTotalCost
  const costs = await this.demurrageService.calculateTotalCost(containerNumber, {
    mode: 'forecast',
    plannedDates: {
      plannedPickupDate: option.plannedPickupDate,
      plannedUnloadDate: option.plannedUnloadDate,
      plannedReturnDate: option.plannedReturnDate
    },
    includeTransport: true,
    warehouse: option.warehouse,
    truckingCompany: option.truckingCompany
  });
  
  // ✅ 补充滞箱费（如果缺失）
  if (!costs.detentionCost && option.plannedReturnDate) {
    const detentionResult = await this.demurrageService.predictDetentionForReturnDate(
      containerNumber,
      option.plannedReturnDate
    );
    costs.detentionCost = detentionResult.detentionCost;
    costs.totalCost += costs.detentionCost;
  }
  
  return costs;
}
```

## 📝 经验教训

### 开发规范改进

1. **创建新服务时必须考虑复用**
   ```typescript
   // ❌ 错误：自己重新实现
   class NewService {
     private calculateCost() { /* 新实现 */ }
   }
   
   // ✅ 正确：复用现有服务
   class NewService {
     constructor(private demurrageService: DemurrageService) {}
     
     private calculateCost() {
       return this.demurrageService.calculateTotalCost(/* ... */);
     }
   }
   ```

2. **预测性计算必须标注模式**
   ```typescript
   interface CostOptions {
     mode: 'actual' | 'forecast'; // 必填！
     plannedDates?: { /* ... */ }; // forecast 模式必填
   }
   ```

3. **费用计算必须使用权威源**
   - ✅ 滞港费/滞箱费/堆存费：`ext_demurrage_standards`
   - ✅ 运输费：`trucking_port_mapping`
   - ❌ 禁止硬编码默认值
