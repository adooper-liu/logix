### **2. 成本数据持久化**

**表名**: `ext_cost_forecast_vs_actual`

**关键字段**:

- `forecast_*`: 预测信息（7 项费用明细）
- `actual_*`: 实际信息（后续填充）
- `cost_variance`: 差异（自动生成）
- `variance_percent`: 差异百分比（自动生成）
- `suggested_*`: 优化建议
- `potential_savings`: 潜在节省

**自动计算字段**:

```sql
cost_variance = actual_total_cost - forecast_total_cost
variance_percent = (cost_variance / forecast_total_cost) * 100
```

## 📊 **三种策略对比**

| 策略          | 提=送=卸   | 堆场费 | 加急费   | 适用场景   |
| ------------- | ---------- | ------ | -------- | ---------- |
| **Direct**    | ✅         | ❌     | ❌       | 常规模式   |
| **Drop off**  | ❌ (提<送) | ✅     | ❌       | 车队有堆场 |
| **Expedited** | ✅         | ❌     | ✅ ($50) | 免费期紧急 |

## 💡 **优化算法逻辑**

```
1. 计算当前方案成本（基准线）
   ↓
2. 探索窗口：[-7 天，+7 天] = 15 天
   ↓
3. 对每一天评估所有可行策略
   ├─ 过滤过去的日期
   ├─ 过滤周末（可选）
   ├─ 过滤仓库无档期
   └─ 评估 Direct/Drop off/Expedited
   ↓
4. 找到成本最低的方案
   ↓
5. 返回 Top 3 方案供用户选择
```

## ⚠️ **常见错误**

### **❌ 错误 1：不检查仓库档期**

```typescript
// 错误：可能选到已满的日期
const option = {
  plannedPickupDate: candidateDate,
  // ❌ 没有检查 isWarehouseAvailable
};
```

### **✅ 正确写法**

```typescript
// 正确：先检查档期
if (!(await isWarehouseAvailable(warehouse, candidateDate))) {
  continue; // 跳过此日期
}
```

---

### **❌ 错误 2：忽略免费期**

```typescript
// 错误：Expedited 只能在免费期内使用
const option = {
  plannedPickupDate: afterLastFreeDate, // 已过免费期
  strategy: "Expedited", // ❌ 无效
};
```

### **✅ 正确写法**

```typescript
// 正确：检查免费期
const strategies = ["Direct", ...(hasYard ? ["Drop off"] : []), ...(candidateDate <= lastFreeDate ? ["Expedited"] : [])];
```
