### 核心改进

#### 严格遵循映射关系

```typescript
// ❌ 旧逻辑：回退到查询所有仓库
const warehouses = await this.warehouseRepo.find({...});

// ✅ 新逻辑：只返回映射链中的仓库
const portMappings = await this.truckingPortMappingRepo.find({...});
const warehouseMappings = await this.warehouseTruckingMappingRepo.find({...});
return warehouses;  // 仅映射中的仓库
```

#### 使用实际费用

```typescript
// ❌ 旧逻辑：基于距离计算
const distance = 25 miles;
const cost = distance × 2.5 × multiplier = $62.5;

// ✅ 新逻辑：直接使用配置费用
const transportFee = mapping.transport_fee = $150;
return transportFee;
```

## 📋 数据库表确认

### 现有表结构 ✅

#### 1. 仓库能力表

**表名**: `dict_warehouses`  
**字段**: `daily_unload_capacity` (INT, DEFAULT 10)

**占用表**: `ext_warehouse_daily_occupancy`
- `planned_count`: 已计划数量
- `capacity`: 总容量（支持日历规则）
- `remaining`: 剩余容量

#### 2. 车队能力表

**表名**: `dict_trucking_companies`  
**字段**: 
- `daily_capacity` (INT, DEFAULT 10) ✅
- `daily_return_capacity` (INT) ✅
- `yard_daily_capacity` (INT) ✅

**占用表**: `ext_trucking_slot_occupancy`
- `planned_trips`: 已计划趟数
- `capacity`: 总容量（支持日历规则）
- `remaining`: 剩余容量

#### 3. 港口 - 车队映射表

**表名**: `dict_trucking_port_mapping`

| 字段 | 用途 | 类型 | 状态 |
|------|------|------|------|
| `yard_capacity` | 堆场容量（Drop 模式每天货柜量） | DECIMAL | ✅ 已有 |
| `standard_rate` | 收费标准（每天 USD/天） | DECIMAL | ✅ 已有 |
| `yard_operation_fee` | 堆场操作费（一次性 USD/柜） | DECIMAL | ✅ 已有 |
| `transport_fee` | **拖卡费（每次运输总费用 USD）** | DECIMAL | ✅ Phase 2 新增 |

## 🎯 费用计算逻辑

### Direct 模式

```typescript
transportCost = mapping.transport_fee;  // $150
```

### Drop off 模式

```typescript
storageDays = calculateStorageDays(option);  // 3 天
storageCost = mapping.standard_rate × storageDays + mapping.yard_operation_fee
            = $50 × 3 + $100 = $250
```

### Expedited 模式

```typescript
expeditedCost = mapping.transport_fee × 1.5
              = $150 × 1.5 = $225
```
