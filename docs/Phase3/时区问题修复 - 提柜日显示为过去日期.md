# 时区问题修复 - 提柜日显示为过去日期

## 🐛 问题描述

**现象**：

- 当前日期：2026-03-26
- 排产预览显示的提柜日：2026-03-25（过去日期）
- 用户疑问：为什么提柜日是过去日期？

**错误日志**：

```
RangeError: Invalid time value
    at Date.toISOString (<anonymous>)
    at IntelligentSchedulingService.scheduleSingleContainer (line 329)
```

## 🔍 根本原因

### 时区差异导致的日期计算错误

**问题链路**：

```
数据库存储：ETA = "2026-03-25T00:00:00Z" (UTC)
    ↓
JavaScript 解析：new Date("2026-03-25T00:00:00Z")
    ↓
太平洋时间 (UTC-7): "2026-03-24T17:00:00-07:00"
    ↓
计算清关日：2026-03-24 (早了一天！)
    ↓
计算提柜日：清关日 + 1 = 2026-03-25
    ↓
用户看到：3-25 日（但今天是 3-26 日）❌
```

### 代码问题点

#### ❌ 问题代码 1：直接解析 ISO 日期字符串

```typescript
// 第 297 行
const clearanceDate = destPo.eta || destPo.ata;

// 第 308 行
const plannedCustomsDate = new Date(clearanceDate);
// 问题：clearanceDate 是 "2026-03-25T00:00:00Z"
// JavaScript 会按 UTC 解析，然后转换为本地时间
// 在太平洋时间会变成 2026-03-24 17:00:00
```

#### ❌ 问题代码 2：使用 `setHours()` 设置今天

```typescript
// 第 324-325 行
const today = new Date();
today.setHours(0, 0, 0, 0);
// 问题：today 包含当前时间（例如 2026-03-26T14:30:00）
// setHours 后变成 2026-03-26T00:00:00（本地时间）
// 但如果 plannedPickupDate 是 UTC 时间，比较时会出错
```

#### ❌ 问题代码 3：直接比较 Date 对象

```typescript
// 第 326 行
if (plannedPickupDate < today) {
  // 问题：plannedPickupDate 可能包含时区偏移
  // today 是本地时间 00:00:00
  // 比较结果不准确
}
```

## ✅ 修复方案

### 核心原则：**使用日期字符串（YYYY-MM-DD）进行日期计算和比较**

#### ✅ 修复 1：强制使用本地时间创建 Date 对象

```typescript
// 修复前
const plannedCustomsDate = new Date(clearanceDate);

// 修复后
const plannedCustomsDate = new Date(clearanceDate + "T00:00:00");
// 添加 'T00:00:00' 强制 JavaScript 使用本地时间解析
// 避免 UTC 转本地时间导致的日期偏移
```

**原理**：

- `new Date("2026-03-25")` → 按 UTC 解析 → 太平洋时间 = 2026-03-24 17:00
- `new Date("2026-03-25T00:00:00")` → 按本地时间解析 → 太平洋时间 = 2026-03-25 00:00

#### ✅ 修复 2：验证日期有效性

```typescript
// 在计算提柜日前验证
let plannedPickupDate = await this.calculatePlannedPickupDate(plannedCustomsDate, destPo.lastFreeDate);

// ✅ 验证日期有效性
if (!plannedPickupDate || isNaN(plannedPickupDate.getTime())) {
  logger.warn(`[IntelligentScheduling] Invalid pickup date calculated for ${container.containerNumber}`);
  return {
    containerNumber: container.containerNumber,
    success: false,
    message: "计算提柜日失败",
    ...containerInfo,
  };
}
```

#### ✅ 修复 3：使用日期字符串比较

```typescript
// 修复前
const today = new Date();
today.setHours(0, 0, 0, 0);
if (plannedPickupDate < today) {
  // 时区不一致，比较结果错误
}

// 修复后
const todayStr = today.toISOString().split("T")[0]; // "2026-03-26"
const pickupDateStr = plannedPickupDate.toISOString().split("T")[0]; // "2026-03-25"

if (pickupDateStr < todayStr) {
  // 字符串比较，完全规避时区问题
  plannedPickupDate = new Date(todayStr + "T00:00:00");
  plannedCustomsDate.setTime(plannedPickupDate.getTime());
  plannedCustomsDate.setDate(plannedCustomsDate.getDate() - 1);
}
```

**原理**：

- 所有日期都转换为 ISO 字符串的日期部分（YYYY-MM-DD）
- 字符串比较不受时区影响
- 创建 Date 时统一使用 `+ 'T00:00:00'` 格式

    ↓

## 📚 技术要点

### 1. JavaScript 日期解析规则

```javascript
// UTC 解析（带 Z 或 +00:00）
new Date("2026-03-25T00:00:00Z"); // UTC 时间
new Date("2026-03-25T00:00:00+00:00"); // UTC 时间

// 本地时间解析（无时区标识）
new Date("2026-03-25"); // 本地时间 00:00
new Date("2026-03-25T00:00:00"); // 本地时间 00:00
```

### 2. 日期比较最佳实践

```javascript
// ❌ 错误：直接比较 Date 对象（受时区影响）
if (date1 < date2) {
}

// ✅ 正确：转换为日期字符串比较
const date1Str = date1.toISOString().split("T")[0];
const date2Str = date2.toISOString().split("T")[0];
if (date1Str < date2Str) {
}
```

### 3. 创建无时区问题的 Date 对象

```javascript
// ❌ 错误：可能按 UTC 解析
new Date("2026-03-25");

// ✅ 正确：强制按本地时间解析
new Date("2026-03-25T00:00:00");
```
