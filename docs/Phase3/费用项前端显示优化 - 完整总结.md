### **3. 平均费用统计**

#### **计算公式**

```typescript
平均费用 = 总费用 / 成功排产的货柜数
```

#### **颜色类型**

```typescript
if (avg === 0) return 'success'        // 🟢 绿色
if (avg <= 100) return ''              // ⚪ 默认色
if (avg <= 500) return 'warning'       // 🟠 橙色
return 'danger'                        // 🔴 红色
```

#### **业务价值**

- 📊 **整体成本水平**: 快速了解平均每柜成本
- 🎯 **异常判断**: 如果 avg > 预期，说明整体成本偏高
- 💡 **决策支持**: 是否需要重新优化排产方案

---

### **4. 费用区间统计**

#### **展示格式**

```
费用区间：$[min] ~ $[max]
示例：$0 ~ $1,200
```

#### **业务分析**

**区间大小分析**:
- **小区间** ($0 ~ $200): 费用分布均匀，排产稳定
- **中间距** ($0 ~ $800): 有一定差异，属正常范围
- **大区间** ($0 ~ $2000+): 离散度大，存在异常高费用

**使用场景**:
1. **识别异常**: max 远大于 min → 检查最高费用的货柜
2. **优化空间**: 重点关注接近 max 的货柜
3. **成本控制**: 分析高费用原因，制定优化策略

## 🔧 技术实现亮点

### **1. 计算属性优化**

```typescript
// 高效的响应式计算
const averageCost = computed(() => {
  const successWithCost = props.previewResults
    .filter(r => r.success && r.estimatedCosts?.totalCost)
  if (successWithCost.length === 0) return 0
  return totalEstimatedCost.value / successWithCost.length
})

// 复用已计算的 totalEstimatedCost.value，避免重复计算
```

### **2. 函数式编程**

```typescript
// 纯函数，无副作用
const getAmountClass = (amount: number): string => {
  if (amount === 0) return 'amount-zero'
  if (amount <= 100) return 'amount-low'
  if (amount <= 500) return 'amount-medium'
  if (amount <= 1000) return 'amount-high'
  return 'amount-critical'
}
```
