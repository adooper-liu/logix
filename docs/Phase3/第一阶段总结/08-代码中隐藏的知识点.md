## 2. 硬编码值清理 🔥

### 2.1 应该提取为常量的值

```typescript
// ❌ 硬编码数字
const FREE_STORAGE_DAYS = 7
if (days > 3)

// ✅ 使用常量
const ALERT_THRESHOLD_DAYS = 3
if (days > ALERT_THRESHOLD_DAYS)

// ❌ 硬编码字符串
const DEFAULT_COUNTRY = 'GB'

// ✅ 使用常量
const DEFAULT_COUNTRY_CODE = 'GB'
```

### 2.2 硬编码颜色

```scss
// ❌ 硬编码颜色
background-color: #409EFF

// ✅ 使用 SCSS 变量
background-color: $primary-color

// ❌ 硬编码状态颜色
.status-warning { color: #FFA500 }

// ✅ 使用主题变量
.status-warning { color: $warning-color }
```

## 3. 性能优化技巧

### 3.1 缓存策略

```typescript
// 使用 computed 缓存
const filteredContainers = computed(() => {
  return containers.value.filter(c => c.status === 'active')
})

// 使用 Redis 缓存
const cacheKey = `container:${containerNumber}`
const cached = await redis.get(cacheKey)
if (cached) return JSON.parse(cached)
```

### 3.2 懒加载

```typescript
// 路由懒加载
const ContainerDetail = () => import('@/views/ContainerDetail.vue')

// 组件懒加载
const HeavyChart = defineAsyncComponent(() => import('./HeavyChart.vue'))
```

### 3.3 防抖节流

```typescript
// 搜索框防抖
const debouncedSearch = debounce((query) => {
  fetchContainers({ search: query })
}, 300)

// 按钮防抖
const handleClick = throttle(() => {
  submitForm()
```

## 4. 错误处理模式

### 4.1 Try-Catch 模式

```typescript
async function fetchContainer(id: string) {
  try {
    const container = await containerService.getById(id)
    return container
  } catch (error) {
    logger.error('获取货柜失败', { id, error })
    throw new BusinessException('货柜不存在')
  }
}
```

### 4.2 前端错误拦截

```typescript
// Axios 拦截器
axios.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) {
      // 未授权
      router.push('/login')
    }
    ElMessage.error(error.message)
    return Promise.reject(error)
  }
)
```
