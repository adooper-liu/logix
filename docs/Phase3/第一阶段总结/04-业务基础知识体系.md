## 1. 货柜物流全流程

### 1.1 七层流转架构 🔥

│  ┌─────────────────────────────────────────────────────────────┐    │

│  │ 备货单号、客户、销往国家、价格条款、货物汇总                   │    │

### 1.4 多港经停逻辑 🔥

**港口类型定义**:

```sql
-- process_port_operations.port_type
CREATE TYPE port_type_enum AS ENUM (
  'origin',      -- 起运港
  'transit',     -- 中转港
  'destination'  -- 目的港
);
```

**port_sequence 序号作用**:

```sql
-- 同一货柜可能有多次港口操作记录
SELECT * FROM process_port_operations 
WHERE container_number = 'TCKU1234567'
ORDER BY port_sequence;

-- 结果示例:
-- | port_sequence | port_type   | port_code | eta_dest_port | ata_dest_port |
-- |---------------|-------------|-----------|---------------|---------------|
-- | 1             | origin      | CNNGB     | 2026-01-10    | 2026-01-12    |
-- | 2             | transit     | SGSIN     | 2026-01-20    | 2026-01-22    |
-- | 3             | destination | GBFXT    | 2026-02-01    | 2026-02-03    |
```

**统计规则 - 只统计目的港**:

```typescript
// 统计只查 port_type = 'destination'
// 避免中转港数据混入
const ARRIVED_TODAY_SUBQUERY = `
  SELECT c.container_number
  FROM biz_containers c
  INNER JOIN process_port_operations po ON c.container_number = po.container_number
  WHERE po.port_type = 'destination'  -- 只统计目的港
  AND po.port_sequence = (
    SELECT MAX(po2.port_sequence)
    FROM process_port_operations po2
    WHERE po2.container_number = po.container_number
    AND po2.port_type = 'destination'
  )
`;
```

### 2.1 价格条款

| 条款 | 英文 | 含义 | 责任划分 |
|------|------|------|---------|

## 5. 费用体系详解

### 5.1 滞港费/滞箱费概念 🔥

| 费用类型 | 英文 | 简称 | 收费阶段 |
|----------|------|------|---------|
| **滞港费** | Demurrage | - | 货物在港口堆放超期 |
| **滞箱费** | Detention | - | 集装箱使用超期 |
| **超期使用费** | Per Diem | - | 通用超期计费 |

**免堆期 vs 免箱期**:

```
时间线:
┌─────────────────────────────────────────────────────────────────┐
│                    船到港                                        │
│                      │                                           │
│                      │ 免堆期 (Free Storage Days)               │
│                      ├─────────────────────────────────────────│
│                      │                                           │
│                      ▼                                           │
│                   提柜日                                        │
│                      │                                           │
│                      │ 免箱期 (Free Detention Days)              │
│                      ├─────────────────────────────────────────│
│                      │                                           │
│                      ▼                                           │
│                   还箱日                                        │
│                      │                                           │
│                      │ 超期计费                                   │
│                      ▼                                           │
└─────────────────────────────────────────────────────────────────┘

费用计算:
Demurrage = max(0, 在港天数 - 免堆期) × 日费率
Detention = max(0, 用箱天数 - 免箱期) × 日费率
```

### 5.2 滞港费标准匹配条件 🔥

**四个字段必须同时满足**:

```sql
-- dict_demurrage_standards 表（假设）
SELECT * FROM dict_demurrage_standards
WHERE destination_country = :countryCode
  AND destination_port = :portCode
  AND shipping_company = :carrierCode
  AND freight_forwarder = :forwarderCode
  AND effective_date <= :currentDate
  AND (expiry_date IS NULL OR expiry_date >= :currentDate)
```

### 5.3 计算模式区分 🔥

| 模式 | 用途 | 使用场景 |
|------|------|---------|
| **actual** | 实际费用计算 | 已完成物流，计算真实费用 |
| **forecast** | 费用预测 | 未完成物流，预测潜在费用 |

```typescript
// backend/src/services/demurrage.service.ts

interface DemurrageCalculationParams {
  containerNumber: string
  calculationMode: 'actual' | 'forecast'  // 核心字段
}

// actual 模式: 使用实际日期
const startDate = calculationDates.ataDestPort || calculationDates.gateInTime
const endDate = calculationDates.gateOutTime || calculationDates.emptyReturnTime

// forecast 模式: 使用计划日期
const startDate = calculationDates.ataDestPort || calculationDates.gateInTime
const endDate = calculationDates.plannedPickupDate || calculationDates.plannedReturnDate
```

### 5.4 关键日期时间线 🔥

**三个时间概念区分**:

| 概念 | 计算方式 | 显示颜色 | 含义 |
|------|---------|---------|------|
| **历时** | 上一节点结束 → 当前节点开始 | 蓝色 | 历史效率分析 |
| **倒计时** | 当前节点开始 → 下一节点开始 | 绿色/橙色 | 剩余时间 |
| **超期** | 当前时间 - 当前节点开始 - 标准耗时 | 红色 | 风险预警 |

**示例 - 最晚提柜日**:

```
最晚提柜日: 2026-02-10
标准耗时: 7天 (168小时)
当前时间: 2026-02-15

计算:
当前时间 - 最晚提柜日 = 5天
标准耗时 = 7天

结果: 未超期，还剩 2天
显示: "倒计时 2天" (绿色)
```
