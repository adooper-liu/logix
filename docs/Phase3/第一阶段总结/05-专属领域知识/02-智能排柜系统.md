## 1. 智能排柜概述

### 1.1 什么是智能排柜？

**智能排柜**是 LogiX 物流系统的核心功能之一，根据货柜的ETA、目的港、仓库可用性等信息，**自动计算并推荐最优的物流计划日期**。

### 1.2 核心目标

```
┌─────────────────────────────────────────────────────────────────────┐

│                                                                     │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐       │
│  │  约束校验    │ -> │  计划生成    │ -> │  成本优化    │       │
│  │  (容量限制)  │    │  (日期计算)   │    │  (方案推荐)   │       │
```

## 2. 业务场景与流程

### 2.1 典型业务流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  备货单创建  │ --> │   海运出运   │ --> │  ETA到港    │ --> │  智能排柜   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                                                                        │
       ┌─────────────────────────────────────────────────────────────┘
       ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  计划提柜   │ --> │  计划清关   │ --> │  计划卸柜   │ --> │  计划还箱   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

### 2.2 五个核心日期

智能排柜以**卸柜日(plannedUnloadDate)**为核心锚点，向前向后推导其他日期：

| 日期字段 | 说明 | 计算逻辑 |
|---------|------|---------|
| `plannedCustomsDate` | 计划清关日 | 提柜日 - 1天 |
| `plannedPickupDate` | 计划提柜日 | 清关日 + 1天，若已过期则至少为今天 |
| `plannedDeliveryDate` | 计划送仓日 | 提柜日 + 运输天数 |
| `plannedUnloadDate` | 计划卸柜日 | **锚点**：基于仓库可用性计算 |
| `plannedReturnDate` | 计划还箱日 | 卸柜日 + 在仓天数 |

## 3. 核心算法与实现 🔥

### 3.1 智能排柜核心逻辑

**位置**: `backend/src/services/intelligentScheduling.service.ts`

```typescript
/**
 * 智能排柜核心流程
 * 
 * 1. 获取待排产货柜列表
 * 2. 按ETA排序（先到先得）
 * 3. 对每个货柜：
 *    a. 确定锚点日期（卸柜日）
 *    b. 向前向后推导其他日期
 *    c. 校验容量约束
 *    d. 保存计划日期
 */
async batchSchedule(request: ScheduleRequest): Promise<BatchScheduleResponse> {
  // Step 1: 获取待排产货柜
  const containers = await this.getContainersToSchedule(request);
  
  // Step 2: 按ETA排序（先到先得）
  containers.sort((a, b) => {
    const etaA = a.etaDestPort?.getTime() || 0;
    const etaB = b.etaDestPort?.getTime() || 0;
    return etaA - etaB;
  });
  
  // Step 3: 逐个排产
  for (const container of containers) {
    const result = await this.calculateOptimalDates(container);
    results.push(result);
  }
  
  return { success: true, total, successCount, failedCount, results };
}
```

### 3.2 锚点日期计算算法

```typescript
/**
 * 计算卸柜日（锚点）
 * 
 * 算法逻辑：
 * 1. 从提柜日开始，向后查找最早可卸柜的日期
 * 2. 考虑仓库日产能限制
 * 3. 考虑仓库可用日期（排除休息日）
 */
private async calculatePlannedUnloadDate(
  plannedPickupDate: Date,
  warehouse: Warehouse,
  dailyCapacity: number
): Promise<Date> {
  let currentDate = new Date(plannedPickupDate);
  currentDate.setDate(currentDate.getDate() + 1); // 提柜后一天
  
  while (true) {
    // 检查是否是可用日期
    const isAvailable = await this.checkWarehouseAvailability(
      warehouse, 
      currentDate
    );
    
    // 检查容量是否已满
    const occupancy = await this.getWarehouseOccupancy(
      warehouse, 
      currentDate
    );
    
    if (isAvailable && occupancy < dailyCapacity) {
      return currentDate; // 找到可用日期
    }
    
    // 向后推一天
    currentDate.setDate(currentDate.getDate() + 1);
    
    // 安全上限：防止无限循环
    if (currentDate > plannedPickupDate + 30 days) {
      throw new Error('无法找到可用卸柜日期');
    }
  }
}
```

### 3.3 容量校验

```typescript
/**
 * 检查仓库容量
 */
private async checkWarehouseCapacity(
  warehouseId: string,
  date: Date,
  quantity: number
): Promise<boolean> {
  const occupancy = await this.warehouseOccupancyRepo.findOne({
    where: {
      warehouseId,
      date: DateOnlyUtils.toDateOnly(date),
    },
  });
  
  const usedCapacity = occupancy?.occupiedSlots || 0;
  const maxCapacity = await this.getWarehouseMaxCapacity(warehouseId);
  
  return (usedCapacity + quantity) <= maxCapacity;
}
```

## 4. 卸柜方案类型

### 4.1 三种卸柜方案

| 方案类型 | 说明 | 适用场景 |
|---------|------|---------|
| **Direct** | 直送模式 | 货物紧急，直接送到仓库卸柜 |
| **Drop off** | 卸货模式 | 货物到港后暂存，再安排送仓 |
| **Expedited** | 加急模式 | 需要额外加急处理 |

### 4.2 方案选择逻辑

```typescript
/**
 * 根据约束条件选择最优卸柜方案
 */
private selectUnloadOption(
  container: Container,
  warehouse: Warehouse,
  truckingCompany: TruckingCompany
): 'Direct' | 'Drop off' | 'Expedited' {
  // 检查是否满足 Direct 条件
  const canDirect = await this.checkDirectConditions(
    container, 
    warehouse, 
    truckingCompany
  );
  if (canDirect) return 'Direct';
  
  // 检查是否满足 Expedited 条件
  const canExpedited = await this.checkExpeditedConditions(
    container, 
    truckingCompany
  );
  if (canExpedited) return 'Expedited';
  
  // 默认使用 Drop off
  return 'Drop off';
}
```

## 5. 成本优化机制

### 5.1 成本优化服务

**文件**: `backend/src/services/schedulingCostOptimizer.service.ts`

```typescript
export class SchedulingCostOptimizerService {
  /**
   * 评估方案总成本
   */
  async evaluateTotalCost(option: UnloadOption): Promise<CostBreakdown> {
    const costs = {
      transportationCost: 0,
      warehouseStorageCost: 0,
      laborCost: 0,
    };
    
    // 计算运输成本
    costs.transportationCost = await this.calculateTransportationCost(option);
    
    // 计算仓储成本
    costs.warehouseStorageCost = await this.calculateWarehouseCost(option);
    
    // 计算人工成本
    costs.laborCost = await this.calculateLaborCost(option);
    
    // 加急费用
    if (option.type === 'Expedited') {
      costs.expeditedFee = await this.calculateExpeditedFee(option);
    }
    
    return costs;
  }
  
  /**
   * 对比多个方案
   */
  async compareOptions(options: UnloadOption[]): Promise<OptionComparison> {
    const evaluations = await Promise.all(
      options.map(opt => this.evaluateTotalCost(opt))
    );
    
    // 找出最优方案
    const cheapest = evaluations.reduce((min, curr) => 
      curr.total < min.total ? curr : min
    );
    
    return { evaluations, recommended: cheapest };
  }
}
```

## 8. 典型问题与解决方案

### 8.1 容量不足

**问题**: 仓库容量已满，无法安排卸柜

**解决方案**:
```typescript
// 向后查找下一个可用日期
async function findNextAvailableDate(
  warehouseId: string,
  startDate: Date,
  maxDays: number = 30
): Promise<Date> {
  for (let i = 0; i < maxDays; i++) {
    const checkDate = new Date(startDate);
    checkDate.setDate(checkDate.getDate() + i);
    
    const hasCapacity = await checkWarehouseCapacity(
      warehouseId, 
      checkDate
    );
    
    if (hasCapacity) return checkDate;
  }
  
  throw new Error('30天内无可用容量');
}
```

### 8.2 车队不可用

**问题**: 选定的车队在指定日期不可用

**解决方案**:
```typescript
// 检查车队可用性并尝试备选
async function findAvailableTrucking(
  warehouseId: string,
  date: Date
): Promise<string | null> {
  // 1. 获取仓库绑定的车队列表
  const mappings = await getWarehouseTruckingMappings(warehouseId);
  
  // 2. 按优先级检查
  for (const mapping of mappings) {
    const hasCapacity = await checkTruckingCapacity(
      mapping.truckingCompanyId,
      date
    );
    
    if (hasCapacity) return mapping.truckingCompanyId;
  }
  
  return null; // 无可用车队
}
```
