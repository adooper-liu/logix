## 1. 状态机概述

### 1.1 为什么需要状态机？

```
┌─────────────────────────────────────────────────────────────────────┐
│                      状态机核心价值                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ✅ 规范化流程                                                     │
│     - 明确定义每种状态                                              │
│     - 规定合法转换路径                                              │
│                                                                     │
│  ✅ 防止非法流转                                                   │
│     - 状态A不能直接跳到状态D                                        │
│     - 必须按顺序流转                                                │
│                                                                     │
│  ✅ 业务逻辑统一                                                   │
│     - 所有货柜使用同一套规则                                        │
│     - 易于维护和扩展                                                │
│                                                                     │
│  ✅ 统计/预警基础                                                  │
│     - 按状态统计货柜数量                                            │
│     - 根据状态触发预警                                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 两套状态系统

LogiX 项目中存在**两套状态系统**，需要明确区分：

| 系统 | 用途 | 数据来源 |
|------|------|---------|
| **业务状态机** | 桑基图、统计、滞港费 | 主后端 `calculateLogisticsStatus` |
| **路径状态机** | 物流路径可视化 | 物流路径微服务 GraphQL |

### 3.2 合法转换规则

```typescript
/**
 * 简化状态的合法流转规则
 */
export const SimplifiedStatusTransitions: Record<SimplifiedStatus, SimplifiedStatus[]> = {
  [SimplifiedStatus.NOT_SHIPPED]: [
    SimplifiedStatus.SHIPPED
  ],
  
  [SimplifiedStatus.SHIPPED]: [
    SimplifiedStatus.IN_TRANSIT,
    SimplifiedStatus.AT_PORT,
    SimplifiedStatus.PICKED_UP // 支持跳转：直接提柜
  ],
  
  [SimplifiedStatus.IN_TRANSIT]: [
    SimplifiedStatus.AT_PORT,
    SimplifiedStatus.PICKED_UP // 支持跳转
  ],
  
  [SimplifiedStatus.AT_PORT]: [
    SimplifiedStatus.PICKED_UP,
    SimplifiedStatus.UNLOADED // 支持跳转：直接卸柜
  ],
  
  [SimplifiedStatus.PICKED_UP]: [
    SimplifiedStatus.UNLOADED,
    SimplifiedStatus.RETURNED_EMPTY // 支持跳转：直接还箱
  ],
  
  [SimplifiedStatus.UNLOADED]: [
    SimplifiedStatus.RETURNED_EMPTY
  ],
  
  [SimplifiedStatus.RETURNED_EMPTY]: []  // 终态，不可再转换
}
```

### 3.3 转换验证

```typescript
/**
 * 验证状态转换是否合法
 */
function isValidTransition(
  fromStatus: SimplifiedStatus,
  toStatus: SimplifiedStatus
): boolean {
  const allowedTransitions = SimplifiedStatusTransitions[fromStatus]
  return allowedTransitions?.includes(toStatus) ?? false
}
```

## 4. calculateLogisticsStatus 实现 🔥

### 4.1 核心算法

```typescript
// backend/src/services/containerStatus.service.ts

/**
 * 计算货柜的物流状态
 * 基于实际时间字段判断
 */
function calculateLogisticsStatus(container: Container): SimplifiedStatus {
  
  // 1. 检查是否已还箱（终态）
  if (container.returnTime || container.logisticsStatus === SimplifiedStatus.RETURNED_EMPTY) {
    return SimplifiedStatus.RETURNED_EMPTY
  }
  
  // 2. 检查是否已卸柜
  if (container.unloadTime || hasUnloadedInWarehouse(container)) {
    return SimplifiedStatus.UNLOADED
  }
  
  // 3. 检查是否已提柜
  if (container.gateOutTime || hasPickupRecord(container)) {
    return SimplifiedStatus.PICKED_UP
  }
  
  // 4. 检查是否已到港（目的港）
  if (container.ataDestPort || hasArrivedAtDestination(container)) {
    return SimplifiedStatus.AT_PORT
  }
  
  // 5. 检查是否在途（有离港时间）
  if (container.departureTime || container.ataOrigin) {
    return SimplifiedStatus.IN_TRANSIT
  }
  
  // 6. 检查是否已出运（有装船时间）
  if (container.loadedTime || container.actualShipDate) {
    return SimplifiedStatus.SHIPPED
  }
  
  // 7. 默认：未出运
  return SimplifiedStatus.NOT_SHIPPED
}

/**
 * 检查是否已到达目的港
 */
function hasArrivedAtDestination(container: Container): boolean {
  // 有 ATA 记录
  if (container.portOperations?.some(
    po => po.portType === 'destination' && po.ataDestPort
  )) {
    return true
  }
  
  // 状态为已到港
  if (container.logisticsStatus === SimplifiedStatus.AT_PORT) {
    return true
  }
  
  return false
}
```

### 4.2 与飞驼集成

```typescript
/**
 * 飞驼数据同步后触发状态机重算
 */
async function recalculateAfterFeituoSync(containerNumber: string) {
  // 1. 更新核心时间字段
  await updateContainerCoreFields(containerNumber)
  
  // 2. 重新计算物流状态
  const newStatus = await calculateLogisticsStatus(containerNumber)
  
  // 3. 更新货柜状态
  await containerRepository.update(
    { containerNumber },
    { logisticsStatus: newStatus }
  )
  
  return newStatus
}
```

## 6. 状态历史追踪

### 6.1 状态事件表

```sql
-- 记录每次状态变更
CREATE TABLE ext_container_status_events (
  id SERIAL PRIMARY KEY,
  container_number VARCHAR(50) NOT NULL,
  status_code VARCHAR(20),           -- 状态码
  status_name_cn VARCHAR(100),       -- 状态中文名
  occurred_at TIMESTAMP,             -- 发生时间
  location_code VARCHAR(10),         -- 位置代码
  port_type VARCHAR(20),            -- 港口类型
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 6.2 查询状态变更历史

```sql
-- 查询货柜状态变更历史
SELECT 
  status_name_cn,
  occurred_at,
  location_code,
  port_type
FROM ext_container_status_events
WHERE container_number = :containerNumber
ORDER BY occurred_at DESC;
```
