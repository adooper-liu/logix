## 1. 飞驼系统概述

### 1.1 什么是飞驼？

**飞驼 (FeiTuo)** 是一个物流跟踪系统，提供：
- 集装箱全流程状态追踪
- 船舶/港口信息查询
- 费用数据同步

**在 LogiX 中的角色**:
```
┌─────────────┐    API    ┌─────────────┐    适配    ┌─────────────┐
│  飞驼系统   │ ────────▶ │  LogiX后端  │ ────────▶ │  业务表     │
│ (外部数据源) │           │ (适配转换)   │           │ (biz_*)     │
└─────────────┘           └─────────────┘           └─────────────┘
```

### 1.2 数据更新频率

- **实时推送**: 有重大状态变更时主动推送
- **定时拉取**: 每小时/每天批量同步
- **按需查询**: 前端查看详情时实时查询

### 1.3 飞驼提供的数据类型

| 数据类型 | 说明 | 对应表 |
|---------|------|--------|
| 状态事件 | 箱子的每一次状态变更 | ext_feituo_status_events |
| 原始数据 | 未经处理的原始API数据 | ext_feituo_raw_data |
| 装载记录 | 货物装载信息 | ext_container_loading_records |
| HOLD记录 | 扣货/查验信息 | ext_container_hold_records |
| 费用记录 | 滞港费等费用数据 | ext_container_charges |

## 3. 状态码到字段映射逻辑 🔥

### 3.1 核心字段映射表

**飞驼状态码 → 业务核心字段的映射**:

| 飞驼状态码 | 含义 | 映射到的核心字段 | 端口类型 |
|-----------|------|----------------|---------|
| ATA | 实际到港 | ata_dest_port | destination |
| ETA | 预计到港 | eta_dest_port | destination |
| FETA | 修正ETA | eta_correction | destination |
| GTIN | 进场 | gate_in_time | destination |
| GTOT | 提柜 | gate_out_time | destination |
| RCVE | 还空箱 | return_time | - |
| DISC | 卸船 | discharge_time | destination |
| ARRIVE | 到港 | ata_dest_port | destination |
| DEPARTED | 离港 | departure_time | origin |

### 3.2 映射实现代码

```typescript
// backend/src/services/feituo/FeituoSmartDateUpdater.ts

/** 
 * 根据状态码获取对应的核心字段名 
 */
function getCoreFieldName(statusCode: string): string | null {
  const fieldMapping: Record<string, string> = {
    'ATA': 'ata_dest_port',
    'ATA_DEST_PORT': 'ata_dest_port',
    'ETA': 'eta_dest_port',
    'ETA_DEST_PORT': 'eta_dest_port',
    'FETA': 'eta_correction',
    'ETA_CORRECTION': 'eta_correction',
    'GTIN': 'gate_in_time',
    'GTOT': 'gate_out_time',
    'RCVE': 'return_time',
    'DISC': 'discharge_time',
    'DISCHARGED': 'discharge_time',
    'ARRIVE': 'ata_dest_port',
    'DEPARTED': 'departure_time',
    'DEPARTURE': 'departure_time'
  }
  return fieldMapping[statusCode] || null
}

/** 
 * 根据状态码推断港口类型 
 */
function getPortTypeForStatusCode(statusCode: string): 'origin' | 'transit' | 'destination' {
  const originStatuses = ['DEPARTED', 'DEPARTURE', 'LOADED', 'STUFFED']
  const transitStatuses = ['TRANSIT', 'TRANSIT_ARRIVED', 'TRANSIT_DEPARTED']
  
  if (originStatuses.includes(statusCode)) return 'origin'
  if (transitStatuses.includes(statusCode)) return 'transit'
  return 'destination' // 默认目的港
}
```

### 3.3 port_type 推断逻辑

```typescript
/** 
 * 推断港口操作记录的 port_type
 */
function inferPortType(
  statusCode: string, 
  existingRecords: PortOperation[]
): 'origin' | 'transit' | 'destination' {
  
  // 如果已有记录，根据序列推断
  const maxSequence = Math.max(...existingRecords.map(r => r.portSequence || 0))
  
  if (maxSequence === 0) return 'origin'  // 第一条为起运港
  if (maxSequence === 1) return 'transit' // 第二条为中转港
  return 'destination' // 最后为目的港
}
```

// backend/src/services/feituoImport.service.ts

async function syncFeituoData(containerNumber: string) {
  // 1. 拉取飞驼数据
  const rawData = await feituoApi.fetch(containerNumber)
  
  // 2. 写入原始数据表
  await this.saveRawData(containerNumber, rawData)
  
  // 3. 解析并写入状态事件表
  const events = this.parseStatusEvents(rawData)
  await this.saveStatusEvents(containerNumber, events)
  
  // 4. 更新业务核心字段
  await this.updateContainerCoreFields(containerNumber, events)
  
  // 5. 触发状态机重算
  await this.recalculateLogisticsStatus(containerNumber)
}

## 6. 飞驼API与状态机集成 🔥

### 6.1 集成方案

**方案**: 核心字段更新 + 状态机重计算

```typescript
/**
 * 更新货柜核心时间字段
 * 飞驼状态码 → 核心字段映射
 */
async updateContainerCoreFields(
  containerNumber: string, 
  events: FeituoStatusEvent[]
) {
  for (const event of events) {
    const fieldName = getCoreFieldName(event.statusCode)
    if (!fieldName) continue
    
    // 只更新更晚的时间
    const currentValue = await this.getFieldValue(containerNumber, fieldName)
    if (!currentValue || new Date(event.occurredAt) > new Date(currentValue)) {
      await this.setFieldValue(containerNumber, fieldName, event.occurredAt)
    }
  }
}

/**
 * 触发状态机重新计算
 */
async recalculateLogisticsStatus(containerNumber: string) {
  // 调用主后端状态机
  const status = calculateLogisticsStatus(containerNumber)
  
  // 更新货柜表
  await this.containerRepository.update(
    { containerNumber },
    { logisticsStatus: status }
  )
}
```

## 7. 典型问题与解决方案

### 7.1 问题1: 字段映射缺失

**现象**: 某些飞驼状态码没有对应的核心字段

**原因**: 映射表不完整

**解决方案**:
```typescript
// 扩展映射表
const fieldMapping: Record<string, string> = {
  // 已有映射
  'ATA': 'ata_dest_port',
  // 新增映射
  'ON_BOARD': 'loaded_time',  // 装船时间
  'PASSED_CUSTOMS': 'customs_clearance_time',  // 清关时间
}
```

### 7.2 问题2: 时区差异

**现象**: 飞驼返回的时间与本地时间不一致

**原因**: 飞驼API使用UTC时间

**解决方案**:
```typescript
// 统一转换为本地时间
function convertToLocalTime(utcTime: string): Date {
  const date = new Date(utcTime)
  // 飞驼返回的时间已经是北京时间，无需额外转换
  return date
}
```

### 7.3 问题3: 重复数据

**现象**: 同一状态码多次触发更新

**原因**: 飞驼推送重复/历史数据未去重

**解决方案**:
```typescript
// 根据 status_code + occurred_at 去重
async function saveStatusEvent(event: FeituoStatusEvent) {
  const existing = await this.statusEventRepo.findOne({
    where: {
      containerNumber: event.containerNumber,
      statusCode: event.statusCode,
      occurredAt: event.occurredAt
    }
  })
  
  if (existing) return  // 跳过重复
  
  await this.statusEventRepo.save(event)
}
```
