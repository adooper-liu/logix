# 提柜日时区问题修复 - 本地日期字符串

## 🐛 问题现象

**用户反馈**：

- 后端兜底逻辑应该调整为明天（2026-03-27）
- 前端显示却是今天（2026-03-26）
- 怀疑是时区转换问题

**截图证据**：

- 提柜日、送仓日、卸柜日均显示为 2026-03-26
- 还箱日显示为 2026-03-27（明天）

---

## 🔍 根本原因

### UTC 转换导致的日期偏移

**问题链路**：

```typescript
// 假设当前时间：2026-03-26 17:00:00 PST (太平洋时间，UTC-8)

// 1. 获取今天日期字符串
const today = new Date();
const todayStr = today.toISOString().split("T")[0];
// today.toISOString() = "2026-03-27T01:00:00.000Z" (已经是明天！)
// todayStr = "2026-03-27" ❌ 错误！本地时间还是 3-26

// 2. 计算明天
const tomorrow = new Date(todayStr + "T00:00:00");
tomorrow.setDate(tomorrow.getDate() + 1);
// tomorrow = "2026-03-28T00:00:00.000Z" (UTC)
// 转本地时间 = 2026-03-27 16:00:00 PST

// 3. 返回给前端
plannedPickupDate: plannedPickupDate.toISOString().split("T")[0];
// = "2026-03-28" ❌ 又偏移了一天！
```

### 时区转换陷阱

**太平洋标准时间 (PST) = UTC-8**

| 本地时间         | UTC 时间         | toISOString().split('T')[0] |
| ---------------- | ---------------- | --------------------------- |
| 2026-03-26 17:00 | 2026-03-27 01:00 | "2026-03-27" ❌             |
| 2026-03-26 23:00 | 2026-03-27 07:00 | "2026-03-27" ❌             |
| 2026-03-27 00:00 | 2026-03-27 08:00 | "2026-03-27" ✅             |

**结论**：

- 在**下午/晚上**调用 `toISOString()` 会得到**明天的日期**
- 导致 `todayStr` 比本地实际日期**早一天**

---

## ✅ 修复方案

### 使用本地时间格式化

```typescript
// ✅ 修复：使用本地日期字符串，避免 UTC 转换
const today = new Date();
const todayStr = today
  .toLocaleDateString("zh-CN", {
    year: "numeric",
    month: "2-digit",
    day: "2-digit",
  })
  .replace(/\//g, "-"); // "2026-03-26"

const pickupDateStr = plannedPickupDate
  .toLocaleDateString("zh-CN", {
    year: "numeric",
    month: "2-digit",
    day: "2-digit",
  })
  .replace(/\//g, "-");

if (pickupDateStr <= todayStr) {
  // 调整为明天
  const tomorrow = new Date(todayStr + "T00:00:00");
  tomorrow.setDate(tomorrow.getDate() + 1);
  plannedPickupDate = tomorrow;

  const tomorrowStr = tomorrow
    .toLocaleDateString("zh-CN", {
      year: "numeric",
      month: "2-digit",
      day: "2-digit",
    })
    .replace(/\//g, "-");

  logger.debug(`Pickup date adjusted to tomorrow (${tomorrowStr})`);
}
```

## 🎯 toLocaleDateString 方法

### 语法

```typescript
date.toLocaleDateString(locale, options);
```

### 参数

- `locale`: `'zh-CN'` (使用本地时区)
- `options`:
  - `year: 'numeric'` - 数字年份
  - `month: '2-digit'` - 两位月份
  - `day: '2-digit'` - 两位日期

### 返回值

- `"2026/03/26"` (格式取决于 locale)
- `.replace(/\//g, '-')` → `"2026-03-26"`

## 🚨 注意事项

### 1. 时区选择

- ❌ 不要使用固定的时区（如 `'America/Los_Angeles'`）
- ✅ 使用本地时区：`toLocaleDateString('zh-CN', {...})`

### 2. 日期格式

- `toLocaleDateString()` 返回的格式取决于 locale
- 使用 `.replace(/\//g, '-')` 统一为 `YYYY-MM-DD` 格式

### 3. 与 ISO 字符串的区别

```typescript
// ISO 字符串（UTC）
date.toISOString() // "2026-03-27T01:00:00.000Z"

// 本地日期字符串
date.toLocaleDateString('zh-CN', {...}) // "2026/03/26"
```
