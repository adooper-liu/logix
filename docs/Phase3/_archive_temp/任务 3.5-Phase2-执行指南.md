## ❌ 原问题

### PostgreSQL 语法错误

**错误的 SQL**:
```sql
ALTER TABLE dict_trucking_port_mapping 
ADD COLUMN IF NOT EXISTS transport_fee DECIMAL(10,2) DEFAULT 0 COMMENT '拖卡费（每次运输总费用 USD）';
-- ERROR: syntax error at or near "COMMENT"
```

**原因**: PostgreSQL 不支持在 `ALTER TABLE` 语句中直接使用 `COMMENT` 关键字

---

## ✅ 解决方案

### 正确的 SQL 语法

PostgreSQL 需要将 `ADD COLUMN` 和 `COMMENT` 分开执行：

```sql
-- 1. 添加字段
ALTER TABLE dict_trucking_port_mapping 
ADD COLUMN IF NOT EXISTS transport_fee DECIMAL(10,2) DEFAULT 0;

-- 2. 添加注释
COMMENT ON COLUMN dict_trucking_port_mapping.transport_fee IS '拖卡费（每次运输总费用 USD）';
```

## ⚠️ 常见问题

### 问题 1: "relation does not exist"

**错误信息**:
```
ERROR: relation "dict_trucking_port_mapping" does not exist
```

**原因**: 表不存在

**解决方案**:
```sql
-- 检查表是否存在
SELECT table_name FROM information_schema.tables 
WHERE table_name = 'dict_trucking_port_mapping';

-- 如果不存在，需要先创建表
-- 参考：migrations/create_trucking_port_mapping.sql
```

### 问题 2: "column already exists"

**错误信息**:
```
ERROR: column "transport_fee" of relation "dict_trucking_port_mapping" already exists
```

**原因**: 字段已存在

**解决方案**: 这是正常的，说明字段已经添加过了。继续执行后续语句即可。

### 问题 3: "permission denied"

**错误信息**:
```
ERROR: permission denied for table dict_trucking_port_mapping
```

**原因**: 权限不足

**解决方案**:
```sql
-- 使用有权限的用户执行（如 postgres）
psql -U postgres -d logix -f migrations/add_cost_optimization_mapping_fields.sql
```
