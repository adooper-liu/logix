## 四、数据更新流程

### 4.1 状态事件处理流程

```
飞驼 Excel 导入
    ↓
ext_feituo_status_events (状态事件表)
    ↓
shouldUpdateCoreField(statusCode, hasOccurred)
    ↓
只有 hasOccurred=true 的事件才更新核心字段
    ↓
getCoreFieldName(statusCode) → 获取核心字段名
    ↓
getPortTypeForStatusCode(statusCode) → 获取港口类型
    ↓
根据港口类型查找/创建对应的 port_operations 记录
    ↓
更新该记录的核心时间字段
```

## 六、特殊说明

### 6.1 LOBD 和 DLPT 都更新 shipment_date

- **LOBD** (Loaded on Board): 装船时间
- **DLPT** (Departed): 离港时间

这两个状态码都更新 `process_sea_freight.shipment_date`，但：
- LOBD 通常发生在起运港
- DLPT 可能发生在起运港或中转港

实际使用中，**后发生的事件会覆盖先前的值**。

### 6.2 RCVE 更新的是还空箱表

**RCVE** (Empty Returned) 状态码比较特殊：
- 它不更新 `process_port_operations`
- 而是更新 `process_empty_return.return_time`

这是因为还空箱是一个独立的操作环节。

### 6.3 FETA 状态码

**FETA** (Final ETA) 是交货地预计到港时间：
- 映射到 `eta` 字段
- 港口类型为 `destination`
- 优先级高于普通的 ETA 状态码

## 七、常见问题排查

### Q1: 为什么某个状态码没有更新日期？

**检查点**：
1. 该状态码是否在 `getCoreFieldName` 映射表中
2. 事件的 `has_occurred` 是否为 `true`
3. 对应的港口类型是否正确匹配
4. 数据库中是否已存在对应的港口操作记录

### Q2: 如何确认状态码的港口类型？

```sql
-- 查询状态码与港口类型的对应关系
SELECT DISTINCT
    event_code AS 状态码，
    event_place AS 地点，
    CASE 
        WHEN event_place LIKE '%宁波%' THEN 'origin'
        WHEN event_place LIKE '%费利克斯托%' THEN 'destination'
        ELSE 'transit'
    END AS 推断港口类型
FROM ext_feituo_status_events
WHERE container_number = '你的货柜号';
```

### Q3: 多个港口记录如何区分？

通过 `port_sequence` 字段区分：
- `port_sequence = 1`: 起运港
- `port_sequence = 2`: 中转港（如有）
- `port_sequence = 3`: 目的港

## 八、总结

飞驼状态码映射遵循以下规则：

1. **每个状态码对应一个核心字段**
2. **每个状态码对应一个港口类型**
3. **只有已发生的事件 (has_occurred=true) 才更新核心字段**
4. **多港经停时，根据港口类型更新对应的记录**
5. **后发生的事件会覆盖先前同字段的值**

掌握这些规则，就能准确理解飞驼数据如何影响系统的核心时间表。
