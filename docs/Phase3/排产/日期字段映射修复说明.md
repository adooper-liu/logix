# 排产日期字段映射修复

## 问题描述

在排产确认保存功能中，存在字段映射错误：

### 错误代码

```typescript
// ❌ 错误的实现（scheduling.controller.ts 第 2432 行）
trucking.pickupDate = new Date(plannedData.plannedPickupDate);
trucking.returnDate = new Date(plannedData.plannedReturnDate);
```

### 问题表现

- 数据库中 `planned_pickup_date` 字段为 `NULL`
- 数据库中 `pickup_date` 字段包含计划提柜日期
- 前端 Shipment 表格中 `act` (实际) 标签显示的是计划日期，而非实际日期

### 影响范围

1. **数据混乱**：计划提柜日和实际提柜日混在同一个字段
2. **业务逻辑错误**：无法区分计划日期和实际日期
3. **飞驼系统覆盖**：飞驼导入的实际提柜日会覆盖排产的计划日期

## 修复方案

### 1. 代码修复

**文件**: `backend/src/controllers/scheduling.controller.ts`

**修复后代码**:

```typescript
// ✅ 正确的实现（第 2432-2433 行）
trucking.plannedPickupDate = new Date(plannedData.plannedPickupDate);
trucking.plannedReturnDate = new Date(plannedData.plannedReturnDate);
```

### 2. 数据迁移

**脚本**: `backend/scripts/fix_scheduling_date_mapping.sql`

**执行步骤**:

```bash
# 1. 备份数据（可选但推荐）
docker exec -i logix-postgres pg_dump -U logix -t process_trucking_transport > backup_$(date +%Y%m%d).sql

# 2. 执行修复脚本
docker exec -i logix-postgres psql -U logix -d logix < backend/scripts/fix_scheduling_date_mapping.sql

# 3. 验证修复结果
docker exec -it logix-postgres psql -U logix -d logix -c "SELECT container_number, planned_pickup_date, pickup_date, pickup_date_source FROM process_trucking_transport WHERE planned_pickup_date IS NOT NULL LIMIT 10;"
```

### 3. 字段语义说明

| 字段名                | 用途       | 数据来源          | 说明                               |
| --------------------- | ---------- | ----------------- | ---------------------------------- |
| `planned_pickup_date` | 计划提柜日 | 排产系统计算      | 智能排产系统计算的计划日期         |
| `pickup_date`         | 实际提柜日 | 飞驼系统/业务导入 | 实际提柜操作日期                   |
| `pickup_date_source`  | 数据来源   | 系统自动标记      | `feituo` \| `business` \| `manual` |

## 验证方法

### 前端验证

1. 打开 Shipment 列表页面
2. 查看"提柜日期"列
3. 确认显示三个日期：
   - `lfd` - 最晚提柜日（来自 `last_free_date`）
   - `pla` - 计划提柜日（来自 `planned_pickup_date`）✅
   - `act` - 实际提柜日（来自 `pickup_date`）✅

### 数据库验证

```sql
-- 检查修复后的数据
SELECT
  container_number,
  planned_pickup_date AS "计划提柜日",
  pickup_date AS "实际提柜日",
  pickup_date_source AS "数据来源"
FROM process_trucking_transport
WHERE container_number IN (
  SELECT container_number
  FROM biz_containers
  WHERE schedule_status = 'issued'
)
ORDER BY container_number;
```

## 相关实体定义

### TruckingTransport 实体

```typescript
@Entity("process_trucking_transport")
export class TruckingTransport {
  @PrimaryColumn({ type: "varchar", length: 50, name: "container_number" })
  containerNumber: string;

  @Column({ type: "date", nullable: true, name: "planned_pickup_date" })
  plannedPickupDate?: Date; // ← 计划提柜日

  @Column({ type: "timestamp", nullable: true, name: "pickup_date" })
  pickupDate?: Date; // ← 实际提柜日

  @Column({ type: "varchar", length: 20, nullable: true, name: "pickup_date_source" })
  pickupDateSource?: string; // ← 数据来源
}
```
