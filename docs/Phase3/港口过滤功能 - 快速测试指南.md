## 🔍 **问题诊断**

### **问题 1: 港口选择器不显示**

**可能原因**:
1. `overview.ports` 为空数组
2. 后端未返回 ports 字段

**排查步骤**:
```javascript
// 在浏览器控制台执行
console.log('overview.ports:', overview.value.ports);
// 如果为空，检查后端日志
```

**解决方案**:
- 检查后端 `getSchedulingOverview` 是否添加了 `ports` 字段
- 检查 SQL 查询是否正确

---

### **问题 2: 选择港口后数量不变**

**可能原因**:
1. `portCode` 未传递到后端
2. SQL 过滤条件未生效

**排查步骤**:
```javascript
// 在浏览器控制台查看网络请求
// Network 面板 → 查看 /scheduling/overview 请求

// 检查请求参数
params: {
  country: 'US',
  portCode: 'USLAX' // ← 是否存在？
}
```

**解决方案**:
- 检查 `loadOverview` 函数是否正确传递 `portCode`
- 检查后端 SQL 是否包含 `portCondition`

---

### **问题 3: 预览排产仍为全量**

**可能原因**:
1. `handlePreviewSchedule` 未传递 `portCode`
2. Service 层未应用过滤

**排查步骤**:
```javascript
// 查看 Network 请求
POST /api/v1/scheduling/batch-schedule

// 检查请求体
{
  "country": "US",
  "portCode": "USLAX", // ← 是否存在？
  "dryRun": true
}
```

**解决方案**:
- 确认 `handlePreviewSchedule` 代码中添加了 `portCode` 参数
- 检查 Service 层 `getContainersToSchedule` 方法

---

### **问题 4: 后端报错 "column po.port_code does not exist"**

**可能原因**:
- 数据库表 `process_port_operations` 中没有 `port_code` 字段

**排查步骤**:
```sql
-- 检查表结构
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'process_port_operations';
```

**解决方案**:
```sql
-- 如果确实缺少该字段，需要添加迁移脚本
ALTER TABLE process_port_operations 
ADD COLUMN IF NOT EXISTS port_code VARCHAR(10);
```
