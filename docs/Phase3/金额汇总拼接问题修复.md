# 金额汇总拼接问题修复

## 问题描述

在排产预览模态框中，"预估总费用"显示为字符串拼接而不是数值累加。

### 问题现象（截图证据）

- **预估总费用**: `$01710700.00001710700.00001710700.00001710700.00001710700.0000`（字符串拼接）
- **单个货柜总费用**: `$1710700.0000`（**数值本身错误**，应为 `$1,710.70`）
- **平均费用**: `$NaN`（计算错误）

### 问题确认

**关键发现**：单个货柜的费用显示为 `$1,710,700.00` 而不是 `$1,710.70`！

这说明问题不在前端的汇总计算，而在**后端的费用计算本身**！

### 根本原因推测

后端 `calculateSingleDemurrage` 函数计算出来的 `totalAmount` 值本身就是错误的（`1710700.00` 而不是 `1710.70`）。

可能的原因：
1. `ratePerDay` 的值错误（如 `17107` 而不是 `17.107`）
2. `chargeDays` 的值错误（如 `100000` 而不是 `100`）
3. `tiers` 中的费率配置错误

## 修复方案

### 第一步：前端类型修复（已完成）

**修改文件**

`frontend/src/views/scheduling/components/SchedulingPreviewModal.vue`

**修改内容**

```typescript
// 修改前
const totalEstimatedCost = computed(() => {
  return props.previewResults
    .filter(r => r.success && r.estimatedCosts?.totalCost)
    .reduce((sum, r) => sum + (r.estimatedCosts?.totalCost || 0), 0)
})

// 修改后
const totalEstimatedCost = computed((): number => {
  const results = props.previewResults.filter(r => r.success && r.estimatedCosts?.totalCost)
  return results.reduce((sum, r) => {
    const cost = r.estimatedCosts?.totalCost ?? 0
    return sum + cost
  }, 0)
})
```

**修复要点**

1. ✅ **显式返回类型**: `computed((): number => ...)` 确保返回数值类型
2. ✅ **空值处理**: 使用 `??` 空值合并运算符代替 `||`，更精确地处理 `null` 和 `undefined`
3. ✅ **类型安全**: 明确的类型注解帮助 TypeScript 正确推断
