## 🎯 优化需求

### 用户反馈

> "http://localhost:5173/#/scheduling 界面的逻辑与布优化，问题检查，日历中的国家与仓库选择 位置 要移动到 日历头部位置"

### 原始问题

**优化前的布局流程**:
```
┌─────────────────────────────────────┐
│ 📅 每日能力日历                      │
│ [🏭 仓库] [🚛 车队] [刷新] [手动设置] │ ← 卡片头部
├─────────────────────────────────────┤
│           FullCalendar 日历          │  ← 日历视图
├─────────────────────────────────────┤
│ 🟢 工作日：10  🟴 周末：0 ...        │  ← 图例
├─────────────────────────────────────┤
│ 统计：总天数 | 工作日 | 周末 | 平均   │  ← 统计信息
├─────────────────────────────────────┤
│ [选择国家] [选择仓库] [查看：WH001]   │  ← ❌ 资源选择器（在底部）
└─────────────────────────────────────┘
```

**问题**:
- ❌ 资源选择器（国家 + 仓库/车队）在卡片最底部
- ❌ 用户需要先滚动到底部选择资源，才能看到日历
- ❌ 操作流程不直观：选择 → 查看

### 优化目标

✅ **将资源选择器移到日历头部**:
```
┌─────────────────────────────────────┐
│ 📅 每日能力日历                      │
│ [🏭 仓库] [🚛 车队] [刷新] [手动设置] │ ← 卡片头部
├─────────────────────────────────────┤
│ [选择国家] [选择仓库] [查看：WH001]   │  ← ✅ 资源选择器（移到顶部）
├─────────────────────────────────────┤
│           FullCalendar 日历          │  ← 日历视图
├─────────────────────────────────────┤
│ 🟢 工作日：10  🟴 周末：0 ...        │  ← 图例
├─────────────────────────────────────┤
│ 统计：总天数 | 工作日 | 周末 | 平均   │  ← 统计信息
└─────────────────────────────────────┘
```

## 💡 设计原则

### 1. 就近原则

**解释**: 相关的控件应该放在一起

**应用**:
- 资源选择器紧挨着日历
- 选择后立即看到结果
- 减少视线移动距离

### 2. 优先级原则

**解释**: 重要的、常用的功能放在更显眼的位置

**应用**:
- 资源选择是第一步操作 → 放在顶部
- 日历展示是核心功能 → 紧随其后
- 统计信息是辅助信息 → 放在底部

### 3. 流畅性原则

**解释**: 操作流程应该顺畅无阻

**应用**:
- 从上到下的自然流程
- 无需滚动页面
- 一键操作，即时反馈
