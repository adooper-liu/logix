## 🔍 **关键发现**

### **发现 1: 有两种完全不同的堆存费** ⭐

#### **对比表**

| 维度 | 港口存储费 (Storage Cost) | 外部堆场堆存费 (Yard Storage Cost) |
|------|--------------------------|----------------------------------|
| **英文名称** | Storage Charge | Yard Storage Fee |
| **数据源** | `ext_demurrage_standards` | `dict_trucking_port_mapping` |
| **字段** | `rate_per_day`, `tiers`, `free_days` | `standard_rate`, `yard_operation_fee` |
| **收取方** | 港口/码头 | 拖车车队 |
| **类别** | D&D 费用 | 运输环节附加费 |
| **计算方式** | 阶梯费率 | 固定费率 + 操作费 |
| **计费区间** | 卸船日 → 提柜日 | 提柜日 → 送仓日 ⭐ |
| **适用场景** | 所有模式 | 仅真 Drop off 模式 |
| **免费天数** | 有 | 无 |

#### **重要性**

这是两个**完全不同**的费用项，经常混淆：
- ❌ ~~错误理解~~: "堆存费只有一种"
- ✅ **正确理解**: "有两种堆存费，数据源、收取方、计算方式都不同"

---

### **发现 2: 送仓日是判断堆场使用的关键** ⭐

#### **判断逻辑修复**

**旧逻辑（错误）**:
```typescript
// ❌ 比较 提 vs 卸
if (pickupDayStr !== unloadDayStr) {
  // 计算费用
}
```

**新逻辑（正确）**:
```typescript
// ✅ 比较 提 vs 送
const plannedDeliveryDate = plannedUnloadDate; // Drop off: 送 = 卸

if (pickupDayStr !== deliveryDayStr) {
  // 计算费用
}
```

#### **场景对比**

| 场景 | 提 | 送 | 卸 | 是否收费 | 原因 |
|------|---|---|---|---------|------|
| Live load | D1 | D1 | D1 | ❌ 不收费 | 直接送仓，不经堆场 |
| 假 Drop off | D1 | D1 | D1 | ❌ 不收费 | 直接送仓，未使用堆场 |
| 真 Drop off | D1 | D3 | D3 | ✅ 收费 | 在堆场放了 3 天 |

**关键结论**:
- 判断标准：`提 ≠ 送` 而不是 `提 ≠ 卸`
- 原因：送仓日是货柜离开堆场的日期
