# 免费期字段完整分析

**问题**: `lastFreeDate` 到底指的是哪个免费期？  
**答案**: 需要区分 **滞港费免费期** 和 **滞箱费免费期**

---

## 📊 **1. 免费期的定义与区别**

### **A. 滞港费免费期 (Demurrage Free Time)**

**定义**: 集装箱在**港口码头内**的免费存放时间  
**计算起点**: 船舶卸柜日 (Discharge Date)  
**计算终点**: 提柜日 (Pickup Date)  

**公式**:
```
滞港天数 = 提柜日 - 卸柜日
滞港费 = max(0, 滞港天数 - 免费天数) × 阶梯费率
```

**业务场景**:
- 货柜到港后，收货人有一定天数的免费堆存期
- 超过免费期未提柜 → 产生滞港费 (Demurrage)
- 由船公司收取

**数据库字段**:
```sql
-- process_port_operations 表
last_free_date DATE,              -- 最后免费日（基于卸船日 + 免费天数）
free_storage_days INT,            -- 免费堆存天数（如 7 天）
```

**示例**:
```
卸柜日：2026-03-20
免费天数：7 天
最后免费日：2026-03-27

提柜日：2026-03-25 → 无滞港费
提柜日：2026-03-30 → 滞港费 = 3 天 × $150/天 = $450
```

---

### **B. 滞箱费免费期 (Detention Free Time)**

**定义**: 使用**集装箱箱体**的免费时间  
**计算起点**: 提柜日 (Pickup Date)  
**计算终点**: 还箱日 (Return Date)  

**公式**:
```
滞箱天数 = 还箱日 - 提柜日
滞箱费 = max(0, 滞箱天数 - 免费天数) × 日费率
```

**业务场景**:
- 提柜后，收货人有空箱使用的免费期
- 超过免费期未还箱 → 产生滞箱费 (Detention)
- 由船公司或车队收取

**数据库字段**:
```sql
-- process_port_operations 表
free_detention_days INT,          -- 免费滞箱天数（如 5 天、7 天）
```

**示例**:
```
提柜日：2026-03-25
免费天数：7 天
最晚还箱日：2026-04-01

还箱日：2026-03-30 → 无滞箱费
还箱日：2026-04-05 → 滞箱费 = 4 天 × $100/天 = $400
```

---

### **C. 免费离港期 (Free Off-Terminal Days)**

**定义**: 货物从港口运出的免费时间（较少使用）  
**计算起点**: 卸柜日  
**计算终点**: 货物离开港口的日期  

**数据库字段**:
```sql
free_off_terminal_days INT,       -- 免费离港天数
```

## 🔍 **2. 当前系统中的 lastFreeDate**

### **数据来源**

**表**: `process_port_operations`  
**字段**: `last_free_date`

**含义**: **滞港费的最后免费日**（不是滞箱费！）

**计算逻辑**:
```sql
-- 在 port_operation_generate_dates.sql 中
last_free_date = discharged_time::date + free_storage_days
```

**代码位置**: `intelligentScheduling.service.ts`

```typescript
// 获取目的港操作记录
const destPo = container.portOperations?.find(
  (po: any) => po.portType === 'destination'
);

// 使用 last_free_date 作为免费期截止
const lastFreeDate = destPo.lastFreeDate;
```

## ⚠️ **3. 混淆点分析**

### **问题 1: 名称混淆**

**现状**:
- `lastFreeDate` 在代码中被简称为"免费期截止"
- 没有明确说明是**滞港费**还是**滞箱费**

**影响**:
- 开发者误以为是一个通用的免费期
- 成本计算时可能用错基准

---

### **问题 2: 还箱日计算时的误用**

**代码**: `intelligentScheduling.service.ts:585-590`

```typescript
// ❌ 错误：用滞港费的 lastFreeDate 推算滞箱费的还箱日
if (emptyReturn?.lastReturnDate) {
  lastReturnDate = new Date(emptyReturn.lastReturnDate);
} else if (destPo.lastFreeDate) {
  // fallback: 从 lastFreeDate + 免费用箱天数计算（默认 7 天）
  lastReturnDate = new Date(destPo.lastFreeDate);
  lastReturnDate.setDate(lastReturnDate.getDate() + 7);
}
```

**问题**:
- `destPo.lastFreeDate` 是**滞港费免费期**（基于卸柜日）
- 加上 7 天后得到的是**假设的还箱日**
- 但这个还箱日与实际合同中的**滞箱费免费期**可能不一致

**正确做法**:
应该从 `EmptyReturn` 表获取真实的 `lastReturnDate`（基于提柜日 + 滞箱费免费天数）

---

### **问题 3: 成本优化时的误用**

**代码**: `schedulingCostOptimizer.service.ts`

```typescript
// 生成 Expedited 方案（仅在免费期内）
if (candidateDate <= lastFreeDate) {
  strategies.push('Expedited');
}
```

**问题**:
- 这里的 `lastFreeDate` 用于判断是否可以免滞港费 ✅ 正确
- 但**没有考虑滞箱费免费期** ❌ 不完整

**应该同时检查**:
```typescript
const isWithinDemurrageFreePeriod = candidateDate <= demurrageLastFreeDate;
const isWithinDetentionFreePeriod = (returnDate - pickupDate) <= detentionFreeDays;
```

## ✅ **4. 正确的数据模型**

### **A. 两个独立的免费期**

```typescript
interface PortOperation {
  // 滞港费相关
  lastFreeDate: DATE;             // 最后免费日（基于卸柜日）
  freeStorageDays: INT;           // 免费堆存天数
  
  // 滞箱费相关
  freeDetentionDays: INT;         // 免费滞箱天数（从提柜日开始）
  lastReturnDate: DATE;           // 最晚还箱日（提柜日 + freeDetentionDays）
}
```

---

### **B. 计算逻辑分离**

```typescript
// ✅ 正确：分别计算两种免费期
class CostCalculator {
  // 计算滞港费
  calculateDemurrage(pickupDate: Date, dischargeDate: Date, freeDays: number): number {
    const daysOutsideFreePeriod = daysBetween(dischargeDate, pickupDate) - freeDays;
    return max(0, daysOutsideFreePeriod) * tieredRate;
  }
  
  // 计算滞箱费
  calculateDetention(pickupDate: Date, returnDate: Date, freeDays: number): number {
    const detentionDays = daysBetween(pickupDate, returnDate) - freeDays;
    return max(0, detentionDays) * dailyRate;
  }
}
```

## 📊 **8. 业务影响分析**

### **当前情况（混淆使用）**

**问题**:
- 用滞港费的 `lastFreeDate` 推算还箱日
- 可能导致滞箱费计算不准确
- 成本优化建议可能有偏差

**示例**:
```
滞港费免费期：2026-03-27 (卸柜日 03-20 + 7 天)
推算还箱日：2026-04-03 (03-27 + 7 天)

实际滞箱费免费期：5 天（合同约定）
实际最晚还箱日：2026-03-30 (提柜日 03-25 + 5 天)

差异：3 天 → 可能导致滞箱费计算错误
```

**好处**:
1. ✅ 滞港费计算准确（使用正确的免费天数）

3. ✅ 成本优化建议更可靠
4. ✅ 用户可以看到两种费用的明细
