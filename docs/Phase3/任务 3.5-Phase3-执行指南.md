## ❌ 原问题

### 生成列插入错误

**错误的 SQL**:
```sql
INSERT INTO ext_warehouse_daily_occupancy 
(warehouse_code, date, planned_count, capacity, remaining, created_at, updated_at)
VALUES (..., capacity_value, ...);
-- ERROR: cannot insert a non-DEFAULT value into column "remaining"
-- DETAIL: Column "remaining" is a generated column.
```

**原因**: `remaining` 字段是**生成列**（Generated Column），由数据库自动计算，不能手动插入值

---

## ✅ 解决方案

### 正确的 SQL 语法

不指定 `remaining` 字段，让数据库自动计算：

```sql
-- ✅ 正确：不指定 remaining 字段
INSERT INTO ext_warehouse_daily_occupancy 
(warehouse_code, date, planned_count, capacity, created_at, updated_at)
VALUES (
  wh_record.warehouse_code,
  start_date + i,
  0,  -- planned_count
  capacity_value,  -- capacity
  NOW(),
  NOW()
)
ON CONFLICT (warehouse_code, date) DO UPDATE SET
  capacity = EXCLUDED.capacity,
  updated_at = NOW();
```

**数据库会自动计算**:
```sql
remaining = capacity - planned_count
```

## ⚠️ 常见问题

### 问题 1: "no data returned"

**现象**: 查询档期数据时无结果

**原因**: 
1. 没有活跃的仓库或车队
2. 日期范围不对

**解决方案**:
```sql
-- 检查是否有活跃仓库
SELECT warehouse_code, warehouse_name, status, daily_unload_capacity 
FROM dict_warehouses 
WHERE status = 'ACTIVE';

-- 检查是否有活跃车队
SELECT company_code, company_name, status, daily_capacity 
FROM dict_trucking_companies 
WHERE status = 'ACTIVE';

-- 如果没有，需要添加测试数据
INSERT INTO dict_warehouses (warehouse_code, warehouse_name, property_type, country, status, daily_unload_capacity)
VALUES ('WH001', '1 号仓库', '自营仓', 'US', 'ACTIVE', 10);

INSERT INTO dict_trucking_companies (company_code, company_name, country, status, daily_capacity)
VALUES ('TC001', '顺达拖车', 'US', 'ACTIVE', 10);
```

### 问题 2: 周末 capacity 不为 0

**现象**: 周末的 capacity 仍然有值

**原因**: 数据不是通过 SQL 脚本生成的，或者手动修改过

**解决方案**:
```sql
-- 重新运行初始化脚本
psql -U postgres -d logix -f migrations/add_calendar_based_capacity.sql

-- 或者手动更新周末数据
UPDATE ext_warehouse_daily_occupancy 
SET capacity = 0, remaining = 0
WHERE EXTRACT(DOW FROM date) IN (0, 6);  -- 周六、周日
```

### 问题 3: remaining 计算不正确

**现象**: remaining ≠ capacity - planned_count

**原因**: 如果 `remaining` 字段定义为 `GENERATED ALWAYS AS (...) STORED`，则会自动计算

**解决方案**: 检查表结构
```sql
\d ext_warehouse_daily_occupancy

-- 如果是普通列，可以手动更新
UPDATE ext_warehouse_daily_occupancy 
SET remaining = capacity - planned_count;

-- 如果是生成列，无需操作，数据库会自动维护
```
