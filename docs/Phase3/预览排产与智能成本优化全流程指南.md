## 1. 核心概念与术语

### 1.1 状态定义

| 状态 | 英文 | 说明 |
|------|------|------|
| 待排产 | initial | 货柜刚导入，尚未进行排产 |
| 已排产 | issued | 已完成排产，生成计划日期 |
| 已派工 | dispatched | 排产结果已下发给仓库/车队 |

### 1.2 五个计划日期

| 字段 | 中文名 | 计算逻辑 |
|------|--------|----------|
| plannedCustomsDate | 计划清关日 | 提柜日 - 1 天 |
| plannedPickupDate | 计划提柜日 | 清关日 + 1 天（若过期则至少为今天） |
| plannedDeliveryDate | 计划送仓日 | 提柜日 + 运输天数 |
| plannedUnloadDate | 计划卸柜日 | **核心锚点**，基于仓库可用性和日产能计算 |
| plannedReturnDate | 计划还箱日 | 卸柜日 + 在仓天数 |

### 1.3 三种卸柜方式

| 方式 | 英文 | 适用场景 | 成本特点 |
|------|------|----------|----------|
| 直装 | Direct | 提柜日当天有产能 | 无堆场费 |
| Drop off | Drop off | 提柜日无产能，需先拉到堆场 | 有堆场操作费 + 堆存费 |
| 加急 | Expedited | 免费期即将到期，紧急处理 | 有加急费 |

### 1.4 关键日期口径

**最后免费日 (last_free_date)**:
- 计算公式：基准日 + (免费天数 - 1) 天
- 基准日优先级：修正 ETA → ETA → ATA → 实际卸船日
- 示例：ETA=2026-02-16，免费天数=7 → last_free_date=2026-02-22

**最晚还箱日 (last_return_date)**:
- 计算公式：(实际提柜日 或 last_free_date) + 免费用箱天数
- 免费天数来自滞箱费标准表
- 示例：last_free_date=2026-02-22，免费用箱天数=7 → last_return_date=2026-03-01

### 2.1 完整业务流程

```
┌─────────────────────────────────────────────────────────────┐
│                    预览排产流程                              │
├─────────────────────────────────────────────────────────────┤
│ 1. 查询待排产货柜（schedule_status = initial）               │
│ 2. 按清关可放行日排序（先到先得）                            │
│ 3. 为每个货柜匹配滞港费标准，计算最晚提柜日                  │
│ 4. 根据目的港选择候选仓库和车队                             │
│ 5. 计算五个计划日期                                         │
│ 6. 预估成本（Demurrage、Detention、Transportation）         │
│ 7. 返回预览结果（不保存）                                    │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                  智能成本优化流程                            │
├─────────────────────────────────────────────────────────────┤
│ 1. 生成所有可行方案（Direct/Drop off/Expedited）            │
│ 2. 评估每个方案的总成本                                     │
│ 3. 选择成本最低的方案                                       │
│ 4. 提供优化建议（可节省金额）                               │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                  应用选择方案流程                            │

│ 1. 用户确认应用方案                                         │
│ 2. 执行正式排产（dryRun=false）                             │
```

```typescript
async batchSchedule(request: ScheduleRequest): Promise<BatchScheduleResponse> {
  const { dryRun = false } = request
  
  // 1. 查询待排产货柜
  const containers = await this.getContainersToSchedule(request)
  
  // 2. 预填充 lastFreeDate（用于后续计算）
  await this.preFillLastFreeDates(containers)
  
  // 3. 对每个货柜进行排产
  const results = []
  for (const container of containers) {
    const result = await this.scheduleSingleContainer(container, request)
    results.push(result)
  }
  
  return {
    total: containers.length,
    successCount: results.filter(r => r.success).length,
    results
  }
}
```

```typescript
// backend/src/services/intelligentScheduling.service.ts
private async scheduleSingleContainer(
  container: Container, 
  request: ScheduleRequest
): Promise<ScheduleResult> {
  // 1. 获取目的港信息
  const destPo = container.portOperations?.find(po => po.portType === 'destination')
  
  // 2. 获取最后免费日
  const lastFreeDate = destPo?.lastFreeDate
    ? new Date(destPo.lastFreeDate)
    : dateTimeUtils.addDays(new Date(destPo.eta || Date.now()), 7)
  
  // 3. 计算计划清关日
  const plannedCustomsDate = await this.calculatePlannedCustomsDate()
  
  // 4. 计算计划提柜日 (= 清关日 + 1 天，且 ≤ last_free_date)
  const plannedPickupDate = await this.calculatePlannedPickupDate(
    plannedCustomsDate, 
    lastFreeDate
  )
  
  // 5. 计算计划卸柜日（核心锚点）
  const { plannedUnloadDate, warehouse, truckingCompany, unloadMode } = 
    await this.calculatePlannedUnloadDate(
      container, 
      plannedPickupDate, 
      lastFreeDate
    )
  
  // 6. 计算计划还箱日
  const plannedReturnDate = await this.calculatePlannedReturnDate(
    container, 
    plannedUnloadDate, 
    lastFreeDate
  )
  
  // 7. 预估成本
  const estimatedCosts = await this.calculateEstimatedCosts(
    container.containerNumber,
    plannedPickupDate,
    plannedUnloadDate,
    plannedReturnDate,
    unloadMode,
    warehouse,
    truckingCompany
  )
  
  // 8. 返回结果（dryRun 模式下不保存）
  return {
    containerNumber: container.containerNumber,
    success: true,
    plannedData: {
      plannedCustomsDate,
      plannedPickupDate,
      plannedDeliveryDate: plannedUnloadDate, // Drop off: 送 = 卸
      plannedUnloadDate,
      plannedReturnDate
    },
    estimatedCosts,
    warehouseName: warehouse.warehouseName
  }
}
```

### 4.1 成本优化服务入口

```typescript
// backend/src/services/schedulingCostOptimizer.service.ts
async optimizePickupDate(
  containerNumber: string,
  warehouse: Warehouse,
  truckingCompany: TruckingCompany,
  basePickupDate: Date,
  lastFreeDate?: Date
): Promise<{
  suggestedPickupDate: Date;
  suggestedStrategy: 'Direct' | 'Drop off' | 'Expedited';
  originalCost: number;
  optimizedCost: number;
  savings: number;
  savingsPercent: number;
  alternatives: Array<{
    pickupDate: Date;
    strategy: 'Direct' | 'Drop off' | 'Expedited';
    totalCost: number;
    savings: number;
  }>;
}> {
  // 1. 获取货柜信息
  const container = await this.containerRepo.findOne({
    where: { containerNumber },
    relations: ['portOperations']
  })
  
  // 2. 从权威数据源获取免费期（SKILL 原则）
  const demurrageResult = await this.demurrageService.calculateForContainer(
    containerNumber
  )
  
  let effectiveLastFreeDate: Date
  if (demurrageResult.result?.calculationDates?.lastPickupDateComputed) {
    // 优先使用 DB 中的 last_free_date
    effectiveLastFreeDate = new Date(
      demurrageResult.result.calculationDates.lastPickupDateComputed
    )
  } else if (lastFreeDate) {
    // 其次使用传入参数
    effectiveLastFreeDate = lastFreeDate
  } else {
    // 最后兜底：basePickupDate + 7 天
    effectiveLastFreeDate = dateTimeUtils.addDays(basePickupDate, 7)
  }
  
  // 3. 计算当前方案成本
  const currentOption: UnloadOption = {
    containerNumber,
    warehouse,
    plannedPickupDate: basePickupDate,
    strategy: truckingCompany.hasYard ? 'Drop off' : 'Direct',
    isWithinFreePeriod: basePickupDate <= effectiveLastFreeDate
  }
  
  const currentBreakdown = await this.evaluateTotalCost(currentOption)
  const originalCost = currentBreakdown.totalCost
  
  // 4. 向前/向后探索最多 7 天
  const searchRange = 7
  const alternatives = []
  
  for (let offset = -searchRange; offset <= searchRange; offset++) {
    const candidateDate = dateTimeUtils.addDays(basePickupDate, offset)
    
    // 跳过周末（如果配置了）
    if (this.isWeekend(candidateDate) && await this.shouldSkipWeekends()) {
      continue
    }
    
    // 检查仓库档期
    if (!(await this.isWarehouseAvailable(warehouse, candidateDate))) {
      continue
    }
    
    // 评估成本
    const option: UnloadOption = {
      containerNumber,
      warehouse,
      plannedPickupDate: candidateDate,
      strategy: this.determineStrategy(offset, candidateDate, effectiveLastFreeDate),
      isWithinFreePeriod: candidateDate <= effectiveLastFreeDate
    }
    
    const breakdown = await this.evaluateTotalCost(option)
    
    alternatives.push({
      pickupDate: candidateDate,
      strategy: option.strategy,
      totalCost: breakdown.totalCost,
      savings: originalCost - breakdown.totalCost
    })
  }
  
  // 5. 选择最优方案
  const best = alternatives.sort((a, b) => a.totalCost - b.totalCost)[0]
  
  return {
    suggestedPickupDate: best.pickupDate,
    suggestedStrategy: best.strategy,
    originalCost,
    optimizedCost: best.totalCost,
    savings: originalCost - best.totalCost,
    savingsPercent: ((originalCost - best.totalCost) / originalCost) * 100,
    alternatives
  }
}
```

### 4.2 生成所有可行方案

```typescript
// backend/src/services/schedulingCostOptimizer.service.ts
async generateAllFeasibleOptions(
  container: Container,
  plannedPickupDate: Date,
  lastFreeDate: Date
): Promise<UnloadOption[]> {
  const options: UnloadOption[] = []
  
  // 1. 获取候选仓库
  const warehouses = await this.getCandidateWarehouses(
    container.countryCode,
    container.portCode
  )
  
  // 2. 为每个仓库生成 Direct/Drop off 方案
  for (const warehouse of warehouses) {
    // 搜索免费期前 7 天到后 2 天
    for (let offset = -7; offset <= 2; offset++) {
      const candidateDate = dateTimeUtils.addDays(plannedPickupDate, offset)
      
      // 确保在合理范围内
      if (candidateDate < new Date()) continue
      if (candidateDate > dateTimeUtils.addDays(lastFreeDate, 2)) continue
      
      // 跳过周末
      if (this.isWeekend(candidateDate) && await this.shouldSkipWeekends()) {
        continue
      }
      
      // 检查仓库档期
      if (!(await this.isWarehouseAvailable(warehouse, candidateDate))) {
        continue
      }
      
      options.push({
        containerNumber: container.containerNumber,
        warehouse,
        plannedPickupDate: candidateDate,
        strategy: 'Direct',
        isWithinFreePeriod: candidateDate <= lastFreeDate
      })
      
      // 如果车队有堆场，生成 Drop off 方案
      const truckingCompanies = await this.getCandidateTruckingCompanies(warehouse)
      for (const trucking of truckingCompanies.filter(t => t.hasYard)) {
        options.push({
          containerNumber: container.containerNumber,
          warehouse,
          plannedPickupDate: candidateDate,
          strategy: 'Drop off',
          truckingCompany: trucking,
          isWithinFreePeriod: candidateDate <= lastFreeDate
        })
      }
    }
  }
  
  // 3. 如果是紧急情况（≤2 天），生成 Expedited 方案
  const expeditedOptions = await this.generateExpeditedOptions(
    container, 
    lastFreeDate
  )
  options.push(...expeditedOptions)
  
  return options
}
```

### 4.3 成本评估逻辑

```typescript
// backend/src/services/schedulingCostOptimizer.service.ts
async evaluateTotalCost(option: UnloadOption): Promise<CostBreakdown> {
  const breakdown: CostBreakdown = {
    demurrageCost: 0,
    detentionCost: 0,
    storageCost: 0,
    transportationCost: 0,
    yardStorageCost: 0,
    handlingCost: 0,
    totalCost: 0
  }
  
  // 1. 计算距离（用于估算运输费）
  const distance = this.distanceMatrix[option.warehouse.portCode]?.[option.warehouse.warehouseCode] || 50
  
  // 2. 基础运费
  breakdown.transportationCost = distance * 2 // $2/英里
  
  // 3. 堆场操作费（Drop off 模式）
  if (option.strategy === 'Drop off' && option.truckingCompany?.hasYard) {
    const tpMapping = await this.truckingPortMappingRepo.findOne({
      where: {
        truckingCompanyId: option.truckingCompany.companyCode,
        portCode: option.warehouse.portCode
      }
    })
    
    breakdown.yardStorageCost = Number(tpMapping?.yardOperationFee || 0)
    
    // 堆场堆存费（按 2 天估算）
    const dailyYardRate = Number(tpMapping?.standardRate || 0)
    breakdown.yardStorageCost += dailyYardRate * 2
  }
  
  // 4. 加急费（Expedited 模式）
  if (option.strategy === 'Expedited') {
    breakdown.handlingCost = 100 // 固定加急费
  }
  
  // 5. 总计
  breakdown.totalCost = 
    breakdown.demurrageCost + 
    breakdown.detentionCost + 
    breakdown.storageCost + 
    breakdown.transportationCost + 
    breakdown.yardStorageCost + 
    breakdown.handlingCost
  
  return breakdown
}
```

```typescript
// backend/src/services/intelligentScheduling.service.ts
async batchSchedule(request: ScheduleRequest): Promise<BatchScheduleResponse> {
  const { dryRun = false } = request
  
  // ... 前面与预览流程相同 ...

    const result = await this.scheduleSingleContainer(container, request)

    if (!dryRun && result.success) {
      // 1. 更新货柜 schedule_status

  }
```

### 5.3 写回最后提柜日和最后还箱日

```typescript
// backend/src/services/demurrage.service.ts
async runScheduledFreeDateUpdate(options: {
  limitLastFree: number
  limitLastReturn: number
}): Promise<{
  lastFreeWritten: number
  lastReturnWritten: number
  lastFreeProcessed: number
  lastReturnProcessed: number
}> {
  const stats = {
    lastFreeWritten: 0,
    lastReturnWritten: 0,
    lastFreeProcessed: 0,
    lastReturnProcessed: 0
  }
  
  // 1. 计算并更新最后提柜日
  const lastFreeRecords = await this.computeLastFreeDates(options.limitLastFree)
  stats.lastFreeProcessed = lastFreeRecords.length
  
  for (const record of lastFreeRecords) {
    const destPo = await this.portOperationRepo.findOne({
      where: { id: record.poId }
    })
    
    if (destPo && record.lastFreeDate) {
      destPo.lastFreeDate = record.lastFreeDate
      await this.portOperationRepo.save(destPo)
      stats.lastFreeWritten++
    }
  }
  
  // 2. 计算并更新最后还箱日
  const lastReturnRecords = await this.computeLastReturnDates(options.limitLastReturn)
  stats.lastReturnProcessed = lastReturnRecords.length
  
  for (const record of lastReturnRecords) {
    const destPo = await this.portOperationRepo.findOne({
      where: { id: record.poId }
    })
    
    if (destPo && record.lastReturnDate) {
      destPo.lastReturnDate = record.lastReturnDate
      await this.portOperationRepo.save(destPo)
      stats.lastReturnWritten++
    }
  }
  
  return stats
}
```

## 7. 使用规范与最佳实践

### 7.1 必须遵守的原则

#### SKILL 原则

1. **数据库优先**: 所有免费期数据必须以 process_port_operations.last_free_date 为准
2. **禁止反向**: 不要通过 UPDATE 语句修补数据，要删除错误数据后重新计算
3. **日期口径**: 所有日期筛选必须使用统一的 DateRangePicker 组件

#### 数据一致性

1. **唯一数据源**: last_free_date 只能从 process_port_operations 表读取
2. **禁止硬编码**: 不要写死免费天数，必须从 dict_demurrage_standards 读取
3. **事务保证**: 批量更新必须在事务中执行

### 7.3 常见错误与避免方法

#### 错误 1: 直接修改数据库

```sql
-- ❌ 错误：直接 UPDATE 修补
UPDATE process_port_operations 
SET last_free_date = '2026-03-25' 
WHERE container_number = 'CNT123456'

-- ✅ 正确：删除错误数据，修复逻辑，重新计算
DELETE FROM process_port_operations WHERE container_number = 'CNT123456'
-- 然后调用 DemurrageService.recalculate()
```

#### 错误 2: 硬编码免费天数

```typescript
// ❌ 错误：硬编码
const freeDays = 7

// ✅ 正确：从字典表读取
const standard = await demurrageStandardRepo.findOne({
  where: { countryCode, portCode, shippingLine }
})
const freeDays = standard.freeDays
```

#### 错误 3: 忽略仓库档期

```typescript
// ❌ 错误：不检查档期就直接安排
const plannedUnloadDate = addDays(pickupDate, 1)

// ✅ 正确：检查档期后再确定
for (let offset = 0; offset <= 7; offset++) {
  const candidate = addDays(pickupDate, offset)
  if (await isWarehouseAvailable(warehouse, candidate)) {
    plannedUnloadDate = candidate
    break
  }
}
```

### 7.4 性能优化建议

1. **批量处理**: 使用 batchSchedule 而非逐个调用 scheduleSingleContainer
2. **缓存配置**: 调度配置（如 skip_weekends）应该缓存，避免频繁查库
3. **并行计算**: 多个货柜的成本评估可以并行执行
4. **索引优化**: 确保 schedule_status、last_free_date 等字段有索引
