# 提柜日调整为明天 - 业务规则修复

## 🐛 问题描述

**现象**：

- 当前日期：2026-03-26
- 排产预览显示的提柜日：2026-03-25（过去日期）
- 用户要求：即使 `etaBufferDays=0`，提柜日也应该是**明天**（2026-03-27），而不是过去或今天

**业务规则**：

> 提柜日不能安排在过去或今天，必须是明天或未来的日期

---

## 🔍 问题分析

### 原有逻辑

```typescript
if (pickupDateStr < todayStr) {
  // 提柜日是过去日期，调整为今天
  plannedPickupDate = new Date(todayStr + "T00:00:00");
}
```

**问题**：

- ❌ 只检查"过去日期"，不检查"今天"
- ❌ 调整为"今天"，但今天已经来不及安排提柜

### 业务场景

假设今天是 **2026-03-26**：

1. **ETA = 2026-02-11**（很早的日期）
2. **etaBufferDays = 0**（不顺延）
3. **计算清关日** = ETA + 0 = 2026-02-11
4. **计算提柜日** = 清关日 + 1 = 2026-02-12
5. **兜底逻辑**：提柜日 < 今天 → 调整

**原有逻辑结果**：

- 提柜日 = 2026-03-26（今天）❌ 来不及安排

**新逻辑结果**：

- 提柜日 = 2026-03-27（明天）✅ 合理

---

## ✅ 修复方案

### 修改逻辑

```typescript
if (pickupDateStr <= todayStr) {
  // 提柜日是过去日期或今天，调整为明天
  const tomorrow = new Date(todayStr + "T00:00:00");
  tomorrow.setDate(tomorrow.getDate() + 1);
  plannedPickupDate = tomorrow;
  plannedCustomsDate.setTime(plannedPickupDate.getTime());
  plannedCustomsDate.setDate(plannedCustomsDate.getDate() - 1); // 保持 提=清关 +1
}
```

**关键变化**：

1. **条件修改**：`< todayStr` → `<= todayStr`（包含今天）
2. **调整目标**：今天 → 明天
3. **联动调整**：清关日 = 提柜日 - 1 天

---

## 📊 日期计算示例

### 场景 1：ETA 是过去日期

```typescript
今天 = 2026-03-26
ETA = 2026-02-11
etaBufferDays = 0

// 计算过程
清关日 = ETA + 0 = 2026-02-11
提柜日 = 清关日 + 1 = 2026-02-12

// 兜底逻辑（触发）
提柜日 (2026-02-12) <= 今天 (2026-03-26) → 调整为明天
明天 = 2026-03-26 + 1 = 2026-03-27 ✅

// 联动调整
清关日 = 提柜日 - 1 = 2026-03-26
```

**最终结果**：

- 清关日：2026-03-26（今天）
- 提柜日：2026-03-27（明天）✅

---

### 场景 2：ETA 是未来日期

```typescript
今天 = 2026-03-26
ETA = 2026-04-01
etaBufferDays = 0

// 计算过程
清关日 = ETA + 0 = 2026-04-01
提柜日 = 清关日 + 1 = 2026-04-02

// 兜底逻辑（不触发）
提柜日 (2026-04-02) > 今天 (2026-03-26) → 不调整 ✅

// 最终结果
清关日：2026-04-01
提柜日：2026-04-02
```

---

### 场景 3：ETA 是今天

```typescript
今天 = 2026-03-26
ETA = 2026-03-26
etaBufferDays = 0

// 计算过程
清关日 = ETA + 0 = 2026-03-26
提柜日 = 清关日 + 1 = 2026-03-27

// 兜底逻辑（触发）
提柜日 (2026-03-27) > 今天 (2026-03-26) → 不调整 ✅

// 最终结果
清关日：2026-03-26
提柜日：2026-03-27
```

**注意**：这个场景下，虽然 ETA 是今天，但提柜日是明天，符合业务要求。

## 🔧 业务规则总结

### 提柜日安排原则

1. **正常情况下**：
   - 提柜日 = 清关日 + 1 天
   - 清关日 = ETA + etaBufferDays

2. **兜底逻辑**：
   - 如果提柜日 ≤ 今天 → 调整为明天
   - 清关日 = 提柜日 - 1 天（保持 提=清关 +1）

3. **业务约束**：
   - ✅ 提柜日必须是明天或未来日期
   - ❌ 不能是今天或过去日期
   - ✅ 给实际操作预留足够时间

---

## 🚨 注意事项

1. **日期比较使用字符串**：避免时区问题

   ```typescript
   const todayStr = today.toISOString().split("T")[0]; // "2026-03-26"
   ```

2. **联动调整**：调整提柜日时，清关日也要相应调整

   ```typescript
   plannedCustomsDate.setDate(plannedCustomsDate.getDate() - 1);
   ```

3. **日志记录**：记录调整后的日期，便于调试
   ```typescript
   logger.debug(`Pickup date adjusted to tomorrow (${tomorrowStr})`);
   ```
