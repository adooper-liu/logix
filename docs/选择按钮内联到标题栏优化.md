# 选择按钮内联到标题栏优化

## 📋 优化概述

将**选择操作区**（全选成功项、取消全选）从独立容器移动到**标题栏内**，与确认保存按钮在同一行，体现"选择 → 确认"的一站式操作流程。

## 🎯 优化目标

### 问题

- 选择操作区独立成行，占用额外空间
- 选择按钮与确认按钮分离，操作流程不连贯
- 用户需要上下移动视线，操作效率低
- 布局不够紧凑

### 目标

- **内联布局**：选择按钮与确认按钮在同一行
- **流程连贯**：从左到右：选择 → 确认
- **节省空间**：移除独立容器，减少行数
- **提升效率**：一眼看到所有操作按钮

## ✨ 调整内容

### 1. 布局调整

**优化前：**

```
┌─────────────────────────────────────────────┐
│ 标题 + 确认保存 + 放弃                       │
├─────────────────────────────────────────────┤
│ [已选] [全选] [取消]                         │ ← 独立一行
├─────────────────────────────────────────────┤
│ 统计徽章...                                  │
└─────────────────────────────────────────────┘
```

**优化后：**

```
┌─────────────────────────────────────────────┐
│ 标题 + [已选] [全选] [取消] │ [确认] [放弃] │ ← 同一行
├─────────────────────────────────────────────┤
│ 统计徽章...                                  │
└─────────────────────────────────────────────┘
```

### 2. 视觉分组

```
标题栏（同一行）
┌──────────────────────────────────────────┐
│ 排产预览 │ [已选] [全选] [取消] │ [确认] [放弃] │
│          │    选择操作区    │ 确认操作区 │
│          │ ← 分隔线 →      │            │
└──────────────────────────────────────────┘
```

## 🎨 设计要点

### 1. 视觉层次清晰

**标题栏（从左到右）：**

- 左侧：标题 + 标签
- 中间：选择操作区（已选 + 全选 + 取消）
- 右侧：确认操作区（确认保存 + 放弃）

### 2. 操作流程明确

**选择 → 确认 一气呵成：**

```
[已选 5 个] [全选] [取消]  →  [确认保存] [放弃]
     ↓                            ↓
选择要保存的                    立即确认
```

### 3. 视觉分组

**分隔线设计：**

- 选择操作区右侧：`1px solid #dcdfe6`
- 视觉上自然分组
- 引导从左到右的操作流程

### 4. 色彩搭配

- **已选标签**：橙色（`type="warning"`）
- **全选按钮**：蓝色边框（`plain`）
- **取消按钮**：灰色边框（`plain`）
- **分隔线**：灰色（`#dcdfe6`）
- **确认保存**：绿色（`type="success"`）
- **放弃**：灰色边框（`plain`）
