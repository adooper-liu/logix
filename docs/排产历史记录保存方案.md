# 排产历史记录保存方案

## 📋 方案概述

本方案用于记录每次排产确认的完整历史，包括：

- **排产方案详情**：策略、日期、资源、费用等
- **历史版本管理**：同一货柜多次排产的版本号递增
- **备选方案记录**：保存其他可行方案供对比分析
- **审计追踪**：操作人、操作时间、操作类型

### 5. 触发器：自动版本号递增

```sql
CREATE TRIGGER trg_increment_scheduling_version
BEFORE INSERT ON hist_scheduling_records
FOR EACH ROW
EXECUTE FUNCTION increment_scheduling_version();
```

**功能**：

- 新版本递增：`scheduling_version = MAX(version) + 1`
- 旧版本标记：将原 CONFIRMED 记录标记为 SUPERSEDED

#### 2.1 查询单柜历史

```typescript
/**
 * GET /api/v1/scheduling/history/:containerNumber
 * Query params: page, limit, startDate, endDate
 */
getSchedulingHistory = async (req: Request, res: Response) => {
  const { containerNumber } = req.params;
  const { page = 1, limit = 10, startDate, endDate } = req.query;

  const query = this.schedulingHistoryRepo.createQueryBuilder("history").where("history.containerNumber = :containerNumber", { containerNumber });

  if (startDate) query.andWhere("history.operatedAt >= :startDate", { startDate });
  if (endDate) query.andWhere("history.operatedAt <= :endDate", { endDate });

  query
    .orderBy("history.schedulingVersion", "DESC")
    .skip((pageNum - 1) * limitNum)
    .take(limitNum);

  const [records, total] = await query.getManyAndCount();

  res.json({ success: true, data: { containerNumber, total, page, limit, records } });
};
```

#### 2.2 查询最新记录

```typescript
/**
 * GET /api/v1/scheduling/history/latest
 * Query params: containerNumbers[]
 */
getLatestSchedulingHistory = async (req: Request, res: Response) => {
  const { containerNumbers } = req.query;

  let query = this.schedulingHistoryRepo
    .createQueryBuilder("history")
    .innerJoin(
      (qb) => qb.select("containerNumber", "containerNumber").addSelect("MAX(schedulingVersion)", "maxVersion").from("hist_scheduling_records", "h").groupBy("containerNumber"),
      "latest",
      "history.containerNumber = latest.containerNumber AND history.schedulingVersion = latest.maxVersion",
    );

  if (containerNumbers) {
    query.where("history.containerNumber IN (:...containerNumbers)", { containerNumbers });
  }

  const records = await query.getMany();
  res.json({ success: true, data: records });
};
```

#### 2.3 保存历史记录（内部方法）

```typescript
/**
 * 在确认保存排产时调用
 * @param containerNumber 货柜号
 * @param previewResult 预览结果（包含完整方案信息）
 * @param operatedBy 操作人
 * @param manager 数据库事务管理器
 */
private async saveSchedulingHistory(
  containerNumber: string,
  previewResult: any,
  operatedBy: string,
  manager: any
): Promise<void> {
  const history = new SchedulingHistory();

  // 基本信息
  history.containerNumber = containerNumber;
  history.schedulingMode = previewResult.mode || 'AUTO';
  history.strategy = previewResult.strategy || 'Direct';

  // 日期信息
  history.plannedPickupDate = previewResult.plannedPickupDate ? new Date(previewResult.plannedPickupDate) : undefined;
  history.plannedDeliveryDate = previewResult.plannedDeliveryDate ? new Date(previewResult.plannedDeliveryDate) : undefined;
  history.plannedUnloadDate = previewResult.plannedUnloadDate ? new Date(previewResult.plannedUnloadDate) : undefined;
  history.plannedReturnDate = previewResult.plannedReturnDate ? new Date(previewResult.plannedReturnDate) : undefined;

  // 资源信息
  history.warehouseCode = previewResult.warehouseCode || undefined;
  history.truckingCompanyCode = previewResult.truckingCompanyCode || undefined;

  // 费用信息
  if (previewResult.costBreakdown) {
    history.totalCost = previewResult.costBreakdown.totalCost || undefined;
    history.demurrageCost = previewResult.costBreakdown.demurrageCost || undefined;
    // ... 其他费用
  }

  // 备选方案
  if (previewResult.alternatives && Array.isArray(previewResult.alternatives)) {
    history.alternativeSolutions = previewResult.alternatives;
  }

  // 操作信息
  history.operatedBy = operatedBy || 'SYSTEM';
  history.operationType = 'CREATE';
  history.schedulingStatus = 'CONFIRMED';

  await manager.save(history);
}
```

### 3. 集成到确认保存流程

文件：`backend/src/controllers/scheduling.controller.ts`

在 `confirmSchedule` 方法中，保存排产结果时同时调用：

```typescript
private async savePreviewResults(previewResults: any[]): Promise<any> {
  const queryRunner = AppDataSource.createQueryRunner();
  await queryRunner.connect();
  await queryRunner.startTransaction();

  try {
    for (const previewResult of previewResults) {
      const { containerNumber } = previewResult;

      // 1. 保存排产结果到流程表（现有逻辑）
      await this.saveSchedulingResult(previewResult, queryRunner.manager);

      // 2. 💾 新增：保存历史记录
      await this.saveSchedulingHistory(
        containerNumber,
        previewResult,
        'SYSTEM',  // 或从认证信息获取
        queryRunner.manager
      );

      // 3. 扣减资源档期（现有逻辑）
      await this.occupyCapacity(previewResult, queryRunner.manager);
    }

    await queryRunner.commitTransaction();
    return { success: true, message: '排产成功' };
  } catch (error) {
    await queryRunner.rollbackTransaction();
    throw error;
  } finally {
    await queryRunner.release();
  }
}
```

## 📝 注意事项

1. **性能优化**：
   - 大数据量时考虑分区表（按月/年）
   - 定期归档旧数据（如超过 1 年的记录）

2. **数据一致性**：
   - 历史记录保存与排产结果保存在同一事务中
   - 失败时回滚，避免数据不一致

3. **扩展性**：
   - 可添加更多分析字段（如排产耗时、资源使用率等）
   - 可集成到 BI 系统进行数据分析

4. **安全性**：
   - 建议添加权限控制（仅授权用户可查询历史）
   - 敏感操作记录操作人信息
