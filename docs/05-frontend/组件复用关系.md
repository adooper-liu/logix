## 四、组件逻辑关系

### 4.1 时间显示组件层次

```
DurationDisplay (时长显示 - 基础)
    │
    ├── KeyDatesTimeline.vue      # 使用: 历时/倒计时/超期
    │
    └── NodeDurationDisplay       # 使用: 物流路径节点
            │
            └── LogisticsPathTab.vue
```

**逻辑说明**:
- `DurationDisplay`: 通用时长显示组件
- `NodeDurationDisplay`: 扩展支持节点特定样式(历时/超期)
- `KeyDatesTimeline`: 关键日期时间线，整合多种时长显示

### 4.2 日期编辑组件层次

```
ContainerDateEditDialog (日期编辑对话框)
    │
    └── ContainerDetailSidebar.vue # 使用: 编辑关键日期
```

### 4.3 甘特图组件层次

```
SimpleGanttChart (简化版)
    │
    ├── GanttStatisticsPanel      # 统计
    ├── GanttSearchBar            # 搜索
    ├── DateRangeSelector         # 日期选择
    ├── GanttTimelineHeader       # 时间轴
    ├── GanttPortGroup            # 港口分组
    │       └── 货柜圆点
```

## 五、组件状态管理

### 5.1 Props 传递模式

```
父组件 → 子组件 (单向数据流)
    │
    ├── Shipments.vue
    │       │
    │       └── props: { containers, filters, pagination }
    │
    └── SchedulingFilterBar.vue
            │
            └── props: { dateRange, filters }
```

### 5.2 事件通信模式

```
子组件 → 父组件 (事件)
    │
    ├── DateRangePicker
    │       │
    │       └── emit('update:modelValue', value)
    │
    └── ContainerContextMenu
            │
            └── emit('action', { type, container })
```

### 5.3 插槽(Slot)模式

```
通用组件提供插槽
    │
    ├── ContainerDetailSkeleton
    │       │
    │       └── <slot name="header"> / <slot name="content">
    │
    └── AlertCenter
            │
            └── <slot> 默认内容
```

## 六、复用组件设计原则

### 6.1 通用组件识别标准

| 条件 | 示例 |
|------|------|
| 被3+模块使用 | DateRangePicker |
| 功能独立完整 | UniversalImport |
| 无业务耦合 | DurationDisplay |
| 可配置性强 | GanttChart |

### 6.2 业务组件封装

```
业务组件 = 通用组件 + 业务逻辑
    │
    ├── DemurrageDetailSection
    │       │
    │       ├── DemurrageCalculator  # 通用
    │       ├── API调用              # 业务
    │       └── 业务状态管理          # 业务
    │
    └── LastPickupDateDisplay
            │
            ├── 样式                 # 通用
            ├── 格式化逻辑           # 通用
            └── 业务数据转换         # 业务
```

### 6.3 组件分层

```
第一层: 基础UI组件 (Element Plus)
    ├── el-button, el-input, el-table, el-dialog...
    │
第二层: 通用业务组件
    ├── DateRangePicker, DurationDisplay, SankeyChart...
    │
第三层: 业务模块组件
    ├── DemurrageDetailSection, GanttPortGroup...
    │
第四层: 页面级组件
    ├── Shipments.vue, Dashboard.vue...
```

## 七、组件复用最佳实践

### 7.1 新增组件决策树

```
需要新增组件?
    │
    ├── 是否被多个模块使用?
    │       │
    │       ├── 是 → 放入 common/
    │       │
    │       └── 否 → 是否功能独立完整?
    │               │
    │               ├── 是 → 放入 common/
    │               │
    │               └── 否 → 放入对应模块
    │
    └── 组件是否可以进一步拆分?
            │
            ├── 是 → 拆分为子组件
            │
            └── 否 → 保持组件完整性
```

### 7.2 组件命名规范

| 类型 | 命名示例 | 说明 |
|------|----------|------|
| 通用组件 | DateRangePicker | 功能+类型 |
| 业务组件 | DemurrageDetailSection | 业务+功能+类型 |
| 展示组件 | StatsCard | 功能+类型 |
| 容器组件 | ContainerDetailSidebar | 容器+内容 |
| 骨架屏 | ContainerDetailSkeleton | 容器+Skeleton |

### 7.3 组件文档要求

```
通用组件必须有:
    │
    ├── Props 接口定义
    │       │
    │       └── type Props = { modelValue?: DateRange }
    │
    ├── Events 事件定义
    │       │
    │       └── emit('update:modelValue', value)
    │
    ├── Slots 插槽定义
    │       │
    │       └── <slot name="header"> 默认内容
    │
    └── 使用示例
            │
            └── <DateRangePicker v-model="range" />
```
