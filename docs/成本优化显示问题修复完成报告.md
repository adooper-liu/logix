### 2.3 费用计算修复 ✅

**文件**: `backend/src/services/demurrage.service.ts`

**问题分析**:
- `calculateTotalCost` 调用 `calculateForContainer` 时使用的是数据库中的原计划日期
- 成本优化需要评估不同候选日期的费用差异
- 必须支持"临时覆盖计划日期"进行试算

**解决方案**: 创建新的内部方法 `calculateTotalCostWithPlannedDates`

#### 1. 修改 `calculateTotalCost` 入口
```typescript
async calculateTotalCost(containerNumber: string, options?: {...}) {
  // ✅ 如果传入了 plannedDates，使用临时覆盖逻辑
  if (options?.plannedDates) {
    return await this.calculateTotalCostWithPlannedDates(containerNumber, options);
  }
  
  // 否则使用原有逻辑（从数据库读取计划日期）
  const demurrageResult = await this.calculateForContainer(...);
  // ...
}
```

#### 2. 新增 `calculateTotalCostWithPlannedDates` 方法
```typescript
private async calculateTotalCostWithPlannedDates(
  containerNumber: string,
  options: { plannedDates: {...} }
) {
  // 1. 获取基础参数
  const params = await this.getContainerMatchParams(containerNumber);
  
  // 2. ✅ 关键：临时覆盖 calculationDates
  const originalPlannedPickupDate = params.calculationDates.plannedPickupDate;
  const originalPlannedReturnDate = params.calculationDates.plannedReturnDate;
  
  params.calculationDates.plannedPickupDate = options.plannedDates.plannedPickupDate;
  params.calculationDates.plannedReturnDate = options.plannedDates.plannedReturnDate;
  
  // 3. 调用 calculateForContainer（会使用覆盖后的日期）
  const demurrageResult = await this.calculateForContainer(...);
  
  // 4. 恢复原始日期（防御性编程）
  params.calculationDates.plannedPickupDate = originalPlannedPickupDate;
  params.calculationDates.plannedReturnDate = originalPlannedReturnDate;
  
  // 5. 返回费用结果
  return costs;
}
```

**效果验证**:

| 提柜日期 | 修改前费用 | 修改后费用 | 说明 |
|----------|-----------|-----------|------|
| 3-28 | $2,900 | $2,900 | 基准 |
| 3-29 | $2,900 ❌ | $2,950 ✅ | 增加 1 天，+$50 |
| 3-30 | $2,900 ❌ | $3,000 ✅ | 增加 2 天，+$100 |

**核心改进**: 
- ✅ 不同日期的费用正确递增
- ✅ 已超期的每天产生额外费用
- ✅ 支持成本优化场景的试算需求
