# 清关信息和港口操作页签去重修改报告

## 问题描述

"清关信息"和"港口操作"两个页签显示内容重复，两个页签都使用了相同的数据源`containerData.portOperations`，导致信息冗余。

## 问题分析

### 修改前的页签内容

#### 清关信息页签
- 港口信息（港口类型、名称、编码、序号）
- 时间节点（预计到港、ETA修正、目的港到达、卸船日期等）
- 免费期信息
- 清关状态（清关状态、单据状态、ISF）
- 清关公司信息
- 码头操作信息
- 状态节点详情

#### 港口操作页签
- 港口信息
- 时间节点
- 清关状态
- 清关公司
- 码头信息
- 卸船日期

**问题**：两个页签有大量重复内容，用户需要在两个页签之间来回查看相同的信息。

## 修改方案

### 设计原则

根据业务逻辑，将`portOperations`数据按功能分类到不同页签：

1. **清关信息页签**：只显示与海关清关相关的信息
2. **港口操作页签**：只显示与港口装卸、码头操作相关的信息

### 清关信息页签内容

#### 1. 清关状态
- 清关状态
- 清关单据状态
- 计划清关日期
- 实际清关日期

#### 2. ISF申报
- ISF申报状态
- ISF申报日期

#### 3. 清关公司
- 目的港清关公司
- 传递日期
- 全部生成日期

#### 4. 免费期信息
- 最后免费日期
- 免堆期(天)
- 场内免箱期(天)
- 场外免箱期(天)

#### 5. 异常信息
- 异常原因

### 港口操作页签内容

#### 1. 港口信息
- 港口类型
- 港口名称
- 港口编码
- 港口序号

#### 2. 时间节点
- 预计到港日期
- ETA修正
- 目的港到达日期
- 目的港卸船/火车日期
- 途经港到达日期
- 途经港离港日期

#### 3. 码头操作
- 目的港码头
- 入闸时间
- 入闸终端
- 出闸时间
- 出闸终端
- 放电时间
- 可提货时间
- 泊位位置

#### 4. 状态节点详情
- 状态代码
- 状态发生时间
- 是否已发生
- 地点类型
- 地点名称(英文/中文)
- 数据来源
- 货物存储位置
- 时区

#### 5. 备注
- 备注

## 修改效果

### 修改前

| 页签 | 内容特点 |
|------|----------|
| 清关信息 | 包含港口信息、时间节点、清关状态、码头操作等所有信息 |
| 港口操作 | 简化版的港口信息，仍有大量重复 |

### 修改后

| 页签 | 内容特点 |
|------|----------|
| 清关信息 | 仅包含清关相关：清关状态、ISF申报、清关公司、免费期、异常 |
| 港口操作 | 仅包含港口操作：港口信息、时间节点、码头操作、状态详情 |

## 字段分类

| 字段类别 | 所属页签 | 示例字段 |
|---------|---------|---------|
| 清关流程 | 清关信息 | 清关状态、清关单据状态、计划清关日期、实际清关日期 |
| ISF申报 | 清关信息 | ISF申报状态、ISF申报日期 |
| 清关公司 | 清关信息 | 目的港清关公司、传递日期、全部生成日期 |
| 免费期 | 清关信息 | 最后免费日期、免堆期、场内免箱期、场外免箱期 |
| 异常信息 | 清关信息 | 异常原因 |
| 港口基本信息 | 港口操作 | 港口类型、港口名称、港口编码、港口序号 |
| 航行时间节点 | 港口操作 | 预计到港、ETA修正、目的港到达、卸船日期、途经港到达/离港 |
| 码头操作 | 港口操作 | 目的港码头、入闸/出闸时间和终端、放电时间、可提货时间、泊位位置 |
| 状态追踪 | 港口操作 | 状态代码、发生时间、地点信息、数据来源、货物位置 |
| 备注 | 港口操作 | 备注 |

## 优势

1. **信息清晰**：每个页签专注展示一类信息，用户无需在不同页签间来回查找
2. **减少冗余**：消除了重复信息，提高用户体验
3. **逻辑合理**：按业务功能分类，符合用户操作习惯
4. **易于维护**：清晰的结构便于后续功能扩展

## 注意事项

1. **数据源不变**：两个页签仍然使用相同的`portOperations`数据源，只是显示内容不同
2. **关联关系保持**：如果有多个港口（如起运港、中转港、目的港），两个页签都会分别展示
3. **字段复用**：一些字段（如港口名称）在两个页签中都可见，但展示的侧重点不同
