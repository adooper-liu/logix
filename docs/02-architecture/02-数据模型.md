# 数据模型

LogiX 7层流转数据模型。

> **权威来源**: `backend/sql/schema/03_create_tables.sql`

## 表关系

```
备货单 (biz_replenishment_orders)
    ↓
货柜 (biz_containers) ──→ 海运 (process_sea_freight)
    ↓
港口操作 (process_port_operations) ←→ 拖卡运输 (process_trucking_transport)
    ↓
仓库操作 (process_warehouse_operations)
    ↓
还空箱 (process_empty_returns)
```

## 物流状态

| 状态 | 代码 | 说明 |
|------|------|------|
| 未出运 | `not_shipped` | 尚未装船 |
| 已出运 | `shipped` | 已装船 |
| 在途 | `in_transit` | 航行中 |
| 已到港 | `at_port` | 到达港口 |
| 已提柜 | `picked_up` | 已提柜到仓库/目的地 |
| 已卸柜 | `unloaded` | 已卸货 |
| 已还箱 | `returned_empty` | 已归还空箱 |

## 港口类型

| 类型 | 代码 | 说明 |
|------|------|------|
| 起运港 | `origin` | 货物出发港口 |
| 中转港 | `transit` | 途径港口 |
| 目的港 | `destination` | 最终目的港口 |

## 多港经停

一个货柜可以有多个港口操作记录，通过 `port_type` 和 `port_sequence` 区分：

```sql
SELECT * FROM process_port_operations
WHERE container_number = 'xxxxx'
ORDER BY port_sequence;
```

统计时只统计目的港（port_type = 'destination'），排除中转港。
