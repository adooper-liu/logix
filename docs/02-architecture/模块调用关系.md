## 四、核心服务依赖图

### 4.1 依赖关系

```
IntelligentSchedulingService
    │
    ├── ✅ 依赖 DemurrageService
    │       (计算预估费用)
    │
    ├── ✅ 依赖 ContainerService
    │       (查询货柜数据)
    │
    ├── ✅ 依赖 ContainerStatusService
    │       (更新状态)
    │
    └── ✅ 依赖 ExternalDataService
            (飞驼数据)

SchedulingCostOptimizerService
    │
    ├── ✅ 依赖 IntelligentSchedulingService
    │       (生成候选方案)
    │
    └── ✅ 依赖 DemurrageService
            (计算成本)

DemurrageService
    │
    ├── ✅ 依赖 ContainerRepository
    │
    ├── ✅ 依赖 SeaFreightRepository
    │
    ├── ✅ 依赖 PortOperationRepository
    │
    ├── ✅ 依赖 TruckingTransportRepository
    │
    └── ✅ 依赖 ExtDemurrageStandardRepository
            (费用标准)
```

### 4.2 无依赖关系（可独立使用）

| 服务 | 说明 |
|------|------|
| ContainerService | 基础CRUD，无特殊依赖 |
| LogisticsPathService | 纯查询，无写操作 |
| AlertService | 预警服务，独立 |
| CostAnalyticsService | 分析服务，独立 |

## 六、模块边界

| 模块 | 职责 | 边界 |
|------|------|------|
| 货柜管理 | 货柜CRUD、查询 | 只读/写 biz_containers |
| 排产 | 排产算法、成本优化 | 读写 process_trucking_transport, process_warehouse_operations |
