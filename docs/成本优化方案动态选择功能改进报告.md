# 成本优化方案动态选择功能改进报告

## 📋 概述

本次改进实现了**成本优化方案的动态选择功能**，用户可以选择不同的优化方案，所有相关数据（费用明细、优化节省、优化建议）都会**实时更新**。

**版本**: v1.6.0  
**完成时间**: 2026-03-27  
**状态**: ✅ 已完成

---

## 🎯 改进目标

### 问题清单

1. ❌ **费用对比卡片溢出** - 卡片尺寸过大，超出容器
2. ❌ **费用明细固定显示** - 无论选择哪个方案，费用明细都显示最优方案数据
3. ❌ **优化节省固定显示** - 左侧的优化节省金额不会随方案选择而变化
4. ❌ **无法切换方案** - 用户只能被动接受最优方案，无法选择其他方案

### 改进要求

1. ✅ **减小卡片尺寸** - 解决溢出问题
2. ✅ **动态费用明细** - 根据选择的方案实时更新费用明细
3. ✅ **动态优化节省** - 左侧的优化节省金额根据选择的方案变化
4. ✅ **方案选择功能** - 用户可以点击选择任意方案
5. ✅ **遵循 SKILL 原则** - 单一事实来源，数据一致性

## 🔧 核心改进

### 1. 响应式状态管理

**新增状态变量**：

```typescript
// ✅ 关键新增：跟踪用户选择的方案（默认为最优方案）
const selectedAlternativeIndex = ref(0)

// ✅ 关键新增：获取当前选择的方案
const selectedAlternative = computed(() => {
  const allAlts = props.report.allAlternatives || []
  return allAlts[selectedAlternativeIndex.value] || allAlts[0]
})

// ✅ 关键新增：根据选择的方案动态计算优化后的费用
const currentOptimizedCost = computed(() => {
  const alt = selectedAlternative.value
  if (!alt) return optimizedCost.value
  
  return {
    total: alt.totalCost,
    pickupDate: alt.pickupDate,
    strategy: alt.strategy,
    breakdown: alt.breakdown,
  }
})

// ✅ 关键新增：根据选择的方案动态计算节省金额
const currentSavings = computed(() => {
  const orig = originalCost.value.total
  const opt = currentOptimizedCost.value.total
  const amount = orig - opt
  const percentage = orig > 0 ? (amount / orig * 100) : 0
  
  return {
    amount: amount > 0 ? amount : 0,
    percentage: amount > 0 ? percentage : 0,
    explanation: generateSavingsExplanation(currentOptimizedCost.value, originalCost.value)
  }
})
```

**数据流**：

```
用户点击方案卡片
    ↓
selectedAlternativeIndex 更新
    ↓
selectedAlternative 计算新方案
    ↓
currentOptimizedCost 计算新费用
    ↓
currentSavings 计算新节省金额
    ↓
breakdownTableData 更新费用明细
    ↓
界面实时更新
```

## 🔍 技术实现细节

### SKILL 原则遵循

#### 1. **S - Single source of truth (单一事实来源)**

```typescript
// ✅ 单一事实来源：selectedAlternative
const selectedAlternative = computed(() => {
  const allAlts = props.report.allAlternatives || []
  return allAlts[selectedAlternativeIndex.value] || allAlts[0]
})

// ✅ 所有数据都从这里派生
const currentOptimizedCost = computed(() => { /* ... */ })
const currentSavings = computed(() => { /* ... */ })
const breakdownTableData = computed(() => { /* ... */ })
```

#### 2. **K - Keep it simple (保持简单)**

```typescript
// ✅ 简单的状态管理
const selectedAlternativeIndex = ref(0)  // 只需要一个索引

// ✅ 自动更新
const currentSavings = computed(() => {
  // 自动响应 selectedAlternativeIndex 的变化
})
```

#### 3. **I - Interface consistency (接口一致性)**

```typescript
// ✅ 所有计算属性都返回相同的结构
interface OptimizedCost {
  total: number
  pickupDate: string
  strategy: string
  breakdown: CostBreakdownItem
}

// ✅ currentOptimizedCost 和 optimizedCost 结构相同
const currentOptimizedCost = computed<OptimizedCost>(() => { /* ... */ })
```

#### 4. **L - Logical data flow (逻辑数据流)**

```
用户操作 → 更新状态 → 计算属性 → 界面更新
   ↓          ↓          ↓          ↓
点击方案  →  index   →  alt    →  重新渲染
```

#### 5. **L - Loose coupling (松耦合)**

```typescript
// ✅ 组件内部状态管理，不依赖外部
const selectedAlternativeIndex = ref(0)

// ✅ 通过 props 接收数据，通过 emits 发送事件
const emit = defineEmits<Emits>()
const handleAccept = () => {
  emit('accept', selectedAlternative.value)
}
```

##  已知问题与解决方案

### 问题 1：ECharts 图表溢出

**问题**：图表高度 400px，容器高度 240px，导致溢出

**解决**：
- CostTrendChart.vue: `height: '400px'` → `height: '100%'`
- OptimizationResultCard.vue: `.trend-chart-container` 使用 `height: 100%` + `min-height: 240px`

### 问题 2：SCSS 注释语法错误

**问题**：使用 `//` 注释导致编译错误

**解决**：改用 CSS 注释 `/* */`

```scss
// ❌ 错误
.cost-value {
  font-size: 18px;  // ✅ 关键修复：减小字体
}

/* ✅ 正确 */
.cost-value {
  font-size: 18px;  /* ✅ 关键修复：减小字体 */
}
```

### 问题 3：函数重复声明

**问题**：`handleSelectAlternative` 等函数被重复定义

**解决**：删除旧的函数定义，只保留新的实现

## 🎓 经验总结

### 成功经验

1. **单一事实来源**：使用一个状态变量控制所有数据
2. **响应式系统**：充分利用 Vue 的响应式特性
3. **计算属性**：自动缓存，提高性能
4. **组件化**：保持组件职责单一

### 踩坑记录

1. **SCSS 注释**：必须使用 `/* */` 而不是 `//`
2. **图表高度**：使用百分比高度时要确保父元素有高度
3. **函数重复**：避免在同一作用域内重复声明函数
4. **事件冒泡**：使用 `.stop` 修饰符阻止不必要的冒泡
