# 时间线节点显示规则修复 - 实际提柜节点历时显示

## 📋 问题描述

### 原始问题

用户反馈：实际提柜节点显示"已超期 2 天→还空箱"是错误的。

**截图信息**：

- 实际提柜日：2026/03/28
- 最晚还箱日：2026/04/04
- 当前日期：2026/03/30
- 显示：已超期 2 天→还空箱 ❌

**正确显示应为**：历时 2 天 ✅

---

## 🔍 业务场景分析

### 时间线节点分类

#### 1. **实际发生节点**（业务已完成）

- 出运
- ATA（实际到港）
- 卸船
- **实际提柜**
- 实际还箱

**特点**：

- ✅ 日期已发生（早于当前时间）
- ✅ 业务已流转到下一环节
- ✅ 应该显示**历时**（从该节点到现在的天数，或到下一节点的天数）

#### 2. **计划节点**（预期时间）

- ETA（预计到港）
- 修正 ETA

**特点**：

- 📅 可能是过去或未来日期
- 📊 如果后一节点已发生 → 显示历时
- ⏳ 如果后一节点未发生 → 显示超期/倒计时

#### 3. **关键节点**（ deadline 约束）

- 最晚提柜（LFD）
- 最晚还箱（LRD）

**特点**：

- ⚠️ 有免费期标准（如 7 天）
- 🔴 可能超期
- 🟠 可能预警
- 🟢 可能倒计时
- 🔵 已完成后显示历时

---

## 🎯 业务规则定义

### 规则 1：实际发生节点总是显示历时

**判断条件**：

```typescript
const actualEventLabels = ["出运", "ATA", "卸船", "实际提柜", "实际还箱"];
if (actualEventLabels.includes(event.label)) {
  return true; // hasNextNode = true
}
```

**显示逻辑**：

- **历时计算** = 当前日期 - 节点日期
- **显示文案**：`历时 X 天`
- **颜色**：蓝色（info）

**示例**：

```
实际提柜：2026/03/28
当前日期：2026/03/30
显示：历时 2 天（蓝色）
```

---

### 规则 2：关键节点的显示逻辑

#### 场景 A：最晚提柜 + 实际提柜已发生

**条件**：

- 最晚提柜日 < 当前时间
- 实际提柜日 < 当前时间

**显示**：

- 最晚提柜 → 显示**历时**（实际提柜 - 最晚提柜）
- 如果 历时 > 免费期 → 超期（红色）
- 如果 历时 ≤ 免费期 → 历时（蓝色）

**示例**：

```
最晚提柜：2026/03/25
实际提柜：2026/03/28
免费期：7 天
显示：历时 3 天（蓝色，未超期）
```

---

#### 场景 B：最晚提柜 + 实际提柜未发生

**条件**：

- 最晚提柜日 < 当前时间
- 实际提柜日 = null

**显示**：

- 超期天数 = 当前日期 - 最晚提柜日
- 显示：`已超期 X 天→还空箱`（红色脉冲）

**示例**：

```
最晚提柜：2026/03/20
当前日期：2026/03/30
实际提柜：未发生
显示：已超期 10 天→还空箱（红色）
```

---

#### 场景 C：最晚还箱 + 实际还箱已发生

**条件**：

- 最晚还箱日 < 当前时间
- 实际还箱日 < 当前时间

**显示**：

- 最晚还箱 → 显示**历时**（实际还箱 - 最晚还箱）
- 如果 历时 > 免费期 → 超期（红色）
- 如果 历时 ≤ 免费期 → 历时（蓝色）

**示例**：

```
最晚还箱：2026/04/01
实际还箱：2026/04/03
免费期：7 天
显示：历时 2 天（蓝色，未超期）
```

---

#### 场景 D：最晚还箱 + 实际还箱未发生

**条件**：

- 最晚还箱日 < 当前时间
- 实际还箱日 = null

**显示**：

- 超期天数 = 当前日期 - 最晚还箱日
- 显示：`已超期 X 天→实际还箱`（红色脉冲）

**示例**：

```
最晚还箱：2026/03/25
当前日期：2026/03/30
实际还箱：未发生
显示：已超期 5 天→实际还箱（红色）
```

---

#### 场景 E：最晚还箱（未来日期）

**条件**：

- 最晚还箱日 > 当前时间

**显示**：

- 倒计时天数 = 最晚还箱日 - 当前日期
- 如果 倒计时 ≤ 3 天 → 橙色预警
- 如果 倒计时 > 3 天 → 绿色正常

**示例**：

```
最晚还箱：2026/04/04
当前日期：2026/03/30
显示：倒计时 5 天（绿色）
```

## 🔧 代码修复

### 文件 1：`frontend/src/views/shipments/components/KeyDatesTimeline.vue`

**修改位置**：`getEffectiveHasNextNode` 函数

**修改内容**：

```typescript
const getEffectiveHasNextNode = (event: TimelineEvent, index: number, allEvents: TimelineEvent[]): boolean => {
  // ✅ 新增：实际发生节点总是显示历时
  const actualEventLabels = ["出运", "ATA", "卸船", "实际提柜", "实际还箱"];
  if (actualEventLabels.includes(event.label)) {
    return true;
  }

  // 最晚提柜：检查实际提柜是否已发生
  if (event.label === "最晚提柜") {
    const pickupDate = dates.value?.pickupDateActual;
    if (!pickupDate) return false;
    const pickupDateObj = typeof pickupDate === "string" ? new Date(pickupDate) : pickupDate;
    return pickupDateObj < new Date();
  }

  // 最晚还箱：检查实际还箱是否已发生
  if (event.label === "最晚还箱") {
    const returnTime = dates.value?.returnTime;
    if (!returnTime) return false;
    const returnTimeObj = typeof returnTime === "string" ? new Date(returnTime) : returnTime;
    return returnTimeObj < new Date();
  }

  // 其他节点：检查后续节点是否有实际日期
  for (let i = index + 1; i < allEvents.length; i++) {
    const nextEvent = allEvents[i];
    if (nextEvent.label !== "当前") {
      if (nextEvent.date < new Date()) {
        return true;
      }
    }
  }

  return false;
};
```

### 文件 2：`frontend/src/components/common/DurationDisplay.vue`

**修改位置**：`displayType` 计算属性

**修改内容**：

```typescript
const displayType = computed(() => {
  // 显式指定模式
  if (props.mode !== "auto") {
    if (props.mode === "countdown") return "countdown";
    if (props.mode === "elapsed") return "elapsed";
    if (props.mode === "overdue") return "overdue";
  }

  // 自动判断模式
  // 优先判断：如果有后一节点（且后一节点已发生），显示历时
  if (props.hasNextNode) {
    return "elapsed";
  }

  // 如果是当前正在进行的节点，显示超期/倒计时
  if (props.isCurrentNode) {
    const time = overdueTime.value;
    if (!time) return "elapsed";
    return time < 0 ? "countdown" : "overdue";
  }

  // 无后一节点：根据时间与标准判断
  const time = overdueTime.value;
  if (!time) return "elapsed";
  if (time < 0) return "countdown";

  // ✅ 新增：实际发生节点显示历时
  const actualEventLabels = ["出运", "ATA", "卸船", "实际提柜", "实际还箱"];
  if (actualEventLabels.includes(props.label || "") && prevDateObj.value) {
    return "elapsed";
  }

  // 实际还箱：业务终点，显示历时
  if (props.label === "实际还箱" && prevDateObj.value) {
    return "elapsed";
  }

  // 关键节点在过去：显示超期
  if (isKeyNode.value) {
    return "overdue";
  }

  // 非关键节点在过去：显示超期
  return "overdue";
});
```

## 📝 总结

### 核心业务逻辑

1. **实际发生节点**（实际提柜、实际还箱等）是业务已完成的标志，应该显示**历时**
2. **关键节点**（最晚提柜、最晚还箱）才需要显示超期/倒计时
3. **超期**只针对**未完成**的 deadline，不针对**已完成**的业务节点

### 修复范围

- ✅ `KeyDatesTimeline.vue` - 节点 `hasNextNode` 判断逻辑
- ✅ `DurationDisplay.vue` - 显示类型判断逻辑
- ✅ 实际提柜节点显示修正
- ✅ 其他实际发生节点（出运、ATA、卸船、实际还箱）一致性处理

### 影响范围
