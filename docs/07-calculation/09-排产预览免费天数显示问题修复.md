# 排产预览免费天数显示问题修复

## 📋 问题描述

用户在排产预览页面中发现，最晚提柜日（LFD）和最晚还箱日（LRD）的 Tooltip 中"免费天数"显示为 `-`，而不是实际的数字（如 7 天）。

## 🔍 问题分析

### 数据流链路

```
后端计算 → API 返回 → 前端转换 → 表格显示 → Tooltip 展示
```

### 可能的问题点

1. **后端计算问题**：免费天数未正确计算
2. **API 返回问题**：字段未包含在返回数据中
3. **前端转换问题**：字段未在数据转换时传递
4. **Tooltip 显示问题**：字段值未正确渲染

## 🔧 故障排查

### 问题 1：后端日志中没有免费天数

**可能原因**：

- 滞港费标准数据缺失
- 免费天数计算逻辑错误

**排查步骤**：

1. 检查 `ext_demurrage_standards` 表是否有数据
2. 检查 `country_code` 是否匹配
3. 查看 `demurrage.service.ts` 中的计算日志

### 问题 2：前端日志中没有免费天数

**可能原因**：

- API 返回数据中没有字段
- 数据转换时字段丢失

**排查步骤**：

1. 在浏览器开发者工具中查看 Network 标签
2. 检查 API 响应中是否包含 `pickupFreeDays` 和 `returnFreeDays`
3. 检查数据转换逻辑是否正确传递字段

### 问题 3：Tooltip 显示为 `-`

**可能原因**：

- 字段值为 `undefined` 或 `null`
- 字段类型为数字但值为 0

**排查步骤**：

1. 在浏览器控制台检查 `row.pickupFreeDays` 的值
2. 确认值是否为数字类型
3. 如果值为 0，需要修改判断逻辑：

   ```vue
   <!-- 修改前 -->
   {{ row.pickupFreeDays !== undefined ? row.pickupFreeDays : "-" }}

   <!-- 修改后 -->
   {{ row.pickupFreeDays !== undefined && row.pickupFreeDays !== null ? row.pickupFreeDays : "-" }}
   ```
