# 时间线节点显示修复 - 实际业务节点不应显示超期

## 📋 问题描述

在实际业务节点（出运、ATA、实际提柜）上，显示了"实际超期 X 天"，这是不合理的。

### 问题截图

```
出运 → 显示"实际超期 61 天" ❌
ATA → 显示"实际超期 1 天" ❌
实际提柜 → 显示"实际超期 7 天" ❌
```

---

## ✅ 业务规则

### 节点分类

**① 实际业务节点**（已真实发生）：
- 出运
- ATA（实际到港）
- 卸船
- 实际提柜
- 实际还箱

**显示规则**：
- ✅ **总是显示"历时"**（蓝色）
- ❌ 不显示"超期"或"倒计时"

**原因**：
- 实际业务节点是已经发生的业务事实
- 不存在"计划"与"实际"的对比
- 只显示从该节点到现在经过的天数（历时）

**② 计划与预警节点**（用于参照）：
- ETA（预计到港）
- 修正 ETA
- 最晚提柜（LFD）
- 最晚还箱（LRD）

**显示规则**：
- 如果后续实际节点已发生 → 显示"历时"（实际 - 计划 的对比）
  - 差值 > 0：实际超期（橙色）
  - 差值 < 0：实际提前（绿色）
  - 差值 = 0：按时完成（蓝色）
- 如果后续实际节点未发生 → 显示"倒计时/超期"
  - 日期在未来：倒计时（绿色）
  - 日期在过去：超期（红色）

---

## 🔧 修复方案

### 修改文件：`DurationDisplay.vue`

**修改位置**：`displayType` 计算属性

**修改前**：
```typescript
const displayType = computed((): 'countdown' | 'elapsed' | 'overdue' => {
  if (props.mode !== 'auto') {
    return props.mode as 'countdown' | 'elapsed' | 'overdue'
  }

  if (props.hasNextNode) {
    return 'elapsed'
  }

  const time = overdueTime.value
  if (!time) return 'elapsed'
  if (time < 0) return 'countdown'
  
  // 日期在过去：显示超期
  return 'overdue'
})
```

**修改后**：
```typescript
const displayType = computed((): 'countdown' | 'elapsed' | 'overdue' => {
  // 显式指定模式
  if (props.mode !== 'auto') {
    return props.mode as 'countdown' | 'elapsed' | 'overdue'
  }

  // ✅ 实际业务节点：总是显示历时（因为业务已真实发生）
  const actualEventLabels = ['出运', 'ATA', '卸船', '实际提柜', '实际还箱']
  if (actualEventLabels.includes(props.label || '')) {
    return 'elapsed'
  }

  // ✅ 计划与预警节点：根据后一节点和时间判断
  
  // 有后一节点：显示历时（实际 - 计划）
  if (props.hasNextNode) {
    return 'elapsed'
  }

  // 无后一节点：根据时间判断
  const time = overdueTime.value
  if (!time) return 'elapsed'
  if (time < 0) return 'countdown'
  
  // 日期在过去：显示超期
  return 'overdue'
})
```

**关键修改**：
- 添加了 `actualEventLabels` 数组
- 实际业务节点直接返回 `'elapsed'`，不进入后续的超期判断

## 📝 核心原则

1. **实际业务节点**：总是显示历时（蓝色），因为业务已真实发生
2. **计划预警节点**：根据后续实际节点是否发生来决定显示方式
3. **清晰的视觉区分**：
   - 蓝色（历时）= 业务事实
   - 绿色（倒计时）= 未来计划
   - 橙色（实际超期）= 计划 vs 实际对比（晚于计划）
   - 红色（超期）= 计划在过去但未完成
