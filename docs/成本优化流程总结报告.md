```
PASS src/tests/cost-optimizer-category.test.ts
  √ 应该为原计划超期的情况生成 forward 搜索日期
  √ 应该为免费期内的情况生成 backward 搜索日期  
  √ 应该过滤掉早于原计划的日期（免费期内）
  √ 应该允许早于原计划的日期（已超期）
```

**关键验证点**:
- ✅ Free Period (免费期内): backward search，从免费期截止日往前找
- ✅ Overdue (已超期): forward search，从今天开始往后找
- ✅ 日期约束正确应用:
  - 不能是过去 (date >= today)
  - 不能早于原计划 (date >= basePickupDate，仅免费期内)
  - 不能是当天 (date != today，除非原计划就是当天)

**示例输出**:
```
场景 1: 原计划超期 (lastFreeDate=2026-03-27, basePickupDate=2026-03-28)
→ forward search: [2026-03-27, 2026-03-28, 2026-03-29, ..., 2026-04-03] (8 天)

场景 2: 免费期内 (lastFreeDate=2026-03-30, basePickupDate=2026-03-28)
→ backward search: [2026-03-30, 2026-03-29, 2026-03-28] (3 天)
```

### ⚠️ 发现的问题

#### 问题 1: 分类逻辑存在缺陷（高优先级）

**当前行为**:
```typescript
// Line 594-608
const daysUntilFreezeExpires = Math.ceil(
  (lastFreeOnly.getTime() - today.getTime()) / (1000 * 60 * 60 * 24)
);

const category = {
  category: daysUntilFreezeExpires >= 0 ? 'within_free_period' : 'overdue',
  remainingDays: daysUntilFreezeExpires,
  ...
};
```

**问题场景**:
```
输入:
  today = 2026-03-27
  lastFreeDate = 2026-03-27
  basePickupDate = 2026-03-28

计算:
  daysUntilFreezeExpires = 0
  category = 'within_free_period' ❌

期望:
  category = 'overdue' ✅ (因为原计划明天，但免费期今天结束，已经超期了)
```

**根本原因**:
分类逻辑只考虑了 `lastFreeDate` 和 `today` 的关系，没有考虑 `lastFreeDate` 和 `basePickupDate` 的关系。

**影响**:
- 当 `lastFreeDate < basePickupDate` 时，错误地判断为"免费期内"
- 导致使用 backward 策略而不是 forward 策略
- 生成的搜索范围为空（所有日期被过滤）

**修复方案**:
```typescript
// 新增：判断原计划是否已经超期
const isOriginalPlanOverdue = lastFreeOnly < basePickupOnly;

if (isOriginalPlanOverdue) {
  // 原计划超期，即使今天还在免费期内，也是 overdue
  category = 'overdue';
} else {
  // 原计划未超期，根据剩余天数判断
  category = daysUntilFreezeExpires >= 0 ? 'within_free_period' : 'overdue';
}
```

**状态**: ⏳ 待修复

**日期推导**:
```
提柜日 = 送仓日 = 卸柜日 = 还箱日
```

**费用构成**:
```

detentionCost     = f(提柜日，卸柜日，免费期，费率)
storageCost       = f(提柜日，港口费率)
transportationCost = f(距离，运输费率)
yardStorageCost   = $0 (不是 Drop off)
handlingCost      = $0 (不是 Expedited)
totalCost         = Σ 以上费用
```

### Drop off 模式

**日期推导**:
```
提柜日 < 送仓日 = 卸柜日
还箱日 = 卸柜日 + 3
```

**费用构成**:
```
demurrageCost     = f(提柜日，免费期，费率)
detentionCost     = f(提柜日，卸柜日，免费期，费率)
storageCost       = f(提柜日，港口费率)
transportationCost = f(距离，运输费率)
yardStorageCost   = f(堆存天数，堆场费率) + 操作费
                    堆存天数 = 送仓日 - 提柜日
handlingCost      = $0
totalCost         = Σ 以上费用
```
