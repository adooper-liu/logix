# 通用字典映射：当前存了哪些、为何前端没内容

## 一、现在项目里「存」了什么映射？

- **数据库表**：`dict_universal_mapping`（需先执行迁移才会存在）。
- **当前状态**：若从未执行过迁移，**表不存在**，因此库里**没有任何映射数据**。
- 映射数据只存在于这张表里；没有别的配置文件或代码里的“默认映射列表”。

## 二、为何前端没有内容？

1. **表不存在**  
   后端接口（如 `GET /api/v1/dict-mapping/universal/type/PORT`、`GET .../stats/summary`）会查 `dict_universal_mapping`。若表未建，会报错：**relation "dict_universal_mapping" does not exist**，接口返回 500，前端拿不到数据，列表和统计都为空。

2. **表存在但无数据**  
   执行迁移后表是空的，没有插入任何种子数据，所以前端列表依然为空，需要你在「通用字典映射管理」里新增，或通过脚本/Excel 导入。

## 三、如何让前端有内容？

### 步骤 1：执行迁移（建表 + 函数）

**当前项目 Docker 配置**（以仓库内 compose 文件为准）：

| 环境 | Compose 文件 | 服务名 | 容器名 | 用户 | 数据库 |
|------|--------------|--------|--------|------|--------|
| 开发 | `docker-compose.timescaledb.yml` | `postgres` | `logix-timescaledb-dev` | `postgres` | `logix_db` |
| 生产 | `docker-compose.timescaledb.prod.yml` | `postgres` | `logix-timescaledb-prod` | 来自 `.env` 的 `DB_USERNAME` | `DB_DATABASE` |

一律用 **docker compose -f &lt;文件&gt; exec** 执行迁移（不依赖容器名，由 compose 找对应容器）。

**1. 先启动 Postgres**（在项目根目录 `D:\Gihub\logix`）：

- 开发：`docker compose -f docker-compose.timescaledb.yml up -d postgres`
- 生产：`docker compose -f docker-compose.timescaledb.prod.yml --env-file .env up -d postgres`

**2. 执行迁移**

在**项目根目录**执行（PowerShell 下用管道，不要用 `<`）：

- **开发环境**：
  ```powershell
  Get-Content backend/migrations/create_universal_dict_mapping.sql -Raw | docker compose -f docker-compose.timescaledb.yml exec -T postgres psql -U postgres -d logix_db
  ```
- **生产环境**（需已配置 `.env` 中的 `DB_USERNAME`、`DB_DATABASE`）：
  ```powershell
  Get-Content backend/migrations/create_universal_dict_mapping.sql -Raw | docker compose -f docker-compose.timescaledb.prod.yml --env-file .env exec -T postgres psql -U $env:DB_USERNAME -d $env:DB_DATABASE
  ```

Bash / Git Bash / WSL（可用 `<` 时）：

```bash
# 开发
docker compose -f docker-compose.timescaledb.yml exec -T postgres psql -U postgres -d logix_db < backend/migrations/create_universal_dict_mapping.sql
```

**PowerShell 一键执行**：在项目根目录执行 `.\backend\scripts\init-universal-dict-mapping.ps1`，会使用**开发** compose 执行迁移并做简单校验（需已 `docker compose -f docker-compose.timescaledb.yml up -d postgres`）。

执行成功后会出现：

- 表：`dict_universal_mapping`
- 函数：`get_standard_code`, `get_all_mappings_by_type`, `search_mappings_fuzzy`, `get_standard_codes_batch`

此时再打开前端「通用字典映射管理」，接口不再因表不存在而 500，但列表仍可能为空（因为还没数据）。

### 步骤 2：添加映射数据

- **方式 A**：在前端页面「通用字典映射管理」里，按字典类型（港口、国家、船公司等）逐条「新增映射」。
- **方式 B**：用 SQL 或脚本向 `dict_universal_mapping` 插入数据（注意字段与表结构一致，如 `dict_type`, `target_table`, `target_field`, `standard_code`, `name_cn` 等）。

插入时需与现有字典表（如 `dict_ports`, `dict_countries`）里的标准代码一致，这样「名称 → 标准代码」才会在导入、查询时生效。

## 四、小结

| 问题 | 说明 |
|------|------|
| 现在项目都存了哪些映射？ | 只存在表 `dict_universal_mapping` 里；若未建表则当前没有任何映射。 |
| 为何前端没有内容？ | 1）表未建导致接口报错；2）或表已建但无数据。 |
| 怎么有内容？ | 1）先执行 `create_universal_dict_mapping.sql`；2）再在页面或库里添加映射数据。 |

迁移文件路径：`backend/migrations/create_universal_dict_mapping.sql`。
