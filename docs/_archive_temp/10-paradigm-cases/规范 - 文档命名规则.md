## 📋 命名格式标准

### 基本格式

```
分类前缀 - 描述名称【可选中文说明】.md

示例:
✅ 索引 - 案例库总索引 [CASE_STUDY_INDEX].md
✅ 指南 - 案例库使用指南.md
✅ 专题 - 成本优化完整案例集.md
```

### 组成部分

1. **分类前缀**（必需）
   - 标识文档类型和用途
   - 使用 2-4 个汉字
   - 后跟短横线 `-`

2. **描述名称**（必需）
   - 简洁明了的描述
   - 可以使用英文或中文
   - 推荐 10-30 个字符

3. **中文说明**（可选）
   - 方括号 `[]` 包裹
   - 便于快速理解
   - 用于英文缩写的情况

### ❌ 错误示例

```markdown
## 缺少分类前缀
❌ 案例库总索引.md          → ✅ 索引 - 案例库总索引.md
❌ 成本优化案例集.md         → ✅ 专题 - 成本优化案例集.md
❌ 使用指南.md              → ✅ 指南 - 案例库使用指南.md

## 分类不明确
❌ 成本优化文档.md          → ✅ 专题 - 成本优化完整案例集.md
❌ 修复 Bug 的记录.md        → ✅ 失败案例 - 成本优化日期错误修复.md
❌ 实施方案.md             → ✅ 方案 - 智能排产成本优化设计.md

❌ 关于我们在做成本优化的时候遇到的各种问题和解决方案的详细说明.md
→ ✅ 案例 - 成本优化问题解决方案集.md

❌ 重要文档.md             → ✅ 核心 - 开发范式案例库.md
❌ 参考资料.md             → ✅ 索引 - 外部参考资料.md
❌ 笔记.md                → ✅ 笔记 - 架构设计学习心得.md
```

## 🔧 文件重命名流程

### Step 1: 识别文档类型

```markdown
问题：这个文档是什么类型？
- 是索引/目录？ → 使用"索引"
- 是使用说明？ → 使用"指南"
- 是具体案例？ → 使用"案例"
- 是技术方案？ → 使用"方案"
- 是检查清单？ → 使用"清单"
- 是完成报告？ → 使用"报告"
```

### Step 2: 选择分类前缀

```markdown
根据文档内容选择最匹配的前缀：
- 成功案例 → "案例"或"成功案例"
- 失败教训 → "失败案例"
- 最佳实践 → "实践"
- 常见错误 → "反模式"
```

### Step 3: 编写描述名称

```markdown
要求:
✅ 简洁明了（10-30 字）
✅ 准确描述内容
✅ 避免模糊词汇
✅ 可以使用英文缩写

❌ 太长：关于成本优化项目中如何处理日期边界条件的技术方案设计
✅ 合适：方案 - 成本优化日期边界处理
```

### Step 4: 添加可选说明

```markdown
适用于以下情况:
- 有英文缩写：索引 - 案例库总索引 [CASE_STUDY_INDEX]
- 需要强调：核心 - 开发范式案例库 [LDF-CASES]
- 版本号：报告 - 文档整理完成报告 [v2.0]
```

### Step 5: 执行重命名

```bash
# PowerShell 示例
Rename-Item "旧文件名.md" "新文件名.md"

# 批量重命名示例
Get-ChildItem "*.md" | Where-Object { $_.Name -match "成本优化" } | 
  Rename-Item -NewName { "案例-" + $_.Name }
```

## 📊 命名决策树

```
开始
  ↓
这是什么类型的文档？
  ├─ 索引/目录 → 使用"索引"
  ├─ 使用说明 → 使用"指南"
  ├─ 技术规范 → 使用"规范"
  ├─ 具体案例 → 
  │   ├─ 成功经验 → "案例"或"成功案例"
  │   ├─ 失败教训 → "失败案例"
  │   └─ 最佳做法 → "实践"
  ├─ 技术方案 → 使用"方案"
  ├─ 检查清单 → 使用"清单"
  ├─ 测试验证 → 使用"测试"或"验证"
  ├─ 完成报告 → 使用"报告"
  └─ 其他 → 选择最接近的分类
    ↓
名称是否清晰明了？
  ├─ 是 → 完成命名
  └─ 否 → 简化描述名称
    ↓
是否需要附加说明？
  ├─ 是 → 添加 [中文说明]
  └─ 否 → 直接完成
```

## 🎯 特殊场景处理

### 场景 1: 系列文档

```markdown
同一主题的多个文档:
- 案例 - 成本优化逻辑分析.md
```


## 📋 检查清单

### 命名质量检查

在最终确定文件名之前，请检查：

- [ ] 是否包含分类前缀？
- [ ] 前缀选择是否准确？
- [ ] 描述名称是否简洁（10-30 字）？
- [ ] 名称是否能准确反映内容？
- [ ] 是否避免了模糊词汇？
- [ ] 如有英文缩写，是否添加了中文说明？
- [ ] 文件名长度是否合理（<50 字符）？
- [ ] 是否与现有文档命名风格一致？

### 一致性检查

- [ ] 同类文档是否使用了相同前缀？
- [ ] 系列文档命名是否有连贯性？
- [ ] 是否与索引中的引用一致？
- [ ] 链接到其他文档的路径是否正确？

