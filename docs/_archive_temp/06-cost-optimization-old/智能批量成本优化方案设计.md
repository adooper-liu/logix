## 一、需求分析

### 1.1 业务目标

**智能批量成本优化**的核心目标：

1. ✅ **分类处理**：按免费期状态分类（免费期内 vs 已超期）
2. ✅ **优先级排序**：超期的优先处理
3. ✅ **资源约束**：考虑仓库能力限制
4. ✅ **成本最优**：在免费期内尽量延后日期（成本=0，日期最大化）
5. ✅ **效率优化**：分类子集减少数据处理量

### 1.2 核心策略

```
批量货柜
  ↓
分类 → 【免费期内】vs 【已超期】
  ↓                    ↓
剩余天数倒序         超期天数倒序
  ↓                    ↓
优先处理超期        能力允许时处理
  ↓
优化策略：
- 超期的：尽量往后排（成本为 0，日期最大化）
- 免费期内的：从免费日往前优化
```

### 4.2 核心算法

#### 4.2.1 分类算法

```typescript
async categorizeContainers(
  containerNumbers: string[]
): Promise<{
  withinFreePeriod: ContainerCategory[]; // 免费期内
  overdue: ContainerCategory[]; // 已超期
}> {
  const categories = {
    withinFreePeriod: [],
    overdue: []
  };

  for (const containerNumber of containerNumbers) {
    // 1. 获取货柜信息
    const container = await this.containerRepo.findOne({
      where: { containerNumber },
      relations: ['portOperations']
    });

    // 2. 获取免费期
    const lastFreeDate = await this.getLastFreeDate(containerNumber);
    const plannedPickupDate = container.plannedData?.plannedPickupDate;

    // 3. 计算剩余天数
    const remainingDays = dateTimeUtils.daysBetween(
      new Date(), // 当前日期
      lastFreeDate
    );

    // 4. 分类
    const category = remainingDays >= 0 ? 'within_free_period' : 'overdue';

    const categoryItem: ContainerCategory = {
      containerNumber,
      category,
      plannedPickupDate,
      lastFreeDate,
      remainingDays,
      originalCost: 0, // TODO: 计算
      warehouseCode: container.warehouseCode,
      truckingCompanyId: container.truckingCompanyId
    };

    categories[category].push(categoryItem);
  }

  // 5. 排序（倒序）
  // 免费期内：剩余天数少的优先（紧急）
  categories.withinFreePeriod.sort((a, b) => a.remainingDays - b.remainingDays);

  // 已超期：超期天数多的优先（最紧急）
  categories.overdue.sort((a, b) => a.remainingDays - b.remainingDays); // 负数，小的更紧急

  return categories;
}
```

#### 4.2.2 优先级排序算法

```typescript
assignPriorities(
  withinFreePeriod: ContainerCategory[],
  overdue: ContainerCategory[]
): PriorityContainer[] {
  const priorityQueue: PriorityContainer[] = [];

  // 1. 已超期的优先级最高（优先级 1-N）
  overdue.forEach((container, index) => {
    priorityQueue.push({
      ...container,
      priority: index + 1 // 超期的优先级最高
    });
  });

  // 2. 免费期内的优先级较低（优先级 N+1~M）
  withinFreePeriod.forEach((container, index) => {
    priorityQueue.push({
      ...container,
      priority: overdue.length + index + 1
    });
  });

  return priorityQueue;
}
```

#### 4.2.3 智能优化算法

```typescript
async optimizeContainerWithCapacity(
  container: PriorityContainer,
  warehouseCapacities: Map<string, Map<string, WarehouseCapacity>>,
  truckingCompany: TruckingCompany
): Promise<PriorityContainer> {
  const warehouseCode = container.warehouseCode;
  const basePickupDate = container.plannedPickupDate;
  const lastFreeDate = container.lastFreeDate;

  // 1. 如果是已超期的，尽量往前排（日期越早越好，减少继续产生费用）
  if (container.category === 'overdue') {
    // 从当前日期开始，往前找第一个有能力的日期（尽早处理）
    let optimalDate = new Date(); // 从今天开始

    while (optimalDate <= dateTimeUtils.addDays(basePickupDate, 7)) {
      const capacity = warehouseCapacities
        .get(warehouseCode)
        ?.get(optimalDate.toISOString());

      if (capacity && capacity.availableCapacity > 0) {
        // 有能力，检查成本
        const cost = await this.evaluateCost(
          container.containerNumber,
          optimalDate,
          truckingCompany
        );

        // 超期的只能尽量减少损失，找到最早可用的日期
        container.optimizedPickupDate = optimalDate;
        container.optimizedCost = cost;
        container.savings = container.originalCost - cost;
        break;
      }

      // 往后推一天（找下一个可用日期）
      optimalDate = dateTimeUtils.addDays(optimalDate, 1);
    }

    // 如果找不到，使用原计划日期
    if (!container.optimizedPickupDate) {
      container.optimizedPickupDate = basePickupDate;
      container.optimizedCost = container.originalCost;
      container.savings = 0;
    }
  }

  // 2. 如果是免费期内的，从免费日往前优化（日期越大越好）
  if (container.category === 'within_free_period') {
    // 从免费期截止日开始，往前找第一个有能力的日期（尽量靠近免费期）
    let optimalDate = lastFreeDate;

    while (optimalDate >= dateTimeUtils.addDays(basePickupDate, -7)) {
      const capacity = warehouseCapacities
        .get(warehouseCode)
        ?.get(optimalDate.toISOString());

      if (capacity && capacity.availableCapacity > 0) {
        // 有能力，检查成本
        const cost = await this.evaluateCost(
          container.containerNumber,
          optimalDate,
          truckingCompany
        );

        if (cost === 0) {
          // 成本为 0，找到最优解（尽量靠近免费期，日期最大化）
          container.optimizedPickupDate = optimalDate;
          container.optimizedCost = cost;
          container.savings = container.originalCost - cost;
          break;
        }
      }

      // 往前推一天
      optimalDate = dateTimeUtils.addDays(optimalDate, -1);
    }

    // 如果找不到，使用原计划日期
    if (!container.optimizedPickupDate) {
      container.optimizedPickupDate = basePickupDate;
      container.optimizedCost = container.originalCost;
      container.savings = 0;
    }
  }

  return container;
}
```

#### 4.2.4 仓库能力预计算

```typescript
async precomputeWarehouseCapacities(
  warehouseCodes: string[],
  dateRange: { start: Date; end: Date }
): Promise<Map<string, Map<string, WarehouseCapacity>>> {
  const capacities = new Map<string, Map<string, WarehouseCapacity>>();

  for (const warehouseCode of warehouseCodes) {
    const warehouseCapacities = new Map<string, WarehouseCapacity>();

    // 查询仓库能力
    const occupancies = await this.warehouseOccupancyRepo.find({
      where: {
        warehouseCode,
        date: Between(dateRange.start, dateRange.end)
      }
    });

    for (const occupancy of occupancies) {
      const dateKey = occupancy.date.toISOString();
      warehouseCapacities.set(dateKey, {
        warehouseCode,
        date: occupancy.date,
        totalCapacity: occupancy.totalCapacity || 100, // 默认 100
        usedCapacity: occupancy.usedCapacity || 0,
        availableCapacity: (occupancy.totalCapacity || 100) - (occupancy.usedCapacity || 0)
      });
    }

    capacities.set(warehouseCode, warehouseCapacities);
  }

  return capacities;
}
```

### 4.3 主流程

```typescript
async batchOptimize(
  containerNumbers: string[],
  basePickupDate: Date,
  lastFreeDate?: Date
): Promise<{
  results: PriorityContainer[];
  summary: {
    totalContainers: number;
    withinFreePeriodCount: number;
    overdueCount: number;
    totalSavings: number;
    optimizedCount: number;
  };
}> {
  log.info(`[BatchOptimizer] Starting batch optimization for ${containerNumbers.length} containers`);

  // 1. 分类货柜
  const categories = await this.categorizeContainers(containerNumbers);

  log.info(`[BatchOptimizer] Categorized:`);
  log.info(`  - Within free period: ${categories.withinFreePeriod.length}`);
  log.info(`  - Overdue: ${categories.overdue.length}`);

  // 2. 分配优先级
  const priorityQueue = this.assignPriorities(
    categories.withinFreePeriod,
    categories.overdue
  );

  // 3. 预计算仓库能力
  const warehouseCodes = [...new Set(containerNumbers.map(cn => {
    // 从分类中获取 warehouseCode
    const cat = [...categories.withinFreePeriod, ...categories.overdue]
      .find(c => c.containerNumber === cn);
    return cat?.warehouseCode;
  }).filter(Boolean) as string[])];

  const dateRange = {
    start: dateTimeUtils.addDays(basePickupDate, -7),
    end: dateTimeUtils.addDays(basePickupDate, 30)
  };

  const warehouseCapacities = await this.precomputeWarehouseCapacities(
    warehouseCodes,
    dateRange
  );

  // 4. 按优先级优化每个货柜
  const results: PriorityContainer[] = [];

  for (const container of priorityQueue) {
    log.info(
      `[BatchOptimizer] Processing ${container.containerNumber} ` +
      `(Priority: ${container.priority}, Category: ${container.category})`
    );

    // 获取车队信息
    const truckingCompany = await this.truckingCompanyRepo.findOne({
      where: { companyCode: container.truckingCompanyId }
    });

    if (!truckingCompany) {
      log.warn(`[BatchOptimizer] Trucking company not found for ${container.containerNumber}`);
      continue;
    }

    // 优化
    const optimizedContainer = await this.optimizeContainerWithCapacity(
      container,
      warehouseCapacities,
      truckingCompany
    );

    results.push(optimizedContainer);

    // 更新仓库能力（占用一个位置）
    if (optimizedContainer.optimizedPickupDate) {
      const capacity = warehouseCapacities
        .get(container.warehouseCode)
        ?.get(optimizedContainer.optimizedPickupDate.toISOString());

      if (capacity) {
        capacity.usedCapacity++;
        capacity.availableCapacity--;
      }
    }
  }

  // 5. 统计
  const summary = {
    totalContainers: containerNumbers.length,
    withinFreePeriodCount: categories.withinFreePeriod.length,
    overdueCount: categories.overdue.length,
    totalSavings: results.reduce((sum, r) => sum + (r.savings || 0), 0),
    optimizedCount: results.filter(r => r.optimizedPickupDate !== r.plannedPickupDate).length
  };

  log.info(`[BatchOptimizer] Completed:`, summary);

  return {
    results,
    summary
  };
}
```

## 五、优化策略详解

### 5.1 分类策略

#### 为什么分类？

1. **减少计算量**：只计算必要的货柜
2. **优先级清晰**：超期的优先处理
3. **资源优化**：避免浪费仓库能力

#### 分类标准

| 类别     | 剩余天数 | 优先级 | 策略                 |
| -------- | -------- | ------ | -------------------- |
| 免费期内 | >= 0     | 低     | 从免费日往前优化     |
| 已超期   | < 0      | 高     | 尽量往后排（成本=0） |

---

### 5.2 优先级策略

#### 为什么倒序排列？

**免费期内**：

- 剩余天数少的更紧急（优先级高）
- 例：剩余 1 天 > 剩余 5 天

**已超期**：

- 超期天数多的更紧急（优先级高）
- 例：超期 10 天 > 超期 3 天

#### 优先级计算

```typescript
priority = category === "overdue" ? index + 1 : overdue.length + index + 1;
```

**效果**：

- 所有超期的优先级 < 免费期内的
- 超期越久，优先级越高（数字越小）
- 免费期越短，优先级越高

---

### 5.3 仓库能力策略

#### 为什么考虑仓库能力？

1. **资源有限**：仓库每天处理能力有限
2. **避免冲突**：多个货柜不能同一天
3. **实际可行**：优化结果必须可执行

#### 能力检查逻辑

```typescript
if (capacity.availableCapacity > 0) {
  // 有能力，可以安排
} else {
  // 无能力，找其他日期
}
```

#### 能力更新

```typescript
// 优化成功后，占用一个位置
capacity.usedCapacity++;
capacity.availableCapacity--;
```

---

### 5.4 成本优化策略

#### 超期货柜（🔴 紧急）

**目标**：尽早处理，减少继续产生费用

**策略**：

1. 从**当前日期**开始（今天）
2. 往后找第一个有能力的日期
3. 检查成本（虽然已经超期，但尽早处理可以减少损失）
4. 使用最早可用的日期

**原因**：

- 已经超期，每天都在产生费用
- **越早处理，损失越小**
- 日期越早越好（尽快解决）

**示例**：

```
超期 10 天的货柜：
- 原计划：3-28（超期 10 天，费用 $1000）
- 优化后：3-27（超期 9 天，费用 $900）✅ 更早，费用更少
- 而不是：3-30（超期 12 天，费用 $1200）❌ 更晚，费用更多
```

---

#### 免费期内的货柜（🟢 正常）

**目标**：成本为 0，日期最大化

**策略**：

1. 从免费期截止日开始
2. 往前找第一个有能力的日期
3. 检查成本是否为 0
4. 如果为 0，使用该日期（靠近免费期，日期最大化）
5. 如果不为 0，继续往前找

**原因**：

- 在免费期内，成本都是 0
- **靠近免费期有利于灵活调整**
- 给其他货柜留出空间
- 日期越大（越晚），灵活性越高

**示例**：

```
免费期截止日 3-30 的货柜：
- 原计划：3-25（成本 $0，但太早）
- 优化后：3-30（成本 $0，最晚，灵活性最高）✅
- 而不是：3-20（成本 $0，但太早，不灵活）❌
```

  const summary = {
    totalContainers: containerNumbers.length,
    withinFreePeriodCount: categories.withinFreePeriod.length,
    overdueCount: categories.overdue.length,
    totalSavings: results.reduce((sum, r) => sum + (r.savings || 0), 0),
    optimizedCount: results.filter(r => r.optimizedPickupDate !== r.plannedPickupDate).length
  };

  log.info(`[BatchOptimizer] Completed:`, summary);

  return {
    results,
    summary
  };
}
