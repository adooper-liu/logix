### Step 2: 验证日期约束逻辑

**测试场景 A: 原计划超期**

```
输入:
  today = 2026-03-27
  lastFreeDate = 2026-03-27
  basePickupDate = 2026-03-28

期望行为:
  category = 'overdue' (因为 lastFreeDate < basePickupDate)
  strategy = 'forward'
  searchDates = [2026-03-27, 2026-03-28, ..., 2026-04-03]

当前行为:
  ❌ category = 'within_free_period' (BUG!)
  ❌ strategy = 'backward'
  ❌ searchDates = [] (所有日期被过滤)
```

**测试方法**:

1. 打开 `d:\Gihub\logix\backend\src\services\schedulingCostOptimizer.service.ts`
2. 找到 Line 594-608 (categorizeSingleContainer 方法)
3. 查看分类逻辑
4. 找到 Line 857 (selectOptimizationStrategy 调用)
5. 查看策略选择

**验证命令**:

```bash
# 添加调试日志后运行
npm run start
# 然后调用成本优化 API，观察日志输出
```

### Step 3: 验证费用计算逻辑

**测试场景 B: Direct 模式，免费期内**

```
输入:
  pickupDate = 2026-03-27
  strategy = 'Direct'
  lastFreeDate = 2026-03-27
  distance = 50 miles
  transportRate = $14/mile

期望输出:
  demurrageCost = $0
  detentionCost = $0
  storageCost = $0
  transportationCost = 50 * 14 = $700
  yardStorageCost = $0
  handlingCost = $0
  totalCost = $700
```

**测试场景 C: Drop off 模式，超期**

```
输入:
  pickupDate = 2026-03-28
  strategy = 'Drop off'
  lastFreeDate = 2026-03-27
  distance = 50 miles
  transportRate = $14/mile
  yardRate = $100/day
  yardOpFee = $100

期望输出:
  pickupDate = 2026-03-28
  unloadDate = 2026-03-30 (提 +2)
  returnDate = 2026-04-02 (卸 +3)

  demurrageCost = $0 (假设未超期)
  detentionCost = $0 (假设未超期)
  storageCost = $2,200 (假设$100/天 × 22 天)
  transportationCost = $700
  yardStorageCost = (2 天 × $100) + $100 = $300
  handlingCost = $0
  totalCost = $3,200
```

**验证方法**:

1. 调用成本优化 API
2. 查看后端日志中的费用明细
3. 对比期望值和实际值

**API 调用示例**:

```bash
curl -X POST http://localhost:3000/api/scheduling/ECMU5399797/optimize \
  -H "Content-Type: application/json" \
  -d '{
    "warehouseCode": "WH001",
    "truckingCompanyId": "TC001",
    "basePickupDate": "2026-03-28"
  }'
```

**查看日志**:

```bash
# 查看实时日志
tail -f backend.log | grep CostOptimizer
```

## 🐛 已知问题列表

### 高优先级

1. **分类逻辑错误**
   - **现象**: 当 `lastFreeDate < basePickupDate` 时判断为"免费期内"
   - **影响**: 使用错误的搜索策略，导致搜索范围为空
   - **修复方案**: 增加 `isOriginalPlanOverdue` 判断
   - **状态**: ⏳ 待修复

2. **四日期缺失**
   - **现象**: Alternative 接口缺少 deliveryDate、unloadDate、returnDate
   - **影响**: 用户无法看到完整的日期链
   - **修复方案**: 扩展接口并在后端推导日期
   - **状态**: ⏳ 待实现

### 中优先级

3. **审计功能缺失**
   - **现象**: 没有保存优化历史
   - **影响**: 无法追溯优化决策
   - **修复方案**: 创建审计表并保存记录
   - **状态**: ⏳ 待实现
