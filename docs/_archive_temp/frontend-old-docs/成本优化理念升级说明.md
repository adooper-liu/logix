## 🎯 一、理念对比

### 传统排产理念（当前实现）

```
业务目标：完成排产
决策依据：仓库产能可用性
排产流程:
  1. 清关日 = ETA || ATA
  2. 提柜日 = 清关日 + 1 (受 last_free_date 约束)
  3. 卸柜日 = 查找最早可用仓库档期
  4. 送柜日/还箱日 = 根据模式推导

问题:
❌ 没有考虑滞港费成本
❌ 可能导致不必要的费用
❌ 缺少成本透明度
```

### 成本优化理念（新方案）

```
业务目标：总成本最小化
决策依据：成本效益分析
排产流程:
  1. 卸柜日是瓶颈 ← 先确定候选范围
  2. 评估每个方案的总成本
     ├─ 滞港费
     ├─ 堆存费
     └─ 运输费
  3. 选择成本最低方案
  4. 反推其他日期

优势:
✅ 充分利用免费期
✅ 避免不必要费用
✅ 成本透明可解释
```

## 💡 二、核心洞察

### 关键发现

**用户的深刻见解**:

> "卸柜日是关键：先根据仓库产能找最早可用卸柜日，再评估用港口的免费天数，是否会产生滞港费"

这个洞察揭示了：

1. **卸柜日是真正的瓶颈**
   - 不是提柜日，因为提柜日可以灵活调整
   - 不是清关日，因为清关日相对固定
   - 而是卸柜日，因为受仓库产能硬约束

2. **成本应该驱动决策**
   - 不只是"能找到什么位置"
   - 而是"哪个位置成本最低"
   - 需要综合比较各种成本

3. **免费期是宝贵资源**
   - 应该在免费期内完成提柜
   - 如果无法避免滞港，要权衡替代方案
   - Drop off + 外部堆场可能更经济

## 🔧 四、算法升级

### 4.1 从"正向推导"到"成本优化"

#### 旧算法（正向推导）

```typescript
// 1. 计算提柜日
pickupDate = clearanceDate + 1

// 2. 查找卸柜日
unloadDate = findEarliestAvailableWarehouse(pickupDate)

// 3. 推导其他日期
deliveryDate = (mode === 'Live load') ? pickupDate : unloadDate
returnDate = (mode === 'Live load') ? unloadDate : unloadDate + 1

// ❌ 问题：没有考虑成本
```

#### 新算法（成本优化）

```typescript
// 1. 生成卸柜日候选
const candidates = await getUnloadDateCandidates(warehouse, startDate, lastFreeDate);

// 2. 评估每个候选的总成本
const costOptions = await Promise.all(
  candidates.map(async (candidate) => {
    const demurrage = calculateDemurrage(candidate.date, lastFreeDate);
    const storage = calculateStorage(candidate.date, truckingCompany);
    const transport = calculateTransport(truckingCompany);
    
    return {
      date: candidate.date,
      totalCost: demurrage + storage + transport,
      details: { demurrage, storage, transport }
    };
  })
);

// 3. 选择成本最低方案
const optimal = costOptions.sort((a, b) => a.totalCost - b.totalCost)[0];

// 4. 反推其他日期
pickupDate = calculatePickupDate(optimal.date, lastFreeDate);
deliveryDate = (mode === 'Live load') ? pickupDate : optimal.date;
returnDate = (mode === 'Live load') ? optimal.date : optimal.date + 1;

// ✅ 优势：成本最小化
```

## 📞 十、快速参考

### 核心公式

```typescript
// 总成本 = 滞港费 + 堆存费 + 运输费
totalCost = demurrageCost + storageCost + transportCost

// 滞港费 = 阶梯式累进
demurrageCost = sum(tier[i].rate * days_in_tier[i])

// 堆存费 = 天数 × 日费率
storageCost = storageDays × dailyRate

// 成本节省 = 原方案 - 优化方案
savings = baselineCost - optimizedCost
```

### 决策树

```
是否需要产生滞港费？
├─ 否 → 安排最早可用卸柜日
└─ 是 → 启动优化流程
    ├─ 有堆场车队可用？
    │   ├─ 是 → Drop off + 内部堆场
    │   └─ 否 → 继续判断
    ├─ 外部堆场可用？
    │   ├─ 是 → Drop off + 外部堆场
    │   └─ 否 → 接受滞港
    └─ 成本对比
        - 滞港费 > 堆存费 × 1.3 → 选择堆场
        - 滞港费 <= 堆存费 × 1.3 → 接受滞港
```
