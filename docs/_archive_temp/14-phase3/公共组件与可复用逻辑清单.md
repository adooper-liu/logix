## 2. 基础通用组件

### 2.1 日期时间相关

#### DateTimePicker
路径：components/common/DateTimePicker.vue  
用途：统一的日期时间选择器  
功能：
- 支持日期 + 时间选择
- 快捷选项（今天、明天、本周等）
- 禁用过去日期配置
- 统一的格式化输出

使用示例:
``vue
<template>
  <DateTimePicker 
    v-model="selectedDate"
    :disabled-date="(date) => date < new Date()"
    format="YYYY-MM-DD HH:mm"
    placeholder="请选择日期时间"
  />
</template>
```

注意事项：
- 所有需要日期时间选择的场景都应使用此组件
- 不要直接使用 el-date-picker 裸组件

---

#### DateTimeDisplay
路径：components/common/DateTimeDisplay.vue  
用途：统一的日期时间展示格式  
功能：
- 自动格式化日期时间
- 支持相对时间显示（"3 小时前"）
- 多语言支持
- Tooltip 显示完整时间

使用示例:
``vue
<template>
  <DateTimeDisplay 
    :value="container.etaDestPort"
    show-relative
    tooltip-format="YYYY-MM-DD HH:mm:ss"
  />
</template>
```

注意事项：
- 所有货柜时间字段展示都应使用此组件
- 确保全项目时间格式一致

#### DateRangePicker

```typescript

## 8. 使用规范与最佳实践

### 8.1 组件复用原则

应该做的:

1. 优先使用现有组件
```typescript
// 正确：使用现有组件
<DateTimePicker v-model="date" />

// 错误：重复造轮子
<el-date-picker 
  v-model="date"
  :disabled-date="..."
  :shortcuts="..."
/>
```

2. 提取公共逻辑到 Composable
```typescript
// 正确：提取为 composable
export function useContainerLogic() {
  const { data, loading } = useFetch(...)
  return { data, loading }
}

// 错误：在每个组件中重复
onMounted(async () => {
  loading.value = true
  data.value = await fetchApi()
  loading.value = false
})
```

3. 使用统一的类型定义
```typescript
// 正确：使用共享类型
import { Container } from '@/types/container'

// 错误：自己定义
interface MyContainer {
  containerNumber: string
  // ...
}
```

4. 遵循命名规范
```typescript
// 正确：Composable 命名
useShipmentsTable()
useContainerDetail()

// 正确：组件命名
ContainerGanttChart.vue
DateTimePicker.vue

// 错误：命名不规范
shipment_table()
gantt.vue
```

禁止做的:

1. 不要重复造轮子
```typescript
// 禁止：自己实现倒计时
const countdown = ref('')
setInterval(() => {
  countdown.value = calculateCountdown(targetDate)
}, 1000)

// 正确：使用现有 composable
const { countdown } = useContainerCountdown(targetDate)
```

2. 不要硬编码颜色和文本
```typescript
// 禁止：硬编码
const color = '#F56C6C'
const status = '已到港'

// 正确：使用统一配置
const { danger } = useColors()
const status = getLogisticsStatusText('at_port')
```

3. 不要直接调用 API
```typescript
// 禁止：直接在组件中调用
const loadData = async () => {
  const res = await fetch('/api/containers')
  data.value = res.data
}

// 正确：使用 composable
const { containers, loading } = useShipmentsTable()
```

### 8.2 Composable 使用规范

#### 命名规则
``typescript
// ✅ 正确：use + PascalCase
useShipmentsTable
useContainerDetail
useGanttLogic

// ❌ 错误：不符合规范
use_shipments_table
usecontainerDetail
```

#### 返回值组织
``typescript
// ✅ 正确：清晰的返回值结构
export function useShipmentsTable() {
  // 状态
  const containers = ref<Container[]>([])
  const loading = ref(false)
  
  // 计算属性
  const filteredContainers = computed(() => ...)
  
  // 方法
  const handleSortChange = () => {...}
  const handlePageChange = () => {...}
  
  return {
    // 状态
    containers,
    loading,
    
    // 计算属性
    filteredContainers,
    
    // 方法
    handleSortChange,
    handlePageChange
  }
}
```

#### 副作用管理
``typescript
// ✅ 正确：清理副作用
export function useEventListener(
  target: EventTarget,
  event: string,
  handler: (e: Event) => void
) {
  onMounted(() => {
    target.addEventListener(event, handler)
  })
  
  onUnmounted(() => {
    target.removeEventListener(event, handler)
  })
}
```

### 8.3 组件开发流程

#### 1. 需求分析
- 是否有类似组件？→ **复用或重构**
- 是否通用？→ **放入 common 目录**
- 是否业务特定？→ **放入对应业务目录**

#### 2. 组件设计
``typescript
// 确定组件职责（单一职责原则）
// 确定 Props 接口
interface Props {
  modelValue: T
  disabled?: boolean
}

// 确定 Events
const emit = defineEmits<{
  'update:modelValue': [value: T]
  'change': [value: T]
}>()

// 确定 Slots
defineSlots<{
  default: () => any
  header: () => any
}>()
```

#### 3. 类型定义
``typescript
// 在 types/index.ts 或组件同目录 types.ts 定义
export interface ComponentType {
  // 类型定义
}
```

#### 4. 单元测试
``typescript
describe('ComponentName', () => {
  it('should render correctly', () => {...})
  it('should emit events correctly', () => {...})
})
```

#### 5. 文档编写
```
## ComponentName
**用途**: 一句话描述
**Props**: 属性列表
**Events**: 事件列表
**Slots**: 插槽列表
**示例**: 使用代码
```

### 8.4 性能优化建议

#### 1. 避免不必要的计算
``typescript
// ✅ 正确：使用 computed
const filteredList = computed(() => {
  return list.value.filter(item => item.active)
})

// ❌ 错误：在 template 中直接计算
// <div v-for="item in list.filter(i => i.active)" />
```

#### 2. 使用虚拟滚动
``typescript
// ✅ 正确：大数据量使用虚拟滚动
const { visibleItems } = useVirtualScroll(items, itemHeight)

// ❌ 错误：直接渲染大量 DOM
// <div v-for="item in items" /> <!-- items.length = 10000 -->
```

#### 3. 懒加载组件
``typescript
// ✅ 正确：按需加载
const HeavyComponent = defineAsyncComponent(() => 
  import('./HeavyComponent.vue')
)

// ❌ 错误：一次性加载所有组件
```

## 10. 维护与更新

### 10.1 新增组件流程

1. **评估必要性**
   - 是否真的需要新组件？
   - 能否扩展现有组件？

2. **确定分类**
   - 通用组件 → `components/common/`
   - 业务组件 → `components/{business}/`
   - Composable → `composables/`

3. **编写代码**
   - 遵循命名规范
   - 完整的类型定义
   - 单元测试

4. **编写文档**
   - 更新本文档
   - 添加使用示例

5. **Code Review**
   - 检查是否复用现有组件
   - 检查代码质量
   - 检查性能影响

### 10.2 组件废弃流程

1. **标记 deprecated**
   ```typescript
   /**
    * @deprecated 使用 NewComponent 替代
    */
   ```

2. **提供迁移指南**
   ````
   ## 迁移步骤
   1. 替换组件名
   2. 更新 Props
   3. 测试功能
   ```

3. **保留一个版本周期**
   - 下一个大版本移除
