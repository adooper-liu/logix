## 一、需求总结

### 1.1 业务目标

智能排柜系统的核心目标是根据货柜的 ETA/ATA、目的港、仓库可用性等信息，自动计算并推荐最优的物流计划日期，以最小化总成本（包括滞港费、滞箱费、堆存费、运输费等）。

### 1.2 核心功能需求

#### 1.2.1 五个计划日期计算

以卸柜日 (plannedUnloadDate) 为核心锚点，向前向后推导其他日期：

| 日期字段 | 方向 | 计算逻辑 | 约束条件 |
|---------|------|---------|---------|
| plannedCustomsDate | 向前 | ETA + buffer 天数 | ≥ today |
| plannedPickupDate | 向前 | 清关日 + 1 天 | ≤ lastFreeDate, ≥ today |
| plannedDeliveryDate | 向前 | 提柜日 + 运输天数 | - |
| plannedUnloadDate | 锚点 | 基于仓库可用性计算 | 受仓库产能限制 |
| plannedReturnDate | 向后 | 卸柜日 + 在仓天数 | ≤ lastReturnDate |

#### 1.2.2 三种卸柜方案

1. **Direct (直送模式)**
   - 适用场景：货物紧急，直接送到仓库卸柜
   - 特点：提柜日 = 送柜日 = 卸柜日
   
2. **Drop off (卸货模式)**
   - 适用场景：货物到港后暂存堆场，再安排送仓
   - 特点：提柜日 < 送柜日 = 卸柜日
   - 优势：可利用堆场缓冲，避免滞港费

3. **Expedited (加急模式)**
   - 适用场景：免费期即将到期，需要加急处理
   - 特点：在免费期内完成卸柜
   - 成本：需支付加急操作费

#### 1.2.3 成本优化引擎

**成本构成**:
```typescript
interface CostBreakdown {
  demurrageCost: number;      // 滞港费
  detentionCost: number;      // 滞箱费
  storageCost: number;        // 港口存储费
  yardStorageCost: number;    // 外部堆场堆存费 (Drop off 专属)
  transportationCost: number; // 运输费
  handlingCost: number;       // 操作费 (加急费等)
  totalCost: number;          // 总成本
}
```

**优化算法**:
- 生成所有可行方案（搜索窗口±7 天）
- 评估每个方案的总成本
- 选择成本最低的方案
- 支持多方案对比和用户选择

#### 1.2.4 日历能力配置

**仓库能力管理**:
- 每日卸柜容量限制
- 休息日配置（跳过周末）
- 特殊日期配置

**车队能力管理**:
- 每日还箱容量限制
- 堆场容量管理
- 港口 - 仓库映射关系

### 1.3 技术需求

1. **数据库优先原则**
   - 所有数据从数据库读取
   - 禁止使用前端输入作为权威源
   - 配置项存储在 dict_scheduling_config 表

2. **日期计算精度**
   - 所有日期计算使用统一的 dateTimeUtils 工具
   - 考虑时区和时间部分的影响
   - 使用 DateOnly 格式存储日期

3. **性能要求**
   - 批量排产支持分页处理（limit/skip）
   - 并发计算滞港费（CONCURRENCY=5）
   - 缓存仓库档期查询结果

4. **可维护性**
   - 完整的日志记录
   - 清晰的错误处理
   - 可配置的参数管理

```typescript
  const options: UnloadOption[] = [];
  
  // 1. 获取候选仓库列表
  const warehouses = await this.getCandidateWarehouses(
    countryCode, 
    portCode
  );

  for (const warehouse of warehouses) {

        strategy: 'Direct',
        isWithinFreePeriod: candidateDate <= lastFreeDate
      });
    }
  }

  const dropOffOptions = await this.generateDropOffOptions(...);
  options.push(...dropOffOptions);

  const expeditedOptions = await this.generateExpeditedOptions(...);
  options.push(...expeditedOptions);
```

2. **成本评估逻辑** (L333-475)
```typescript
async evaluateTotalCost(option: UnloadOption): Promise<CostBreakdown> {
  const breakdown: CostBreakdown = {
    demurrageCost: 0,
    detentionCost: 0,
    storageCost: 0,
    yardStorageCost: 0,
    transportationCost: 0,
    handlingCost: 0,
    totalCost: 0
  };
  
  // 1. 根据策略计算计划日期
  const plannedPickupDate = option.plannedPickupDate;
  let actualPlannedUnloadDate = option.plannedUnloadDate;
  
  if (!actualPlannedUnloadDate) {
    if (option.strategy === 'Drop off') {
      // Drop off: 假设提柜后 2 天卸柜
      actualPlannedUnloadDate = dateTimeUtils.addDays(
        option.plannedPickupDate, 
        2
      );
    } else {
      // Direct/Expedited: 提=卸
      actualPlannedUnloadDate = option.plannedPickupDate;
    }
  }
  
  // 2. 估算还箱日
  let plannedReturnDate: Date;
  if (option.strategy === 'Drop off') {
    plannedReturnDate = dateTimeUtils.addDays(actualPlannedUnloadDate, 3);
  } else {
    plannedReturnDate = actualPlannedUnloadDate;
  }
  
  // 3. 使用 DemurrageService 计算所有费用
  const totalCostResult = await this.demurrageService.calculateTotalCost(
    option.containerNumber,
    {
      mode: 'forecast',
      plannedDates: {
        plannedPickupDate,
        plannedUnloadDate: actualPlannedUnloadDate,
        plannedReturnDate
      },
      includeTransport: true,
      warehouse: option.warehouse,
      truckingCompany: option.truckingCompany,
      unloadMode: option.strategy === 'Drop off' ? 'Drop off' : 'Live load'
    }
  );
  
  // 4. 填充成本明细
  breakdown.demurrageCost = totalCostResult.demurrageCost;
  breakdown.detentionCost = totalCostResult.detentionCost;
  breakdown.storageCost = totalCostResult.storageCost;
  breakdown.transportationCost = totalCostResult.transportationCost;
  
  // 5. 计算外部堆场堆存费（Drop off 模式专属）
  if (option.strategy === 'Drop off' && option.truckingCompany?.hasYard) {
    const yardStorageDays = dateTimeUtils.daysBetween(
      option.plannedPickupDate,
      plannedDeliveryDate
    );
    breakdown.yardStorageCost = 
      (yardStorageDays * yardDailyRate) + yardOperationFee;
  }
  
  // 6. 加急费
  if (option.strategy === 'Expedited') {
    breakdown.handlingCost = await this.getConfigNumber(
      'expedited_handling_fee', 
      50
    );
  }
  
  // 7. 总成本
  breakdown.totalCost = 
    breakdown.demurrageCost +
    breakdown.detentionCost +
    breakdown.storageCost +
    breakdown.transportationCost +
    breakdown.yardStorageCost +
    breakdown.handlingCost;
  
  return breakdown;
}
```

3. **成本优化建议逻辑** (L712-904)
```typescript
async suggestOptimalUnloadDate(...) {
  // 1. 获取货柜信息
  const container = await this.containerRepo.findOne({
    where: { containerNumber },
    relations: ['portOperations']
  });
  
  // 2. 获取准确的免费期截止日（从 DB）
  const demurrageResult = await this.demurrageService.calculateForContainer(
    containerNumber
  );
  const effectiveLastFreeDate = demurrageResult.result?.calculationDates
    ?.lastPickupDateComputed || lastFreeDate;
  
  // 3. 计算当前方案成本
  const currentOption: UnloadOption = {
    containerNumber,
    warehouse,
    plannedPickupDate: basePickupDate,
    strategy: truckingCompany.hasYard ? 'Drop off' : 'Direct',
    truckingCompany,
    isWithinFreePeriod: basePickupDate <= effectiveLastFreeDate
  };
  const originalCost = (await this.evaluateTotalCost(currentOption)).totalCost;
  
  // 4. 向前/向后探索最多 7 天
  const candidates = [];
  for (let offset = -7; offset <= 7; offset++) {
    const candidateDate = dateTimeUtils.addDays(basePickupDate, offset);
    
    // 跳过过去和周末
    if (candidateDate < new Date() || this.isWeekend(candidateDate)) {
      continue;
    }
    
    // 检查仓库档期
    if (!await this.isWarehouseAvailable(warehouse, candidateDate)) {
      continue;
    }
    
    // 评估不同策略
    const strategies = [
      'Direct',
      ...(truckingCompany.hasYard ? ['Drop off'] : []),
      ...(candidateDate <= effectiveLastFreeDate ? ['Expedited'] : [])
    ];
    
    for (const strategy of strategies) {
      const option: UnloadOption = {
        containerNumber,
        warehouse,
        plannedPickupDate: candidateDate,
        strategy,
        truckingCompany
      };
      
      const breakdown = await this.evaluateTotalCost(option);
      candidates.push({
        pickupDate: candidateDate,
        strategy,
        totalCost: breakdown.totalCost
      });
    }
  }
  
  // 5. 找到成本最低的方案
  const optimalCandidate = candidates.reduce(
    (min, curr) => curr.totalCost < min.totalCost ? curr : min
  );
  
  const optimizedCost = optimalCandidate.totalCost;
  const savings = originalCost - optimizedCost;
  const savingsPercent = originalCost > 0 
    ? (savings / originalCost) * 100 
    : 0;
  
  return {
    suggestedPickupDate: optimalCandidate.pickupDate,
    suggestedStrategy: optimalCandidate.strategy,
    originalCost,
    optimizedCost,
    savings,
    savingsPercent,
    alternatives: candidates.sort((a, b) => a.totalCost - b.totalCost).slice(0, 3)
  };
}
```

```typescript
async isWarehouseAvailable(
  warehouse: Warehouse, 
  date: Date
): Promise<boolean> {
  // 使用智能日历能力确保档期记录存在
  const occupancy = await smartCalendarCapacity.ensureWarehouseOccupancy(
    warehouse.warehouseCode,
    date
  );
  
  if (!occupancy) {
    log.warn(`No occupancy record for ${warehouse.warehouseCode} on ${date}`);
    return false;
  }
  
  // 检查剩余容量
  const available = occupancy.remaining > 0;
  
  if (!available) {
    log.debug(
      `Warehouse ${warehouse.warehouseCode} not available on ${date}: ` +
      `capacity=${occupancy.capacity}, ` +
      `planned=${occupancy.plannedCount}, ` +
      `remaining=${occupancy.remaining}`
    );
  }
  
  return available;
}
```

```typescript
async decrementWarehouseOccupancy(
  warehouseCode: string,
  date: Date
): Promise<void> {
  const queryDate = new Date(date);
  queryDate.setHours(0, 0, 0, 0);
  
  const occupancy = await this.warehouseOccupancyRepo.findOne({
    where: { warehouseCode, date: queryDate }
  });
  
  if (occupancy) {
    occupancy.plannedCount += 1;
    await this.warehouseOccupancyRepo.save(occupancy);
  } else {
    // 获取仓库默认容量
    const warehouse = await this.warehouseRepo.findOne({
      where: { warehouseCode }
    });
    const capacity = warehouse?.dailyUnloadCapacity || 10;
    
    await this.warehouseOccupancyRepo.insert({
      warehouseCode,
      date: queryDate,
      capacity,
      plannedCount: 1,
      remaining: capacity - 1
    });
  }
}
```

#### 2.3.3 候选仓库选择

```typescript
async getCandidateWarehouses(
  countryCode: string,
  portCode: string
): Promise<Warehouse[]> {
  // 1. 获取港口 - 车队映射
  const truckingPortMappings = await this.truckingPortMappingRepo.find({
    where: { country: countryCode, portCode, isActive: true }
  });
  
  const truckingCompanyIds = truckingPortMappings.map(
    m => m.truckingCompanyId
  );
  
  // 2. 通过车队 - 仓库映射获取仓库
  if (truckingCompanyIds.length > 0) {
    const warehouseTruckingMappings = await this.warehouseTruckingMappingRepo.find({
      where: {
        country: countryCode,
        truckingCompanyId: In(truckingCompanyIds),
        isActive: true
      }
    });
    
    const warehouseCodes = warehouseTruckingMappings.map(
      m => m.warehouseCode
    );
    
    if (warehouseCodes.length > 0) {
      const warehouses = await this.warehouseRepo.find({
        where: {
          warehouseCode: In(warehouseCodes),
          status: 'ACTIVE'
        }
      });
      
      return warehouses;
    }
  }
  
  // 3. Fallback: 返回同一国家的活跃仓库
  return await this.warehouseRepo.find({
    where: { country: countryCode, status: 'ACTIVE' }
  });
}
```

```typescript
private async getConfigNumber(
```

## 四、关键技术决策

### 4.1 为什么以卸柜日为锚点？

**原因**:
1. 仓库资源是最紧缺的约束
2. 卸柜日决定了仓库档期占用
3. 其他日期都可以基于卸柜日推导

**优势**:
- 优先保证仓库资源可用性
- 避免因仓库满负荷导致排产失败
- 符合"先到先得"的业务规则

### 4.2 为什么使用启发式算法？

**原因**:
1. 问题规模大（数百个货柜）
2. 约束条件复杂（仓库、车队、免费期等）
3. 需要实时响应（用户操作）

**启发式策略**:
- 按 ETA 排序（先到先得）
- 贪心选择最早可用日期
- 局部搜索优化（±7 天成本对比）

### 4.3 为什么分离两个 Service？

**职责分离**:
- `IntelligentSchedulingService`: 规则引擎，负责日期计算和档期扣减
- `SchedulingCostOptimizerService`: 优化引擎，负责成本评估和方案推荐

**优势**:
- 单一职责原则
- 便于测试和维护
- 支持独立演进

## 五、典型问题与解决方案

### 5.1 仓库容量不足

**问题表现**:
```
排产失败：Warehouse WH001 has no available capacity on 2026-03-20
```

**解决方案**:
```typescript
// 向后查找最近可用日期
async findEarliestAvailableDay(
  warehouseCode: string,
  startDate: Date
): Promise<Date | null> {
  for (let i = 0; i < 30; i++) {
    const checkDate = dateTimeUtils.addDays(startDate, i);
    
    // 跳过周末
    if (this.isWeekend(checkDate) && await this.shouldSkipWeekends()) {
      continue;
    }
    
    // 检查产能
    const occupancy = await this.getOccupancy(warehouseCode, checkDate);
    if (!occupancy || occupancy.remaining > 0) {
      return checkDate;
    }
  }
  return null; // 30 天内都无容量
}
```

### 5.3 Drop off 模式堆场费未计算

**问题表现**:
```
Drop off 方案的总成本比 Direct 低，但实际应该更高
```

**根因**:
- 未判断是否实际使用了堆场
- 未从 TruckingPortMapping 读取堆场费率

**解决方案**:
```typescript
// ✅ 正确做法：判断提柜日 < 送仓日，才计算堆场费
if (option.strategy === 'Drop off' && truckingCompany.hasYard) {
  const plannedDeliveryDate = actualPlannedUnloadDate; // Drop off: 送=卸
  
  const pickupDayStr = option.plannedPickupDate.toISOString().split('T')[0];
  const deliveryDayStr = plannedDeliveryDate.toISOString().split('T')[0];
  
  if (pickupDayStr !== deliveryDayStr) {
    // ✅ 提 < 送，说明货柜在堆场存放了
    const yardStorageDays = dateTimeUtils.daysBetween(
      option.plannedPickupDate,
      plannedDeliveryDate
    );
    
    // 从 TruckingPortMapping 获取堆场费率
    const truckingPortMapping = await this.truckingPortMappingRepo.findOne({
      where: {
        country: countryCode,
        portCode,
        truckingCompanyId: option.truckingCompany.companyCode,
        isActive: true
      }
    });
    
    if (truckingPortMapping) {
      breakdown.yardStorageCost = 
        (truckingPortMapping.standardRate || 0) * yardStorageDays +
        (truckingPortMapping.yardOperationFee || 0);
    }
  }
}
```
