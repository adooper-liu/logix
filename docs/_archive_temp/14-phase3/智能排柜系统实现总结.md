## 系统概述

### 1.1 系统定位

智能排柜系统是 LogiX 物流管理平台的**核心业务引擎**，负责自动化安排集装箱的整个物流流程：

```
到港 (ATA/ETA) → 清关 → 提柜 → 送仓 → 卸柜 → 还箱
```

### 1.2 核心价值

1. **自动化排产**: 根据到港日期、免费期、仓库产能自动计算最优排产计划
2. **成本优化**: 对比多种卸柜方案 (Live load vs Drop off)，选择成本最低方案
3. **资源管理**: 管理仓库日产能、车队档期，避免资源冲突
4. **智能决策**: 基于规则引擎和成本模型，提供智能推荐

### 1.3 技术栈

- **后端**: Node.js + TypeScript + TypeORM
- **前端**: Vue 3 + Element Plus + Vite
- **数据库**: PostgreSQL (业务数据) + TimescaleDB (时序数据)
- **缓存**: Redis
- **搜索**: Elasticsearch

## 功能模块详解

### 3.1 智能排产引擎

**核心文件**: `backend/src/services/intelligentScheduling.service.ts` (1284 行)

#### 主流程：batchSchedule

```typescript
async batchSchedule(request: ScheduleRequest): Promise<BatchScheduleResponse> {
  // 1. 获取待排产货柜
  const containers = await getContainersToSchedule(request);
  // 过滤：scheduleStatus IN ['initial', 'issued']
  // 加载：portOperations, seaFreight, replenishmentOrders, customer

  // 2. 排序 (先到先得)
  const sortedContainers = sortByClearanceDate(containers);
  // 优先级：ATA > ETA
  // 同日内：按 lastFreeDate 升序

  // 3. 滞港费预计算 (批量写回 lastFreeDate)
  // 并发计算 (CONCURRENCY=5)，避免首批延迟

  // 4. 分步排产 (支持分页)
  const toProcess = sortedContainers.slice(skip, skip + limit);
  for (const container of toProcess) {
    const result = await scheduleSingleContainer(container, request);
    results.push(result);
  }

  // 5. 返回结果
  return {
    success: true,
    total: containers.length,
    successCount: results.filter(r => r.success).length,
    results,
    hasMore: skip + limit < sortedContainers.length
  };
}
```

#### 单柜排产：scheduleSingleContainer

**10 步核心流程**:

```
步骤 1: 获取目的港操作记录
  └─ 提取 ATA/ETA、lastFreeDate

步骤 2: 计算计划清关日
  └─ plannedCustomsDate = ETA (无则用 ATA)

步骤 3: 计算计划提柜日
  └─ plannedPickupDate = 清关日 + 1 天
  └─ 约束：≤ lastFreeDate
  └─ 下限：至少为今天
  └─ 跳过周末 (如果 skip_weekends=true)

步骤 4: 确定候选仓库
  └─ 映射链：港口 → 车队 → 仓库
  └─ 严格匹配 dict_trucking_port_mapping + dict_warehouse_trucking_mapping
  └─ 无映射返回 []，不回退

步骤 5: 选择最早可用仓库
  └─ 从提柜日起查询仓库日产能
  └─ 检查 ext_warehouse_daily_occupancy
  └─ 返回最早有产能的仓库和卸柜日

步骤 6: 选择车队
  └─ 严格匹配映射关系
  └─ 检查车队档期占用

步骤 7: 决定卸柜方式
  └─ hasYard = true → Drop off (提<送=卸)
  └─ hasYard = false → Live load (提=送=卸)
  └─ 自动调整：无堆场但提≠卸时，调整为 Live load

步骤 8: 计算计划还箱日
  └─ Live load: 还=卸 (同日)
  └─ Drop off: 还=卸 +1
  └─ 约束：≤ lastReturnDate (如有)

步骤 9: 选择清关公司
  └─ 根据国家匹配
  └─ 无匹配使用 "UNSPECIFIED"

步骤 10: 更新数据库 (dryRun 模式跳过)
  └─ 更新 Container/PortOperation/TruckingTransport/
      WarehouseOperation/EmptyReturn
  └─ 扣减仓库产能 (decrementWarehouseOccupancy)
  └─ 扣减车队档期 (decrementTruckingOccupancy)
  └─ 扣减还箱档期 (decrementFleetReturnOccupancy, Drop off 模式)
  └─ 同步状态 (updateStatus)
```

#### 关键约束规则

**五节点日期约束**:

```
清关 <= 提柜 <= 送仓 <= 卸柜 <= 还箱
```

**卸柜方式规则**:

```
Live load (无堆场):
  提柜日 = 送仓日 = 卸柜日
  还箱日 = 卸柜日

Drop off (有堆场):
  提柜日 < 送仓日 = 卸柜日
  还箱日 = 卸柜日 + 1
```

```typescript
async calculateWarehouseCapacity(warehouseCode, date): Promise<number> {
  // 1. 获取仓库基础产能
  const warehouse = await getWarehouse(warehouseCode);
  const baseCapacity = warehouse.dailyUnloadCapacity; // 默认 10

  // 2. 判断是否为休息日
  const isRestDay = await isRestDay(date);
  if (isRestDay) return 0;

  // 3. 应用工作日倍率
  const config = await getCalendarConfig();
  const finalCapacity = baseCapacity * config.weekdayMultiplier;

  return finalCapacity;
}
```

### 5.1 核心 API 端点

#### POST /api/v1/scheduling/batch-schedule

**功能**: 批量排产

**请求**:

```typescript
interface BatchScheduleRequest {
  country?: string; // 国家过滤 (GB, US, CA)
  startDate?: string; // 开始日期 (YYYY-MM-DD)
  endDate?: string; // 结束日期 (YYYY-MM-DD)
  forceSchedule?: boolean; // 是否强制排产 (忽略已排产状态)
  containerNumbers?: string[]; // 指定柜号列表 (用于单柜模拟)
  limit?: number; // 每批处理数量 (默认：全部)
  skip?: number; // 跳过数量 (分页用)
  dryRun?: boolean; // 预览模式 (true=只计算不保存)
}
```

**响应**:

```typescript
interface BatchScheduleResponse {
  success: boolean;
  total: number; // 总货柜数
  successCount: number; // 成功数
  failedCount: number; // 失败数
  results: ScheduleResult[]; // 排产结果列表
  hasMore?: boolean; // 是否还有待排产货柜
}

interface ScheduleResult {
  containerNumber: string;
  success: boolean;
  message?: string;
  plannedData?: {
    plannedCustomsDate: string; // 计划清关日
    plannedPickupDate: string; // 计划提柜日
    plannedDeliveryDate: string; // 计划送仓日
    plannedUnloadDate: string; // 计划卸柜日
    plannedReturnDate: string; // 计划还箱日
    truckingCompanyId: string; // 车队 ID
    warehouseId: string; // 仓库 ID
    unloadModePlan: string; // Live load / Drop off
    customsBrokerCode: string; // 清关公司代码
    estimatedCosts?: object; // 预估费用 (dryRun 模式)
  };
  destinationPort?: string; // 目的港代码
  destinationPortName?: string; // 目的港名称
  warehouseName?: string; // 仓库名称
  etaDestPort?: string; // 预计到港日
  ataDestPort?: string; // 实际到港日
}
```

## 核心算法

### 6.1 日期计算算法

#### 计划提柜日计算

```typescript
/**
 * 计算计划提柜日
 * 规则：= 清关日 + 1 天，且 ≤ lastFreeDate
 * 下限：至少为今天
 */
private async calculatePlannedPickupDate(
  customsDate: Date,
  lastFreeDate?: Date
): Promise<Date> {
  const pickupDate = new Date(customsDate);
  pickupDate.setDate(pickupDate.getDate() + 1); // 清关后次日提柜

  // 约束：不能超过免费期
  if (lastFreeDate) {
    const lastFree = new Date(lastFreeDate);
    lastFree.setHours(0, 0, 0, 0);
    if (pickupDate > lastFree) {
      pickupDate.setTime(lastFree.getTime());
    }
  }

  // 跳过周末 (如果配置了 skip_weekends = true)
  await this.skipWeekendsIfNeeded(pickupDate);

  return pickupDate;
}

/**
 * 跳过周末
 */
private async skipWeekendsIfNeeded(date: Date): Promise<void> {
  const config = await this.schedulingConfigRepo.findOne({
    where: { configKey: 'skip_weekends' }
  });

  const shouldSkipWeekends = config?.configValue === 'true';
  if (!shouldSkipWeekends) return;

  // 跳过周六 (6) 和周日 (0)
  while (date.getDay() === 0 || date.getDay() === 6) {
    date.setDate(date.getDate() + 1);
  }
}
```

#### 计划还箱日计算

```typescript
/**
 * 计算计划还箱日
 * 规则：
 * - Live load: 还 = 卸 (同日)
 * - Drop off: 还 = 卸 + 1
 * 约束：≤ lastReturnDate (如有)
 */
private calculatePlannedReturnDate(
  unloadDate: Date,
  unloadMode: string,
  lastReturnDate?: Date
): Date {
  const returnDate = new Date(unloadDate);

  // 根据卸柜方式决定还箱日
  if (unloadMode === 'Drop off') {
    returnDate.setDate(returnDate.getDate() + 1); // Drop off 次日还箱
  }

  // 不能超过最晚还箱日
  if (lastReturnDate) {
    const lastReturn = new Date(lastReturnDate);
    lastReturn.setHours(0, 0, 0, 0);
    if (returnDate > lastReturn) {
      returnDate.setTime(lastReturn.getTime());
    }
  }

  // 兜底：还箱日 >= 卸柜日
  const unloadOnly = new Date(unloadDate);
  unloadOnly.setHours(0, 0, 0, 0);
  if (returnDate < unloadOnly) {
    returnDate.setTime(unloadOnly.getTime());
    if (unloadMode === 'Drop off') {
      returnDate.setDate(returnDate.getDate() + 1);
    }
  }

  return returnDate;
}
```

### 6.2 仓库选择算法

```typescript
/**
 * 找最早有产能的仓库和卸柜日
 */
private async findEarliestAvailableWarehouse(
  warehouses: Warehouse[],
  startDate: Date
): Promise<{ warehouse: Warehouse; plannedUnloadDate: Date } | null> {
  // 按仓库优先级排序
  const sortedWarehouses = warehouses.sort((a, b) => {
    // 优先级：is_default > 自营仓 > 平台仓 > 第三方仓 > warehouse_code
    if (a.isDefault !== b.isDefault) return a.isDefault ? -1 : 1;

    const typePriority = IntelligentSchedulingService.PROPERTY_TYPE_PRIORITY;
    const aPriority = typePriority[a.propertyType] || 999;
    const bPriority = typePriority[b.propertyType] || 999;
    if (aPriority !== bPriority) return aPriority - bPriority;

    return a.warehouseCode.localeCompare(b.warehouseCode);
  });

  // 对每个仓库，从 startDate 开始找最早可用日
  for (const warehouse of sortedWarehouses) {
    const availableDate = await this.findEarliestAvailableDay(
      warehouse.warehouseCode,
      startDate
    );

    if (availableDate) {
      return { warehouse, plannedUnloadDate: availableDate };
    }
  }

  return null; // 所有仓库产能都不足
}

/**
 * 找仓库最早可用日
 */
private async findEarliestAvailableDay(
  warehouseCode: string,
  startDate: Date
): Promise<Date | null> {
  let date = new Date(startDate);
  date.setHours(0, 0, 0, 0);

  // 从 startDate 起最多查 30 天
  for (let i = 0; i < 30; i++) {
    const occupancy = await this.warehouseOccupancyRepo.findOne({
      where: { warehouseCode, date }
    });

    const warehouse = await this.warehouseRepo.findOne({
      where: { warehouseCode }
    });
    const capacity = warehouse?.dailyUnloadCapacity || 10;

    if (!occupancy || occupancy.plannedCount < capacity) {
      return date; // 找到可用日
    }

    // 产能已满，检查下一天
    date.setDate(date.getDate() + 1);
  }

  return null; // 30 天内都无产能
}
```

### 6.3 车队选择算法

```typescript
/**
 * 选择车队 (严格匹配映射关系)
 */
private async selectTruckingCompany(
  warehouseCode: string,
  portCode?: string,
  plannedPickupDate?: Date,
  countryCode?: string
): Promise<TruckingCompany | null> {
  const checkDate = plannedPickupDate || new Date();
  const dateOnly = new Date(checkDate);
  dateOnly.setHours(0, 0, 0, 0);

  // 1. 通过映射链获取候选车队
  const mappings = await this.warehouseTruckingMappingRepo.find({
    where: { warehouseCode }
  });

  const candidateIds = mappings.map(m => m.truckingCompanyId);

  // 2. 过滤服务该港口的车队
  const portMappings = await this.truckingPortMappingRepo.find({
    where: {
      country: countryCode,
      portCode,
      isActive: true
    }
  });

  const servingPortIds = portMappings.map(m => m.truckingCompanyId);
  const validCandidates = candidateIds.filter(id =>
    servingPortIds.includes(id)
  );

  // 3. 检查档期，返回第一个有产能的车队
  for (const truckingId of validCandidates) {
    const occupancy = await this.truckingOccupancyRepo.findOne({
      where: {
        truckingCompanyId: truckingId,
        date: dateOnly,
        portCode,
        warehouseCode
      }
    });

    const trucking = await this.truckingCompanyRepo.findOne({
      where: { companyCode: truckingId }
    });
    const capacity = trucking?.dailyCapacity || 10;

    if (!occupancy || occupancy.plannedTrips < capacity) {
      return trucking;
    }
  }

  return null; // 无可用车辆
}
```

### 6.4 成本计算算法

```typescript
/**
 * 评估卸柜方案总成本
 */
async evaluateTotalCost(option: UnloadOption): Promise<CostBreakdown> {
  const breakdown: CostBreakdown = {
    transportCost: 0,
    storageCost: 0,
    laborCost: 0,
    expeditedFee: 0,
    externalStorageFee: 0,
    totalCost: 0
  };

  // 1. 运输成本 (基于距离)
  const distance = this.getDistance(option.warehouse.portCode, option.warehouse.code);
  const transportRate = await this.getTransportRate();
  breakdown.transportCost = distance * transportRate;

  // 2. 仓储成本
  const storageDays = this.calculateStorageDays(option);
  const storageRate = await this.getStorageRate();
  breakdown.storageCost = storageDays * storageRate;

  // 3. 人工成本
  const laborRate = await this.getLaborRate();
  breakdown.laborCost = laborRate;

  // 4. 加急费 (如果选择 Expedited 策略)
  if (option.strategy === 'Expedited') {
    breakdown.expeditedFee = await this.getExpeditedFee();
  }

  // 5. 外部存储费 (如果超过免费期)
  if (!option.isWithinFreePeriod) {
    breakdown.externalStorageFee = await this.getExternalStorageFee();
  }

  // 6. 总成本
  breakdown.totalCost =
    breakdown.transportCost +
    breakdown.storageCost +
    breakdown.laborCost +
    breakdown.expeditedFee +
    breakdown.externalStorageFee;

  return breakdown;
}
```

## 性能优化

### 9.1 批量处理优化

```typescript
// 并发计算滞港费 (CONCURRENCY=5)
const CONCURRENCY = 5;
const lastFreeByCn: Record<string, Date> = {};

for (let i = 0; i < numbers.length; i += CONCURRENCY) {
  const batch = numbers.slice(i, i + CONCURRENCY);
  const settled = await Promise.allSettled(batch.map((cn) => this.demurrageService.calculateForContainer(cn)));

  for (let j = 0; j < batch.length; j++) {
    const s = settled[j];
    const computed = s.status === "fulfilled" ? s.value?.result?.calculationDates?.lastPickupDateComputed : null;
    if (computed) lastFreeByCn[batch[j]] = new Date(computed);
  }
}
```

**优势**:

- 避免串行计算导致的延迟
- 每批 5 个并发，平衡性能和资源
- 使用 Promise.allSettled 容忍部分失败

### 9.2 分步排产

```typescript
const BATCH_SIZE = 3; // 每批 3 个
let skip = 0;
let hasMore = true;

while (hasMore) {
  const result = await api.batchSchedule({
    limit: BATCH_SIZE,
    skip,
  });

  // 暂停，等待用户确认
  const continueConfirm = await ElMessageBox.confirm("是否继续？");
  if (!continueConfirm) break;

  skip += BATCH_SIZE;
  hasMore = result.hasMore;
}
```

**优势**:

- 避免一次性处理大量数据
- 用户可随时中断
- 降低数据库压力

### 9.3 档期预写回

```typescript
// 仅对本批货柜计算滞港费并写回 lastFreeDate
// 避免全量写回导致首批延迟
for (const container of toProcess) {
  const destPo = container.portOperations?.find((po) => po.portType === "destination");
  if (destPo && lastFreeByCn[container.containerNumber]) {
    destPo.lastFreeDate = lastFreeByCn[container.containerNumber];
  }
}
```

**优势**:

- 首批返回更快
- 按需计算，避免浪费

### 10.1 常见问题 Top 5

#### Q1: 排产失败 - "无映射关系中的仓库"

**原因**: 映射链不完整

**诊断 SQL**:

```sql
-- 检查车队 - 港口映射
SELECT * FROM dict_trucking_port_mapping
WHERE port_code = 'GBFXT' AND country = 'GB';

-- 检查仓库 - 车队映射
SELECT * FROM dict_warehouse_trucking_mapping
WHERE trucking_company_id = 'TC001';
```

**解决方案**:

```sql
-- 补充映射关系
INSERT INTO dict_trucking_port_mapping (trucking_company_id, port_code, country)
VALUES ('TC001', 'GBFXT', 'GB');

INSERT INTO dict_warehouse_trucking_mapping (warehouse_code, trucking_company_id)
VALUES ('WH001', 'TC001');
```

#### Q2: 产能不足

**原因**: 当日产能已满

**诊断 SQL**:

```sql
-- 查看占用情况
SELECT * FROM ext_warehouse_daily_occupancy
WHERE warehouse_code = 'WH001'
  AND date BETWEEN '2026-03-24' AND '2026-03-30'
ORDER BY date;
```

**解决方案**:

```sql
-- 临时增加产能
UPDATE warehouse
SET daily_unload_capacity = 30
WHERE warehouse_code = 'WH001';

-- 或手动调整占用表
UPDATE ext_warehouse_daily_occupancy
SET capacity = 30, remaining = capacity - planned_count
WHERE warehouse_code = 'WH001' AND date = '2026-03-25';
```

#### Q3: 日期计算异常 (未跳过周末)

**原因**: 周末配置未生效

**诊断 SQL**:

```sql
-- 检查配置
SELECT * FROM dict_scheduling_config
WHERE config_key IN ('skip_weekends', 'weekend_days');
```

**解决方案**:

```sql
-- 更新配置
INSERT INTO dict_scheduling_config (config_key, config_value, description)
VALUES ('skip_weekends', 'true', '排产时跳过周末')
ON CONFLICT (config_key) DO UPDATE SET config_value = 'true';
```

**原因**: updateStatus 执行失败
