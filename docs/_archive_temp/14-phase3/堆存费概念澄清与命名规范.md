## 📋 核心概念对比

| 维度 | 港口存储费（Storage Charge） | 外部堆场堆存费（Yard Storage Fee） |
|------|---------------------------|----------------------------------|
| **英文名称** | Storage Charge / Port Storage Fee | Yard Storage Fee / External Storage Fee |
| **收取方** | 港口/码头 | 拖车车队 |
| **数据源** | `ext_demurrage_standards` 表 | `dict_trucking_port_mapping` 表 |
| **字段映射** | `chargeTypeCode = 'STORAGE'` | `standard_rate` + `yard_operation_fee` |
| **判断条件** | 匹配到 STORAGE 类型标准 | `yard_capacity > 0` + Drop off 模式 |
| **费用范畴** | D&D 费用类型之一 | 运输环节附加费 |
| **计费区间** | 到港日 → 提柜日 | 卸柜日 → 还箱日（堆场存放期间） |
| **免费期** | 有（根据标准 freeDays） | 无（从第一天开始收费） |
| **阶梯费率** | 支持 | 不支持（固定费率） |
| **计算逻辑** | 与滞港费相同的复杂逻辑 | 简单公式：`rate * days + operation_fee` |

## 🔍 详细说明

### 1️⃣ 港口存储费（Storage Charge）

#### 定义
港口/码头为货柜在港区堆场内提供的存储服务收取的费用。

#### 业务场景
- 货柜到达目的港后，在港口指定的堆场存放
- 从到港日开始计算，到提柜日结束
- 通常有免费期（如 3-7 天）
- 超期后按阶梯费率加收

#### 计算逻辑
```typescript
// 位置：demurrage.service.ts - calculateForContainer()

// 1. 匹配标准
const standards = await matchStandards(containerNumber);
const storageStd = standards.find(s => isStorageCharge(s));

// 2. 计算免费期截止日
const freeDays = storageStd.freeDays || 0;
const lastFreeDate = addDays(arrivalDate, freeDays - 1);

// 3. 计算计费天数
const chargeDays = max(0, daysBetween(lastFreeDate, pickupDate));

// 4. 应用阶梯费率
const totalCost = calculateSingleDemurrage(
  arrivalDate,
  pickupDate,
  freeDays,
  ratePerDay,
  tiers, // 阶梯费率 [{fromDay: 1, toDay: 7, rate: 50}, ...]
  currency,
  freeDaysBasis // 工作日/自然日
);
```

#### 代码位置
- **主计算**: `demurrage.service.ts:1265-1476`
- **工具函数**: `isStorageCharge()` (Line 390-396)
- **汇总入口**: `calculateTotalCost()` (Line 3155-3156)

#### 示例
```
货柜 ECMU1234567
- 到港日：2026-03-20
- 提柜日：2026-03-28
- 免费期：5 天（自然日）
- 免费期截止：2026-03-24
- 计费天数：4 天（03-25 ~ 03-28）
- 费率：$50/天（前 7 天）
- 港口存储费：$200
```

### 2️⃣ 外部堆场堆存费（Yard Storage Fee）

#### 定义
拖车车队为其自有或合作的堆场设施收取的货柜临时存放费用。

#### 业务场景
- **仅 Drop off 模式**：提柜 < 送仓 = 卸柜
- 车队将货柜从港口提到自己的堆场
- 暂时存放数天（通常 3-5 天）
- 等待仓库有空闲档期后再送到仓库卸柜

#### 前提条件
```sql
-- dict_trucking_port_mapping 表
SELECT 
  trucking_company_id,
  yard_capacity,      -- 堆场容量（>0 表示有堆场）
  standard_rate,      -- 每日堆存费率
  yard_operation_fee  -- 一次性操作费
FROM dict_trucking_port_mapping
WHERE yard_capacity > 0;  -- ✅ 只有 >0 才提供堆场服务
```

#### 计算逻辑
```typescript
// 位置：待重构（当前分散在调用方）

// 仅在 Drop off 模式下计算
if (option.strategy === 'Drop off' && truckingCompany.hasYard) {
  const yardStorageDays = 3; // 预计堆放天数（可根据仓库档期调整）
  
  const yardStorageFee = 
    truckingPortMapping.standardRate * yardStorageDays + 
    truckingPortMapping.yardOperationFee;
  
  costs.yardStorageCost = yardStorageFee;
}
```

#### 示例
```
货柜 ECMU1234567 - Drop off 模式
- 提柜日：2026-03-25
- 卸柜日：2026-03-28（堆场存放 3 天）
- 车队堆场容量：50 TEU
- 堆存费率：$80/天
- 操作费：$50（一次性）
- 外部堆场堆存费：$80 × 3 + $50 = $290
```

## ❌ 常见误解与澄清

### 误解 1：所有堆存费都是一种费用

**❌ 错误理解**：
> "堆存费就是堆存费，为什么要分两种？"

**✅ 正确理解**：
- **港口存储费**：港口收取，D&D 范畴，有免费期和阶梯费率
- **外部堆场堆存费**：车队收取，运输附加费，无免费期，固定费率
- **两者完全不同**，就像"停车费"和"过路费"的区别

---

### 误解 2：TruckingPortMapping.standardRate 是港口存储费率

**❌ 错误理解**：
> "从 TruckingPortMapping 读取 standardRate 作为堆存费率"

**✅ 正确理解**：
- `dict_trucking_port_mapping.standardRate` 是**车队堆场的每日费率**
- 用于计算**外部堆场堆存费**（Drop off 模式专属）
- **不是**港口的 Storage Charge 费率
- 港口存储费应从 `ext_demurrage_standards` 读取

---

### 误解 3：Drop off 模式只需要计算一种堆存费

**❌ 错误理解**：
> "Drop off 模式的堆存费就是外部堆场堆存费"

**✅ 正确理解**：
Drop off 模式可能产生**两笔**堆存相关费用：

1. **港口存储费**（Storage Charge）
   - 如果货柜在港口堆场存放（提柜延迟）
   - 从到港日到实际提柜日
   - 由 `calculateForContainer()` 自动计算

2. **外部堆场堆存费**（Yard Storage Fee）
   - 如果货柜在车队堆场存放
   - 从提柜日到送仓库卸柜
   - 额外计算，加到总成本中

**示例场景**：
```
货柜 ECMU1234567
- 到港日：2026-03-20
- 提柜日：2026-03-25（延迟 5 天，产生港口存储费）
- 卸柜日：2026-03-28（车队堆场存放 3 天，产生外部堆场堆存费）

费用明细：
- 港口存储费：$50 × 2 天 = $100（假设免费期 3 天）
- 外部堆场堆存费：$80 × 3 天 + $50 = $290
- 总计：$390（两笔堆存费同时存在）
```

### 场景 2: 预览排产 / 自动排柜成本优化

```typescript
// intelligentScheduling.service.ts / schedulingCostOptimizer.service.ts

// 1. 使用统一的 calculateTotalCost 计算 D&D 费用
const totalCostResult = await this.demurrageService.calculateTotalCost(containerNumber, {
  mode: 'forecast',
  plannedDates: { plannedPickupDate, plannedUnloadDate, plannedReturnDate },
  includeTransport: true,
  warehouse,
  truckingCompany,
  unloadMode
});

// 2. D&D 费用已包含港口存储费
const portStorageCost = totalCostResult.storageCost; // ✅ 来自 ext_demurrage_standards

// 3. 如果是 Drop off 模式且有堆场，额外计算外部堆场堆存费
let yardStorageCost = 0;
if (unloadMode === 'Drop off' && truckingCompany.hasYard) {
  const truckingPortMapping = await this.truckingPortMappingRepo.findOne({
    where: {
      country: warehouse.country,
      portCode: destPo.portCode,
      truckingCompanyId: truckingCompany.companyCode,
      isActive: true
    }
  });
  
  if (truckingPortMapping) {
    const yardStorageDays = 3; // 预计堆放天数
    yardStorageCost = 
      truckingPortMapping.standardRate * yardStorageDays + 
      truckingPortMapping.yardOperationFee;
  }
}

// 4. 总成本
const totalCost = 
  totalCostResult.totalCost + // D&D + 运输费
  yardStorageCost;            // 外部堆场堆存费（如有）
```

## 📝 命名规范总结

### 中文命名
- ✅ **港口存储费**：指港口收取的 Storage Charge
- ✅ **外部堆场堆存费**：指车队收取的 Yard Storage Fee
- ❌ 避免单独使用"堆存费"（容易混淆）

### 英文命名
- ✅ **Storage Charge** / **Port Storage Fee**
- ✅ **Yard Storage Fee** / **External Storage Fee**
- ❌ 避免单独使用"Storage Fee"

### 代码变量命名
```typescript
// ✅ 推荐
portStorageCost      // 港口存储费
yardStorageCost      // 外部堆场堆存费
yardStorageFee       // 外部堆场堆存费（别名）

// ❌ 避免
storageCost          // 不明确，可能混淆
storageFee           // 不明确
```
