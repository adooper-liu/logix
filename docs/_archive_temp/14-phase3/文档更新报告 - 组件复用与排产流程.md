### 2. 预览排产与智能成本优化全流程指南

文件路径：`docs/Phase3/预览排产与智能成本优化全流程指南.md`  
文档行数：931 行  
维护目标：统一预览排产和成本优化的实现口径

#### 核心内容

**整体流程概览**:
```
预览排产 → 智能成本优化 → 应用选择方案 → 正式排产
```

**五个计划日期**:
1. plannedCustomsDate - 计划清关日（提柜日 - 1 天）
2. plannedPickupDate - 计划提柜日（清关日 + 1 天）
3. plannedDeliveryDate - 计划送仓日（提柜日 + 运输天数）
4. plannedUnloadDate - **核心锚点**（基于仓库可用性和日产能）
5. plannedReturnDate - 计划还箱日（卸柜日 + 在仓天数）

**三种卸柜方式**:
- Direct（直装）：提柜日当天有产能，无堆场费
- Drop off：提柜日无产能，需先拉到堆场，有堆场操作费 + 堆存费
- Expedited（加急）：免费期即将到期，紧急处理，有加急费

**智能成本优化流程**:
1. 从权威数据源获取 last_free_date（process_port_operations.last_free_date）
2. 生成所有可行方案（Direct/Drop off/Expedited）
3. 评估每个方案的总成本
4. 选择成本最低的方案
5. 提供优化建议（可节省金额）

**关键代码位置**:
- IntelligentSchedulingService.batchSchedule() - 批量排产
- SchedulingCostOptimizerService.optimizePickupDate() - 成本优化
- DemurrageService.calculateTotalCost() - 费用计算
- useShipmentsSchedule.executeDirectSchedule() - 前端调用

#### 遵循 SKILL 原则

**数据库优先**:
- last_free_date 只能从 process_port_operations 表读取
- 禁止通过 UPDATE 语句修补数据
- 正确流程：删除错误数据 → 修复映射/逻辑 → 重新计算

**日期口径统一**:
- 所有日期筛选必须使用统一的 DateRangePicker 组件
- 后端口径：actual_ship_date（备货单）→ shipment_date（海运）

**开发检查清单**:
- 开发前排产功能前必须理解的 6 个关键问题
- 开发成本优化功能前必须检查的 5 个要点
- 常见错误与避免方法（3 个典型案例）

3. **第三步**：实践开发
