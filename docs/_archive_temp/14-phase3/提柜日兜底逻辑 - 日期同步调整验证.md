# 提柜日兜底逻辑 - 日期同步调整验证

## 📊 日期计算链

当提柜日被调整为明天时，**所有后续日期都会同步调整**！

### 计算顺序

```
提柜日调整（明天）
    ↓
卸柜日（从提柜日起查仓库可用档期）
    ↓
送仓日（提柜日或卸柜日）
    ↓
还箱日（卸柜日或卸柜日 +1）
```

## 🔍 详细计算逻辑

### 1️⃣ 提柜日调整

```typescript

  const tomorrow = new Date(todayStr + "T00:00:00");
  tomorrow.setDate(tomorrow.getDate() + 1);
```

**结果**：

- 提柜日：明天（2026-03-27）
- 清关日：今天（2026-03-26）


**结果**：

- Live load：卸柜日 = 提柜日 = 明天
- Drop off：卸柜日 ≥ 提柜日（明天或后天）

### 3️⃣ 送仓日计算

```typescript
// L480-484
const plannedDeliveryDate = this.calculatePlannedDeliveryDate(
  plannedPickupDate,  // ← 使用调整后的提柜日
  unloadMode,
  unloadDate          // ← 使用调整后的卸柜日

// L877-887
private calculatePlannedDeliveryDate(pickupDate, unloadMode, unloadDate) {
  if (unloadMode === 'Live load') {
    return new Date(pickupDate); // 提 = 送
  }
  return new Date(unloadDate);   // Drop off: 送 = 卸
}
```

**结果**：

- Live load：送仓日 = 提柜日 = 明天
- Drop off：送仓日 = 卸柜日 ≥ 明天

---

### 4️⃣ 还箱日计算

```typescript
// L511-515
const plannedReturnDate = this.calculatePlannedReturnDate(
  unloadDate,       // ← 使用调整后的卸柜日
  unloadMode,
  lastReturnDate
);

// L894-921
private calculatePlannedReturnDate(unloadDate, unloadMode, lastReturnDate) {
  const returnDate = new Date(unloadDate);
  if (unloadMode === 'Drop off') {
    returnDate.setDate(returnDate.getDate() + 1); // Drop off: 还 = 卸 +1
  }
  // Live load: 还 = 卸
}
```

**结果**：

- Live load：还箱日 = 卸柜日 ≥ 明天
- Drop off：还箱日 = 卸柜日 +1 ≥ 后天

## 📋 完整日期链示例

### 场景：今天是 2026-03-26，ETA 是 2026-02-11

#### 初始计算（触发兜底）

```
ETA = 2026-02-11
etaBufferDays = 0

清关日 = ETA + 0 = 2026-02-11
提柜日 = 清关日 + 1 = 2026-02-12

兜底逻辑：提柜日 (2026-02-12) ≤ 今天 (2026-03-26)
→ 调整为明天
```

#### 调整后结果

**假设仓库在提柜日（明天）有档期**：

| 节点   | Live load            | Drop off             |
| ------ | -------------------- | -------------------- |
| 清关日 | 2026-03-26（今天）   | 2026-03-26（今天）   |
| 提柜日 | 2026-03-27（明天）✅ | 2026-03-27（明天）✅ |
| 送仓日 | 2026-03-27（明天）✅ | 2026-03-27（明天）✅ |
| 卸柜日 | 2026-03-27（明天）✅ | 2026-03-27（明天）✅ |
| 还箱日 | 2026-03-27（明天）✅ | 2026-03-28（后天）✅ |

**日期关系**：

- ✅ 清关日 < 提柜日
- ✅ 提柜日 ≤ 送仓日 ≤ 卸柜日 ≤ 还箱日
- ✅ 所有日期都是未来日期

## ✅ 结论

**是的，所有日期都会同步调整！**

### 原因

1. **提柜日是基准**：卸柜日从提柜日开始查询
2. **送仓日依赖提/卸**：送仓日 = 提柜日 或 卸柜日
3. **还箱日依赖卸柜日**：还箱日 = 卸柜日 或 卸柜日 +1

### 数据流

```
提柜日调整
    ↓ (作为参数传入)
卸柜日查询
    ↓ (作为参数传入)
送仓日计算
    ↓ (作为参数传入)
还箱日计算
```

### 业务规则

- ✅ 提柜日 ≤ 送仓日 ≤ 卸柜日 ≤ 还箱日
- ✅ 所有日期都是未来日期
- ✅ 日期关系始终保持正确
