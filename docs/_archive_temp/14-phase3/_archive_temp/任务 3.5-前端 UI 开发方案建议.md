## 💡 方案对比

### 方案 A：集成到现有智能排柜界面

**架构设计**:
```
SchedulingVisual.vue (扩展现有界面)
├── 现有功能保持不变
└── 新增"成本优化"标签页
    ├── CostEvaluationPanel.vue      # 成本评估面板
    ├── OptionComparisonTable.vue    # 方案对比表格
    ├── OptimizationSuggestion.vue   # 优化建议弹窗
    └── CostBreakdownChart.vue       # 成本明细图表
```

**优点**:
1. ✅ **复用现有组件**
   - 可直接使用现有的 `DateRangePicker`、`DemurrageCalculationPanel`
   - 可复用资源配置、占用情况的数据加载逻辑
   - 可共享排产结果数据

2. ✅ **用户体验一致**
   - 用户已在排柜界面，无需跳转
   - 操作流程连贯：排产 → 评估成本 → 选择方案
   - 统一的 UI 风格（Element Plus + 现有配色）

3. ✅ **开发效率高**
   - 无需重新搭建框架
   - 可直接调用现有的 `batchSchedule` API
   - 可复用现有的状态管理、缓存机制

4. ✅ **数据流简单**
   ```typescript
   // 直接调用现有 service
   const result = await schedulingService.batchSchedule(params);
   // 然后对每个 option 进行成本评估
   const costs = await Promise.all(
     result.results.map(option => 
       costOptimizerService.evaluateTotalCost(option)
     )
   );
   ```

**缺点**:
1. ❌ **界面复杂度高**
   - 现有界面已有 1910 行代码，继续扩展会更大
   - 功能过多可能导致用户认知负担

2. ❌ **性能风险**
   - 同时渲染多个复杂组件（甘特图、资源占用图、成本图表）
   - 大量数据计算可能阻塞主线程

3. ❌ **维护困难**
   - 单一文件过大（超过 2000 行）
   - 职责不够清晰（排产逻辑 + 成本优化逻辑混合）

**适用场景**:
- ✅ 成本优化是排产流程的自然延伸
- ✅ 用户需要在排产后立即查看成本评估
- ✅ 希望快速上线，减少开发量

## 🎯 推荐方案：方案 C（混合方案）

### 理由

1. **业务角度**
   - 成本优化是排产流程的自然延伸，不应割裂
   - 用户需要在排产后立即看到成本评估
   - 独立的成本管理中心是长期目标，但非当务之急

2. **技术角度**
   - 可充分利用现有组件和 API
   - 开发周期短（预计 2-3 天完成 Phase 1+2）
   - 代码质量可控，后续可优化

3. **用户体验角度**
   - 一站式完成排产和成本优化
   - 学习成本低（界面风格一致）
   - 操作流程符合直觉

4. **项目管理角度**
   - 风险最低（复用成熟代码）
   - 进度可控（分阶段交付）
   - 可扩展性强（未来可独立）

## 🛠️ 技术细节

### 后端 API 设计

**1. 评估单个方案成本**
```typescript
POST /api/v1/scheduling/evaluate-cost

Request:
{
  containerNumber: string,
  option: {
    warehouse: string,
    unloadDate: string,
    strategy: 'Direct' | 'Drop off' | 'Expedited',
    truckingCompany?: string
  }
}

Response:
{
  success: true,
  data: {
    containerNumber: string,
    option: UnloadOption,
    costBreakdown: CostBreakdown,
    recommendation?: string
  }
}
```

**2. 对比多个方案**
```typescript
POST /api/v1/scheduling/compare-options

Request:
{
  containerNumber: string,
  options: UnloadOption[]
}

Response:
{
  success: true,
  data: {
    containerNumber: string,
    comparisons: Array<{
      option: UnloadOption,
      costBreakdown: CostBreakdown,
      rank: number
    }>,
    recommendedOption: {
      option: UnloadOption,
      reason: string,
      savings: number
    }
  }
}
```

**3. 获取推荐方案**
```typescript
GET /api/v1/scheduling/recommend-option/:containerNumber

Response:
{
  success: true,
  data: {
    containerNumber: string,
    optimalOption: UnloadOption,
    costBreakdown: CostBreakdown,
    alternatives: UnloadOption[],
    reasoning: string
  }
}
```

### 前端组件设计

**CostEvaluationPanel.vue**
```vue
<template>
  <div class="cost-evaluation-panel">
    <el-card title="成本评估">
      <!-- 方案选择 -->
      <UnloadOptionSelector 
        v-model="selectedOption" 
        :options="availableOptions"
        @change="handleOptionChange"
      />
      
      <!-- 成本明细 -->
      <CostBreakdownDisplay 
        v-if="costBreakdown" 
        :data="costBreakdown"
      />
      
      <!-- 成本可视化 -->
      <CostVisualization 
        v-if="costBreakdown" 
        :data="costBreakdown"
        type="pie"
      />
      
      <!-- 操作按钮 -->
      <div class="actions">
        <el-button @click="evaluate">重新评估</el-button>
        <el-button type="primary" @click="select">选择此方案</el-button>
      </div>
    </el-card>
  </div>
</template>

<script setup lang="ts">
import { ref, watch } from 'vue';
import { costOptimizationService } from '@/services/costOptimization';
import type { UnloadOption, CostBreakdown } from '@/types/scheduling';

const props = defineProps<{
  containerNumber: string;
  availableOptions: UnloadOption[];
}>();

const emit = defineEmits<{
  optionSelected: [option: UnloadOption];
}>();

const selectedOption = ref<UnloadOption | null>(null);
const costBreakdown = ref<CostBreakdown | null>(null);
const loading = ref(false);

const handleOptionChange = async (option: UnloadOption) => {
  loading.value = true;
  try {
    const result = await costOptimizationService.evaluateCost(
      props.containerNumber,
      option
    );
    costBreakdown.value = result.data.costBreakdown;
  } finally {
    loading.value = false;
  }
};

const evaluate = () => {
  if (selectedOption.value) {
    handleOptionChange(selectedOption.value);
  }
};

const select = () => {
```
