## 🚀 部署步骤

### 1. 生产环境构建

**后端**:
```bash
cd backend
npm run build
npm run start:prod
```

**前端**:
```bash
cd frontend
npm run build
# 将 dist 目录部署到 Web 服务器
```

### 2. 数据库迁移

```bash
# 执行配置项迁移
psql -h <host> -U <user> -d logix -f migrations/add_cost_optimization_config.sql
```

### 3. 环境变量配置

**后端 .env**:
```bash
# 确保以下配置存在
DATABASE_URL=postgresql://...
PORT=3000
NODE_ENV=production
```

**前端 .env.production**:
```bash
VITE_API_BASE_URL=/api/v1
```

### 4. 重启服务

```bash
# 后端
pm2 restart logix-backend

# 前端（Nginx）
sudo systemctl restart nginx
```

## ⚠️ 常见问题排查

### Q1: 配置项不存在

**症状**: 成本计算为 0 或 NaN

**解决**:
```sql
-- 检查配置表
SELECT * FROM dict_scheduling_config;

-- 如果为空，重新执行迁移脚本
```

### Q2: TypeORM 元数据错误

**症状**: 启动时报实体元数据错误

**解决**:
```bash
# 清除缓存
rm -rf backend/dist
rm -rf backend/node_modules/.cache

# 重新编译
npm run build
```

### Q3: ECharts 不显示

**症状**: 饼图区域空白

**解决**:
```bash
# 确保安装了 echarts
npm install echarts --save

# 检查导入语句
import * as echarts from 'echarts'
```

### Q4: 组件通信失败

**症状**: 选择方案后无反应

**解决**:
- 检查 Props 类型定义
- 检查 Events 抛出
- 使用 Vue DevTools 调试
