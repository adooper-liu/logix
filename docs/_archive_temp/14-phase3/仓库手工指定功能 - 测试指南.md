## 📋 **测试清单**

### **1️⃣ 后端 API 测试**

#### **测试 1: 获取候选仓库列表 API**

**接口**: `GET /api/v1/scheduling/warehouses?portCode=LAX&countryCode=US`

**测试步骤**:

```bash
# 使用 curl 测试
curl -X GET "http://localhost:3000/api/v1/scheduling/warehouses?portCode=LAX&countryCode=US" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 或使用 Postman
GET http://localhost:3000/api/v1/scheduling/warehouses?portCode=LAX&countryCode=US
```

**预期响应**:

```json
{
  "success": true,
  "data": [
    {
      "warehouseCode": "US-W001",
      "warehouseName": "LA Warehouse",
      "propertyType": "自营仓",
      "country": "US",
      "dailyUnloadCapacity": 10,
      "status": "ACTIVE",
      "address": "123 Main St, Los Angeles"
    },
    {
      "warehouseCode": "US-W002",
      "warehouseName": "Platform Warehouse",
      "propertyType": "平台仓",
      "country": "US",
      "dailyUnloadCapacity": 5,
      "status": "ACTIVE",
      "address": "456 Elm St, Los Angeles"
    }
  ]
}
```

**测试场景**:

- ✅ 正常情况：提供有效的 portCode 和 countryCode
- ❌ 缺少 portCode: 应返回 400 错误
- ❌ 缺少 countryCode: 应返回 400 错误
- ❌ 无效港口代码：应返回空数组
- ❌ 无映射关系：应返回空数组

### **2️⃣ 后端排产功能测试**

#### **测试 2: 手工指定仓库排产**

**接口**: `POST /api/v1/containers/batch-schedule`

**请求体**:

```json
{
  "designatedWarehouseMode": true,
  "designatedWarehouseCode": "US-W001",
  "containerNumbers": ["CONT001", "CONT002"],
  "dryRun": false
}
```

**预期响应**:

```json
{
  "success": true,
  "total": 2,
  "successCount": 2,
  "failedCount": 0,
  "results": [
    {
      "containerNumber": "CONT001",
      "success": true,
      "message": "使用指定仓库 LA Warehouse 排产成功",
      "plannedData": {
        "warehouseId": "US-W001",
        "warehouseName": "LA Warehouse",
        "truckingCompanyId": "TRUCK_001",
        "unloadMode": "Drop off"
      }
    }
  ]
}
```

**错误场景测试**:

1. ❌ 仓库不存在：

   ```json
   {
     "designatedWarehouseMode": true,
     "designatedWarehouseCode": "INVALID_WAREHOUSE",
     "containerNumbers": ["CONT001"]
   }
   ```

   预期：返回错误消息"指定的仓库 INVALID_WAREHOUSE 不存在或已停用"

2. ❌ 仓库不在映射中：

   ```json
   {
     "designatedWarehouseMode": true,
     "designatedWarehouseCode": "US-W003",
     "containerNumbers": ["CONT001"],
     "portCode": "LAX"
   }
   ```

   预期：返回错误消息"指定的仓库 US-W003 不可用于该港口 LAX（请配置映射关系）"

3. ❌ 无到港日期：
   预期：返回错误消息"无到港日期（ATA/ETA），无法排产"

4. ❌ 仓库无产能：
   预期：返回错误消息"指定的仓库 XXX 在可预见的时间内无产能"

### **3️⃣ 前端功能测试**

#### **测试 3: 手工指定仓库对话框**

**前置条件**:

1. 有至少一个待排产的货柜
2. 货柜有目的港信息

**测试步骤**:

**Step 1: 选择货柜**

- 在排产页面选择一个或多个货柜
- 确认"手工指定"按钮可用（非禁用状态）

**Step 2: 打开对话框**

- 点击"手工指定"按钮
- 验证对话框弹出，标题为"手工指定仓库"

**Step 3: 验证仓库列表加载**

- 检查仓库列表是否按类型分组显示（自营仓、平台仓、第三方仓）
- 验证只显示可用于当前港口的仓库
- 验证仓库信息显示完整（代码、名称、类型、可用性）

**Step 4: 选择仓库**

- 选择一个仓库
- 验证下方显示仓库详细信息
- 验证信息包括：名称、类型、国家、地址

**Step 5: 选择柜号（可选）**

- 选择一个或多个柜号，或留空
- 验证提示信息："留空表示对所有选中柜生效"

**Step 6: 确认排产**

- 点击"确认排产"按钮
- 验证调用排产 API
- 验证显示执行日志
- 验证刷新数据

**Step 7: 取消操作**

- 点击"取消"按钮
- 验证对话框关闭
- 验证未调用 API

### **4️⃣ 集成测试**

#### **测试 4: 端到端流程**

**场景 A: 手工指定单个仓库**

1. 选择 3 个发往 LAX 的货柜
2. 点击"手工指定"
3. 选择"US-W001 - LA Warehouse"（自营仓）
4. 点击"确认排产"
5. 验证：
   - ✅ 3 个货柜都使用该仓库
   - ✅ 排产状态变为"已排产"
   - ✅ 仓库产能扣减
   - ✅ 车队档期扣减

**场景 B: 批量手工指定**

1. 筛选所有发往 LAX 的平台仓货柜（假设 5 个）
2. 批量选择这些货柜
3. 点击"手工指定"
4. 选择"US-W002 - Platform Warehouse"（平台仓）
5. 点击"确认排产"
6. 验证：
   - ✅ 5 个货柜都使用平台仓
   - ✅ 排产结果正确
   - ✅ 数据统计更新

**场景 C: 混合模式**

1. 对大部分货柜使用"智能排产"（自动选自营仓）
2. 对其中的特殊货柜（如 2 个）使用"手工指定"（选择平台仓）
3. 验证：
   - ✅ 两种模式互不干扰
   - ✅ 各自排产结果正确
   - ✅ 数据一致性良好

### **5️⃣ 边界条件测试**

#### **测试 5: 异常处理**

| 测试项           | 操作步骤                       | 预期结果                   |
| ---------------- | ------------------------------ | -------------------------- |
| **仓库不存在**   | 指定不存在的仓库代码           | 显示错误提示，不执行排产   |
| **仓库已停用**   | 指定 status != 'ACTIVE' 的仓库 | 显示错误提示，不执行排产   |
| **无映射关系**   | 指定不在港口映射中的仓库       | 显示错误提示，提示配置映射 |
| **无可用产能**   | 指定已满负荷的仓库             | 显示错误提示，建议其他日期 |
| **车队匹配失败** | 仓库无对应车队服务             | 显示错误提示，无法找到车队 |
| **网络异常**     | 断网或服务器宕机               | 显示友好错误提示           |
| **权限不足**     | 普通用户尝试手工指定           | 按钮禁用或提示无权限       |

### **6️⃣ 性能测试**

#### **测试 6: 大批量排产**

**场景**: 一次性手工指定 50 个货柜

- ✅ 数据一致性良好
