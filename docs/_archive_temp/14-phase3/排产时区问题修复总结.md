# 排产时区问题修复总结

## 📋 问题汇总

### 2026-03-26 发现的问题

#### 问题 1：提柜日显示为过去日期（3-25 日）

**现象**：

- 当前日期：2026-03-26
- 排产预览显示提柜日：2026-03-25（过去日期）
- 货币符号显示：英国仓库显示 `$` 而不是 `£`

**根本原因**：

1. **时区问题**：ETA 日期使用 UTC 存储，JavaScript 解析时转换为本地时间导致日期偏移
2. **日期比较问题**：直接使用 `Date` 对象比较，受时区影响不准确
3. **国家信息缺失**：`plannedData` 对象缺少 `warehouseCountry` 字段

---

#### 问题 2：Invalid Date 错误导致排产失败

**错误日志**：

```
RangeError: Invalid time value
    at Date.toISOString (<anonymous>)
    at IntelligentSchedulingService.scheduleSingleContainer (line 329)

2026-03-26 13:08:30 [error]: [IntelligentScheduling] Error scheduling container ECMU5381817: Invalid time value
2026-03-26 13:08:30 [error]: [IntelligentScheduling] Error scheduling container ECMU5399586: Invalid time value
```

**根本原因**：

- 日期计算过程中未验证日期有效性
- 如果输入日期无效，`toISOString()` 会抛出 `RangeError`
- 缺少防御式编程机制

### 修复 2：时区问题导致日期偏移

**文件**：`backend/src/services/intelligentScheduling.service.ts`

**修改 1**：强制使用本地时间创建 Date 对象（L308）

```typescript
// 修复前
const plannedCustomsDate = new Date(clearanceDate);

// 修复后
const plannedCustomsDate = new Date(clearanceDate + "T00:00:00");
// 添加 'T00:00:00' 强制 JavaScript 使用本地时间解析
```

**修改 2**：使用日期字符串比较（L326-338）

```typescript
// 修复前
const today = new Date();
today.setHours(0, 0, 0, 0);
if (plannedPickupDate < today) {
}

// 修复后
const todayStr = today.toISOString().split("T")[0];
const pickupDateStr = plannedPickupDate.toISOString().split("T")[0];

if (pickupDateStr < todayStr) {
  plannedPickupDate = new Date(todayStr + "T00:00:00");
  plannedCustomsDate.setTime(plannedPickupDate.getTime());
  plannedCustomsDate.setDate(plannedCustomsDate.getDate() - 1);
}
```

**效果**：

- ✅ 清关日 = ETA（正确日期，无偏移）
- ✅ 提柜日 = 清关日 + 1（正确计算）
- ✅ 不受服务器时区影响

## 🎯 技术要点

### 1. JavaScript 日期解析规则

```javascript
// UTC 解析（带 Z 或 +00:00）
new Date("2026-03-25T00:00:00Z"); // UTC 时间
new Date("2026-03-25T00:00:00+00:00"); // UTC 时间

// 本地时间解析（无时区标识）
new Date("2026-03-25"); // 本地时间 00:00
new Date("2026-03-25T00:00:00"); // 本地时间 00:00 ✅
```

### 2. 日期验证最佳实践

```javascript
// ✅ 验证 Date 对象是否有效
function isValidDate(date: Date): boolean {
  return date instanceof Date && !isNaN(date.getTime());
}

// ✅ 使用示例
if (!plannedPickupDate || isNaN(plannedPickupDate.getTime())) {
  logger.warn('Invalid date detected');
  return error;
}
```

### 3. 安全的日期转字符串

```javascript
// ✅ 先验证再转换
if (!date || isNaN(date.getTime())) {
  return new Date().toISOString().split("T")[0]; // 回退到今天
}
return date.toISOString().split("T")[0];
```

### 4. 多层验证策略

```
输入验证 → customsDate 是否有效
   ↓
计算验证 → calculatePlannedPickupDate 返回值
   ↓
使用验证 → toISOString() 前检查
   ↓
安全使用 ✅
```
