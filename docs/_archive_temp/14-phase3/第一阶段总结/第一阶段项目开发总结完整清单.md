## 3. 技术选型与架构决策

### 3.1 后端技术栈

| 技术 | 版本 | 选型理由 | 替代方案 | 实际效果 |
|------|------|---------|---------|---------|
| Node.js | 18+ | 高性能、生态丰富 | Java/Python | ✅ |
| TypeScript | 5.3+ | 类型安全、开发体验 | JavaScript | ✅ |
| Express | 4.18+ | 轻量、灵活、成熟 | NestJS/Koa | ✅ |
| TypeORM | 0.3+ | 支持 PostgreSQL、Active Record 模式 | Prisma/Sequelize | ⚠️ 有坑 |
| PostgreSQL | 14+ | 强大的关系型数据库 | MySQL/MariaDB | ✅ |
| TimescaleDB | latest | 时序数据优化 | InfluxDB | ✅ |
| Redis | 7+ | 缓存、会话管理 | Memcached | ✅ |
| Socket.IO | 4.6+ | 实时通信 | WebSocket 原生 | ✅ |
| Winston | 3.11+ | 日志管理 | Bunyan/Pino | ✅ |

**深度分析**:

- [ ] **为什么选择 TypeORM？**
  - 优点：Active Record 模式、迁移工具完善
  - 缺点：命名策略问题、复杂查询支持弱
  - 踩过的坑：SnakeNamingStrategy 问题
  - 解决方案：自定义 NamingStrategy + 手动指定字段名

- [ ] **为什么引入 TimescaleDB？**
  - 业务需求：大量时间序列数据（状态事件、监控数据）
  - 性能优势：自动分区、高效聚合查询
  - 实施难度：需要学习曲线
  - 实际收益：查询性能提升 X%

### 5.5 滞港费计算 ⭐🔥

**需要深度总结的知识点**:

- [ ] **滞港费计算模型** 🔥
  ```
  Demurrage = max(0, 实际在港天数 - 免堆期) × 日费率
  
  Detention = max(0, 实际用箱天数 - 免箱期) × 日费率
  ```

- [ ] **计算模式区分** 🔥
  - actual 模式：使用实际日期计算
  - forecast 模式：使用计划日期预测

- [ ] **标准匹配条件** 🔥
  - 进口国 (destination_country)
  - 目的港 (destination_port)
  - 船公司 (shipping_company)
  - 货代公司 (freight_forwarder)
  - 生效日期范围

- [ ] **日期计算逻辑** 🔥
  - 入口日期：ATA / GTIN
  - 出口日期：GTOT / RCVE
  - 免费截止日期：ATA + Free Storage Days
  - 最后提柜日期：Last Free Date

- [ ] **关键日期时间线** 🔥（新增）
  - 历时：上一节点结束 → 当前节点开始
  - 倒计时：当前节点开始 → 下一节点开始
  - 超期：当前节点开始 + 标准耗时 < 当前时间
