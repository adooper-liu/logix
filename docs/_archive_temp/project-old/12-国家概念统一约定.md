# 国家概念统一约定

## 一、核心定义

**「国家」作为字段名，在本项目中实质指：该国分公司（子公司）**，对应客户类型为 `SUBSIDIARY` 的客户。

- **dict_countries.code**：国家代码（如 US、CA、GB），为标准 ISO 代码。
- **别名**：UK 为 GB 的别名，后端 `normalizeCountryCode` 会将 UK 规范化为 GB。
- **该国分公司**：`biz_customers` 中 `customer_type_code = 'SUBSIDIARY'` 且 `country = dict_countries.code` 的客户。
- **约定**：每个国家代码（code）默认对应一个子公司类型客户；该国分公司名称 = 该子公司客户的 `customer_name`。

## 二、country 字段统一规则

### 2.1 存值规则

| 表 | 字段 | 存值类型 | 说明 |
|----|------|----------|------|
| dict_countries | code | 国家代码 | 主键，如 US、CA、GB |
| biz_customers | country | **国家代码** | 外键 → dict_countries.code |
| dict_warehouses | country | **国家代码** | 仓库所在国 |
| dict_trucking_port_mapping | country | **国家代码** | 车队-港口映射所属国（该国分公司） |
| dict_warehouse_trucking_mapping | country | **国家代码** | 仓库-车队映射所属国（该国分公司） |
| dict_overseas_companies | country | **国家代码** | 海外公司所在国 |
| dict_ports | country | 国家代码/名称 | 港口所在国（可为代码或名称，建议统一为 code） |
| dict_trucking_companies | country | **国家代码** | 拖车公司所在国 |
| dict_customs_brokers | country | **国家代码** | 清关公司所在国 |
| biz_replenishment_orders | **sell_to_country** | **特例** | 见下文 |

### 2.2 sell_to_country 特例

`sell_to_country` 为历史/业务兼容字段，**当前存值为子公司名称（customer_name）**，而非国家代码。

- **用途**：与 `biz_customers.customer_name` 关联，用于国家过滤、展示等。
- **数据链**：`order.sell_to_country = cust.customer_name` → `cust.country` = 国家代码。
- **业务含义**：销往该国分公司 = 销往该子公司客户。
- **转换**：需要国家代码时，应通过 `biz_customers` 关联：`cust.customer_name = order.sell_to_country` 取 `cust.country`；或通过 `order.customer_code` 关联客户取 `cust.country`。

### 2.3 国家过滤 SQL 统一写法

```sql
-- 通过 sell_to_country（子公司名称）关联取国家代码
LEFT JOIN biz_customers cust ON cust.customer_name = order.sell_to_country
WHERE cust.country = :countryCode   -- :countryCode 为 dict_countries.code

-- 或通过 customer_code 关联（更可靠）
LEFT JOIN biz_customers cust ON cust.customer_code = order.customer_code
WHERE cust.country = :countryCode
```

**优先使用 customer_code 关联**，因 customer_name 可能重名或变更。

## 三、各模块使用约定

### 3.1 统计与过滤（DateFilterBuilder / DateRangeSubquery）

- 国家筛选值：`dict_countries.code`（如 US、CA）。
- 数据链：`order.sell_to_country` 或 `order.customer_code` → `cust.country`。
- 当前实现：`cust.customer_name = order.sell_to_country`，建议逐步支持 `cust.customer_code = order.customer_code` 作为主路径。

### 3.2 智能排柜（getCandidateWarehouses）

- `dict_warehouses.country` 存国家代码。
- 备货单的 `sell_to_country` 存子公司名称，**需先转换为国家代码** 再与 `dict_warehouses.country` 匹配。
- 转换方式：通过 `order.customer_code` 查 `biz_customers.country`，或通过 `cust.customer_name = sell_to_country` 查 `cust.country`。
- **完整关联逻辑**：见 [13-销往国家到排柜资源关联逻辑.md](./13-销往国家到排柜资源关联逻辑.md)。

### 3.3 映射表（dict_warehouse_trucking_mapping、dict_trucking_port_mapping）

- `country` 字段存 **国家代码**，表示「该国分公司」的映射。
- 与 `dict_warehouses.country`、`biz_customers.country` 一致。

### 3.4 清关行/车队/仓库缺省

- 当字典无该国记录时，采用「XX清关行」「XX车队」「XX仓库」占位符（XX = dict_countries.name_cn）。
- 详见 [13-销往国家到排柜资源关联逻辑.md](./13-销往国家到排柜资源关联逻辑.md) 第 3.1 节。

### 3.5 Excel 导入

- 「销往国家」列映射到 `sell_to_country`。
- Excel 中可能填子公司名称（如 AOSOM LLC）或国家代码；若填国家代码，需在导入时或展示层做兼容（或通过字典映射为子公司名称再存）。

## 四、小结

| 概念 | 含义 |
|------|------|
| country 字段 | 存 dict_countries.code，表示该国分公司 |
| 该国分公司 | 子公司类型客户，customer_type_code = 'SUBSIDIARY' |
| sell_to_country | 特例：存子公司名称，用于与 customer 关联 |
| 国家筛选/匹配 | 统一用 dict_countries.code |
| 清关行缺省 | dict_customs_brokers 无记录时用「XX清关行」（XX=国家中文名） |

新增表或字段时，凡表示「该国分公司」的，一律用 `country` 字段存 `dict_countries.code`。

## 五、检查与迁移

- **检查脚本**：`cd backend && npm run check:country-tables`，列出含 country 的表、智能排柜必需表、country 值规范性。
- **智能排柜必需表**：`dict_warehouses`、`dict_trucking_port_mapping`、`dict_warehouse_trucking_mapping`。
- **迁移脚本**：`migrations/normalize_country_uk_to_gb.sql`，将子公司名称→国家代码、UK→GB、BEL→BE，并从仓库补全映射表 country。
