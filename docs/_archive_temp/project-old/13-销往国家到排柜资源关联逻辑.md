# 销往国家到排柜资源关联逻辑

> 根据货柜的销往国家（sell_to_country）查找并关联排柜所需的清关行、车队、仓库信息。

---

## 一、数据链总览

```
biz_replenishment_orders.sell_to_country (子公司名称)
    │
    ├─► biz_customers.customer_name = sell_to_country
    │       └─► cust.country = 国家代码 (dict_countries.code)
    │
    └─► 或 order.customer_code → biz_customers.customer_code
            └─► cust.country = 国家代码

国家代码 (countryCode) 作为统一过滤键
    │
    ├─► 清关行: dict_customs_brokers WHERE country = countryCode
    │
    ├─► 仓库: dict_warehouses WHERE country = countryCode
    │
    └─► 车队: dict_trucking_port_mapping / dict_warehouse_trucking_mapping
            WHERE country = countryCode
```

---

## 二、第一步：销往国家 → 国家代码

### 2.1 sell_to_country 含义

- **存值**：子公司名称（如 `AOSOM LLC`、`MH STAR UK LTD`），非国家代码
- **来源**：备货单 `biz_replenishment_orders.sell_to_country`
- **约定**：见 [12-国家概念统一约定.md](./12-国家概念统一约定.md)

### 2.2 解析为国家代码

**方式一：通过 customer_code（推荐）**

```sql
SELECT cust.country
FROM biz_replenishment_orders o
JOIN biz_customers cust ON cust.customer_code = o.customer_code
WHERE o.container_number = :containerNumber
```

**方式二：通过 customer_name（sell_to_country）**

```sql
SELECT cust.country
FROM biz_replenishment_orders o
JOIN biz_customers cust ON cust.customer_name = o.sell_to_country
WHERE o.container_number = :containerNumber
```

**方式三：已是国家代码**

若 `sell_to_country` 已是 2–3 位大写（如 `US`、`GB`），可直接使用；`UK` 需规范为 `GB`。

---

## 三、第二步：国家代码 → 清关行

### 3.1 清关行/车队/仓库缺省约定（项目统一）

**当字典中无该国记录时，采用「XX清关行」「XX车队」「XX仓库」作为替代**，用于智能排柜、展示等场景。

| 资源 | 有记录 | 无记录（占位符） | 示例 |
|------|--------|------------------|------|
| 清关行 | broker_code / broker_name | `{国家中文名}清关行` | 美国 → `美国清关行` |
| 车队 | trucking_company_id / trucking_company_name | `{国家中文名}车队` | 美国 → `美国车队` |
| 仓库 | warehouse_code / warehouse_name | `{国家中文名}仓库` | 美国 → `美国仓库` |

- **XX**：来自 `dict_countries.name_cn`（国家中文名）
- **用途**：智能排柜时若未匹配到对应资源，写入或展示用该占位符，避免空值
- **全项目生效**：列表、甘特图、详情、排柜引擎等统一采用此约定

### 3.2 表

| 表 | 字段 | 说明 |
|----|------|------|
| dict_customs_brokers | country | 清关公司所在国，存国家代码 |
| dict_customs_brokers | broker_code | 清关公司代码 |

### 3.3 查询

```sql
SELECT broker_code, broker_name, broker_name_en, contact_phone, contact_email
FROM dict_customs_brokers
WHERE country = :countryCode AND status = 'ACTIVE'
ORDER BY broker_name;
```

**无记录时**：`customs_broker_code` / 展示值 = `(SELECT name_cn FROM dict_countries WHERE code = :countryCode) || '清关行'`，如 `美国清关行`。

### 3.4 写入

清关公司写入 `process_port_operations.customs_broker_code`（目的港操作记录）。无字典记录时写入占位符（如 `美国清关行`）。

---

## 四、第三步：国家代码 + 目的港 → 仓库

### 4.1 仓库缺省

无该国仓库记录时，展示/写入占位符「XX仓库」（XX = dict_countries.name_cn），见 3.1。

### 4.2 表

| 表 | 字段 | 说明 |
|----|------|------|
| dict_warehouses | country | 仓库所在国 |
| dict_trucking_port_mapping | country, port_code | 该国分公司下，港口与车队的映射 |
| dict_warehouse_trucking_mapping | country, warehouse_code, trucking_company_id | 该国分公司下，仓库与车队的映射 |

### 4.3 查询逻辑

**方式一：仅按国家**

```sql
SELECT warehouse_code, warehouse_name, warehouse_name_en, country, city, address
FROM dict_warehouses
WHERE country = :countryCode AND status = 'ACTIVE';
```

**方式二：通过港口→车队→仓库推导（推荐）**

1. 目的港：`process_port_operations.port_code`（port_type = 'destination'）
2. 该国能服务该港口的车队
3. 该车队能服务的该国仓库

```sql
-- 1. 该国能服务该港口的车队
SELECT trucking_company_id
FROM dict_trucking_port_mapping
WHERE country = :countryCode AND port_code = :portCode AND is_active = true;

-- 2. 这些车队能服务的该国仓库
SELECT DISTINCT warehouse_code
FROM dict_warehouse_trucking_mapping
WHERE country = :countryCode
  AND trucking_company_id IN (:truckingCompanyIds)
  AND is_active = true;

-- 3. 取该国仓库且 warehouse_code 在推导结果中
SELECT * FROM dict_warehouses
WHERE country = :countryCode
  AND warehouse_code IN (:warehouseCodes)
  AND status = 'ACTIVE';
```

---

## 五、第四步：国家代码 + 仓库 + 港口 → 车队

### 5.1 车队缺省

无该国车队记录时，展示/写入占位符「XX车队」（XX = dict_countries.name_cn），见 3.1。

### 5.2 表

| 表 | 字段 | 说明 |
|----|------|------|
| dict_warehouse_trucking_mapping | country, warehouse_code, trucking_company_id | 该国分公司下，仓库与车队的映射 |
| dict_trucking_port_mapping | country, port_code, trucking_company_id | 该国分公司下，港口与车队的映射 |
| dict_trucking_companies | company_code | 车队主表 |

### 5.3 查询逻辑

**方式一：按仓库→车队**

```sql
SELECT trucking_company_id
FROM dict_warehouse_trucking_mapping
WHERE country = :countryCode
  AND warehouse_code = :warehouseCode
  AND is_active = true;
```

**方式二：按港口→车队（与仓库映射取交集）**

```sql
-- 能服务该港口的该国车队
SELECT trucking_company_id
FROM dict_trucking_port_mapping
WHERE country = :countryCode AND port_code = :portCode AND is_active = true;

-- 能服务该仓库的该国车队
SELECT trucking_company_id
FROM dict_warehouse_trucking_mapping
WHERE country = :countryCode AND warehouse_code = :warehouseCode AND is_active = true;

-- 取交集：既能服务港口又能服务仓库的车队
```

## 六、完整流程（智能排柜用）

```text
输入：货柜 container_number
  │
  ├─ 1. 备货单 → sell_to_country / customer_code
  ├─ 2. 目的港操作 → port_code (目的港)
  └─ 3. resolveCountryCode → countryCode
  │
  ├─ 清关行：dict_customs_brokers WHERE country = countryCode
  │
  ├─ 候选仓库：
  │     dict_trucking_port_mapping(country, port_code) → truckingCompanyIds
  │     dict_warehouse_trucking_mapping(country, truckingCompanyId IN) → warehouseCodes
  │     dict_warehouses(country, warehouse_code IN) → 候选仓库列表
  │
  └─ 车队：
        dict_warehouse_trucking_mapping(country, warehouse_code) → truckingCompanyIds
        取有剩余能力的车队
```

## 七、API 与代码引用

| 资源 | API | 过滤参数 |
|------|-----|----------|
| 清关行 | GET /api/v1/dict/customs-brokers | ?country=US |
| 仓库 | GET /api/v1/dict/warehouses | ?country=US |
| 车队 | GET /api/v1/dict/trucking-companies | ?country=US |

**智能排柜引擎**：`backend/src/services/intelligentScheduling.service.ts`

- `resolveCountryCode()`：sell_to_country → countryCode
- `getCandidateWarehouses(countryCode, portCode)`：仓库
- `selectTruckingCompany(warehouseCode, portCode, plannedPickupDate, countryCode)`：车队

**清关行**：智能排柜时优先从 `dict_customs_brokers` 按 country 选取；若无记录则采用「XX清关行」占位符（见 3.1）。
