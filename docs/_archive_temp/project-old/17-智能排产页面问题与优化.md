## 六、API 与数据流

### 5.1 概览

```
GET /api/v1/scheduling/overview?startDate=&endDate=
  → 待排产数量（initial + issued，可选日期范围）
  → 仓库列表（code, name, country, dailyCapacity）
  → 车队列表（code, name, country）
```

### 5.2 排产

```
POST /api/v1/scheduling/batch-schedule
Body: { country?, startDate?, endDate?, limit?, skip? }
  → intelligentSchedulingService.batchSchedule
  → 按需写回 last_free_date → 排序 → 逐柜排产
```

### 5.3 排产结果字段

```typescript
interface ScheduleResult {
  containerNumber: string;
  success: boolean;
  message?: string;
  // 货柜基本信息（用于前端展示）
  destinationPort?: string;       // 目的港代码
  destinationPortName?: string;    // 目的港名称
  warehouseName?: string;           // 仓库名称
  etaDestPort?: string;            // 预计到港日
  ataDestPort?: string;            // 实际到港日
  plannedData?: {
    plannedPickupDate?: string;    // 计划提柜日
    plannedDeliveryDate?: string;  // 计划送仓日
    plannedReturnDate?: string;    // 计划还箱日
    // ...
  };
}
```
