## 一、方案概述

### 1.1 目标

为 LogiX 项目接入 AI 能力，实现：
- 📊 **自然语言查询**：Text-to-SQL，将用户问题转为数据库查询
- 🔍 **数据验证**：自动发现数据质量问题并提供修复建议
- 🤖 **智能调度**：基于历史数据预测最佳排产时间
- 💬 **AI 对话**：智能助手，回答业务问题、生成报表

## 二、多模型接入

### 2.1 支持的模型

| 模型 | 用途 | 场景 |
|------|------|------|
| **硅基流动 (DeepSeek)** | 主力模型 | Text-to-SQL、数据分析 |
| **OpenAI GPT-4** | 复杂推理 | 调度优化、异常分析 |

| **Ollama (本地)** | 隐私场景 | 敏感数据处理 |

### 2.2 模型配置

```typescript
// backend/.env

SILICON_FLOW_API_KEY=sk-xxx

# Ollama (本地)
OLLAMA_BASE_URL=http://localhost:11434

# 默认模型
DEFAULT_MODEL=deepseek-chat
```

## 三、核心功能模块

### 3.1 Text-to-SQL 引擎

```
用户: "显示过去一周到达美国的货柜"
    ↓
AI 分析意图 + 提取实体(美国=US, 时间=过去一周)
    ↓
生成 SQL + 安全检查(仅 SELECT, 无危险操作)
    ↓
执行查询 + 格式化结果
    ↓
返回: 表格/图表/导出
```

**安全策略**：
- 仅允许 SELECT 查询
- 禁止 INSERT/UPDATE/DELETE/DROP/TRUNCATE
- 超时限制（30秒）
- 结果集限制（1000行）

### 3.2 RAG 知识库

- 自动同步文档到向量库
- 业务规则、API文档、操作手册
- 自然语言问答

### 3.3 数据质量分析

- 📊 **数据完整性**：缺失值、异常值检测
- 🚨 **异常检测**：滞港费异常、延误预警
- 📈 **趋势分析**：业务趋势预测

### 3.4 智能调度优化

- 基于历史数据预测最佳排产时间
- 资源冲突检测与建议
- 优先级自动排序

## 四、API 设计

### 4.1 AI 对话

```typescript
// POST /api/v1/ai/chat
{
  message: "显示本周到港的货柜统计",
  context: { scope: "US" },  // 可选上下文
  options: {
    execute: true,     // 是否执行
    preview: true      // 预览SQL模式
  }
}
```

### 4.2 Text-to-SQL

```typescript
// POST /api/v1/ai/text-to-sql
{
  query: "哪些货柜的滞港费超过1000美元",
  tables: ["containers", "demurrage_records"]  // 可选限制范围
}

// Response
{
  success: true,
  sql: "SELECT c.container_number, d.amount ...",
  data: [...],
  rowCount: 15
}
```

### 4.3 数据分析

```typescript
// POST /api/v1/ai/analyze
{
  type: "quality|anomaly|trend",
  table: "biz_containers",
  params: { ... }
}
```

### 4.4 调度优化建议

```typescript
// POST /api/v1/ai/scheduling-suggest
{
  containerNumbers: ["CBHU1234567"],
  targetDate: "2024-03-20"
}
```

## 九、注意事项

1. **数据安全**: 敏感数据不出网，使用本地模型处理
2. **成本控制**: 限制 API 调用次数和返回行数
3. **错误处理**: AI 返回可能不可靠，需有兜底方案
4. **用户体验**: 长时间运行的查询需要进度提示

## 十、AI 对话页排查（/#/ai-chat）

| 问题 | 排查 | 解决 |
|------|------|------|
| 404 / 请求失败 | 前端 API 路径需含 `/v1` | `ai.ts` 已改为 `/v1/ai/*` |
| 未配置 | 状态显示「未配置」 | 在 `.env` 设置 `SILICON_FLOW_API_KEY` |
| SQL 预览不显示 | 发送查询类问题 | 自动调用 textToSqlPreview，成功则展开预览 |
| 执行 SQL 失败 | 原 textToSqlExecute 传的是 SQL 非自然语言 | 已新增 `executeRawSql` + `POST /ai/execute-sql` |
| 超时 | 默认 10s 可能不足 | chat/textToSqlPreview 已设 timeout: 60000 |
