# AI 智能排柜集成方案

> 在 AI 对话中集成智能排柜能力，让用户通过自然语言触发排产

---

## 一、目标

用户可在 AI 对话页（`/#/ai-chat`）通过自然语言触发智能排柜，例如：

- 「帮我排产」
- 「开始智能排柜」
- 「对美国方向 3 月的货柜执行排产」
- 「排产本周到港的货柜」

---

## 二、现有能力

| 组件 | 路径/API | 说明 |
|------|----------|------|
| 智能排柜 API | `POST /api/v1/scheduling/batch-schedule` | 入参 country、startDate、endDate、forceSchedule、containerNumbers、limit、skip |
| 排产概览 | `GET /api/v1/scheduling/overview?startDate=&endDate=` | 待排产数量、仓库/车队产能 |
| AI 对话 | `POST /api/v1/ai/chat` | 当前支持 Text-to-SQL、知识库问答 |
| 前端服务 | `containerService.batchSchedule()` | 已封装排产调用 |

## 三、集成架构

```
用户: "帮我排产美国方向本周的货柜"
         │
         ▼
┌─────────────────────────────────────────────────────────┐
│  AI Chat 意图识别                                         │
│  • 排产意图: 排产/智能排柜/开始排产/一键排产                │
│  • 参数抽取: 国家(美国/US)、日期范围(本周)                  │
└─────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────┐
│  调用 intelligentSchedulingService.batchSchedule()      │
│  或 POST /api/v1/scheduling/batch-schedule               │
└─────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────┐
│  返回排产结果 → AI 格式化回复                             │
│  成功 X 个 / 失败 Y 个，可跳转甘特图/排产页                │
└─────────────────────────────────────────────────────────┘
```

## 四、实现方案

### 方案 A：后端意图识别 + 工具调用（推荐）

在 `ai.controller.ts` 的 `chat` 流程中增加排产意图分支：

1. **意图检测**：当用户消息匹配排产关键词时，走排产分支
2. **参数解析**：从消息或 context 中解析 country、startDate、endDate
3. **调用排产**：调用 `intelligentSchedulingService.batchSchedule()`
4. **格式化回复**：将排产结果注入 system prompt，让 AI 生成自然语言回复

**优点**：无需改前端，后端统一处理  
**缺点**：参数解析依赖关键词，复杂表达可能不准

### 方案 B：AI 生成结构化指令 + 工具执行

1. 在 system prompt 中说明排产能力与参数格式
2. 要求 AI 在检测到排产意图时返回 JSON：`{ "action": "schedule", "params": { "country": "US", ... } }`
3. 后端解析 AI 返回，若为 action=schedule 则调用排产 API，再二次请求 AI 生成回复

**优点**：AI 可理解更复杂的自然语言  
**缺点**：两次 LLM 调用，延迟与成本更高

### 方案 C：独立 AI 排产接口

新增 `POST /api/v1/ai/schedule`：

- 入参：`{ message: "帮我排产美国方向" }`
- 后端：意图识别 + 参数抽取（规则或 LLM）→ 调用 batchSchedule → 返回结果

**优点**：职责清晰，可单独测试  
**缺点**：与 chat 分离，用户需知道「排产」和「查询」是不同入口

## 五、推荐实现（方案 A 细化）

### 5.1 后端修改

**文件**：`backend/src/ai/controllers/ai.controller.ts`

在 `chat` 方法中，在 `needsDataQuery` 判断之前增加排产意图判断：

```typescript
// 排产意图检测
const schedulePatterns = ['排产', '智能排柜', '一键排产', '开始排产', '执行排产', '帮我排柜'];
const isScheduleIntent = schedulePatterns.some(p => message.includes(p));

if (isScheduleIntent) {
  // 从 context 或默认值获取参数
  const country = context?.country ?? context?.scopedCountryCode;
  const startDate = context?.startDate ?? getDefaultStartDate(); // 如本年首日
  const endDate = context?.endDate ?? getDefaultEndDate();       // 如今天

  const scheduleResult = await intelligentSchedulingService.batchSchedule({
    country, startDate, endDate, forceSchedule: false
  });

  // 将结果注入 prompt，让 AI 生成回复
  // 或直接返回格式化结果，不再调用 LLM
}
```

**参数解析增强**（可选）：用简单正则从消息中抽取「美国」「3月」「本周」等，映射为 country、startDate、endDate。

### 5.2 前端修改

**文件**：`frontend/src/views/ai/Chat.vue`

- 在发送消息时，将当前页面的 **日期范围**、**国家筛选** 作为 `context` 传给 chat API
- 若 AI 返回中包含 `scheduleResult`，可展示「查看排产结果」「跳转甘特图」按钮

**文件**：`frontend/src/services/ai.ts`

- `chat` 方法支持传入 `context: { startDate, endDate, country }`

### 5.3 知识库补充

在 `knowledgeBase` 中增加排产相关说明，便于 AI 回答「如何排产」「排产规则」等问题。

## 六、实施步骤

| 步骤 | 任务 | 预估 |

| 3 | 前端：Chat 页传入日期、国家 context | 0.5 天 |

## 七、安全与确认

- **写操作确认**：排产会更新 `process_trucking_transport`、`process_warehouse_operations` 等，建议在首次触发时弹窗确认
- **权限**：复用现有排产权限，AI 仅作为入口
- **日志**：记录 AI 触发的排产请求（用户、参数、结果）
