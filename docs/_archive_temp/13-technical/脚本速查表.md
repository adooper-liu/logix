## 🎯 **脚本分层架构（重要）**

LogiX项目的脚本采用**三层架构设计**，职责清晰、互不干扰：

```
┌─────────────────────────────────────────┐
│  层级 3: 应用层 (Application Layer)     │
│  ┌───────────────────────────────────┐  │
│  │ start-logix-dev.ps1    ⭐推荐     │  │
│  │ stop-logix-dev.ps1               │  │
│  │ - 一键启动全栈开发环境            │  │
│  │ - 包含：数据库 + 后端 + 前端 + 微服务  │  │
│  │ - 适用：日常开发工作               │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
              ↑ 调用（独立运行）
┌─────────────────────────────────────────┐
│  层级 2: 服务层 (Service Layer)         │
│  ┌───────────────────────────────────┐  │
│  │ prod-start.bat                    │  │
│  │ prod-stop.bat                     │  │
│  │ prod-logs.bat                     │  │
│  │ - 生产环境部署                     │  │
│  │ - 仅 Docker 容器，无应用进程        │  │
│  │ - 适用：生产部署、测试环境          │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
              ↑ 调用（独立运行）
┌─────────────────────────────────────────┐
│  层级 1: 基础设施层 (Infrastructure)    │
│  ┌───────────────────────────────────┐  │
│  │ tsdb-start.bat       💾 仅数据库   │  │
│  │ tsdb-stop.bat                     │  │
│  │ tsdb-db.bat                       │  │
│  │ tsdb-logs.bat                     │  │
│  │ tsdb-info.bat                     │  │
│  │ tsdb-restart.bat                  │  │
│  │ tsdb-clean.bat         ⚠️ 危险     │  │
│  │ - TimescaleDB 时序数据库           │  │
│  │ - 适用：只需要数据库的场景          │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

## 💾 **执行 SQL 脚本的正确方式（Docker 容器）**

**重要**: 数据库运行在 Docker 容器中，**不能直接使用 `psql` 命令**！

### ✅ 正确方式

#### 方式 1: 使用专用脚本（推荐）⭐

```powershell
# 进入交互式 CLI
tsdb-db.bat

# 进入后可以使用 psql 命令
\l                     # 列出数据库
\dt                    # 列出表
\d table_name          # 查看表结构
SELECT * FROM table;   # 查询
\q                     # 退出
```

#### 方式 2: Docker exec 执行 SQL 文件

```powershell
# 标准格式
docker exec -i <容器名> psql -U <用户> -d <数据库> < <SQL 文件路径>

# 示例：初始化数据库
docker exec -i logix-timescaledb-prod psql -U logix_user -d logix_db < backend/scripts/init-database-complete.sql

# 示例：清理测试数据
docker exec -i logix-timescaledb-prod psql -U logix_user -d logix_db < scripts/cleanup-test-data.sql

# 示例：验证表结构
docker exec -i logix-timescaledb-prod psql -U logix_user -d logix_db < scripts/verify-tables.sql
```

#### 方式 3: Docker exec 直接执行 SQL 命令

```powershell
# 单条 SQL 命令
docker exec -i logix-timescaledb-prod psql -U logix_user -d logix_db -c "SELECT COUNT(*) FROM biz_containers;"

# 查看所有表
docker exec -i logix-timescaledb-prod psql -U logix_user -d logix_db -c "\dt"

# 查看表结构
docker exec -i logix-timescaledb-prod psql -U logix_user -d logix_db -c "\d biz_containers"

# 查看 TimescaleDB 信息
docker exec -i logix-timescaledb-prod psql -U logix_user -d logix_db -c "SELECT * FROM timescaledb_information.hypertables;"
```

#### 方式 4: 先复制文件到容器再执行

```powershell
# 适合大型脚本或需要多次执行的脚本

# 1. 复制 SQL 文件到容器
docker cp backend/scripts/init-database-complete.sql logix-timescaledb-prod:/tmp/init.sql

# 2. 在容器内执行
docker exec -i logix-timescaledb-prod psql -U logix_user -d logix_db -f /tmp/init.sql
```

### ❌ 错误方式（无法执行）

```powershell
# 这些命令无法执行，因为数据库在 Docker 容器内！
psql -f backend/scripts/init-database-complete.sql  # ❌ 无法连接到容器内的数据库
psql -U logix_user -d logix_db                      # ❌ 无法连接
psql -h localhost -U logix_user -d logix_db         # ❌ 端口可能未映射到宿主机
```

### 🔧 实用技巧

#### 创建别名简化命令

```powershell
# PowerShell 配置文件 (profile.ps1)
function Invoke-SqlLogix {
    param([string]$file)
    docker exec -i logix-timescaledb-prod psql -U logix_user -d logix_db < $file
}

# 使用
Invoke-SqlLogix backend/scripts/init-database-complete.sql
```

#### 批量执行多个脚本

```powershell
$scripts = @(
    'backend/scripts/script1.sql',
    'backend/scripts/script2.sql'
)

foreach ($script in $scripts) {
    Write-Host "Executing $script..."
    docker exec -i logix-timescaledb-prod psql -U logix_user -d logix_db < $script
}
```
