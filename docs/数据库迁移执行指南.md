# LogiX 数据库迁移执行指南

**版本**：1.0  
**日期**：2026-03-19  
**适用场景**：重建数据库后必须执行的迁移脚本及顺序

---

## 一、执行流程总览

```
01_drop_all_tables
       ↓
03_create_tables
       ↓
02_init_dict_tables_final
       ↓
04_fix_constraints
       ↓
05_init_warehouses
       ↓
【阶段 A】CREATE 表迁移（新增表）
       ↓
【阶段 B】ALTER 迁移（新增列/修改表）
       ↓
【阶段 C】数据迁移（可选）
```

---

## 二、主初始化脚本（必须按序执行）

| 顺序 | 脚本路径 | 用途 | 说明 |
|------|----------|------|------|
| 1 | `backend/01_drop_all_tables.sql` | 删除所有表 | 重建前必须执行 |
| 2 | `backend/03_create_tables.sql` | 创建核心表结构 | 字典表、业务表、流程表、扩展表 |
| 3 | `backend/02_init_dict_tables_final.sql` | 初始化字典数据 | 国家、港口、柜型、船公司等 |
| 4 | `backend/04_fix_constraints.sql` | 外键与约束 | FK、CHECK、索引 |
| 5 | `backend/05_init_warehouses.sql` | 初始化仓库 | TRUNCATE + INSERT 129 个仓库 |

**执行命令示例**（在项目根目录）：

```bash
psql $DATABASE_URL -f backend/01_drop_all_tables.sql
psql $DATABASE_URL -f backend/03_create_tables.sql
psql $DATABASE_URL -f backend/02_init_dict_tables_final.sql
psql $DATABASE_URL -f backend/04_fix_constraints.sql
psql $DATABASE_URL -f backend/05_init_warehouses.sql
```

## 三、阶段 A：CREATE 表迁移（必须执行）

以下迁移**创建新表**，03_create_tables 中不包含。**必须按顺序执行**。

| 顺序 | 脚本路径 | 创建的表 | 依赖 |
|------|----------|----------|------|
| A1 | `migrations/add_feituo_import_tables.sql` | ext_feituo_import_batch, table1, table2 | biz_containers |
| A2 | `migrations/add_demurrage_standards_and_records.sql` | ext_demurrage_standards, ext_demurrage_records | biz_containers |
| A3 | `migrations/add_inspection_records.sql` | ext_inspection_records, ext_inspection_events | biz_containers |
| A4 | `migrations/add_sys_data_change_log.sql` | sys_data_change_log | 无 |
| A5 | `migrations/create_universal_dict_mapping.sql` | dict_universal_mapping | 无 |
| A6 | `migrations/create_resource_occupancy_tables.sql` | ext_warehouse_daily_occupancy, ext_trucking_slot_occupancy, ext_yard_daily_occupancy | dict_warehouses, dict_trucking_companies |
| A7 | `migrations/add_ext_feituo_places.sql` | ext_feituo_places | biz_containers |
| A8 | `migrations/add_ext_feituo_status_events.sql` | ext_feituo_status_events | biz_containers |

**注意**：若 03 已包含 flow_definitions、flow_instances、dict_yards 等，则无需再执行 `create_flow_definitions_table.sql`、`create_flow_instances_table.sql` 等重复建表脚本。

## 四、阶段 B：ALTER 迁移（必须执行）

以下迁移**修改已有表**（ADD COLUMN、ALTER TYPE 等）。**必须在阶段 A 之后执行**，顺序可适当调整。

| 顺序 | 脚本路径 | 操作对象 | 依赖 |
|------|----------|----------|------|
| B1 | `migrations/add_feituo_raw_data_by_group.sql` | ext_feituo_import_table1, table2 | A1 |
| B2 | `migrations/add_destination_port_to_demurrage_records.sql` | ext_demurrage_records | A2 |
| B3 | `migrations/add_demurrage_record_permanence.sql` | ext_demurrage_records | A2 |
| B4 | `migrations/add_demurrage_calculation_mode.sql` | process_port_operations, ext_demurrage_records | A2 |
| B5 | `migrations/add_status_event_terminal_name.sql` | ext_container_status_events | 03 |
| B6 | `migrations/add_schedule_status.sql` | biz_containers | 03 |
| B7 | `migrations/add_country_to_dict_tables.sql` | dict_customs_brokers, dict_trucking_companies | 03 |
| B8 | `migrations/add_daily_capacity_to_trucking_companies.sql` | dict_trucking_companies | 03 |
| B9 | `migrations/add_daily_unload_capacity_to_warehouses.sql` | dict_warehouses | 03 |
| B10 | `migrations/add_country_to_warehouse_trucking_mapping.sql` | dict_warehouse_trucking_mapping | 03 |
| B11 | `migrations/add_trucking_return_and_yard_capacity.sql` | dict_trucking_companies | 03 |
| B12 | `migrations/add_feituo_port_operation_fields.sql` | process_port_operations | 03 |
| B13 | `migrations/add_hold_date_fields.sql` | process_port_operations | 03 |
| B14 | `migrations/add_actual_loading_date.sql` | process_sea_freight | 03 |
| B15 | `migrations/fix_port_field_length.sql` | process_sea_freight 等 | 03 |
| B16 | `migrations/add_container_number_to_replenishment_orders.sql` | biz_replenishment_orders | 03 |
| B17 | `migrations/add_last_free_date_mode.sql` | process_port_operations | 03 |

## 五、阶段 C：数据迁移（可选）

| 脚本路径 | 用途 | 说明 |
|----------|------|------|
| `migrations/add_savannah_port.sql` | 插入 USSAV 萨凡纳港 | 建议先执行 |
| `migrations/add_common_ports.sql` | 插入 22 个常用港口 | ON CONFLICT DO NOTHING |
| `migrations/normalize_country_uk_to_gb.sql` | 国家代码 UK→GB | 若有历史数据 |
| `migrations/backfill_last_free_date.sql` | 回填 last_free_date | 仅对有业务数据时 |
| `migrations/backfill_last_return_date.sql` | 回填还箱日期 | 同上 |
| `migrations/backfill_customer_code_from_sell_to_country.sql` | 回填 customer_code | 同上 |

## 六、日期类型迁移（按需）

若 03 中部分字段为 DATE，需统一为 TIMESTAMP 时：

| 脚本 | 说明 |
|------|------|
| `migrations/unify-datetime-types.sql` | 将 process_port_operations、process_sea_freight 等 DATE 转为 TIMESTAMP |
| `migrations/convert_date_to_timestamp.sql` | 同上，带条件判断，可重复执行 |

**注意**：两脚本功能重叠，**二选一**执行即可。新建库若 03 已用 TIMESTAMP，可跳过。

---

## 七、不建议在重建后执行的脚本

以下脚本为**数据修复/一次性回填**，重建后无历史数据时无需执行：

| 脚本 | 说明 |
|------|------|
| `migrations/batch-update-all-statuses.sql` | 批量更新物流状态 |
| `migrations/fix-at-port-status.sql` | 修复 at_port 状态 |
| `migrations/update-container-statuses.sql` | 更新货柜状态 |
| `migrations/insert_empty_return_data.sql` | 插入还箱测试数据 |
| `migrations/rollback_timestamp_to_date.sql` | 回滚 timestamp→date（与统一 TIMESTAMP 相反） |

---

## 八、01_drop_all_tables 补充说明

若已新增 `ext_feituo_places`、`ext_feituo_status_events`，需在 `01_drop_all_tables.sql` 中增加：

```sql
DROP TABLE IF EXISTS ext_feituo_places CASCADE;
DROP TABLE IF EXISTS ext_feituo_status_events CASCADE;
```

建议放在「删除扩展表」区块，`ext_feituo_import_table1` 之前。

## 九、一键执行脚本示例

```bash
#!/bin/bash
# 重建数据库后完整迁移（项目根目录执行）
set -e
DB_URL="${DATABASE_URL:-postgresql://localhost:5432/logix}"

echo "=== 1. 主初始化 ==="
psql "$DB_URL" -f backend/01_drop_all_tables.sql
psql "$DB_URL" -f backend/03_create_tables.sql
psql "$DB_URL" -f backend/02_init_dict_tables_final.sql
psql "$DB_URL" -f backend/04_fix_constraints.sql
psql "$DB_URL" -f backend/05_init_warehouses.sql

echo "=== 2. 阶段 A: CREATE 表 ==="
psql "$DB_URL" -f migrations/add_feituo_import_tables.sql
psql "$DB_URL" -f migrations/add_demurrage_standards_and_records.sql
psql "$DB_URL" -f migrations/add_inspection_records.sql
psql "$DB_URL" -f migrations/add_sys_data_change_log.sql
psql "$DB_URL" -f migrations/create_universal_dict_mapping.sql
psql "$DB_URL" -f migrations/create_resource_occupancy_tables.sql
psql "$DB_URL" -f migrations/add_ext_feituo_places.sql
psql "$DB_URL" -f migrations/add_ext_feituo_status_events.sql

echo "=== 3. 阶段 B: ALTER 表 ==="
psql "$DB_URL" -f migrations/add_feituo_raw_data_by_group.sql
psql "$DB_URL" -f migrations/add_destination_port_to_demurrage_records.sql
psql "$DB_URL" -f migrations/add_demurrage_record_permanence.sql
psql "$DB_URL" -f migrations/add_demurrage_calculation_mode.sql
psql "$DB_URL" -f migrations/add_status_event_terminal_name.sql
psql "$DB_URL" -f migrations/add_schedule_status.sql
psql "$DB_URL" -f migrations/add_country_to_dict_tables.sql
psql "$DB_URL" -f migrations/add_daily_capacity_to_trucking_companies.sql
psql "$DB_URL" -f migrations/add_daily_unload_capacity_to_warehouses.sql
psql "$DB_URL" -f migrations/add_country_to_warehouse_trucking_mapping.sql
psql "$DB_URL" -f migrations/add_trucking_return_and_yard_capacity.sql
psql "$DB_URL" -f migrations/add_feituo_port_operation_fields.sql
psql "$DB_URL" -f migrations/add_hold_date_fields.sql
psql "$DB_URL" -f migrations/add_actual_loading_date.sql
psql "$DB_URL" -f migrations/fix_port_field_length.sql
psql "$DB_URL" -f migrations/add_container_number_to_replenishment_orders.sql
psql "$DB_URL" -f migrations/add_last_free_date_mode.sql

echo "=== 4. 阶段 C: 数据（可选）==="
psql "$DB_URL" -f migrations/add_savannah_port.sql
psql "$DB_URL" -f migrations/add_common_ports.sql

echo "=== 完成 ==="
```

## 十、迁移清单速查表

| 类别 | 必须 | 可选 |
|------|------|------|
| 主初始化 | 01, 03, 02, 04, 05 | - |
| CREATE 表 | A1–A8 | create_flow_*, create_resource_* 若 03 已含则跳过 |
| ALTER 表 | B1–B15 | 视业务需要 |
| 数据 | - | add_savannah_port, add_common_ports |
| 日期类型 | - | unify-datetime-types（若 03 有 DATE） |
