### 问题 2: 类型通用化过度（中优先级）

**问题描述**:
```typescript
interface Props {
  alternatives?: any[] // 使用 any 失去类型检查
}
```

**影响**:
- ⚠️ 无法在编译时检查 alternative 对象结构
- ⚠️ IDE 智能提示减弱

**建议**:

#### 方案 A: 创建通用接口（推荐）
```typescript
interface ChartAlternative {
  pickupDate: string
  totalCost: number
  strategy?: string
  savings?: number
  [key: string]: any
}

interface Props {
  alternatives?: ChartAlternative[]
  containerNumber?: string
}
```

**优点**:
- ✅ 保留基本类型检查
- ✅ IDE 智能提示
- ✅ 灵活性高

**优先级**: 🟡 中

---

### 问题 3: 容器编号未传递（低优先级）

**问题描述**:
```vue
<CostTrendChart :container-number="''" />
```

**影响**:
- ⚠️ 图表标题缺少柜号信息
- ⚠️ 个性化程度降低

**建议**:
```vue
<!-- 从父组件传递 containerNumber -->
<CostTrendChart 
  :alternatives="report.allAlternatives" 
  :container-number="report.containerNumber || ''"
/>
```

**优先级**: 🔵 低
