## 1. 货柜物流全流程

### 1.1 七层流转架构

LogiX项目采用 7 层流转架构，每层对应一张核心业务表：

**第 1 层：备货单 (biz_replenishment_orders)**
- 主键：order_number
- 字段：备货单号、客户、销往国家、价格条款、货物汇总

**第 2 层：货柜 (biz_containers)**
- 主键：container_number
- 字段：集装箱号、柜型、货物描述、物流状态

**第 3-6 层：过程表（并行）**
- process_sea_freight - 海运信息
- process_port_operations - 港口操作
- process_trucking_transports - 拖卡运输
- process_warehouse_operations - 仓库操作

**第 7 层：还空箱 (process_empty_return)**
- 外键：container_number
- 字段：还箱时间、还箱地点、还箱状态

### 1.2 核心表关系

从数据库脚本提取的表关系定义：


### 1.3 物流状态流转

简化状态（7 个）- 用于桑基图和统计：

```typescript
// backend/src/utils/logisticsStatusMachine.ts
export enum SimplifiedStatus {
  NOT_SHIPPED = 'not_shipped',      // 未出运
  SHIPPED = 'shipped',              // 已出运/已装船
  IN_TRANSIT = 'in_transit',        // 在途
  AT_PORT = 'at_port',              // 已到港（中转港或目的港）
  PICKED_UP = 'picked_up',          // 已提柜
  UNLOADED = 'unloaded',            // 已卸柜
  RETURNED_EMPTY = 'returned_empty'  // 已还箱
}
```

状态流转顺序：
1. not_shipped（未出运）→ shipped（装船/出运）
2. shipped → in_transit（在途）或 at_port（直接到港）
3. in_transit → at_port（到港）
4. at_port → picked_up（提柜）或 unloaded（卸柜）或 returned_empty（直接还箱）
5. picked_up → unloaded（卸柜）或 returned_empty（还箱）
6. unloaded → returned_empty（还箱）

### 1.4 多港经停逻辑

港口类型定义：

```sql
-- process_port_operations.port_type
CREATE TYPE port_type_enum AS ENUM (
  'origin',      -- 起运港
  'transit',     -- 中转港
  'destination'  -- 目的港
);
```

判断规则：
- 起运港：POL (Port of Loading)
- 中转港：via 或 transshipment at
- 目的港：POD (Port of Discharge) / Final Destination

示例：
```
Vessel: ABC123
POL: Shekou, China
via: Singapore (中转港)
POD: Savannah, USA (目的港)
```

## 2. 国际贸易术语

### 2.1 常用贸易术语

| 术语 | 全称 | 中文 | 含义 |
|------|------|------|------|
| **EXW** | Ex Works | 工厂交货 | 买方承担所有运输责任和费用 |
| **FOB** | Free On Board | 装运港船上交货 | 卖方负责装船，买方负责海运和后续 |
| **CIF** | Cost, Insurance and Freight | 成本加保险费加运费 | 卖方负责海运费和保险 |
| **DDP** | Delivered Duty Paid | 完税后交货 | 卖方承担所有费用和风险 |

### 2.2 价格条款应用

在备货单中的使用：
- EXW: 买方自行安排提货和运输
- FOB: 卖方负责送到起运港并装船
- CIF: 卖方负责海运费和保险
- DDP: 卖方负责全程运输和清关

## 3. 港口与船务知识

### 3.1 关键概念

**ETD (Estimated Time of Departure)**: 预计离港时间  
**ETA (Estimated Time of Arrival)**: 预计到港时间  
**ATD (Actual Time of Departure)**: 实际离港时间  
**ATA (Actual Time of Arrival)**: 实际到港时间  

**BL Number (Bill of Lading Number)**: 提单号  
**MBL Number (Master Bill of Lading)**: 主提单号  
**HBL Number (House Bill of Lading)**: 分提单号  

**POL (Port of Loading)**: 装货港  
**POD (Port of Discharge)**: 卸货港  
**Final Destination**: 最终目的地  

---

### 3.2 船期计算

航程时间估算：
- 深圳/香港 → 美国西岸（LA/LB）：14-18 天
- 深圳/香港 → 美国东岸（Savannah）：28-35 天
- 深圳/香港 → 欧洲基本港：30-40 天

影响因素：
- 直航 vs 中转
- 船公司航线安排
- 季节性拥堵（如巴拿马运河）

## 5. 费用体系详解

### 5.1 海运相关费用

**海运费 (Ocean Freight)**:
- 基本海运费
- 燃油附加费 (BAF)
- 货币贬值附加费 (CAF)

**起运港费用**:
- THC (Terminal Handling Charge): 码头操作费
- DOC (Document Fee): 文件费
- Seal Fee: 封条费

**目的港费用**:
- DTHC: 目的港码头操作费
- ISPS: 港口安保费
- Custom Clearance: 报关费

---

### 5.2 滞期费 (Demurrage & Detention)

**Demurrage (滞港费)**:
- 定义：集装箱在码头超过免堆期产生的费用
- 计费起点：Last Free Date 次日
- 费率：USD 50-150/天（因港口而异）

**Detention (滞箱费)**:
- 定义：集装箱超出免费使用期产生的费用
- 计费起点：提柜后超过免费期
- 费率：USD 50-100/天

**免堆期 (Free Time)**:
- 通常：7-14 天
- 可申请延长：VIP 客户或特殊情况
- 计算方式：按自然日或工作日（因港口而异）

---

### 5.3 拖车费用

**提柜费**:
- 从堆场提柜到工厂
- 距离计费：近距离 USD 200-400，远距离 USD 400-800

**送柜费**:
- 从工厂送柜到码头
- 距离计费：同上

**特殊费用**:
- 等待费：超过免费等待时间（通常 2 小时）
- 加班费：非工作时间操作
- 特殊车型费：超重、超长货物

---

### 5.4 仓储费用

**卸柜费**:
- 按体积计费：USD 2-5/CBM
- 按件数计费：USD 0.5-2/件

**仓储费**:
- 免租期：通常 7-30 天
- 超期费：USD 0.5-2/CBM/天

**操作费**:
- 分拣费：USD 0.3-1/CBM
- 贴标费：USD 0.1-0.5/件
- 打包费：USD 5-20/托盘
