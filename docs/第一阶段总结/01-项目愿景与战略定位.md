## 1. 项目命名体系

### 1.1 项目名称

**LogiX —— 乐捷行 智慧物流系统**

### 1.2 命名寓意

```
LogiX = Logistics（物流） + sIX（六大核心要素）+ X（无限可能）
```

- **Logi**: 代表物流（Logistics），项目的核心领域
- **sIX**: 代表六大核心要素（货柜、库位、仓位、订单、包裹、员工操作）
- **X**: 代表无限可能，"+1"全供应链，寓意以物流六要素为基石，开启供应链的无限未来

## 2. 项目发起背景

### 2.1 为什么要做这个系统？

#### 业务痛点（As-Is）

在项目启动前，客户面临以下痛点：

**痛点 1: 数据分散，难以统一管理**
```
❌ Excel 表格满天飞
   - 备货单一个表
   - 货柜信息一个表
   - 物流跟踪一个表
   - 费用记录又一个表
   
❌ 数据孤岛严重
   - 仓库不知道货柜何时到港
   - 车队不知道何时去提柜
   - 财务不知道产生了多少费用
```

**痛点 2: 依赖人工经验，效率低下**
```
❌ 排柜靠人工判断
   - 老师傅凭经验决定卸哪个仓库
   - 新人完全不知道怎么排
   - 不同人排出不同方案
   
❌ 预警靠人工记忆
   - 记不住哪个货柜快逾期了
   - 等到发现已经产生滞港费
   - 每天要花 2 小时核对数据
```

**痛点 3: 外部数据对接困难**
```
❌ 飞驼系统数据手动导入
   - 每周导出一次 Excel
   - 手动复制到内部表格
   - 容易出错且时效性差
   
❌ 状态更新不及时
   - 货柜已经到港了还不知道
   - 等收到通知已经产生费用
   - 无法提前安排后续操作
```

**痛点 4: 成本核算不清晰**
```
❌ 不知道每个方案的实际成本
   - 只看到海运费
   - 没算上滞港费、堆场费
   - 隐性成本被忽略
   
❌ 无法对比不同方案的优劣
   - Direct 真的比 Drop off 便宜吗？
   - 加急是否值得？
   - 缺乏数据支撑决策
```

#### 理想状态（To-Be）

我们希望通过系统实现：

```
✅ 数据统一管理
   - 所有货柜信息在一个平台
   - 一键查询完整链路
   - 实时数据更新

✅ 智能决策支持
   - 系统自动推荐最优方案
   - 提前预警潜在风险
   - 成本透明可对比

✅ 自动化流程
   - 外部数据自动同步
   - 状态变更自动通知
   - 报表自动生成

✅ 降本增效
   - 减少人工判断错误
   - 避免不必要的滞期费
   - 优化资源配置
```

## 2. 核心功能定位

### 2.1 智能排柜系统的核心价值

#### 什么是排柜？

**定义**: 根据货柜的特性、目的地、客户要求等因素，决定采用何种卸柜方案（仓库、车队、时间）的决策过程。

**排柜的重要性**:
```
一个好的排柜方案可以：
✅ 节省 $500-$2000 的成本
✅ 缩短 1-2 天的运输时间
✅ 避免产生滞港费
✅ 提高仓库和车队的利用率
```

#### 智能排柜解决什么问题？

**问题 1: 方案选择困难**
```
场景：
  有一个 40HQ 货柜要到港，有 3 个仓库可选，5 个车队可用
  
传统方式:
  ❌ 凭经验选择"看起来不错"的方案
  ❌ 打电话问各个仓库和车队是否有空位
  ❌ 无法确定是否是最优方案
  
智能排柜:
  ✅ 系统自动评估所有组合（3×5=15 种方案）
  ✅ 计算每种方案的成本和时间
  ✅ 推荐综合评分最高的方案
  ✅ 提供详细对比数据
```

**问题 2: 成本核算复杂**
```
成本构成:
  - 运输费（距离 × 费率 × 柜型系数）
  - 仓储费（日费率 × 存储天数）
  - 人工费（工时 × 工时费率）
  - 加急费（如需要）
  - 外部存储费（如超出免费期）
  
传统方式:
  ❌ 用计算器一项项按
  ❌ 容易算错或遗漏
  ❌ 无法横向对比
  
智能排柜:
  ✅ 自动计算所有费用
  ✅ 明细化展示成本构成
  ✅ 一键对比多个方案
```

**问题 3: 约束条件多**
```
约束条件:
  - 仓库容量限制（不能超负荷）
  - 车队运力限制（每天能拉多少柜）
  - 时间窗口限制（某些仓库周末不收货）
  - 特殊货物要求（如危险品、冷冻品）
  
智能排柜:
  ✅ 自动过滤不满足约束的方案
  ✅ 在可行方案中选择最优
  ✅ 当资源冲突时智能调配
```

#### 核心价值主张

```
🎯 让每一次排柜都成为最优决策

💡 从"人工经验驱动"升级为"数据智能驱动"

📈 实现成本、时效、资源利用率的平衡优化
```

### 2.2 物流可视化监控的定位

#### 为什么要可视化？

**物流的特点**:
```
⏰ 时效性强
   - 货柜每天都在移动
   - 每个节点都有时间要求
   - 错过时间窗口会产生费用

🔗 环节众多
   - 起运港 → 海运 → 目的港 → 提柜 → 运输 → 卸货 → 还箱
   - 每个环节都可能出问题
   - 需要全局视角掌控

📊 数据量大
   - 一个货柜有 50+ 个字段
   - 每天有 10-20 个新货柜
   - 历史数据累积成千上万条
```

**可视化解决的问题**:

```
❌ 问题：Excel 表格无法直观展示时间线
   
✅ 解决方案：甘特图
   - 横轴是时间
   - 纵轴是货柜
   - 一眼看出哪些货柜即将到期
   - 一眼看出资源占用情况
```

#### 可视化监控的核心功能

**1. 甘特图展示**
```
功能:
  - 4 种泳道视图（按到港、按计划提柜、按最晚提柜、按最晚还箱）
  - 颜色区分状态（正常、预警、逾期）
  - 鼠标悬停显示详情
  - 点击跳转到详情页

价值:
  - 一眼看清所有货柜的时间安排
  - 快速识别需要优先处理的货柜
  - 合理安排工作优先级
```

**2. 智能预警**
```
预警规则:
  ⚠️ Warning: 即将超期（未来 3 天内）
  🔴 Danger: 逾期未提（已过最晚提柜日）

呈现方式:
  - 甘特图上边框高亮 + 脉冲动画
  - 统计面板实时计数
  - Tooltip 显示详细信息

价值:
  - 变被动为主动
  - 避免产生额外费用
  - 提升客户满意度
```

**3. 统计面板**
```
关键指标:
  - 总货柜数
  - 已提柜数
  - 未提柜数
  - 即将超期数
  - 逾期未提数
  - 已还箱数

价值:
  - 管理者一眼看清整体状况
  - 发现异常趋势
  - 支持资源调配决策
```

### 2.3 成本优化决策的支持能力

#### 成本优化的三个层次

**层次 1: 单一货柜优化**
```
场景: 单个货柜选择卸柜方案

优化方法:
  - 对比 Direct vs Drop off vs Expedited
  - 考虑仓库费率差异
  - 考虑车队运费差异
  - 考虑时间成本

输出:
  - 推荐最优方案
  - 显示预计节省金额
```

**层次 2: 批量货柜优化**
```
场景: 同一批次有多个货柜要到港

优化方法:
  - 合并运输（拼车）
  - 错峰安排（避免仓库拥堵）
  - 优先级排序（先处理紧急的）

输出:
  - 批量排柜计划
  - 资源分配方案
```

**层次 3: 全局资源优化**
```
场景: 考虑仓库和车队的长期利用率

优化方法:
  - 均衡各仓库 workload
  - 提高车队满载率
  - 避免资源闲置或过载

输出:
  - 资源利用率报告
  - 瓶颈识别
  - 扩容建议
```

#### 成本优化算法的核心要素

```typescript
// 成本计算公式
总成本 = 运输成本 + 仓储成本 + 人工成本 + 加急费 + 外部存储费

// 各项成本计算
运输成本 = 基础费率 × 距离 × 柜型系数
仓储成本 = 日费率 × 存储天数
人工成本 = 工时 × 工时费率
加急费 = expedited_handling_fee (固定费用)
外部存储费 = external_storage_daily_rate × 超出天数

// 推荐算法评分
评分 = 成本权重 (40%) + 时间权重 (30%) + 
      资源利用率权重 (20%) + 客户偏好权重 (10%)
```

### 2.4 数据导入自动化的意义

#### 为什么要自动化导入？

**手工录入的问题**:
```
❌ 效率低
   - 一个货柜 50+ 个字段
   - 每天 10-20 个新货柜
   - 纯手工录入需要 2-3 小时/天

❌ 易出错
   - 看错行、输错数
   - 漏填必填字段
   - 格式不统一

❌ 数据不一致
   - 不同人录入习惯不同
   - 缩写、简称混用
   - 后期对账困难
```

#### 自动化导入的价值

**价值 1: 效率提升**
```
Before: 2-3 小时/天
After:  10 分钟/天（上传文件 + 确认）

提升：10 倍以上
```

**价值 2: 准确性保证**
```
✅ 字段映射自动处理
✅ 数据格式自动转换
✅ 必填项自动验证
✅ 错误数据自动拦截
```

**价值 3: 标准化推动**
```
✅ 统一列名规范（支持别名）
✅ 统一数据格式（日期、数字）
✅ 统一业务规则（外键关联）
✅ 统一质量标准（验证规则）
```

### 4.1 ROI（投资回报率）计算

#### 投入成本

**开发成本**（第一阶段）:
```
人力成本:
  - 产品经理：1 人 × 3 月 × ¥30,000/月 = ¥90,000
  - 架构师：1 人 × 2 月 × ¥40,000/月 = ¥80,000
  - 后端开发：2 人 × 3 月 × ¥25,000/月 = ¥150,000
  - 前端开发：2 人 × 3 月 × ¥25,000/月 = ¥150,000
  - UI 设计：0.5 人 × 2 月 × ¥20,000/月 = ¥20,000
  
总计：¥490,000

基础设施:
  - 服务器（首年）：¥20,000
  - 域名和证书：¥5,000
  - 开发工具：¥10,000
  
总计：¥35,000

第一阶段总投入：¥525,000
```

#### 产出收益

**直接收益**（年度）:
```
节省人力成本:
  - 物流操作员：2 人 × ¥8,000/月 × 12 月 = ¥192,000
  - 数据统计员：1 人 × ¥6,000/月 × 12 月 = ¥72,000
  
节省滞港费:
  - 减少 50% 滞港费 = ¥30,000/年

提升资源利用率:
  - 车队和仓库优化 = ¥50,000/年

年度直接收益：¥344,000
```

**间接收益**（年度，难量化但重要）:
```
客户满意度提升:
  - 客户留存率提升
  - 口碑传播带来新客户
  - 预估价值：¥100,000/年

决策质量提升:
  - 避免因错误决策导致的损失
  - 抓住更多商机
  - 预估价值：¥200,000/年

团队能力提升:
  - 新人培训成本降低
  - 员工工作效率提升
  - 预估价值：¥50,000/年

年度间接收益：¥350,000
```

#### ROI 计算

```
年度总收益 = ¥344,000 + ¥350,000 = ¥694,000
初始投入 = ¥525,000

ROI = (年度总收益 - 初始投入) / 初始投入 × 100%
    = (¥694,000 - ¥525,000) / ¥525,000 × 100%
    = 32.2%

投资回收期 = 初始投入 / 月度收益
           = ¥525,000 / (¥694,000 / 12)
           = 约 9 个月
```

## 5. 项目命名体系

### 5.1 LogiX 的含义

**命名由来**:
```
LogiX = Logo + Logistics + X

Logo: 
  - 品牌标识
  - 易于记忆和传播

Logistics:
  - 物流行业属性
  - 体现核心业务领域

X:
  - 无限可能（X 代表未知数）
  - 智能化（X 代表科技因子）
  - 扩展性（X 代表可扩展）
```

**品牌定位**:
```
🎯 智能物流管理的代名词

💡 让物流管理更简单、更智能

🚀 用科技驱动物流行业升级
```

### 5.2 模块命名逻辑

#### 表名前缀体系

```sql
dict_*       -- Dictionary（字典表）
               - 基础数据、配置项
               - 如：dict_ports（港口字典）

biz_*        -- Business（业务表）
               - 核心业务数据
               - 如：biz_containers（货柜表）

process_*    -- Process（过程表）
               - 业务流程数据
               - 如：process_port_operations（港口操作）

ext_*        -- Extension（扩展表）
               - 外部系统数据
               - 如：ext_feituo_status_events（飞驼状态事件）

flow_*       -- Flow（流程表）
               - 流程定义和实例
               - 如：flow_definitions（流程定义）
```

#### 功能模块命名

```
智能排柜系统 (Intelligent Scheduling System)
  - 核心：排柜决策优化
  - 关键词：成本优化、智能推荐

  - 核心：时间线展示

  - 关键词：规则引擎、分级预警

  - 关键词：Excel、映射、验证
```
