## 1. 项目命名体系

### 1.1 项目名称

**LogiX —— 乐捷行 智慧物流系统**

### 1.2 命名寓意

LogiX = Logistics（物流）+ sIX（六大核心要素）+ X（无限可能）

## 2. 项目发起背景

### 2.1 为什么要做这个系统？

#### 业务痛点（As-Is）

**痛点 1: 数据分散，难以统一管理**

- Excel 表格分散：备货单、货柜信息、物流跟踪、费用记录分别管理
- 数据孤岛严重：仓库不知道货柜到港时间，车队不知道提柜时间，财务不知道费用情况

**痛点 2: 依赖人工经验，效率低下**

- 排柜靠人工判断：老师傅凭经验，新人不会操作，不同人排出不同方案
- 预警靠人工记忆：记不住逾期时间，发现时已产生滞港费，每天花 2 小时核对数据

**痛点 3: 外部数据对接困难**

- 飞驼系统数据手动导入：每周导出 Excel，手动复制，容易出错且时效性差
- 状态更新不及时：货柜到港不知道，收到通知已产生费用，无法提前安排

**痛点 4: 成本核算不清晰**

- 不知道每个方案的实际成本：只看到海运费，没算滞港费、堆场费等隐性成本
- 无法对比不同方案的优劣：Direct 是否真的比 Drop off 便宜，加急是否值得

#### 理想状态（To-Be）

希望通过系统实现：

1. **数据统一管理**
   - 所有货柜信息在一个平台
   - 一键查询完整链路
   - 实时数据更新

2. **智能决策支持**
   - 系统自动推荐最优方案
   - 提前预警潜在风险
   - 成本透明可对比

3. **自动化流程**
   - 外部数据自动同步
   - 状态变更自动通知
   - 报表自动生成

4. **降本增效**
   - 减少人工判断错误
   - 避免不必要的滞期费
   - 优化资源配置

### 2.3 期望达到的商业价值

#### 定量价值（可衡量）

**节省人力成本**:

- Before: 每天花 2 小时核对数据，每周花 4 小时制作报表，每月花 8 小时核算费用
- After: 数据自动汇总，报表一键生成，费用自动计算
- 年度节省：约 780 小时 ≈ 97 个工作日

**降低滞期费用**:

- Before: 平均每月产生滞港费，部分因忘记免堆期导致
- After: 预警系统提前 3 天提醒，智能排柜优化时间安排
- 预计减少：50% 的滞港费，年度节省约$30,000

#### 定性价值（不可量化但重要）

1. **提升客户满意度**
   - 快速响应客户查询
   - 准确告知货物状态
   - 减少人为错误

2. **增强团队协同**
   - 信息透明共享
   - 减少沟通成本
   - 提升工作效率

3. **支持业务扩张**
   - 系统化支撑更大业务量
   - 新人快速上手
   - 降低对老员工依赖

## 3. 核心功能定位

### 3.1 第一阶段已实现功能

1. **全局可视化追踪**
   - 货柜从起运港到目的港全流程追踪
   - 甘特图可视化展示
   - 实时状态更新

2. **滞港费自动计算**
   - 根据 Last Free Date 自动计算
   - 区分 Demurrage 和 Detention
   - 支持多种计算模式

3. **智能排柜系统**
   - 成本优化算法
   - 自动推荐最优卸柜方案
   - 考虑多种约束条件

4. **实时风险预警**
   - 即将超期/逾期未提预警
   - 多级预警（P0-P3）
   - 预警传播机制

5. **数据导入中心**
   - Excel 批量导入
   - 飞驼系统数据同步
   - 数据校验和纠错

### 3.2 各功能模块关系

```
数据导入 → 数据库存储 → 业务逻辑处理 → 前端展示
   ↓           ↓              ↓            ↓
飞驼 Excel  货柜/备货单    滞港费计算    甘特图
外部 API    状态事件       智能排柜      预警标识
                        风险预警        统计面板
```

### 6.3 方法论实践

**"看见 - 理解 - 执行"三步曲**:

1. **看见**（See）
   - 全要素映射（货柜、库位、订单等）
   - 全链路可视（从起运到交付）
   - 实时数据更新

2. **理解**（Understand）
   - 数据分析（趋势、异常）
   - 风险评估（逾期、甩柜）
   - 成本对比（不同方案）

3. **执行**（Act）
   - 智能推荐（最优方案）
   - 自动调度（资源分配）
   - 风险处置（预警响应）
