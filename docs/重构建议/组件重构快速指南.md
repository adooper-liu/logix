# 组件重构快速指南

## 🎯 何时需要重构？

### 警告信号 🚨

- [ ] **文件超过 500 行** - 需要拆分
- [ ] **函数超过 100 行** - 需要提取
- [ ] **一个组件做 3 件以上的事** - 违反单一职责
- [ ] **复制粘贴代码** - 需要提取为公共逻辑
- [ ] **难以理解代码在做什么** - 需要重构命名和结构
- [ ] **测试覆盖率低于 50%** - 难以测试的信号

### Step 2: 制定计划

#### 选项 A: 提取 Composable（适合业务逻辑）

**适用场景**:
- ✅ 多个组件需要复用相同逻辑
- ✅ 逻辑复杂，难以测试
- ✅ 包含 API 调用、数据处理

**示例**:
```typescript
// ❌ 重构前 - 组件内重复逻辑
const handleSchedule = async () => {
  // 100 行代码...
}

// ✅ 重构后 - Composable
// composables/useSchedulingFlow.ts
export function useSchedulingFlow() {
  const handleBatchSchedule = async (params) => {
    // 逻辑实现
  }
  return { handleBatchSchedule }
}
```

---

#### 选项 B: 拆分子组件（适合 UI 模块）

**适用场景**:
- ✅ 模板中有大段独立功能区块
- ✅ 职责不单一
- ✅ 代码量过大

**示例**:
```vue
<!-- ❌ 重构前 - 巨型组件 -->
<template>
  <div>
    <!-- 顶部操作栏 (100 行) -->
    <!-- 统计栏 (50 行) -->
    <!-- 结果表格 (300 行) -->
    <!-- 优化面板 (200 行) -->
  </div>
</template>

<!-- ✅ 重构后 - 拆分为子组件 -->
<template>
  <div>
    <TopActionBar />
    <SchedulingStatsBar />
    <ResultDisplay>
      <ResultTable />
    </ResultDisplay>
    <CostOptimizationPanel />
  </div>
</template>
```

### Step 3: 执行重构

#### 3.1 提取 Composable 的步骤

```bash
# 1. 创建文件
touch composables/useYourLogic.ts

# 2. 复制逻辑代码
# 从原组件中复制相关代码到新文件

# 3. 定义输入输出
export function useYourLogic(options: {
  // 输入参数
  param1: string
  param2: number
}) {
  // 内部状态
  const state = ref()
  
  // 方法
  const method1 = () => {}
  
  // 返回
  return {
    state,
    method1,
  }
}

# 4. 在原组件中使用
import { useYourLogic } from '@/composables/useYourLogic'

const { state, method1 } = useYourLogic({ param1: 'value' })
```

## 🛠️ 常用重构模式

### 模式 1: 提取工具函数

```typescript
// ❌ 重构前
const formatCost = (cost: number) => {
  return `$${cost.toFixed(2)}`
}

// ✅ 重构后
// utils/format.ts
export const formatCurrency = (amount: number, currency = 'USD') => {
  return `${currency}$${amount.toFixed(2)}`
}
```

---

### 模式 2: 提取 Composable

```typescript
// ❌ 重构前 - 组件内重复
const logs = ref([])
const addLog = (message: string, type = 'info') => {
  logs.value.push({ time: new Date(), message, type })
}

// ✅ 重构后
// composables/useExecutionLog.ts
export function useExecutionLog() {
  const logs = ref([])
  
  const addLog = (message: string, type = 'info') => {
    logs.value.push({
      time: new Date().toLocaleTimeString(),
      message,
      type,
    })
  }
  
  const clearLogs = () => {
    logs.value = []
  }
  
  return {
    logs,
    addLog,
    clearLogs,
  }
}
```

## 📐 代码组织规范

### Composable 文件结构

```typescript
// composables/useExample.ts
import { ref, computed } from 'vue'

// 类型定义
interface UseExampleOptions {
  param1: string
  param2: number
}

interface UseExampleReturn {
  state: Ref<string>
  method1: () => void
}

/**
 * 示例 Composable
 * @param options - 配置参数
 */
export function useExample(options: UseExampleOptions): UseExampleReturn {
  // ============================================================================
  // 状态
  // ============================================================================
  const state = ref('')
  
  // ============================================================================
  // 计算属性
  // ============================================================================
  const computedValue = computed(() => {
    return state.value
  })
  
  // ============================================================================
  // 方法
  // ============================================================================
  const method1 = () => {
    // 实现
  }
  
  // ============================================================================
  // 返回
  // ============================================================================
  return {
    state,
    method1,
  }
}
```

### 子组件文件结构

```vue
<!-- components/ExampleComponent.vue -->
<template>
  <div class="example-component">
    <!-- 内容 -->
  </div>
</template>

<script setup lang="ts">
// ============================================================================
// 类型定义
// ============================================================================
interface Props {
  data: any
  loading?: boolean
}

interface Emits {
  (e: 'update', value: any): void
  (e: 'action', type: string): void
}

// ============================================================================
// Props & Emits
// ============================================================================
const props = withDefaults(defineProps<Props>(), {
  loading: false,
})

const emit = defineEmits<Emits>()

// ============================================================================
// 状态
// ============================================================================
const localState = ref('')

// ============================================================================
// 方法
// ============================================================================
const handleAction = () => {
  emit('action', 'type')
}

// ============================================================================
// 导出（可选）
// ============================================================================
defineExpose({
  localState,
})
</script>

<style scoped>
.example-component {
  /* 样式 */
}
</style>
```

## 🎯 重构决策树

```
需要重构吗？
├─ 是，逻辑重复 → 提取 Composable
├─ 是，UI 模块独立 → 拆分子组件
├─ 是，函数太长 → 提取工具函数
└─ 是，组件太大 → 拆分 + 提取

如何提取？
├─ 多个组件需要 → Composable
├─ 只在一个组件用 → 组件内方法
└─ 纯工具函数 → utils 目录

如何拆分？
├─ 职责清晰 → 直接拆分
├─ 有依赖关系 → 先提取逻辑
└─ 不确定 → 保持现状，后续优化
```

## ⚠️ 常见陷阱

### ❌ 过度设计

```typescript
// ❌ 错误：为了一句话逻辑创建 Composable
export function useFormatNumber() {
  return (n: number) => n.toFixed(2)
}

// ✅ 正确：直接内联
const formatted = `${num.toFixed(2)}`
```

---

### ❌ 拆分过细

```vue
<!-- ❌ 错误：10 行代码也拆分 -->
<ParentComponent>
  <ChildComponent1 />
  <ChildComponent2 />
  <ChildComponent3 />
  <!-- ... 20 个子组件 -->
</ParentComponent>

<!-- ✅ 正确：适度拆分 -->
<ParentComponent>
  <Section1 />
  <Section2 />
  <Section3 />
</ParentComponent>
```

---

### ❌ 忽略测试

```typescript
// ❌ 错误：重构后不测试
refactor()
commit('重构完成')

// ✅ 正确：完整的测试
refactor()
runTests() // 单元测试
runE2ETests() // E2E 测试
manualTest() // 手动测试
commit('重构完成，测试通过')
```
