# 优化报告 - 费用明细显示优化

## 🎯 优化需求

根据用户反馈，需要优化费用明细列的显示效果：

1. ✅ **移除"总费用"文字** - 直接显示金额
2. ✅ **紧凑布局** - 减少明细费项与金额的间距

---

## 🔧 优化方案

### 1️⃣ 移除"总费用"标签

**优化前：**
```vue
<span class="cost-tree-label">{{ node.label }}</span>
<!-- 显示：总费用 -->
```

**优化后：**
```vue
<span class="cost-tree-label" v-if="data.level === 1">{{ node.label }}</span>
<!-- 只显示明细费项，根节点不显示标签 -->
```

**效果对比：**

    ├─ 港口存储费    $2200.00

### 2️⃣ 紧凑布局优化

#### CSS 样式调整

**优化前：**
```css

  padding-right: 8px;
}

.cost-tree-label {
  /* 无 flex 设置 */

.cost-tree-value {

**优化后：**

  padding: 2px 0;  /* ✅ 减少垂直间距 */
}

/* ✅ 根节点不缩进 */
.cost-tree :deep(.el-tree-node.is-leaf .el-tree-node__content) {
  padding-left: 0;

```

## 🎨 设计原则

### 1. 信息密度优化
- **移除冗余文字**："总费用"三个字不必要
- **突出核心信息**：金额是最重要的信息
- **视觉层次清晰**：根节点金额更突出

### 2. 空间利用率
- **减少间距**：从 4px 减少到 2px
- **固定宽度**：金额列固定 100px，对齐一致
- **弹性布局**：标签自动填充剩余空间

### 3. 可读性提升
- **右对齐**：金额右对齐符合财务习惯
- **层次分明**：根节点和子节点视觉区分
- **紧凑不拥挤**：减少间距但保持可读性

## 💡 技术实现细节

### 模板优化

```vue
<template #default="{ node, data }">
  <span class="cost-tree-node">
    <!-- ✅ 只显示明细费项的标签（level === 1） -->
    <span class="cost-tree-label" v-if="data.level === 1">
      {{ node.label }}
    </span>
    
    <!-- ✅ 所有节点都显示金额 -->
    <span 
      class="cost-tree-value" 
      :class="data.level === 0 ? 'cost-value-total' : getAmountClass(data.value)"
    >
      {{ formatCurrency(data.value, row.plannedData?.warehouseCountry || 'US') }}
    </span>
  </span>
</template>
```

### 树形结构数据

```typescript
const buildCostTree = (costs: any, _country: string) => {
  const tree: any[] = []
  const children: any[] = []
  
  // 子节点（明细费项）- level: 1
  if (costs.demurrageCost) {
    children.push({
      label: '滞港费',
      value: costs.demurrageCost,
      level: 1,  // ✅ 明细费项
    })
  }
  
  // ... 其他明细费用
  
  // 根节点（总费用）- level: 0
  tree.push({
    label: '总费用',  // 这个 label 不会显示
    value: costs.totalCost,
    level: 0,  // ✅ 根节点
    children: children,
  })
  
  return tree
}
```
