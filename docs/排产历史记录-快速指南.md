## 🔍 SQL 查询示例

### 1. 查看货柜所有历史

```sql
SELECT 
  scheduling_version AS "版本",
  strategy AS "策略",
  planned_pickup_date AS "提柜日",
  warehouse_code AS "仓库",
  total_cost AS "总费用",
  operated_at AS "操作时间",
  scheduling_status AS "状态"
FROM hist_scheduling_records
WHERE container_number = 'MSKU1234567'
ORDER BY scheduling_version DESC;
```

### 2. 查看最新有效排产

```sql
SELECT * FROM v_latest_scheduling
WHERE container_number IN ('MSKU1234567', 'MSKU7654321');
```

### 3. 统计某仓库的排产情况

```sql
SELECT 
  warehouse_code,
  COUNT(*) AS "货柜数",
  AVG(total_cost) AS "平均成本",
  MAX(operated_at) AS "最后排产时间"
FROM hist_scheduling_records
WHERE warehouse_code = 'WH001'
  AND scheduling_status = 'CONFIRMED'
GROUP BY warehouse_code;
```

### 4. 查看版本变更历史

```sql
SELECT 
  scheduling_version,
  strategy,
  total_cost,
  operated_at,
  operated_by
FROM hist_scheduling_records
WHERE container_number = 'MSKU1234567'
ORDER BY operated_at;
```

## 💡 使用场景

### 场景 1：追溯排产决策

**问题**：为什么这个货柜选择了 Drop off 策略？

**操作**：
1. 查询历史记录：`GET /api/v1/scheduling/history/MSKU1234567`
2. 查看备选方案：检查 `alternative_solutions` 字段
3. 对比成本差异：比较不同版本的 `total_cost`

### 场景 2：分析成本优化效果

**问题**：重新排产后节省了多少成本？

**SQL**：
```sql
SELECT 
  scheduling_version,
  strategy,
  total_cost,
  LAG(total_cost) OVER (ORDER BY scheduling_version) AS prev_cost,
  total_cost - LAG(total_cost) OVER (ORDER BY scheduling_version) AS savings
FROM hist_scheduling_records
WHERE container_number = 'MSKU1234567';
```

### 场景 3：审计操作记录

**问题**：谁在什么时候修改了排产方案？

**SQL**：
```sql
SELECT 
  container_number,
  operated_by,
  operated_at,
  operation_type,
  strategy,
  remarks
FROM hist_scheduling_records
WHERE operated_by = 'zhangsan'
  AND operated_at >= '2026-03-01'
ORDER BY operated_at DESC;
```

## 🎨 前端集成示例

### Vue 3 组件

```vue
<template>
  <el-button @click="showHistory = true">查看历史</el-button>
  
  <el-dialog v-model="showHistory" title="排产历史" width="900px">
    <el-table :data="historyRecords" stripe>
      <el-table-column prop="schedulingVersion" label="版本" width="80" />
      <el-table-column prop="strategy" label="策略" width="100">
        <template #default="{ row }">
          <el-tag :type="getStrategyTagType(row.strategy)">
            {{ row.strategy }}
          </el-tag>
        </template>
      </el-table-column>
      <el-table-column prop="plannedPickupDate" label="提柜日" />
      <el-table-column prop="plannedUnloadDate" label="卸柜日" />
      <el-table-column prop="totalCost" label="总费用" align="right">
        <template #default="{ row }">
          ${{ Number(row.totalCost || 0).toFixed(2) }}
        </template>
      </el-table-column>
      <el-table-column prop="operatedAt" label="操作时间" />
      <el-table-column prop="operatedBy" label="操作人" />
      <el-table-column prop="schedulingStatus" label="状态" width="100">
        <template #default="{ row }">
          <el-tag :type="getStatusTagType(row.schedulingStatus)">
            {{ getStatusText(row.schedulingStatus) }}
          </el-tag>
        </template>
      </el-table-column>
    </el-table>
    
    <el-pagination
      v-model:current-page="page"
      v-model:page-size="limit"
      :total="total"
      layout="total, prev, pager, next"
      @current-change="loadHistory"
    />
  </el-dialog>
</template>

<script setup lang="ts">
import { ref } from 'vue';
import { getSchedulingHistory } from '@/api/scheduling';

const props = defineProps<{ containerNumber: string }>();
const showHistory = ref(false);
const historyRecords = ref([]);
const page = ref(1);
const limit = ref(10);
const total = ref(0);

const getStrategyTagType = (strategy: string) => {
  const map: Record<string, string> = {
    'Direct': 'success',
    'Drop off': 'warning',
    'Expedited': 'danger'
  };
  return map[strategy] || 'info';
};

const getStatusText = (status: string) => {
  const map: Record<string, string> = {
    'CONFIRMED': '已确认',
    'CANCELLED': '已取消',
    'SUPERSEDED': '已替代'
  };
  return map[status] || status;
};

const loadHistory = async () => {
  const result = await getSchedulingHistory(
    props.containerNumber,
    page.value,
    limit.value
  );
  
  historyRecords.value = result.data.records;
  total.value = result.data.total;
};
</script>
```

## ⚠️ 常见问题

### Q1: 为什么查询返回空数组？

**A**: 可能原因：
1. 数据库表未创建 → 执行迁移脚本
2. 该货柜没有排产记录 → 先进行排产操作
3. 货柜号错误 → 检查 containerNumber 参数

### Q2: 版本号如何递增？

**A**: 每次确认保存排产时，数据库触发器自动递增版本号。例如：
- 第 1 次排产：version = 1
- 第 2 次排产：version = 2（同时 version 1 标记为 SUPERSEDED）

### Q3: 如何查看备选方案？

**A**: 查询 `alternative_solutions` 字段（JSONB 类型）：

```sql
SELECT 
  container_number,
  scheduling_version,
  alternative_solutions
FROM hist_scheduling_records
WHERE container_number = 'MSKU1234567';
```

### Q4: 历史记录会影响性能吗？

**A**: 
- 保存操作：在事务中执行，影响极小（<10ms）
- 查询操作：有索引优化，通常 <50ms
- 大数据量时建议定期归档（如超过 1 年的数据）

## 📊 数据字典

### 排产模式 (scheduling_mode)

| 值 | 说明 | 使用场景 |
|----|------|----------|
| MANUAL | 手动排产 | 用户手动指定日期和资源 |
| AUTO | 自动排产 | 系统智能推荐最优方案 |
| BATCH | 批量排产 | 批量优化多个货柜 |

### 排产策略 (strategy)

| 值 | 说明 | 特点 |
|----|------|------|
| Direct | 直送 | 提柜 = 卸柜，当天完成 |
| Drop off | 卸后还 | 提柜 < 卸柜，货柜在堆场暂存 |
| Expedited | 加急 | 免费期内快速处理 |

### 操作类型 (operation_type)

| 值 | 说明 |
|----|------|
| CREATE | 新建排产 |
| UPDATE | 更新排产 |
| CANCEL | 取消排产 |

### 排产状态 (scheduling_status)

| 值 | 说明 |
|----|------|
| CONFIRMED | 已确认（当前有效） |
| CANCELLED | 已取消 |
| SUPERSEDED | 已被新版本替代 |
