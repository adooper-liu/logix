# 📱 前端查看排产历史记录 - 完整使用指南

## 🎯 功能概览

排产历史记录功能已完整集成到前端，支持两种查看方式：

| 方式 | 组件 | 适用场景 |
|------|------|----------|
| **内联展示** | `SchedulingHistoryCard.vue` | 在 Shipments 列表中快速查看某货柜的历史 |

### **4. API 调用方式**

如果你想手动调用 API：

```typescript
import api from '@/services/api';

// 查询单柜历史（分页）
async function loadContainerHistory(containerNumber: string) {
  try {
    const response = await api.get(`/scheduling/history/${containerNumber}`, {
      params: {
        page: 1,
        limit: 50
      }
    });
    
    console.log('历史记录:', response.data.data.records);
    // 输出示例：
    // [
    //   {
    //     id: 2,
    //     containerNumber: 'MSKU001',
    //     schedulingVersion: 2,
    //     strategy: 'Drop off',
    //     totalCost: 520.00,
    //     schedulingStatus: 'CONFIRMED',
    //     operatedAt: '2026-03-27T10:30:00Z'
    //   },
    //   {
    //     id: 1,
    //     containerNumber: 'MSKU001',
    //     schedulingVersion: 1,
    //     strategy: 'Direct',
    //     totalCost: 485.50,
    //     schedulingStatus: 'SUPERSEDED',
    //     operatedAt: '2026-03-27T10:00:00Z'
    //   }
    // ]
  } catch (error) {
    console.error('加载失败:', error);
  }
}

// 批量查询最新记录
async function loadLatestHistories(containerNumbers: string[]) {
  try {
    const response = await api.get('/scheduling/history/latest', {
      params: {
        containerNumbers: containerNumbers
      }
    });
    
    console.log('最新记录:', response.data.data);
  } catch (error) {
    console.error('加载失败:', error);
  }
}

// 使用示例
loadContainerHistory('MSKU001');
loadLatestHistories(['MSKU001', 'MSKU002', 'MSKU003']);
```

### **核心字段**

|--------|------|------|------|

## 🐛 常见问题

### **Q1: 组件不显示？**

**检查点：**
1. 确认容器号正确传递
2. 检查 API 路径是否正确
3. 查看浏览器控制台是否有错误

**调试代码：**
```typescript
watch(() => props.containerNumber, (newVal) => {
  console.log('容器号变化:', newVal);
});
```

---

### **Q2: 历史记录为空？**

**可能原因：**
1. 该货柜还未进行过排产
2. 数据库表未创建
3. 后端服务未启动

**验证步骤：**
```bash
# 1. 检查数据库表
docker exec -it logix-postgres psql -U postgres -d logix -c "\dt hist_scheduling_records"

# 2. 测试 API
curl http://localhost:8080/api/v1/scheduling/history/MSKU001

# 3. 查看后端日志
docker logs logix-backend --tail 50 | grep "history"
```

---

### **Q3: 费用显示为 "-"？**

**原因：** 该版本的费用数据可能未保存

**解决方案：** 检查后端 `saveSchedulingHistory` 方法是否正确提取了 `costBreakdown` 数据

## 🚀 性能优化建议

### **1. 分页加载**

对于历史记录较多的情况，建议启用分页：

```vue
<SchedulingHistoryCard 
  :container-number="record.containerNumber"
  :page-size="10"
/>
```

### **2. 缓存数据**

使用 Vue 的 `keep-alive` 缓存组件：

```vue
<keep-alive>
  <SchedulingHistoryCard 
    v-if="showHistory"
    :container-number="record.containerNumber"
  />
</keep-alive>
```

### **3. 防抖搜索**

在独立查询页面中实现防抖：

```typescript
import { debounce } from 'lodash-es';

const debouncedSearch = debounce(() => {
  handleSearch();
}, 500);

watch(() => searchForm.containerNumber, debouncedSearch);
```
