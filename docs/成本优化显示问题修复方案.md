### 🔴 问题 2: 优化建议违反逻辑

**现象**: 
- 已超期货柜，建议从 3-28 调整到 3-30（往后推 2 天）
- 违反了"超期的必须尽早处理"原则

**根本原因**: 

在 `suggestOptimalUnloadDate` 方法中：

```typescript
// Line 857-861: 生成智能搜索范围
const searchDates = this.generateSearchRange(
  basePickupDate,      // 3-28
  effectiveLastFreeDate, // 假设 3-25（已超期）
  strategy             // OVERDUE 策略
);

// Line 1242-1250: generateSearchRange 实现
if (strategy.searchDirection === 'forward') {
  // ✅ 已超期：从今天开始往后找（尽早处理）
  for (let offset = strategy.searchStartOffset; offset <= strategy.searchEndOffset; offset++) {
    const date = dateTimeUtils.addDays(today, offset);
    // ❌ 问题：这里过滤了 date >= basePickupDate
    if (date >= basePickupDate) {  // 3-28
      dates.push(date);
    }
  }
}
```

**问题分析**:

1. **搜索方向正确**: `forward` = 从今天开始往后找
2. **过滤条件错误**: `date >= basePickupDate` 导致：
   - 如果今天是 3-26
   - `basePickupDate` 是 3-28
   - 搜索范围：3-26, 3-27, 3-28, 3-29, 3-30...
   - 但过滤后只剩：3-28, 3-29, 3-30...
   - **错失了 3-26, 3-27 这两个更早的日期！**

3. **逻辑矛盾**: 
   - 已超期应该"尽早处理"
   - 但过滤条件保证"不早于原计划"
   - 导致无法找到比原计划更早的日期

**修复方案**:

```typescript
// 方案 A: 移除过滤条件（推荐）
if (strategy.searchDirection === 'forward') {
  for (let offset = strategy.searchStartOffset; offset <= strategy.searchEndOffset; offset++) {
    const date = dateTimeUtils.addDays(today, offset);
    // ✅ 移除过滤：允许找到比原计划更早的日期
    dates.push(date);
  }
}

// 方案 B: 反向过滤（仅用于免费期内）
if (strategy.searchDirection === 'forward') {
  for (let offset = strategy.searchStartOffset; offset <= strategy.searchEndOffset; offset++) {
    const date = dateTimeUtils.addDays(today, offset);
    // ✅ 只在免费期内才过滤：不能早于原计划
    if (category.category === 'within_free_period' && date < basePickupDate) {
      continue; // 跳过
    }
    dates.push(date);
  }
}
```

### 🔴 问题 3: 费用计算错误

**现象**: 
- 3-28 日 $2,900
- 3-30 日也显示 $2,900（错误！）
- 实际应该是：3-30 日 > 3-28 日（因为又过了 2 天）

**根本原因**: 

在 `evaluateTotalCost` 方法中调用 `calculateForContainer` 时：

```typescript
// Line 1380-1390: evaluateTotalCost 实现
const demurrageResult = await this.demurrageService.calculateForContainer(
  option.containerNumber,
  {
    freeDateWriteMode: 'none',
    // ❌ 问题：没有传入 plannedPickupDate 参数！
    // 导致 DemurrageService 使用实际日期而非计划日期
  }
);
```

**问题分析**:

1. **参数缺失**: `calculateForContainer` 需要知道"计划提柜日"才能准确计算
2. **使用实际日期**: 如果不传参数，服务会使用当前日期（今天）作为截止日
3. **结果错误**: 
   - 3-28 日计算：从今天到 3-28 的费用 → $2,900
   - 3-30 日计算：从今天到 3-30 的费用 → 应该 > $2,900
   - 但因为都没传参数，都用今天计算 → 都是 $2,900

**修复方案**:

```typescript
// 在 evaluateTotalCost 方法中添加参数
const demurrageResult = await this.demurrageService.calculateForContainer(
  option.containerNumber,
  {
    freeDateWriteMode: 'none',
    // ✅ 传入计划提柜日
    plannedPickupDate: option.plannedPickupDate
  }
);
```
