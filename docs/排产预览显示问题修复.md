# 排产预览显示问题修复

## 问题描述

在排产预览/确认功能中发现两个显示问题：

1. **执行日志费用项显示不完整**：只显示了部分费用（运输费、卸货费、仓储费、其他），缺少重要的 D&D 相关费用项（滞港费、滞箱费、D&D 合并费、堆场堆存费等）。

2. **资源状态中仓库占用率显示异常**：占用率显示为 0%，但状态显示为"正常"，原因是预加载逻辑缺失仓库档期数据加载。

## 修复内容

### 2. 仓库档期预加载逻辑修复

**问题原因**: 预加载函数 `preloadCapacityData` 中只处理了车队档期请求，缺少仓库档期请求的处理逻辑。

**修复内容**: 在预加载逻辑中添加仓库档期的批量加载。

```typescript
// 处理仓库请求
const warehouseChunks = Array.from(warehouseRequests).reduce((acc, key, i) => {
  if (i % MAX_CONCURRENT === 0) acc.push([]);
  acc[acc.length - 1].push(key);
  return acc;
}, [] as string[][]);

for (const chunk of warehouseChunks) {
  const promises = chunk.map(async (key) => {
    const [type, code, date] = key.split(":");
    try {
      if (type === "warehouse") {
        await getWarehouseCapacityText({
          plannedData: {
            warehouseId: code, // ✅ warehouseId 就是 warehouseCode
            plannedUnloadDate: date,
          },
        });
      }
    } catch (error) {
      console.warn(`[预加载] ${key} 失败:`, error);
    }
  });
  await Promise.all(promises);
}
```

**同时修复**: 仓库代码获取逻辑，直接使用 `warehouseId`（因为后端返回的 `warehouseId` 就是 `warehouseCode`）：

```typescript
// 收集仓库
// ✅ 修复：warehouseId 就是 warehouseCode，直接使用
const warehouseCode = row.plannedData.warehouseId;
const unloadDate = row.plannedData.plannedUnloadDate;
if (warehouseCode && unloadDate) {
  const key = `warehouse:${warehouseCode}:${unloadDate}`;
  warehouseRequests.add(key);
}
```

## 技术要点

1. **费用项完整性**: 确保所有后端返回的费用字段都能在前端展示
2. **条件显示**: 只显示金额不为 0 的费用项，避免日志冗长
3. **排序逻辑**: 按 D&D 相关费用优先展示，便于用户快速识别风险
4. **预加载优化**: 仓库和车队档期数据都在渲染前批量加载，避免 429 错误
5. **并发控制**: 限制并发请求数为 10，避免服务器压力过大
6. **字段映射**: 明确 `warehouseId` 就是 `warehouseCode`，避免参数传递错误
