## 🔍 根本原因分析

### 问题 1: 原方案日期提取错误 ❌

**前端代码** (`SchedulingVisual.vue:1754`):

```typescript
optimizationReport.value = {
  originalCost: {
    total: typeof originalCost === "number" ? originalCost : 0,

    strategy: firstAlt?.strategy || suggestedStrategy || "Direct",
```

**问题分析**:

- `firstAlt?.pickupDate` 是 `alternatives[0]`，即优化候选方案的第一个
- `alternatives` 是后端返回的**优化候选方案列表**（前后 15 天探索）
- **不是**货柜的原始计划日期！

**正确逻辑**:

- 原方案应该使用 `basePickupDate`（原始计划日期，3-28）
- 优化后使用 `suggestedPickupDate`（建议日期，3-30）

### 问题 2: 成本计算没有使用计划日期 ❌

**后端代码** (`schedulingCostOptimizer.service.ts:3421`):

```typescript
async evaluateTotalCost(option: UnloadOption): Promise<CostBreakdown> {
  // ...

  // ❌ 调用 calculateForContainer 时没有传入计划日期
  const demurrageResult = await this.calculateForContainer(containerNumber, {
    freeDateWriteMode: 'none'
  });

  // ❌ calculateForContainer 使用的是货柜的实际日期，不是计划日期
  breakdown.demurrageCost = totalCostResult.demurrageCost;
  breakdown.detentionCost = totalCostResult.detentionCost;
  breakdown.storageCost = totalCostResult.storageCost;
  breakdown.transportationCost = totalCostResult.transportationCost;

  return breakdown;
}
```

**问题分析**:

1. `evaluateTotalCost` 接收了 `option.plannedPickupDate`（计划日期）
2. 但调用 `calculateForContainer` 时**没有传入计划日期**
3. `calculateForContainer` 使用的是货柜的**实际日期**（或当前日期）
4. 导致**不同计划日期的成本计算结果相同**

**影响**:

- 3-28 提柜的成本 = 3-29 提柜的成本 = 3-30 提柜的成本
- **滞港费没有递增**（应该每天增加）
- **优化失去意义**（因为成本都一样）

**问题分析**:

`calculateForContainer` 方法的设计是用于**计算实际发生的费用**，它基于：

- 实际提柜日 (`pickupDateActual`)
- 实际还箱日 (`returnTime`)
- 免费期截止日 (`lastFreeDate`)

但成本优化需要**预测不同计划日期的费用**，这需要：

- 计划提柜日 (`plannedPickupDate`)
- 计划还箱日 (`plannedReturnDate`)
- 基于计划日期重新计算免费期和计费天数
