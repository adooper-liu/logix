## 📋 **场景驱动选择**

### 场景 1: 早上开始工作 ☀️

```bash
# 最佳实践
.\start-logix-dev.ps1

# 等待服务就绪（约 60 秒）
# - 数据库：localhost:5432
# - 后端：http://localhost:3001
# - 前端：http://localhost:5173
# - 微服务：http://localhost:4000
```

### 场景 2: 只需要数据库 🗄️

```powershell
# 运行 SQL 脚本或查询数据
tsdb-start.bat
tsdb-db.bat

# 或使用 Docker exec 执行 SQL
docker exec -i logix-timescaledb-prod psql -U logix_user -d logix_db < backend/scripts/init-database-complete.sql

# 或直接使用 Makefile
make tsdb-up
make tsdb-db
```

### 场景 3: 查看日志 🔍

```bash
# 查看所有服务日志
tsdb-logs.bat

# 查看特定服务日志
tsdb-logs.bat postgres      # 数据库
tsdb-logs.bat backend       # 后端
tsdb-logs.bat redis         # Redis
tsdb-logs.bat grafana       # Grafana

# Docker Compose 方式
docker-compose -f docker-compose.timescaledb.prod.yml logs -f backend
```

### 场景 4: 进入数据库 💻

```powershell
# 使用批处理脚本（推荐）
tsdb-db.bat

# 或使用 Docker 命令
docker exec -it logix-timescaledb-prod psql -U logix_user -d logix_db

# 常用 SQL 命令
\l                      # 列出所有数据库
\dt                     # 列出所有表
\d table_name           # 查看表结构
SELECT * FROM table;    # 查询数据
\q                      # 退出
```

### 场景 5: 晚上下班 🌙

```bash
# 停止所有服务
.\stop-logix-dev.ps1

# 或仅停止数据库
tsdb-stop.bat
```

## ⚠️ **危险操作警告**

### 🔴 极度危险（会删除所有数据）

```bash
tsdb-clean.bat              # 清理所有 TimescaleDB 数据
make tsdb-clean             # Makefile 版本
scripts/cleanup-*.sql       # 清理业务数据
```

**使用前必须**:
1. ✅ 备份数据库：`pg_dump -U logix logix > backup.sql`
2. ✅ 确认是否需要
3. ✅ 通知团队成员

### 🟡 高度危险（会修改数据）

```bash
scripts/fix-*.sql           # 修复类脚本
scripts/update-*.sql        # 更新类脚本
backend/scripts/*.sql       # 批量修改脚本
```

**使用前必须**:
1. ✅ 在测试环境先验证
2. ✅ 备份相关数据
3. ✅ 准备好回滚方案

### ✅ 安全操作（只读）

```bash
scripts/check-*.sql         # 检查类脚本
scripts/query-*.sql         # 查询类脚本
scripts/verify-*.sql        # 验证类脚本
tsdb-info.bat               # 统计信息查询
```

## 🆘 **故障排查速查**

### 问题 1: Docker 未运行

```bash
# Windows
# 启动 Docker Desktop
"C:\Program Files\Docker\Docker\Docker Desktop.exe"

# 或使用智能启动脚本
.\start-docker-smart.bat
```

### 问题 2: 端口被占用

```bash
# 查看端口占用
netstat -ano | findstr :5432
netstat -ano | findstr :3001
netstat -ano | findstr :5173

# 停止占用进程（谨慎使用）
Stop-Process -Id <PID> -Force
```

### 问题 3: 数据库无法连接

```bash
# 检查容器状态
docker ps | grep timescaledb

# 查看数据库日志
tsdb-logs.bat postgres

# 重启数据库
tsdb-stop.bat
tsdb-start.bat
```

### 问题 4: Node.js 进程残留

```bash
# Windows：强制停止所有 Node 进程
Get-Process node | Stop-Process -Force

# 或使用任务管理器手动结束
```

## 💡 **最佳实践**

### ✅ 推荐做法

```bash
# 1. 使用统一脚本启动环境
.\start-logix-dev.ps1

# 2. 数据库操作使用专用脚本
tsdb-start.bat          # 启动
tsdb-db.bat             # 进入
tsdb-logs.bat backend   # 查看日志

# 3. 定期清理测试数据
psql -f scripts/cleanup-test-data.sql

# 4. 离开时停止服务
.\stop-logix-dev.ps1
```

### ❌ 避免做法

```bash
# 1. 直接 docker-compose up（缺少依赖检查）
# 2. 手动杀进程（可能导致数据损坏）
# 3. 跳过备份直接清理数据
# 4. 在生产环境运行测试脚本
```
