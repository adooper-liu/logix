# 429 错误修复 - 渲染时禁止调用异步 API

## 问题描述

在排产预览页面，执行日志中出现大量 **429 Too Many Requests** 错误：

```
GET http://localhost:5173/api/v1/scheduling/resources/capacity/range?... 429 (Too Many Requests)
获取仓库档期失败：AxiosError: Request failed with status code 429
```

## 根因分析

### 1. 问题现象
- 表格渲染时频繁调用档期检查 API
- 每个单元格渲染都会触发 `getWarehouseCapacityText()` 异步请求
- 5 个货柜可能触发数十次 API 请求

### 2. 根本原因

**违反 Vue 渲染规范**：在模板渲染函数中调用异步 API

```vue
<!-- ❌ 错误示例 -->
<el-tag :type="getWarehouseCapacityType(row)">
  {{ getWarehouseCapacityLabel(row) }}
</el-tag>

<script>
// 这个方法在每次渲染时都会被调用
const getWarehouseCapacityType = (row: any) => {
  // ... 检查缓存
  
  // ❌ 如果缓存不存在，触发异步请求
  getWarehouseCapacityText(row)  // ← 异步调用 API
  
  return 'info'
}
</script>
```

### 3. 为什么会重复调用？

Vue 的响应式系统会在以下情况触发重新渲染：
- 表格数据变化
- 表格高度调整
- 用户交互（排序、筛选）
- 父组件更新

每次重新渲染都会再次调用这些函数，导致：
- 同一数据被重复请求
- 缓存失效
- API 限流（429）

## 解决方案

### 核心原则

**渲染函数必须是纯函数**：
- ✅ 只能读取已缓存的数据
- ❌ 禁止触发新的异步请求
- ❌ 禁止产生副作用

### 修复方案

#### 1. 预加载所有数据（已实现）

在预览完成后，批量预加载所有档期数据：

```typescript
// ✅ 正确：预加载所有数据
const preloadCapacityData = async (results: any[]) => {
  const warehouseRequests = new Set<string>()
  
  // 收集所有需要请求的仓库
  results.forEach(row => {
    const warehouseCode = row.plannedData.warehouseCode
    const unloadDate = row.plannedData.plannedUnloadDate
    if (warehouseCode && unloadDate) {
      warehouseRequests.add(`warehouse:${warehouseCode}:${unloadDate}`)
    }
  })
  
  // 批量并发请求（限制并发数）
  const MAX_CONCURRENT = 10
  // ... 并发请求并缓存
}

// 在预览完成后调用
await preloadCapacityData(result.results)
```

#### 2. 渲染函数只读缓存

```typescript
// ✅ 正确：只读缓存，不触发请求
const getWarehouseCapacityType = (row: any) => {
  const warehouseCode = row.plannedData?.warehouseId
  const unloadDate = row.plannedData?.plannedUnloadDate
  
  if (!warehouseCode || !unloadDate) return 'info'
  
  const cacheKey = `warehouse:${warehouseCode}:${unloadDate}`
  
  // ✅ 只检查缓存，不触发新请求
  if (capacityCache.value.has(cacheKey)) {
    const data = capacityCache.value.get(cacheKey)
    const text = typeof data === 'string' ? data : data.status
    if (text === '超负荷') return 'danger'
    if (text === '紧张') return 'warning'
    return 'success'
  }
  
  // ✅ 缓存不存在时返回默认值，不触发请求
  return 'info'
}
```

#### 3. 完整的修复清单

| 函数 | 修复前 | 修复后 |
|------|--------|--------|
| `getWarehouseCapacityType` | ❌ 调用 `getWarehouseCapacityText()` | ✅ 只读缓存 |
| `getWarehouseOccupancyRate` | ❌ 调用 `getWarehouseCapacityText()` | ✅ 只读缓存 |
| `getTruckingCapacityType` | ❌ 调用 `getTruckingCapacityText()` | ✅ 只读缓存 |
| `getTruckingOccupancyRate` | ❌ 调用 `getTruckingCapacityText()` | ✅ 只读缓存 |
| `getWarehouseCapacityLabel` | ✅ 只读缓存 | ✅ 保持只读 |
| `getTruckingCapacityLabel` | ✅ 只读缓存 | ✅ 保持只读 |

## 技术实现细节

### 缓存结构

```typescript
const capacityCache = ref<Map<string, any>>(new Map())

// 缓存格式
cacheKey = `warehouse:${warehouseCode}:${unloadDate}`
cacheValue = { 
  status: '正常' | '紧张' | '超负荷',
  occupancyRate: 75.5  // 百分比
}
```

### 数据流

```
1. 预览排产
   ↓
2. 获取排产结果
   ↓
3. 预加载档期数据（批量请求，限制并发）
   ↓
4. 数据存入缓存
   ↓
5. 渲染表格
   ↓
6. 渲染函数从缓存读取（不触发新请求）
```

## 相关规范

### Vue 渲染规范

> **渲染函数必须是纯函数**
> - 不应该有副作用
> - 不应该触发异步操作
> - 不应该修改状态

参考：[Vue 官方文档 - 计算属性 vs 方法](https://cn.vuejs.org/guide/essentials/computed.html#computed-vs-methods)

### SKILL 开发原则

- **S**imple: 保持渲染函数简单
- **K**ISS: 避免过度设计
- **I**diomatic: 遵循 Vue 最佳实践
- **L**azy: 延迟加载，提前缓存
- **L**ightweight: 轻量级渲染

## 经验教训

### 1. Vue 渲染函数的陷阱

```typescript
// ❌ 错误：渲染时触发异步
const renderCell = (row: any) => {
  const data = await fetchApi()  // ← 绝对禁止
  return h('div', data)
}

// ✅ 正确：使用缓存
const renderCell = (row: any) => {
  const data = cache.get(row.key)  // ← 只读缓存
  return h('div', data)
}
```

### 2. 预加载策略

- **时机**：数据获取完成后立即预加载
- **批量**：合并相同类型的请求
- **限流**：控制并发数避免阻塞
- **缓存**：统一缓存格式便于访问

### 3. 错误处理

```typescript
try {
  await preloadCapacityData(results)
} catch (error) {
  console.warn('[预加载] 部分数据加载失败，使用降级逻辑')
  // 降级：返回默认状态
}
```
