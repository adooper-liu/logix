# 车队数据架构优化方案

## 当前问题分析

### 1. 主键设计不一致

**dict_trucking_companies（主数据表）**

```typescript
@PrimaryColumn({ type: 'varchar', length: 100, name: 'company_code' })
companyCode: string;  // 使用业务代码作为主键
```

**dict_trucking_port_mapping（映射表）**

```typescript
@PrimaryGeneratedColumn()
id: number;  // 自增 ID 作为主键

@Column({ type: 'varchar', length: 50, name: 'trucking_company_id' })
truckingCompanyId: string;  // 但关联字段是 varchar
```

**问题：**

- 主数据表使用 `company_code`（业务主键）
- 映射表使用 `id`（代理主键）+ `trucking_company_id`（业务外键）
- 类型长度不一致：主表 100，映射表 50

### 2. 数据冗余

两个映射表都存储了 `trucking_company_name`：

```typescript
@Column({ type: 'varchar', length: 200, nullable: true, name: 'trucking_company_name' })
truckingCompanyName: string;
```

**问题：**

- 违反了数据库第三范式（3NF）
- 车队名称变更时，需要同步更新多个表
- 浪费存储空间

### 3. 缺少外键约束

当前没有数据库级别的外键约束，导致：

- 可以插入不存在的车队代码
- 删除车队后，映射记录成为孤儿数据
- 数据完整性依赖应用层保证

## 优化方案

### 方案一：添加外键约束（推荐优先实施）✅

**优点：**

- 保证数据完整性
- 级联删除/更新
- 改动最小

**实施步骤：**

1. **执行迁移脚本**

```bash
psql -U postgres -d logix -f migrations/add_trucking_foreign_keys.sql
```

2. **修改实体类字段长度**

```typescript
// TruckingPortMapping.ts 和 WarehouseTruckingMapping.ts
@Column({ type: 'varchar', length: 100, name: 'trucking_company_id' })  // 从 50 改为 100
truckingCompanyId: string;
```

3. **应用层逻辑调整**

- 导入时必须先创建车队
- 查询时使用 JOIN 获取车队信息

### 方案二：完全规范化（长期目标）🎯

**核心思想：**

- 映射表只存储 `company_code`
- 移除所有冗余的 `trucking_company_name` 字段
- 通过视图或查询时 JOIN 获取完整信息

**实施步骤：**

#### 1. 修改数据库表结构

```sql
-- 移除冗余字段
ALTER TABLE dict_trucking_port_mapping DROP COLUMN trucking_company_name;
ALTER TABLE dict_warehouse_trucking_mapping DROP COLUMN trucking_company_name;

-- 创建视图方便查询
CREATE VIEW v_trucking_port_mapping_full AS
SELECT
    tpm.id,
    tpm.country,
    tpm.trucking_company_id,
    tc.company_name AS trucking_company_name,
    tc.company_name_en AS trucking_company_name_en,
    tpm.port_code,
    tpm.port_name,
    tpm.yard_capacity,
    tpm.standard_rate,
    tpm.unit,
    tpm.yard_operation_fee,
    tpm.transport_fee,
    tpm.mapping_type,
    tpm.is_default,
    tpm.is_active,
    tpm.remarks
FROM dict_trucking_port_mapping tpm
LEFT JOIN dict_trucking_companies tc ON tpm.trucking_company_id = tc.company_code;
```

#### 2. 修改后端 Entity

```typescript
// TruckingPortMapping.ts
@Entity("dict_trucking_port_mapping")
export class TruckingPortMapping {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ type: "varchar", length: 50, name: "country" })
  country: string;

  // 只保留代码，不保留名称
  @Column({ type: "varchar", length: 100, name: "trucking_company_id" })
  truckingCompanyId: string;

  // 可选：添加关系定义
  @ManyToOne(() => TruckingCompany)
  @JoinColumn({ name: "trucking_company_id", referencedColumnName: "companyCode" })
  truckingCompany?: TruckingCompany;

  // ... 其他字段
}
```

#### 3. 修改后端查询

```typescript
// 使用 JOIN 查询
const mappings = await truckingPortMappingRepository.createQueryBuilder("tpm").leftJoinAndSelect("tpm.truckingCompany", "tc").where("tpm.country = :country", { country }).getMany();

// 返回时自动包含车队信息
return {
  success: true,
  data: mappings.map((m) => ({
    ...m,
    truckingCompanyName: m.truckingCompany?.companyName,
    truckingCompanyNameEn: m.truckingCompany?.companyNameEn,
  })),
};
```

#### 4. 修改前端显示

前端代码不需要大改，因为 API 返回的数据结构保持一致。

## 导入流程优化

### 当前流程（需要改进）❌

```
Excel → 解析数据 → 尝试创建映射
           ↓
    发现车队不存在 → 创建车队
           ↓
    再次创建映射
```

### 优化后流程 ✅

```
Excel → 解析数据 → 验证车队是否存在
           ↓
    不存在 → 拒绝导入，提示先创建车队
           ↓
    存在 → 直接创建映射（外键自动关联）
```

或者更友好的方式：

```
Excel → 解析数据 → 自动创建/更新车队
           ↓
    自动生成 company_code
           ↓
    创建映射（外键自动关联）
```

## 数据完整性保障

### 外键约束的作用

```sql
FOREIGN KEY (trucking_company_id)
REFERENCES dict_trucking_companies(company_code)
ON DELETE CASCADE    -- 删除车队时，自动删除相关映射
ON UPDATE CASCADE;   -- 更新车队代码时，自动更新映射
```

### 应用层验证

即使有外键约束，应用层也应该：

1. 导入前验证车队是否存在
2. 提供友好的错误提示
3. 支持批量创建车队

## 性能优化

### 索引策略

```sql
-- 外键字段索引（加速 JOIN）
CREATE INDEX idx_trucking_port_mapping_company_id
ON dict_trucking_port_mapping(trucking_company_id);

-- 复合索引（常用查询条件）
CREATE INDEX idx_trucking_port_country_active
ON dict_trucking_port_mapping(country, is_active);
```

### 查询优化

```typescript
// ❌ 避免 N+1 查询
const mappings = await repo.find();
for (const m of mappings) {
  const company = await truckingRepo.findOne({ where: { code: m.truckingCompanyId } });
}

// ✅ 使用 JOIN 一次性查询
const mappings = await repo.createQueryBuilder("tpm").leftJoinAndSelect("tpm.truckingCompany", "tc").getMany();
```
