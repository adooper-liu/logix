# 滞港费计算：问题归纳与前端展示说明

## 一、业务与数据层（常见问题）

1. **匹配到多条标准，界面只有一条「计费」行**  
   - `ext_demurrage_standards` 中 `is_chargeable = 'N'` 的多条记录可同时命中。  
   - **滞箱费 / Demurrage & Detention（合并项）** 依赖 **实际提柜日**（`process_trucking_transport.pickup_date`）；无提柜则 **不参与计费**，进入 **「已匹配暂不计算」** 列表。  
   - **滞港 / Storage 等** 可在无提柜时用「当天」作截止，进入 **「已计费明细」**。

2. **计费天数 > 0，金额为 0**  
   - 命中标准的 **`rate_per_day` 与 `tiers` 均为空** 时，公式无法产生金额。需在标准表中维护费率。

3. **备货单仅有 `customer_name`、无 `customer_code`**  
   - 匹配仍可用客户名称对齐 `foreign_company_name`；建议补全 `customer_code` 以便关联 `biz_customers`。

4. **提柜为空但已有 `return_time`**  
   - 数据需业务核对；滞箱/合并项仍以 **提柜** 为前置条件。

## 二、前端展示约定（LogiX）

- **合计金额** = 下方 **「已计费明细」** 各行金额之和，**不含**「暂不计算」项。  
- **顶部四维度标签** 取 **首条匹配标准**（同组合下通常一致）；并展示 **客户/境外公司名称**（若接口返回）。  
- **费率缺失** 时在表格内标注 **「未配置」**，避免误解为计算错误。
