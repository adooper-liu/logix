**项目**: LogiX 物流管理系统  
**相关文档**: [返回目录](./README.md)

## 1. 核心计算逻辑

### 成本构成

```
TotalCost = DemurrageCost + DetentionCost + StorageCost
```

其中:
- **Demurrage (滞港费)**: 超过免费期后，货物滞留在港口的费用
- **Detention (滞箱费)**: 超过免费期后，集装箱滞留在手中的费用
- **Storage (堆存费)**: 港口堆存费用

### 优化目标

在满足约束条件下，找到**总成本最低**的候选日期组合。

## 2. 场景模拟

### 场景 1: 不同候选日期的成本对比

**货柜参数**:
- 货柜号: ECMU5397691
- ATA: 2026-02-11
- 免费天数: 7天
- 日费率: $50/天

**候选日期与成本**:

| 候选提柜日 | rangeEnd | chargeDays | 滞港费 | 成本 |
|------------|----------|------------|--------|------|
| 2026-03-27 (今天) | 2026-03-27 | 38天 | $1,900 | $1,900 |
| 2026-03-30 | 2026-03-30 | 35天 | $1,750 | $1,750 |
| 2026-03-31 | 2026-03-31 | 34天 | $1,700 | $1,700 |

**计算过程 (候选日: 2026-03-31)**:
```
rangeEnd = max(2026-03-31, 2026-03-27) = 2026-03-31
lastFreeDate = 2026-02-11 + 7 - 1 = 2026-02-17
chargeStart = 2026-02-18
chargeDays = 2026-03-31 - 2026-02-18 + 1 = 41天 ❌

修正: rangeEnd = min(2026-03-31, 2026-03-27) = 2026-03-27
chargeDays = 2026-03-27 - 2026-02-18 + 1 = 38天 ❌

问题: paramsOverride 覆盖无效!

修复后:
rangeEnd = max(plannedPickupDate, today) 正确理解
- 候选日 03-27: rangeEnd = 03-27 → chargeDays = 38
- 候选日 03-30: rangeEnd = 03-30 → chargeDays = 35 ✅
- 候选日 03-31: rangeEnd = 03-31 → chargeDays = 34 ✅
```

**结论**: 候选日期越晚，chargeDays 越少，费用越低

### 场景 2: 免费期内 vs 超期

**货柜参数**:
- ATA: 2026-03-20
- 免费天数: 7天
- 日费率: $80/天

**不同计算日期**:

| 计算日期 | 计划提柜日 | 剩余免费天 | 费用 |
|----------|------------|------------|------|
| 2026-03-22 | 2026-03-25 | 2天 | $0 |
| 2026-03-26 | 2026-03-25 | -1天(超1天) | $80 |
| 2026-03-28 | 2026-03-25 | -3天(超3天) | $240 |

**计算过程 (2026-03-28)**:
```
lastFreeDate = 2026-03-20 + 7 - 1 = 2026-03-26
endDate = min(2026-03-28, 2026-03-25) = 2026-03-25

因为 03-25 <= 03-26 (lastFreeDate)
所以 chargeDays = 0, 费用 = $0 ❌

修正: endDate = max(today, plannedPickupDate)
2026-03-28 > 2026-03-25 → endDate = 2026-03-28

chargeStart = 2026-03-27
chargeDays = 2026-03-28 - 2026-03-27 + 1 = 2天
费用 = 2 × $80 = $160 ❌

最终修正: rangeEnd = max(today, plannedPickupDate) 只对预测模式
actual模式: rangeEnd = min(today, plannedPickupDate)
```

### 场景 3: 批量优化 - 分类处理

**货柜列表**:

| 货柜 | ATA | 免费日 | 状态 |
|------|-----|--------|------|
| C1 | 02-01 | 02-07 | 超期20天 |
| C2 | 02-15 | 02-21 | 超期7天 |
| C3 | 03-20 | 03-26 | 免费期内2天 |
| C4 | 03-25 | 03-31 | 免费期内4天 |

**优化策略**:

```
Step 1: 分类
├── 超期组: C1 (超20天), C2 (超7天)
└── 免费期组: C3 (剩2天), C4 (剩4天)

Step 2: 优先级排序
├── 超期组 (优先): C1 > C2 (超期越多越优先)
└── 免费期组: C4 > C3 (剩余越少越优先)

Step 3: 仓库能力检查
├── C1 → 分配到 04-01
├── C2 → 分配到 04-02
├── C3 → 分配到 03-28
└── C4 → 分配到 03-29
```

**优化结果对比**:

| 货柜 | 原计划日 | 优化后 | 节省费用 |
|------|----------|--------|----------|
| C1 | 超期20天 | 04-01 | $0 (超期成本=0) |
| C2 | 超期7天 | 04-02 | $0 |
| C3 | 03-26 | 03-28 | $0 (免费期内) |
| C4 | 03-31 | 03-29 | $0 |
| **合计** | - | - | **$0** |
