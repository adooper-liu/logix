## 1. 系统概述

### 1.1 功能定位

智能排柜系统是 LogiX 物流管理系统的核心模块，用于自动化安排集装箱的提送卸还计划。

**核心价值**:

- 🎯 自动化排产：减少人工干预，提高排产效率
- 💰 成本优化：通过智能算法降低滞港费、堆场费等
- 📊 资源平衡：合理分配仓库、车队、堆场资源
- ⚠️ 风险管控：避免超期、产能超限等问题

### 1.2 业务流程

```
货柜到港 (ETA/ATA)
    ↓
清关计划日
    ↓
提柜日 (pickup) ──→ 送柜日 (delivery) ──→ 卸柜日 (unload) ──→ 还箱日 (return)
         │              │                    │                   │
         └──────────────┴────────────────────┴───────────────────┘
                        约束条件：提 ≤ 送 ≤ 卸 ≤ 还
```

### 1.3 卸柜方式

| 方式          | 适用场景   | 时间关系     | 特点               |
| ------------- | ---------- | ------------ | ------------------ |
| **Live load** | 车队无堆场 | 提 = 送 = 卸 | 直接送仓，无堆场费 |
| **Drop off**  | 车队有堆场 | 提 < 送 = 卸 | 先放堆场，灵活调度 |

## 2. 核心功能

### 2.1 智能排产引擎

#### 功能特性

- ✅ 多约束条件校验（仓库容量、车队能力、还箱期限）
- ✅ 周末跳过（可配置）
- ✅ Live/Drop 模式自动判断
- ✅ 档期占用管理（仓库、车队、堆场）
- ✅ 清关公司自动匹配

#### 排产流程

```typescript
1. 获取目的港操作记录（ETA/ATA/last_free_date）
   ↓
2. 计算计划清关日、提柜日
   ↓
3. 选择候选仓库（基于港口→车队→仓库映射链）
   ↓
4. 选择车队（基于仓库映射，检查 has_yard）
   ↓
5. 确定卸柜方式（has_yard ? Drop off : Live load）
   ↓
6. 找最早可用卸柜日（检查仓库容量）
   ↓
7. 计算送柜日、还箱日
   ↓
8. 扣减档期（仓库、车队送柜、车队还箱）
   ↓
9. 更新数据库，返回结果
```

### 2.2 档期管理

#### 仓库日产能

**表**: `ext_warehouse_daily_occupancy`

| 字段           | 类型        | 说明                                 |
| -------------- | ----------- | ------------------------------------ |
| warehouse_code | VARCHAR(50) | 仓库编码                             |
| date           | DATE        | 日期                                 |
| planned_count  | INTEGER     | 已计划卸柜数                         |
| capacity       | INTEGER     | 总容量（来自 daily_unload_capacity） |
| remaining      | INTEGER     | 剩余可用容量                         |

**使用逻辑**:

```sql
-- 查询某仓库某日是否可用
SELECT remaining FROM ext_warehouse_daily_occupancy
WHERE warehouse_code = 'WH001' AND date = '2026-03-20';

-- 扣减容量（排产时）
UPDATE ext_warehouse_daily_occupancy
SET planned_count = planned_count + 1,
    remaining = remaining - 1
WHERE warehouse_code = 'WH001' AND date = '2026-03-20';
```

#### 车队送柜档期

**表**: `ext_trucking_slot_occupancy`

| 字段                | 类型        | 说明                          |
| ------------------- | ----------- | ----------------------------- |
| trucking_company_id | VARCHAR(50) | 车队 ID                       |
| date                | DATE        | 送柜日期                      |
| port_code           | VARCHAR(50) | 目的港（可选）                |
| warehouse_code      | VARCHAR(50) | 仓库（可选）                  |
| planned_trips       | INTEGER     | 已计划送柜次数                |
| capacity            | INTEGER     | 总容量（来自 daily_capacity） |

#### 车队还箱档期

**表**: `ext_trucking_return_slot_occupancy`

| 字段                | 类型        | 说明                                 |
| ------------------- | ----------- | ------------------------------------ |
| trucking_company_id | VARCHAR(50) | 车队 ID                              |
| slot_date           | DATE        | 还箱日期                             |
| planned_count       | INTEGER     | 已计划还箱数                         |
| capacity            | INTEGER     | 总容量（来自 daily_return_capacity） |

**注意**: 仅在 Drop off 模式下扣减此表。

### 2.3 映射关系

#### 三级映射链

```
目的港 (port_code)
    ↓ dict_trucking_port_mapping
车队 (trucking_company_id)
    ↓ dict_warehouse_trucking_mapping
仓库 (warehouse_code)
```

**配置要求**:

1. 必须在 `dict_trucking_port_mapping` 中配置港口 → 车队
2. 必须在 `dict_warehouse_trucking_mapping` 中配置车队 → 仓库
3. 无映射关系的仓库/车队不会被选中

## 4. 算法逻辑

### 4.1 核心公式

#### 计划清关日

```typescript
clearanceDate = ETA || ATA; // 优先 ETA，无则 ATA
```

#### 计划提柜日

```typescript
pickupDate = clearanceDate + 1天

约束:
1. pickupDate ≤ last_free_date（避免滞港费）
2. pickupDate ≥ today（不能回到过去）
3. pickupDate ≠ 周末（如果 skip_weekends = true）
```

#### 卸柜方式判断

```typescript
if (truckingCompany.hasYard === true) {
  unloadMode = "Drop off"; // 提 < 送 = 卸
} else {
  unloadMode = "Live load"; // 提 = 送 = 卸

  // 如果提≠卸，调整为 Live load
  if (pickupDay !== unloadDay) {
    unloadDay = findEarliestAvailableDay(warehouse, pickupDate);
  }
}
```

#### 计划送柜日

```typescript
if (unloadMode === "Live load") {
  deliveryDate = pickupDate; // 提 = 送
} else {
  deliveryDate = unloadDate; // 送 = 卸
}
```

#### 计划还箱日

```typescript
if (unloadMode === 'Live load') {
  returnDate = unloadDate  // 还 = 卸
} else {
  returnDate = unloadDate + 1  // Drop off: 还 = 卸 + 1
}

约束:
returnDate ≤ lastReturnDate（如果有）
```

### 4.2 档期扣减逻辑

#### 仓库档期扣减

```typescript
async decrementWarehouseOccupancy(warehouseCode, date):
  occupancy = findOne({ warehouseCode, date })

  if (occupancy exists):
    occupancy.plannedCount += 1
    occupancy.remaining -= 1
    save(occupancy)
  else:
    capacity = warehouse.dailyUnloadCapacity || 10
    insert({ warehouseCode, date, plannedCount: 1, capacity, remaining: capacity-1 })
```

#### 车队送柜档期扣减

```typescript
async decrementTruckingOccupancy(truckingCompanyId, date, portCode, warehouseCode):
  occupancy = findOne({ truckingCompanyId, date, portCode, warehouseCode })

  if (occupancy exists):
    occupancy.plannedTrips += 1
    save(occupancy)
  else:
    capacity = truckingCompany.dailyCapacity || 10
    insert({ truckingCompanyId, date, portCode, warehouseCode, plannedTrips: 1, capacity })
```

#### 车队还箱档期扣减（仅 Drop off）

```typescript
async decrementFleetReturnOccupancy(truckingCompanyId, returnDate):
  if (unloadMode !== 'Drop off') return

  occupancy = findOne({ truckingCompanyId, slotDate: returnDate })

  if (occupancy exists):
    occupancy.plannedCount += 1
    occupancy.remaining -= 1
    save(occupancy)
  else:
    capacity = truckingCompany.dailyReturnCapacity || truckingCompany.dailyCapacity || 20
    insert({ truckingCompanyId, slotDate: returnDate, plannedCount: 1, capacity, remaining: capacity-1 })
```

### 4.3 仓库选择算法

#### 候选仓库过滤

```typescript
getCandidateWarehouses(countryCode, portCode):
  // 1. 港口→车队映射
  portMappings = find({ portCode, country: countryCode })
  truckingCompanyIds = portMappings.map(m => m.truckingCompanyId)

  // 2. 车队→仓库映射
  warehouseMappings = find({
    truckingCompanyId: In(truckingCompanyIds),
    country: countryCode
  })
  warehouseCodes = unique(warehouseMappings.map(m => m.warehouseCode))

  // 3. 返回仓库列表
  warehouses = find({
    warehouseCode: In(warehouseCodes),
    country: countryCode,
    status: 'ACTIVE'
  })

  return sortWarehousesByPriority(warehouses, warehouseMappings)
```

#### 仓库排序规则

```typescript
sortWarehousesByPriority(warehouses, mappings):
  defaultWarehouseCodes = mappings.filter(m => m.isDefault).map(m => m.warehouseCode)

  sort(warehouses) by:
    1. is_default (在 mapping 中 isDefault=true)
    2. property_type (自营仓 > 平台仓 > 第三方仓)
    3. warehouse_code (字典序)
```

## 8. 故障排查

### 8.1 常见问题

#### 问题 1: 排产失败 - "无映射关系中的仓库"

**原因**:

- `dict_trucking_port_mapping` 中无该港口的映射
- `dict_warehouse_trucking_mapping` 中无对应车队的映射

**解决方案**:

```sql
-- 检查映射关系
SELECT * FROM dict_trucking_port_mapping WHERE port_code = 'CA_VAN';
SELECT * FROM dict_warehouse_trucking_mapping WHERE trucking_company_id = 'TRUCK_001';

-- 添加缺失的映射
INSERT INTO dict_trucking_port_mapping (...) VALUES (...);
INSERT INTO dict_warehouse_trucking_mapping (...) VALUES (...);
```

#### 问题 2: 排产失败 - "仓库产能不足"

**原因**:

- 仓库在选定日期的 planned_count >= capacity

**解决方案**:

```sql
-- 查看仓库产能使用情况
SELECT date, planned_count, capacity, remaining
FROM ext_warehouse_daily_occupancy
WHERE warehouse_code = 'WH_001'
ORDER BY date DESC LIMIT 10;

-- 临时增加产能
UPDATE dict_warehouses
SET daily_unload_capacity = 50
WHERE warehouse_code = 'WH_001';
```

#### 问题 3: 排产失败 - "无映射关系中的车队"

**原因**:

- `dict_warehouse_trucking_mapping` 中无该仓库的映射
- 所有候选车队的档期已满

**解决方案**:

```sql
-- 检查映射关系
SELECT * FROM dict_warehouse_trucking_mapping
WHERE warehouse_code = 'WH_001';

-- 查看车队档期使用情况
SELECT trucking_company_id, date, planned_trips, capacity, remaining
FROM ext_trucking_slot_occupancy
WHERE date >= CURRENT_DATE
ORDER BY date LIMIT 20;
```

#### 问题 4: 周末未跳过

**原因**:

- `skip_weekends` 配置为 false 或 NULL

**解决方案**:

```sql
-- 检查配置
SELECT config_value FROM dict_scheduling_config
WHERE config_key = 'skip_weekends';

-- 启用周末跳过
UPDATE dict_scheduling_config
SET config_value = 'true'
WHERE config_key = 'skip_weekends';
```
