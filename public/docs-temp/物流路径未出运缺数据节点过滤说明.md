# 物流路径"未出运 缺数据"节点过滤说明

**修改时间**: 2026-04-02  
**修改人**: 刘志高

---

## 需求

在前端界面不显示"未出运 缺数据"节点，但在后端状态定义中保留该状态。

---

## 实现方案

### 前端过滤逻辑

**文件**: `frontend/src/views/shipments/components/LogisticsPathTab.vue`

**修改内容**:

在 `nodesByStage` 计算属性中添加节点过滤逻辑：

```typescript
const nodesByStage = computed(() => {
  if (!path.value?.nodes?.length) return [];

  // ... 运输模式过滤逻辑

  // 过滤掉不需要显示的节点（如"未出运 缺数据"）
  const shouldShowNode = (node: StatusNode) => {
    // 不显示"未出运 缺数据"节点
    if (node.status === "NOT_SHIPPED" && (node.rawData as { noData?: boolean })?.noData) {
      return false;
    }
    return true;
  };

  path.value!.nodes.forEach((node, globalIndex) => {
    // 先过滤掉不需要显示的节点
    if (!shouldShowNode(node)) return;

    // ... 分组逻辑
  });

  return Object.values(groups).sort((a, b) => a.order - b.order);
});
```

---

## 技术细节

### 判断条件

```typescript
node.status === "NOT_SHIPPED" && (node.rawData as { noData?: boolean })?.noData;
```

**解释**:

- `node.status === 'NOT_SHIPPED'`: 状态为"未出运"
- `(node.rawData as { noData?: boolean })?.noData`: `rawData.noData` 为 `true`，表示这是缺数据的占位节点

**同时满足以上两个条件** → 不显示该节点

## 后端保留状态定义

**文件**: `logistics-path-system/backend/src/services/statusPathFromDb.ts`

后端仍然生成和返回"未出运 缺数据"节点，保持完整的状态链：

```typescript
const STANDARD_MODE_TEMPLATE = [
  { order: 1, status: StandardStatus.NOT_SHIPPED, label: "未出运", statuses: [StandardStatus.NOT_SHIPPED] },
  // ... 其他阶段

```

**好处**:

1. 保持状态链完整性
2. 便于调试和日志分析
3. 如果后续需要显示，只需修改前端过滤逻辑

## 注意事项

### 1. 仅前端过滤

此修改**仅影响前端显示**，不影响：

- 后端数据生成
- API 返回值
- 数据库存储
- 其他业务逻辑

### 2. rawData.noData 字段

`noData` 字段由后端在生成占位节点时设置：

```typescript
// 后端代码示例
rawData: {
  noData: true,  // 标记为缺数据节点
  // ... 其他字段
}
```

### 3. 扩展性

如需过滤其他节点，可在 `shouldShowNode` 函数中添加更多条件：

```typescript

    return false
  }

  // 不显示其他特定节点
  if (/* 其他条件 */) {
    return false
  }

  return true
}
```

## 常见问题

### Q1: 为什么不直接在后端移除这个节点？

**A**:

- 后端保留完整状态链，便于调试和数据分析
- 前端可能在不同场景下需要不同的显示策略
- 分离关注点：后端提供完整数据，前端决定如何展示

### Q2: 如果以后需要显示这个节点怎么办？

**A**:

- 方法 1: 删除或注释掉 `shouldShowNode` 中的过滤逻辑
- 方法 2: 添加配置项，允许用户切换是否显示

### Q3: 其他"缺数据"节点会显示吗？

**A**:

- 会显示。当前只过滤"未出运 缺数据"
- 其他阶段的缺数据节点（如"进港 缺数据"）仍会显示
- 如需过滤，可在 `shouldShowNode` 中添加相应条件
