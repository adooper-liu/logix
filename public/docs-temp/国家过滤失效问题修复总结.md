# 国家过滤失效问题修复总结

## 问题描述

**现象**：

- 用户选择国家：**意大利（IT）**
- 点击"按最晚还箱 - 紧急：倒计时 3 天内"卡片（显示 3 个货柜）
- 表格过滤出 **6 条记录**
- 其中 **3 条是英国费利克斯托** 的货柜

**根本原因**：前端 `container.ts` 服务使用了独立的 axios 实例，而不是全局 `api.ts` 实例，导致国家代码请求头未正确添加。

---

## 修复内容

### 0. 关键问题发现（响应拦截器导致的双重解包）

**问题**：在修复过程中引入了新的错误

**原因分析**：

- `api.ts` 的响应拦截器已经返回 `response.data`
- `container.ts` 中又再次访问 `response.data`
- 导致实际返回的是 `response.data.data`，而不是 `response.data`

**修复方法**：

```typescript
// ❌ 错误
async getContainers(filters): Promise<ContainerResponse> {
  const response = await api.get('/containers', { params })
  return response.data as ContainerResponse  // ❌ 双重解包
}

// ✅ 正确
async getContainers(filters): Promise<ContainerResponse> {
  const response = await api.get('/containers', { params })
  return response as ContainerResponse  // ✅ api.get() 已返回 response.data
}
```

**影响范围**：所有 18 个方法都需要修改 `return response.data` 为 `return response`

## 技术要点

### 1. Axios 响应拦截器的返回值

**关键理解**：

```typescript
// api.ts 的响应拦截器
apiClient.interceptors.response.use(
  (response: AxiosResponse) => {
    return response.data; // ✅ 返回 response.data，而不是完整 response
  },
  (error) => Promise.reject(error),
);

// api.get() 的实际返回值
const response = await api.get("/containers");
// response = response.data（后端返回的实际数据）
//         ^ 已经是 { success: true, items: [], pagination: {} }
```

**常见错误模式**：

```typescript
// ❌ 错误示例
async getContainers() {
  const response = await api.get('/containers')
  return response.data  // ❌ 访问 undefined.data
}

// ✅ 正确示例
async getContainers() {
  const response = await api.get('/containers')
  return response  // ✅ 直接返回（已经是 response.data）
}
```

**为什么这样设计**：

- ✅ 简化调用：`const data = await api.get()` 直接获取数据
- ✅ 统一错误处理：拦截器统一处理 401、500 等错误
- ✅ 减少冗余：不需要每次都写 `response.data`

---

### 2. 为什么使用全局 api 实例？

**优点**：

- ✅ 统一拦截器逻辑（Token、国家代码、缓存控制）
- ✅ 避免重复代码
- ✅ 更容易维护和调试
- ✅ 保证所有请求都添加国家代码

**独立实例的问题**：

- ❌ 需要重复实现拦截器
- ❌ 容易出现不一致
- ❌ 难以维护
- ❌ 可能遗漏某些请求头

### 2. 国家代码流转链路

```
前端选择国家
  ↓
localStorage.setItem('logix_scoped_country_code', 'IT')
  ↓
Pinia store: scopedCountryCode = 'IT'
  ↓
api.ts 拦截器：config.headers['X-Country-Code'] = 'IT'
  ↓
HTTP 请求头：X-Country-Code: IT
  ↓
后端 scopedCountryMiddleware 读取请求头
  ↓
runWithScope({ countryCode: 'IT' }, next)
  ↓
AsyncLocalStorage 存储国家代码
  ↓
LastReturnStatisticsService.getDistribution()
  ↓
getScopedCountryCode() = 'IT'
  ↓
ContainerQueryBuilder.addCountryFilters(query)
  ↓
LEFT JOIN biz_customers cust ON ...
WHERE cust.country = 'IT'
  ↓
只返回意大利的货柜 ✅
```

### 3. AsyncLocalStorage 原理

后端使用 Node.js 的 `AsyncLocalStorage` 实现请求作用域：

```typescript
// requestContext.ts
import { AsyncLocalStorage } from "async_hooks";

const requestStorage = new AsyncLocalStorage<RequestScope>();

export function runWithScope<T>(scope: RequestScope, fn: () => T): T {
  return requestStorage.run(scope, fn);
}

export function getScopedCountryCode(): string | undefined {
  const scope = requestStorage.getStore();
  return scope?.countryCode;
}
```

**工作原理**：

- `runWithScope` 创建一个异步作用域
- 在该作用域内的所有异步操作都能访问 `scope`
- 类似线程本地存储（Thread-Local Storage）

## 总结

### 问题根源

1. **原始问题**：`container.ts` 使用独立 axios 实例，拦截器可能未正确工作，导致 `X-Country-Code` 请求头缺失
2. **修复引入的新问题**：响应拦截器已返回 `response.data`，但 `container.ts` 中再次访问 `.data`，导致双重解包

### 修复方案

1. 使用全局 `api.ts` 实例
2. 删除自定义拦截器
3. **修正所有方法的返回值**：从 `return response.data` 改为 `return response`

### 修复效果

- ✅ 国家代码正确传递
- ✅ 统计和查询正确应用国家过滤
- ✅ API 返回值正确，不会出现 undefined

### 代码改进

- 删除 46 行自定义 axios 实例代码
- 新增 4 行导入全局 api
- 修改 18 个方法调用
- **修正 22 处返回值（从 `.data` 改为直接返回）**
