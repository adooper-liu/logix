# 排产预览模式与正式模式切换指南

## 📌 核心概念

排产可视化页面有两种显示模式：

### 1. **预览模式**（`isPreviewMode = true`）

- **用途**：查看排产方案，但不保存到数据库
- **数据来源**：`previewResults`（内存中的预览结果）
- **特征**：
  - 卡片标题显示"排产预览"
  - 显示警告标签"⚠️ 预览模式，未保存"
  - 显示"确认保存"、"放弃"按钮
  - 表格支持多选（可以选择要保存的方案）
  - 不显示"导出"和"甘特图"按钮
  - 数据不会持久化，刷新后消失

### 2. **正式模式**（`isPreviewMode = false`）

- **用途**：查看已保存的排产结果
- **数据来源**：`scheduleResult`（从后端加载的已保存结果）
- **特征**：
  - 卡片标题显示"排产结果"
  - 无警告标签
  - 显示"导出"和"甘特图"按钮
  - 表格不支持多选（数据已保存）
  - 数据已持久化到数据库

---

## 🔄 模式切换流程

### 从 **正式模式** → **预览模式**

**触发操作**：点击"排产"按钮（`handlePreviewSchedule`）

**切换逻辑**（第 1889 行）：

```typescript
// ✅ 直接显示预览结果，不再弹出对话框
isPreviewMode.value = true;
scheduleResult.value = null; // 清空正式结果
```

**完整流程**：

1. 用户点击"排产"按钮
2. 调用 `executeSchedulingFlow({ dryRun: true })`
3. 后端返回预览结果（不保存到数据库）
4. 前端转换数据格式（第 1818-1886 行）
5. 设置 `isPreviewMode.value = true`
6. 清空 `scheduleResult.value`
7. 将结果存入 `previewResults.value`
8. 自动滚动到结果区域
9. 预加载档期数据

---

### 从 **预览模式** → **正式模式**

**触发操作**：点击"确认保存"按钮（`handleConfirmSave`）

**切换逻辑**（第 1981 行）：

```typescript
// 清空预览状态
isPreviewMode.value = false;
previewResults.value = [];
selectedPreviewContainers.value = [];
```

**完整流程**：

1. 用户在预览结果中勾选要保存的方案
2. 点击"确认保存"按钮
3. 调用 `containerService.confirmSchedule({ previewResults })`
4. 后端将预览结果保存到数据库
5. 设置 `scheduleResult.value` 为已保存的结果
6. 设置 `isPreviewMode.value = false`
7. 清空 `previewResults.value` 和选中项
8. 刷新档期数据
9. 重新加载概览数据（更新待排产数量）

---

### 从 **预览模式** → **正式模式**（放弃）

**触发操作**：点击"放弃"按钮（`handleDiscardPreview`）

**切换逻辑**（第 2012 行）：

```typescript
isPreviewMode.value = false;
previewResults.value = [];
selectedPreviewContainers.value = [];
```

**完整流程**：

1. 用户点击"放弃"按钮
2. 清空所有预览数据
3. 回到初始状态（无数据）
4. 需要重新点击"排产"或从数据库加载

| 状态                        | 预览模式       | 正式模式   |
| --------------------------- | -------------- | ---------- |
| `isPreviewMode`             | `true`         | `false`    |
| `previewResults`            | 有数据         | 空数组     |

| `selectedPreviewContainers` | 用户勾选的柜号 | 空数组     |

| 警告标签                    | 显示           | 不显示     |
| 表格多选                    | 支持           | 不支持     |
| 确认保存按钮                | 显示           | 不显示     |
| 放弃按钮                    | 显示           | 不显示     |
| 导出按钮                    | 不显示         | 显示       |
| 甘特图按钮                  | 不显示         | 显示       |

## 🎯 用户操作路径

### 路径 1：预览 → 保存 → 正式

```
初始状态（正式模式，无数据）
  ↓ 点击"排产"
预览模式（显示预览结果）
  ↓ 勾选方案 → 点击"确认保存"
正式模式（显示已保存结果）
```

### 路径 2：预览 → 放弃 → 初始

```
初始状态（正式模式，无数据）
  ↓ 点击"排产"
预览模式（显示预览结果）
  ↓ 点击"放弃"
初始状态（正式模式，无数据）
```

### 路径 3：直接加载已保存数据

```
访问页面（从数据库加载）
  ↓ 有已保存数据
正式模式（显示已保存结果）
```

## 🔍 数据流向

### 预览模式数据流

```
用户点击"排产"
  ↓
前端调用 API（dryRun: true）
  ↓
后端计算方案（不保存）
  ↓
返回预览结果
  ↓
前端存入 previewResults
  ↓
设置 isPreviewMode = true
  ↓
UI 显示预览结果（临时显示）
```

### 正式模式数据流

```
用户点击"确认保存"
  ↓
前端调用 API（传递 previewResults）
  ↓
后端保存到数据库
  ↓
返回保存结果
  ↓
前端存入 scheduleResult
  ↓
设置 isPreviewMode = false
  ↓
UI 显示已保存结果（持久化）
```

## ⚠️ 注意事项

### 1. **预览数据是临时的**

- 预览模式下的数据只存在于内存中
- 刷新页面或切换路由会丢失
- 必须点击"确认保存"才能持久化

### 2. **正式模式需要刷新数据**

- 保存成功后会自动调用 `loadOverview()`
- 更新待排产数量统计
- 刷新档期数据缓存

### 3. **模式切换是互斥的**

- 同一时间只能是预览模式或正式模式
- 切换时会清空另一种模式的数据
- `isPreviewMode` 是唯一的判断标准

### 4. **甘特图按钮只在正式模式显示**

- 预览模式下不能查看甘特图
- 必须先保存排产结果
- 确保甘特图显示的是已保存的数据

## 📝 开发建议

### 添加新模式相关功能时

1. **判断当前模式**：

   ```typescript
   if (isPreviewMode.value) {
     // 预览模式逻辑
   } else {
     // 正式模式逻辑
   }
   ```

2. **访问数据源**：

   ```typescript
   const data = isPreviewMode.value ? previewResults.value : scheduleResult.value?.results;
   ```

3. **切换模式时清理数据**：

   ```typescript
   // 从预览切换到正式
   isPreviewMode.value = false;
   previewResults.value = [];
   selectedPreviewContainers.value = [];

   // 从正式切换到预览
   isPreviewMode.value = true;
   scheduleResult.value = null;
   ```

## 🎨 UI 设计意图

### 预览模式的设计理念

- **临时性**：明确告知用户这是预览，未保存
- **选择性**：允许用户选择满意的方案保存
- **可放弃**：可以放弃不满意的方案
- **审查性**：提供详细的数据和费用信息供审查

### 正式模式的设计理念

- **持久化**：数据已保存到数据库
- **可操作**：可以导出、查看甘特图
- **稳定性**：数据不会因刷新而丢失
- **可追溯**：显示的是历史已保存记录
