### 3. 查询层：从请求上下文获取国家代码

**文件**：`backend/src/services/statistics/common/DateFilterBuilder.ts`

```typescript
static addCountryFilters(
  query: SelectQueryBuilder<any>,
  countryCode?: string
): SelectQueryBuilder<any> {
  // 优先使用传入的 countryCode，否则从请求上下文获取
  const raw =
    (countryCode !== undefined && countryCode !== null
      ? String(countryCode).trim()
      : getScopedCountryCode()) || '';

  const code = normalizeCountryCode(raw);
  if (!code) return query; // 无国家代码时不过滤

  // 添加客户表连接和国家过滤条件
  query.leftJoin('biz_customers', 'cust', TYPEORM_ORDER_CUSTOMER_JOIN_ON);
  query.andWhere('cust.country = :countryCode', { countryCode: code });
  return query;
}
```

**获取方法**：`backend/src/utils/requestContext.ts`

```typescript
export function getScopedCountryCode(): string | undefined {
  const scope = requestStorage.getStore(); // 从 AsyncLocalStorage 获取
  return scope?.countryCode;
}
```

## 三、国家过滤逻辑

### 1. 过滤条件

```sql
-- 连接客户表
LEFT JOIN biz_customers cust ON (
  cust.customer_code = order.customer_code
  OR (cust.customer_code IS NULL AND sf.sell_to_country_code = cust.code)
)
-- 国家过滤
WHERE cust.country = :countryCode
```

### 2. 过滤效果

- **有国家代码时**：只显示该国家的货柜（通过客户表关联）
- **无国家代码时**：显示所有货柜（不过滤）
- **无备货单货柜**：不显示（当有国家过滤时）

### ❓ 问题 2：选择国家后显示 0 条数据？

**可能原因**：

- 该国家确实没有符合条件的货柜
- 客户表国家字段未正确配置
- 备货单客户关联缺失

**排查方法**：

```sql
-- 检查该国家的货柜数量
SELECT COUNT(DISTINCT c.container_number) as count
FROM biz_containers c
INNER JOIN biz_replenishment_orders ro ON c.container_number = ro.container_number
INNER JOIN biz_customers cust ON ro.customer_code = cust.customer_code
WHERE cust.country = 'CA'
  AND c.logistics_status IN ('PICKED_UP', 'UNLOADED');

-- 检查客户表国家分布
SELECT cust.country, COUNT(DISTINCT c.container_number) as count
FROM biz_containers c
INNER JOIN biz_replenishment_orders ro ON c.container_number = ro.container_number
INNER JOIN biz_customers cust ON ro.customer_code = cust.customer_code
GROUP BY cust.country;
```
