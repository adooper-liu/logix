## 🚀 P0级（立即修复 - 本周内）

### 任务1: 修复日期解析函数
**优先级**: 🔴 **最高 - 影响40%导入**

**问题**: `parseDate`只解析日期，忽略时间部分
```typescript
// 错误: "2025/7/18 1:37" → 2025-07-18 00:00:00 (丢失01:37)
// 正确: "2025/7/18 1:37" → 2025-07-18 01:37:00
```

**代码位置**: `feituoImport.service.ts:105-116`

**修复时间**: 30分钟

**影响范围**:
- 所有时间字段（ATA/ETA/ETD/ATD）
- 滞港费计算（时间精度丢失）
- 倒计时显示（小时部分丢失）

**测试验证**:
```typescript
// 测试用例
expect(parseDate("2025/7/18 1:37")).toBe("2025-07-18T01:37:00");
expect(parseDate("2025/7/18")).toBe("2025-07-18T00:00:00");
expect(parseDate("2025-07-18T01:37:00Z")).toBe("2025-07-18T01:37:00");
```

### 任务3: 优化状态码映射
**优先级**: 🔴 **最高 - 影响20%业务逻辑**

**问题**: BDAR/POCA/DSCH都映射到`ata_dest_port`，导致时间被覆盖
```typescript
// 错误:
BDAR(抵港) 2025/9/3 18:00 → ata_dest_port
POCA(靠泊) 2025/9/4 05:04 → ata_dest_port (覆盖!)

// 正确:
BDAR(抵港) 2025/9/3 18:00 → ata_dest_port
POCA(靠泊) 2025/9/4 05:04 → berthing_time
DSCH(卸船) 2025/9/8 15:04 → dest_port_unload_date
```

**代码位置**: 
- `FeiTuoStatusMapping.ts:22-23`
- `feituoImport.service.ts:645-665`

**修复时间**: 45分钟

**影响范围**:
- ATA时间准确性（抵港vs靠泊）
- 滞港费计算（免费期起算日错误）
- 状态机判断（是否到港）

**解决方案**:
1. 新增字段`berthing_time`到`port_operation`表
2. 修改映射: 'POCA' → 'berthing_time'
3. 增加逻辑: 不覆盖更早的时间

**测试验证**:
```typescript
// 测试用例
// 输入: BDAR=2025-09-03 18:00, POCA=2025-09-04 05:04
// 期望: ataDestPort=2025-09-03 18:00, berthingTime=2025-09-04 05:04
```

### 任务4: 增加字段长度验证
**优先级**: 🟡 **中 - 影响15%导入**

**问题**: VARCHAR字段超长导致数据库错误
```sql
-- 错误:
container_number VARCHAR(50) 
插入: "KOCU4312744-EXTRALONG-SUFFIX" (60字符) → ERROR

-- 正确:
插入: "KOCU4312744-EXTRALONG-SUFFIX" (60字符) → 截断为50字符 + WARNING
```

**代码位置**: 所有实体字段赋值处

**修复时间**: 30分钟

**影响字段**:
- container_number (50)
- mbl_number (50)
- port_code (50)
- status_code (20)

**解决方案**:
```typescript
function validateAndTruncate(val: string, maxLength: number): string {
  if (val.length > maxLength) {
    logger.warn(`Value exceeds max length ${maxLength}, truncated: ${val}`);
    return val.substring(0, maxLength);
  }
  return val;
}
```

### 任务6: 增加时间顺序验证
**优先级**: 🟡 **中 - 影响数据质量**

**问题**: 不验证业务时间顺序（提柜>到港>卸船>还箱）

**代码位置**: 新增验证方法

**修复时间**: 1小时

**验证规则**:
```
提柜时间 ≥ 到港时间     ⚠️ 警告（提前提柜）
卸船时间 ≥ 到港时间     ✅ 正常
还箱时间 ≥ 提柜时间     ✅ 正常
```

**实现**:
```typescript
private validateTimeSequence(containerNumber: string, events: TimeEvent[]): string[] {
  const errors: string[] = [];
  
  if (pickup < arrival) {
    errors.push(`提柜时间(${pickup})早于到港时间(${arrival})`);
  }
  
  return errors;
}
```
