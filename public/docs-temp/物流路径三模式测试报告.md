  const statusCodes = events.map((e) => e.status_code).filter(Boolean);

  // 优先级：海铁联运 > 驳船联运 > 纯海运
  if (hasRailEvents) return TransportMode.SEA_RAIL;

## 常见问题

### Q1: 为什么没有铁路/驳船测试数据？

**A**: 当前数据库主要包含纯海运货柜数据。需要导入特殊运输模式的货柜：

- 海铁联运：美加线货柜（经美国/加拿大港口后铁路转运）
- 驳船联运：支线港口货柜（经驳船集疏到枢纽港）

### Q2: 如何快速生成测试数据？

**A**: 可以手动插入测试事件：

```sql
-- 海铁联运测试数据
INSERT INTO ext_container_status_events
  (container_number, status_code, occurred_at, description)
VALUES
  ('HMMU6232154', 'IRLB', NOW(), '铁路装车'),
  ('HMMU6232154', 'IRDP', NOW() + INTERVAL '2 hours', '铁路运输'),
  ('HMMU6232154', 'IRAR', NOW() + INTERVAL '1 day', '铁路到达'),
  ('HMMU6232154', 'IRDS', NOW() + INTERVAL '1 hour', '铁路卸箱');
```

### Q3: 模式识别会误判吗？

**A**: 理论上不会。识别逻辑基于实际发生的事件：

- 只有铁路事件 → SEA_RAIL
- 只有驳船事件 → FEEDER
- 都没有 → STANDARD
- 都有 → 优先判为 SEA_RAIL（符合业务优先级）
