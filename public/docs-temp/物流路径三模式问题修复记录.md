## 问题描述

根据用户反馈的界面截图，发现以下问题：

1. **阶段映射错误**：
   - POCA（靠泊）显示在"到港"阶段 ❌ 应该显示在"到港"阶段 ✅
   - FETA（交货地抵达）显示在"提柜"阶段 ❌ 应该显示在"提柜"阶段 ✅

2. **预计标签未显示**：界面上没有看到黄色的"预计"标签

3. **运输模式标识未显示**：顶部没有显示"纯海运"标签

4. **铁路/驳船段样式缺失**：没有看到特殊的颜色和图标

---

## 问题根因分析

### 1. POCA 和 FETA 状态码映射错误

**后端映射错误**：
```typescript
// ❌ 错误映射
'POCA': StandardStatus.AVAILABLE,  // 可提
'FETA': StandardStatus.DELIVERY_ARRIVED,  // 交货地抵达

// ✅ 正确映射（根据数据库实际定义）
'POCA': StandardStatus.BERTHED,  // 靠泊
'FETA': StandardStatus.DELIVERY_ARRIVED,  // 交货地抵达
```

**前端 STAGE_MAP 缺失注释**：
```typescript
// 需要添加注释说明
BERTHED: { stage: 'arrival', label: '到港', order: 6 },  // POCA 靠泊
DELIVERY_ARRIVED: { stage: 'pickup', label: '提柜', order: 10 },  // FETA 交货地抵达
```

### 2. 预计标签未显示

**可能原因**：
1. 后端 `is_estimated` 字段未正确设置
2. 前端 `isEstimatedNode` 函数判断逻辑错误
3. 模板条件 `v-if="isEstimatedNode(item.node)"` 不成立

**验证步骤**：
```sql
-- 检查数据库中 is_estimated 字段
SELECT event_code, description_cn, is_estimated 
FROM ext_container_status_events 
WHERE container_number = 'HMMU6232153'
ORDER BY event_time;
```

### 3. 运输模式标识未显示

**可能原因**：
1. 后端 `transportMode` 未正确返回
2. 前端 `path.transportMode` 为 undefined
3. 模板条件 `v-if="path.transportMode"` 不成立

**验证步骤**：
```typescript
// 检查 API 返回值
const path = await LogisticsPathService.getStatusPath(containerNumber);
console.log('transportMode:', path.transportMode);
```

### 4. 铁路/驳船段样式缺失

**原因**：当前测试货柜是纯海运模式，没有铁路/驳船事件

**验证**：
- 纯海运货柜：不显示铁路/驳船段 ✅
- 海铁联运货柜：应显示铁路段（蓝色边框 + 火车图标）⏳ 待测试
- 驳船联运货柜：应显示驳船段（绿色边框 + 小船图标）⏳ 待测试

## 待实施的修复

### 修复 4: 预计标签数据源

**如果数据库 `is_estimated` 字段为空**：

需要在后端设置预计状态：

```typescript
// logistics-path-system/backend/src/services/statusPathFromDb.ts
function eventToNode(e: DbEvent): StatusNode {
  return {
    // ... 其他字段
    rawData: {
      eventCode: e.status_code,
      eventId: e.id,
      dataSource: e.data_source,
      isEstimated: e.is_estimated ?? false,  // 确保有默认值
      ...(e.raw_data || {}),
    },
  };
}
```

### 修复 5: 运输模式识别逻辑

**如果需要手动指定运输模式**：

可以在后端添加手动覆盖逻辑：

```typescript
function identifyTransportMode(
  events: DbEvent[], 
  containerNumber: string
): TransportMode {
  // 1. 优先从配置表读取
  const modeFromConfig = await getTransportModeFromConfig(containerNumber);
  if (modeFromConfig) return modeFromConfig;
  
  // 2. 基于事件识别
  const statusCodes = events.map((e) => e.status_code).filter(Boolean);
  
  // 3. 检查铁路/驳船事件
  const hasRailEvents = statusCodes.some((code) => 
    ["IRLB", "IRDP", "IRAR", "IRDS"].includes(code)
  );
  
  const hasFeederEvents = statusCodes.some((code) => 
    ["FDLB", "FDDP", "FDBA", "FDDC"].includes(code)
  );
  
  // 4. 返回识别结果
  if (hasRailEvents) return TransportMode.SEA_RAIL;
  if (hasFeederEvents) return TransportMode.FEEDER;
  return TransportMode.STANDARD;
}
```
