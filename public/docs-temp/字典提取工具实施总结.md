## 提取能力

### 船公司提取
- **识别列名**："船公司", "船公司名称", "shipping company"
- **标准化**：自动提取代码（如"CMA CGM" → "CMA"）
- **已知列表**：14家主要船公司（CMA, YML, WHL, MSK, MSC, HPL, HMM, ONE, EMC, COSCO, OOCL, ZIM, PIL, APL）

### 港口提取
- **识别列名**："目的港", "目的港名称", "port of discharge"
- **标准化**：识别标准代码格式（如CNSHA, FRLEH）
- **已知列表**：25+主要港口（中美欧主要港口）

### 状态码提取
- **识别列名**："状态码", "status code", "事件代码"
- **标准化**：转为大写
- **已知列表**：20+状态码（RCVE, STSP, GITM, LOBD, DLPT, BDAR, POCA, DSCH, STCS, FDDP, FDLB, FDBA等）

### 运输模式提取
- **识别列名**："运输方式", "transport mode", "运输模式"
- **标准化**：映射常见变体（如"海运" → "大船"）
- **支持模式**：大船、驳船、卡车、铁路、多式联运

## 验证功能

### 映射完整性验证
验证提取的数据与系统已知映射的匹配情况：

1. **状态码验证**（严格匹配）
   - 显示已映射和未映射的状态码
   - 未映射状态码需要更新配置

2. **船公司验证**（宽松匹配）
   - 显示已映射和未映射的船公司
   - 未映射显示警告

3. **港口验证**（宽松匹配）
   - 显示已映射和未映射的港口
   - 未映射显示信息提示

### 验证报告
- Markdown格式导出
- 包含完整验证结果
- 提供未映射项目的处理建议

## 导出功能

### CSV导出
- 格式：数据类型,值,数量
- 用途：数据分析、备份、共享
- 包含所有提取的字典数据

### SQL导出
- 自动生成INSERT语句
- 包含4个字典表：
  - `dict_shipping_companies` 船公司字典
  - `dict_ports` 港口字典
  - `dict_status_codes` 状态码字典
  - `dict_transport_modes` 运输模式字典
- 用途：直接初始化数据库字典表

### 验证报告导出
- Markdown格式
- 包含完整的映射验证结果
- 提供处理建议

## 多式联运场景支持

### 测试场景覆盖

#### 1. 驳船+大船联运
- 驳船状态码：STSP, FDLB, FDBA
- 大船状态码：LOBD, DLPT, BDAR
- 港口中转：CNSHA → FRLEH

#### 2. 铁路+卡车联运
- 铁路状态码：RAIL, GTIB
- 卡车状态码：GTOB, GITM, TCHR
- 内陆运输：DEHAM → 仓库

#### 3. 状态机映射验证
- 验证飞驼状态码到系统状态机的映射
- 确保多式联运下的状态转换正确
- 识别缺失的状态码映射

## 使用方法

### 访问方式

**方式1**: 直接访问
```
http://localhost:5173/#/import/dictionary-extractor
```

**方式2**: 系统菜单
- 导航到"数据导入"
- 点击"字典数据提取"

### 操作流程

1. **上传Excel**
   - 准备飞驼数据Excel文件
   - 拖拽或选择文件上传

2. **提取数据**
   - 点击"提取字典数据"按钮
   - 等待解析和提取完成

3. **验证映射**
   - 点击"验证状态码映射"按钮
   - 查看验证结果

4. **导出数据**
   - 选择导出格式（CSV/SQL/报告）
   - 下载文件并使用
