### 2. 甘特图拖拽策略矩阵实现

**业务背景**：
- Live load 模式：提柜=送仓=卸柜（硬约束）
- Drop off 模式：提柜≤送仓=卸柜（软约束）

**实现位置**：`frontend/src/components/common/gantt/useGanttLogic.ts`

#### 2.2 handleDrop 函数增强

**核心逻辑**：
1. 获取货柜的卸柜模式（从 trucking 或 warehouse）
2. 判断是否为 Live load 模式
3. 根据模式和拖拽方向决定同步行为：
   - **Live load + 任意方向**：强制同步送柜日和卸柜日
   - **Drop off + 向前拖拽**：标记需要询问用户
   - **Drop off + 向后拖拽 + 日期冲突**：自动同步

**代码示例**：
```typescript
if (field === 'plannedPickupDate') {
  const currentDate = trucking?.plannedPickupDate 
    ? dayjs(trucking.plannedPickupDate).format('YYYY-MM-DD')
    : null
  const isForwardDrag = currentDate && new Date(newDate) < new Date(currentDate)
  
  if (isLiveLoad) {
    // Live load 模式：强制同步
    updateData['plannedDeliveryDate'] = newDate
    updateData['plannedUnloadDate'] = newDate
  } else if (isForwardDrag) {
    // Drop off 向前拖拽：等待用户确认
    console.log('[handleDrop] Drop off 模式向前拖拽，将在 dragend 时询问用户')
  } else {
    // Drop off 向后拖拽：检查是否需要自动同步
    const deliveryDate = trucking?.plannedDeliveryDate
      ? dayjs(trucking.plannedDeliveryDate).format('YYYY-MM-DD')
      : null
    
    if (deliveryDate && new Date(newDate) > new Date(deliveryDate)) {
      updateData['plannedDeliveryDate'] = newDate
      updateData['plannedUnloadDate'] = newDate
    }
  }
}
```

#### 2.3 handleDragEnd 函数重构

**核心逻辑**：
1. 解构 pending 数据，包含新增的字段
2. 判断是否需要二次确认（Drop off + 向前拖拽 + 无自动同步）
3. 如果需要确认，弹窗询问用户
4. 根据用户选择决定是否同步其他字段

**代码示例**：
```typescript
// 判断是否需要额外确认
if (!isLiveLoad && updateField === 'plannedPickupDate' && !extraUpdateData) {
  const isForwardDrag = currentDate && new Date(newDate) < new Date(currentDate)
  if (isForwardDrag) {
    needAdditionalConfirm = true
  }
}

// 如果需要额外确认
if (needAdditionalConfirm) {
  ElMessageBox.confirm(
    '当前为 Drop off 模式，向前拖拽提柜日可能导致送柜日/卸柜日晚于提柜日。是否同步调整？',
    '确认同步调整',
    {
      confirmButtonText: '同步调整',
      cancelButtonText: '仅调整提柜日',
      type: 'warning',
    }
  )
    .then(async () => {
      finalUpdateData['plannedDeliveryDate'] = newDate
      finalUpdateData['plannedUnloadDate'] = newDate
      await executeUpdateWithData(container, finalUpdateData)
    })
    .catch((err) => {
      if (err === 'cancel') {
        // 用户选择仅调整提柜日
        await executeUpdateWithData(container, finalUpdateData)
      }
    })
}
```

#### 2.4 新增 executeUpdateWithData 函数

**目的**：支持同时更新多个字段（提柜日、送柜日、卸柜日）

```typescript
const executeUpdateWithData = async (
  container: Container, 
  updateData: Record<string, string>
) => {
  try {
    console.log('[executeUpdateWithData] 更新数据:', updateData)
    
    const result = await containerService.updateSchedule(
      container.containerNumber, 
      updateData
    )

    if (result.success) {
      ElMessage.success({ message: '日期调整成功', duration: 2000 })
      await loadData()
    } else {
      handleUpdateError(result)
    }
  } catch (error: any) {
    handleUpdateException(error)
  }
}
```

## 测试验证

### 场景 1: Live load 模式拖拽提柜日
- **前置条件**：卸柜模式 = Live load，三个日期相同
- **操作**：拖拽提柜日到新日期
- **预期**：三个日期自动同步，无弹窗

### 场景 2: Drop off 模式向前拖拽
- **前置条件**：卸柜模式 = Drop off，提柜日 < 送柜日 = 卸柜日
- **操作**：向前拖拽提柜日
- **预期**：弹窗询问是否同步

### 场景 3: Drop off 向后拖拽（冲突）
- **前置条件**：卸柜模式 = Drop off，新提柜日 > 原送柜日
- **操作**：向后拖拽提柜日
- **预期**：自动同步，无弹窗

### 场景 4: Drop off 向后拖拽（无冲突）
- **前置条件**：卸柜模式 = Drop off，新提柜日 < 原送柜日
- **操作**：向后拖拽提柜日
- **预期**：仅更新提柜日，不同步其他字段

1. **卸柜模式获取优先级**：
   ```typescript

   ```

2. **拖拽方向判断**：
   ```typescript
   const isForwardDrag = new Date(newDate) < new Date(currentDate)
   ```

3. **日期冲突检测**：
   ```typescript
   if (deliveryDate && new Date(newDate) > new Date(deliveryDate)) {
     // 需要自动同步
   }
   ```

4. **双层 RAF 延迟弹窗**：
   - 避免首次点击被消费
   - 确保拖拽状态完全清理后再弹窗

## 注意事项

1. **后端校验保留**：原有的顺序约束校验（提柜日 ≤ 送柜日 ≤ 卸柜日）仍然有效
2. **状态更新**：日期变更后会自动触发货柜状态重新计算
3. **缓存清理**：更新成功后会清理相关缓存
4. **还箱日重算**：当前实现未包含还箱日重算逻辑，后续可根据需要添加

## 后续优化建议

1. **送柜日拖拽限制**：送柜日是派生字段，建议设置为只读，不允许直接拖拽
2. **还箱日自动重算**：当卸柜日变更后，自动重算还箱日（卸柜日 + N天）
3. **批量拖拽支持**：支持同时拖拽多个货柜的日期
4. **拖拽预览**：拖拽过程中实时显示将要更新的日期
