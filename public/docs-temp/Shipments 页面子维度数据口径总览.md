## 一、维度概述

**按状态维度**是基于货柜的物流状态（logistics_status）进行分类统计，反映货柜在7层流转过程中的实时分布状态。

### 1.1 统计范围

- **包含**: 全部货柜（不限状态、不限港口类型）
- **业务用途**: 监控货柜在全流程中的实时分布状态

### 1.2 状态流转图

```
┌─────────────┐
│  未出运    │ (not_shipped)
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  已出运    │ (shipped)
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   在途     │ (in_transit)
└──────┬──────┘
       │
       ▼
┌─────────────┐     ┌─────────────┐
│ 已到目的港 │     │ 已到中转港 │ (at_port + transit)
│ (at_port)  │     └─────────────┘
└──────┬──────┘           │
       │                  ▼
       ▼           ┌─────────────┐
┌─────────────┐    │   在途     │
│  已提柜    │    │(in_transit) │
│ (picked_up)│    └──────┬──────┘
└──────┬──────┘           │
       │                  ▼
       ▼           ┌─────────────┐
┌─────────────┐    │ 已到目的港 │
│  已卸柜    │    │ (at_port)  │
│ (unloaded)  │    └─────────────┘
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  已还箱    │ (returned_empty)
└─────────────┘
```

## 二、子维度口径详情

### 2.1 未出运 (not_shipped)

| 条件            | 要求        | 说明             |
| --------------- | ----------- | ---------------- |
| logisticsStatus | not_shipped | 货柜状态为未出运 |

**业务含义**: 货柜仍在备货仓库，尚未安排出运

**典型场景**:

- 备货单已创建，但还未装船
- 等待船期
- 海关清关未完成

---

### 2.2 已出运 (shipped)

| 条件            | 要求    | 说明                    |
| --------------- | ------- | ----------------------- |
| logisticsStatus | shipped | 货柜状态为已出运/已装船 |

**业务含义**: 货柜已装船，海运开始

**典型场景**:

- 货柜已装载到船舶上
- 提单已签发
- 海运阶段开始

**可流转至**:

- 在途 (in_transit)
- 已到目的港 (at_port)
- 已提柜 (picked_up) - 直达情况

---

### 2.3 在途 (in_transit)

| 条件            | 要求       | 说明           |
| --------------- | ---------- | -------------- |
| logisticsStatus | in_transit | 货柜状态为在途 |

**业务含义**: 货柜正在海上运输中

**典型场景**:

- 从起运港到目的港的航行中
- 可能会经过中转港停靠
- ETA日期已确定

**可流转至**:

- 已到目的港 (at_port)
- 已提柜 (picked_up) - 直达情况

---

### 2.4 已到目的港 (at_port)

| 条件            | 要求      | 说明             |
| --------------- | --------- | ---------------- |
| logisticsStatus | at_port   | 货柜状态为已到港 |
| currentPortType | ≠ transit | 不是中转港       |

**业务含义**: 货柜已到达最终目的港

**典型场景**:

- 船舶靠泊目的港
- ATA日期已记录
- 等待清关
- 等待提柜安排

**可流转至**:

- 已提柜 (picked_up)
- 已卸柜 (unloaded)

---

### 2.5 已到中转港 (arrived_at_transit)

| 条件            | 要求    | 说明                 |
| --------------- | ------- | -------------------- |
| currentPortType | transit | 当前港口类型为中转港 |

**业务含义**: 货柜已到达中转港，将在中转港转运

**典型场景**:

- 船舶靠泊中转港
- 等待转运船舶
- 可能会停留数天

**可流转至**:

- 在途 (in_transit) - 继续前往目的港
- 已到目的港 (at_port)

---

### 2.6 已提柜 (picked_up)

| 条件            | 要求      | 说明             |
| --------------- | --------- | ---------------- |
| logisticsStatus | picked_up | 货柜状态为已提柜 |

**业务含义**: 货柜已被拖车提离港口

**典型场景**:

- 拖车已完成提柜操作
- 货柜正在运往仓库/客户
- pickup_date已记录

**可流转至**:

- 已卸柜 (unloaded)
- 已还箱 (returned_empty) - 直达情况

---

### 2.7 已卸柜 (unloaded)

| 条件            | 要求     | 说明             |
| --------------- | -------- | ---------------- |
| logisticsStatus | unloaded | 货柜状态为已卸柜 |

**业务含义**: 货柜已在仓库/客户处完成卸货

**典型场景**:

- 货物已从集装箱中取出
- 空箱准备归还
- 卸货完成时间已记录

**可流转至**:

- 已还箱 (returned_empty)

---

### 2.8 已还箱 (returned_empty)

| 条件            | 要求           | 说明             |
| --------------- | -------------- | ---------------- |
| logisticsStatus | returned_empty | 货柜状态为已还箱 |

**业务含义**: 空箱已归还到指定地点

**典型场景**:

- 空箱已还到堆场
- return_time已记录
- 货柜流程结束

**流转终点**: 流程完成，无后续状态

## 三、状态分类汇总

### 3.1 按流程阶段分类

| 阶段         | 包含状态               | 说明          |
| ------------ | ---------------------- | ------------- |
| **出运前**   | 未出运                 | 备货阶段      |
| **海运阶段** | 已出运、在途           | 海上运输      |
| **到港阶段** | 已到目的港、已到中转港 | 港口作业      |
| **陆运阶段** | 已提柜                 | 拖车运输      |
| **还箱阶段** | 已卸柜、已还箱         | 仓库作业/还箱 |

### 3.2 按进度分类

| 进度       | 包含状态                                             | 说明       |
| ---------- | ---------------------------------------------------- | ---------- |
| **未开始** | 未出运                                               | 流程未启动 |
| **进行中** | 已出运、在途、已到目的港、已到中转港、已提柜、已卸柜 | 流程进行中 |
| **已完成** | 已还箱                                               | 流程已完成 |

### 3.3 按风险分类

| 风险级别   | 包含状态                       | 说明             |
| ---------- | ------------------------------ | ---------------- |
| **低风险** | 未出运、已还箱                 | 不产生滞港费     |
| **中风险** | 已出运、在途                   | 关注ETA          |
| **高风险** | 已到目的港、已到中转港、已提柜 | 关注免租期倒计时 |

## 四、特殊状态说明

### 4.1 at_port 状态的动态显示

`at_port` 状态会根据 `currentPortType` 动态显示：

| currentPortType | 显示文本   | 说明           |
| --------------- | ---------- | -------------- |
| destination     | 已到目的港 | 到达最终目的港 |
| transit         | 已到中转港 | 到达中转港     |
| null/其他       | 已到港     | 默认显示       |

### 4.2 状态流转约束

根据 `SimplifiedStatusTransitions`，合法的状态流转路径：

```
未出运 → 已出运
已出运 → 在途、已到目的港、已提柜
在途 → 已到目的港、已提柜
已到目的港 → 已提柜、已卸柜
已提柜 → 已卸柜、已还箱
已卸柜 → 已还箱
已还箱 → (无后续状态)
```

**非法流转示例**:

- ❌ 未出运 → 在途 (中间跳过已出运)
- ❌ 已提柜 → 未出运 (不能回退)
- ❌ 已还箱 → 已卸柜 (已完成的流程不能回退)

## 五、与其他维度的关系

### 5.1 与"按到港"维度的关系

| 子维度     | 按状态 | 按到港                   | 关系       |
| ---------- | ------ | ------------------------ | ---------- |
| 未出运     | -      | -                        | 无交集     |
| 已出运     | ✅     | -                        | 仅状态分类 |
| 在途       | ✅     | 已逾期/3日内/7日内/7日后 | 有交集     |
| 已到目的港 | ✅     | 今日/今日之前            | 有交集     |

### 5.2 与"按提柜"维度的关系

| 子维度     | 按状态 | 按计划提柜                         | 关系     |
| ---------- | ------ | ---------------------------------- | -------- |

| 已提柜     | ✅     | 今日实际提柜                       | 部分重合 |
| 已卸柜     | ✅     | -                                  | 无交集   |

| 子维度     | 按状态 | 最晚提柜                           | 关系     |
| ---------- | ------ | ---------------------------------- | -------- |

### 5.4 与"最晚还箱"维度的关系

| 子维度 | 按状态 | 最晚还箱                           | 关系     |
| ------ | ------ | ---------------------------------- | -------- |
| 已提柜 | ✅     | 已逾期/紧急/警告/正常/缺最后还箱日 | 部分重合 |
| 已卸柜 | ✅     | 已逾期/紧急/警告/正常/缺最后还箱日 | 部分重合 |

## 六、数据口径总结

| 子维度     | 条件                                                    | 业务含义       |
| ---------- | ------------------------------------------------------- | -------------- |
| 未出运     | logisticsStatus = not_shipped                           | 货柜在备货仓库 |
| 已出运     | logisticsStatus = shipped                               | 货柜已装船     |
| 在途       | logisticsStatus = in_transit                            | 货柜海上运输中 |
| 已到目的港 | logisticsStatus = at_port AND currentPortType ≠ transit | 货柜到达目的港 |
| 已到中转港 | currentPortType = transit                               | 货柜到达中转港 |
| 已提柜     | logisticsStatus = picked_up                             | 货柜被拖车提离 |
| 已卸柜     | logisticsStatus = unloaded                              | 货物已卸空     |
| 已还箱     | logisticsStatus = returned_empty                        | 空箱已归还     |

### 二、按计划提柜维度

用途：只是针对所有实际已到目的港且未完成实际提柜的货柜进行拖车安排，其他状态的货柜不考虑。应与按按状态维度的已到目港进行汇总进行对比一致。
子集范围：所有已到目的港、未提柜状态。
汇总= 待安排提柜 + 已安排提柜（逾期未提柜 + 今日计划提柜 + 7日后计划提柜+ 3天内计划提柜）

#### 1. 待安排提柜

| 条件           | 要求  | 说明                |
| -------------- | ----- | ------------------- |
| 已到目的港     | ✅ 是 | 有ATA且不是transit  |
| 无计划提柜日期 | ✅ 是 | 无plannedPickupDate |
| 无拖卡运输记录 | ✅ 是 | 还未安排拖车        |
| 无实际提柜日期 | ✅ 是 | pickupDate为空      |

#### 2. 已逾期提柜 修改为 逾期未提柜

| 条件            | 要求  | 说明                |
| --------------- | ----- | ------------------- |
| 有拖卡运输记录  | ✅ 是 | 有安排拖车          |
| 有计划提柜日期  | ✅ 是 | 有plannedPickupDate |
| 无实际提柜日期  | ✅ 是 | pickupDate为空      |
| 计划日期 < 今天 | ✅ 是 | 计划日期已过期      |

#### 3. 今日计划提柜

| 条件            | 要求  | 说明                |
| --------------- | ----- | ------------------- |
| 有拖卡运输记录  | ✅ 是 | 有安排拖车          |
| 有计划提柜日期  | ✅ 是 | 有plannedPickupDate |
| 无实际提柜日期  | ✅ 是 | pickupDate为空      |
| 计划日期 = 今天 | ✅ 是 | 今天计划提柜        |

#### 4. 今日实际提柜 删除这个维度

| 条件            | 要求  | 说明         |
| --------------- | ----- | ------------ |
| 有实际提柜日期  | ✅ 是 | 有pickupDate |
| 实际日期 = 今天 | ✅ 是 | 今天实际提柜 |

#### 5. 3天内计划提柜

| 条件                        | 要求  | 说明                |
| --------------------------- | ----- | ------------------- |
| 有拖卡运输记录              | ✅ 是 | 有安排拖车          |
| 有计划提柜日期              | ✅ 是 | 有plannedPickupDate |
| 无实际提柜日期              | ✅ 是 | pickupDate为空      |
| 计划日期 ∈ [今天, 今天+3天] | ✅ 是 | 3天内计划           |

#### 6. 7天内计划提柜

| 条件                            | 要求  | 说明                |
| ------------------------------- | ----- | ------------------- |
| 有拖卡运输记录                  | ✅ 是 | 有安排拖车          |
| 有计划提柜日期                  | ✅ 是 | 有plannedPickupDate |
| 无实际提柜日期                  | ✅ 是 | pickupDate为空      |
| 计划日期 ∈ (今天+3天, 今天+7天] | ✅ 是 | 3-7天内计划         |

### 三、按最晚提柜维度

用途：也针对所有实际已到目的港且未完成提柜的货柜进行拖车安排，其他状态的货柜不考虑。作为计划提柜维度的补充。

子集范围：所有已到目的港、未提柜状态。

| 情况     | 有计划日期                    | 有实际提柜日期 | 应纳入统计              |
| -------- | ----------------------------- | -------------- | ----------------------- |
| 基础条件 | 已到目的港（有ATA且≠transit） | -              | ✅ 必须                 |
| 1        | ❌ 无                         | ❌ 无          | ✅ 是（完全未安排）     |
| 2        | ✅ 有                         | ❌ 无          | ✅ 是（已安排但未执行） |
| 3        | ✅ 有                         | ✅ 有          | ❌ 否（已完成提柜）     |
| 4        | ❌ 无                         | ✅ 有          | ❌ 否（已完成提柜）     |

#### 1. 已逾期

| 条件                | 要求  | 说明                     |
| ------------------- | ----- | ------------------------ |
| 基础条件            | ✅ 是 | 满足基础条件子集中的数据 |
| 有lastFreeDate      | ✅ 是 | 有最后免费日             |
| lastFreeDate < 今天 | ✅ 是 | 已逾期                   |

#### 2. 紧急

| 条件                            | 要求  | 说明                     |
| ------------------------------- | ----- | ------------------------ |
| 基础条件                        | ✅ 是 | 满足基础条件子集中的数据 |
| 有lastFreeDate                  | ✅ 是 | 有最后免费日             |
| lastFreeDate ∈ [今天, 今天+3天] | ✅ 是 | 剩余≤3天                 |

#### 3. 警告

| 条件                                | 要求  | 说明                     |
| ----------------------------------- | ----- | ------------------------ |
| 基础条件                            | ✅ 是 | 满足基础条件子集中的数据 |
| 有lastFreeDate                      | ✅ 是 | 有最后免费日             |
| lastFreeDate ∈ (今天+3天, 今天+7天] | ✅ 是 | 剩余3-7天                |

#### 4. 正常

| 条件                    | 要求  | 说明                     |
