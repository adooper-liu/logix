## 问题清单

根据用户反馈，发现以下问题需要修复：

### ✅ 已修复

1. **POCA 状态码映射错误**
   - 问题：POCA 被映射到 `StandardStatus.AVAILABLE`（可提）
   - 根因：映射表定义错误
   - 修复：修改为 `StandardStatus.BERTHED`（靠泊）

2. **FETA 状态码映射注释缺失**
   - 问题：FETA 映射缺少注释说明
   - 根因：开发时未添加注释
   - 修复：添加注释 `// 交货地抵达`

3. **前端 STAGE_MAP 注释缺失**
   - 问题：BERTHED 和 DELIVERY_ARRIVED 缺少注释
   - 根因：未同步后端映射
   - 修复：添加注释 `// POCA 靠泊` 和 `// FETA 交货地抵达`

4. **飞驼同步代码 is_estimated 字段修正**
   - 问题：`ext_container_status_events` 实体中 `isEstimated` 为非持久化字段
   - 根因：数据库添加了字段，但实体未同步更新
   - 修复：将 `isEstimated` 改为持久化字段，添加 `@Column` 装饰器

### ⏳ 待修复

5. **运输模式标识未显示**
   - 问题：顶部没有显示“纯海运”标签
   - 可能原因：
     - API 未返回 `transportMode`
     - 前端未正确接收
   - 排查：需要检查 API 返回值

6. **铁路/驳船段样式缺失**
   - 问题：没有看到特殊的颜色和图标
   - 原因：当前测试货柜是纯海运模式
   - 方案：导入海铁联运和驳船联运测试数据

## 经验教训

### 1. 状态码映射必须基于数据库实际定义

**教训**: POCA 被错误映射到 AVAILABLE，实际应该是 BERTHED

**改进**:
- ✅ 映射前必须查看数据库实际数据
- ✅ 不能凭感觉或猜测定义映射关系
- ✅ 映射表必须有注释说明

### 2. 数据库字段必须先验证

**教训**: `is_estimated` 字段在代码中使用了，但数据库表和实体中没有

**改进**:
- ✅ 开发前必须先验证数据库表结构
- ✅ 使用 `fix-verification` 技能检查字段是否存在
- ✅ 代码和数据库必须同步更新
- ✅ **重要**: 修改数据库表后，必须同步更新 TypeORM 实体
  - 添加 `@Column` 装饰器
  - 指定正确的类型和默认值
  - 从非持久化字段列表中移除

### 3. STAGE_MAP 必须完整

**教训**: 前端 STAGE_MAP 缺少某些状态的映射

**改进**:
- ✅ STAGE_MAP 必须包含所有 StandardStatus 枚举值
- ✅ 新增状态码时必须同步更新 STAGE_MAP
- ✅ 添加注释说明状态码的来源和含义

### 4. 运输模式识别逻辑要健壮

**教训**: 仅基于事件代码识别运输模式可能不准确

**改进**:
- ✅ 优先从配置表读取运输模式
- ✅ 基于实际事件识别作为 fallback
- ✅ 支持手动覆盖运输模式
