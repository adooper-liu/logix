## 一、核心问题分析

### 问题1: ETA相关分组缺少状态筛选 ⚠️

**文档要求**:
> ETA分组的特殊筛选条件:只包含"已出运但未到港之后状态"的货柜(`shipped/in_transit/at_port`)

**后端当前实现**:
```typescript
// 所有ETA相关查询(overdue, within3Days, within7Days, over7Days)
.where('container.logisticsStatus IN (:...targetStatuses)', { targetStatuses })
// targetStatuses = ['shipped', 'in_transit', 'at_port', 'picked_up', 'unloaded']
```

**影响**:
- 已提柜(`picked_up`)的货柜被错误归类到"3日内预计到港"等ETA分组
- 违反业务逻辑:已提柜的货柜不需要用ETA预测到港时间

**修正方案**:
```typescript
// ETA分组只包含 shipped/in_transit/at_port 状态
const etaTargetStatuses = ['shipped', 'in_transit', 'at_port'];
.where('container.logisticsStatus IN (:...etaTargetStatuses)', { etaTargetStatuses })
```
