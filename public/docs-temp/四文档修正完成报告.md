#### 问题 1：API 路径不一致

**现象**: 文档写 `POST /calculate`，实际是 `GET /calculate/:containerNumber`

**根因**: 开发时未及时更新文档

**解决**: 
1. 审查所有路由文件
2. 逐一比对文档中的 API 端点
3. 更新为正确的路径和方法

**教训**: 文档必须与代码同步更新

#### 问题 2：Composables 命名不准确

**现象**: 文档写 `useContainers()`，实际是 `useShipmentsTable()`

**根因**: 重构后未更新文档

**解决**:
1. 读取实际 Vue 文件确认 imports
2. 逐一比对文档中的 Composables
3. 更新为正确的命名

**教训**: Composables 需要基于代码验证

#### 问题 3：组件不存在

**现象**: 文档中提到 `SchedulingControls.vue`，但实际不存在

**根因**: 设计变更或命名变更未同步

**解决**:
1. 使用 file_search 验证组件存在性
2. 标记不存在的组件
3. 删除或更新为实际组件名

**教训**: 组件列表需要实际验证
