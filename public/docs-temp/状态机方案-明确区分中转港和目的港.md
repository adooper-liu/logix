## 1. 状态机定义

### 1.1 状态流转图

```
未出运 (not_shipped)
  ↓ shipment_date 有值
在途 (in_transit)
  ↓ 已到中转港 (transit有ATA)
已到中转港 (at_transit_port)
  ↓ 已到目的港 (destination有ATA)
已到港 (at_port)
  ↓ pickup_date 有值
已提柜 (picked_up)
  ↓ WMS已确认（wms_status='WMS已完成' OR ebs_status='已入库' OR wms_confirm_date有值）
已卸柜 (unloaded)
  ↓ return_time 有值
已还箱 (returned_empty)
```

### 1.2 状态与对应表/字段

| 状态 | 表名 | 关键字段 | 判断条件 |
|------|------|----------|----------|
| not_shipped | - | - | 初始状态，无其他操作记录 |
| in_transit | process_sea_freight | shipment_date | 有出运记录，未到任何港口 |
| at_transit_port | process_port_operations | ata_dest_port (transit) | 已到中转港（transit有ATA） |
| at_port | process_port_operations | ata_dest_port (destination) | 已到目的港（destination有ATA） |
| picked_up | process_trucking_transport | pickup_date | 有提柜记录 |
| unloaded | process_warehouse_operations | wms_status, ebs_status, wms_confirm_date | WMS已确认 |
| returned_empty | process_empty_returns | return_time | 有还箱记录 |

### 1.3 特殊字段说明

| 字段 | 表名 | 业务含义 | 用途 |
|------|------|----------|------|
| last_free_date | process_port_operations | 最晚提柜日 | "按最晚提柜日"统计 |
| lastReturnDate | process_empty_returns | 最晚还箱日 | "按最晚还箱日"统计 |

### 1.4 中转港 vs 目的港 - 明确区分

**中转港 (at_transit_port)**:
- port_type = 'transit'
- 有实际到港时间 (ata_dest_port IS NOT NULL)
- 目的港无ATA (destination的ata_dest_port IS NULL)
- 表示货柜已到达中转港口，正在等待转运

**目的港 (at_port)**:
- port_type = 'destination'
- 有实际到港时间 (ata_dest_port IS NOT NULL)
- 表示货柜已到达最终目的港口

**关键区别**:
- `at_transit_port` - 途中状态，还在中转
- `at_port` - 到达状态，已到最终目的港

## 2. 状态判断逻辑

### 2.1 核心判断函数

```typescript
interface DateInfo {
  // 海运信息
  shipmentDate?: Date;

  // 港口操作
  hasTransit: boolean;           // 有中转港记录
  hasDestinationAta: boolean;    // 目的港有ATA
  lastFreeDate?: Date;           // 最晚提柜日

  // 拖卡运输
  pickupDate?: Date;

  // 仓库操作
  wmsStatus?: string;            // WMS状态
  ebsStatus?: string;            // EBS状态
  wmsConfirmDate?: Date;         // WMS确认日期
  warehouseArrivalDate?: Date;   // 仓库到达日期

  // 还箱操作
  lastReturnDate?: Date;         // 最晚还箱日
  returnTime?: Date;             // 还箱时间
}

async determineStatus(containerNumber: string): Promise<string> {
  const dates = await this.getDateInfo(containerNumber);

  // 1. 已还箱（优先级最高）
  if (dates.returnTime) return 'returned_empty';

  // 2. 已卸柜（WMS已确认）
  if (this.isWmsConfirmed(dates)) return 'unloaded';

  // 3. 已提柜
  if (dates.pickupDate) return 'picked_up';

  // 4. 已到目的港（destination有ATA，不包括中转港）
  if (dates.hasDestinationAta) return 'at_port';

  // 5. 已到中转港（transit有ATA，但destination无ATA）
  if (dates.hasTransit) return 'at_transit_port';

  // 6. 在途（有出运日期，但未到任何港口）
  if (dates.shipmentDate) return 'in_transit';

  // 7. 未出运
  return 'not_shipped';
}

/**
 * 判断是否WMS已确认（已卸柜）
 */
private isWmsConfirmed(dates: DateInfo): boolean {
  // 满足任一条件即可
  return dates.wmsStatus === 'WMS已完成' ||
         dates.ebsStatus === '已入库' ||
         dates.wmsConfirmDate !== null;
}
```

### 2.2 获取日期信息

```typescript
async getDateInfo(containerNumber: string): Promise<DateInfo> {
  const result = await this.containerRepository.query(`
    SELECT
      -- 海运信息
      sf.shipment_date as "shipmentDate",

      -- 港口操作（多港经停）
      CASE WHEN EXISTS (
        SELECT 1 FROM process_port_operations po2
        WHERE po2.container_number = c.container_number
          AND po2.port_type = 'transit'
      ) THEN 1 ELSE 0 END as "hasTransit",
      CASE WHEN EXISTS (
        SELECT 1 FROM process_port_operations po3
        WHERE po3.container_number = c.container_number
          AND po3.port_type = 'destination'
          AND po3.ata_dest_port IS NOT NULL
      ) THEN 1 ELSE 0 END as "hasDestinationAta",
      po.last_free_date as "lastFreeDate",

      -- 拖卡运输
      tt.pickup_date as "pickupDate",

      -- 仓库操作
      wo.wms_status as "wmsStatus",
      wo.ebs_status as "ebsStatus",
      wo.wms_confirm_date as "wmsConfirmDate",
      wo.warehouse_arrival_date as "warehouseArrivalDate",

      -- 还箱操作
      er.last_return_date as "lastReturnDate",
      er.return_time as "returnTime"
    FROM biz_containers c
    LEFT JOIN process_sea_freight sf ON c.container_number = sf.container_number
    LEFT JOIN process_port_operations po ON c.container_number = po.container_number AND po.port_type = 'destination'
    LEFT JOIN process_trucking_transport tt ON c.container_number = tt.container_number
    LEFT JOIN process_warehouse_operations wo ON c.container_number = wo.container_number
    LEFT JOIN process_empty_returns er ON c.container_number = er.container_number
    WHERE c.container_number = $1
    LIMIT 1
  `, [containerNumber]);

  return result[0] || {};
}
```

### 2.3 状态判断优先级总结

| 优先级 | 状态 | 判断条件 | 说明 |
|--------|------|----------|------|
| 1 (最高) | returned_empty | returnTime IS NOT NULL | 有还箱记录 |
| 2 | unloaded | WMS已确认 | 已卸柜入库 |
| 3 | picked_up | pickupDate IS NOT NULL | 已提柜 |
| 4 | at_port | hasDestinationAta = true | **已到目的港** |
| 5 | at_transit_port | hasTransit = true AND hasDestinationAta = false | **已到中转港** |
| 6 | in_transit | shipmentDate IS NOT NULL AND hasDestinationAta = false AND hasTransit = false | 在运输途中 |
| 7 (最低) | not_shipped | 以上条件都不满足 | 未出运 |

## 3. 数据修复方案

### 3.1 批量修复SQL

#### 修复: 更新所有货柜状态

```sql
-- 批量更新所有货柜状态（使用修正后的状态机规则）
UPDATE biz_containers c
SET logistics_status =
  CASE
    -- 1. 已还箱（优先级最高）
    WHEN EXISTS (
      SELECT 1 FROM process_empty_returns er
      WHERE er.container_number = c.container_number
      AND er.return_time IS NOT NULL
    ) THEN 'returned_empty'

    -- 2. 已卸柜
    WHEN EXISTS (
      SELECT 1 FROM process_warehouse_operations wo
      WHERE wo.container_number = c.container_number
      AND (
        wo.wms_status = 'WMS已完成'
        OR wo.ebs_status = '已入库'
        OR wo.wms_confirm_date IS NOT NULL
      )
    ) THEN 'unloaded'

    -- 3. 已提柜
    WHEN EXISTS (
      SELECT 1 FROM process_trucking_transport tt
      WHERE tt.container_number = c.container_number
      AND tt.pickup_date IS NOT NULL
    ) THEN 'picked_up'

    -- 4. 已到目的港（只有目的港有ATA，不包括中转港）
    WHEN EXISTS (
      SELECT 1 FROM process_port_operations po
      WHERE po.container_number = c.container_number
      AND po.port_type = 'destination'
      AND po.ata_dest_port IS NOT NULL
    ) THEN 'at_port'

    -- 5. 已到中转港（有中转港记录，但目的港无ATA）
    WHEN EXISTS (
      SELECT 1 FROM process_port_operations transit_po
      WHERE transit_po.container_number = c.container_number
      AND transit_po.port_type = 'transit'
      AND transit_po.ata_dest_port IS NOT NULL
    ) AND NOT EXISTS (
      SELECT 1 FROM process_port_operations dest_po
      WHERE dest_po.container_number = c.container_number
      AND dest_po.port_type = 'destination'
      AND dest_po.ata_dest_port IS NOT NULL
    ) THEN 'at_transit_port'

    -- 6. 在途（有出运日期，未到任何港口）
    WHEN EXISTS (
      SELECT 1 FROM process_sea_freight sf
      WHERE sf.container_number = c.container_number
      AND sf.shipment_date IS NOT NULL
    ) THEN 'in_transit'

    -- 7. 未出运
    ELSE 'not_shipped'
  END,
  updated_at = NOW();
```

## ❓ 常见问题

### Q1: 为什么需要 `at_transit_port` 状态？

**A**: 因为中转港和目的港的业务含义完全不同:
- 中转港: 货柜还在运输途中,只是停靠
- 目的港: 货柜已到达最终目的地,可以开始后续操作

如果都用 `at_port`,会导致:
- 统计不准确（"按到港"无法区分）
- 用户混淆（无法知道货柜在中转还是到达）
- 操作错误（在中转港时尝试提柜）

### Q2: 有中转港的货柜状态流转?

**A**: 状态流转: `in_transit` → `at_transit_port` → `at_port`

- 出运后: `in_transit`
- 到达中转港: `at_transit_port`
- 到达目的港: `at_port`
- 后续状态: `picked_up` → `unloaded` → `returned_empty`

### Q3: 多港经停场景如何处理?

**A**: 只要目的港(destination)有ATA,状态就是 `at_port`

- 不管经过多少个中转港
- 不管中转港是否有ATA
- 只看目的港的ATA

## 📝 总结

### 核心原则

1. **明确区分**: 中转港和目的港是两种完全不同的状态
2. **优先级清晰**: 从高到低逐级判断,不会误判
3. **简单直接**: 删除复杂的 `isAtPort` 方法,逻辑更清晰
4. **易于理解**: 7个状态,每个状态都有明确的业务含义

### 关键要点

1. **at_port**: 只表示已到目的港 (destination有ATA)
2. **at_transit_port**: 只表示已到中转港 (transit有ATA,destination无ATA)
3. **in_transit**: 表示在运输途中,未到任何港口
4. **优先级**: returned_empty > unloaded > picked_up > at_port > at_transit_port > in_transit > not_shipped

### 实施优先级

1. **高优先级**: 数据修复、ContainerStatusService实现
2. **中优先级**: 前端状态枚举更新、统计卡片更新
3. **低优先级**: 桑基图更新、状态流转可视化
