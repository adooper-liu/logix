# 国家过滤失效问题排查与修复

## 问题描述

**现象**：
- 用户选择国家：**意大利（IT）**
- 点击"按最晚还箱 - 紧急：倒计时 3 天内"卡片（显示 3 个货柜）
- 表格过滤出 **6 条记录**
- 其中 **3 条是英国费利克斯托** 的货柜

**问题**：国家过滤失效，显示了其他国家的货柜

---

## 根本原因

经过排查，发现前端 `container.ts` 服务中**没有正确应用国家过滤**：

### 问题 1：container.ts 使用独立的 axios 实例

**文件**：`frontend/src/services/container.ts`

```typescript
class ContainerService {
  constructor() {
    this.api = axios.create({
      baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:3001/api/v1',
      timeout: 120000,
    })
    
    // 虽然有拦截器，但可能存在时序问题
    this.api.interceptors.request.use(
      config => {
        const appStore = useAppStore()
        if (appStore.scopedCountryCode) {
          config.headers['X-Country-Code'] = appStore.scopedCountryCode
        }
        return config
      }
    )
  }
}
```

**问题**：
1. `container.ts` 创建了自己的 axios 实例，而不是使用 `api.ts` 的全局实例
2. 拦截器中调用 `useAppStore()` 可能存在时序问题
3. 没有日志记录，无法确认是否成功添加请求头

### 问题 2：后端统计服务可能未正确读取国家代码

虽然后端中间件 `scopedCountryMiddleware` 会读取 `X-Country-Code` 并写入 `AsyncLocalStorage`，但需要验证：
1. 前端请求是否正确发送了 `X-Country-Code` 头
2. 中间件是否在统计接口之前执行
3. `LastReturnStatisticsService` 是否正确调用了 `addCountryFilters`

查看日志输出：

### 方法 3：数据库查询验证

执行 SQL 验证该国家的货柜数量：

```sql
-- 检查意大利的"紧急：倒计时 3 天内"货柜数量
SELECT COUNT(DISTINCT c.container_number) as count
FROM biz_containers c
INNER JOIN biz_replenishment_orders ro ON c.container_number = ro.container_number
INNER JOIN biz_customers cust ON ro.customer_code = cust.customer_code
INNER JOIN process_trucking_transport tt ON c.container_number = tt.container_number
LEFT JOIN process_empty_return er ON c.container_number = er.container_number
WHERE cust.country = 'IT'
  AND c.logistics_status IN ('PICKED_UP', 'UNLOADED')
  AND tt.pickup_date IS NOT NULL
  AND (er.return_time IS NULL OR NOT EXISTS (
    SELECT 1 FROM process_empty_return er2 
    WHERE er2.container_number = c.container_number 
    AND er2.return_time IS NOT NULL
  ))
  AND er.last_return_date IS NOT NULL
  AND er.last_return_date >= CURRENT_DATE
  AND er.last_return_date < CURRENT_DATE + INTERVAL '3 days';
```

## 修复方案

### 方案 1：使用全局 api 实例（推荐）

**修改文件**：`frontend/src/services/container.ts`

```typescript
// 修改前
import axios from 'axios'

class ContainerService {
  private api: AxiosInstance
  
  constructor() {
    this.api = axios.create({
      baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:3001/api/v1',
      timeout: 120000,
    })
    // ... 拦截器
  }
}

// 修改后
import { api } from './api'  // ✅ 使用全局 api 实例

class ContainerService {
  // ❌ 删除自定义 api 实例
  // ❌ 删除拦截器
  
  async getStatisticsDetailed(startDate?: string, endDate?: string) {
    const params: any = {}
    if (startDate) params.startDate = startDate
    if (endDate) params.endDate = endDate
    
    // ✅ 使用全局 api，自动包含国家代码
    const response = await api.get('/containers/statistics-detailed', { params })
    return response.data
  }
}
```

**优点**：
- 统一使用 `api.ts` 的拦截器，保证所有请求都添加国家代码
- 避免重复代码
- 更容易维护

**缺点**：
- 需要修改所有方法调用

---

### 方案 2：修复独立 axios 实例的拦截器

如果必须使用独立实例，确保拦截器正确工作：

```typescript
class ContainerService {
  private api: AxiosInstance
  
  constructor() {
    this.api = axios.create({
      baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:3001/api/v1',
      timeout: 120000,
    })
    
    this.api.interceptors.request.use(
      config => {
        const token = localStorage.getItem('token')
        if (token) {
          config.headers.Authorization = `Bearer ${token}`
        }
        
        // ✅ 确保正确获取国家代码
        try {
          const appStore = useAppStore()
          if (appStore?.scopedCountryCode) {
            config.headers['X-Country-Code'] = appStore.scopedCountryCode
            console.log('[ContainerService] Adding country header:', appStore.scopedCountryCode)
          }
        } catch (error) {
          console.error('[ContainerService] Failed to get country code:', error)
        }
        
        return config
      },
      error => Promise.reject(error)
    )
  }
}
```

## 长期解决方案

### 1. 统一使用全局 api 实例

所有 service 都应该使用 `api.ts` 的全局实例，而不是自己创建：

```typescript
// ✅ 正确做法
import { api } from '@/services/api'

class ContainerService {
  async getContainers() {
    return api.get('/containers')
  }
}

// ❌ 错误做法
class ContainerService {
  private api = axios.create({...})  // 不要自己创建
}
```

### 2. 添加请求日志

在所有 service 的拦截器中添加日志：

```typescript
this.api.interceptors.request.use(config => {
  console.log('[API Request]', {
    url: config.url,
    headers: config.headers,
    country: config.headers['X-Country-Code']
  })
  return config
})
```

### 3. 添加后端验证

在所有统计接口添加国家代码验证：

```typescript
const countryCode = getScopedCountryCode();
if (!countryCode) {
  logger.warn('[Statistics] No country code in request');
}
```
