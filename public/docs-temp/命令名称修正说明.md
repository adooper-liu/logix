## 📋 修正内容

```bash

npm run quality              # validate + test

# 或快速检查（不含测试）

# CI 环境专用
```

## 🔍 验证依据

### 根目录 `package.json` 实际配置

```json
{
  "scripts": {
    "quality": "npm run validate && npm run test",
    "quality:ci": "npm run test && npm run type-check:backend && npm run lint:backend",
    "validate": "npm run type-check && npm run lint && npm run lint:naming",
    "test": "npm run test:backend && npm run test:frontend"
  }
}
```

**结论**: 根目录没有 `quality-check` 脚本，应使用 `quality` 或 `validate`。

## ✅ 修正原则

### 1. 唯一事实标准

所有文档中的命令必须与 `package.json` 实际配置一致，避免新人照抄失败。

### 2. 推荐优先级

```bash
# 首选：完整质量检查
npm run quality

# 次选：快速检查（开发中频繁使用）
npm run validate

# CI 环境：专用命令
npm run quality:ci
```

### 3. 分步执行选项

如需单独执行某项检查：

```bash
# 后端
cd backend && npm test                    # 单元测试
cd backend && npm run test:integration    # 集成测试

# 前端
cd frontend && npm test                   # 单元测试
cd frontend && npm run test:e2e           # E2E 测试
```
