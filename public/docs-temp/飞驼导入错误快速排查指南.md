# 飞驼数据导入错误快速排查指南

## 🚨 紧急错误（立即停止导入）

### 错误1: `缺少集装箱号`
```
表现: 导入失败，错误日志显示"缺少集装箱号"
原因: Excel中"集装箱号"列为空
解决:
  1. 检查Excel列名是否为"集装箱号"或"集装箱号（一）"
  2. 确保所有行都有集装箱号
  3. 重新导入
```

### 错误2: `duplicate key value violates unique constraint`
```
表现: 数据库主键冲突错误
原因: 同一集装箱号已存在
解决:
  1. 检查是否为重复导入
  2. 使用更新模式（UPSERT）而非插入模式
  3. 删除旧数据后重新导入
```

## ⚠️ 警告错误（继续导入，但需关注）

### 警告1: 日期解析失败
```
表现: 日期字段显示为NULL或1970-01-01
原因: 日期格式不正确
示例: "2025/7/18 1:37" 格式不受支持
解决:
  1. 检查日期格式是否为 YYYY/MM/DD HH:mm
  2. 或转换为 YYYY-MM-DD HH:mm:ss 格式
  3. 查看日志确认具体失败的字段
```

### 警告2: 字段被截断
```
表现: 日志显示"Value exceeds max length"
原因: 字段长度超过数据库限制
常见字段:
  - container_number: 最长50字符
  - mbl_number: 最长50字符
  - status_code: 最长20字符
解决:
  1. 缩短字段长度
  2. 或联系开发团队扩展数据库字段
```

### 警告3: 时间顺序异常
```
表现: 日志显示"时间顺序错误: 提柜时间早于到港时间"
原因: 飞驼数据源时间逻辑错误
示例:
  提柜时间: 2025/7/2 8:01
  到港时间: 2025/7/17 22:55
解决:
  1. 核对飞驼原始数据
  2. 手动修正错误时间
  3. 联系船公司确认实际时间
```

## 🔍 数据问题（导入成功，但数据异常）

### 问题1: 多港经停数据丢失
```
表现: 只显示起运港和目的港，缺少中转港
原因: 运输方式数组解析失败
检查:
  1. Excel中"运输方式"列格式是否正确
  2. 应为: ["驳船", "大船"] 或 "驳船,大船"
  3. 查看日志是否有"parseArrayField"错误
解决:
  1. 修正Excel格式
  2. 或转换为标准JSON数组格式
  3. 重新导入
```

### 问题2: ATA/ETA时间相同
```
表现: 实际到港时间和预计到港时间显示相同
原因: BDAR(抵港)和POCA(靠泊)都映射到ata_dest_port
示例:
  BDAR: 2025/9/3 18:00 (抵港)
  POCA: 2025/9/4 05:04 (靠泊，覆盖了抵港时间)
解决:
  1. 查看status_events表确认原始状态
  2. 手动修正ata_dest_port为正确的抵港时间
  3. 开发团队将优化映射逻辑
```

### 问题3: 甩柜标记未显示
```
表现: 柜号被甩，但系统中未标记
原因: "是否甩柜"字段未正确映射
检查:
  1. Excel中"是否甩柜"列值是否为 TRUE/FALSE
  2. 或是否为"是"/"否"
  3. 查看container.is_rolled字段
解决:
  1. 确认字段名正确("是否甩柜"或"是否用柜")
  2. 手动更新container.is_rolled = true
  3. 补充甩柜原因和时间
```

### 问题4: 费用信息未导入
```
表现: ext_container_charges表无数据
原因: 费用信息数组解析失败
检查:
  1. Excel中"费用类型"列格式: ["STORAGE", "DEMURRAGE"]
  2. "费用"列格式: ["150.00", "200.00"]
  3. 查看日志是否有"mergeChargeInfo"错误
解决:
  1. 统一数组格式
  2. 确保费用为数字格式
  3. 重新导入
```

## 📊 批量导入问题

### 问题: 导入速度慢/超时
```
表现: 导入1000条数据需要30分钟以上
原因: 逐条查询数据库，无批量优化
临时解决:
  1. 分批导入，每批<500条
  2. 避开高峰期
  3. 增加网络超时时间
长期解决:
  1. 开发团队优化批量导入逻辑
  2. 使用批量查询替代逐条查询
```

## 🛠️ 排查命令

### 查看导入日志
```bash
# 查看最近导入批次
cd backend
npm run logs | grep "FeituoImport"

# 查看具体批次错误
npm run logs | grep "batch_id=123"
```

### 数据库检查
```sql
-- 检查最近导入批次
SELECT * FROM ext_feituo_import_batch 
ORDER BY created_at DESC LIMIT 10;

-- 查看批次错误详情
SELECT id, error_details FROM ext_feituo_import_batch 
WHERE id = 123;

-- 检查特定柜号导入状态
SELECT * FROM ext_feituo_import_table1 
WHERE container_number = 'KOCU4312744';

-- 检查多港经停
SELECT port_type, port_name, port_sequence 
FROM process_port_operations 
WHERE container_number = 'KOCU4312744'
ORDER BY port_sequence;

-- 检查费用信息
SELECT charge_type, charge_amount, status 
FROM ext_container_charges 
WHERE container_number = 'KOCU4312744';
```

### 验证数据质量
```sql
-- 检查缺失MBL
create view temp_missing_mbl as
select container_number from biz_containers 
where bill_of_lading_number is null;

-- 检查时间异常
create view temp_time_anomaly as
select c.container_number, 
       po.ata_dest_port, 
       tt.pickup_date
from biz_containers c
join process_port_operations po on c.container_number = po.container_number
join process_trucking_transport tt on c.container_number = tt.container_number
where po.port_type = 'destination'
and tt.pickup_date < po.ata_dest_port;

-- 检查字段截断
select container_number, 
       length(container_number) as len
from biz_containers 
where length(container_number) > 45;
```

## 📝 导入前检查清单

- [ ] Excel格式为.xlsx或.csv
- [ ] 列名与飞驼导出完全一致
- [ ] 集装箱号列无空值
- [ ] 日期格式为 YYYY/MM/DD HH:mm
- [ ] 数组字段使用 ["a","b"] 格式
- [ ] 数值字段无千分位逗号
- [ ] 文本字段无特殊字符
- [ ] 文件大小 < 10MB（超过请分批）
- [ ] 网络连接稳定
- [ ] 已备份相关数据
