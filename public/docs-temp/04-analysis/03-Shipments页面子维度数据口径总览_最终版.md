## 一、按到港维度

### 子集范围

**所有已经出运的货柜**，包括：已出运、在途、已到目的港、已提柜、已卸柜、已还箱状态。

### 分类结构

```
按到港总计 (全部已出运货柜)
├── ① 实际已到目的港 (有ATA)
│   ├── 今日到港
│   └── 今日之前到港
│       ├── 今日之前到港未提柜
│       └── 今日之前到港已提柜
└── ② 未到目的港 (无ATA)
    ├── 已逾期未到港
    ├── 3日内预计到港
    ├── 7日内预计到港
    ├── 7日后预计到港
    └── 其他记录 (无ETA)
```

### 汇总关系

```
按到港总计 = 实际已到目的港 + 未到目的港

实际已到目的港 = 今日到港 + 今日之前到港未提柜 + 今日之前到港已提柜

未到目的港 = 已逾期未到港 + 3日内预计到港 + 7日内预计到港 + 7日后预计到港 + 其他记录
```

### 分类说明

#### ① 实际已到目的港 (有ATA)

**筛选条件**: 货柜有目的港的实际到港时间(ATA)，且港口类型不是中转港

**使用时间**: ATA (实际到港时间)

**包含状态**: shipped/in_transit/at_port/picked_up/unloaded/returned_empty

| 子分类                 | 条件                                                   | 说明               |
| ---------------------- | ------------------------------------------------------ | ------------------ |
| **今日到港**           | ATA = 今天                                             | 今天刚到港         |
| **今日之前到港未提柜** | ATA < 今天 且 状态 ≠ picked_up/unloaded/returned_empty | 之前到港但还没提柜 |
| **今日之前到港已提柜** | ATA < 今天 且 状态 = picked_up/unloaded/returned_empty | 之前到港且已提柜   |

#### ② 未到目的港 (无ATA)

**筛选条件**: 货柜没有目的港的实际到港时间(ATA)，只有预计到港时间(ETA)或两者都没有

**使用时间**: ETA (预计到港时间)

**包含状态**: **仅包含** shipped/in_transit/at_port (已出运但未到港之后状态)

**为什么要限制状态?**

- `picked_up` 和 `unloaded` 状态的货柜已经提柜，不需要用ETA预测到港时间
- 这些货柜应该归类到"今日之前到港已提柜"分类中

| 子分类            | 条件                       | 说明                         |
| ----------------- | -------------------------- | ---------------------------- |
| **已逾期未到港**  | ETA < 今天                 | 预计到港时间已过期，还未到港 |
| **3日内预计到港** | ETA ∈ [今天, 今天+3天]     | 预计3天内到港(含今天)        |
| **7日内预计到港** | ETA ∈ (今天+3天, 今天+7天] | 预计3-7天内到港              |
| **7日后预计到港** | ETA > 今天+7天             | 预计7天后到港                |
| **其他记录**      | ETA 为空                   | 没有预计到港时间             |

### 详细条件说明

#### 1. 今日到港

| 条件                   | 要求  | 说明               |
| ---------------------- | ----- | ------------------ |
| 有ATA                  | ✅ 是 | 货柜有实际到港日期 |
| ATA = 今天             | ✅ 是 | ATA日期等于今天    |
| 港口类型 = destination | ✅ 是 | 是目的港(非中转港) |

#### 2. 今日之前到港未提柜

| 条件                   | 要求                                | 说明                       |
| ---------------------- | ----------------------------------- | -------------------------- |
| 有ATA                  | ✅ 是                               | 货柜有实际到港日期         |
| ATA < 今天             | ✅ 是                               | ATA日期早于今天            |
| 港口类型 = destination | ✅ 是                               | 是目的港(非中转港)         |
| 物流状态               | ≠ picked_up/unloaded/returned_empty | 未提柜、未卸柜、未还箱状态 |

#### 3. 今日之前到港已提柜

| 条件                   | 要求                                | 说明                       |
| ---------------------- | ----------------------------------- | -------------------------- |
| 有ATA                  | ✅ 是                               | 货柜有实际到港日期         |
| ATA < 今天             | ✅ 是                               | ATA日期早于今天            |
| 港口类型 = destination | ✅ 是                               | 是目的港(非中转港)         |
| 物流状态               | = picked_up/unloaded/returned_empty | 已提柜、已卸柜或已还箱状态 |

#### 4. 已逾期未到港

| 条件       | 要求                       | 说明                   |
| ---------- | -------------------------- | ---------------------- |
| 物流状态   | shipped/in_transit/at_port | 已出运但未到港之后状态 |
| 有ETA      | ✅ 是                      | 有预计到港日期         |
| 无ATA      | ✅ 是                      | 还没有实际到港         |
| ETA < 今天 | ✅ 是                      | ETA已过期              |

#### 5. 3日内预计到港

| 条件                   | 要求                       | 说明                   |
| ---------------------- | -------------------------- | ---------------------- |
| 物流状态               | shipped/in_transit/at_port | 已出运但未到港之后状态 |
| 有ETA                  | ✅ 是                      | 有预计到港日期         |
| 无ATA                  | ✅ 是                      | 还没有实际到港         |
| ETA ∈ [今天, 今天+3天] | ✅ 是                      | 3天内到港(含今天)      |

#### 6. 7日内预计到港

| 条件                       | 要求                       | 说明                   |
| -------------------------- | -------------------------- | ---------------------- |
| 物流状态                   | shipped/in_transit/at_port | 已出运但未到港之后状态 |
| 有ETA                      | ✅ 是                      | 有预计到港日期         |
| 无ATA                      | ✅ 是                      | 还没有实际到港         |
| ETA ∈ (今天+3天, 今天+7天] | ✅ 是                      | 3-7天内到港            |

#### 7. 7日后预计到港

| 条件           | 要求                       | 说明                   |
| -------------- | -------------------------- | ---------------------- |
| 物流状态       | shipped/in_transit/at_port | 已出运但未到港之后状态 |
| 有ETA          | ✅ 是                      | 有预计到港日期         |
| 无ATA          | ✅ 是                      | 还没有实际到港         |
| ETA > 今天+7天 | ✅ 是                      | 7天后到港              |

#### 8. 其他记录

| 条件     | 要求                       | 说明                   |
| -------- | -------------------------- | ---------------------- |
| 物流状态 | shipped/in_transit/at_port | 已出运但未到港之后状态 |
| 无ATA    | ✅ 是                      | 还没有实际到港         |
| 无ETA    | ✅ 是                      | 没有预计到港时间       |

### 举例说明

| 货柜状态           | ATA             | ETA        | 属于哪类分类       | 理由                   |
| ------------------ | --------------- | ---------- | ------------------ | ---------------------- |
| 在途 (in_transit)  | ❌              | 2025-02-01 | 3日内预计到港      | 未实际到港，ETA在3天内 |
| 已到港 (at_port)   | ✅ (2025-01-01) | 2025-01-01 | 今日之前到港未提柜 | 已实际到港，但未提柜   |
| 已提柜 (picked_up) | ✅ (2025-01-01) | 2025-01-01 | 今日之前到港已提柜 | 已实际到港，已提柜     |
| 已出运 (shipped)   | ❌              | 2024-12-01 | 已逾期未到港       | ETA已过期，还未到港    |
| 在途 (in_transit)  | ❌              | ❌         | 其他记录           | 无ETA，无法预测        |

### 关键点

1. **互斥性**: 每个货柜只被归类到其中一个子分类，不会重复计数
2. **ATA vs ETA**:
   - 有ATA的货柜使用ATA分类(实际已到港)
   - 无ATA的货柜使用ETA分类(未到港，预测)
3. **状态筛选**:
   - ATA分组包含所有已出运状态(shipped/in_transit/at_port/picked_up/unloaded/returned_empty)
   - ETA分组只包含已出运但未到港之后状态(shipped/in_transit/at_port)
4. **港口类型**: 只统计目的港(port_type = 'destination')，排除中转港

## 二、按提柜计划维度

### 子集范围

**仅针对按到港中的"实际已到目的港"但还没有实际提柜的货柜**。

### 分类结构

```
按提柜计划总计 (实际已到目的港但未提柜)
├── ① 有安排拖车计划
│   ├── 逾期未提柜
│   ├── 今日计划提柜
│   ├── 3天内计划提柜
│   └── 7天内计划提柜
└── ② 没有安排拖车计划
    └── 待安排提柜
```

### 汇总关系

```
按提柜计划总计 = 有安排拖车计划 + 没有安排拖车计划

有安排拖车计划 = 逾期未提柜 + 今日计划提柜 + 3天内计划提柜 + 7天内计划提柜

没有安排拖车计划 = 待安排提柜
```

### 基础筛选条件

| 条件       | 要求                                      | 说明                          |
| ---------- | ----------------------------------------- | ----------------------------- |
| 已到目的港 | ✅ 是                                     | 有ATA且港口类型 = destination |
| 物流状态   | ≠ picked_up, ≠ unloaded, ≠ returned_empty | 未提柜、未卸柜、未还箱状态    |
| 无实际提柜 | ✅ 是                                     | pickupDate为空                |

### 分类说明

#### ① 有安排拖车计划

**筛选条件**: 存在拖卡运输记录(`trucking_transport`表),且未实际提柜

| 子分类            | 条件                                 | 说明                    |
| ----------------- | ------------------------------------ | ----------------------- |
| **逾期未提柜**    | plannedPickupDate < 今天             | 计划提柜日期已过期      |
| **今日计划提柜**  | plannedPickupDate = 今天             | 今天计划提柜            |
| **3天内计划提柜** | plannedPickupDate ∈ [今天, 今天+3]   | 3天内计划提柜（含今天） |
| **7天内计划提柜** | plannedPickupDate ∈ (今天+3, 今天+7] | 3-7天内计划提柜         |

#### ② 没有安排拖车计划

**筛选条件**: 不存在拖卡运输记录(`trucking_transport`表),且未实际提柜

| 子分类         | 条件           | 说明               |
| -------------- | -------------- | ------------------ |
| **待安排提柜** | 无拖卡运输记录 | 还没有安排拖车计划 |

### 详细条件说明

#### 1. 逾期未提柜

| 条件                     | 要求  | 说明                       |
| ------------------------ | ----- | -------------------------- |
| 基础条件                 | ✅ 是 | 已到目的港且未提柜         |
| 有拖卡运输记录           | ✅ 是 | 存在trucking_transport记录 |
| 有计划提柜日期           | ✅ 是 | plannedPickupDate不为空    |
| 无实际提柜日期           | ✅ 是 | pickupDate为空             |
| plannedPickupDate < 今天 | ✅ 是 | 计划日期已过期             |

#### 2. 今日计划提柜

| 条件                     | 要求  | 说明                       |
| ------------------------ | ----- | -------------------------- |
| 基础条件                 | ✅ 是 | 已到目的港且未提柜         |
| 有拖卡运输记录           | ✅ 是 | 存在trucking_transport记录 |
| 有计划提柜日期           | ✅ 是 | plannedPickupDate不为空    |
| 无实际提柜日期           | ✅ 是 | pickupDate为空             |
| plannedPickupDate = 今天 | ✅ 是 | 今天计划提柜               |

#### 3. 3天内计划提柜

| 条件                               | 要求  | 说明                       |
| ---------------------------------- | ----- | -------------------------- |
| 基础条件                           | ✅ 是 | 已到目的港且未提柜         |
| 有拖卡运输记录                     | ✅ 是 | 存在trucking_transport记录 |
| 有计划提柜日期                     | ✅ 是 | plannedPickupDate不为空    |
| 无实际提柜日期                     | ✅ 是 | pickupDate为空             |
| plannedPickupDate ∈ [今天, 今天+3] | ✅ 是 | 3天内计划提柜              |

#### 4. 7天内计划提柜

| 条件                                 | 要求  | 说明                       |
| ------------------------------------ | ----- | -------------------------- |
| 基础条件                             | ✅ 是 | 已到目的港且未提柜         |
| 有拖卡运输记录                       | ✅ 是 | 存在trucking_transport记录 |
| 有计划提柜日期                       | ✅ 是 | plannedPickupDate不为空    |
| 无实际提柜日期                       | ✅ 是 | pickupDate为空             |
| plannedPickupDate ∈ (今天+3, 今天+7] | ✅ 是 | 3-7天内计划提柜            |

#### 5. 待安排提柜

| 条件           | 要求  | 说明                         |
| -------------- | ----- | ---------------------------- |
| 基础条件       | ✅ 是 | 已到目的港且未提柜           |
| 无拖卡运输记录 | ✅ 是 | 不存在trucking_transport记录 |
| 无实际提柜     | ✅ 是 | pickupDate为空               |

### 关键点

1. **与按到港维度的关系**:
   - 按提柜计划维度的子集 = 按到港维度中的"实际已到目的港且未提柜"货柜
   - 即: `今日到港 + 今日之前到港未提柜`

2. **互斥性**: 每个货柜只被归类到其中一个子分类,不会重复计数

3. **拖卡记录的重要性**:
   - 有拖卡记录 = 有安排拖车计划
   - 无拖卡记录 = 没有安排拖车计划(待安排提柜)

4. **实际提柜日期**:
   - 所有分类都要求无实际提柜日期(pickupDate为空)
   - 已实际提柜的货柜不在按提柜计划维度中

### 举例说明

| 货柜情况                 | ATA | plannedPickupDate | pickupDate | 拖卡记录 | 属于哪类分类   | 理由             |
| ------------------------ | --- | ----------------- | ---------- | -------- | -------------- | ---------------- |
| 到港5天,已安排拖车但逾期 | ✅  | 3天前             | ❌         | ✅       | 逾期未提柜     | 计划日期已过期   |
| 到港1天,计划今天提柜     | ✅  | 今天              | ❌         | ✅       | 今日计划提柜   | 今天计划提柜     |
| 到港2天,计划3天后提柜    | ✅  | 3天后             | ❌         | ✅       | 3天内计划提柜  | 3天内计划提柜    |
| 到港4天,计划5天后提柜    | ✅  | 5天后             | ❌         | ✅       | 7天内计划提柜  | 3-7天内计划提柜  |
| 到港1天,未安排拖车       | ✅  | ❌                | ❌         | ❌       | 待安排提柜     | 没有安排拖车计划 |
| 到港且已提柜             | ✅  | 3天前             | ✅         | ✅       | 不在统计范围内 | 已实际提柜       |
| 未到港,在途中            | ❌  | 5天后             | ❌         | ✅       | 不在统计范围内 | 未到目的港       |

## 四、按最晚还箱维度

按最晚还箱 (有实际提柜或有拖卡记录，且未还箱)

### 5.2 时间计算辅助函数

```typescript
// useGanttHelpers.ts
export const getRemainingTime = (targetDate: Date) => {
  const today = new Date();
  today.setHours(0, 0, 0, 0);
  const target = new Date(targetDate);
  target.setHours(0, 0, 0, 0);

  const diffTime = target.getTime() - today.getTime();
  const diffDays = Math.ceil(diffTime / (1000 * 60 * 60 * 24));

  return {
    days: diffDays >= 0 ? diffDays : 0,
    isExpired: diffDays < 0,
  };
};
```
