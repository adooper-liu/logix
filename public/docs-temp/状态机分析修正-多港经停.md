## ✅ 修正结论

### 问题1：46个`at_port`状态缺少到港记录

**原因**：这些货柜正在**中转港（transit）**，还未到达目的港（destination）

**验证结果**：
```
container_number | port_type   | port_sequence | ata_dest_port
----------------+-------------+---------------+---------------
CAAU9304575     | origin      | 1             |
CAAU9304575     | transit     | 2             |  ← 当前位置
CAAU9304575     | destination | 3             |
```

**状态判断逻辑（修正版）**：
```sql
-- 正确的到港判断
CASE
  -- 已到中转港：有transit记录，且destination无ata
  WHEN EXISTS (
    SELECT 1 FROM process_port_operations po
    WHERE po.container_number = c.container_number
      AND po.port_type = 'transit'
  )
  AND NOT EXISTS (
    SELECT 1 FROM process_port_operations po
    WHERE po.container_number = c.container_number
      AND po.port_type = 'destination'
      AND po.ata_dest_port IS NOT NULL
  )
  THEN '已到中转港'

  -- 已到目的港：destination有ata
  WHEN EXISTS (
    SELECT 1 FROM process_port_operations po
    WHERE po.container_number = c.container_number
      AND po.port_type = 'destination'
      AND po.ata_dest_port IS NOT NULL
  )
  THEN '已到目的港'

  ELSE '未到港'
END
```

## 📊 修正后的状态机定义

考虑到多港经停场景，状态机应该是：

```
未出运 (not_shipped)
  ↓ 有出运记录
在途 (in_transit)
  ↓ 已到中转港 OR 已到目的港
已到港 (at_port)
  - 已到中转港（等待到目的港）
  - 已到目的港（等待提柜）
  ↓ 有提柜记录
已提柜 (picked_up)
  ↓ 有还箱记录
已还箱 (returned_empty)
```

**注意**：
- `at_port`状态包含两个子状态：
  - **已到中转港**：有transit记录，destination无ata
  - **已到目的港**：destination有ata
- 前端需要区分这两个子状态进行展示

## 🎯 状态判断逻辑（完整版）

```typescript
function determineStatus(containerNumber: string): string {
  if (hasReturnTime(containerNumber)) return 'returned_empty';
  if (hasPickupDate(containerNumber)) return 'picked_up';

  // 判断是否到港（包括中转港和目的港）
  if (hasArrivedAtTransit(containerNumber) || hasArrivedAtDestination(containerNumber)) {
    return 'at_port';
  }

  if (hasShipmentDate(containerNumber)) return 'in_transit';
  return 'not_shipped';
}

function hasArrivedAtTransit(containerNumber: string): boolean {
  // 有transit记录
  const hasTransit = await this.portOperationRepository.exists({
    where: { containerNumber, portType: 'transit' }
  });

  if (!hasTransit) return false;

  // destination还没有ata（说明还在中转）
  const destinationHasAta = await this.portOperationRepository.exists({
    where: {
      containerNumber,
      portType: 'destination',
      ataDestPort: Not(IsNull())
    }
  });

  return !destinationHasAta;
}

function hasArrivedAtDestination(containerNumber: string): boolean {
  // destination有ata
  return await this.portOperationRepository.exists({
    where: {
      containerNumber,
      portType: 'destination',
      ataDestPort: Not(IsNull())
    }
  });
}
```

## 📋 "按最晚还箱日"统计的真正逻辑

根据多港经停机制，"按最晚还箱日"应该统计：

### 目标集定义

```sql
-- 目标集：已提柜 或 已到目的港
SELECT c.container_number
FROM biz_containers c
WHERE
  -- 条件1：已提柜（有拖卡记录）
  EXISTS (
    SELECT 1 FROM process_trucking_transport tt
    WHERE tt.container_number = c.container_number
    AND tt.pickup_date IS NOT NULL
  )

  OR

  -- 条件2：已到目的港（destination有ata，且有lastReturnDate）
  EXISTS (
    SELECT 1 FROM process_port_operations po
    LEFT JOIN process_empty_returns er ON po.container_number = er."containerNumber"
    WHERE po.container_number = c.container_number
      AND po.port_type = 'destination'
      AND po.ata_dest_port IS NOT NULL
      AND er."lastReturnDate" IS NOT NULL
  )

  -- 排除：已还箱
  AND c.logistics_status != 'returned_empty'
```

```sql
-- 简化版本：按状态统计，不需要额外约束
SELECT logistics_status, COUNT(*)
FROM biz_containers

-- at_port: 92（包括已到中转港和已到目的港）
-- in_transit: 85
-- picked_up: 54
-- returned_empty: 119
```

**前端应该基于状态来展示，而不是基于复杂的约束条件！**

## 🔑 关键要点

1. **多港经停机制**：
   - `port_type = 'transit'`：中转港
   - `port_type = 'destination'`：目的港
   - 只有`destination.ata_dest_port`才是真正的到港

2. **at_port状态的含义**：
   - 包括"已到中转港"和"已到目的港"
   - 前端需要区分这两个子状态

3. **统计应该基于状态**：
   - 不需要在查询中添加复杂的约束条件
   - 状态维护逻辑确保数据的准确性
   - 如果状态不对，应该修复状态，而不是在统计中绕过

## ❓ 仍需确认的问题

1. **lastFreeDate vs lastReturnDate**：
   - `lastFreeDate`：港口的最后免费日（process_port_operations表）
   - `lastReturnDate`：货代的最后还箱日（process_empty_returns表）
   - 这两个字段可能不同，需要明确业务含义

2. **"按最晚还箱日"到底要统计什么**：
   - 如果是统计"未还箱的货柜"，应该是`picked_up`状态
   - 如果是统计"需要还箱的货柜"，可能也包括`at_port`状态
   - 需要明确业务需求

3. **unloaded状态**：
   - 是否需要这个状态？
   - 如果需要，为什么当前数据中完全没有使用？

4. **46个at_port（在中转港）的货柜**：
   - 这些货柜的lastReturnDate是从哪里来的？
   - 在中转港就应该有最后还箱日吗？
