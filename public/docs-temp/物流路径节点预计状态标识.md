# 物流路径节点预计状态标识功能

## 问题描述

在货柜详情页的物流路径视图中，所有节点都没有标识是否为"预计"状态，导致用户误解。例如：
- 已还空箱节点显示具体时间，但实际上是预计时间（is_estimated=1）
- 用户会误以为已经实际发生

## 解决方案

### 后端修改（logistics-path-system 微服务）

**文件**: `logistics-path-system/backend/src/services/statusPathFromDb.ts`

1. **扩展 DbEvent 接口**：添加 `is_estimated` 字段
   ```typescript
   interface DbEvent {
     // ... 其他字段
     is_estimated: boolean | null;
   }
   ```

2. **修改 SQL 查询**：从 `ext_container_status_events` 表查询 `is_estimated` 字段
   ```sql
   SELECT id, container_number, status_code, status_name, occurred_at, location, description, data_source, raw_data, is_estimated
   FROM ext_container_status_events
   WHERE container_number = $1
   ORDER BY occurred_at ASC NULLS LAST, id ASC
   LIMIT 100
   ```

3. **在 eventToNode 函数中传递 isEstimated**：
   ```typescript
   rawData: {
     eventCode: e.status_code,
     eventId: e.id,
     dataSource: e.data_source,
     isEstimated: e.is_estimated ?? false,  // 新增
     ...(e.raw_data || {}),
   }
   ```

### 前端修改

**文件**: `frontend/src/views/shipments/components/LogisticsPathTab.vue`

1. **添加判断函数**：
   ```typescript
   /** 判断是否为预计节点（isEstimated=true） */
   const isEstimatedNode = (node: StatusNode): boolean => {
     return !!(node.rawData as { isEstimated?: boolean })?.isEstimated
   }
   ```

2. **在节点卡片上显示"预计"标签**：
   ```vue
   <div class="stage-node-desc-row">
     <span class="stage-node-desc">{{ item.node.description }}</span>
     <span v-if="item.node.isAlert" class="alert-badge">异常</span>
     <el-tag
       v-if="isEstimatedNode(item.node)"
       size="small"
       type="warning"
       class="estimated-tag"
     >
       预计
     </el-tag>
   </div>
   ```

3. **在节点详情面板中显示"预计"标签**：
   ```vue
   <el-tag
     v-if="isEstimatedNode(selectedNode)"
     type="warning"
     size="small"
     class="detail-estimated-tag"
   >
     预计
   </el-tag>
   ```

4. **添加样式**：
   - 预计节点的边框和背景色（黄色边框，淡黄色背景）
   - 标签间距调整

## 数据来源

`ext_container_status_events` 表的 `is_estimated` 字段来自：
- 飞驼 API 同步：`s.isEsti === 'Y'`（见 `backend/src/services/externalDataService.ts` 第 238 行）
- 飞驼 Excel 导入：映射字段"是否预计"（见 `frontend/src/configs/importMappings/feituo.ts` 第 185 行）

## 效果展示

### 修改前
- 所有节点看起来都一样，无法区分实际发生和预计
- 用户会误以为预计时间是实际发生时间

### 修改后
- 预计节点显示黄色"预计"标签
- 节点卡片有淡黄色左边框和背景色
- 节点详情面板也显示"预计"标签
- 用户可以清楚识别哪些是预计数据

## 测试验证

1. 打开货柜详情页（如 `HMMU6232153`）
2. 切换到"物流路径"标签
3. 查看"还箱"阶段的"已还空箱"节点
4. 应该显示黄色"预计"标签（因为数据库中 `is_estimated=1`）
5. 点击节点打开详情面板，也应该显示"预计"标签

## 注意事项

- 预计事件不代表实际发生，仅用于预测和计划
- 实际事件会覆盖预计事件（当飞驼 API 返回实际时间时）
- 核心业务字段（如 `return_time`、`shipment_date`）只根据实际事件更新
