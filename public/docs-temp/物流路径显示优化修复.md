### 1. DELIVERY_ARRIVED (FETA) 阶段修正 ✅

**问题**：FETA（交货地抵达）被错误分配到"提柜"阶段

**修复**：
- 后端：移动到"抵港"阶段（`STANDARD_MODE_TEMPLATE` order 7）
- 前端：`STAGE_MAP` 中映射到 `arrival` 阶段（order 6）

**代码位置**：
- `logistics-path-system/backend/src/services/statusPathFromDb.ts` 第 91 行
- `frontend/src/views/shipments/components/LogisticsPathTab.vue` 第 701 行

### 2. STCS (提柜 (货)) 阶段增加 ✅

**问题**：STCS（提柜后运输中）只在"提柜"阶段，但业务上应该也在"可提"阶段显示

**修复**：
- "可提"阶段同时接受 `AVAILABLE` 和 `IN_TRANSIT_TO_DEST`
- 这样 STCS 事件会在"可提"阶段显示，不会报"缺数据"

**代码位置**：
- `logistics-path-system/backend/src/services/statusPathFromDb.ts` 第 101 行
- `logistics-path-system/backend/src/utils/pathValidator.ts` 第 44 行（流转规则）

**业务解释**：
- STCS = Gate Out for Delivery（提柜后运输中）
- 代表已经提柜，说明已经过了"可提"阶段
- 但为了显示完整性，在"可提"阶段也接受该状态

## 状态流转规则更新

### 关键流转

**允许跳过可提货直接提柜**：
```typescript
DISCHARGED -> [
  AVAILABLE,              // 正常流程：先可提货
  IN_TRANSIT_TO_DEST,     // STCS: 跳过可提货直接提柜
  GATE_OUT,               // 直接提柜
  HOLD,                   // 各种滞留
]
```

**铁路/驳船卸箱后提柜**：
```typescript
RAIL_DISCHARGED -> [
  GATE_OUT,               // 铁路卸箱后提柜
  IN_TRANSIT_TO_DEST,     // STCS: 铁路卸箱后直接提柜运输
  HOLD,
]

FEEDER_DISCHARGED -> [
  GATE_OUT,               // 驳船卸船后提柜
  IN_TRANSIT_TO_DEST,     // STCS: 驳船卸船后直接提柜运输
  HOLD,
]
```
