# 前端查看历史排产记录指南

## 📌 功能概述

LogiX 系统提供了专门的**排产历史记录查询页面**，用于查看和管理所有已保存的排产记录。

---

## 🎯 访问路径

### 方式一：通过路由直接访问

访问 URL：`/scheduling/history`

```typescript
// 路由配置位置：frontend/src/router/index.ts
{
  path: 'history',
```

如果系统有侧边栏菜单，可以在以下位置找到：

## 🖥️ 页面功能

### 1. **搜索区域**

提供两个搜索条件：

#### a) 货柜号搜索
- 输入框支持模糊查询
- 支持 `allow-clear` 一键清空
- 示例：`HMMU6232153`

#### b) 时间范围搜索
- 使用 `a-range-picker` 日期选择器
- 可选择开始日期和结束日期
- 按操作时间（`operatedAt`）筛选

#### c) 查询按钮
- 🔍 **查询**：执行搜索
- 🔄 **重置**：清空所有条件

---

### 2. **结果列表**

表格显示以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| **货柜号** | 集装箱编号 | HMMU6232153 |
| **版本** | 排产版本号（带标签） | v1, v2 |
| **策略** | 排产策略（带标签） | 直提、甩挂、加急 |
| **总费用** | 预估总费用（高亮显示） | $1,250.00 |
| **状态** | 排产状态（徽章显示） | 生效中、已作废、已取消 |
| **操作人** | 执行操作的用户 | admin, SYSTEM |
| **操作时间** | 操作发生的时间 | 2026-04-02 14:30:00 |
| **操作** | 查看详情按钮 | - |

---

### 3. **详情抽屉**

点击"查看详情"按钮后，从右侧滑出抽屉，显示完整信息：

#### 基本信息
- 货柜号
- 版本号
- 策略
- 状态

#### 日期信息
- 提柜日期（`plannedPickupDate`）
- 送仓日期（`plannedDeliveryDate`）
- 卸柜日期（`plannedUnloadDate`）
- 还箱日期（`plannedReturnDate`）

#### 资源信息
- 仓库（`warehouseName` / `warehouseCode`）
- 车队（`truckingCompanyName` / `truckingCompanyCode`）

#### 费用明细
- 总费用（高亮显示）
- 滞港费（`demurrageCost`）
- 滞箱费（`detentionCost`）
- 堆存费（`storageCost`）
- 运输费（`transportationCost`）

#### 审计信息
- 操作人（`operatedBy`）
- 操作时间（`operatedAt`）
- 操作类型（CREATE / UPDATE / CANCEL）

## ⚠️ 注意事项

### 1. **只显示最新版本的逻辑**

API 端点 `/scheduling/history/latest` 返回的是每个货柜的最新版本记录，而不是全部历史记录。

如需查看某个货柜的完整版本历史，需要：
- 在后端添加新的 API 端点（如 `/scheduling/history/all/:containerNumber`）
- 或修改现有 API 增加参数控制（如 `?includeAllVersions=true`）

### 2. **状态含义**

| 状态 | 含义 | 说明 |
|------|------|------|
| **CONFIRMED** | 生效中 | 当前正在使用的排产方案 |
| **SUPERSEDED** | 已作废 | 被新版本替代的旧版本 |
| **CANCELLED** | 已取消 | 用户手动取消的排产方案 |

### 3. **版本号规则**

- 每次重新排产并确认后，版本号 +1
- 格式：`v1`, `v2`, `v3`...
- 使用 `<a-tag>` 标签显示，便于识别

### 4. **操作类型**

| 类型 | 说明 |
|------|------|
| **CREATE** | 创建新的排产记录 |
| **UPDATE** | 更新现有排产记录 |
| **CANCEL** | 取消排产记录 |
