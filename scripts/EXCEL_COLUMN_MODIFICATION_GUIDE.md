## 📝 操作步骤

### 步骤 1: 打开 Excel 文件

```bash
双击打开您的货柜数据 Excel 文件
```

### 步骤 2: 找到错误的列名

```
定位到列名为 "销往国家" 的列
┌─────────────┐
│ 销往国家    │  ← 需要修改这一列的列名
├─────────────┤
│ AOSOM US LLC│
│ XYZ Trading │
└─────────────┘
```

### 步骤 3: 修改列名

**方法 A: 直接编辑（推荐）**
```
1. 双击列头单元格 "销往国家"
2. 删除原文字
3. 输入 "客户名称"
4. 按 Enter 确认
```

**方法 B: 使用公式栏**
```
1. 选中列头单元格 "销往国家"
2. 在顶部公式栏中修改为 "客户名称"
3. 按 Enter 确认
```

### 步骤 4: 保存文件

```bash
Ctrl + S  # 保存修改
```

### 步骤 5: 重新上传

```
1. 打开通用导入组件页面
2. 上传修改后的 Excel 文件
3. 系统会正确识别：
   - 列名："客户名称" → 映射到 customer_name ✅
   - 值：AOSOM US LLC → 存储为客户名称 ✅
   - 自动推断：根据关键词填充 sell_to_country ✅
```

## 🔍 技术原理

### 为什么修改列名后能正常工作？

```typescript
// 配置 1: 客户名称
{
  excelField: '客户名称',  // ← 精确匹配 Excel 列名
  field: 'customer_name'
}

// 配置 2: 销往国家（带自动填充）
{
  excelField: '销往国家',
  field: 'sell_to_country',
  transform: (value, row) => {
    if (value) return value  // Excel 有值直接用
    
    // Excel 无值时，从 customer_name 推断
    if (row?.customer_name === 'AOSOM US LLC') {
      return '美国'  // 检测到 US 关键词
    }
    
    return null
  }
}
```

## ⚠️ 注意事项

### 1. 确保列名准确

- ✅ 使用 **"客户名称"** (标准名称)
- ❌ 不要用 "销往国家" (会造成混淆)
- ❌ 不要用其他变体（如"客户"、"公司名称"等）

### 2. 检查其他列名

建议同时检查其他列名是否规范：

| 正确的列名 | 常见错误 | 状态 |
|-----------|---------|------|
| 备货单号 | 订单号、PO 号 | ⚠️ 需统一 |
| 主备货单号 | 主单号 | ⚠️ 需统一 |
| 客户名称 | 销往国家（错） | ✅ 已修正 |
| 销往国家 | 进口国 | ⚠️ 可用别名 |
| 箱数合计 | 总箱数 | ⚠️ 需统一 |

### 3. 保存备份

修改前建议保存原始文件的备份：

```bash
原始文件.xlsx  →  原始文件_备份.xlsx
修改后：原始文件.xlsx (列名已修正)
```
